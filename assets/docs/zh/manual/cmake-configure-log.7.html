简介¶

CMake 会记录一份运行日志（称为 configure log），用于记录配置（Configure）步骤中发生的特定事件。configure log 并不包含配置项目时打印的所有输出、错误或消息。它记录的是有关特定事件的详细信息，例如 try_compile() 对工具链的检查，旨在用于调试构建树的配置。

对于人工查看，此版本的 CMake 会将 configure log 写入以下文件：

${CMAKE_BINARY_DIR}/CMakeFiles/CMakeConfigureLog.yaml

代码说明

  • ${CMAKE_BINARY_DIR}/CMakeFiles/CMakeConfigureLog.yaml：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_BINARY_DIR：顶层构建目录的绝对路径。

然而，日志文件的位置和名称在未来的 CMake 版本中可能会发生变化。读取 configure log 的工具应使用 configureLog 查询向 cmake-file-api(7) 获取其位置。详情请参阅下方的 日志版本控制 部分。

日志结构¶

configure log 的设计既便于机器阅读，也便于人工阅读。

日志文件是一个 YAML 文档流，包含零个或多个由文档标记分隔的 YAML 文档。每个文档以 --- 文档标记行开始，包含一个记录了 CMake 一次“配置”步骤事件的单个 YAML 映射；如果配置步骤正常结束，则以 ... 文档标记行结尾。

---
events:
  -
    kind: "try_compile-v1"
    # (other fields omitted)
  -
    kind: "try_compile-v1"
    # (other fields omitted)
...

代码说明

  • ---：这一行给出数据项或文本内容，用于补充当前示例。

  • events:：这一行声明字段 events 及其取值。

  • -：这一行给出数据项或文本内容，用于补充当前示例。

  • kind: "try_compile-v1"：这一行声明字段 kind 及其取值。

  • # (other fields omitted)：这一行给出数据项或文本内容，用于补充当前示例。

  • -：这一行给出数据项或文本内容，用于补充当前示例。

  • kind: "try_compile-v1"：这一行声明字段 kind 及其取值。

  • # (other fields omitted)：这一行给出数据项或文本内容，用于补充当前示例。

  • ...：这一行给出数据项或文本内容，用于补充当前示例。

每当 CMake 配置构建树并记录新事件时，就会向日志中追加一个新的文档。

每个文档根映射的键包括：

events

一个 YAML 块序列节点，对应于 CMake 一次“配置”步骤中记录的事件。每个事件都是一个 YAML 节点，包含下方记录的 事件种类 之一。

日志版本控制¶

每种 事件种类 都是独立进行版本控制的。事件日志条目提供的键集取决于其主要版本。当记录一个事件时，运行中的 CMake 版本所知的该事件种类的最新版本会被写入日志。

读取 configure log 的工具必须忽略其无法理解的事件种类和版本。

  • 未来的 CMake 版本可能会引入新的事件种类或版本。

  • 如果使用不同版本的 CMake 重新配置现有的构建树，日志可能包含同一事件种类的不同版本。

  • 如果 cmake-file-api(7) 查询请求一个或多个 configureLog 对象版本，日志可能包含同一事件的多个条目，每个条目带有该事件种类不同的版本。

IDE 应在运行 CMake 之前，通过写入 cmake-file-api(7) 查询来请求特定的 configureLog 对象版本，然后仅按照 file-api 回复中的描述来读取 configure log。

文本块编码¶

为了使日志便于人工阅读，文本块始终使用 YAML 字面量块标量 (|) 表示。由于字面量块标量不支持转义，反斜杠和不可打印字符会在应用层进行编码：

  • \\ 编码为一个反斜杠。

  • \xXX 使用两个十六进制数字 XX 编码一个字节。

事件种类¶

每种事件种类都由以下形式的 YAML 映射表示：

kind: "<kind>-v<major>"
backtrace:
  - "<file>:<line> (<function>)"
checks:
  - "Checking for something"
#...event-specific keys...

代码说明

  • kind: "<kind>-v<major>"：这一行声明字段 kind 及其取值。

  • backtrace:：这一行声明字段 backtrace 及其取值。

  • - "<file>:<line> (<function>)"：这一行声明字段 - "<file> 及其取值。

  • checks:：这一行声明字段 checks 及其取值。

  • - "Checking for something"：这一行给出数据项或文本内容，用于补充当前示例。

  • #...event-specific keys...：这一行给出数据项或文本内容，用于补充当前示例。

所有事件共有的键包括：

种类

一个标识事件种类和主要版本的字符串。

回溯

一个 YAML 块序列，报告事件发生时 CMake 源代码位置的调用栈，从最近到最远。每个节点是一个字符串，指定一个格式为 <file>:<line> (<function>) 的位置。

checks

一个可选键，当事件发生时存在至少一个挂起的 message(CHECK_START) 时会出现。其值为一个 YAML 块序列，报告挂起的检查栈，从最近到最远。每个节点是一个包含挂起检查消息的字符串。

额外的映射键特定于每种（版本化的）事件种类，如下所述。

事件种类 message¶

message(CONFIGURE_LOG) 命令记录 message 事件。

message 事件只有一个主要版本，即版本 1。

message-v1 事件¶

message-v1 事件是一个 YAML 映射。

kind: "message-v1"
backtrace:
  - "CMakeLists.txt:123 (message)"
checks:
  - "Checking for something"
message: |
  # ...

代码说明

  • kind: "message-v1"：这一行声明字段 kind 及其取值。

  • backtrace:：这一行声明字段 backtrace 及其取值。

  • - "CMakeLists.txt:123 (message)"：这一行声明字段 - "CMakeLists.txt 及其取值。

  • checks:：这一行声明字段 checks 及其取值。

  • - "Checking for something"：这一行给出数据项或文本内容，用于补充当前示例。

  • message: |：这一行声明字段 message 及其取值。

  • # ...：这一行给出数据项或文本内容，用于补充当前示例。

message-v1 映射特有的键包括：

message

一个 YAML 字面量块标量，包含使用我们的 文本块编码 表示的消息文本。

事件种类 try_compile¶

try_compile() 命令记录 try_compile 事件。

try_compile 事件只有一个主要版本，即版本 1。

try_compile-v1 事件¶

try_compile-v1 事件是一个 YAML 映射。

kind: "try_compile-v1"
backtrace:
  - "CMakeLists.txt:123 (try_compile)"
checks:
  - "Checking for something"
description: "Explicit LOG_DESCRIPTION"
directories:
  source: "/path/to/.../TryCompile-01234"
  binary: "/path/to/.../TryCompile-01234"
cmakeVariables:
  SOME_VARIABLE: "Some Value"
buildResult:
  variable: "COMPILE_RESULT"
  cached: true
  stdout: |
    # ...
  exitCode: 0

代码说明

  • kind: "try_compile-v1"：这一行声明字段 kind 及其取值。

  • backtrace:：这一行声明字段 backtrace 及其取值。

  • - "CMakeLists.txt:123 (try_compile)"：这一行声明字段 - "CMakeLists.txt 及其取值。

  • checks:：这一行声明字段 checks 及其取值。

  • - "Checking for something"：这一行给出数据项或文本内容，用于补充当前示例。

  • description: "Explicit LOG_DESCRIPTION"：这一行声明字段 description 及其取值。

  • directories:：这一行声明字段 directories 及其取值。

  • source: "/path/to/.../TryCompile-01234"：这一行声明字段 source 及其取值。

  • binary: "/path/to/.../TryCompile-01234"：这一行声明字段 binary 及其取值。

  • cmakeVariables:：这一行声明字段 cmakeVariables 及其取值。

  • SOME_VARIABLE: "Some Value"：这一行声明字段 SOME_VARIABLE 及其取值。

  • buildResult:：这一行声明字段 buildResult 及其取值。

  • variable: "COMPILE_RESULT"：这一行声明字段 variable 及其取值。

  • cached: true：这一行声明字段 cached 及其取值。

  • stdout: |：这一行声明字段 stdout 及其取值。

  • # ...：这一行给出数据项或文本内容，用于补充当前示例。

  • exitCode: 0：这一行声明字段 exitCode 及其取值。

try_compile-v1 映射特有的键包括：

描述

一个可选键，在使用 LOG_DESCRIPTION <text> 选项时出现。其值为包含描述 <text> 的字符串。

directories

描述与编译尝试关联的目录的映射。它包含以下键：

源

指定 try_compile() 项目源目录的字符串。

binary

指定 try_compile() 项目二进制目录的字符串。对于非项目调用，这通常与源目录相同。

cmakeVariables

一个可选键，在 CMake 自动或由于 CMAKE_TRY_COMPILE_PLATFORM_VARIABLES 变量而将变量传播到测试项目时出现。其值为从变量名到其值的映射。

buildResult

描述编译测试代码结果的映射。它包含以下键：

variable

指定存储尝试构建测试项目结果的 CMake 变量名的字符串。

cached

一个布尔值，指示上述结果 variable 是否存储在 CMake 缓存中。

stdout

一个 YAML 字面量块标量，包含构建测试项目的输出，使用我们的 文本块编码 表示。这包含来自标准输出 (stdout) 和标准错误 (stderr) 的构建输出。

exitCode

指定尝试构建测试项目时构建工具退出代码的整数。

事件种类 try_run¶

try_run() 命令记录 try_run 事件。

try_run 事件只有一个主要版本，即版本 1。

try_run-v1 事件¶

try_run-v1 事件是一个 YAML 映射。

kind: "try_run-v1"
backtrace:
  - "CMakeLists.txt:456 (try_run)"
checks:
  - "Checking for something"
description: "Explicit LOG_DESCRIPTION"
directories:
  source: "/path/to/.../TryCompile-56789"
  binary: "/path/to/.../TryCompile-56789"
buildResult:
  variable: "COMPILE_RESULT"
  cached: true
  stdout: |
    # ...
  exitCode: 0
runResult:
  variable: "RUN_RESULT"
  cached: true
  stdout: |
    # ...
  stderr: |
    # ...
  exitCode: 0

代码说明

  • kind: "try_run-v1"：这一行声明字段 kind 及其取值。

  • backtrace:：这一行声明字段 backtrace 及其取值。

  • - "CMakeLists.txt:456 (try_run)"：这一行声明字段 - "CMakeLists.txt 及其取值。

  • checks:：这一行声明字段 checks 及其取值。

  • - "Checking for something"：这一行给出数据项或文本内容，用于补充当前示例。

  • description: "Explicit LOG_DESCRIPTION"：这一行声明字段 description 及其取值。

  • directories:：这一行声明字段 directories 及其取值。

  • source: "/path/to/.../TryCompile-56789"：这一行声明字段 source 及其取值。

  • binary: "/path/to/.../TryCompile-56789"：这一行声明字段 binary 及其取值。

  • buildResult:：这一行声明字段 buildResult 及其取值。

  • variable: "COMPILE_RESULT"：这一行声明字段 variable 及其取值。

  • cached: true：这一行声明字段 cached 及其取值。

  • stdout: |：这一行声明字段 stdout 及其取值。

  • # ...：这一行给出数据项或文本内容，用于补充当前示例。

  • exitCode: 0：这一行声明字段 exitCode 及其取值。

  • runResult:：这一行声明字段 runResult 及其取值。

  • variable: "RUN_RESULT"：这一行声明字段 variable 及其取值。

  • cached: true：这一行声明字段 cached 及其取值。

  • stdout: |：这一行声明字段 stdout 及其取值。

  • # ...：这一行给出数据项或文本内容，用于补充当前示例。

  • stderr: |：这一行声明字段 stderr 及其取值。

  • # ...：这一行给出数据项或文本内容，用于补充当前示例。

  • exitCode: 0：这一行声明字段 exitCode 及其取值。

try_run-v1 映射特有的键包括 try_compile-v1 事件 记录的键，以及：

runResult

描述运行测试代码结果的映射。它包含以下键：

variable

指定存储尝试运行测试可执行文件结果的 CMake 变量名的字符串。

cached

一个布尔值，指示上述结果 variable 是否存储在 CMake 缓存中。

stdout

一个可选键，在测试项目构建成功时出现。其值为一个 YAML 字面量块标量，包含运行测试可执行文件的输出，使用我们的 文本块编码 表示。

如果使用了 RUN_OUTPUT_VARIABLE，stdout 和 stderr 会被合并捕获，因此这将包含两者。否则，这仅包含 stdout 输出。

stderr

一个可选键，在测试项目构建成功且未使用 RUN_OUTPUT_VARIABLE 选项时出现。其值为一个 YAML 字面量块标量，包含运行测试可执行文件的输出，使用我们的 文本块编码 表示。

如果使用了 RUN_OUTPUT_VARIABLE，stdout 和 stderr 会被合并捕获在 stdout 键中，此键将不会出现。否则，这将包含 stderr 输出。

exitCode

一个可选键，在测试项目构建成功时出现。其值为指定退出代码的整数，或者是包含尝试运行测试可执行文件时产生的错误消息的字符串。

事件种类 find¶

find_file()、find_path()、find_library() 和 find_program() 命令记录 find 事件。

find 事件只有一个主要版本，即版本 1。

find-v1 事件¶

在 4.1 版本中新增。

find-v1 事件是一个 YAML 映射。

kind: "find-v1"
backtrace:
  - "CMakeLists.txt:456 (find_program)"
mode: "program"
variable: "PROGRAM_PATH"
description: "Docstring for variable"
settings:
  SearchFramework: "NEVER"
  SearchAppBundle: "NEVER"
  CMAKE_FIND_USE_CMAKE_PATH: true
  CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH: true
  CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH: true
  CMAKE_FIND_USE_CMAKE_SYSTEM_PATH: true
  CMAKE_FIND_USE_INSTALL_PREFIX: true
names:
  - "name1"
  - "name2"
candidate_directories:
  - "/path/to/search"
  - "/other/path/to/search"
  - "/path/to/found"
  - "/further/path/to/search"
searched_directories:
  - "/path/to/search"
  - "/other/path/to/search"
found: "/path/to/found/program"

代码说明

  • kind: "find-v1"：这一行声明字段 kind 及其取值。

  • backtrace:：这一行声明字段 backtrace 及其取值。

  • - "CMakeLists.txt:456 (find_program)"：这一行声明字段 - "CMakeLists.txt 及其取值。

  • mode: "program"：这一行声明字段 mode 及其取值。

  • variable: "PROGRAM_PATH"：这一行声明字段 variable 及其取值。

  • description: "Docstring for variable"：这一行声明字段 description 及其取值。

  • settings:：这一行声明字段 settings 及其取值。

  • SearchFramework: "NEVER"：这一行声明字段 SearchFramework 及其取值。

  • SearchAppBundle: "NEVER"：这一行声明字段 SearchAppBundle 及其取值。

  • CMAKE_FIND_USE_CMAKE_PATH: true：这一行声明字段 CMAKE_FIND_USE_CMAKE_PATH 及其取值。

  • CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH: true：这一行声明字段 CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH 及其取值。

  • CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH: true：这一行声明字段 CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH 及其取值。

  • CMAKE_FIND_USE_CMAKE_SYSTEM_PATH: true：这一行声明字段 CMAKE_FIND_USE_CMAKE_SYSTEM_PATH 及其取值。

  • CMAKE_FIND_USE_INSTALL_PREFIX: true：这一行声明字段 CMAKE_FIND_USE_INSTALL_PREFIX 及其取值。

  • names:：这一行声明字段 names 及其取值。

  • - "name1"：这一行给出数据项或文本内容，用于补充当前示例。

  • - "name2"：这一行给出数据项或文本内容，用于补充当前示例。

  • candidate_directories:：这一行声明字段 candidate_directories 及其取值。

  • - "/path/to/search"：这一行给出数据项或文本内容，用于补充当前示例。

  • - "/other/path/to/search"：这一行给出数据项或文本内容，用于补充当前示例。

  • - "/path/to/found"：这一行给出数据项或文本内容，用于补充当前示例。

  • - "/further/path/to/search"：这一行给出数据项或文本内容，用于补充当前示例。

  • searched_directories:：这一行声明字段 searched_directories 及其取值。

  • - "/path/to/search"：这一行给出数据项或文本内容，用于补充当前示例。

  • - "/other/path/to/search"：这一行给出数据项或文本内容，用于补充当前示例。

  • found: "/path/to/found/program"：这一行声明字段 found 及其取值。

find-v1 映射特有的键包括：

mode

描述执行搜索的命令的字符串。为 file、path、program 或 library 之一。

variable

搜索存储结果的变量。

描述

变量的文档字符串。

settings

搜索活动状态下的搜索设置。

SearchFramework

描述如何执行框架搜索的字符串。为 FIRST、LAST、ONLY 或 NEVER 之一。请参阅 CMAKE_FIND_FRAMEWORK。

SearchAppBundle

描述如何执行应用程序包搜索的字符串。为 FIRST、LAST、ONLY 或 NEVER 之一。请参阅 CMAKE_FIND_APPBUNDLE。

CMAKE_FIND_USE_CMAKE_PATH

指示搜索时是否使用 CMake 特定缓存变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_PATH。

CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH

指示搜索时是否使用 CMake 特定环境变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH。

CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH

指示搜索时是否使用平台特定环境变量的布尔值。请参阅 CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH。

CMAKE_FIND_USE_CMAKE_SYSTEM_PATH

指示搜索时是否使用平台特定 CMake 变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_SYSTEM_PATH。

CMAKE_FIND_USE_INSTALL_PREFIX

指示搜索时是否使用安装前缀的布尔值。请参阅 CMAKE_FIND_USE_INSTALL_PREFIX。

names

查询查找的名称。

candidate_directories

搜索期间按顺序查看的候选目录。

searched_directories

搜索过程中按顺序查看的目录。

found

表示找到的值的字符串，如果未找到则为 false。

search_context

变量名到其指定的搜索路径的映射（取决于变量，可能为字符串或字符串数组）。环境变量被包裹在 ENV{ 和 } 之间，否则使用 CMake 变量。仅使用指定了路径的变量。

package_stack

对象数组，其中包含由 find_package() 调用提供的路径栈。

package_paths

调用栈中由 find_package() 命令提供的路径。

事件种类 find_package¶

在 4.1 版本中新增。

find_package() 命令记录 find_package 事件。

find_package 事件只有一个主要版本，即版本 1。

find_package-v1 事件¶

find_package-v1 事件是一个 YAML 映射。

kind: "find_package-v1"
backtrace:
  - "CMakeLists.txt:456 (find_program)"
name: "PackageName"
components:
  -
    name: "Component"
    required: true
    found: true
configs:
  -
    filename: PackageNameConfig.cmake
    kind: "cmake"
  -
    filename: packagename-config.cmake
    kind: "cmake"
version_request:
  version: "1.0"
  version_complete: "1.0...1.5"
  min: "INCLUDE"
  max: "INCLUDE"
  exact: false
settings:
  required: "optional"
  quiet: false
  global: false
  policy_scope: true
  bypass_provider: false
  hints:

find_package-v1 映射特有的键包括：

名称

所请求包的名称。

组件

如果存在，为一个对象数组，包含以下字段：

名称

组件名称。

required

指示组件是必须的还是可选的布尔值。

found

指示组件是否找到的布尔值。

configs

如果存在，为一个对象数组，指示要搜索的配置文件。

filename

配置文件的文件名。

种类

文件种类。为 cmake 或 cps 之一。

version_request

指示搜索版本约束的对象。

版本

所需最低版本。

version_complete

用户提供的版本范围。

min

指示版本范围下界是 INCLUDE 还是 EXCLUDE。

max

指示版本范围上界是 INCLUDE 还是 EXCLUDE。

exact

指示是否请求 EXACT 版本匹配的布尔值。

settings

搜索活动状态下的搜索设置。

required

搜索的需求请求。为 optional、optional_explicit、required_explicit、required_from_package_variable 或 required_from_find_variable 之一。

quiet

指示搜索是否为 QUIET 的布尔值。

global

指示是否提供了 GLOBAL 关键字的布尔值。

policy_scope

指示是否提供了 NO_POLICY_SCOPE 关键字的布尔值。

bypass_provider

指示是否提供了 BYPASS_PROVIDER 关键字的布尔值。

hints

作为 HINTS 提供的路径数组。

names

搜索时使用的包名称数组，由 NAMES 提供。

search_paths

要搜索的路径数组，由 PATHS 提供。

path_suffixes

搜索时使用的后缀数组，由 PATH_SUFFIXES 提供。

registry_view

搜索所请求的 REGISTRY_VIEW。

paths

搜索活动状态下的路径设置。

CMAKE_FIND_USE_CMAKE_PATH

指示搜索时是否使用 CMake 特定缓存变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_PATH。

CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH

指示搜索时是否使用 CMake 特定环境变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_ENVIRONMENT_PATH。

CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH

指示搜索时是否使用平台特定环境变量的布尔值。请参阅 CMAKE_FIND_USE_SYSTEM_ENVIRONMENT_PATH。

CMAKE_FIND_USE_CMAKE_SYSTEM_PATH

指示搜索时是否使用平台特定 CMake 变量的布尔值。请参阅 CMAKE_FIND_USE_CMAKE_SYSTEM_PATH。

CMAKE_FIND_USE_INSTALL_PREFIX

指示搜索时是否使用安装前缀的布尔值。请参阅 CMAKE_FIND_USE_INSTALL_PREFIX。

CMAKE_FIND_USE_CMAKE_PACKAGE_REGISTRY

指示是否在 CMake 包注册表中搜索该包的布尔值。请参阅 CMAKE_FIND_USE_PACKAGE_REGISTRY。

CMAKE_FIND_USE_SYSTEM_PACKAGE_REGISTRY

指示是否在系统 CMake 包注册表中搜索该包的布尔值。请参阅 CMAKE_FIND_USE_SYSTEM_PACKAGE_REGISTRY。

CMAKE_FIND_ROOT_PATH_MODE

指示当前生效的根路径模式的字符串，由 CMAKE_FIND_ROOT_PATH_BOTH、ONLY_CMAKE_FIND_ROOT_PATH 和 NO_CMAKE_FIND_ROOT_PATH 参数选择。

candidates

已拒绝的候选路径数组。每个元素包含以下键：

path

被考量的文件路径。对于依赖提供程序，该值为 dependency_provider::<COMMAND_NAME> 格式。

mode

查找该文件的模式。为 module、cps、cmake 或 provider 之一。

reason

路径被拒绝的原因。为 insufficient_version、no_exist、ignored、no_config_file 或 not_found 之一。

message

如果存在，为一个描述为什么该包被视为未找到的字符串。

found

如果包已找到，则为有关找到文件的信息。如果未找到，则为 null。可用键包括：

path

找到该包的模块或配置的路径。对于依赖提供程序，该值为 dependency_provider::<COMMAND_NAME> 格式。

mode

考量该路径的模式。为 module、cps、cmake 或 provider 之一。

版本

报告的包版本。

search_context

变量名到其指定的搜索路径的映射（取决于变量，可能为字符串或字符串数组）。环境变量被包裹在 ENV{ 和 } 之间，否则使用 CMake 变量。仅使用指定了路径的变量。

package_stack

对象数组，其中包含由 find_package() 调用提供的路径栈。

package_paths

调用栈中由 find_package() 命令提供的路径。
