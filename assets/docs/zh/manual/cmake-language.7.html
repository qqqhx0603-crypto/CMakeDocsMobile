组织结构¶

CMake 输入文件使用“CMake 语言”编写，位于命名为 CMakeLists.txt 或以 .cmake 为扩展名的源文件中。术语“列表文件”（listfile）是对任何此类包含 CMake 命令并由工具处理的源文件的统称。

项目中的 CMake 语言源文件组织为：

  • 目录（CMakeLists.txt）、

  • 脚本（<script>.cmake）以及

  • 模块（<module>.cmake）。

目录¶

当 CMake 处理项目源码树时，入口点是顶级源码目录中名为 CMakeLists.txt 的源文件。该文件可能包含完整的构建规范，或者使用 add_subdirectory() 命令将子目录添加到构建中。该命令添加的每个子目录也必须包含一个 CMakeLists.txt 文件作为该目录的入口点。对于每个处理了 CMakeLists.txt 文件的源码目录，CMake 都会在构建树中生成一个对应的目录，作为默认的工作目录和输出目录。

脚本¶

单个 <script>.cmake 源文件可以通过使用 cmake(1) 命令行工具并配合 -P 选项，以*脚本模式*进行处理。脚本模式仅运行给定的 CMake 语言源文件中的命令，而不生成构建系统。它不允许使用定义构建目标或操作的 CMake 命令。

模块¶

无论是在目录还是脚本中，CMake 语言代码都可以使用 include() 命令在当前作用域内加载 <module>.cmake 源文件。有关 CMake 发行版附带模块的文档，请参阅 cmake-modules(7) 手册页。项目源码树也可以提供自己的模块，并在 CMAKE_MODULE_PATH 变量中指定它们的位置。

语法¶

编码¶

CMake 语言源文件可以使用 7 位 ASCII 文本编写，以确保在所有支持的平台上具有最大的可移植性。换行符可以编码为 \n 或 \r\n，但在读取输入文件时会被统一转换为 \n。

请注意，此实现支持 8 位编码，因此在支持此编码的系统 API 的平台上，源文件可以采用 UTF-8 编码。此外，CMake 3.2 及更高版本在 Windows（微软桌面系统） 上支持以 UTF-8 编码的源文件（使用 UTF-16 调用系统 API）。此外，CMake 3.0 及更高版本允许源文件以 UTF-8 字节顺序标记 (BOM) 开头。

4.3 版本新增：cmake_host_system_information() 命令的 LOCALE_CHARSET 查询会返回预期的字符集编码。

源文件¶

CMake 语言源文件由零个或多个命令调用组成，这些调用由换行符分隔，并可选择使用空格和注释。

file         ::= file_element*
file_element ::= command_invocation line_ending |
                 (bracket_comment|space)* line_ending
line_ending  ::= line_comment? newline
space        ::= <match '[ \t]+'>
newline      ::= <match '\n'>

请注意，任何不在命令参数或括号注释内的源文件行都可以以行注释结尾。

命令调用¶

命令调用是一个名称，后跟由空格分隔且用圆括号括起来的参数。

command_invocation  ::= space* identifier space* '(' arguments ')'
identifier          ::= <match '[A-Za-z_][A-Za-z0-9_]*'>
arguments           ::= argument? separated_arguments*
separated_arguments ::= separation+ argument? |
                        separation* '(' arguments ')'
separation          ::= space | line_ending

例如

add_executable(hello world.c)

代码说明

  • add_executable(hello world.c)：调用 add_executable() 命令，定义一个可执行目标。

命令名称不区分大小写。参数中未加引号的嵌套括号必须平衡。每个 ( 或 ) 都会作为字面量无引号参数传递给命令调用。这可用于在调用 if() 命令时括住条件。例如：

if(FALSE AND (FALSE OR TRUE)) # evaluates to FALSE

代码说明

  • if(FALSE AND (FALSE OR TRUE)) # evaluates to FALSE：这一行继续补充当前命令或表达式的内容；条件运算符：AND 表示逻辑与：两侧条件都为真时结果为真；OR 表示逻辑或：任一侧条件为真时结果为真。

注意

3.0 之前的 CMake 版本要求命令名称标识符至少为 2 个字符。

2.8.12 之前的 CMake 版本会静默接受紧随引号参数之后且没有空格分隔的无引号参数或引号参数。为了兼容性，CMake 2.8.12 及更高版本接受此类代码，但会产生警告。

命令参数¶

在命令调用中有三种类型的参数：

argument ::= bracket_argument | quoted_argument | unquoted_argument

括号参数¶

括号参数受 Lua 长括号语法启发，将内容括在长度相同的开始和结束“括号”之间。

bracket_argument ::= bracket_open bracket_content bracket_close
bracket_open     ::= '[' '='* '['
bracket_content  ::= <any text not containing a bracket_close with
                     the same number of '=' as the bracket_open>
bracket_close    ::= ']' '='* ']'

左括号的写法是 [ 后跟零个或多个 =，再跟一个 [。相应的右括号写法是 ] 后跟相同数量的 =，再跟一个 ]。括号不可嵌套。总是可以为左右括号选择唯一的长度，以包含其他长度的右括号。

括号参数的内容由左括号和右括号之间的所有文本组成，但左括号后紧跟的一个换行符（如果有）会被忽略。不对括起来的内容进行任何求值（例如转义序列或变量引用）。括号参数总是作为一个单一参数传递给命令调用。

例如

message([=[
This is the first line in a bracket argument with bracket length 1.
No \-escape sequences or ${variable} references are evaluated.
This is always one argument even though it contains a ; character.
The text does not end on a closing bracket of length 0 like ]].
It does end in a closing bracket of length 1.
]=])

代码说明

  • message([=[：这一行给出示例文本或输出，用来说明：message([=[；运算符：= 表示赋值或键值关联。

  • This is the first line in a bracket argument with bracket length 1.：这是示例中的英文标题或状态文本，用于说明紧随其后的命令、阶段或输出。

  • No \-escape sequences or ${variable} references are evaluated.：这一行给出示例文本或输出，用来说明：No \-escape sequences or ${variable} references are evaluated.；变量说明：variable：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • This is always one argument even though it contains a ; character.：这一行给出示例文本或输出，用来说明：This is always one argument even though it contains a ; character.

  • The text does not end on a closing bracket of length 0 like ]].：这一行给出示例文本或输出，用来说明：The text does not end on a closing bracket of length 0 like ]].

  • It does end in a closing bracket of length 1.：这是示例中的英文标题或状态文本，用于说明紧随其后的命令、阶段或输出。

  • ]=])：这一行给出示例文本或输出，用来说明：]=])；运算符：= 表示赋值或键值关联。

注意

3.0 之前的 CMake 版本不支持括号参数。它们将左括号解释为无引号参数的开始。

引号参数¶

引号参数将内容括在双引号字符之间。

quoted_argument     ::= '"' quoted_element* '"'
quoted_element      ::= <any character except '\' or '"'> |
                        escape_sequence |
                        quoted_continuation
quoted_continuation ::= '\' newline

引号参数的内容由双引号之间的所有文本组成。其中的转义序列和变量引用都会被求值。引号参数总是作为一个单一参数传递给命令调用。

例如

message("This is a quoted argument containing multiple lines.
This is always one argument even though it contains a ; character.
Both \\-escape sequences and ${variable} references are evaluated.
The text does not end on an escaped double-quote like \".
It does end in an unescaped double quote.
")

代码说明

  • message("This is a quoted argument containing multiple lines.：这一行继续补充当前命令或表达式的内容。

  • This is always one argument even though it contains a ; character.：这一行继续补充当前命令或表达式的内容。

  • Both \\-escape sequences and ${variable} references are evaluated.：这一行继续补充当前命令或表达式的内容；变量说明：variable：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • The text does not end on an escaped double-quote like \".：这一行继续补充当前命令或表达式的内容。

  • It does end in an unescaped double quote.：这一行继续补充当前命令或表达式的内容；变量说明：quote：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • ")：这一行继续补充当前命令或表达式的内容。

任何以奇数个反斜杠结尾的行上的最后一个 \ 被视为行连接符，并与紧随其后的换行符一起被忽略。例如：

message("\
This is the first line of a quoted argument. \
In fact it is the only line but since it is long \
the source code uses line continuation.\
")

代码说明

  • message("\：这一行继续补充当前命令或表达式的内容。

  • This is the first line of a quoted argument. \：这一行继续补充当前命令或表达式的内容。

  • In fact it is the only line but since it is long \：这一行继续补充当前命令或表达式的内容。

  • the source code uses line continuation.\：这一行继续补充当前命令或表达式的内容。

  • ")：这一行继续补充当前命令或表达式的内容。

注意

3.0 之前的 CMake 版本不支持使用 \ 进行行连接。如果引号参数中的行以奇数个 \ 结尾，它们会报错。

无引号参数¶

无引号参数不被任何引号语法括起来。它不能包含任何空格、(、)、#、" 或 \，除非这些字符被反斜杠转义。

unquoted_argument ::= unquoted_element+ | unquoted_legacy
unquoted_element  ::= <any character except whitespace or one of '()#"\'> |
                      escape_sequence
unquoted_legacy   ::= <see note in text>

无引号参数的内容由连续的允许字符或转义字符块组成。其中的转义序列和变量引用都会被求值。生成的数值会以列表划分为元素的方式进行划分。每个非空元素都作为参数传递给命令调用。因此，一个无引号参数可以作为零个或多个参数传递给命令调用。

例如

foreach(arg
    NoSpace
    Escaped\ Space
    This;Divides;Into;Five;Arguments
    Escaped\;Semicolon
    )
  message("${arg}")
endforeach()

代码说明

  • foreach(arg：这一行继续补充当前命令或表达式的内容；变量说明：arg：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • NoSpace：这一行继续补充当前命令或表达式的内容。

  • Escaped\ Space：这一行继续补充当前命令或表达式的内容。

  • This;Divides;Into;Five;Arguments：这一行继续补充当前命令或表达式的内容。

  • Escaped\;Semicolon：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

  • message("${arg}")：调用 message() 输出提示信息，方便观察脚本执行结果；变量说明：arg：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • endforeach()：这一行结束或切换上一段控制结构的分支。

注意

为了支持遗留的 CMake 代码，无引号参数也可以包含双引号字符串（"..."，可能包含水平空白）和 Make 风格的变量引用（$(MAKEVAR)）。

未转义的双引号必须平衡，不能出现在无引号参数的开头，并被视为内容的一部分。例如，无引号参数 -Da="b c"、-Da=$(v) 和 a" "b"c"d 均按字面意思解释。它们可以分别写成引号参数 "-Da=\"b c\""、"-Da=$(v)" 和 "a\" \"b\"c\"d"。

Make 风格的引用按字面意思作为内容的一部分处理，不会进行变量扩展。它们被视为单个参数的一部分（而不是作为分开的 $、(、MAKEVAR 和 ) 参数）。

上述“无引号遗留”产生式代表了此类参数。我们不建议在新的代码中使用遗留的无引号参数。相反，请使用引号参数或括号参数来表示内容。

转义序列¶

转义序列是一个 \ 后跟一个字符。

escape_sequence  ::= escape_identity | escape_encoded | escape_semicolon
escape_identity  ::= '\' <match '[^A-Za-z0-9;]'>
escape_encoded   ::= '\t' | '\r' | '\n'
escape_semicolon ::= '\;'

紧跟在非字母数字字符后面的 \ 仅对其进行转义，将其作为字面量字符处理而不解释其语法。\t、\r 或 \n 分别表示制表符、回车符或换行符。在任何变量引用之外的 \; 自身编码为字符 ;，但可在无引号参数中使用它来编码 ;，而不将其作为参数分割符。在变量引用内部的 \; 编码为字面量 ; 字符。（另请参阅策略 CMP0053 文档以了解历史考量。）

变量引用¶

变量引用的形式为 ${<variable>}，并在引号参数或无引号参数内部进行求值。变量引用被指定变量或缓存条目的值替换；如果两者均未设置，则替换为空字符串。变量引用可以嵌套并从内向外求值，例如 ${outer_${inner_variable}_variable}。

字面量变量引用可以由字母数字字符、字符 /_.+- 以及转义序列组成。嵌套引用可用于评估任何名称的变量。另请参阅策略 CMP0053 文档以了解历史考量，以及为什么虽然从技术上允许 $ 但不建议这样做。

变量部分记录了变量名的作用域以及如何设置它们的值。

环境变量引用的形式为 $ENV{<variable>}。有关更多信息，请参阅环境变量部分。

缓存变量引用的形式为 $CACHE{<variable>}，它被替换为指定的缓存条目的值，而不检查相同名称的普通变量。如果缓存条目不存在，则替换为空字符串。有关更多信息，请参阅 CACHE。

if() 命令具有特殊的条件语法，允许在短形式中使用 <variable> 而不是 ${<variable>} 进行变量引用。但是，环境变量总是需要作为 $ENV{<variable>} 引用。

注释¶

注释以 # 字符开头，前提是该字符不在括号参数、引号参数内，或未作为无引号参数的一部分被 \ 转义。注释有两种类型：括号注释和行注释。

括号注释¶

# 后紧跟 bracket_open 构成了括号注释，其内容包含整个括号包围的区域。

bracket_comment ::= '#' bracket_argument

例如

#[[This is a bracket comment.
It runs until the close bracket.]]
message("First Argument\n" #[[Bracket Comment]] "Second Argument")

代码说明

  • #[[This is a bracket comment.：这是配置或脚本注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • It runs until the close bracket.]]：这一行给出示例文本或输出，用来说明：It runs until the close bracket.]]

  • message("First Argument\n" #[[Bracket Comment]] "Second Argument")：这一行给出示例文本或输出，用来说明：message("First Argument\n" #[[Bracket Comment]] "Second Argument")

注意

3.0 之前的 CMake 版本不支持括号注释。它们将开头的 # 解释为行注释的开始。

行注释¶

没有紧跟 bracket_open 的 # 构成了行注释，其内容延伸至行尾。

line_comment ::= '#' <any text not starting in a bracket_open
                 and not containing a newline>

例如

# This is a line comment.
message("First Argument\n" # This is a line comment :)
        "Second Argument") # This is a line comment.

代码说明

  • # This is a line comment.：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • message("First Argument\n" # This is a line comment :)：调用 message() 输出提示信息，方便观察脚本执行结果。

  • "Second Argument") # This is a line comment.：这一行继续补充当前命令或表达式的内容。

控制结构¶

条件块¶

if()/elseif()/else()/endif() 命令用于界定需要条件执行的代码块。

循环¶

foreach()/endforeach() 和 while()/endwhile() 命令界定循环执行的代码块。在这些块内，可以使用 break() 命令提前终止循环，而 continue() 命令可用于立即开始下一次迭代。

命令定义¶

macro()/endmacro() 和 function()/endfunction() 命令界定用于后续调用记录的代码块。

变量¶

变量是 CMake 语言中存储的基本单位。它们的值始终为字符串类型，尽管某些命令可能会将这些字符串解释为其他类型的值。set() 和 unset() 命令明确设置或取消设置变量，但其他命令也具有修改变量的语义。变量名区分大小写，几乎可以由任何文本组成，但我们建议仅使用字母数字字符加 _ 和 - 组成名称。

变量具有动态作用域。每个“设置”或“取消设置”操作都会在当前作用域中创建绑定。

块作用域

block() 命令可以为变量绑定创建一个新作用域。

函数作用域

由 function() 命令创建的命令定义，其创建的命令在被调用时，会在一个新的变量绑定作用域内处理所记录的命令。在该作用域内“设置”或“取消设置”变量，仅在当前函数及其中嵌套的调用中可见，函数返回后不可见。

目录作用域

源树中的每个目录都有自己的变量绑定。在处理目录的 CMakeLists.txt 文件之前，CMake 会复制父目录中当前定义的所有变量绑定（如果有）来初始化新的目录作用域。当 CMake 脚本通过 cmake -P 处理时，会在一个“目录”作用域内绑定变量。

不在函数调用内部进行的“设置”或“取消设置”操作会绑定到当前目录作用域。

持久缓存

CMake 存储了一组独立的“缓存”变量或“缓存条目”，它们的值在项目构建树中的多次运行期间保持不变。缓存条目具有独立的作用域，只能通过显式请求修改，例如使用 set() 和 unset() 命令的 CACHE 选项。

求值变量引用时，CMake 首先在函数调用栈（如果有）中搜索绑定，然后回退到当前目录作用域中的绑定（如果有）。如果找到“设置”绑定，则使用其值。如果找到“取消设置”绑定或未找到绑定，CMake 随后会搜索缓存条目。如果找到缓存条目，则使用其值。否则，变量引用求值为空字符串。可以使用 $CACHE{VAR} 语法执行直接的缓存条目查找。

cmake-variables(7) 手册记录了许多由 CMake 提供，或者在项目代码设置时对 CMake 有意义的变量。

注意

CMake 保留了以以下方式开头的标识符：

  • 以 CMAKE_ 开头（大写、小写或大小写混合），或者

  • 以 _CMAKE_ 开头（大写、小写或大小写混合），或者

  • 以 _ 开头，后跟任何 CMake 命令 的名称。

环境变量¶

环境变量与普通变量类似，区别如下：

作用域

环境变量在 CMake 进程中具有全局作用域。它们永远不会被缓存。

引用

变量引用使用 ENV 操作符，形式为 $ENV{<variable>}。

初始化

CMake 环境变量的初始值是调用进程的值。可以使用 set() 和 unset() 命令更改值。这些命令仅影响正在运行的 CMake 进程，而不影响整个系统环境。更改后的值不会写回调用进程，也不会被后续的构建或测试进程看到。

请参阅 cmake -E env 命令行工具，以便在修改后的环境中运行命令。

检查

请参阅 cmake -E environment 命令行工具以显示所有当前环境变量。

列表¶

尽管 CMake 中的所有值都存储为字符串，但在某些上下文中（例如在对无引号参数进行求值时），字符串可以被视为列表。在此类上下文中，字符串通过在不遵循不等数量 [ 和 ] 字符且不紧跟在 \ 之后的 ; 字符处拆分，从而划分为列表元素。序列 \; 不会分割值，但在生成的元素中会被替换为 ;。

元素列表表示为通过 ; 连接的分隔字符串。例如，set() 命令将多个值存储到目标变量中作为列表。

set(srcs a.c b.c c.c) # sets "srcs" to "a.c;b.c;c.c"

代码说明

  • set(srcs a.c b.c c.c) # sets "srcs" to "a.c;b.c;c.c"：这一行继续补充当前命令或表达式的内容；变量说明：srcs：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

列表旨在用于简单的用例，例如源文件列表，不应将其用于复杂的数据处理任务。大多数构造列表的命令不会转义列表元素中的 ; 字符，从而导致嵌套列表被展平。

set(x a "b;c") # sets "x" to "a;b;c", not "a;b\;c"

代码说明

  • set(x a "b;c") # sets "x" to "a;b;c", not "a;b\;c"：这一行继续补充当前命令或表达式的内容；变量说明：x：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

通常，列表不支持包含 ; 字符的元素。为了避免问题，请考虑以下建议：

  • 许多 CMake 命令、变量和属性的接口都接受分号分隔的列表。除非这些接口记录了直接支持或某种转义/编码分号的方法，否则请避免向这些接口传递包含分号的列表元素。

  • 构建列表时，在元素中用其他未使用的占位符代替 ;。然后在处理列表元素时，再用 ; 替换该占位符。例如，以下代码使用 | 代替 ; 字符：

    set(mylist a "b|c")
foreach(entry IN LISTS mylist)
  string(REPLACE "|" ";" entry "${entry}")
  # use "${entry}" normally
endforeach()

    代码说明

    • set(mylist a "b|c")：调用 set() 命令，把变量 mylist 设为给定的值；变量说明：mylist：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

    • foreach(entry IN LISTS mylist)：开始一个 foreach() 循环，后续语句会按列表元素依次执行；变量说明：entry：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

    • string(REPLACE "|" ";" entry "${entry}")：调用 string() 命令，执行字符串处理；变量说明：entry：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

    • # use "${entry}" normally：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中；变量说明：entry：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

    • endforeach()：这一行结束或切换上一段控制结构的分支。

    ExternalProject 模块的 LIST_SEPARATOR 选项就是一个使用此方法的接口示例。

  • 在生成器表达式列表中，请使用 $<SEMICOLON> 生成器表达式。

  • 在命令调用中，尽可能使用引号参数语法。被调用的命令将接收带有分号的参数内容，而不会分割。而无引号参数会在分号处被分割。

  • 在 function() 实现中，避免使用 ARGV 和 ARGN，它们无法区分值中的分号和分隔值的分号。相反，建议使用命名位置参数以及 ARGC 和 ARGV# 变量。当使用 cmake_parse_arguments() 解析参数时，建议使用其 PARSE_ARGV 签名，该签名使用 ARGV# 变量。

    请注意，此方法不适用于 macro() 实现，因为它们使用占位符而不是实际变量来引用参数。
