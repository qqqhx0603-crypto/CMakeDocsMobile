简介¶

CMake 可以查找并使用 Qt 4、Qt 5 和 Qt 6 库。Qt 4 库通过 CMake 自带的 FindQt4 查找模块进行查找，而 Qt 5 和 Qt 6 库则是使用 Qt 5 和 Qt 6 自带的“配置文件包（Config-file Packages）”进行查找。有关 CMake 包的更多信息，请参阅 cmake-packages(7)；关于您所使用的 Qt 版本，请参阅 Qt cmake 手册。

Qt 4、Qt 5 和 Qt 6 可以在同一个 CMake 构建系统 中一起使用

cmake_minimum_required(VERSION 3.16 FATAL_ERROR)

project(Qt4_5_6)

set(CMAKE_AUTOMOC ON)

find_package(Qt6 COMPONENTS Widgets DBus REQUIRED)
add_executable(publisher publisher.cpp)
target_link_libraries(publisher Qt6::Widgets Qt6::DBus)

find_package(Qt5 COMPONENTS Gui DBus REQUIRED)
add_executable(subscriber1 subscriber1.cpp)
target_link_libraries(subscriber1 Qt5::Gui Qt5::DBus)

find_package(Qt4 REQUIRED)
add_executable(subscriber2 subscriber2.cpp)
target_link_libraries(subscriber2 Qt4::QtGui Qt4::QtDBus)

一个 CMake 目标不能链接到多个 Qt 版本。如果尝试这样做，或者由于传递性目标依赖项评估导致这种情况，则会发出诊断信息。

Qt 构建工具¶

Qt 依赖一些捆绑的工具进行代码生成，例如用于元对象代码生成的 moc，用于控件布局和填充的 uic，以及用于虚拟文件系统内容生成的 rcc。如果满足适当的条件，这些工具可以由 cmake(1) 自动调用。自动工具调用可用于 Qt 4 到 6 版本。

AUTOMOC¶

AUTOMOC 目标属性控制 cmake(1) 是否检查目标中的 C++ 文件以确定它们是否需要运行 moc，并创建在适当时间执行 moc 的规则。

如果在头文件中找到 AUTOMOC_MACRO_NAMES 中的宏，则会对该文件运行 moc。结果将放入一个名为 moc_<basename>.cpp 的文件中。如果在 C++ 实现文件中找到该宏，moc 输出将按照 Qt 约定放入名为 <basename>.moc 的文件中。用户必须在 C++ 实现文件中使用预处理器 #include 包含 <basename>.moc。

包含的 moc_*.cpp 和 *.moc 文件将在 <AUTOGEN_BUILD_DIR>/include 目录中生成，该目录会自动添加到目标的 INCLUDE_DIRECTORIES 中。

  • 这与 CMake 3.7 及以下版本不同；详情请参阅其文档。

  • 对于 多配置生成器，包含目录为 <AUTOGEN_BUILD_DIR>/include_<CONFIG>。

  • 请参阅 AUTOGEN_BUILD_DIR。

未包含的 moc_<basename>.cpp 文件将在自定义文件夹中生成以避免名称冲突，并包含在编译到目标中的单独文件中，命名为 <AUTOGEN_BUILD_DIR>/mocs_compilation.cpp 或 <AUTOGEN_BUILD_DIR>/mocs_compilation_$<CONFIG>.cpp。

  • 请参阅 AUTOGEN_BUILD_DIR。

moc 命令行将使用被调用目标的 COMPILE_DEFINITIONS 和 INCLUDE_DIRECTORIES 目标属性，以及相应的构建配置。

AUTOMOC 目标属性可以通过设置 CMAKE_AUTOMOC 变量来预设所有后续目标。AUTOMOC_MOC_OPTIONS 目标属性可以被填充以设置传递给 moc 的选项。CMAKE_AUTOMOC_MOC_OPTIONS 变量可以被填充以预设所有后续目标的选项。

要搜索的其他宏名称可以添加到 AUTOMOC_MACRO_NAMES。

可以使用 AUTOMOC_DEPEND_FILTERS 从源代码中提取额外的 moc 依赖文件名。

可以通过启用 SKIP_AUTOMOC 或更广泛的 SKIP_AUTOGEN 来将 C++ 源文件从 AUTOMOC 处理中排除。

AUTOUIC¶

AUTOUIC 目标属性控制 cmake(1) 是否检查目标中的 C++ 文件以确定它们是否需要运行 uic，并创建在适当时间执行 uic 的规则。

如果找到匹配 <path>ui_<basename>.h 的预处理器 #include 指令，并且存在 <basename>.ui 文件，则将执行 uic 以生成适当的文件。<basename>.ui 文件将在以下位置搜索

  1. <source_dir>/<basename>.ui

  2. <source_dir>/<path><basename>.ui

  3. <AUTOUIC_SEARCH_PATHS>/<basename>.ui

  4. <AUTOUIC_SEARCH_PATHS>/<path><basename>.ui

其中 <source_dir> 是 C++ 文件的目录，AUTOUIC_SEARCH_PATHS 是额外搜索路径的列表。

生成的 ui_*.h 文件被放置在 <AUTOGEN_BUILD_DIR>/include 目录中，该目录会自动添加到目标的 INCLUDE_DIRECTORIES 中。

  • 这与 CMake 3.7 及以下版本不同；详情请参阅其文档。

  • 对于 多配置生成器，包含目录为 <AUTOGEN_BUILD_DIR>/include_<CONFIG>。

  • 请参阅 AUTOGEN_BUILD_DIR。

AUTOUIC 目标属性可以通过设置 CMAKE_AUTOUIC 变量来预设所有后续目标。AUTOUIC_OPTIONS 目标属性可以被填充以设置传递给 uic 的选项。CMAKE_AUTOUIC_OPTIONS 变量可以被填充以预设所有后续目标的选项。AUTOUIC_OPTIONS 源文件属性可以设置在 <basename>.ui 文件上，以便为该文件设置特定选项。这会覆盖 AUTOUIC_OPTIONS 目标属性中的选项。

目标可以填充 INTERFACE_AUTOUIC_OPTIONS 目标属性，其中包含调用 uic 时应使用的选项。这必须与依赖目标中 AUTOUIC_OPTIONS 目标属性的内容一致。CMAKE_DEBUG_TARGET_PROPERTIES 变量可用于跟踪此类 INTERFACE_AUTOUIC_OPTIONS 的来源目标。这意味着为 Qt 提供替代翻译系统的库可以指定运行 uic 时应使用的选项

add_library(KI18n klocalizedstring.cpp)
target_link_libraries(KI18n Qt6::Core)

# KI18n uses the tr2i18n() function instead of tr().  That function is
# declared in the klocalizedstring.h header.
set(autouic_options
  -tr tr2i18n
  -include klocalizedstring.h
)

set_property(TARGET KI18n APPEND PROPERTY
  INTERFACE_AUTOUIC_OPTIONS ${autouic_options}
)

链接到从上游导出的目标的消费项目，在通过 IMPORTED 目标链接的结果下，当由 AUTOUIC 运行 uic 时，会自动使用适当的选项

set(CMAKE_AUTOUIC ON)
# Uses a libwidget.ui file:
add_library(LibWidget libwidget.cpp)
target_link_libraries(LibWidget
  KF5::KI18n
  Qt5::Widgets
)

可以通过启用 SKIP_AUTOUIC 或更广泛的 SKIP_AUTOGEN 来将源文件从 AUTOUIC 处理中排除。

AUTORCC¶

AUTORCC 目标属性控制 cmake(1) 是否创建在适当时间对具有后缀 .qrc 的源文件执行 rcc 的规则。

add_executable(myexe main.cpp resource_file.qrc)

AUTORCC 目标属性可以通过设置 CMAKE_AUTORCC 变量来预设所有后续目标。AUTORCC_OPTIONS 目标属性可以被填充以设置传递给 rcc 的选项。CMAKE_AUTORCC_OPTIONS 变量可以被填充以预设所有后续目标的选项。AUTORCC_OPTIONS 源文件属性可以设置在 <name>.qrc 文件上，以便为该文件设置特定选项。这会覆盖 AUTORCC_OPTIONS 目标属性中的选项。

可以通过启用 SKIP_AUTORCC 或更广泛的 SKIP_AUTOGEN 来将源文件从 AUTORCC 处理中排除。

<ORIGIN>_autogen 目标¶

moc 和 uic 工具作为 CMake 生成的合成 <ORIGIN>_autogen 自定义目标 的一部分执行。默认情况下，该 <ORIGIN>_autogen 目标继承 <ORIGIN> 目标的依赖项（请参阅 AUTOGEN_ORIGIN_DEPENDS）。可以通过将目标添加到 AUTOGEN_TARGET_DEPENDS 目标属性，将目标依赖项添加到 <ORIGIN>_autogen 目标。

注意

如果使用 Qt 5.15 或更高版本，并且生成器是 Ninja 或 Makefile 生成器，请参阅 <ORIGIN>_autogen_timestamp_deps 目标。

<ORIGIN>_autogen_timestamp_deps 目标¶

如果使用 Qt 5.15 或更高版本，并且生成器是 Ninja 或 Makefile 生成器，除了 <ORIGIN>_autogen 目标外，还会创建 <ORIGIN>_autogen_timestamp_deps 目标。此目标没有任何源文件或命令要执行，但它具有之前由 Qt 5.15 之前的 <ORIGIN>_autogen 目标继承的依赖项。这些依赖项将作为自定义命令的仅顺序依赖项列表，而不会强制自定义命令重新执行。

Visual Studio 生成器¶

当使用 Visual Studio 生成器 时，CMake 会生成一个 PRE_BUILD 自定义命令，而不是 <ORIGIN>_autogen 自定义目标（对于 AUTOMOC 和 AUTOUIC）。但这并不总是可行的，当满足以下任一情况时，会使用 <ORIGIN>_autogen 自定义目标：

  • <ORIGIN> 目标依赖于 GENERATED 文件，这些文件没有被 AUTOMOC 和 AUTOUIC 通过 SKIP_AUTOMOC、SKIP_AUTOUIC、SKIP_AUTOGEN 或 CMP0071 排除

  • AUTOGEN_TARGET_DEPENDS 列出了一个源文件

  • CMAKE_GLOBAL_AUTOGEN_TARGET 已启用

Windows 上的 qtmain.lib¶

用于 QtGui 库的 Qt 4 和 5 导入目标 指定了 Qt 自带的 qtmain.lib 静态库将由所有启用了 WIN32_EXECUTABLE 的相关可执行文件链接。

要禁用此行为，请为基于 Qt 5 的目标启用 Qt5_NO_LINK_QTMAIN 目标属性，或为基于 Qt 4 的目标启用 QT4_NO_LINK_QTMAIN 目标属性。

add_executable(myexe WIN32 main.cpp)
target_link_libraries(myexe Qt4::QtGui)

add_executable(myexe_no_qtmain WIN32 main_no_qtmain.cpp)
set_property(TARGET main_no_qtmain PROPERTY QT4_NO_LINK_QTMAIN ON)
target_link_libraries(main_no_qtmain Qt4::QtGui)
