简介¶

CMake 生成器负责为原生构建系统编写输入文件。对于一个构建目录，必须精确选择一个 CMake 生成器，以确定要使用哪种原生构建系统。此外，还可以选择一个 额外生成器 作为某些 命令行构建工具生成器 的变体，用于为辅助 IDE 生成项目文件。

CMake 生成器是平台相关的，因此每个生成器可能仅在特定平台上可用。cmake(1) 命令行工具的 --help 输出中列出了当前平台可用的生成器。使用其 -G 选项来为新的构建目录指定生成器。cmake-gui(1) 在创建新构建目录时提供了交互式选择生成器的功能。

CMake 生成器¶

命令行构建工具生成器¶

这些生成器支持命令行构建工具。为了使用它们，必须从一个已针对所选编译器和构建工具配置好环境的命令行提示符中启动 CMake。

  • Borland Makefile（构建脚本）s
  • MSYS Makefile（构建脚本）s
  • MinGW Makefile（构建脚本）s
  • NMake Makefile（构建脚本）s
  • NMake Makefile（构建脚本）s JOM
  • Unix Makefile（构建脚本）s
  • Watcom WMake

Ninja（构建工具） 生成器¶

  • Ninja（构建工具）

FASTBuild 生成器¶

  • FASTBuild

IDE 构建工具生成器¶

这些生成器支持集成开发环境 (IDE) 项目文件。由于 IDE 会自行配置环境，因此可以从任何环境中启动 CMake。

额外生成器¶

自 3.27 版本起弃用： 对“额外生成器”的支持已被弃用，并将从 CMake 的未来版本中移除。IDE 可以使用 cmake-file-api(7) 来查看 CMake 生成的项目构建树。

CMake 生成器 在 cmake(1) 命令行工具的 --help 输出中列出，其中一些可能包含指定辅助 IDE 额外生成器的变体。此类生成器名称的形式为 <extra-generator> - <main-generator>。以下是 CMake 已知的额外生成器。

  • CodeBlocks
  • CodeLite
  • Eclipse CDT4
  • Kate
  • Sublime Text 2
