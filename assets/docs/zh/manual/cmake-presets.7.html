简介¶

3.19 版本新增。

CMake 用户经常面临的一个问题是：如何与他人共享项目的常用配置方式。这可能是为了支持 CI 构建，或者是为了方便经常使用相同构建设置的用户。CMake 支持两个主要文件：CMakePresets.json 和 CMakeUserPresets.json，允许用户指定常用的配置选项并与他人共享。CMake 还支持通过 include 字段包含其他文件。

CMakePresets.json 和 CMakeUserPresets.json 位于项目的根目录中。它们的格式完全相同，且都是可选的（尽管如果指定了 --preset，则至少需要存在一个）。CMakePresets.json 用于指定项目范围内的构建详情，而 CMakeUserPresets.json 则用于开发者指定其个人的本地构建详情。

CMakePresets.json 可以提交到版本控制系统中，而 CMakeUserPresets.json 则不应提交。例如，如果项目使用 Git，可以跟踪 CMakePresets.json，并将 CMakeUserPresets.json 添加到 .gitignore 中。

格式¶

这些文件是以对象为根节点的 JSON 文档。

{
  "version": 10,
  "cmakeMinimumRequired": {
    "major": 3,
    "minor": 23,
    "patch": 0
  },
  "$comment": "An example CMakePresets.json file",
  "include": [
    "otherThings.json",
    "moreThings.json"
  ],
  "configurePresets": [
    {
      "$comment": [
        "This is a comment row.",
        "This is another comment,",
        "just because we can do it"
      ],
      "name": "default",
      "displayName": "Default Config",
      "description": "Default build using Ninja generator",
      "generator": "Ninja",
      "binaryDir": "${sourceDir}/build/default",
      "cacheVariables": {
        "FIRST_CACHE_VARIABLE": {
          "type": "BOOL",
          "value": "OFF"
        },
        "SECOND_CACHE_VARIABLE": "ON"
      },
      "environment": {
        "MY_ENVIRONMENT_VARIABLE": "Test",
        "PATH": "$env{HOME}/ninja/bin:$penv{PATH}"
      },
      "vendor": {
        "example.com/ExampleIDE/1.0": {
          "autoFormat": true
        }
      }
    },
    {
      "name": "ninja-multi",
      "inherits": "default",
      "displayName": "Ninja Multi-Config",
      "description": "Default build using Ninja Multi-Config generator",
      "generator": "Ninja Multi-Config"
    },
    {
      "name": "windows-only",
      "inherits": "default",
      "displayName": "Windows-only configuration",
      "description": "This build is only available on Windows",
      "condition": {
        "type": "equals",
        "lhs": "${hostSystemName}",
        "rhs": "Windows"
      }
    }
  ],
  "buildPresets": [
    {
      "name": "default",
      "configurePreset": "default"
    }
  ],
  "testPresets": [
    {
      "name": "default",
      "configurePreset": "default",
      "output": {"outputOnFailure": true},
      "execution": {"noTestsAction": "error", "stopOnFailure": true}
    }
  ],
  "packagePresets": [
    {
      "name": "default",
      "configurePreset": "default",
      "generators": [
        "TGZ"
      ]
    }
  ],
  "workflowPresets": [
    {
      "name": "default",
      "steps": [
        {
          "type": "configure",
          "name": "default"
        },
        {
          "type": "build",
          "name": "default"
        },
        {
          "type": "test",
          "name": "default"
        },
        {
          "type": "package",
          "name": "default"
        }
      ]
    }
  ],
  "vendor": {
    "example.com/ExampleIDE/1.0": {
      "autoFormat": false
    }
  }
}

指定版本 10 或更高的预设文件可以在 JSON 对象的任何层级使用键 $comment 添加注释，以提供文档说明。

根对象识别以下字段：

$schema

一个可选字符串，提供指向描述此 JSON 文档结构的 JSON 架构的 URI。该字段用于支持 JSON 架构的编辑器中的验证和自动补全。它不会影响文档自身的行为。如果未指定此字段，JSON 文档仍然有效，但使用 JSON 架构进行验证和自动补全的工具可能无法正常工作。

版本

一个必需的整数，表示 JSON 架构的版本。有关支持的版本以及它们添加时对应的 CMake 版本，请参阅 版本。

cmakeMinimumRequired

一个可选对象，表示构建此项目所需的 CMake 最低版本。此对象包含以下字段：

主要版本

一个可选整数，表示主版本号。

次要版本

一个可选整数，表示次版本号。

patch

一个可选整数，表示补丁版本号。

include

一个可选字符串数组，表示要包含的文件。如果文件名不是绝对路径，则相对于当前文件计算。这在指定版本 4 或更高的预设文件中是被允许的。有关包含文件的约束条件，请参阅 包含。

vendor

一个可选映射，包含特定于供应商的信息。除验证其是否为映射外，CMake 不会解释此字段的内容。但是，键应当是供应商特定的域名，后跟以 / 分隔的路径。例如，Example IDE 1.0 可以使用 example.com/ExampleIDE/1.0。每个字段的值可以是供应商想要的任何内容，尽管通常是一个映射。

configurePresets

一个可选的 配置预设 (Configure Preset) 对象数组。这在指定版本 1 或更高的预设文件中是被允许的。

buildPresets

一个可选的 构建预设 (Build Preset) 对象数组。这在指定版本 2 或更高的预设文件中是被允许的。

testPresets

一个可选的 测试预设 (Test Preset) 对象数组。这在指定版本 2 或更高的预设文件中是被允许的。

packagePresets

一个可选的 打包预设 (Package Preset) 对象数组。这在指定版本 6 或更高的预设文件中是被允许的。

workflowPresets

一个可选的 工作流预设 (Workflow Preset) 对象数组。这在指定版本 6 或更高的预设文件中是被允许的。

包含 (Includes)¶

在文件版本 4 及以后，CMakePresets.json 和 CMakeUserPresets.json 可以通过 include 字段包含其他文件。被这些文件包含的文件也可以包含其他文件。如果 CMakePresets.json 和 CMakeUserPresets.json 同时存在，即使没有 include 字段，在所有格式版本中，CMakeUserPresets.json 都会隐式地包含 CMakePresets.json。

如果预设文件包含从另一个文件继承的预设，则必须直接或间接地包含该文件。文件之间不允许存在包含循环。如果 a.json 包含 b.json，则 b.json 不能包含 a.json。但是，同一个文件可以从相同或不同的文件中被多次包含。

直接或间接从 CMakePresets.json 包含的文件应保证由项目提供。CMakeUserPresets.json 可以包含来自任何位置的文件。

从版本 7 开始，include 字段支持 宏展开，但仅限于 $penv{} 宏展开。从版本 9 开始，也支持其他宏展开，但 $env{} 和预设特定的宏（即源自预设定义内字段（如 presetName）的宏）除外。

配置预设 (Configure Preset)¶

configurePresets 数组的每个条目都是一个 JSON 对象，可能包含以下字段：

名称

一个必需的字符串，表示预设的机器友好名称。此标识符用于 cmake --preset 选项。在同一目录下的 CMakePresets.json 和 CMakeUserPresets.json 的并集中，不能有两个名称相同的配置预设。但是，配置预设可以与构建、测试、打包或工作流预设重名。

hidden

一个可选的布尔值，指定预设是否应被隐藏。如果预设被隐藏，则不能在 --preset 参数中使用，不会出现在 CMake GUI 中，并且不需要具有有效的 generator 或 binaryDir（即使是从继承中获取）。hidden 预设旨在用作其他预设通过 inherits 字段进行继承的基础。

inherits

一个可选字符串数组，表示要继承的预设名称。此字段也可以是一个字符串，等同于包含一个字符串的数组。

默认情况下，预设将从 inherits 预设中继承所有字段（name、hidden、inherits、description 和 displayName 除外），但可以根据需要重写它们。如果多个 inherits 预设为同一字段提供冲突的值，则优先选择 inherits 数组中靠前的预设。

预设只能继承自同一文件或其包含的文件（直接或间接）中定义的预设。CMakePresets.json 中的预设不能继承自 CMakeUserPresets.json 中的预设。

condition

一个可选的 条件 (Condition) 对象。这在指定版本 3 或更高的预设文件中是被允许的。

vendor

一个可选映射，包含特定于供应商的信息。除验证其是否为映射外，CMake 不会解释此字段的内容。但是，它应遵循与根级 vendor 字段相同的约定。如果供应商使用自己的每个预设的 vendor 字段，则在适当时应以合理的方式实现继承。

displayName

一个可选字符串，包含预设的人类友好名称。

描述

一个可选字符串，包含预设的人类友好描述。

generator

一个可选字符串，表示要用于预设的 生成器 (generator)。如果未指定 generator，则必须从 inherits 预设中继承（除非该预设是 hidden 的）。在版本 3 或更高版本中，可以省略此字段以回退到常规的生成器发现过程。

请注意，对于 Visual Studio 生成器，与命令行 -G 参数不同，不能在生成器名称中包含平台名称。请改用 architecture 字段。

architecture, toolset

可选字段，分别表示支持它们的 生成器 的平台和工具集。

有关 architecture 的可能值，请参阅 cmake -A 选项；有关 toolset，请参阅 cmake -T 选项。

每个字段可以是字符串，也可以是包含以下字段的对象：

value

一个可选字符串，表示该值。

strategy

一个可选字符串，告诉 CMake 如何处理 architecture 或 toolset 字段。有效值为：

"set"

设置相应的值。对于不支持相应字段的生成器，这将导致错误。

"external"

即使生成器支持该值，也不要设置它。例如，如果预设使用 Ninja 生成器，而 IDE 知道如何从 architecture 和 toolset 字段设置 Visual C++ 环境，这会非常有用。在这种情况下，CMake 将忽略该字段，但 IDE 可以在调用 CMake 之前使用它们来设置环境。

如果未提供 strategy 字段，或者如果该字段使用字符串形式而不是对象形式，则其行为与 "set" 相同。

toolchainFile

一个可选字符串，表示工具链文件的路径。此字段支持 宏展开。如果指定了相对路径，则相对于构建目录计算；如果未找到，则相对于源代码目录计算。此字段的优先级高于任何 CMAKE_TOOLCHAIN_FILE 的值。在指定版本 3 或更高的预设文件中是被允许的。

graphviz

一个可选字符串，表示 graphviz 输入文件的路径，其中将包含项目中的所有库和可执行文件依赖关系。有关更多详细信息，请参阅 cmake --graphviz 的文档。

此字段支持 宏展开。如果指定了相对路径，则相对于当前工作目录计算。在指定版本 10 或更高的预设文件中是被允许的。

binaryDir

一个可选字符串，表示输出二进制目录的路径。此字段支持 宏展开。如果指定了相对路径，则相对于源代码目录计算。如果未指定 binaryDir，则必须从 inherits 预设中继承（除非该预设是 hidden 的）。在版本 3 或更高版本中，可以省略此字段。

installDir

一个可选字符串，表示安装目录的路径，该路径将用作 CMAKE_INSTALL_PREFIX 变量。此字段支持 宏展开。如果指定了相对路径，则相对于源代码目录计算。在指定版本 3 或更高的预设文件中是被允许的。

cmakeExecutable

一个可选字符串，表示用于此预设的 CMake 可执行文件的路径。这保留供 IDE 使用，CMake 本身不会使用。使用此字段的 IDE 应展开其中的任何宏。

cacheVariables

一个缓存变量的可选映射。键是变量名称（不能为空字符串），值可以是 null、布尔值（等同于 "TRUE" 或 "FALSE" 且类型为 BOOL）、表示变量值的字符串（支持 宏展开），或是包含以下字段的对象：

type

一个可选字符串，表示变量的类型。

value

一个必需的字符串或布尔值，表示变量的值。布尔值等同于 "TRUE" 或 "FALSE"。此字段支持 宏展开。

缓存变量通过 inherits 字段继承，预设的变量将是其自身 cacheVariables 与其所有父级 cacheVariables 的并集。如果此并集中的多个预设定义了相同的变量，则应用 inherits 的标准规则。将变量设置为 null 会导致它不被设置，即使它是从另一个预设继承而来的。

environment

一个可选的环境变量映射。键是变量名称（不能为空字符串），值可以是 null 或表示变量值的字符串。无论流程的环境是否为其提供了值，每个变量都会被设置。

此字段支持 宏展开，并且此映射中的环境变量可以相互引用，并且可以以任何顺序排列，只要这些引用不会导致循环即可（例如，如果 ENV_1 是 $env{ENV_2}，则 ENV_2 不能是 $env{ENV_1}）。$penv{NAME} 允许通过仅访问父环境中的值来向现有环境变量添加或追加值。

环境变量通过 inherits 字段继承，预设的环境将是其自身 environment 与其所有父级 environment 的并集。如果此并集中的多个预设定义了相同的变量，则应用 inherits 的标准规则。将变量设置为 null 会导致它不被设置，即使它是从另一个预设继承而来的。

warnings

一个可选对象，指定要启用的警告。该对象可能包含以下字段：

开发

一个可选布尔值。等同于在命令行上传递 -Wdev 或 -Wno-dev。如果 errors.dev 设置为 true，则此项不能设置为 false。

已弃用

一个可选布尔值。等同于在命令行上传递 -Wdeprecated 或 -Wno-deprecated。如果 errors.deprecated 设置为 true，则此项不能设置为 false。

uninitialized

一个可选布尔值。将其设置为 true 等同于在命令行上传递 --warn-uninitialized。

unusedCli

一个可选布尔值。将其设置为 false 等同于在命令行上传递 --no-warn-unused-cli。

systemVars

一个可选布尔值。将其设置为 true 等同于在命令行上传递 --check-system-vars。

errors

一个可选对象，指定要启用的错误。该对象可能包含以下字段：

开发

一个可选布尔值。等同于在命令行上传递 -Werror=dev 或 -Wno-error=dev。如果 warnings.dev 设置为 false，则此项不能设置为 true。

已弃用

一个可选布尔值。等同于在命令行上传递 -Werror=deprecated 或 -Wno-error=deprecated。如果 warnings.deprecated 设置为 false，则此项不能设置为 true。

debug

一个可选对象，指定调试选项。该对象可能包含以下字段：

output

一个可选布尔值。将其设置为 true 等同于在命令行上传递 --debug-output。

tryCompile

一个可选布尔值。将其设置为 true 等同于在命令行上传递 --debug-trycompile。

find

一个可选布尔值。将其设置为 true 等同于在命令行上传递 --debug-find。

trace

一个可选对象，指定追踪选项。这在指定版本 7 的预设文件中是被允许的。该对象可能包含以下字段：

mode

一个可选字符串，指定追踪模式。有效值为：

on

导致打印所有进行的调用及其来源的追踪信息。等同于在命令行上传递 --trace。

off

不会打印所有调用的追踪信息。

expand

导致打印所有进行的调用及其来源的带有变量展开的追踪信息。等同于在命令行上传递 --trace-expand。

format

一个可选字符串，指定追踪的格式输出。有效值为：

人工

以人类可读的格式打印每一行追踪。这是默认格式。等同于在命令行上传递 --trace-format=human。

json-v1

将每一行打印为单独的 JSON 文档。等同于在命令行上传递 --trace-format=json-v1。

源

一个可选字符串数组，表示要追踪的源文件路径。此字段也可以是一个字符串，等同于包含一个字符串的数组。等同于在命令行上传递 --trace-source。

redirect

一个可选字符串，指定追踪输出文件的路径。等同于在命令行上传递 --trace-redirect。

构建预设 (Build Preset)¶

buildPresets 数组的每个条目都是一个 JSON 对象，可能包含以下字段：

名称

一个必需的字符串，表示预设的机器友好名称。此标识符用于 cmake --build --preset 选项。在同一目录下的 CMakePresets.json 和 CMakeUserPresets.json 的并集中，不能有两个名称相同的构建预设。但是，构建预设可以与配置、测试、打包或工作流预设重名。

hidden

一个可选的布尔值，指定预设是否应被隐藏。如果预设被隐藏，则不能在 --preset 参数中使用，并且不需要具有有效的 configurePreset（即使是从继承中获取）。hidden 预设旨在用作其他预设通过 inherits 字段进行继承的基础。

inherits

一个可选字符串数组，表示要继承的预设名称。此字段也可以是一个字符串，等同于包含一个字符串的数组。

默认情况下，预设将从 inherits 预设中继承所有字段（name、hidden、inherits、description 和 displayName 除外），但可以根据需要重写它们。如果多个 inherits 预设为同一字段提供冲突的值，则优先选择 inherits 数组中靠前的预设。

预设只能继承自同一文件或其包含的文件（直接或间接）中定义的预设。CMakePresets.json 中的预设不能继承自 CMakeUserPresets.json 中的预设。

condition

一个可选的 条件 (Condition) 对象。这在指定版本 3 或更高的预设文件中是被允许的。

vendor

一个可选映射，包含特定于供应商的信息。除验证其是否为映射外，CMake 不会解释此字段的内容。但是，它应遵循与根级 vendor 字段相同的约定。如果供应商使用自己的每个预设的 vendor 字段，则在适当时应以合理的方式实现继承。

displayName

一个可选字符串，包含预设的人类友好名称。

描述

一个可选字符串，包含预设的人类友好描述。

environment

一个可选的环境变量映射。键是变量名称（不能为空字符串），值可以是 null 或表示变量值的字符串。无论流程的环境是否为其提供了值，每个变量都会被设置。

此字段支持 宏展开，并且此映射中的环境变量可以相互引用，并且可以以任何顺序排列，只要这些引用不会导致循环即可（例如，如果 ENV_1 是 $env{ENV_2}，则 ENV_2 不能是 $env{ENV_1}）。$penv{NAME} 允许通过仅访问父环境中的值来向现有环境变量添加或追加值。

环境变量通过 inherits 字段继承，预设的环境将是其自身 environment 与其所有父级 environment 的并集。如果此并集中的多个预设定义了相同的变量，则应用 inherits 的标准规则。将变量设置为 null 会导致它不被设置，即使它是从另一个预设继承而来的。

注意

对于使用 ExternalProject 且配置预设中具有 ExternalProject 所需环境变量的 CMake 项目，请使用继承该配置预设的构建预设，否则 ExternalProject 将不会设置配置预设中的环境变量。示例：假设主机默认使用一个编译器（例如 Clang），而用户希望使用另一个编译器（例如 GCC）。设置配置预设环境变量 CC 和 CXX，并使用继承该配置预设的构建预设。否则，ExternalProject 可能会使用与顶级 CMake 项目不同的（系统默认）编译器。

configurePreset

一个可选字符串，指定要与此构建预设关联的配置预设名称。如果未指定 configurePreset，则必须从 inherits 预设中继承（除非该预设是 hidden 的）。构建目录是从配置预设中推断出来的，因此构建将在与配置相同的 binaryDir 中进行。

inheritConfigureEnvironment

一个可选布尔值，默认为 true。如果为 true，则来自关联配置预设的环境变量将在所有继承的构建预设环境之后、但在明确指定于此构建预设的环境变量之前被继承。

jobs

一个可选整数。等同于在命令行上传递 --parallel 或 -j。如果值为 0，则等同于传递 --parallel 而省略 <jobs>；或者，可以使用 environment 字段将环境变量 CMAKE_BUILD_PARALLEL_LEVEL 定义为空字符串。

版本 4.3 变更：无论预设文件中的版本如何，此字段都不接受负整数值。

targets

一个可选字符串或字符串数组。等同于在命令行上传递 --target 或 -t。供应商可能会忽略 targets 属性，或隐藏明确指定 targets 的构建预设。此字段支持宏展开。

configuration

一个可选字符串。等同于在命令行上传递 --config。

cleanFirst

一个可选布尔值。如果为 true，则等同于在命令行上传递 --clean-first。

resolvePackageReferences

一个可选字符串，指定包解析模式。这在指定版本 4 或更高的预设文件中是被允许的。

包引用用于定义对来自外部包管理器的包的依赖。目前仅支持与 Visual Studio 生成器 结合使用的 NuGet。如果没有定义包引用的目标，此选项无效。有效值为：

on

在尝试构建之前解析包引用。

off

不会解析包引用。请注意，这可能会在某些构建环境中（例如 .NET SDK 风格的项目）导致错误。

only

仅解析包引用，但不执行构建。

注意

命令行参数 --resolve-package-references 的优先级高于此设置。如果未提供命令行参数且未指定此设置，则将评估特定于环境的缓存变量以决定是否应执行包恢复。

使用 Visual Studio 生成器 时，包引用使用 VS_PACKAGE_REFERENCES 属性定义。包引用使用 NuGet 恢复。可以通过将 CMAKE_VS_NUGET_PACKAGE_RESTORE 变量设置为 OFF 来禁用它。这也可以在配置预设中完成。

verbose

一个可选布尔值。如果为 true，则等同于在命令行上传递 --verbose。

nativeToolOptions

一个可选字符串数组。等同于在命令行上传递 -- 之后的选项。数组值支持宏展开。

测试预设 (Test Preset)¶

testPresets 数组的每个条目都是一个 JSON 对象，可能包含以下字段：

名称

一个必需的字符串，表示预设的机器友好名称。此标识符用于 ctest --preset 选项。在同一目录下的 CMakePresets.json 和 CMakeUserPresets.json 的并集中，不能有两个名称相同的测试预设。但是，测试预设可以与配置、构建、打包或工作流预设重名。

hidden

一个可选的布尔值，指定预设是否应被隐藏。如果预设被隐藏，则不能在 --preset 参数中使用，并且不需要具有有效的 configurePreset（即使是从继承中获取）。hidden 预设旨在用作其他预设通过 inherits 字段进行继承的基础。

inherits

一个可选字符串数组，表示要继承的预设名称。此字段也可以是一个字符串，等同于包含一个字符串的数组。

默认情况下，预设将从 inherits 预设中继承所有字段（name、hidden、inherits、description 和 displayName 除外），但可以根据需要重写它们。如果多个 inherits 预设为同一字段提供冲突的值，则优先选择 inherits 数组中靠前的预设。

预设只能继承自同一文件或其包含的文件（直接或间接）中定义的预设。CMakePresets.json 中的预设不能继承自 CMakeUserPresets.json 中的预设。

condition

一个可选的 条件 (Condition) 对象。这在指定版本 3 或更高的预设文件中是被允许的。

vendor

一个可选映射，包含特定于供应商的信息。除验证其是否为映射外，CMake 不会解释此字段的内容。但是，它应遵循与根级 vendor 字段相同的约定。如果供应商使用自己的每个预设的 vendor 字段，则在适当时应以合理的方式实现继承。

displayName

一个可选字符串，包含预设的人类友好名称。

描述

一个可选字符串，包含预设的人类友好描述。

environment

一个可选的环境变量映射。键是变量名称（不能为空字符串），值可以是 null 或表示变量值的字符串。无论流程的环境是否为其提供了值，每个变量都会被设置。

此字段支持 宏展开，并且此映射中的环境变量可以相互引用，并且可以以任何顺序排列，只要这些引用不会导致循环即可（例如，如果 ENV_1 是 $env{ENV_2}，则 ENV_2 不能是 $env{ENV_1}）。$penv{NAME} 允许通过仅访问父环境中的值来向现有环境变量添加或追加值。

环境变量通过 inherits 字段继承，预设的环境将是其自身 environment 与其所有父级 environment 的并集。如果此并集中的多个预设定义了相同的变量，则应用 inherits 的标准规则。将变量设置为 null 会导致它不被设置，即使它是从另一个预设继承而来的。

configurePreset

一个可选字符串，指定要与此测试预设关联的配置预设名称。如果未指定 configurePreset，则必须从 inherits 预设中继承（除非该预设是 hidden 的）。构建目录是从配置预设中推断出来的，因此测试将在与配置和构建相同的 binaryDir 中运行。

inheritConfigureEnvironment

一个可选布尔值，默认为 true。如果为 true，则来自关联配置预设的环境变量将在所有继承的测试预设环境之后、但在明确指定于此测试预设的环境变量之前被继承。

configuration

条件 (Condition)¶

预设的 condition 字段（在指定版本 3 或更高的预设文件中是被允许的）用于确定是否启用了该预设。例如，这可以用于在 Windows 以外的平台上禁用预设。condition 可以是布尔值、null 或对象。如果是布尔值，则表示预设是启用还是禁用。如果是 null，则启用该预设，但 null 条件不会被可能从该预设继承的任何预设所继承。子条件（例如在 not、anyOf 或 allOf 条件中）不能是 null。如果是对象，它包含以下字段：

type

一个必需字符串，具有以下值之一：

"const"

指示条件是常量。这等同于使用布尔值代替对象。条件对象将具有以下附加字段：

value

一个必需布尔值，为条件的评估提供常量值。

"equals"

"notEquals"

指示条件比较两个字符串以查看它们是否相等（或不相等）。条件对象将具有以下附加字段：

lhs

要比较的第一个字符串。此字段支持宏展开。

rhs

要比较的第二个字符串。此字段支持宏展开。

"inList"

"notInList"

指示条件在字符串列表中搜索字符串。条件对象将具有以下附加字段：

string

一个必需字符串，用于搜索。

list

一个必需字符串数组，用于搜索。此字段支持宏展开，并使用短路评估。

"matches"

"notMatches"

指示条件在字符串中搜索正则表达式。条件对象将具有以下附加字段：

string

一个必需字符串，用于搜索。

regex

一个必需的正则表达式，用于搜索。此字段支持宏展开。

"anyOf"

"allOf"

指示条件是零个或多个嵌套条件的聚合。条件对象将具有以下附加字段：

conditions

一个必需的条件对象数组。这些条件使用短路评估。

"not"

指示条件是另一个条件的倒置。条件对象将具有以下附加字段：

condition

一个必需的条件对象。

宏展开 (Macro Expansion)¶

如上所述，某些字段支持宏展开。宏以 $<macro-namespace>{<macro-name>} 的形式识别。所有宏均在所使用的预设上下文中进行评估，即使该宏位于从另一个预设继承的字段中也是如此。例如，如果 Base 预设将变量 PRESET_NAME 设置为 ${presetName}，而 Derived 预设继承自 Base，则 PRESET_NAME 将被设置为 Derived。

在宏名称末尾不加闭合花括号是错误的。例如，${sourceDir 是无效的。后跟除左花括号（{）及其可能的命名空间之外的任何内容的美元符号（$）将被解释为字面上的美元符号。

已识别的宏包括

${sourceDir}

项目源代码目录的路径（即与 CMAKE_SOURCE_DIR 相同）。

${sourceParentDir}

项目源代码目录的父目录路径。

${sourceDirName}

${sourceDir} 的最后一个文件名部分。例如，如果 ${sourceDir} 是 /path/to/source，则此值将为 source。

${presetName}

在预设的 name 字段中指定的名称。

这是一个特定于预设的宏。

${generator}

在预设的 generator 字段中指定的生成器。对于构建（build）和测试（test）预设，该值将求值为由 configurePreset 指定的生成器。

这是一个特定于预设的宏。

${hostSystemName}

主机操作系统的名称。包含与 CMAKE_HOST_SYSTEM_NAME 相同的值。这在指定版本 3 或更高的预设文件中是允许的。

${fileDir}

包含该宏的预设文件所在的目录路径。这在指定版本 4 或更高的预设文件中是允许的。

${dollar}

字面量美元符号（$）。

${pathListSep}

用于分隔路径列表的原生字符，例如 : 或 ;。

例如，通过将 PATH 设置为 /path/to/ninja/bin${pathListSep}$env{PATH}，${pathListSep} 将扩展为底层操作系统用于在 PATH 中进行连接的字符。

这在指定版本 5 或更高的预设文件中是允许的。

$env{<variable-name>}

名称为 <variable-name> 的环境变量。变量名不能为空字符串。如果变量定义在 environment 字段中，则使用该值，而不是来自父环境的值。如果未定义该环境变量，则求值为空字符串。

请注意，虽然 Windows 环境变量名称是不区分大小写的，但预设中的变量名称仍然区分大小写。在大小写不一致时，这可能会导致意外的结果。为了获得最佳效果，请保持环境变量名称的大小写一致。

$penv{<variable-name>}

与 $env{<variable-name>} 类似，不同之处在于该值仅来自父环境，从不来自 environment 字段。这允许在现有环境变量的前面或后面追加值。例如，将 PATH 设置为 /path/to/ninja/bin:$penv{PATH} 将把 /path/to/ninja/bin 添加到 PATH 环境变量的最前面。这是必要的，因为 $env{<variable-name>} 不允许循环引用。

$vendor{<macro-name>}

供厂商插入其自定义宏的扩展点。CMake 将无法使用包含 $vendor{<macro-name>} 宏的预设，并有效地忽略这些预设。不过，它仍然可以使用同一文件中的其他预设。

CMake 不会尝试解释 $vendor{<macro-name>} 宏。然而，为了避免名称冲突，IDE 厂商应该在 <macro-name> 前添加一个非常简短（最好 <= 4 个字符）的厂商标识符前缀，后跟一个 .，再后跟宏名称。例如，Example IDE 可以使用 $vendor{xide.ideInstallDir}。

版本¶

CMakePresets.json 和 CMakeUserPresets.json 的 JSON 架构遵循一种版本方案，新版本会在较新版本的 CMake 中添加并支持。

下面列出了支持的版本，以及添加它们的 CMake 版本和新功能与更改的摘要。

1

3.19 版本新增。

初始版本支持 配置预设 (Configure Presets) 和 宏扩展 (Macro Expansion)。

2

在 3.20 版本中添加。

  • 添加了 构建预设 (Build Presets)。

  • 添加了 测试预设 (Test Presets)。

3

3.21 版本新增。

  • 为 配置、构建 和 测试预设 添加了 条件 (Condition) 对象。

  • 配置预设 的变更

    • 添加了 installDir 字段。

    • 添加了 toolchainFile 字段。

    • binaryDir 字段现在变为可选。

    • generator 字段现在变为可选。

  • 宏扩展 的变更

    • 添加了 ${hostSystemName} 宏。

4

在版本 3.23 中添加。

  • 添加了 包含 (Includes)，以支持在 CMakePresets.json 和 CMakeUserPresets.json 中包含其他 JSON 文件。

  • 构建预设 的变更

    • 添加了 resolvePackageReferences 字段。

  • 宏扩展 的变更

    • 添加了 ${fileDir} 宏。

5

在 3.24 版本中添加。

  • 测试预设 的变更

    • 在 output 对象中添加了 testOutputTruncation 字段。

  • 宏扩展 的变更

    • 添加了 ${pathListSep} 宏。

6

在 3.25 版本中新增。

  • 添加了 打包预设 (Package Presets)。

  • 添加了 工作流预设 (Workflow Presets)。

  • 测试预设 的变更

    • 在 output 对象中添加了 outputJUnitFile 字段。

7

在 3.27 版本中新增。

  • 配置预设 的变更

    • 添加了 trace 字段。

  • 包含 的变更

    • include 字段现在支持 $penv{} 宏扩展。

8

版本 3.28 新增。

  • 在根对象中添加了 $schema 字段。

9

3.30 版本新增。

  • 包含 的变更

    • include 字段现在支持其他类型的 宏扩展。

10

在版本 3.31 中添加。

  • 添加了可选的 $comment 字段，以支持在 CMakePresets.json 和 CMakeUserPresets.json 中添加注释。

  • 配置预设 的变更

    • 添加了 graphviz 字段。

11

Added in version 4.3.

  • 测试预设 的变更

    • jobs 字段现在接受空字符串，表示省略 <jobs> 的 --parallel。
