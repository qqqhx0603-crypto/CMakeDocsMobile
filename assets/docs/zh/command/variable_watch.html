variable_watch¶

监视 CMake 变量的变化。

variable_watch(<variable> [<command>])

代码说明

  • variable_watch(<variable> [<command>])：调用 variable_watch() 命令，具体行为由这一行传入的参数决定。

如果指定的 <variable> 发生变化，并且没有提供 <command>，则会打印一条消息以告知该变化。

如果提供了 <command>，则会执行该命令。该命令将接收以下参数：COMMAND(<variable> <access> <value> <current_list_file> <stack>)

<variable>

正在访问的变量的名称。

<access>

以下之一：READ_ACCESS、UNKNOWN_READ_ACCESS、MODIFIED_ACCESS、UNKNOWN_MODIFIED_ACCESS 或 REMOVED_ACCESS。 UNKNOWN_ 值仅在变量从未设置时使用。 一旦设置，即使变量稍后被取消设置，在同一次 CMake 运行期间也不会再次使用它们。

<value>

变量的值。 在修改时，这是变量的新（修改后）值。 在删除时，该值为空。

<current_list_file>

执行访问的文件完整路径。

<stack>

当前在文件包含堆栈上的所有文件的绝对路径列表，从最底层的文件开始，到当前处理的文件（即 current_list_file）结束。

请注意，对于某些访问（例如 list(APPEND)），观察者会执行两次，先进行读取访问，然后进行写入访问。 另外请注意，对变量的 if(DEFINED) 查询不会被注册为访问，并且观察者不会被执行。

只有非缓存变量才能使用此命令进行监视。 永远不会监视对缓存变量的访问。 但是，缓存变量 var 的存在会导致对非缓存变量 var 的访问不使用 UNKNOWN_ 前缀，即使非缓存变量 var 从未存在过。
