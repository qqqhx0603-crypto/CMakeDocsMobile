使用指定的源文件向项目中添加一个可执行文件。

普通可执行文件¶

add_executable(<name> <options>... <sources>...)¶

添加一个由命令调用中列出的源文件构建的 可执行文件 目标，名称为 <name>。

选项包括

WIN32

自动设置 WIN32_EXECUTABLE 目标属性。详情请参阅该目标属性的文档。

MACOSX_BUNDLE

自动设置 MACOSX_BUNDLE 目标属性。详情请参阅该目标属性的文档。

EXCLUDE_FROM_ALL

自动设置 EXCLUDE_FROM_ALL 目标属性。详情请参阅该目标属性的文档。

<name> 对应逻辑目标名称，且在项目中必须是全局唯一的。构建出的实际可执行文件名是根据原生平台的约定（例如 <name>.exe 或仅为 <name>）构建的。

3.1 版本新增：add_executable 的源参数可以使用语法为 $<...> 的“生成器表达式”。有关可用表达式，请参阅 cmake-generator-expressions(7) 手册。

3.11 版本新增：如果源文件随后通过 target_sources() 添加，则可以省略源文件。

默认情况下，可执行文件将在对应于调用命令所在的源目录的构建树目录中创建。若要更改此位置，请参阅 RUNTIME_OUTPUT_DIRECTORY 目标属性的文档。若要更改最终文件名的 <name> 部分，请参阅 OUTPUT_NAME 目标属性的文档。

有关定义构建系统属性的更多信息，请参阅 cmake-buildsystem(7) 手册。

若某些源文件经过预处理，且希望在 IDE 中能够访问原始源文件，请参阅 HEADER_FILE_ONLY 的相关说明。

导入的可执行文件¶

add_executable(<name> IMPORTED [GLOBAL])¶

添加一个 导入的可执行目标 以引用位于项目外部的可执行文件。该目标名称的引用方式与项目中构建的任何目标相同，区别在于它默认仅在创建它的目录及其子目录中可见。

选项包括

GLOBAL

使目标名称全局可见。

导入的目标不会生成任何构建规则，且 IMPORTED 目标属性为 True。导入的可执行文件对于从诸如 add_custom_command() 等命令进行便捷引用非常有用。

导入的可执行文件的详情是通过设置以 IMPORTED_ 开头的属性来指定的。其中最重要的属性是 IMPORTED_LOCATION（以及按配置的版本 IMPORTED_LOCATION_<CONFIG>），它指定了磁盘上主可执行文件的位置。有关更多信息，请参阅 IMPORTED_* 属性的文档。

别名可执行文件¶

add_executable(<name> ALIAS <target>)¶

创建一个 别名目标，使得在后续命令中可以使用 <name> 来指代 <target>。<name> 不会作为 make 目标出现在生成的构建系统中。<target> 不可以是 ALIAS。

3.11 版本新增：ALIAS 可以针对 GLOBAL 导入的目标。

3.18 版本新增：ALIAS 可以针对非 GLOBAL 的导入目标。此类别名的作用域限于创建它的目录及其子目录。ALIAS_GLOBAL 目标属性可用于检查别名是否为全局的。

ALIAS 目标可用作读取属性的目标，也可用于自定义命令和自定义目标的可执行文件。它们还可以通过常规的 if(TARGET) 子命令来测试其是否存在。<name> 不得用于修改 <target> 的属性，也就是说，它不能作为 set_property()、set_target_properties()、target_link_libraries() 等的操作数。ALIAS 目标不得被安装或导出。
