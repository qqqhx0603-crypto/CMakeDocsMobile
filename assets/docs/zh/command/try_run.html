尝试编译并运行一些代码。

尝试编译并运行源文件¶

try_run(<runResultVar> <compileResultVar>
        [SOURCES_TYPE <type>]
        {SOURCES <srcfile>...                 |
         SOURCE_FROM_CONTENT <name> <content> |
         SOURCE_FROM_VAR <name> <var>         |
         SOURCE_FROM_FILE <name> <path>       }...
        [LOG_DESCRIPTION <text>]
        [NO_CACHE]
        [NO_LOG]
        [CMAKE_FLAGS <flags>...]
        [COMPILE_DEFINITIONS <defs>...]
        [LINK_OPTIONS <options>...]
        [LINK_LIBRARIES <libs>...]
        [COMPILE_OUTPUT_VARIABLE <var>]
        [COPY_FILE <fileName> [COPY_FILE_ERROR <var>]]
        [<LANG>_STANDARD <std>]
        [<LANG>_STANDARD_REQUIRED <bool>]
        [<LANG>_EXTENSIONS <bool>]
        [RUN_OUTPUT_VARIABLE <var>]
        [RUN_OUTPUT_STDOUT_VARIABLE <var>]
        [RUN_OUTPUT_STDERR_VARIABLE <var>]
        [WORKING_DIRECTORY <var>]
        [ARGS <args>...]
        )

在 3.25 版本中新增。

尝试从一个或多个源文件构建可执行文件。构建成功则在 <compileResultVar> 中返回布尔值 true，构建失败则返回布尔值 false（除非指定了 NO_CACHE，否则该结果会被缓存）。如果构建成功，则运行该可执行文件并将退出代码存储在 <runResultVar> 中（除非指定了 NO_CACHE，否则该结果会被缓存）。如果可执行文件已构建但运行失败，则 <runResultVar> 将被设置为 FAILED_TO_RUN。关于两个命令共有的选项以及测试项目如何构建源文件的信息，请参阅 try_compile() 命令的文档。

必须提供一个或多个源文件。此外，SOURCES 和/或 SOURCE_FROM_* 必须置于其他关键字之前。

3.26 版本新增: 如果不指定 NO_LOG 选项，该命令会记录一个 configure-log try_run 事件。

此命令支持 CMake 3.25 之前的另一种签名方式。为了清晰起见，建议使用上述签名方式。

try_run(<runResultVar> <compileResultVar>
        <bindir> {<srcfile>|SOURCES <srcfile>...}
        [CMAKE_FLAGS <flags>...]
        [COMPILE_DEFINITIONS <defs>...]
        [LINK_OPTIONS <options>...]
        [LINK_LIBRARIES <libs>...]
        [LINKER_LANGUAGE <lang>]
        [COMPILE_OUTPUT_VARIABLE <var>]
        [COPY_FILE <fileName> [COPY_FILE_ERROR <var>]]
        [<LANG>_STANDARD <std>]
        [<LANG>_STANDARD_REQUIRED <bool>]
        [<LANG>_EXTENSIONS <bool>]
        [RUN_OUTPUT_VARIABLE <var>]
        [OUTPUT_VARIABLE <var>]
        [WORKING_DIRECTORY <var>]
        [ARGS <args>...]
        )

选项¶

try_run 特有的选项包括：

COMPILE_OUTPUT_VARIABLE <var>

在给定的变量中报告编译步骤的构建输出。

OUTPUT_VARIABLE <var>

在给定的变量中报告编译构建输出以及运行可执行文件的输出。此选项仅出于遗留原因保留，且仅由旧版 try_run 签名支持。建议优先使用 COMPILE_OUTPUT_VARIABLE 和 RUN_OUTPUT_VARIABLE。

RUN_OUTPUT_VARIABLE <var>

在给定的变量中报告运行可执行文件的输出。

RUN_OUTPUT_STDOUT_VARIABLE <var>

在 3.25 版本中新增。

在给定的变量中报告运行可执行文件产生的标准输出 (stdout)。

RUN_OUTPUT_STDERR_VARIABLE <var>

在 3.25 版本中新增。

在给定的变量中报告运行可执行文件产生的标准错误输出 (stderr)。

WORKING_DIRECTORY <var>

在 3.20 版本中添加。

在给定的目录下运行可执行文件。如果未指定 WORKING_DIRECTORY，可执行文件将在 <bindir> 或当前构建目录中运行。

ARGS <args>...

运行可执行文件时传递给它的附加参数。

其他行为设置¶

设置变量 CMAKE_TRY_COMPILE_CONFIGURATION 以选择构建配置。

  • 对于多配置生成器，此选项用于选择要构建的配置。

  • 对于单配置生成器，此选项在测试项目中设置 CMAKE_BUILD_TYPE。

交叉编译时的行为¶

3.3 版本新增: 运行交叉编译的二进制文件时，使用 CMAKE_CROSSCOMPILING_EMULATOR。

进行交叉编译时，第一步编译出的可执行文件通常无法在构建主机上运行。try_run 命令会检查 CMAKE_CROSSCOMPILING 变量，以检测 CMake 是否处于交叉编译模式。如果是，它仍然会尝试编译可执行文件，但不会尝试运行它（除非设置了 CMAKE_CROSSCOMPILING_EMULATOR 变量）。相反，它会创建缓存变量，这些变量必须由用户手动填充，或者通过在某个 CMake 脚本文件中预设它们，以对应可执行文件如果在目标平台上运行时本应产生的值。这些缓存条目包括：

<runResultVar>

如果可执行文件在目标平台上运行，其预期的退出代码。

<runResultVar>__TRYRUN_OUTPUT

如果可执行文件在目标平台上运行，其预期的标准输出和标准错误输出。仅当使用了 RUN_OUTPUT_VARIABLE 或 OUTPUT_VARIABLE 选项时才会创建。

<runResultVar>__TRYRUN_OUTPUT_STDOUT

在 3.25 版本中新增。

如果可执行文件在目标平台上运行，其预期的标准输出。仅当使用了 RUN_OUTPUT_STDOUT_VARIABLE 或 RUN_OUTPUT_STDERR_VARIABLE 选项时才会创建。

<runResultVar>__TRYRUN_OUTPUT_STDERR

在 3.25 版本中新增。

如果可执行文件在目标平台上运行，其预期的标准错误输出。仅当使用了 RUN_OUTPUT_STDOUT_VARIABLE 或 RUN_OUTPUT_STDERR_VARIABLE 选项时才会创建。

为了使项目的交叉编译更容易，仅在确实需要时才使用 try_run。如果您使用了 try_run，请仅在必要时使用 RUN_OUTPUT_STDOUT_VARIABLE、RUN_OUTPUT_STDERR_VARIABLE、RUN_OUTPUT_VARIABLE 或 OUTPUT_VARIABLE 选项。使用它们将意味着在交叉编译时，必须手动设置缓存变量为可执行文件的输出值。您也可以通过检查 if() 块中的 CMAKE_CROSSCOMPILING 变量来“保护”对 try_run 的调用，并为此情况提供一个易于预设的替代方案。
