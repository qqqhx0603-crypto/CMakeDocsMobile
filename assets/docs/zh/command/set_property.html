set_property¶

在给定的作用域内设置命名属性。

set_property({GLOBAL                                    |
              DIRECTORY [<dir>]                         |
              TARGET    <target>...                     |
              FILE_SET  <file_set>... TARGET <target>   |
              SOURCE    <source>...
                        [DIRECTORY <dirs> ...]
                        [TARGET_DIRECTORY <targets>...] |
              INSTALL   <file>...                       |
              TEST      <test>...
                        [DIRECTORY <dir>]               |
              CACHE     <entry>...}
             [APPEND] [APPEND_STRING]
             PROPERTY <name> [<value>...])

在作用域内的一个或多个对象上设置一个属性。

第一个参数决定了设置属性的作用域。它必须是以下选项之一：

GLOBAL

该作用域是唯一的，不需要名称。

DIRECTORY

作用域默认为当前目录，但其他（已由 CMake 处理过的）目录可以通过完整路径或相对路径来命名。相对路径被视为相对于当前源目录。另请参阅 set_directory_properties() 命令。

3.19 版本新增： <dir> 可以引用二进制目录。

TARGET

作用域可以命名零个或多个现有的目标（target）。另请参阅 set_target_properties() 命令。

别名目标（Alias Targets） 不支持设置目标属性。

FILE_SET

Added in version 4.3.

作用域可以命名零个或多个现有的文件集（file set）。

必须包含以下选项：

TARGET <target>

附加该文件集的目标。

SOURCE

作用域可以命名零个或多个源文件。默认情况下，源文件属性仅对在同一目录（CMakeLists.txt）中添加的目标可见。

3.18 版本新增：可以使用以下一个或两个子选项在其他目录作用域中设置可见性：

DIRECTORY <dirs>...

源文件属性将在每个 <dirs> 目录的作用域中设置。CMake 必须已经知晓这些目录，或者是通过调用 add_subdirectory() 添加的，或者是顶级源目录。相对路径被视为相对于当前源目录。

3.19 版本新增：<dirs> 可以引用二进制目录。

TARGET_DIRECTORY <targets>...

源文件属性将在创建了指定 <targets> 中任意一个的目录作用域中设置（因此这些 <targets> 必须已经存在）。

另请参阅 set_source_files_properties() 命令。

INSTALL

版本 3.1 中新增。

作用域可以命名零个或多个已安装的文件路径。这些路径可供 CPack 使用以影响部署。

属性键和值均可使用生成器表达式（generator expressions）。特定属性可能适用于已安装的文件和/或目录。

路径组件必须使用正斜杠分隔，必须经过规范化且区分大小写。

要使用相对路径引用安装前缀本身，请使用 .。

目前已安装文件的属性仅针对 WIX 生成器定义，其中给定的路径是相对于安装前缀的。

TEST

作用域仅限于调用该命令所在的目录。它可以命名零个或多个现有的测试。另请参阅命令 set_tests_properties()。

对于通过 add_test(NAME) 特征创建的测试，测试属性值可以使用 生成器表达式 来指定。

3.28 版本新增：可以使用以下子选项在其他目录作用域中设置可见性：

DIRECTORY <dir>

测试属性将在 <dir> 目录的作用域内设置。CMake 必须已经知晓该目录，或者是通过调用 add_subdirectory() 添加的，或者是顶级源目录。相对路径被视为相对于当前源目录。<dir> 可以引用二进制目录。

CACHE

作用域必须命名零个或多个现有的缓存条目。

必须的 PROPERTY 选项后面紧跟要设置的属性名称。其余参数用于以分号分隔列表的形式构成属性值。

如果提供了 APPEND 选项，该列表将被附加到任何现有的属性值之后（空值会被忽略且不会被附加）。如果提供了 APPEND_STRING 选项，该字符串将作为字符串附加到任何现有的属性值之后，即它会生成一个更长的字符串，而不是字符串列表。当对定义为支持 INHERITED 行为的属性使用 APPEND 或 APPEND_STRING 时（参见 define_property()），在查找要附加到的初始值时不会发生继承。如果该属性尚未在指定作用域中直接设置，则该命令的行为将如同未提供 APPEND 或 APPEND_STRING 一样。

注意

全局可见的源文件属性 GENERATED。有关详细信息，请参阅其文档。
