target_include_directories¶

向目标（target）添加包含目录。

target_include_directories(<target> [SYSTEM] [AFTER|BEFORE]
  {INTERFACE|PUBLIC|PRIVATE} <dir...
  [{INTERFACE|PUBLIC|PRIVATE} <dir>...]...)

代码说明

  • target_include_directories(<target> [SYSTEM] [AFTER|BEFORE]：这一行继续补充当前命令或表达式的内容。

  • {INTERFACE|PUBLIC|PRIVATE} <dir...：这一行继续补充当前命令或表达式的内容；运算符：< 表示小于比较。

  • [{INTERFACE|PUBLIC|PRIVATE} <dir>...]...)：这一行继续补充当前命令或表达式的内容。

指定编译给定目标时使用的包含目录。指定的 <target> 必须通过诸如 add_executable() 或 add_library() 等命令创建，且不得为 别名目标（ALIAS target（别名目标））。

通过显式使用 AFTER 或 BEFORE，您可以选择追加（append）或前置（prepend），而不受默认行为的影响。

INTERFACE、PUBLIC 和 PRIVATE 关键字用于指定后续参数的 作用域。PRIVATE 和 PUBLIC 项将填充 <target> 的 INCLUDE_DIRECTORIES 属性。PUBLIC 和 INTERFACE 项将填充 <target> 的 INTERFACE_INCLUDE_DIRECTORIES 属性。以下参数指定了要包含的目录。

3.11 版本新增: 允许在 导入目标 上设置 INTERFACE 项。

对同一个 <target> 的重复调用会按调用顺序追加项目。

如果指定了 SYSTEM，编译器将被告知这些目录在某些平台上作为系统包含目录。这可能会产生一些效果，例如抑制警告或在依赖项计算中跳过这些目录下的头文件（详见编译器文档）。此外，无论指定的顺序如何，系统包含目录总是在普通包含目录之后进行搜索。

如果 SYSTEM 与 PUBLIC 或 INTERFACE 一起使用，目标属性 INTERFACE_SYSTEM_INCLUDE_DIRECTORIES 将会被填充指定的目录。

target_include_directories 的参数可以使用语法为 $<...> 的生成器表达式。有关可用表达式，请参阅 cmake-generator-expressions(7) 手册。有关定义构建系统属性的更多信息，请参阅 cmake-buildsystem(7) 手册。

指定的包含目录可以是绝对路径或相对路径。相对路径将被解释为相对于当前源目录（即 CMAKE_CURRENT_SOURCE_DIR），并在存储到相关的目标属性之前转换为绝对路径。如果路径以生成器表达式开头，它将被假定为绝对路径（下文注明的特例除外），并将按原样使用。

包含目录的使用需求在构建树（build-tree）和安装树（install-tree）之间通常有所不同。BUILD_INTERFACE 和 INSTALL_INTERFACE 生成器表达式可用于描述基于使用位置的独立使用需求。INSTALL_INTERFACE 表达式中允许使用相对路径，并将被解释为相对于安装前缀（installation prefix）。不应在 BUILD_INTERFACE 表达式中使用相对路径，因为它们不会被转换为绝对路径。例如

target_include_directories(mylib PUBLIC
  $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include/mylib>
  $<INSTALL_INTERFACE:include/mylib>  # <prefix>/include/mylib
)

代码说明

  • target_include_directories(mylib PUBLIC：这一行继续补充当前命令或表达式的内容。

  • $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include/mylib>：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录；生成器表达式：$<BUILD_INTERFACE:...> 在生成构建系统时按当前目标和配置计算结果。

  • $<INSTALL_INTERFACE:include/mylib> # <prefix>/include/mylib：这一行继续补充当前命令或表达式的内容；生成器表达式：$<INSTALL_INTERFACE:...> 在生成构建系统时按当前目标和配置计算结果。

  • )：这一行继续补充当前命令或表达式的内容。

创建可重定位包¶

请注意，不建议使用依赖项包含目录的绝对路径来填充目标的 INTERFACE_INCLUDE_DIRECTORIES 中的 INSTALL_INTERFACE。这会将在构建包的机器上发现的依赖项包含目录路径硬编码到已安装的包中。

INTERFACE_INCLUDE_DIRECTORIES 中的 INSTALL_INTERFACE 仅适用于指定目标自身提供的头文件所需的包含目录，而不适用于其 INTERFACE_LINK_LIBRARIES 目标属性中列出的传递依赖项所提供的头文件。这些依赖项本身应当是指定了其自身在 INTERFACE_INCLUDE_DIRECTORIES 中头文件位置的目标。

有关创建可重定位包（relocatable packages）时在指定使用需求时需要注意的额外事项的讨论，请参阅 cmake-packages(7) 手册的 创建可重定位包 部分。
