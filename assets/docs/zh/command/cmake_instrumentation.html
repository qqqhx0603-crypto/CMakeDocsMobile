cmake_instrumentation¶

4.3 版本新增。

启用与 CMake Instrumentation API（CMake 插桩 API）的交互。

这允许在项目级别配置插桩。

cmake_instrumentation(
  API_VERSION <version>
  DATA_VERSION <version>
  [HOOKS <hooks>...]
  [OPTIONS <options>...]
  [CALLBACK <callback>]
  [CUSTOM_CONTENT <name> <type> <content>]
)

代码说明

  • cmake_instrumentation( API_VERSION <version> DATA_VERSION <version> [HOOKS <hooks>...] [OPTIONS <options>...] [CALLBACK <callback>] [CUSTOM_CONTENT <name> <type> <content>] )：调用 `cmake_instrumentation()` 命令，具体行为由这一行传入的参数决定。

必须始终提供 API_VERSION 和 DATA_VERSION。

API_VERSION 是整数。目前唯一支持的值是 1。有关详细信息，请参阅 API v1。

DATA_VERSION 是 major 或 major.minor 形式的版本值。目前唯一支持的版本是 1.0。有关详细信息，请参阅 数据版本。

可选关键字 HOOKS、OPTIONS 和 CALLBACK 中的每一个都对应 v1 查询文件 的参数之一。CALLBACK 关键字可以多次提供，以创建多个回调。

每当调用 cmake_instrumentation 时，就会在 <build>/.cmake/instrumentation/v1/query/generated 中生成一个查询文件，以使用提供的参数启用插桩。

自定义 CMake 内容¶

CUSTOM_CONTENT 参数指定了配置阶段中需要包含在每个 v1 CMake 内容文件 中的特定数据。这可用于将插桩数据与有关其配置的特定信息相关联，例如优化级别或其是否属于覆盖率构建的一部分。

CUSTOM_CONTENT 需要 name、type 和 content 参数。

name 是用于标识所报告内容的说明符。

type 指定了应如何解释该内容。支持的值包括：

  • STRING 内容为字符串。

  • BOOL 内容应被解释为布尔值。其判定条件与 if() 对给定值的判定条件一致（即为 true）。

  • LIST 内容为需要解析的 CMake ; 分隔列表。

  • JSON 内容应作为 JSON 字符串进行解析。这可以是数字（如 1 或 5.0）、带引号的字符串（如 \"string\"）、布尔值 true/false，或者可以使用 string(JSON ...) 命令构建的 JSON 对象（如 { \"key\" : \"value\" }）。

content 是要报告的实际内容。

示例¶

以下示例展示了该命令的调用及其对应的 JSON 查询文件。

cmake_instrumentation(
  API_VERSION 1
  DATA_VERSION 1.0
  HOOKS postGenerate preCMakeBuild postCMakeBuild
  OPTIONS staticSystemInformation dynamicSystemInformation trace
  CALLBACK ${CMAKE_COMMAND} -P /path/to/handle_data.cmake
  CALLBACK ${CMAKE_COMMAND} -P /path/to/handle_data_2.cmake
  CUSTOM_CONTENT myString STRING string
  CUSTOM_CONTENT myList   LIST   "item1;item2"
  CUSTOM_CONTENT myObject JSON   "{ \"key\" : \"value\" }"
)

代码说明

  • cmake_instrumentation( API_VERSION 1 DATA_VERSION 1.0 HOOKS postGenerate preCMakeBuild postCMakeBuild OPTIONS staticSystemInformation dynamicSystemInformation trace CALLBACK ${CMAKE_COMMAND} -P /path/to/handle_data.cmake CALLBACK ${CMAKE_COMMAND} -P /path/to/handle_data_2.cmake CUSTOM_CONTENT myString STRING string CUSTOM_CONTENT myList LIST "item1;item2" CUSTOM_CONTENT myObject JSON "{ \"key\" : \"value\" }" )：调用 `cmake_instrumentation()` 命令，具体行为由这一行传入的参数决定。

{
  "version": 1,
  "hooks": [
    "postGenerate", "preCMakeBuild", "postCMakeBuild"
  ],
  "options": [
    "staticSystemInformation", "dynamicSystemInformation", "trace"
  ],
  "callbacks": [
    "/path/to/cmake -P /path/to/handle_data.cmake",
    "/path/to/cmake -P /path/to/handle_data_2.cmake"
  ]
}

代码说明

  • { "version": 1, "hooks": [ "postGenerate", "preCMakeBuild", "postCMakeBuild" ], "options": [ "staticSystemInformation", "dynamicSystemInformation", "trace" ], "callbacks": [ "/path/to/cmake -P /path/to/handle_data.cmake", "/path/to/cmake -P /path/to/handle_data_2.cmake" ] }：这一行给键 `{ "version` 指定对应的值。

这也将导致以下内容包含在每个 v1 CMake 内容文件 中：

"custom": {
  "myString": "string",
  "myList": [
    "item1", "item2"
  ],
  "myObject": {
    "key": "value"
  }
}

代码说明

  • "custom": { "myString": "string", "myList": [ "item1", "item2" ], "myObject": { "key": "value" } }：这一行给键 `custom` 指定对应的值。
