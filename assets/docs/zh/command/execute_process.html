execute_process¶

执行一个或多个子进程。

execute_process(COMMAND <cmd1> [<arguments>]
                [COMMAND <cmd2> [<arguments>]]...
                [WORKING_DIRECTORY <directory>]
                [TIMEOUT <seconds>]
                [RESULT_VARIABLE <variable>]
                [RESULTS_VARIABLE <variable>]
                [OUTPUT_VARIABLE <variable>]
                [ERROR_VARIABLE <variable>]
                [INPUT_FILE <file>]
                [OUTPUT_FILE <file>]
                [ERROR_FILE <file>]
                [OUTPUT_QUIET]
                [ERROR_QUIET]
                [COMMAND_ECHO <where>]
                [OUTPUT_STRIP_TRAILING_WHITESPACE]
                [ERROR_STRIP_TRAILING_WHITESPACE]
                [ENCODING <name>]
                [ECHO_OUTPUT_VARIABLE]
                [ECHO_ERROR_VARIABLE]
                [COMMAND_ERROR_IS_FATAL <ANY|LAST|NONE>])

运行给定的一个或多个命令序列。

命令以管道形式并发执行，每个进程的标准输出通过管道传输到下一个进程的标准输入。所有进程共用一个标准错误管道。

execute_process 在 CMake 配置项目时（即在构建系统生成之前）运行命令。若要创建在构建时运行的自定义命令，请使用 add_custom_target() 和 add_custom_command() 命令。

选项

COMMAND

子进程命令行。

CMake 直接使用操作系统 API 执行子进程。

  • 在 POSIX 平台上，命令行以 argv[] 样式的数组传递给子进程。不会执行中间 shell（命令解释器），因此 shell（命令解释器） 操作符（如 >）会被视为普通参数。

  • 在 Windows（微软桌面系统） 平台上，命令行被编码为一个字符串，使得使用 CommandLineToArgvW 的子进程能够解码出原始参数。

    如果命令运行的是 .exe、.com 或其他可执行文件，则不会执行中间命令解释器，因此 shell（命令解释器） 操作符（如 >）会被视为普通参数。

    如果命令运行的是 .bat 或 .cmd 脚本，它将通过 cmd 命令解释器执行。由于命令解释器不使用 CommandLineToArgvW，因此脚本接收到的某些参数可能会带有额外的引号。

    版本 4.0 中变更： .bat 和 .cmd 脚本现在通过在命令行前添加 cmd /c call 来显式地通过命令解释器执行。在此之前，它们是通过 CreateProcessW 的未定义行为隐式地通过 cmd /c 执行的（不包含 call）。

使用 INPUT_*、OUTPUT_* 和 ERROR_* 选项来重定向标准输入 (stdin)、标准输出 (stdout) 和标准错误 (stderr)。

若要顺序执行多个命令，请多次调用 execute_process，每次调用带有一个 COMMAND 参数。

WORKING_DIRECTORY

指定的目录将被设为子进程的当前工作目录。如果未提供，子进程的工作目录是不确定的。

TIMEOUT

经过指定的秒数（允许小数）后，所有未完成的子进程将被终止，RESULT_VARIABLE 将被设置为一个包含 "timeout" 的字符串。

RESULT_VARIABLE

该变量将被设置为包含最后一个子进程的执行结果。这通常是最后一个子进程返回的整数代码，或描述错误条件的字符串。

RESULTS_VARIABLE <variable>

3.10 版本新增。

该变量将被设置为包含所有进程结果的 分号分隔列表，顺序与给定的 COMMAND 参数一致。每一项都是对应子进程返回的整数代码，或描述错误条件的字符串。

INPUT_FILE <file>

<file> 被连接到第一个 COMMAND 进程的标准输入管道。

OUTPUT_FILE <file>

<file> 被连接到最后一个 COMMAND 进程的标准输出管道。

ERROR_FILE <file>

<file> 被连接到所有 COMMAND 进程的标准错误管道。

版本 3.3 中新增： 如果为 OUTPUT_FILE 和 ERROR_FILE 指定了同一个 <file>，它将被同时用于标准输出和标准错误管道。

OUTPUT_QUIET, ERROR_QUIET

标准输出（对应的 OUTPUT_VARIABLE）或标准错误（对应的 ERROR_VARIABLE）将不会被连接（即变量无内容）。*_FILE 和 ECHO_*_VARIABLE 选项不受此影响。

OUTPUT_VARIABLE, ERROR_VARIABLE

指定的变量将被设置为标准输出和标准错误管道的内容。如果两个管道指定了同一个变量，它们的输出将按产生的顺序合并。

OUTPUT_STRIP_TRAILING_WHITESPACE, ERROR_STRIP_TRAILING_WHITESPACE

从存储在对应的 OUTPUT_VARIABLE 和 ERROR_VARIABLE 变量中的值末尾移除空白字符。

ECHO_OUTPUT_VARIABLE, ECHO_ERROR_VARIABLE

在 3.18 版本中新增。

标准输出或标准错误不会被排他性地重定向到指定的变量。

输出将被复制到指定的变量中，并同时输出到标准输出或标准错误，类似于 Unix 的 tee 命令。

注意

如果同一个管道给出了多个 OUTPUT_* 或 ERROR_* 选项，其优先级未定义。如果未给出任何 OUTPUT_* 或 ERROR_* 选项，输出将与 CMake 进程本身的相应管道共享。

COMMAND_ECHO <where>

版本 3.15 新增。

正在运行的命令将被回显到 <where>，其中 <where> 设置为 STDERR、STDOUT 或 NONE 之一。关于当此选项不存在时的默认行为控制，请参阅 CMAKE_EXECUTE_PROCESS_COMMAND_ECHO 变量。

ENCODING <name>

版本 3.8 新增。

在 Windows（微软桌面系统） 上，用于解码进程输出的编码方式。在其他平台上被忽略。有效的编码名称如下：

NONE

不执行任何解码。假定进程输出的编码方式与 CMake 内部编码（UTF-8）相同。

这是 CMake 3.14 及更早版本的默认设置。

AUTO

使用当前活动控制台的代码页；如果不可用，则使用 ANSI。

这是 CMake 3.15 到 3.30 的默认设置。

ANSI

使用 ANSI 代码页。

OEM

使用原始设备制造商 (OEM) 代码页。

UTF-8

3.11 版本新增。

使用 UTF-8 代码页。

这是自 CMake 3.31 以来的默认设置。请参阅策略 CMP0176。

UTF8

使用 UTF-8 代码页。建议优先使用 UTF-8，以符合 UTF-8 RFC 的命名规范。

COMMAND_ERROR_IS_FATAL <ANY|LAST|NONE>

3.19 版本新增。

COMMAND_ERROR_IS_FATAL 后面的选项决定了遇到错误时的行为：

ANY

ANY：如果命令列表中的任何命令失败，execute_process() 命令将停止并报错。

LAST

LAST：如果命令列表中的最后一个命令失败，execute_process() 命令将停止并报错。列表前部的命令失败不会导致致命错误。

NONE

4.0 版本新增。

NONE：无论是否有命令失败，execute_process() 命令都不会停止并报错。

版本 4.0 中新增： 如果未提供此选项，将检查 CMAKE_EXECUTE_PROCESS_COMMAND_ERROR_IS_FATAL 变量。如果该变量未设置，默认值为 NONE。如果提供了 RESULT_VARIABLE 或 RESULTS_VARIABLE，则 CMAKE_EXECUTE_PROCESS_COMMAND_ERROR_IS_FATAL 将被忽略。
