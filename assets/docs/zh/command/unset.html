unset¶

取消设置变量、缓存变量或环境变量。

取消设置普通变量¶

unset(<variable> [PARENT_SCOPE])¶

从当前作用域移除普通变量，使其变为未定义状态。

如果指定了 PARENT_SCOPE，则变量将从当前作用域的上一级作用域中移除。更多详情请参阅 set() 命令中的相同选项。

注意

在评估形式为 ${VAR} 的 变量引用 时，CMake 首先搜索同名的普通变量。如果不存在此类普通变量，CMake 随后将搜索同名的缓存条目。因此，取消设置普通变量可能会暴露之前被隐藏的缓存变量。若要强制形式为 ${VAR} 的变量引用返回空字符串，请使用 set(<variable> "")，这会清除普通变量的值，但保持其定义状态。

取消设置缓存条目¶

unset(CACHE{<variable>})¶

版本 4.2 中添加。

从缓存中移除 <variable>，使其变为未定义状态。

unset(<variable> CACHE)¶

此语法仅为兼容性目的而支持。建议优先使用另一种语法。

取消设置环境变量¶

unset(ENV{<variable>})¶

从当前可用的 环境变量 中移除 <variable>。后续调用 $ENV{<variable>} 将返回空字符串。

此命令仅影响当前的 CMake 进程，不会影响调用 CMake 的进程、系统全局环境，也不会影响后续构建或测试进程的环境。
