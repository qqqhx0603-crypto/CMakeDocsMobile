install_files¶

自 3.0 版本弃用： 请使用 install(FILES) 命令代替。

此命令已被 install() 命令取代。为了与旧的 CMake 代码兼容而提供。 FILES 形式直接被 install() 命令的 FILES 形式取代。正则表达式形式可以使用 file() 命令的 GLOB 形式更清晰地表达。

install_files(<dir> extension file file ...)

代码说明

  • install_files(<dir> extension file file ...)：调用 install_files() 命令，具体行为由这一行传入的参数决定。

创建规则，将列出的文件与给定的扩展名安装到给定的目录中。只能列出当前源代码树或二进制树中相应位置中存在的文件。如果指定的文件已经具有扩展名，则首先删除该扩展名。这对于提供诸如 foo.cxx 之类的源文件列表，以便安装相应的 foo.h 文件很有用。典型的扩展名是 .h。

install_files(<dir> regexp)

代码说明

  • install_files(<dir> regexp)：调用 install_files() 命令，具体行为由这一行传入的参数决定。

当前源目录中与正则表达式匹配的任何文件都将被安装。

install_files(<dir> FILES file file ...)

代码说明

  • install_files(<dir> FILES file file ...)：调用 install_files() 命令，具体行为由这一行传入的参数决定。

在 FILES 关键字之后列出的任何文件都将从给定的名称显式安装。这种形式允许使用完整路径。

目录 <dir> 相对于安装前缀，安装前缀存储在变量 CMAKE_INSTALL_PREFIX 中。
