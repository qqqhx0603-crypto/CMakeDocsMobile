list¶

对 分号分隔的列表 进行操作。

概要¶

Reading
  list(LENGTH <list> <out-var>)
  list(GET <list> <element index> [<index> ...] <out-var>)
  list(JOIN <list> <glue> <out-var>)
  list(SUBLIST <list> <begin> <length> <out-var>)

Search
  list(FIND <list> <value> <out-var>)

Modification
  list(APPEND <list> [<element>...])
  list(FILTER <list> {INCLUDE | EXCLUDE} REGEX <regex>)
  list(INSERT <list> <index> [<element>...])
  list(POP_BACK <list> [<out-var>...])
  list(POP_FRONT <list> [<out-var>...])
  list(PREPEND <list> [<element>...])
  list(REMOVE_ITEM <list> <value>...)
  list(REMOVE_AT <list> <index>...)
  list(REMOVE_DUPLICATES <list>)
  list(TRANSFORM <list> <ACTION> [...])

Ordering
  list(REVERSE <list>)
  list(SORT <list> [...])

简介¶

list 子命令 APPEND, INSERT, FILTER, PREPEND, POP_BACK, POP_FRONT, REMOVE_AT, REMOVE_ITEM, REMOVE_DUPLICATES, REVERSE 和 SORT 可能会在当前 CMake 变量作用域内创建列表的新值。与 set() 命令类似，list 命令会在当前作用域创建新的变量值，即使列表本身实际上是在父作用域中定义的。要将这些操作的结果向上传递，请使用带有 PARENT_SCOPE 的 set()，或者带有 CACHE INTERNAL 的 set()，也可以使用其他数值传播方式。

注意

CMake 中的列表是一组由 ; 分隔的字符串。可以使用 set() 命令创建列表。例如，set(var a b c d e) 创建了一个包含 a;b;c;d;e 的列表，而 set(var "a b c d e") 则创建了一个字符串或仅包含一个元素的列表。（请注意，宏参数不是变量，因此不能在 LIST 命令中使用。）

单个元素不能包含数量不等且未配对的 [ 和 ] 字符，且不能以反斜杠（\）结尾。详情请参见 分号分隔的列表。

注意

指定索引值时，如果 <element index> 为 0 或更大，则从列表开头开始索引，0 代表第一个元素。如果 <element index> 为 -1 或更小，则从列表末尾开始索引，-1 代表最后一个元素。使用负索引计数时需小心：它们不是从 0 开始的。-0 等同于 0，即第一个列表元素。

读取¶

list(LENGTH <list> <output variable>)¶

返回列表的长度。

list(GET <list> <element index> [<element index> ...] <output variable>)¶

返回列表中指定索引处的元素列表。

list(JOIN <list> <glue> <output variable>)¶

3.12 版本新增。

返回一个使用粘合字符串（glue string）连接所有列表元素的字符串。若要连接多个不属于列表的字符串，请使用 string(JOIN)。

list(SUBLIST <list> <begin> <length> <output variable>)¶

3.12 版本新增。

返回给定列表的子列表。如果 <length> 为 0，将返回空列表。如果 <length> 为 -1，或者列表小于 <begin>+<length>，则返回从 <begin> 开始的剩余列表元素。

搜索¶

list(FIND <list> <value> <output variable>)¶

返回列表中指定元素的索引，如果未找到，则返回 -1。

修改¶

list(APPEND <list> [<element> ...])¶

将元素追加到列表中。如果当前作用域中不存在名为 <list> 的变量，其值将被视为为空，元素会被追加到该空列表中。

list(FILTER <list> <INCLUDE|EXCLUDE> REGEX <regular_expression>)¶

3.6 版本新增。

根据模式包含或移除列表中的项目。在 REGEX 模式下，项目将与给定的正则表达式进行匹配。

有关正则表达式的更多信息，请参阅 string(REGEX)。

list(INSERT <list> <element_index> <element> [<element> ...])¶

在指定索引处插入元素到列表中。指定超出范围的索引是错误的。有效的索引为从 0 到 N（N 为列表长度，含 N）。空列表的长度为 0。如果当前作用域中不存在名为 <list> 的变量，其值将被视为为空，元素会被插入到该空列表中。

list(POP_BACK <list> [<out-var>...])¶

版本 3.15 新增。

如果不提供变量名，则正好移除一个元素。否则，提供 N 个变量名时，将最后 N 个元素的值赋给这些变量，然后从 <list> 中移除这最后 N 个值。

list(POP_FRONT <list> [<out-var>...])¶

版本 3.15 新增。

如果不提供变量名，则正好移除一个元素。否则，提供 N 个变量名时，将前 N 个元素的值赋给这些变量，然后从 <list> 中移除这前 N 个值。

list(PREPEND <list> [<element> ...])¶

版本 3.15 新增。

将元素插入到列表的第 0 位。如果当前作用域中不存在名为 <list> 的变量，其值将被视为为空，元素会被添加到该空列表的前面。

list(REMOVE_ITEM <list> <value> [<value> ...])¶

从列表中移除所有匹配指定项的实例。

list(REMOVE_AT <list> <index> [<index> ...])¶

从列表中移除指定索引处的项目。

list(REMOVE_DUPLICATES <list>)¶

移除列表中的重复项。列表项的相对顺序将被保留，如果遇到重复项，仅保留第一个实例。

list(TRANSFORM <list> <ACTION> [<SELECTOR>] [OUTPUT_VARIABLE <output variable>])¶

3.12 版本新增。

通过对列表中所有元素，或通过指定 <SELECTOR> 对选定元素应用 <ACTION> 来转换列表，并将结果就地存储或存储在指定的输出变量中。

注意

TRANSFORM 子命令不会改变列表中的元素数量。如果指定了 <SELECTOR>，则仅会更改部分元素，其他元素将保持转换前的状态。

<ACTION> 指定应用于列表元素的动作。这些动作与 string() 命令的子命令具有完全相同的语义。<ACTION> 必须是以下内容之一：

APPEND, PREPEND

向列表的每个元素追加或前置指定值。

list(TRANSFORM <list> (APPEND|PREPEND) <value> ...)¶
TOLOWER, TOUPPER

将列表的每个元素转换为小写或大写字符。

list(TRANSFORM <list> (TOLOWER|TOUPPER) ...)¶
STRIP

从列表的每个元素中移除首尾空格。

list(TRANSFORM <list> STRIP ...)¶
GENEX_STRIP

从列表的每个元素中去除所有 生成器表达式。

list(TRANSFORM <list> GENEX_STRIP ...)¶
REPLACE

尽可能多地匹配正则表达式，并为列表的每个元素替换匹配项（与 string(REGEX REPLACE) 语义相同）。

list(TRANSFORM <list> REPLACE <regular_expression> <replace_expression> ...)¶

版本 4.1 变更: ^ 锚点现在仅匹配输入元素的开头，而不是每次重复搜索的开头。请参阅策略 CMP0186。

<SELECTOR> 决定转换列表中的哪些元素。一次只能指定一种选择器。当使用时，<SELECTOR> 必须是以下内容之一：

AT

指定索引列表。

list(TRANSFORM <list> <ACTION> AT <index> [<index> ...] ...)
FOR

指定一个范围，并可选择性地使用增量来遍历该范围。

list(TRANSFORM <list> <ACTION> FOR <start> <stop> [<step>] ...)
REGEX

指定正则表达式。只有匹配该正则表达式的元素才会被转换。

list(TRANSFORM <list> <ACTION> REGEX <regular_expression> ...)

排序¶

list(REVERSE <list>)¶

就地反转列表内容。

list(SORT <list> [COMPARE <compare>] [CASE <case>] [ORDER <order>])¶

按字母顺序对列表进行就地排序。

版本 3.13 新增: 添加了 COMPARE, CASE 和 ORDER 选项。

版本 3.18 新增: 添加了 COMPARE NATURAL 选项。

使用 COMPARE 关键字选择排序的比较方法。<compare> 选项应为以下内容之一：

STRING

按字母顺序对字符串列表进行排序。如果未给出 COMPARE 选项，这是默认行为。

FILE_BASENAME

按文件名（basenames）对路径名列表进行排序。

NATURAL

使用自然顺序（请参阅 strverscmp(3) 手册）对字符串列表进行排序，即连续的数字被视为整体数字进行比较。例如：列表 10.0 1.1 2.1 8.0 2.0 3.1，如果选择 NATURAL 比较，将排序为 1.1 2.0 2.1 3.1 8.0 10.0；如果使用 STRING 比较，则排序为 1.1 10.0 2.0 2.1 3.1 8.0。

使用 CASE 关键字选择区分大小写或不区分大小写的排序模式。<case> 选项应为以下内容之一：

SENSITIVE

列表项区分大小写进行排序。如果未给出 CASE 选项，这是默认行为。

INSENSITIVE

列表项不区分大小写进行排序。仅大小写不同的项，其排序顺序未指定。

要控制排序顺序，可以使用 ORDER 关键字。<order> 选项应为以下内容之一：

ASCENDING

按升序排列列表。如果未给出 ORDER 选项，这是默认行为。

DESCENDING

按降序排列列表。
