source_group¶

定义在 IDE 项目生成中用于源文件的分组。创建源组有两种不同的签名。

source_group(<name> [FILES <src>...] [REGULAR_EXPRESSION <regex>])
source_group(TREE <root> [PREFIX <prefix>] [FILES <src>...])

代码说明

  • source_group(<name> [FILES <src>...] [REGULAR_EXPRESSION <regex>]) source_group(TREE <root> [PREFIX <prefix>] [FILES <src>...])：调用 `source_group()` 命令，具体行为由这一行传入的参数决定。

定义一个组，源文件将被放置到该组中，用于项目文件。这旨在设置 Visual Studio（集成开发环境） 中的文件选项卡。该组的作用域在调用命令的目录中，并适用于在该目录中创建的目标中的源文件。

选项包括

TREE

版本 3.8 新增。

CMake 将自动检测，从 <src> 文件路径中，需要创建的源组，以保持源组结构与项目中的实际文件和目录结构类似。 <src> 文件的路径将被裁剪为相对于 <root>。 如果 src 中的路径不以 root 开头，则命令将失败。

PREFIX

版本 3.8 新增。

位于 <root> 路径下的源组和文件，将被放置在 <prefix> 源组中。

FILES

任何明确指定的源文件都将被放置在组 <name> 中。相对路径相对于当前源目录进行解释。

版本 4.3 中添加：FILES 的参数可以使用 生成器 表达式。

REGULAR_EXPRESSION

任何名称与正则表达式匹配的源文件都将被放置在组 <name> 中。

如果一个源文件匹配多个组，则最后一个明确使用 FILES 列出该文件的组将被优先考虑（如果有）。 如果没有组明确列出该文件，则最后一个其正则表达式与该文件匹配的组将被优先考虑。

组的 <name> 和 <prefix> 参数可以包含正斜杠或反斜杠来指定子组。 反斜杠需要适当转义

source_group(base/subdir ...)
source_group(outer\\inner ...)
source_group(TREE <root> PREFIX sources\\inc ...)

代码说明

  • source_group(base/subdir ...) source_group(outer\\inner ...) source_group(TREE <root> PREFIX sources\\inc ...)：调用 `source_group()` 命令，具体行为由这一行传入的参数决定。

版本 3.18 中添加：允许使用正斜杠 (/) 来指定子组。

为了向后兼容，简写签名

source_group(<name> <regex>)

代码说明

  • source_group(<name> <regex>)：调用 `source_group()` 命令，具体行为由这一行传入的参数决定。

等价于

source_group(<name> REGULAR_EXPRESSION <regex>)

代码说明

  • source_group(<name> REGULAR_EXPRESSION <regex>)：调用 `source_group()` 命令，具体行为由这一行传入的参数决定。
