target_link_directories¶

3.13 版本新增。

向目标添加链接目录。

target_link_directories(<target> [BEFORE]
  {INTERFACE|PUBLIC|PRIVATE} <dir>...
  [{INTERFACE|PUBLIC|PRIVATE} <dir>...]...)

代码说明

  • target_link_directories(<target> [BEFORE]：这一行继续补充当前命令或表达式的内容。

  • {INTERFACE|PUBLIC|PRIVATE} <dir>...：这一行继续补充当前命令或表达式的内容。

  • [{INTERFACE|PUBLIC|PRIVATE} <dir>...]...)：这一行继续补充当前命令或表达式的内容。

指定链接器在链接给定目标时搜索库的路径。每一项可以是绝对路径或相对路径，后者将被解释为相对于当前源目录的路径。这些项将被添加到链接命令中。

名为 <target> 的目标必须是由 add_executable() 或 add_library() 等命令创建的目标，并且不能是 别名目标。

INTERFACE、PUBLIC 和 PRIVATE 关键字用于指定其后各项的作用域。PRIVATE 和 PUBLIC 项将填充 <target> 的 LINK_DIRECTORIES 属性。PUBLIC 和 INTERFACE 项将填充 <target> 的 INTERFACE_LINK_DIRECTORIES 属性（IMPORTED 目标仅支持 INTERFACE 项）。每一项都指定了一个链接目录，并在添加到相关属性之前，如有必要，将转换为绝对路径。对同一个 <target> 的重复调用将按调用顺序追加各项。

如果指定了 BEFORE，则内容将被预置（prepended）到相关属性中，而不是追加到末尾。

target_link_directories 的参数可以使用语法为 $<...> 的生成器表达式。有关可用表达式，请参阅 cmake-generator-expressions(7) 手册。有关定义构建系统属性的更多信息，请参阅 cmake-buildsystem(7) 手册。

注意

此命令很少需要使用，如果存在其他选择，应尽量避免使用。尽可能优先为库提供完整的绝对路径，因为这能确保始终链接到正确的库。find_library() 命令可以提供完整路径，该路径通常可以直接用于 target_link_libraries() 调用中。可能需要库搜索路径的情况包括：

  • 像 Xcode 这样的项目生成器，用户可以在构建时切换目标架构，但无法使用指向库的完整路径，因为它只提供单一架构（即它不是通用二进制文件）。

  • 库本身可能拥有其他私有库依赖项，这些依赖项期望通过 RPATH 机制被找到，但某些链接器无法完全解析这些路径（例如，由于存在像 $ORIGIN 这样的内容）。
