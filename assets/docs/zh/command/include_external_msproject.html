include_external_msproject¶

在由 Visual Studio（集成开发环境） 生成器 生成的解决方案文件中包含外部 Microsoft 项目文件。在其他生成器上会被忽略。

include_external_msproject(projectname location
                           [TYPE projectTypeGUID]
                           [GUID projectGUID]
                           [PLATFORM platformName]
                           dep1 dep2 ...)

代码说明

  • include_external_msproject(projectname location：这一行继续补充当前命令或表达式的内容。

  • [TYPE projectTypeGUID]：这一行继续补充当前命令或表达式的内容。

  • [GUID projectGUID]：这一行继续补充当前命令或表达式的内容。

  • [PLATFORM platformName]：这一行继续补充当前命令或表达式的内容。

  • dep1 dep2 ...)：这一行继续补充当前命令或表达式的内容。

将外部 Microsoft 项目包含到生成的解决方案文件中。这将创建一个名为 [projectname] 的目标。这可用于 add_dependencies() 命令，使其他内容依赖于外部项目。

TYPE、GUID 和 PLATFORM 是可选参数，允许指定项目类型、项目 ID（GUID）和目标平台名称。这对于需要其他于默认值（例如 WIX 项目）的项目非常有用。

3.9 版本新增： 如果导入的项目具有与当前项目不同的配置名称，请设置 MAP_IMPORTED_CONFIG_<CONFIG> 目标属性以指定映射。
