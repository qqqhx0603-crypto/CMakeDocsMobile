add_test¶

向项目添加一个由 ctest(1) 运行的测试。

add_test(NAME <name> COMMAND <command> [<arg>...]
         [CONFIGURATIONS <config>...]
         [WORKING_DIRECTORY <dir>]
         [COMMAND_EXPAND_LISTS])

代码说明

  • add_test(NAME <name> COMMAND <command> [<arg>...] [CONFIGURATIONS <config>...] [WORKING_DIRECTORY <dir>] [COMMAND_EXPAND_LISTS])：调用 `add_test()` 命令，登记一个测试用例。

添加一个名为 <name> 的测试。测试名称可以包含任意字符，必要时可表示为 引号参数 或 方括号参数。请参阅策略 CMP0110。

仅当已调用 enable_testing() 命令时，CMake 才会生成测试。CTest 模块会自动调用 enable_testing，除非 BUILD_TESTING 被设置为 OFF。

使用 add_test(NAME) 签名添加的测试，支持在通过 set_property(TEST) 或 set_tests_properties() 设置的测试属性中使用 生成器表达式。测试属性只能在创建测试所在的目录中设置。

add_test 选项如下：

COMMAND

指定测试命令行。

如果 <command> 指定了一个由 add_executable() 创建的可执行目标

  • 它将自动替换为构建时创建的可执行文件的位置。

  • 3.3 版本新增：目标的 CROSSCOMPILING_EMULATOR（如果已设置）将被用于在宿主机上运行该命令。

    <emulator> <command>

    代码说明

    • <emulator> <command>：这一行给出示例文本或输出，用来说明：<emulator> <command>

    3.29 版本变更：仅在 交叉编译 时使用模拟器。请参阅策略 CMP0158。

  • 3.29 版本新增：目标的 TEST_LAUNCHER（如果已设置）将被用于启动该命令。

    <launcher> <command>

    代码说明

    • <launcher> <command>：这一行给出示例文本或输出，用来说明：<launcher> <command>

    如果同时设置了 CROSSCOMPILING_EMULATOR，则两者都会被使用。

    <launcher> <emulator> <command>

    代码说明

    • <launcher> <emulator> <command>：这一行给出示例文本或输出，用来说明：<launcher> <emulator> <command>

可以使用 生成器表达式 来指定命令。

CONFIGURATIONS

仅在指定的配置中限制执行该测试。

WORKING_DIRECTORY

设置执行测试时所需的测试属性 WORKING_DIRECTORY。如果未指定，测试将在 CMAKE_CURRENT_BINARY_DIR 中运行。工作目录可以使用 生成器表达式 来指定。

COMMAND_EXPAND_LISTS

3.16 版新增。

COMMAND 参数中的列表将被展开，包括那些使用 生成器表达式 创建的列表。

如果测试命令以代码 0 退出，则测试通过。非零退出代码表示测试“失败”。测试属性 WILL_FAIL 可反转此逻辑。请注意，即使 WILL_FAIL 为真，系统级测试失败（如段错误或堆错误）仍会导致测试失败。写入 stdout 或 stderr 的输出由 ctest(1) 捕获，并且仅通过 PASS_REGULAR_EXPRESSION、FAIL_REGULAR_EXPRESSION 或 SKIP_REGULAR_EXPRESSION 测试属性来影响通过/失败状态。

3.16 版本新增：添加了 SKIP_REGULAR_EXPRESSION 属性。

用法示例

add_test(NAME mytest
         COMMAND testDriver --config $<CONFIG>
                            --exe $<TARGET_FILE:myexe>)

代码说明

  • add_test(NAME mytest COMMAND testDriver --config $<CONFIG> --exe $<TARGET_FILE:myexe>)：调用 `add_test()` 命令，登记一个测试用例。

这将创建一个名为 mytest 的测试，其命令运行一个 testDriver 工具，并将配置名称及目标 myexe 生成的可执行文件的完整路径传递给它。

推荐使用上述命令语法，而不是旧的、灵活性较差的形式：

add_test(<name> <command> [<arg>...])

代码说明

  • add_test(<name> <command> [<arg>...])：调用 `add_test()` 命令，登记一个测试用例。

添加一个名为 <name> 的测试，并附带给定的命令行。

与上述 NAME 签名不同，命令行中不支持目标名称。此外，使用此签名添加的测试不支持命令行或测试属性中的 生成器表达式，也不支持 TEST_LAUNCHER 和 CROSSCOMPILING_EMULATOR 目标属性。
