create_test_sourcelist¶

创建一个测试驱动程序，将许多小的测试链接到一个可执行文件中。当使用大型库构建静态可执行文件时，这对于减少所需的总大小很有用。

create_test_sourcelist(<sourceListName> <driverName> <test>... <options>...)¶

从一系列单独的测试源生成一个测试驱动程序源文件，并提供一个可以作为可执行文件构建的组合源列表。

选项包括

<sourceListName>

要在其中存储构建测试驱动程序所需的源文件列表的变量的名称。该列表将包含 <test>... 源和生成的 <driverName> 源。

版本 3.29 中更改：测试驱动程序源以构建树中的绝对路径列出。 之前仅作为 <driverName> 列出。

<driverName>

要生成到构建树中的测试驱动程序源文件的名称。源文件将包含一个 main() 程序入口点，该入口点将分派到命令行中命名的任何测试。

<test>...

要添加到驱动程序二进制文件的测试源文件。每个测试源文件必须在其内部具有与文件名相同（不包括扩展名）的函数。 例如，foo.cxx 测试源可能包含

int foo(int argc, char** argv)

代码说明

  • int foo(int argc, char** argv)：这一行定义或调用一个函数/方法，具体行为取决于参数和上下文；变量说明：foo：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

EXTRA_INCLUDE <header>

指定要 #include 在生成的测试驱动程序源中的头文件。

FUNCTION <function>

指定要使用指向 argc 和 argv 的指针调用的函数。该函数可以提供在 EXTRA_INCLUDE 头文件中

void function(int* pargc, char*** pargv)

代码说明

  • void function(int* pargc, char*** pargv)：这一行定义或调用一个函数/方法，具体行为取决于参数和上下文。

这可用于为每个测试添加额外的命令行处理。

此外，一些 CMake 变量会影响测试驱动程序的生成

CMAKE_TESTDRIVER_BEFORE_TESTMAIN¶

要在调用每个测试的函数之前放置的代码。

CMAKE_TESTDRIVER_AFTER_TESTMAIN¶

要在调用每个测试的函数之后放置的代码。
