define_property¶

定义并记录自定义属性。

define_property(<GLOBAL | DIRECTORY | TARGET | SOURCE |
                 TEST | VARIABLE | CACHED_VARIABLE>
                 PROPERTY <name> [INHERITED]
                 [BRIEF_DOCS <brief-doc> [<docs>...]]
                 [FULL_DOCS <full-doc> [<docs>...]]
                 [INITIALIZE_FROM_VARIABLE <variable>])

代码说明

  • define_property(<GLOBAL | DIRECTORY | TARGET | SOURCE | TEST | VARIABLE | CACHED_VARIABLE> PROPERTY <name> [INHERITED] [BRIEF_DOCS <brief-doc> [<docs>...]] [FULL_DOCS <full-doc> [<docs>...]] [INITIALIZE_FROM_VARIABLE <variable>])：调用 `define_property()` 命令，具体行为由这一行传入的参数决定。

在特定作用域内定义一个属性，供 set_property() 和 get_property() 命令使用。它主要用于定义属性的初始化或继承方式。历史上，该命令还用于关联属性的文档，但这已不再被视为主要用途。

第一个参数决定了属性适用的作用域类型。它必须是以下类型之一：

  • GLOBAL - 与全局命名空间相关联。

  • DIRECTORY - 与单个目录相关联。

  • TARGET - 与单个目标（target）相关联。

  • SOURCE - 与单个源文件相关联。

  • TEST - 与通过 add_test() 命令命名的测试相关联。

  • VARIABLE - 用于记录 CMake 语言变量。

  • CACHED_VARIABLE - 用于记录 CMake 缓存变量。

请注意，与 set_property() 和 get_property() 不同，此处无需提供具体的作用域实例；仅作用域类型很重要。

必需的 PROPERTY 选项后紧跟要定义的属性名称。

如果提供了 INHERITED 选项，则当请求的属性在命令给定的作用域中未设置时，get_property() 命令将向上链接到下一个更高层级的作用域。

  • DIRECTORY 作用域会链接到其父目录的作用域，并继续向上遍历父目录，直到某个目录设置了该属性或没有更多父目录为止。如果顶层目录中仍未找到，它会链接到 GLOBAL 作用域。

  • TARGET、SOURCE 和 TEST 属性会链接到 DIRECTORY 作用域，并根据需要进一步向上链接目录等。

请注意，这种作用域链行为仅适用于对 get_property()、get_directory_property()、get_target_property()、get_source_file_property() 和 get_test_property() 的调用。在“设置”属性时不存在继承行为，因此使用 set_property() 命令的 APPEND 或 APPEND_STRING 选项时，在计算要追加的内容时不会考虑继承的值。

BRIEF_DOCS 和 FULL_DOCS 选项后跟字符串，这些字符串将作为属性的简要和详细文档关联起来。CMake 除了通过 get_property() 命令的相关选项使项目能够获取这些文档外，并不会使用它们。

3.23 版本更改: BRIEF_DOCS 和 FULL_DOCS 选项现在是可选的。

3.23 版本新增: INITIALIZE_FROM_VARIABLE 选项指定了一个变量，属性应从该变量初始化。它只能与目标（target）属性一起使用。<variable> 名称必须以属性名结尾，且不得以 CMAKE_ 或 _CMAKE_ 开头。属性名必须包含至少一个下划线。建议属性名带有项目特定的前缀。

属性重定义¶

一旦为特定类型的作用域定义了属性，它就不能被重定义。尝试通过调用 define_property() 并使用相同的作用域类型和属性名称来重定义现有属性，将会被静默忽略。为两种不同的作用域类型定义相同的属性名称是合法的。

get_property() 可用于确定某个属性是否已为特定作用域类型定义，如果已定义，则可以检查其定义。例如：

# Initial definition
define_property(TARGET PROPERTY MY_NEW_PROP
  BRIEF_DOCS "My new custom property"
)

# Later examination
get_property(my_new_prop_exists
  TARGET NONE
  PROPERTY MY_NEW_PROP
  DEFINED
)

if(my_new_prop_exists)
  get_property(my_new_prop_docs
    TARGET NONE
    PROPERTY MY_NEW_PROP
    BRIEF_DOCS
  )
  # ${my_new_prop_docs} is now set to "My new custom property"
endif()
