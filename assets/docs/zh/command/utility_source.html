utility_source¶

自版本 3.0 起已弃用。请参阅 CMake 策略 CMP0034。

指定第三方工具的源代码树。

utility_source(cache_entry executable_name
               path_to_source [file1 file2 ...])

代码说明

  • utility_source(cache_entry executable_name：这一行继续补充当前命令或表达式的内容。

  • path_to_source [file1 file2 ...])：这一行继续补充当前命令或表达式的内容。

当第三方工具的源代码包含在分发中时，此命令指定其位置和名称。除非 path_to_source 和所有列出的文件存在，否则缓存条目将不会被设置。假定该工具的源代码树将在需要之前构建完成。

当进行交叉编译时，如果执行了 utility_source() 命令，CMake 将会打印一个警告，因为在许多情况下，它被用来构建一个稍后在目标平台上执行的可执行文件。这在交叉编译时不起作用，因为可执行文件只能在其目标平台上运行。因此，在这种情况下，必须手动调整缓存条目，使其指向可在构建主机上运行的可执行文件。
