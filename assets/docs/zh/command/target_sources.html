target_sources¶

版本 3.1 中新增。

向目标添加源文件。

target_sources(<target>
  {INTERFACE|PUBLIC|PRIVATE} <source>...
  [{INTERFACE|PUBLIC|PRIVATE} <source>...]...)

代码说明

  • target_sources(<target> {INTERFACE|PUBLIC|PRIVATE} <source>... [{INTERFACE|PUBLIC|PRIVATE} <source>...]...)：调用 `target_sources()` 命令，为目标补充源文件列表。

指定构建目标和/或其依赖项时要使用的源文件。指定的 <target> 必须是由诸如 add_executable()、add_library() 或 add_custom_target() 等命令创建的，且不能是 别名目标 (ALIAS target（别名目标）)。 <items> 可以使用 生成器表达式 (generator expressions（生成器表达式）)。

3.20 版本新增： <target> 可以是自定义目标。

INTERFACE、PUBLIC 和 PRIVATE 关键字用于指定其后紧跟的源文件路径（<items>）的 作用域。PRIVATE 和 PUBLIC 项将填充 <target> 的 SOURCES 属性，这些属性在构建目标本身时使用。PUBLIC 和 INTERFACE 项将填充 <target> 的 INTERFACE_SOURCES 属性，这些属性在构建依赖项时使用。由 add_custom_target() 创建的目标只能具有 PRIVATE 作用域。

对同一个 <target> 的重复调用会按调用顺序追加项目。

3.3 版本新增： 允许导出带有 INTERFACE_SOURCES 的目标。

3.11 版本新增: 允许在 导入目标 上设置 INTERFACE 项。

3.13 版本变更： 相对源文件路径被解释为相对于当前源目录（即 CMAKE_CURRENT_SOURCE_DIR）。参见策略 CMP0076。

以生成器表达式开头的路径保持不变。当目标的 SOURCE_DIR 属性与 CMAKE_CURRENT_SOURCE_DIR 不同时，请在生成器表达式中使用绝对路径，以确保源文件正确分配给目标。

# WRONG: starts with generator expression, but relative path used
target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:dbgsrc.cpp>")

# CORRECT: absolute path used inside the generator expression
target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:${CMAKE_CURRENT_SOURCE_DIR}/dbgsrc.cpp>")

代码说明

  • # WRONG: starts with generator expression, but relative path used target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:dbgsrc.cpp>") # CORRECT: absolute path used inside the generator expression target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:${CMAKE_CURRENT_SOURCE_DIR}/dbgsrc.cpp>")：这是注释，用来说明：WRONG: starts with generator expression, but relative path used target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:dbgsrc.cpp>") # CORRECT: absolute path used inside the generator expression target_sources(MyTarget PRIVATE "$<$<CONFIG:Debug>:${CMAKE_CURRENT_SOURCE_DIR}/dbgsrc.cpp>")。

有关定义构建系统属性的更多信息，请参阅 cmake-buildsystem(7) 手册。

文件集（File Sets（文件集））¶

在版本 3.23 中添加。

target_sources(<target>
  [{INTERFACE|PUBLIC|PRIVATE}
   [FILE_SET <set> [TYPE <type>] [BASE_DIRS <dirs>...] [FILES <files>...]]...
  ]...)

代码说明

  • target_sources(<target> [{INTERFACE|PUBLIC|PRIVATE} [FILE_SET <set> [TYPE <type>] [BASE_DIRS <dirs>...] [FILES <files>...]]... ]...)：调用 `target_sources()` 命令，为目标补充源文件列表。

向目标添加文件集，或将文件添加到现有文件集。目标可以包含零个或多个命名文件集。每个文件集都有一个名称、一个类型、一个 INTERFACE、PUBLIC 或 PRIVATE 作用域、一个或多个基础目录，以及这些目录中的文件。可接受的类型包括：

HEADERS

打算通过语言的 #include 机制使用的源文件。

CXX_MODULES

版本 3.28 新增。

包含 C++ 接口模块或分区单元的源文件（即那些使用 export 关键字的文件）。此文件集类型在 IMPORTED 目标之外不能具有 INTERFACE 作用域。

可选的默认文件集以其类型命名。该目标不能是自定义目标或 FRAMEWORK 目标。

PRIVATE 或 PUBLIC 文件集中的文件在 IDE 集成中被标记为源文件。此外，HEADERS 文件集中的文件将其 HEADER_FILE_ONLY 属性设置为 TRUE。INTERFACE 或 PUBLIC 文件集中的文件可以通过 install(TARGETS) 命令进行安装，并使用 install(EXPORT) 和 export() 命令进行导出。

每个 target_sources(FILE_SET) 条目均以 INTERFACE、PUBLIC 或 PRIVATE 开头，并接受以下参数：

FILE_SET <set>

要创建或添加到其中的文件集名称。它必须仅包含字母、数字和下划线。以大写字母开头的名称保留给 CMake 预定义的内置文件集。唯一预定义的文件集名称是那些匹配可接受类型名称的名称。所有其他文件集名称不能以大写字母或下划线开头。

TYPE <type>

每个文件集都与特定类型的文件相关联。只能使用上述指定的类型，指定其他任何内容都是错误的。作为一种特殊情况，如果文件集名称是类型名称之一，则无需指定类型，并且可以省略 TYPE <type> 参数。对于所有其他文件集名称，TYPE 是必需的。

BASE_DIRS <dirs>...

文件集基础目录的可选列表。任何相对路径均被视为相对于当前源目录（即 CMAKE_CURRENT_SOURCE_DIR）。如果在首次创建文件集时未指定 BASE_DIRS，则会添加 CMAKE_CURRENT_SOURCE_DIR 的值。此参数支持 生成器表达式。

文件集的任何两个基础目录都不能互为子目录。此要求必须在添加到文件集的所有基础目录中满足，而不仅仅是在对 target_sources() 的单次调用中。

FILES <files>...

要添加到文件集中的文件的可选列表。每个文件必须位于其中一个基础目录中，或者是其中一个基础目录的子目录中。此参数支持 生成器表达式。

如果指定了相对路径，则它们被视为相对于调用 target_sources() 时的 CMAKE_CURRENT_SOURCE_DIR。以 $< 开头的路径是一个例外。此类路径在评估生成器表达式后，被视为相对于目标的源目录。

以下目标属性由 target_sources(FILE_SET) 设置，但通常不应直接操作它们：

对于类型为 HEADERS 的文件集

  • HEADER_SETS

  • INTERFACE_HEADER_SETS

  • HEADER_SET

  • HEADER_SET_<NAME>

  • HEADER_DIRS

  • HEADER_DIRS_<NAME>

对于类型为 CXX_MODULES 的文件集

  • CXX_MODULE_SETS

  • INTERFACE_CXX_MODULE_SETS

  • CXX_MODULE_SET

  • CXX_MODULE_SET_<NAME>

  • CXX_MODULE_DIRS

  • CXX_MODULE_DIRS_<NAME>

target_sources(FILE_SET) 还会修改与包含目录相关的目标属性，如下所示：

INCLUDE_DIRECTORIES

如果 TYPE 为 HEADERS，且文件集的作用域为 PRIVATE 或 PUBLIC，则文件集的所有 BASE_DIRS 将被包装在 $<BUILD_INTERFACE> 中并追加到此属性。

INTERFACE_INCLUDE_DIRECTORIES

如果 TYPE 为 HEADERS，且文件集的作用域为 INTERFACE 或 PUBLIC，则文件集的所有 BASE_DIRS 将被包装在 $<BUILD_INTERFACE> 中并追加到此属性。
