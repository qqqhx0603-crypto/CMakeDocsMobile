get_test_property¶

获取测试的属性。

get_test_property(<test> <property> [DIRECTORY <dir>] <variable>)

代码说明

  • get_test_property(<test> <property> [DIRECTORY <dir>] <variable>)：调用 get_test_property() 命令，具体行为由这一行传入的参数决定。

从测试中获取一个属性。该属性的值存储在指定的 <variable> 中。如果 <test> 未定义，或者测试属性未找到，<variable> 将被设置为 NOTFOUND。如果测试属性被定义为 INHERITED 属性（参见 define_property()），搜索将包括相关的父作用域，如 define_property() 命令中所述。

要获取标准属性列表，可以输入 cmake --help-property-list。

3.28 版本新增： 可以使用以下子选项覆盖目录作用域

DIRECTORY <dir>

测试属性将从 <dir> 目录的作用域中读取。CMake 必须已经知道该源目录，要么是通过调用 add_subdirectory() 添加的，要么 <dir> 是顶级源目录。相对路径被视为相对于当前源目录。 <dir> 可以引用一个构建目录。
