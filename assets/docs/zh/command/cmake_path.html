cmake_path¶

在 3.20 版本中添加。

该命令用于路径操作。它仅处理路径的语法层面，与任何底层文件系统均无交互。路径可以表示一个不存在的路径，甚至是当前文件系统或平台不允许存在的路径。若需与文件系统进行交互的操作，请参见 file() 命令。

注意

cmake_path 命令以构建系统（即宿主平台）的格式处理路径，而非目标系统。在交叉编译时，如果路径包含宿主平台上无法表示的元素（例如宿主不是 Windows（微软桌面系统） 时的驱动器号），结果将不可预测。

约定¶

本文档中使用了以下约定：

<path-var>

始终为变量名。对于期望 <path-var> 作为输入的命令，该变量必须存在，且预期持有一个单一路径。

<input>

字符串字面量，可能包含路径、路径片段，或者根据命令不同包含带有特殊分隔符的多个路径。有关如何解释该参数，请参见各命令的说明。

<input>...

零个或多个字符串字面量参数。

<out-var>

用于存储命令执行结果的变量名。

路径结构与术语¶

路径具有以下结构（所有组件均为可选，且存在一定约束）：

root-name root-directory-separator (item-name directory-separator)* filename

代码说明

  • root-name root-directory-separator (item-name directory-separator)* filename：这一行给出示例文本或输出，用来说明：root-name root-directory-separator (item-name directory-separator)* filename

root-name

标识具有多个根目录的文件系统中的根（例如 "C:" 或 "//myserver"）。它是可选的。

root-directory-separator

目录分隔符，如果存在，表示此路径为绝对路径。如果缺失且 root-name 之后的第一个元素是 item-name，则路径为相对路径。

item-name

非目录分隔符的字符序列。该名称可以标识文件、硬链接、符号链接或目录。识别出两种特殊情况：

  • 由单个点字符 . 组成的项名称是一个目录名，指代当前目录。

  • 由两个点字符 .. 组成的项名称是一个目录名，指代父目录。

上述 (...)* 模式表示可以有零个或多个项名称，多个项由 directory-separator 分隔。()* 字符不属于路径本身。

directory-separator

唯一被识别的目录分隔符是正斜杠 /。如果该字符重复，则被视为单个目录分隔符。换句话说，/usr///////lib 与 /usr/lib 相同。

filename

如果路径不以 directory-separator 结尾，则该路径有一个 filename。filename 实际上是路径的最后一个 item-name，因此它也可以是硬链接、符号链接或目录。

filename 可以有扩展名 (extension)。默认情况下，扩展名定义为从最左侧的句点（包含该句点）开始直到 filename 末尾的子字符串。在接受 LAST_ONLY 关键字的命令中，LAST_ONLY 将解析方式更改为从最右侧的句点开始的子字符串。

对于上述解释存在以下例外情况：

  • 如果 filename 的第一个字符是句点，则该句点被忽略（即像 ".profile" 这样的 filename 被视为没有扩展名）。

  • 如果 filename 为 . 或 ..，则它没有扩展名。

词干 (stem) 是 filename 中扩展名之前的部分。

某些命令引用 root-path。这是 root-name 和 root-directory-separator 的连接，其中一个或两个都可以为空。relative-part 指的是移除了 root-path 后的完整路径。

创建路径变量¶

虽然可以使用普通的 set() 命令小心地创建路径，但建议改用 cmake_path(SET)，因为它会自动将路径转换为所需形式。cmake_path(APPEND) 子命令在需要通过连接片段构建路径时是另一个合适的选择。以下示例比较了构建同一路径的三种方法。

set(path1 "${CMAKE_CURRENT_SOURCE_DIR}/data")

cmake_path(SET path2 "${CMAKE_CURRENT_SOURCE_DIR}/data")

cmake_path(APPEND path3 "${CMAKE_CURRENT_SOURCE_DIR}" "data")

代码说明

  • set(path1 "${CMAKE_CURRENT_SOURCE_DIR}/data")：调用 set() 命令，把变量 path1 设为给定的值；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录；path1：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • cmake_path(SET path2 "${CMAKE_CURRENT_SOURCE_DIR}/data")：调用 cmake_path() 命令，具体行为由这一行传入的参数决定；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • cmake_path(APPEND path3 "${CMAKE_CURRENT_SOURCE_DIR}" "data")：调用 cmake_path() 命令，具体行为由这一行传入的参数决定；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

修改和生成子命令既可以将结果存储在原处，也可以存储在由 OUTPUT_VARIABLE 关键字命名的单独变量中。所有其他子命令必须将结果存储在必需的 <out-var> 变量中。

注意

cmake_path 命令可以直接用于 CMake 提供的代表路径的变量（例如 CMAKE_SOURCE_DIR）。但是，由于该命令可能会修改这些变量，而许多变量在被修改后具有未定义的行为，因此它们绝不应指定为 <out-var>；仅当指定了单独的 <out-var> 时，才可将其指定为 <path-var>。有关 CMake 保留的变量和标识符列表，请参见 cmake-variables(7)。

规范化¶

某些子命令支持规范化 (normalizing)路径。规范化路径的算法如下：

  1. 如果路径为空，则停止（空路径的规范化形式也是空路径）。

  2. 将每个 directory-separator（可能由多个分隔符组成）替换为单个 /（/a///b --> /a/b）。

  3. 移除每个单独的点字符 (.) 以及任何紧随其后的 directory-separator（/a/./b/. --> /a/b）。

  4. 移除每个紧随 directory-separator 和 .. 的 item-name（非 ..），连同任何紧随其后的 directory-separator（/a/b/../c --> a/c）。

  5. 如果有 root-directory，则移除任何 .. 以及其后紧跟的任何 directory-separators。根目录的父目录仍被视为根目录（/../a --> /a）。

  6. 如果最后一个 item-name 是 ..，则移除任何末尾的 directory-separator（../ --> ..）。

  7. 如果此时路径为空，则添加一个 dot（./ 的标准形式为 .）。

分解¶

GET 子命令的以下形式分别从路径中检索不同的组件或组件组。有关每个路径组件的含义，请参见路径结构与术语。

cmake_path(GET <path-var> ROOT_NAME <out-var>)¶
cmake_path(GET <path-var> ROOT_DIRECTORY <out-var>)¶
cmake_path(GET <path-var> ROOT_PATH <out-var>)¶
cmake_path(GET <path-var> FILENAME <out-var>)¶
cmake_path(GET <path-var> EXTENSION [LAST_ONLY] <out-var>)¶
cmake_path(GET <path-var> STEM [LAST_ONLY] <out-var>)¶
cmake_path(GET <path-var> RELATIVE_PART <out-var>)¶
cmake_path(GET <path-var> PARENT_PATH <out-var>)¶

如果路径中不存在请求的组件，<out-var> 中将存储一个空字符串。例如，只有 Windows（微软桌面系统） 系统具有 root-name 的概念，因此当宿主机器非 Windows（微软桌面系统） 时，ROOT_NAME 子命令将始终返回空字符串。

对于 PARENT_PATH，如果 HAS_RELATIVE_PART 子命令返回 false，则结果是 <path-var> 的副本。注意，这意味着根目录被视为具有父目录，该父目录即其自身。当 HAS_RELATIVE_PART 返回 true 时，结果本质上是少了一个元素的 <path-var>。

查询¶

每个 GET 子命令都有一个对应的 HAS_... 子命令，可用于查询某个特定路径组件是否存在。有关每个路径组件的含义，请参见路径结构与术语。

cmake_path(HAS_ROOT_NAME <path-var> <out-var>)¶
cmake_path(HAS_ROOT_DIRECTORY <path-var> <out-var>)¶
cmake_path(HAS_ROOT_PATH <path-var> <out-var>)¶
cmake_path(HAS_FILENAME <path-var> <out-var>)¶
cmake_path(HAS_EXTENSION <path-var> <out-var>)¶
cmake_path(HAS_STEM <path-var> <out-var>)¶
cmake_path(HAS_RELATIVE_PART <path-var> <out-var>)¶
cmake_path(HAS_PARENT_PATH <path-var> <out-var>)¶

上述每个子命令都遵循可预测的模式：如果路径具有关联的组件，则将 <out-var> 设置为 true，否则设置为 false。请注意以下特殊情况：

  • 对于 HAS_ROOT_PATH，仅当 root-name 或 root-directory 中至少有一个非空时，才会返回 true。

  • 对于 HAS_PARENT_PATH，根目录也被视为具有父目录，即其自身。除路径仅包含一个 filename 的情况外，结果均为 true。

cmake_path(IS_ABSOLUTE <path-var> <out-var>)¶

如果 <path-var> 是绝对路径，则将 <out-var> 设置为 true。绝对路径是不需要参照起始位置即可明确标识文件位置的路径。在 Windows（微软桌面系统） 上，这意味着路径必须同时具有 root-name 和 root-directory-separator 才被视为绝对路径。在其他平台上，仅有 root-directory-separator 就足够了。请注意，这意味着在 Windows（微软桌面系统） 上，IS_ABSOLUTE 可能为 false，而 HAS_ROOT_DIRECTORY 可能为 true。

cmake_path(IS_RELATIVE <path-var> <out-var>)¶

这将在 <out-var> 中存储与 IS_ABSOLUTE 相反的结果。

cmake_path(IS_PREFIX <path-var> <input> [NORMALIZE] <out-var>)¶

检查 <path-var> 是否为 <input> 的前缀。

当指定 NORMALIZE 选项时，<path-var> 和 <input> 会在检查前进行规范化。

cmake_path(IS_PREFIX path "/a/b/c/d" result) # result = true
cmake_path(IS_PREFIX path "/a/b" result)     # result = false
cmake_path(IS_PREFIX path "/x/y/z" result)   # result = false

set(path "/a/b")

cmake_path(REMOVE_FILENAME <path-var> [OUTPUT_VARIABLE <out-var>])¶

message("Second path is \"${path}\"")

生成¶

cmake_path(NORMAL_PATH <path-var> [OUTPUT_VARIABLE <out-var>])¶

根据 规范化 中描述的步骤对 <path-var> 进行规范化。

有关输出变量的详细信息，请参见创建路径变量。

cmake_path(RELATIVE_PATH <path-var> [BASE_DIRECTORY <input>] [OUTPUT_VARIABLE <out-var>])¶

修改 <path-var>，使其相对于 BASE_DIRECTORY 参数。如果未指定 BASE_DIRECTORY，则默认基目录将为 CMAKE_CURRENT_SOURCE_DIR。

作为参考，用于计算相对路径的算法与 C++ std::filesystem::path::lexically_relative 使用的算法相同。

有关输出变量的详细信息，请参见创建路径变量。

cmake_path(ABSOLUTE_PATH <path-var> [BASE_DIRECTORY <input>] [NORMALIZE] [OUTPUT_VARIABLE <out-var>])¶

如果 <path-var> 是相对路径（IS_RELATIVE 为 true），则相对于由 BASE_DIRECTORY 选项指定的给定基目录进行计算。如果未指定 BASE_DIRECTORY，则默认基目录将为 CMAKE_CURRENT_SOURCE_DIR。

当指定 NORMALIZE 选项时，路径将在路径计算后进行规范化。

因为 cmake_path() 不访问文件系统，所以符号链接不会被解析，且任何前导波浪号也不会被展开。若要计算解析了符号链接和展开了前导波浪号的真实路径，请改用 file(REAL_PATH) 命令。

有关输出变量的详细信息，请参见创建路径变量。

原生转换¶

对于本节中的命令，原生 (native) 指的是宿主平台，而不是交叉编译时的目标平台。

cmake_path(NATIVE_PATH <path-var> [NORMALIZE] <out-var>)¶

将 CMake 风格的 <path-var> 转换为带有平台特定斜杠的原生路径（Windows（微软桌面系统） 宿主为 \，其他系统为 /）。

当指定 NORMALIZE 选项时，路径将在转换前进行规范化。

有关输出变量的详细信息，请参见创建路径变量。

cmake_path(CONVERT <input> TO_CMAKE_PATH_LIST <out-var> [NORMALIZE])¶

将原生 <input> 路径转换为带有正斜杠 (/) 的 CMake 风格路径。

在 Windows（微软桌面系统） 宿主上，会考虑长文件名标记。输入可以是单个路径或像 $ENV{PATH} 这样的系统搜索路径。搜索路径将被转换为分号分隔的列表（在非 Windows（微软桌面系统） 平台上，这本质上意味着 : 分隔符被替换为 ;）。

当指定 NORMALIZE 选项时，路径将在转换前进行规范化。

有关输出变量的详细信息，请参见创建路径变量。

注意

与大多数其他 cmake_path() 子命令不同，CONVERT 子命令将字面字符串作为输入，而不是变量名。

cmake_path(CONVERT <input> TO_NATIVE_PATH_LIST <out-var> [NORMALIZE])¶

将 CMake 风格的 <input> 路径转换为带有平台特定斜杠的原生路径（Windows（微软桌面系统） 宿主为 \，其他系统为 /）。

输入可以是单个路径或分号分隔的列表。列表将被转换为原生搜索路径（Windows（微软桌面系统） 上以 ; 分隔，其他平台上以 : 分隔）。

当指定 NORMALIZE 选项时，路径将在转换前进行规范化。

有关输出变量的详细信息，请参见创建路径变量。

注意

与大多数其他 cmake_path() 子命令不同，CONVERT 子命令将字面字符串作为输入，而不是变量名。

例如

set(paths "/a/b/c" "/x/y/z")
cmake_path(CONVERT "${paths}" TO_NATIVE_PATH_LIST native_paths)
message("Native path list is \"${native_paths}\"")

  • cmake_path(CONVERT "${paths}" TO_NATIVE_PATH_LIST native_paths)：调用 cmake_path() 命令，具体行为由这一行传入的参数决定；变量说明：paths：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • message("Native path list is \"${native_paths}\"")：调用 message() 输出提示信息，方便观察脚本执行结果；变量说明：native_paths：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

哈希¶

cmake_path(HASH <path-var> <out-var>)¶

计算 <path-var> 的哈希值，使得对于两个比较相等的路径 p1 和 p2（COMPARE ... EQUAL），p1 的哈希值等于 p2 的哈希值。路径在计算哈希值之前总是会进行规范化。

有关输出变量的详细信息，请参见创建路径变量。
