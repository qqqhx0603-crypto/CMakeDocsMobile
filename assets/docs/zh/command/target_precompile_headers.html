target_precompile_headers¶

3.16 版新增。

添加一个需要预编译的头文件列表。

预编译头文件可以通过创建某些头文件的部分处理版本，并在编译过程中使用该版本，而不是重复解析原始头文件，从而加快编译速度。

主要形式¶

target_precompile_headers(<target>
  {INTERFACE|PUBLIC|PRIVATE} <header>...
  [{INTERFACE|PUBLIC|PRIVATE} <header>...]...)

该命令将头文件添加到 <target> 的 PRECOMPILE_HEADERS 和/或 INTERFACE_PRECOMPILE_HEADERS 目标属性中。指定的 <target> 必须是由 add_executable() 或 add_library() 等命令创建的，且不能是 别名目标 (ALIAS target)。

INTERFACE、PUBLIC 和 PRIVATE 关键字用于指定后续参数的 作用域。PRIVATE 和 PUBLIC 项将填充 <target> 的 PRECOMPILE_HEADERS 属性。PUBLIC 和 INTERFACE 项将填充 <target> 的 INTERFACE_PRECOMPILE_HEADERS 属性（导入目标 (IMPORTED targets) 仅支持 INTERFACE 项）。针对同一个 <target> 的重复调用将按调用顺序追加项目。

项目通常应避免在将被 导出 (exported) 的目标上使用 PUBLIC 或 INTERFACE，或者至少应该使用 $<BUILD_INTERFACE:...> 生成器表达式，以防止预编译头出现在已安装的导出目标中。目标的使用者通常应自行控制他们使用的预编译头，而不应被所使用的目标强加（因为预编译头通常不是使用要求）。一个值得注意的例外是，为了在一个地方定义一组常用的预编译头而创建了一个 接口库 (interface library)，然后其他目标私有地链接到该接口库。在这种情况下，接口库的存在专门用于将预编译头传播给其使用者，而使用者实际上仍然处于控制之中，因为它决定是否链接到该接口库。

头文件列表用于生成一个名为 cmake_pch.h|xx 的头文件，该文件用于生成预编译头文件（.pch, .gch, .pchi）制品。cmake_pch.h|xx 头文件将被强制包含（GCC 使用 -include，MSVC 使用 /FI）到所有源文件中，因此源文件不需要包含 #include "pch.h"。

以尖括号（例如 <unordered_map>）或显式双引号（针对 cmake-language(7) 转义，例如 [["other_header.h"]]）指定的头文件名将按原样处理，且必须在包含目录中以供编译器查找。其他头文件名（例如 project_header.h）被解释为相对于当前源目录（例如 CMAKE_CURRENT_SOURCE_DIR），并将按绝对路径包含。例如

target_precompile_headers(myTarget
  PUBLIC
    project_header.h
  PRIVATE
    [["other_header.h"]]
    <unordered_map>
)

target_precompile_headers 的参数可以使用语法为 $<...> 的生成器表达式。请参阅 cmake-generator-expressions(7) 手册以获取可用表达式。$<COMPILE_LANGUAGE:...> 生成器表达式对于指定仅针对一种语言预编译的特定语言头文件（例如 CXX 而非 C）特别有用。在这种情况下，未显式使用双引号或尖括号的头文件名必须通过绝对路径指定。此外，在生成器表达式中使用尖括号时，请务必将结尾的 > 编码为 $<ANGLE-R>。例如

target_precompile_headers(mylib PRIVATE
  "$<$<COMPILE_LANGUAGE:CXX>:${CMAKE_CURRENT_SOURCE_DIR}/cxx_only.h>"
  "$<$<COMPILE_LANGUAGE:C>:<stddef.h$<ANGLE-R>>"
  "$<$<COMPILE_LANGUAGE:CXX>:<cstddef$<ANGLE-R>>"
)

重用预编译头¶

该命令还支持第二种形式，可用于指定一个目标重用来自另一个目标的预编译头文件制品，而不是生成自己的制品。

target_precompile_headers(<target> REUSE_FROM <other_target>)

这种形式会将 PRECOMPILE_HEADERS_REUSE_FROM 属性设置为 <other_target>，并添加一个依赖项，使得 <target> 依赖于 <other_target>。如果在执行 REUSE_FROM 形式时 <target> 的 PRECOMPILE_HEADERS 属性已被设置，CMake 将报错并停止。

注意

REUSE_FROM 形式要求 <target> 和 <other_target> 具有相同的编译器选项、编译器标志和编译器定义集。某些编译器（例如 GCC）如果无法使用预编译头文件，可能会发出警告（-Winvalid-pch）。

另请参阅¶

  • 要禁用特定目标的预编译头，请参阅 DISABLE_PRECOMPILE_HEADERS 目标属性。

  • 要防止在编译特定源文件时使用预编译头，请参阅 SKIP_PRECOMPILE_HEADERS 源文件属性。
