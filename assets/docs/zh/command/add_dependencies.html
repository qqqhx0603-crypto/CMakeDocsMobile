add_dependencies¶

在顶级目标之间添加依赖关系。

add_dependencies(<target> <target-dependency>...)

使一个顶级 <target> 依赖于其他顶级目标，以确保它们在 <target> 之前构建。顶级目标是指由以下命令之一创建的目标：add_executable()、add_library() 或 add_custom_target() 命令（但不是 CMake 生成的目标，例如 install）。

添加到 导入目标 或 接口库 的依赖关系将在其位置上被传递跟踪，因为目标本身不会构建。

3.3 版本新增：允许将依赖关系添加到接口库。

3.8 版本新增：依赖关系将填充 <target> 的 MANUALLY_ADDED_DEPENDENCIES 属性。

3.9 版本变更：Ninja 生成器 为了提高可用并发性，使用了比其他生成器更弱的排序。它们仅保证在 <target> 中的源代码开始编译之前，依赖关系的自定义命令已完成；这确保了生成的源代码可用。

4.1 版本变更：该命令可以不带任何依赖关系被调用。 之前，至少需要一个依赖关系。

另请参阅¶

  • 用于在自定义规则中添加文件级依赖关系的 add_custom_target() 和 add_custom_command() 命令的 DEPENDS 选项。

  • 用于将文件级依赖关系添加到目标文件的 OBJECT_DEPENDS 源文件属性。
