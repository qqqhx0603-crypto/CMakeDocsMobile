cmake_parse_arguments¶

解析函数或宏的参数。

cmake_parse_arguments(<prefix> <options> <one_value_keywords>
                      <multi_value_keywords> <args>...)

cmake_parse_arguments(PARSE_ARGV <N> <prefix> <options>
                      <one_value_keywords> <multi_value_keywords>)

3.5 版本新增：此命令现已原生实现。此前，它定义在 CMakeParseArguments 模块中。

此命令用于宏或函数中。它处理传递给该宏或函数的参数，并定义一组变量来保存各个选项的值。

第一个签名读取在 <args>... 中传递的参数。这可以在 macro() 或 function() 中使用。

3.7 版本新增：PARSE_ARGV 签名仅用于 function() 函数体中。在这种情况下，被解析的参数来自于调用函数的 ARGV# 变量。解析从第 <N> 个参数开始，其中 <N> 是一个无符号整数。这允许参数值中包含 ; 等特殊字符。

<options> 参数包含相应函数或宏的所有选项。这些关键字后面不跟任何值，例如 install() 命令的 OPTIONAL 关键字。

<one_value_keywords> 参数包含此函数或宏的所有后跟一个值的关键字，例如 install() 命令的 DESTINATION 关键字。

<multi_value_keywords> 参数包含此函数或宏的所有可以后跟多个值的关键字，例如 install() 命令的 TARGETS 或 FILES 关键字。

3.5 版本变更：所有关键字必须唯一。每个关键字在 <options>、<one_value_keywords> 或 <multi_value_keywords> 中只能指定一次。如果违反唯一性，将发出警告。

完成后，cmake_parse_arguments 将为 <options>、<one_value_keywords> 和 <multi_value_keywords> 中列出的每个关键字，考虑一个由给定的 <prefix>、"_" 和相应关键字名称组成的变量。对于 <one_value_keywords> 和 <multi_value_keywords>，这些变量将保存参数列表中的相应值，如果未提供关联的关键字，则该变量将处于未定义状态（策略 CMP0174 也可能影响 <one_value_keywords> 的行为）。对于 <options> 关键字，这些变量始终会被定义：如果存在该关键字，则设置为 TRUE；如果不存在，则设置为 FALSE。

所有剩余参数将收集在变量 <prefix>_UNPARSED_ARGUMENTS 中；如果所有参数均被识别，该变量将保持未定义。事后可以检查此变量，以查看宏或函数是否使用了未识别的参数进行调用。

3.15 版本新增：未提供任何值的 <one_value_keywords> 和 <multi_value_keywords> 将被收集在变量 <prefix>_KEYWORDS_MISSING_VALUES 中；如果所有关键字都接收到了值，该变量将保持未定义。可以检查此变量以查看是否存在缺少值的关键字。

3.31 版本变更：如果 <one_value_keyword> 后跟一个空字符串作为值，策略 CMP0174 将控制是否定义相应的 <prefix>_<keyword> 变量。

请仔细选择 <prefix>，以避免与现有变量名冲突。在函数内部使用时，使用前缀 arg 通常是合适的。有一个非常强的惯例是所有关键字都使用全大写，因此该前缀产生的变量形式为 arg_SOME_KEYWORD。这使得代码更具可读性，并最大限度地减少了与缓存变量（其也有全大写的强惯例）冲突的可能性。

function(my_install)
    set(options OPTIONAL FAST)
    set(oneValueArgs DESTINATION RENAME)
    set(multiValueArgs TARGETS CONFIGURATIONS)
    cmake_parse_arguments(PARSE_ARGV 0 arg
        "${options}" "${oneValueArgs}" "${multiValueArgs}"
    )

    # The above will set or unset variables with the following names:
    #   arg_OPTIONAL
    #   arg_FAST
    #   arg_DESTINATION
    #   arg_RENAME
    #   arg_TARGETS
    #   arg_CONFIGURATIONS
    #
    # The following will also be set or unset:
    #   arg_UNPARSED_ARGUMENTS
    #   arg_KEYWORDS_MISSING_VALUES

在宏内部使用时，arg 可能不是合适的前缀，因为代码会影响调用作用域。如果同一作用域内调用的另一个宏在调用 cmake_parse_arguments() 时也使用了 arg，并且两个宏之间有任何公共关键字，那么后续调用的变量可能会覆盖或删除前一个宏调用的变量。因此，建议在 <prefix> 中加入宏名称中唯一的部分，例如 arg_lowercase_macro_name。

macro(my_install)
    set(options OPTIONAL FAST)
    set(oneValueArgs DESTINATION RENAME)
    set(multiValueArgs TARGETS CONFIGURATIONS)
    cmake_parse_arguments(arg_my_install
        "${options}" "${oneValueArgs}" "${multiValueArgs}"
        ${ARGN}
    )
    # ...
endmacro()

macro(my_special_install)
    # NOTE: Has the same keywords as my_install()
    set(options OPTIONAL FAST)
    set(oneValueArgs DESTINATION RENAME)
    set(multiValueArgs TARGETS CONFIGURATIONS)
    cmake_parse_arguments(arg_my_special_install
        "${options}" "${oneValueArgs}" "${multiValueArgs}"
        ${ARGN}
    )
    # ...
endmacro()

假设上述宏依次调用，如下所示

my_install(TARGETS foo bar DESTINATION bin OPTIONAL blub CONFIGURATIONS)
my_special_install(TARGETS barry DESTINATION sbin RENAME FAST)

在完成这两个调用后，以下内容描述了将被设置或取消设置的变量

arg_my_install_OPTIONAL = TRUE
arg_my_install_FAST = FALSE # was not present in call to my_install
arg_my_install_DESTINATION = "bin"
arg_my_install_RENAME <UNSET> # was not present
arg_my_install_TARGETS = "foo;bar"
arg_my_install_CONFIGURATIONS <UNSET> # was not present
arg_my_install_UNPARSED_ARGUMENTS = "blub" # nothing expected after "OPTIONAL"
arg_my_install_KEYWORDS_MISSING_VALUES = "CONFIGURATIONS" # value was missing

arg_my_special_install_OPTIONAL = FALSE # was not present
arg_my_special_install_FAST = TRUE
arg_my_special_install_DESTINATION = "sbin"
arg_my_special_install_RENAME <UNSET> # value was missing
arg_my_special_install_TARGETS = "barry"
arg_my_special_install_CONFIGURATIONS <UNSET> # was not present
arg_my_special_install_UNPARSED_ARGUMENTS <UNSET>
arg_my_special_install_KEYWORDS_MISSING_VALUES = "RENAME"

关键字会终止值列表。如果关键字直接出现在 <one_value_keyword> 之后，则该 <one_value_keyword> 不会接收到任何值，且该关键字会被添加到 <prefix>_KEYWORDS_MISSING_VALUES 变量中。在上面的例子中，对 my_special_install() 的调用包含了 RENAME 关键字，紧接着是 FAST 关键字。在这种情况下，FAST 终止了对 RENAME 关键字的处理。arg_my_special_install_FAST 被设置为 TRUE，arg_my_special_install_RENAME 被取消设置，并且 arg_my_special_install_KEYWORDS_MISSING_VALUES 包含值 RENAME。
