get_filename_component¶

获取完整文件名的特定组件。

3.20 版本更动：此命令已被 cmake_path() 命令取代。但 REALPATH 除外（现由 file(REAL_PATH) 提供），PROGRAM 除外（现可在 separate_arguments(PROGRAM) 中使用）。

3.24 版本更动：提供查询 Windows（微软桌面系统） 注册表能力的未公开功能已被 cmake_host_system_information(QUERY WINDOWS_REGISTRY) 命令取代。

get_filename_component(<var> <FileName> <mode> [CACHE])

代码说明

  • get_filename_component(<var> <FileName> <mode> [CACHE])：调用 `get_filename_component()` 命令，具体行为由这一行传入的参数决定。

将 <var> 设置为 <FileName> 的组件，其中 <mode> 为以下选项之一：

  • DIRECTORY - 不含文件名的目录路径。

  • NAME - 不含目录的文件名。

  • EXT - 文件的最长扩展名（从 d/a.b.c 获取 .b.c）。

  • NAME_WE - 不含目录且不含最长扩展名的文件名。

  • LAST_EXT - 文件的最后一个扩展名（从 d/a.b.c 获取 .c）。

  • NAME_WLE - 不含目录且不含最后一个扩展名的文件名。

  • PATH - DIRECTORY 的旧别名（仅限 CMake <= 2.8.11 使用）。

3.14 版本新增：增加了 LAST_EXT 和 NAME_WLE 模式。

返回的路径使用正斜杠且不包含结尾的斜杠。如果指定了可选的 CACHE 参数，结果变量将被添加到缓存中。

get_filename_component(<var> <FileName> <mode> [BASE_DIR <dir>] [CACHE])

代码说明

  • get_filename_component(<var> <FileName> <mode> [BASE_DIR <dir>] [CACHE])：调用 `get_filename_component()` 命令，具体行为由这一行传入的参数决定。

3.4 版本新增。

将 <var> 设置为 <FileName> 的绝对路径，其中 <mode> 为以下选项之一：

  • ABSOLUTE - 文件的完整路径。

  • REALPATH - 现存文件的完整路径，并解析符号链接。

如果提供的 <FileName> 是相对路径，则它将相对于给定的基础目录 <dir> 进行评估。如果未提供基础目录，则默认基础目录为 CMAKE_CURRENT_SOURCE_DIR。

返回的路径使用正斜杠且不包含结尾的斜杠。如果指定了可选的 CACHE 参数，结果变量将被添加到缓存中。

get_filename_component(<var> <FileName> PROGRAM [PROGRAM_ARGS <arg_var>] [CACHE])

代码说明

  • get_filename_component(<var> <FileName> PROGRAM [PROGRAM_ARGS <arg_var>] [CACHE])：调用 `get_filename_component()` 命令，具体行为由这一行传入的参数决定。

<FileName> 中的程序将在系统搜索路径中查找，或作为完整路径保留。如果 PROGRAM_ARGS 与 PROGRAM 一起出现，则 <FileName> 字符串中存在的任何命令行参数都会从程序名称中分离出来，并存储在 <arg_var> 中。这用于从命令行字符串中分离程序名及其参数。
