set_source_files_properties¶

源文件可以具有影响其构建方式的属性。

set_source_files_properties(<files> ...
                            [DIRECTORY <dirs> ...]
                            [TARGET_DIRECTORY <targets> ...]
                            PROPERTIES <prop1> <value1>
                            [<prop2> <value2>] ...)

代码说明

  • set_source_files_properties(<files> ...：这一行继续补充当前命令或表达式的内容。

  • [DIRECTORY <dirs> ...]：这一行继续补充当前命令或表达式的内容。

  • [TARGET_DIRECTORY <targets> ...]：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES <prop1> <value1>：这一行继续补充当前命令或表达式的内容。

  • [<prop2> <value2>] ...)：这一行继续补充当前命令或表达式的内容。

使用键/值对列表设置与源文件关联的属性。

3.18 版本新增： 默认情况下，源文件属性仅对在同一目录中添加的目标可见（CMakeLists.txt）。可以使用以下一个或两个选项在其他目录范围内设置可见性

DIRECTORY <dirs>...

源文件属性将在每个 <dirs> 目录的作用域中设置。CMake 必须已经知道这些源目录中的每一个，要么是通过调用 add_subdirectory() 添加它们，要么它是顶级源目录。相对路径相对于当前源目录处理。

TARGET_DIRECTORY <targets>...

源文件属性将在创建任何指定 <targets> 的每个目录作用域中设置（因此 <targets> 必须已经存在）。

使用 get_source_file_property() 获取属性值。另请参阅 set_property(SOURCE) 命令。

注意

全局可见的源文件属性 GENERATED。有关详细信息，请参阅其文档。
