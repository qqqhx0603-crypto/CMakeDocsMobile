set¶

将普通变量、缓存变量或环境变量设置为指定的值。关于普通变量和缓存条目的作用域及相互作用，请参阅 cmake-language(7) 变量 文档。

此命令中指定了 <value>... 占位符的签名需要零个或多个参数。多个参数将合并为以 分号分隔的列表，以构成要设置的实际变量值。

设置普通变量¶

set(<variable> <value>... [PARENT_SCOPE])¶

在当前函数或目录作用域中设置或取消设置 <variable>

  • 如果提供了至少一个 <value>...，则将变量设置为该值。

  • 如果不提供值，则取消设置该变量。这等同于 unset(<variable>)。

如果提供了 PARENT_SCOPE 选项，变量将在当前作用域的上一级作用域中设置。每个新目录或 function() 命令都会创建一个新的作用域。作用域也可以通过 block() 命令创建。set(PARENT_SCOPE) 会将变量的值设置到父目录、调用函数或外层作用域中（以实际情况为准）。变量在当前作用域中的先前状态保持不变（例如，如果之前未定义，则依然未定义；如果之前有值，则依然保持该值）。

block(PROPAGATE) 和 return(PROPAGATE) 命令可以用作 set(PARENT_SCOPE) 和 unset(PARENT_SCOPE) 命令的替代方法，用于更新父作用域。

注意

当评估形式为 ${VAR} 的 变量引用 时，CMake 首先会搜索同名的普通变量。如果不存在此类普通变量，CMake 随后会搜索同名的缓存条目。因此，取消设置一个普通变量可能会暴露出之前被遮蔽的缓存变量。若要强制形式为 ${VAR} 的变量引用返回空字符串，请使用 set(<variable> "")，这会清除普通变量，但使其保持定义状态。

设置缓存条目¶

set(CACHE{<variable>} [TYPE <type>] [HELP <helpstring>...] [FORCE] VALUE [<value>...])¶

版本 4.2 中添加。

设置给定的缓存 <variable>（缓存条目）。选项如下：

TYPE <type>

指定缓存条目的类型。<type> 必须是以下类型之一：

BOOL

布尔值 ON/OFF。cmake-gui(1) 提供复选框。

FILEPATH

磁盘上的文件路径。cmake-gui(1) 提供文件对话框。

PATH

磁盘上的目录路径。cmake-gui(1) 提供文件对话框。

STRING

一行文本。cmake-gui(1) 提供文本框，如果设置了 STRINGS 缓存条目属性，则提供下拉选择。

INTERNAL

一行文本。cmake-gui(1) 不显示内部条目。它们可用于在运行期间持久化存储变量。使用此类型意味着包含 FORCE。

如果未指定 TYPE，且缓存变量已存在且其类型不是 UNINITIALIZED，则保留之前指定的类型，否则使用 STRING。

HELP <helpstring>...

<helpstring> 必须指定为一行文本，为 cmake-gui(1) 用户提供选项的简要说明。如果提供了多个字符串，它们将被连接成一个字符串，中间没有分隔符。

如果未指定 HELP，将使用空字符串。

FORCE

由于缓存条目旨在提供用户可设置的值，因此默认情况下不会覆盖现有的缓存条目。使用 FORCE 选项可覆盖现有条目。

VALUE <value>...

要设置为缓存 <variable> 的值列表。此参数必须始终放在最后。

如果缓存条目在调用之前不存在，或者给出了 FORCE 选项，则该缓存条目将被设置为给定值。

注意

如果同名的普通变量已存在，则缓存变量的内容将无法直接访问（参见 变量评估规则）。如果策略 CMP0126 设置为 OLD，则当前作用域中的任何普通变量绑定都将被移除。

缓存条目在调用前可能存在但未设置类型（如果它是用户通过 -D<var>=<value> 选项在 cmake(1) 命令行上创建且未指定类型）。在这种情况下，set 命令将添加类型。此外，如果 <type> 为 PATH 或 FILEPATH，且命令行提供的 <value> 是相对路径，则 set 命令会将该路径视为相对于当前工作目录，并将其转换为绝对路径。

set(<variable> <value>... CACHE <type> <docstring> [FORCE])¶

此签名仅为兼容性目的提供。建议优先使用另一个签名。

设置环境变量¶

set(ENV{<variable>} [<value>])¶

将 环境变量 设置为给定值。后续对 $ENV{<variable>} 的调用将返回此新值。

此命令仅影响当前的 CMake 进程，不影响调用 CMake 的进程，不影响系统整体环境，也不影响后续构建或测试进程的环境。

如果 ENV{<variable>} 后未提供参数，或者 <value> 为空字符串，则此命令将清除该环境变量的任何现有值。

<value> 之后的参数将被忽略。如果发现多余参数，则会发出作者警告。
