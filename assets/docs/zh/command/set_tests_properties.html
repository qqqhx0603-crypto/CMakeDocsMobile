set_tests_properties¶

设置测试的属性。

set_tests_properties(<tests>...
                     [DIRECTORY <dir>]
                     PROPERTIES <prop1> <value1>
                     [<prop2> <value2>]...)

代码说明

  • set_tests_properties(<tests>... [DIRECTORY <dir>] PROPERTIES <prop1> <value1> [<prop2> <value2>]...)：调用 `set_tests_properties()` 命令，具体行为由这一行传入的参数决定。

为测试设置属性。如果未找到测试，CMake 将报告错误。

测试属性值可以使用 生成器 表达式 为通过 add_test(NAME) 签名创建的测试指定。

3.28 版本新增：可以使用以下选项在其他目录范围内设置可见性

DIRECTORY <dir>

测试属性将在 <dir> 目录的范围内设置。CMake 必须已经知道此目录，要么是通过调用 add_subdirectory() 添加的，要么它是顶级源目录。相对路径相对于当前源目录处理。<dir> 可以引用一个构建目录。
