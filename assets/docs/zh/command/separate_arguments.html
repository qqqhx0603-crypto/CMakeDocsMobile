separate_arguments¶

将命令行参数解析为以分号分隔的列表。

separate_arguments(<variable> <mode> [PROGRAM [SEPARATE_ARGS]] <args>)

代码说明

  • separate_arguments(<variable> <mode> [PROGRAM [SEPARATE_ARGS]] <args>)：调用 separate_arguments() 命令，具体行为由这一行传入的参数决定。

将空格分隔的字符串 <args> 解析为项目列表，并将此列表以分号分隔的标准形式存储在 <variable> 中。

此函数用于解析命令行参数。整个命令行必须作为单个字符串传递到参数 <args> 中。

确切的解析规则取决于操作系统。它们由参数 <mode> 指定，该参数必须是以下关键字之一

UNIX_COMMAND

参数由未引用的空格分隔。单引号和双引号对均受尊重。反斜杠转义下一个文字字符（\" 是 "）；没有特殊转义（\n 只是 n）。

WINDOWS_COMMAND

Windows 命令行使用运行时库在启动时构造 argv 所使用的相同语法进行解析。它通过未用双引号括起来的空格分隔参数。反斜杠是文字，除非它们位于双引号之前。有关详细信息，请参阅 MSDN 文章 解析 C 命令行参数。

NATIVE_COMMAND

版本 3.9 中添加。

如果主机系统是 Windows（微软桌面系统），则像 WINDOWS_COMMAND 模式一样进行操作。否则，像 UNIX_COMMAND 模式一样进行操作。

PROGRAM

3.19 版本新增。

假设 <args> 中的第一项是可执行文件，将在系统搜索路径中搜索或保留为完整路径。如果未找到，<variable> 将为空。否则，<variable> 是一个包含 2 个元素的列表

  1. 程序的绝对路径

  2. 存在于 <args> 中的任何命令行参数，以字符串形式

例如

separate_arguments (out UNIX_COMMAND PROGRAM "cc -c main.c")

代码说明

  • separate_arguments (out UNIX_COMMAND PROGRAM "cc -c main.c")：调用 separate_arguments() 命令，具体行为由这一行传入的参数决定。

  • 列表的第一个元素：/path/to/cc

  • 列表的第二个元素：" -c main.c"

SEPARATE_ARGS

如果指定了 PROGRAM 选项的此子选项，命令行参数也将被拆分并存储在 <variable> 中。

例如

separate_arguments (out UNIX_COMMAND PROGRAM SEPARATE_ARGS "cc -c main.c")

代码说明

  • separate_arguments (out UNIX_COMMAND PROGRAM SEPARATE_ARGS "cc -c main.c")：调用 separate_arguments() 命令，具体行为由这一行传入的参数决定。

out 的内容将是：/path/to/cc;-c;main.c

separate_arguments(<var>)

代码说明

  • separate_arguments(<var>)：调用 separate_arguments() 命令，具体行为由这一行传入的参数决定。

将 <var> 的值转换为以分号分隔的列表。所有空格都替换为“;”。这有助于生成命令行。
