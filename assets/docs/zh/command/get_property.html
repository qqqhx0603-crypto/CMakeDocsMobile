get_property¶

获取一个属性。

get_property(<variable>
             <GLOBAL                                                  |
              DIRECTORY [<dir>]                                       |
              TARGET    <target>                                      |
              FILE_SET  <file_set> TARGET <target>                    |
              SOURCE    <source>
                        [DIRECTORY <dir> | TARGET_DIRECTORY <target>] |
              INSTALL   <file>                                        |
              TEST      <test>
                        [DIRECTORY <dir>]                             |
              CACHE     <entry>                                       |
              VARIABLE>
             PROPERTY <name>
             [SET | DEFINED | BRIEF_DOCS | FULL_DOCS])

代码说明

  • get_property(<variable>：这一行继续补充当前命令或表达式的内容。

  • <GLOBAL |：这一行继续补充当前命令或表达式的内容；运算符：< 表示小于比较。

  • DIRECTORY [<dir>] |：这一行继续补充当前命令或表达式的内容。

  • TARGET <target> |：这一行继续补充当前命令或表达式的内容；条件运算符：TARGET 表示判断给定构建目标是否已定义。

  • FILE_SET <file_set> TARGET <target> |：这一行继续补充当前命令或表达式的内容。

  • SOURCE <source>：这一行继续补充当前命令或表达式的内容。

  • [DIRECTORY <dir> | TARGET_DIRECTORY <target>] |：这一行继续补充当前命令或表达式的内容；运算符：| 表示按位或。

  • INSTALL <file> |：这一行继续补充当前命令或表达式的内容。

  • TEST <test>：这一行继续补充当前命令或表达式的内容；条件运算符：TEST 表示判断给定测试是否已定义。

  • [DIRECTORY <dir>] |：这一行继续补充当前命令或表达式的内容。

  • CACHE <entry> |：这一行继续补充当前命令或表达式的内容。

  • VARIABLE>：这一行继续补充当前命令或表达式的内容；运算符：> 表示大于比较。

  • PROPERTY <name>：这一行继续补充当前命令或表达式的内容。

  • [SET | DEFINED | BRIEF_DOCS | FULL_DOCS])：这一行继续补充当前命令或表达式的内容；运算符：| 表示按位或。

从指定作用域的一个对象中获取一个属性。

第一个参数指定用于存储结果的变量。第二个参数确定要从中获取属性的作用域。它必须是以下选项之一：

GLOBAL

作用域是唯一的，不需要名称。

DIRECTORY

作用域默认为当前目录，但可以通过完整路径或相对路径 <dir> 来指定另一个（已被 CMake 处理过的）目录。相对路径被视为相对于当前源目录。另请参阅 get_directory_property() 命令。

3.19 版本新增： <dir> 可以引用二进制目录。

TARGET

作用域必须指定一个现有的目标。另请参阅 get_target_property() 命令。

FILE_SET

4.3 版本新增。

作用域必须指定一个现有的文件集。

以下选项是必需的：

TARGET <target>

附加该文件集的目标。

SOURCE

作用域必须指定一个源文件。默认情况下，源文件属性将从当前源目录的作用域中读取。

3.18 版本加入：目录作用域可以通过以下子选项之一进行覆盖：

DIRECTORY <dir>

源文件属性将从 <dir> 目录的作用域中读取。CMake 必须已经知晓该目录，要么通过调用 add_subdirectory() 添加过该目录，要么 <dir> 本身就是顶层目录。相对路径被视为相对于当前源目录。

3.19 版本新增： <dir> 可以引用二进制目录。

TARGET_DIRECTORY <target>

源文件属性将从创建 <target> 时所在的目录作用域中读取（因此 <target> 必须已经存在）。

另请参阅 get_source_file_property() 命令。

INSTALL

版本 3.1 中新增。

作用域必须指定一个已安装的文件路径。

TEST

作用域必须指定一个现有的测试。另请参阅 get_test_property() 命令。

3.28 版本加入：目录作用域可以通过以下子选项进行覆盖：

DIRECTORY <dir>

测试属性将从 <dir> 目录的作用域中读取。CMake 必须已经知晓该目录，要么通过调用 add_subdirectory() 添加过该目录，要么 <dir> 本身就是顶层目录。相对路径被视为相对于当前源目录。<dir> 可以引用二进制目录。

CACHE

作用域必须指定一个缓存条目。

VARIABLE

作用域是唯一的，不需要名称。

必需的 PROPERTY 选项后面紧跟要获取的属性名称。如果该属性未设置，则返回时调用作用域中的命名 <variable> 将被取消设置，尽管某些属性支持从父作用域继承（如果定义为那种行为，请参阅 define_property()）。

如果给出了 SET 选项，该变量将被设置为一个布尔值，指示该属性是否已被设置。如果给出了 DEFINED 选项，该变量将被设置为一个布尔值，指示该属性是否已被定义（例如通过 define_property() 命令）。

如果给出了 BRIEF_DOCS 或 FULL_DOCS，则变量被设置为包含所请求属性文档的字符串。如果对尚未定义的属性请求文档，则返回 NOTFOUND。

注意

全局可见的源文件属性 GENERATED。有关详细信息，请参阅其文档。
