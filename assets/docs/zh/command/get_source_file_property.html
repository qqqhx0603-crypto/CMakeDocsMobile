get_source_file_property¶

获取源文件的属性。

get_source_file_property(<variable> <file>
                         [DIRECTORY <dir> | TARGET_DIRECTORY <target>]
                         <property>)

代码说明

  • get_source_file_property(<variable> <file> [DIRECTORY <dir> | TARGET_DIRECTORY <target>] <property>)：调用 `get_source_file_property()` 命令，具体行为由这一行传入的参数决定。

获取源文件的属性。属性值将存储在指定的 <variable> 中。如果 <file> 不是源文件，或者未找到该源文件属性，则 <variable> 将被设置为 NOTFOUND。如果该源文件属性被定义为 INHERITED 属性（参见 define_property()），则搜索将包含相关的父作用域，正如 define_property() 命令所描述的那样。

默认情况下，源文件属性将从当前源目录的作用域中读取。

3.18 版本新增：目录作用域可以通过以下子选项之一进行覆盖

DIRECTORY <dir>

源文件属性将从 <dir> 目录的作用域中读取。CMake 必须已经知晓该源目录，即通过调用 add_subdirectory() 添加过该目录，或者 <dir> 是顶级源目录。相对路径将被视为相对于当前源目录的路径。

TARGET_DIRECTORY <target>

源文件属性将从创建 <target> 时所在的目录作用域中读取（因此 <target> 必须已经存在）。

使用 set_source_files_properties() 来设置属性值。源文件属性通常用于控制文件的构建方式。其中一个始终存在的属性是 LOCATION。

注意

全局可见的源文件属性 GENERATED。有关详细信息，请参阅其文档。
