add_custom_target¶

添加一个没有输出的目标，以便它始终会被构建。

add_custom_target(Name [ALL] [command1 [<args1>...]]
                  [COMMAND command2 [<args2>...]] ...
                  [DEPENDS <depend>...]
                  [BYPRODUCTS <file>...]
                  [WORKING_DIRECTORY <dir>]
                  [COMMENT <comment>]
                  [JOB_POOL <job_pool>]
                  [JOB_SERVER_AWARE <bool>]
                  [VERBATIM] [USES_TERMINAL]
                  [COMMAND_EXPAND_LISTS]
                  [SOURCES <source>...])

添加一个指定名称的目标，该目标执行指定的命令。此目标没有输出文件，并且始终被视为已过期，即使命令尝试创建与目标同名的文件也是如此。请使用 add_custom_command() 命令来生成具有依赖关系的文件。默认情况下，没有任何对象依赖于此自定义目标。请使用 add_dependencies() 命令来添加与其它目标之间的依赖关系。

选项包括

ALL

表示此目标应被添加到默认构建目标中，以便每次都运行它（该命令不能称为 ALL）。

BYPRODUCTS

版本 3.2 中新增。

指定该命令预期产生的文件，但这些文件的修改时间在后续构建中可能会也可能不会更新。如果副产品（byproduct）名称是相对路径，它将被解释为相对于当前源目录对应的构建树目录。每个副产品文件都将自动被标记为 GENERATED 源文件属性。

有关此特性的动机，请参阅策略 CMP0058。

Ninja 生成器支持显式指定副产品，以告知 ninja 构建工具在副产品缺失时如何重新生成它们。当其它构建规则（例如自定义命令）依赖于这些副产品时，这也很有用。Ninja 要求任何被其它规则依赖的生成文件必须有相应的构建规则，即使存在仅用于顺序的依赖（order-only dependencies），以确保在依赖方构建之前，副产品已经可用。

Makefile 生成器 将在 make clean 期间删除 BYPRODUCTS 和其它 GENERATED 文件。

3.20 版本加入: BYPRODUCTS 的参数可以使用受限的 生成器表达式。不允许使用目标相关表达式。

3.28 版本变更: 在通过 文件集 (File Sets) 使用的自定义目标中，除非副产品列在非私有文件集中，否则现在它们被视为私有的。请参阅策略 CMP0154。

COMMAND

指定要在构建时执行的命令行。如果指定了多个 COMMAND，它们将按顺序执行，但不一定会被组合成一个有状态的 shell 或批处理脚本。（要运行完整的脚本，请使用 configure_file() 命令或 file(GENERATE) 命令来创建它，然后指定一个 COMMAND 来启动它。）

如果 COMMAND 指定了一个可执行目标名称（由 add_executable() 命令创建），在以下任一条件成立时，它将自动被替换为构建时创建的可执行文件的位置：

  • 该目标未在进行交叉编译（即 CMAKE_CROSSCOMPILING 变量未被设置为 true）。

  • 3.6 版本加入: 该目标正在进行交叉编译，并且提供了模拟器（即其 CROSSCOMPILING_EMULATOR 目标属性已设置）。在这种情况下，CROSSCOMPILING_EMULATOR 的内容将在目标可执行文件路径之前添加到命令中。

如果不满足上述任一条件，则假设该命令名称是构建时在 PATH 中可以找到的程序。

COMMAND 的参数可以使用 生成器表达式。使用 TARGET_FILE 生成器表达式来引用命令行中后续的目标位置（即作为命令参数而非要执行的命令）。

每当以下基于目标的生成器表达式之一被用作要执行的命令，或在命令参数中被提及，都会自动添加目标级依赖，以确保提到的目标在此自定义目标之前构建（请参阅策略 CMP0112）。

  • TARGET_FILE

  • TARGET_LINKER_FILE

  • TARGET_SONAME_FILE

  • TARGET_PDB_FILE

命令和参数是可选的，如果未指定，将创建一个空目标。

COMMENT

在构建时执行命令之前显示给定的消息。

3.26 版本加入: COMMENT 的参数可以使用 生成器表达式。

DEPENDS

引用同一目录（CMakeLists.txt 文件）中使用 add_custom_command() 命令创建的自定义命令的文件和输出。当目标被构建时，它们将被更新。

3.16 版本变更: 如果任何依赖项是同一目录中目标或其任何构建事件的副产品，则会添加目标级依赖，以确保在构建此目标之前，副产品已经可用。

使用 add_dependencies() 命令来添加对其它目标的依赖。

COMMAND_EXPAND_LISTS

版本 3.8 新增。

COMMAND 参数中的列表将被展开，包括那些由 生成器表达式 创建的列表，从而允许正确展开类似 ${CC} "-I$<JOIN:$<TARGET_PROPERTY:foo,INCLUDE_DIRECTORIES>,;-I>" foo.cc 这样的 COMMAND 参数。

JOB_POOL

版本 3.15 新增。

为 Ninja 生成器指定一个 作业池 (pool)。它与 USES_TERMINAL 不兼容，后者隐含使用 console 池。使用非 JOB_POOLS 定义的池会导致 ninja 在构建时报错。

JOB_SERVER_AWARE

版本 3.28 新增。

指定该命令能够识别 GNU Make 作业服务器（job server）。

对于 Unix Makefiles、MSYS Makefiles 和 MinGW Makefiles 生成器，这会为配方行（recipe line）添加 + 前缀。更多信息请参阅 GNU Make 文档。

此选项会被其它生成器静默忽略。

SOURCES

指定要包含在自定义目标中的额外源文件。指定的源文件将被添加到 IDE 项目文件中，以便于编辑，即使它们没有构建规则也是如此。

VERBATIM

命令的所有参数都将为构建工具进行适当的转义，以便被调用的命令能接收到未更改的每个参数。请注意，在 add_custom_target 看到参数之前，CMake 语言处理器已经使用了一层转义。建议使用 VERBATIM，因为它能确保行为正确。当不提供 VERBATIM 时，行为是特定于平台的，因为没有对工具特定的特殊字符进行保护。

USES_TERMINAL

版本 3.2 中新增。

如果可能，该命令将获得对终端的直接访问权限。使用 Ninja 生成器时，这会将命令置于 console 池 中。

WORKING_DIRECTORY

在给定的当前工作目录中执行命令。如果它是相对路径，它将被解释为相对于当前源目录对应的构建树目录。如果未指定，则设置为 CMAKE_CURRENT_BINARY_DIR。

3.13 版本加入: WORKING_DIRECTORY 的参数可以使用 生成器表达式。

Ninja 多配置¶

3.20 版本加入: add_custom_target 支持 Ninja Multi-Config 生成器的跨配置功能。有关更多信息，请参阅生成器文档。
