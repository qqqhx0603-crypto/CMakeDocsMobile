TEST_INCLUDE_FILE¶

自版本 3.10 起已弃用：请使用 TEST_INCLUDE_FILES 目录属性，它可以支持指定多个文件。

目录属性 TEST_INCLUDE_FILE 指定一个 CMake 脚本，当在目录上运行 ctest 时，该脚本将被包含和处理。

如果同时设置了 TEST_INCLUDE_FILE 和 TEST_INCLUDE_FILES 目录属性，则首先包含在 TEST_INCLUDE_FILE 中指定的脚本，然后是 TEST_INCLUDE_FILES 中列出的脚本。

上一主题

INTERPROCEDURAL_OPTIMIZATION_<CONFIG>

下一主题

COMPILE_DEFINITIONS_<CONFIG>
