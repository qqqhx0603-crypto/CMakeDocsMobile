TEST_INCLUDE_FILES¶

3.10 版本新增。

此目录属性指定一个 CMake 脚本列表，当 ctest 在该目录上运行时，这些脚本将被包含和处理。 使用绝对路径，以避免歧义。 脚本文件将按照指定的顺序包含。

TEST_INCLUDE_FILES 脚本在运行 ctest 时被处理，而不是在 cmake 配置阶段。 这些脚本应该像 CTest 控制面板脚本一样编写。 由于配置阶段可用的许多变量和命令在测试阶段不可访问，因此动态生成这些脚本是很常见的。

示例¶

将此目录属性设置为附加一个或多个 CMake 脚本

CMakeLists.txt¶
configure_file(script.cmake.in script.cmake)

set_property(
  DIRECTORY
  APPEND
  PROPERTY TEST_INCLUDE_FILES
    ${CMAKE_CURRENT_BINARY_DIR}/script.cmake
    ${CMAKE_CURRENT_SOURCE_DIR}/foo.cmake
    ${dir}/bar.cmake
)

代码说明

  • configure_file(script.cmake.in script.cmake)：调用 configure_file() 命令，具体行为由这一行传入的参数决定。

  • set_property(：这一行继续补充当前命令或表达式的内容。

  • DIRECTORY：这一行继续补充当前命令或表达式的内容。

  • APPEND：这一行继续补充当前命令或表达式的内容。

  • PROPERTY TEST_INCLUDE_FILES：这一行继续补充当前命令或表达式的内容。

  • ${CMAKE_CURRENT_BINARY_DIR}/script.cmake：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_BINARY_DIR：当前源目录所对应的构建目录。

  • ${CMAKE_CURRENT_SOURCE_DIR}/foo.cmake：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • ${dir}/bar.cmake：这一行继续补充当前命令或表达式的内容；变量说明：dir：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • )：这一行继续补充当前命令或表达式的内容。

script.cmake.in¶
execute_process(
  COMMAND "@CMAKE_COMMAND@" -E echo "script.cmake executed during CTest"
)

代码说明

  • execute_process(：这一行继续补充当前命令或表达式的内容。

  • COMMAND "@CMAKE_COMMAND@" -E echo "script.cmake executed during CTest"：这一行继续补充当前命令或表达式的内容；条件运算符：COMMAND 表示判断给定名称是否为可调用命令。

  • )：这一行继续补充当前命令或表达式的内容。
