自 CMake 3.11 以来的更改包括以下内容。

新特性¶

生成器 (Generators)¶

  • 针对 VS 2017 的 Visual Studio 生成器 现在支持在 CMAKE_GENERATOR_TOOLSET 值中使用 version=14.## 选项（例如通过 cmake(1) 的 -T 选项）来指定工具集版本号。

命令行¶

  • cmake(1) 的 构建项目 模式（cmake --build）增加了 --parallel [<jobs>] 和 -j [<jobs>] 选项，用于指定并行构建级别。它们会映射到原生构建工具的相应选项。

命令¶

  • 增加了 add_compile_definitions() 命令，用于在目录级别设置预处理器定义。该命令取代了 add_definitions()。

  • cmake_minimum_required() 和 cmake_policy(VERSION) 命令现在支持使用 <min>[...<max>] 格式的版本范围。其中 <min> 版本是必需的，但策略的设置基于当前运行的 CMake 版本与 <max> 指定版本中的较旧者。这允许项目指定其已更新版本范围，并避免显式的策略设置。

  • file(GLOB) 和 file(GLOB_RECURSE) 命令新增了一个 CONFIGURE_DEPENDS 标志，它允许构建系统表达对已 glob 匹配的目录内容的依赖。

  • 增加了 file(TOUCH) 和 file(TOUCH_NOCREATE) 命令，以便在不使用 CMake 命令行工具模式配合 execute_process() 的情况下，暴露 TOUCH 功能。

  • find_package() 命令现在会搜索 <PackageName>_ROOT CMake 变量和 <PackageName>_ROOT 环境变量中指定的前缀路径。包根目录以堆栈方式维护，因此查找模块中对所有 find_* 命令的嵌套调用也会搜索这些作为前缀的根目录。参见策略 CMP0074。

  • install() 命令增加了一个可选的 NAMELINK_COMPONENT 参数，允许更改共享库 namelink 的组件。如果未指定，则默认使用 COMPONENT 的值。

  • list() 命令增加了 JOIN 子命令，用于将列表元素用连接字符串拼接起来。

  • list() 命令增加了 SUBLIST 子命令，用于获取列表的子列表。

  • list() 命令增加了 TRANSFORM 子命令，用于对列表元素应用各种字符串转换。

  • project() 命令增加了一个可选的 HOMEPAGE_URL 参数，其作用是设置诸如 PROJECT_HOMEPAGE_URL、<PROJECT-NAME>_HOMEPAGE_URL 和 CMAKE_PROJECT_HOMEPAGE_URL 等变量。

  • string() 命令增加了 JOIN 子命令，用于将输入字符串用连接字符串拼接起来。

  • target_compile_options() 和 add_compile_options() 命令增加了 SHELL: 前缀，用于通过类似 shell 的引用方式指定一组相关的选项。

  • target_link_libraries() 命令现在支持 对象库 (Object Libraries)。链接到对象库会在直接依赖项中使用其目标文件，并传播使用要求。

变量¶

  • 增加了 CMAKE_FOLDER 变量，用于初始化所有目标上的 FOLDER 属性。

  • 定义了 CMAKE_DOTNET_TARGET_FRAMEWORK_VERSION 变量，用于初始化所有 DOTNET_TARGET_FRAMEWORK_VERSION 目标属性。

  • 引入了 CMAKE_PROJECT_VERSION* 变量

    • CMAKE_PROJECT_VERSION

    • CMAKE_PROJECT_VERSION_MAJOR

    • CMAKE_PROJECT_VERSION_MINOR

    • CMAKE_PROJECT_VERSION_PATCH

    • CMAKE_PROJECT_VERSION_TWEAK

  • CMAKE_SUPPRESS_REGENERATION 变量已扩展，支持 Ninja 和 Makefile 生成器。它现在也已载入文档。

  • 定义了 CMAKE_VS_SDK_*_DIRECTORIES 变量，以告知针对 VS 2010 及以上版本的 Visual Studio 生成器 如何填充 .vcxproj 文件中指定 SDK 目录的字段。这些变量包括：

    • CMAKE_VS_SDK_EXCLUDE_DIRECTORIES

    • CMAKE_VS_SDK_EXECUTABLE_DIRECTORIES

    • CMAKE_VS_SDK_INCLUDE_DIRECTORIES

    • CMAKE_VS_SDK_LIBRARY_DIRECTORIES

    • CMAKE_VS_SDK_LIBRARY_WINRT_DIRECTORIES

    • CMAKE_VS_SDK_REFERENCE_DIRECTORIES

    • CMAKE_VS_SDK_SOURCE_DIRECTORIES

  • 增加了 MSVC_TOOLSET_VERSION 变量，用于提供与 MSVC_VERSION 中当前 MSVC 编译器版本相关联的 MSVC 工具集版本。

属性¶

  • 引入了 COMMON_LANGUAGE_RUNTIME 目标属性，用于配置 VS 2010 及以上版本的 Visual Studio 生成器 对托管 C++ 的使用。添加了相应的 IMPORTED_COMMON_LANGUAGE_RUNTIME 目标属性，以支持导入目标的 C++/CLI。

  • 引入了 DOTNET_TARGET_FRAMEWORK_VERSION 目标属性，作为 VS_DOTNET_TARGET_FRAMEWORK_VERSION 的替代品，后者现已被弃用。

  • 添加了 EXPORT_PROPERTIES 目标属性，用于指定自定义的目标属性列表，以包含在由 install(EXPORT) 和 export() 命令导出的目标中。

  • PDB_OUTPUT_DIRECTORY 属性现在支持 生成器表达式。

  • 添加了 TESTS 目录属性，用于保存由 add_test() 命令定义的测试列表。

  • 创建了 VS_DEBUGGER_COMMAND 目标属性，用于为 VS 2010 及以上版本的 Visual Studio 生成器 设置调试命令行。

  • HLSL 源文件属性 VS_SHADER_DISABLE_OPTIMIZATIONS 和 VS_SHADER_ENABLE_DEBUG 增加了对生成器表达式的支持。

  • HLSL 源文件属性 VS_SHADER_OBJECT_FILE_NAME 已添加到 VS 2010 及以上版本的 Visual Studio 生成器 中。该属性指定编译后的着色器对象文件名。

模块¶

  • FindALSA 模块现在提供导入目标。

  • FindCURL 模块现在提供导入目标。

  • FindJPEG 模块现在提供导入目标。

  • FindLibXml2 模块现在提供导入目标。

  • FindMatlab 模块现在支持用于编译和链接 matlab 扩展的 Matlab 运行时编译器 (MCR)。

  • 增加了 FindODBC 模块，用于查找开放数据库互连 (ODBC) 库。

  • FindPkgConfig 模块现在可以将找到的库导出为完整路径，以便通过 target_link_libraries() 命令直接使用。

  • 添加了新的 FindPython3 和 FindPython2 模块，以及一个新的 FindPython 模块，提供定位 Python 环境的新方法。

  • UseSWIG 模块经过全面刷新，现在与生成库的标准 CMake 命令更加一致，并且可以通过属性进行完全配置。

  • UseSWIG 模块现在可以通过 UseSWIG_MODULE_VERSION 变量管理多种行为，以确保旧版本兼容性以及对 SWIG 高级功能（如 %template）更稳健的处理。

  • UseSWIG 模块增加了对 CSHARP 变体包装文件的支持。

  • WriteCompilerDetectionHeader 模块增加了 BARE_FEATURES 选项，用于为新语言特性的确切关键字添加兼容性宏定义。

生成器表达式¶

  • 添加了新的 $<GENEX_EVAL:...> 和 $<TARGET_GENEX_EVAL:target,...> 生成器表达式，以支持评估结果本身就是生成器表达式的场景。

  • 添加了新的 $<IN_LIST:...> 生成器表达式。

  • 添加了新的 $<TARGET_EXISTS:...> 生成器表达式。

  • 添加了新的 $<TARGET_NAME_IF_EXISTS:...> 生成器表达式。

CTest¶

  • ctest_start() 命令已重构，现在您可以直接调用 ctest_start(APPEND)，它将从 TAG 文件中读取所有所需信息。参数解析也变得更加宽松，参数顺序不再那么重要。

  • 添加了 PROCESSOR_AFFINITY 测试属性，用于请求 CTest 在运行测试时绑定 CPU 亲和性，所选处理器集将与设置了该属性且同时运行的其他测试不相交。

CPack¶

  • CPack 模块现在使用 CMAKE_PROJECT_VERSION_MAJOR、CMAKE_PROJECT_VERSION_MINOR 和 CMAKE_PROJECT_VERSION_PATCH 变量来初始化相应的 CPack 变量。

  • 添加了 CPack NuGet 生成器，初步支持 NuGet。

其他¶

  • 编译特性 功能现在支持 C++ 20。除了 cxx_std_20 元特性外，尚未枚举具体特性。

  • 编译特性 功能现在能够识别 VS 2010 以来 MSVC 中 C 特性的可用性。

  • 编译特性 功能现在能够识别德州仪器 (Texas Instruments) C 编译器支持的 C 语言标准。

已弃用和移除的功能¶

  • Visual Studio 8 2005 生成器已被移除。

  • CMake 不再为库目标生成 <tgt>_LIB_DEPENDS 缓存条目。参见策略 CMP0073。

其他更改¶

  • 标记为 SYSTEM 的目录的包含标志现在移到了非系统目录之后。 -isystem 标志会自动执行此操作，因此显式地将它们移至末尾可以使在没有 -isystem 标志的编译器上的行为保持一致。

  • Fortran 依赖扫描现在支持由 Fortran 子模块 隐含的依赖关系。

  • 文件 ${CMAKE_BINARY_DIR}/cmake_install.cmake 的存在和功能现在已记录在 install() 文档中，以便外部打包软件可以利用 CPack 风格的组件安装。

  • CheckIncludeFile 模块中的 check_include_file() 命令现在遵循 CMAKE_REQUIRED_LIBRARIES 变量。参见策略 CMP0075。

  • CheckIncludeFileCXX 模块中的 check_include_file_cxx() 命令现在遵循 CMAKE_REQUIRED_LIBRARIES 变量。参见策略 CMP0075。

  • CheckIncludeFiles 模块中的 check_include_files() 命令现在遵循 CMAKE_REQUIRED_LIBRARIES 变量。参见策略 CMP0075。

  • cmake(1) 的 -E copy_directory 工具现在在源目录不存在时会报错失败。此前，它会通过创建一个空的目标目录而成功。

  • UseSWIG 模块的 swig_add_library() 命令（以及旧版 swig_add_module 命令）现在在 MINGW、MSYS 和 CYGWIN 环境下将 Java 模块的前缀设置为 ""。
