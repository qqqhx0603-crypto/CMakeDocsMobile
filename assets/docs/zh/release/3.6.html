自 CMake 3.5 以来的更改包括以下内容。

新特性¶

生成器 (Generators)¶

  • Ninja 生成器学会了生成 sub/dir/all 形式的伪目标，以驱动子目录的构建。这等同于使用 Makefile 生成器 时的 cd sub/dir; make all。

  • Ninja 生成器现在将系统头文件包含在构建依赖项中，以确保在更新系统软件包时能够正确重新构建。

  • Visual Studio 14 2015 生成器学会了支持 Clang/C2 工具集，例如使用 -T v140_clang_3_7 选项。此功能是实验性的。

命令 (Commands)¶

  • add_custom_command() 和 add_custom_target() 命令学会了如何使用 CROSSCOMPILING_EMULATOR 可执行目标属性。

  • install() 命令新增了一个 EXCLUDE_FROM_ALL 选项，用于将安装规则排除在默认安装之外。

  • list() 命令增加了一个 FILTER 子命令，用于通过正则表达式过滤列表元素。

  • string(TIMESTAMP) 和 file(TIMESTAMP) 命令增加了对 %s 占位符的支持。这是自 UNIX 纪元以来的秒数。

变量 (Variables)¶

  • 引入了 CMAKE_DEPENDS_IN_PROJECT_ONLY 变量，以告知 Makefile 生成器 仅将依赖项扫描限制在项目源代码和构建树中的文件。

  • 引入了一个新的 CMAKE_HOST_SOLARIS 变量，用于指示 CMake 是否在 Oracle Solaris 主机上运行。

  • 添加了 CMAKE_<LANG>_STANDARD_INCLUDE_DIRECTORIES 变量，供工具链文件使用，用于指定要附加到所有编译器命令行中的系统包含目录。

  • CMAKE_<LANG>_STANDARD_LIBRARIES 变量现已编入文档。它旨在供工具链文件使用，用于指定要添加到所有链接器命令行中的系统库。

  • 引入了 CMAKE_NINJA_OUTPUT_PATH_PREFIX 变量，以告知 Ninja 生成器将生成的 build.ninja 文件配置为用作 subninja。

  • 添加了 CMAKE_TRY_COMPILE_PLATFORM_VARIABLES 变量，供工具链文件使用，用于指定必须由 try_compile() 命令传播到测试项目中的特定于平台的变量。

  • 添加了 CMAKE_TRY_COMPILE_TARGET_TYPE 变量，以选择性地告知 try_compile() 命令构建静态库而不是可执行文件。这对于无法在没有自定义标志或脚本的情况下链接二进制文件的交叉编译工具链非常有用。

属性 (Properties)¶

  • 引入了 DEPLOYMENT_REMOTE_DIRECTORY 目标属性，以告知 Visual Studio 9 2008 和 Visual Studio 8 2005 生成器生成用于 WinCE 项目部署和调试器设置的“远程目录”。

  • 引入了 <LANG>_CLANG_TIDY 目标属性及支持的 CMAKE_<LANG>_CLANG_TIDY 变量，以告知 Makefile 生成器 和 Ninja 生成器在编译 C 和 CXX 语言时运行 clang-tidy。

  • 引入了 TIMEOUT_AFTER_MATCH 测试属性，以选择性地告知 CTest 在匹配测试的特定输出后强制执行二次超时。

  • 引入了 VS_CONFIGURATION_TYPE 目标属性，用于为支持 VS 2010 及更高版本的 Visual Studio 生成器 指定自定义项目文件类型。

  • 引入了 VS_STARTUP_PROJECT 目录属性，用于为 Visual Studio 生成器 指定生成的解决方案（.sln 文件）的默认启动项目。

模块 (Modules)¶

  • CMakePushCheckState 模块现在可以推送/弹出/重置 CheckTypeSize 中使用的 CMAKE_EXTRA_INCLUDE_FILES 变量。

  • ExternalProject 模块学会了 GIT_SHALLOW 1 选项，用于对 Git 仓库进行浅克隆。

  • ExternalProject 模块学会了递归初始化 Git 子模块，并在更新时初始化新子模块。使用 GIT_SUBMODULES 选项可限制哪些子模块被初始化和更新。

  • ExternalProject 模块学会了 DOWNLOAD_NO_EXTRACT 1 参数，以跳过提取下载的文件（例如，用于自解压 shell 安装程序或 .msi 文件）。

  • ExternalProject 模块现在在从 git 仓库获取内容时使用 TLS_VERIFY。

  • FindBLAS 和 FindLAPACK 模块学会了支持 OpenBLAS。

  • FindCUDA 模块学会了查找 cublas_device 库。

  • FindGTest 模块的 gtest_add_tests 函数现在会导致 CMake 在测试源发生更改时自动重新运行，以便可以重新扫描它们。

  • 引入了 FindLTTngUST 模块，用于查找 LTTng-UST 库。

  • FindPkgConfig 模块学会了选择性地为已找到的库创建导入目标。

  • FindProtobuf 模块学会了提供 Protobuf_VERSION 变量，并在 find_package() 调用中检查所请求的版本号。

  • InstallRequiredSystemLibraries 模块新增了一个 CMAKE_INSTALL_UCRT_LIBRARIES 选项，以在 Visual Studio 2015 中启用 Windows Universal CRT 库的应用程序本地部署。

平台¶

  • Clang 编译器现在支持 CYGWIN。

  • 增加了对 Bruce C 编译器的支持，编译器 ID 为 Bruce。

CTest¶

  • ctest_update() 命令现在查看 CTEST_GIT_INIT_SUBMODULES 变量，以确定在更新之前是否应更新子模块。

  • ctest_update() 命令现在将在更新时同步子模块。现在将正确拉取添加子模块或更改子模块 URL 的更新。

CPack¶

  • CPack DEB 生成器 学会了在 CPACK_DEBIAN_PACKAGE_SHLIBDEPS 用于依赖项自动检测时，如何处理 CMAKE_INSTALL_RPATH 中的 $ORIGIN。

  • CPack DEB 生成器 学会了如何在软件包包含共享库时生成 DEBIAN/shlibs 控制文件。

  • CPack DEB 生成器 学会了如果软件包将库安装在 ldconfig 控制的位置（例如 /lib/、/usr/lib/），如何生成 DEBIAN/postinst 和 DEBIAN/postrm 文件。

  • CPack DEB 生成器 学会了如果使用多组件设置并且设置了 CPACK_COMPONENT_<compName>_DEPENDS 变量，如何在 Debian 软件包之间生成依赖关系。出于向后兼容性考虑，此功能默认禁用。请参阅 CPACK_DEBIAN_ENABLE_COMPONENT_DEPENDS。

  • CPack DEB 生成器 学会了如何设置自定义软件包文件名，包括如何生成命名正确的 Debian 软件包。

    <PackageName>_<VersionNumber>-<DebianRevisionNumber>_<DebianArchitecture>.deb

    出于向后兼容性考虑，此功能默认禁用。请参阅 CPACK_DEBIAN_FILE_NAME 和 CPACK_DEBIAN_<COMPONENT>_FILE_NAME。

  • CPack DEB 生成器 学会了如何设置软件包发布版本号（与 CPACK_DEBIAN_FILE_NAME 设置的 DEB-DEFAULT 值结合使用时，为软件包文件名中的 DebianRevisionNumber）。请参阅 CPACK_DEBIAN_PACKAGE_RELEASE。

  • CPack DEB 生成器 学会了如何按组件设置软件包架构。请参阅 CPACK_DEBIAN_<COMPONENT>_PACKAGE_ARCHITECTURE。

  • CPack DragNDrop 生成器 学会了一个跳过 /Applications 符号链接的新选项。请参阅 CPACK_DMG_DISABLE_APPLICATIONS_SYMLINK 变量。

  • CPackIFW 模块获得了一个新的 cpack_ifw_update_repository() 命令，用于从远程仓库更新 QtIFW 特定仓库。

  • CPack RPM 生成器 学会了在启用时如何设置 RPM dist 标记作为 RPM Release: 标记的一部分（在某些 Linux 发行版上是强制性的，例如 Fedora）。请参阅 CPACK_RPM_PACKAGE_RELEASE_DIST。

  • CPack RPM 生成器 学会了如何为软件包内容的拥有用户/组和文件/目录权限设置默认值。请参阅 CPACK_RPM_DEFAULT_USER、CPACK_RPM_DEFAULT_GROUP、CPACK_RPM_DEFAULT_FILE_PERMISSIONS、CPACK_RPM_DEFAULT_DIR_PERMISSIONS 及其对应的每个组件设置。

  • CPack RPM 生成器 学会了如何设置用户定义的软件包文件名，如何指定 rpmbuild 应该决定文件名格式，以及如何处理由单个用户定义的 spec 文件生成的多个 rpm 软件包。请参阅 CPACK_RPM_FILE_NAME 和 CPACK_RPM_<component>_FILE_NAME。

  • CPack RPM 生成器 学会了如何正确处理指向生成软件包之外的符号链接。

其他 (Other)¶

  • 编译特性 功能现在可以识别 Intel C++ 编译器版本 12.1 到 16.0 在 UNIX 平台上的支持特性。

已弃用和移除的功能 (Deprecated and Removed Features)¶

  • CMakeForceCompiler 模块及其宏现已弃用。请参阅模块文档了解解释。

  • find_library()、find_path() 和 find_file() 命令不再在非 Windows 平台上从 PATH 环境变量派生的安装前缀中搜索。此行为是在 CMake 3.3 中添加的，旨在支持 Windows 主机，但事实证明在 UNIX 主机上存在问题。仅为了工具而将一些 <prefix>/bin 目录保留在 PATH 中的用户不一定希望搜索任何支持的 <prefix>/lib 目录。可以设置 CMAKE_PREFIX_PATH 环境变量，其中包含 分号分隔的前缀列表，这些前缀将被搜索。

  • Visual Studio 7 .NET 2003 生成器现已弃用，并将于未来的 CMake 版本中移除。

  • Visual Studio 7 生成器（适用于 VS .NET 2002）已被移除。它自 CMake 3.3 起就被弃用。

  • Visual Studio 6 生成器已被移除。它自 CMake 3.3 起就被弃用。

其他更改 (Other Changes)¶

  • cmake.org 提供的预编译 OS X 二进制文件现在需要 OS X 10.7 或更新版本。

  • 在 Linux 和 FreeBSD 平台上，从源代码构建 CMake 本身且不使用系统提供的 libcurl 时，如果系统上找到 OpenSSL，则默认使用它。这为支持通过 https 进行网络通信的命令（例如 file(DOWNLOAD)、file(UPLOAD) 和 ctest_submit()）启用了 SSL/TLS 支持。

  • cmake(1) --build 命令行工具现在拒绝多个 --target 选项并报错，而不是静默忽略除最后一个之外的所有选项。

  • AUTOMOC 现在在不同目录中的多个源文件使用具有相同名称的 #include <moc_foo.cpp> 时会诊断出名称冲突（因为生成的 moc_foo.cpp 文件会发生冲突）。

  • FindBISON 模块的 bison_target() 命令现在通过将 VERBATIM 选项传递给内部 add_custom_command() 调用来支持特殊字符。这可能会破坏为了解决该错误而手动添加转义的客户端。

  • FindFLEX 模块的 flex_target() 命令现在通过将 VERBATIM 选项传递给内部 add_custom_command() 调用来支持特殊字符。这可能会破坏为了解决该错误而手动添加转义的客户端。

  • FindProtobuf 模块的输入和输出变量均已从 PROTOBUF_ 重命名为 Protobuf_，以便与其他 find 模块保持一致。如果提供了旧大小写的输入变量，将予以遵守，并且始终提供旧大小写的输出变量。

  • CPack RPM 生成器 现在支持每个组件特定 CPackRPM 变量中的大写组件名称。例如，名为 foo 的组件现在期望特定于组件的变量为 CPACK_RPM_FOO_PACKAGE_NAME，而之前期望的是 CPACK_RPM_foo_PACKAGE_NAME。变量中的大写组件名称部分与用于其他 CPack 变量的约定兼容。为了向后兼容，旧格式的变量仍然有效，如果同时设置了两个版本的变量，则优先使用旧格式，但未来首选的使用方式是变量中的大写组件名称。以后版本中添加到 CPackRPM 的新变量将仅支持大写组件变量格式。

  • CPack NSIS 生成器的配置文件模板已修复，引用了 CPACK_NSIS_ENABLE_UNINSTALL_BEFORE_INSTALL 选项所使用的卸载程序工具的路径。这避免了依赖不安全的 Windows 功能来运行路径中带有空格的卸载程序工具。
