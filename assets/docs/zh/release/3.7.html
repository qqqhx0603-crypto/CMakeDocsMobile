自 CMake 3.6 以来的变更包括以下内容。

新特性¶

平台¶

  • CMake 现在通过简单的工具链文件支持 Android 交叉编译。

  • Clang 编译器现在可以在 AIX 上使用。

生成器¶

  • Ninja（构建工具） 生成器现在可以在使用具有必要特性的 ninja 工具时，有条件地支持 Fortran。详情请参阅生成器文档。

  • Ninja（构建工具） 生成器现在可以生成形如 sub/dir/{test,install,package} 的伪目标，用于驱动子目录的安装、测试或打包目标的构建。这等同于在 Makefile（构建脚本） 生成器 中使用 cd sub/dir; make {test,install,package}。

  • 新增了 Visual Studio 15 2017 生成器。该生成器处于实验阶段，基于“Visual Studio（集成开发环境） 2017 RC”版本，因为此 VS 版本尚未正式发布。

  • 针对 VS 2010 及更高版本的 Visual Studio（集成开发环境） 生成器 现在可以正确地将 .natvis 源文件放入 VS 项目文件中。

  • Xcode 生成器初步的 Swift 语言支持现在遵循新的 CMAKE_Swift_LANGUAGE_VERSION 变量，以告知 Xcode 源码使用的 Swift 版本。

  • CodeLite 生成器新增了 CMAKE_CODELITE_USE_TARGETS 选项，用于将项目创建方式从项目改为目标。

命令¶

  • add_custom_command() 命令新增了 DEPFILE 选项，该选项与 Ninja（构建工具） 生成器配合使用，可向构建工具提供隐式依赖信息。

  • cmake_parse_arguments() 命令新增了 PARSE_ARGV 模式，用于在 function() 主体内部直接从 ARGC 和 ARGV# 变量中读取参数。

  • export() 命令新增了 ANDROID_MK 选项，用于生成 Android.mk 文件，将 CMake 构建的库引用为 Android NDK 构建系统的预构建库。

  • file(DOWNLOAD) 和 file(UPLOAD) 命令新增了 HTTPHEADER <HTTP-header> 和 USERPWD <username>:<password> 选项。

  • find_library() 和 find_package() 命令现在可以在构建目标为 32 位架构时搜索 lib32/ 目录。请参阅 FIND_LIBRARY_USE_LIB32_PATHS 全局属性。

  • find_package() 命令现在可以通过设置两个新变量 CMAKE_FIND_PACKAGE_SORT_ORDER 和 CMAKE_FIND_PACKAGE_SORT_DIRECTION，按 NAME 或 NATURAL（自然排序）对兼容的库进行排序。

  • if() 命令新增了布尔比较操作符：LESS_EQUAL, GREATER_EQUAL, STRLESS_EQUAL, STRGREATER_EQUAL, VERSION_LESS_EQUAL 以及 VERSION_GREATER_EQUAL。

  • install() 命令新增了 EXPORT_ANDROID_MK 子命令，用于安装 Android.mk 文件，将已安装的库引用为 Android NDK 构建系统的预构建库。

  • string(TIMESTAMP) 和 file(TIMESTAMP) 命令现在支持 %a 和 %b 占位符，分别代表缩写的星期几和月份名称。

  • try_compile() 命令的源文件签名现在会在生成的测试项目中遵循特定配置的标志（例如 CMAKE_<LANG>_FLAGS_DEBUG）。此前仅使用当前工具链的默认此类标志。请参阅策略 CMP0066。

变量¶

  • 添加了变量 CMAKE_FIND_PACKAGE_SORT_ORDER，用于控制 find_package() 命令的排序模式。

  • 添加了变量 CMAKE_FIND_PACKAGE_SORT_DIRECTION，用于控制 find_package() 命令的排序方向。

  • 工具链文件 现在可以设置 CMAKE_<LANG>_FLAGS_INIT 变量，在构建树中首次启用某种语言时，初始化 CMAKE_<LANG>_FLAGS 缓存条目。

  • 工具链文件 现在可以设置 CMAKE_EXE_LINKER_FLAGS_INIT、CMAKE_SHARED_LINKER_FLAGS_INIT 和 CMAKE_MODULE_LINKER_FLAGS_INIT 变量，在构建树中首次启用某种语言时，初始化 CMAKE_EXE_LINKER_FLAGS、CMAKE_SHARED_LINKER_FLAGS 和 CMAKE_MODULE_LINKER_FLAGS 缓存条目。

属性¶

  • 在 Apple 平台上，BUNDLE_EXTENSION 目标属性现在也适用于 Framework 和 App Bundle。

  • 添加了 BINARY_DIR 目录属性，用于获取读取该属性的源目录所对应的二进制目录的绝对路径。

  • 添加了 BUILDSYSTEM_TARGETS 目录属性，用于获取项目中在某个目录下添加的逻辑构建系统目标名称列表。

  • 引入了 LINK_WHAT_YOU_USE 目标属性及支持变量 CMAKE_LINK_WHAT_YOU_USE，用于通过运行 ldd -r -u 检测（在 UNIX 上）已链接但不必要的共享库。

  • 添加了 SOURCE_DIR 目录属性，用于获取与某个目录关联的源目录的绝对路径。

  • 添加了 SUBDIRECTORIES 目录属性，用于获取项目中在某个目录下添加的子目录列表。

  • 添加了 VS_SDK_REFERENCES 目标属性，用于告知 Visual Studio（集成开发环境） 生成器 引用指定的 SDK。

  • 创建了 VS_TOOL_OVERRIDE 源文件属性，用于告知 Visual Studio（集成开发环境） 生成器 对某个源文件使用什么工具。

  • WINDOWS_EXPORT_ALL_SYMBOLS 目标属性现在适用于设置了 ENABLE_EXPORTS 属性的可执行目标。

  • 添加了 XCODE_FILE_ATTRIBUTES 源文件属性，用于告知 Xcode 生成器为该文件在 Xcode 项目属性中生成自定义内容。

模块¶

  • 添加了 AndroidTestUtilities 模块，用于管理向 Android 设备传输测试数据。

  • CheckFortranSourceCompiles 模块的 check_fortran_source_compiles() 命令新增了 SRC_EXT 选项，用于指定自定义测试 Fortran 源文件扩展名。

  • ExternalProject 模块新增了 HTTP_USERNAME 和 HTTP_PASSWORD 选项，用于设置 HTTP 下载凭据。

  • ExternalProject 模块新增了 HTTP_HEADER 选项，用于添加 HTTP 下载标头。

  • FindBISON 模块的 bison_target() 命令新增了 REPORT_FILE 选项，用于指定 bison 的 --report-file= 选项。

  • FindBZip2 模块现在提供导入目标。

  • 引入了 FindICU 模块，用于查找 International Components for Unicode (ICU) 库和程序。

  • FindMatlab 模块现在可以查找 SIMULINK 和 MAT 组件。

  • FindMatlab 模块的 matlab_add_mex() 命令现在可以添加可执行文件和模块。

  • FindMatlab 模块的 matlab_add_unit_test() 命令现在支持内联 Matlab 测试代码。

  • FindOpenCL 模块现在提供导入目标。

  • FindOpenMP 模块现在可以从编译器检测 OpenMP 版本（规范日期）。

  • 添加了 FindVulkan 模块。

  • GenerateExportHeader 模块新增了 CUSTOM_CONTENT_FROM_VARIABLE 选项，用于指定一个包含自定义内容的变量，以便包含在生成的头文件中。

  • GNUInstallDirs 模块新增了 GNUInstallDirs_get_absolute_install_dir() 命令。

  • UseJava 模块新增了 API，用于“导出” jar 目标，以供外部 CMake 项目使用。请参阅 install_jar_exports 和 export_jars 函数。

CTest¶

  • CTest 现在通过新的 FIXTURES_SETUP、FIXTURES_CLEANUP 和 FIXTURES_REQUIRED 测试属性支持测试固件（test fixtures）。当使用正则表达式或 --rerun-failed 限制要运行的测试时，如果任何测试需要该固件，固件的设置（setup）和清理（cleanup）测试将自动添加到执行集中。

  • ctest_configure()、ctest_build()、ctest_test()、ctest_coverage() 和 ctest_upload() 命令新增了 CAPTURE_CMAKE_ERROR 选项，用于捕获命令运行时发生的任何错误并存入变量，避免影响 ctest(1) 进程的返回码。

CPack¶

  • CPack 在 OS X 上新增了 CPack productbuild 生成器。

  • CPack 新增了 CPACK_PACKAGE_CHECKSUM 变量，用于为每个包文件启用校验和文件生成。

  • CPack DEB 生成器 在归档格式设置为 GNU tar 时支持长文件名。请参阅 CPACK_DEBIAN_ARCHIVE_TYPE。

  • CPackIFW 模块新增了 cpack_ifw_add_package_resources() 命令，用于在安装程序二进制文件中包含额外资源。

  • CPackIFW 模块的 cpack_ifw_configure_component() 和 cpack_ifw_configure_component_group() 命令新增了 USER_INTERFACES 选项，用于向 IFW 安装程序添加额外页面列表。

  • CPack RPM 生成器 现在可以按需生成 debuginfo 包。请参阅 CPACK_RPM_DEBUGINFO_PACKAGE 及其按组件版本。

  • CPack RPM 生成器 现在可以按需生成源代码 rpm (SRPM) 包。请参阅 CPACK_RPM_PACKAGE_SOURCES、CPACK_RPM_SOURCE_PKG_BUILD_PARAMS 和 CPACK_RPM_SOURCE_PKG_PACKAGING_INSTALL_PREFIX。

  • CPack NSIS 生成器 现在支持 CPACK_NSIS_<compName>_INSTALL_DIRECTORY，可用于设置组件特定的安装目录。

  • CPack WIX 生成器 现在支持 CPACK_WIX_SKIP_PROGRAM_FOLDER，允许在 ProgramFiles 文件夹之外指定自定义的绝对安装前缀。

  • CPack WIX 生成器 现在支持 CPACK_COMPONENT_<compName>_DISABLED，可用于取消默认安装某组件的选择。

  • CPack WIX 生成器 现在支持用于 Feature 元素的 CPACK_WIX_PATCH_FILE 片段。

  • CPack WIX 生成器 现在支持 CPACK_WIX_ROOT_FEATURE_TITLE 和 CPACK_WIX_ROOT_FEATURE_DESCRIPTION，允许为根特性（root feature）元素指定自定义标题和描述。

其他 (Other)¶

  • cmake(1) 新增了 -E capabilities 选项，用于提供 cmake 工具功能的机器可读（JSON）描述（可用生成器等）。

  • 新增了 cmake-server(7) 模式，通过 JSON 协议向客户端提供有关 CMake 生成的构建系统的语义信息。目前所有协议均为实验性，可能会有变动。

  • cmake(1) 命令新增了 --trace-source=<file> 选项。

  • ccmake(1) 现在支持类 vim 的导航绑定。

  • cmake-gui(1) 新增了一个按钮，用于打开 Visual Studio（集成开发环境） 生成器 和 Xcode 生成器生成的项目文件。

已弃用和移除的功能 (Deprecated and Removed Features)¶

  • 我们不再在 cmake.org 上提供用于下载的 Linux i386 二进制文件，适用于新版本的 CMake。

  • Vim 支持文件 cmake-indent.vim、cmake-syntax.vim 和 cmake-help.vim 已被移除，转而推荐使用 vim-cmake-syntax 项目提供的文件。

  • 放弃了对使用某些编译器构建 CMake 自身的支持：

    • Visual Studio（集成开发环境） 7.1 和 2005 —— 已被 VS 2008 及更高版本取代

    • MinGW.org mingw32 —— 已被 MSYS2 mingw32 和 mingw64 取代

    CMake 仍然支持为使用这些编译器的其他项目生成构建系统。

其他更改 (Other Changes)¶

  • Fortran 依赖扫描器现在支持 Fortran 子模块 (Submodules) 的语法。

  • 来自 vim-cmake-syntax 项目的 Vim 支持文件 indent/cmake.vim 和 syntax/cmake.vim 现在随 CMake 一起发布。
