自 CMake 4.0 以来的变更包括以下内容。

新功能¶

基于文件的 API¶

  • cmake-file-api(7) v1 现在在构建系统生成失败并报错时会写入部分回复。请参阅 v1 回复错误索引。

生成器¶

  • Makefile 生成器 和 Ninja 生成器 增加了对为 Fortran、CUDA 和 HIP 添加链接器启动器（linker launcher）的支持。详情请参阅 CMAKE_<LANG>_LINKER_LAUNCHER 变量和 <LANG>_LINKER_LAUNCHER 目标属性。

命令行¶

  • 当使用 Xcode 生成器时，cmake --build 命令行工具现在可以检测第三方工具是否已将生成的 .xcodeproj 封装在 .xcworkspace 中，并改为通过工作区（workspace）进行构建。

配置日志¶

  • cmake-configure-log(7) 现在会在以下情况记录来自 find_package()（在 CONFIG 模式下）、find_path()、find_file()、find_library() 和 find_program() 命令的事件：首次调用时、结果在“未找到”与“找到”之间转换时，或者通过 --debug-find 命令行选项显式启用时。请参阅 事件类型 find 和 事件类型 find_package。日志记录也可以通过 CMAKE_FIND_DEBUG_MODE 和 CMAKE_FIND_DEBUG_MODE_NO_IMPLICIT_CONFIGURE_LOG 变量进行控制。

编译器¶

  • 来自 Wind River Systems 的 Diab 编译器（版本 5.9.x+）现在支持 ASM、C 和 CXX 语言，编译器 ID 为 Diab。

  • 瑞萨电子（Renesas）编译器现在支持 ASM 和 C 语言，编译器 ID 为 Renesas。

命令¶

  • add_dependencies() 命令现在可以在没有依赖项的情况下调用。

  • cmake_pkg_config() 命令现在支持 IMPORT 和 POPULATE 子命令，用于将 CMake 目标与基于 pkg-config 的依赖项进行对接。

  • project() 命令现在通过 CMAKE_EXPERIMENTAL_EXPORT_PACKAGE_INFO 对 COMPAT_VERSION 关键字提供了实验性支持。

变量¶

  • 添加了 CMAKE_FIND_REQUIRED 变量，用于告知 find_package()、find_path()、find_file()、find_library() 和 find_program() 默认情况下应设为 REQUIRED。这些命令还增加了 OPTIONAL 关键字，以便在特定调用中忽略该变量。

  • CMAKE_<LANG>_COMPILER_ARCHITECTURE_ID 变量现在已为大多数编译器填充，并记录供公开使用。

  • 添加了 CMAKE_<LANG>_ICSTAT 变量和相应的 <LANG>_ICSTAT 目标属性，用于告知 Makefile 生成器 和 Ninja 生成器 在编译 C 和 CXX 语言时与编译器一起运行 IAR icstat 工具。

环境变量¶

  • 添加了 CMAKE_<LANG>_IMPLICIT_LINK_LIBRARIES_EXCLUDE 环境变量，用于可选择性地从检测到的 CMAKE_<LANG>_IMPLICIT_LINK_LIBRARIES 集合中排除特定的库。

属性¶

  • 添加了 AUTOMOC_INCLUDE_DIRECTORIES 目标属性及相关的 CMAKE_AUTOMOC_INCLUDE_DIRECTORIES 变量，用于覆盖从目标的传递包含目录中自动发现 moc 包含文件。

  • MACOSX_PACKAGE_LOCATION 源文件属性现在可以在源目录上设置，并将整个树复制到包（bundle）中。

  • PDB_NAME 和 COMPILE_PDB_NAME 目标属性现在支持 生成器表达式。

模块¶

  • FindASPELL 模块现在提供版本变量、导入的目标以及可选择分别选择 Aspell 库和可执行文件的组件。

  • FindBLAS 和 FindLAPACK 模块现在支持 NVIDIA 性能库 (NVPL)。

  • FindProtobuf 模块的 protobuf_generate(DEPENDENCIES) 命令参数现在接受多个值。

  • FindProtobuf 模块的 protobuf_generate_cpp() 和 protobuf_generate_python() 命令，连同它们的 Protobuf_IMPORT_DIRS 和 PROTOBUF_GENERATE_CPP_APPEND_PATH 提示变量，现已弃用，建议改用 protobuf_generate() 命令。

正则表达式¶

  • string(REGEX MATCHALL)、string(REGEX REPLACE) 和 list(TRANSFORM REPLACE) 命令现在在重复搜索时，正则表达式的 ^ 锚点最多匹配一次（位于输入开头）。请参阅策略 CMP0186。

  • string(REGEX REPLACE) 命令现在允许引用未匹配的组，它们会被替换为空字符串。

  • string(REGEX MATCH)、string(REGEX MATCHALL) 和 string(REGEX REPLACE) 命令现在允许零长度匹配。

CTest¶

  • ctest(1) 增加了 --schedule-random-seed 选项，用于指定数字随机种子，使 ctest --schedule-random 的结果具有确定性，便于复现。

CPack¶

  • CPack NuGet 生成器 增加了 CPACK_NUGET_SYMBOL_PACKAGE 选项，用于生成包含 PDB 文件的 NuGet 符号包。

  • CPack RPM 生成器 增加了 CPACK_RPM_PACKAGE_ENHANCES、CPACK_RPM_PACKAGE_RECOMMENDS 和 CPACK_RPM_PACKAGE_SUPPLEMENTS 变量，以指定相应的 RPM spec 字段。

弃用和移除的功能¶

  • FindGTest 模块的结果变量 GTEST_INCLUDE_DIRS、GTEST_LIBRARIES、GTEST_MAIN_LIBRARIES 和 GTEST_BOTH_LIBRARIES 现已弃用，建议改用 GTest::gtest 和 GTest::gtest_main 导入目标。

  • FindGCCXML 模块已通过策略 CMP0188 弃用。请将项目迁移至 CastXML。

  • FindCABLE 模块已通过策略 CMP0191 弃用。

  • CMakeDetermineVSServicePack 模块已通过策略 CMP0196 弃用。请将项目改用 CMAKE_<LANG>_COMPILER_VERSION 变量。

其他更改¶

  • ExternalProject 模块不再检查 URL 归档文件的扩展名。现在支持 cmake -E tar 可以解压的任何归档类型。

  • FindPython3、FindPython2 和 FindPython 模块现在在交叉编译模式下强制要求制品的一致性。这防止了宿主机制品与目标制品混用。请参阅策略 CMP0190。

  • GNUInstallDirs 模块现在倾向于在安装到特殊前缀时，将 SYSCONFDIR、LOCALSTATEDIR 和 RUNSTATEDIR 默认为绝对路径。请参阅策略 CMP0192。

  • GNUInstallDirs 模块现在会缓存安装前缀为 / 时，带有前导 usr/ 的 CMAKE_INSTALL_* 变量。请参阅策略 CMP0193。

  • install(TARGETS) 命令不再忽略调用时尚未定义的文件集（file set）。target_sources(FILE_SET) 和 install(TARGETS) 的顺序不再具有语义相关性。

  • 启用 ASM 时，不再会意外地成功使用 MSVC 的 cl C 编译器作为汇编器。请参阅策略 CMP0194。

  • MSVC 链接器的 -machine: 标志不再添加到 CMAKE_*_LINKER_FLAGS 变量中。请参阅策略 CMP0197。

  • Visual Studio 生成器 现在会抑制 Visual Studio 的默认编译器标志 -Zc:forScope、-Zc:inline 和 -Zc:wchar_t，以及链接器标志 -dynamicbase、-errorreport、-nxcompat 和 -safeseh（如果项目或用户未指定它们）。这使得构建过程与其他生成器更加一致，也更符合项目和用户实际指定的内容。

  • TARGET_PROPERTY 生成器表达式现在会传递性地评估 LINK_LIBRARIES 和 INTERFACE_LINK_LIBRARIES 目标属性。请参阅策略 CMP0189。

更新¶

自 CMake 4.1.0 以来的变更包括以下内容。

4.1.1¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。

4.1.2¶

  • file(GENERATE) 命令在评估生成器表达式时，现在会使用每个调用点处的策略 CMP0189 的值。之前它使用的是目录 CMakeLists.txt 结束时的值，这与其他所有生成器表达式的评估方式相同。

4.1.3¶

  • 当使用 Xcode 生成器为 macOS 构建时，CMAKE_OSX_DEPLOYMENT_TARGET 现在会再次默认使用宿主机的 macOS 版本（如果该版本早于 macOS SDK 版本），这与 CMake 4.0 之前的行为一致。

  • execute_process() 命令在达到 TIMEOUT 时会再次终止子进程。此问题在 CMake 3.29 中被意外引入。

4.1.4, 4.1.5, 4.1.6¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。
