自 CMake 3.28 以来的变更包括以下内容。

新功能¶

命令行¶

  • cmake(1) -E cat 现在可以通过传递 - 参数来打印标准输入。

生成器¶

  • Visual Studio 生成器 现在支持通过 CMAKE_GENERATOR_TOOLSET 中的 fortran= 字段在 Intel oneAPI Fortran 编译器（ifx）和 Intel 经典 Fortran 编译器（ifort）之间进行选择。

基于文件的 API¶

  • cmake-file-api(7) 的“codemodel”第 2 版 version 字段已更新至 2.7。

  • cmake-file-api(7) 的“codemodel”第 2 版“target”对象增加了一个新的“launchers”字段。

编译器¶

  • Windows 上的 LLVM/Clang 类 GNU 前端（clang++）现在可用于编译 CUDA 语言源文件。

  • Windows 上以 GNU ABI 为目标的编译器（MinGW）现在可用于编译 Objective C (OBJC) 和 Objective C++ (OBJCXX)。这些编译器包括 GNU 编译器（gcc 和 g++）以及 LLVM/Clang 类 GNU 前端（clang 和 clang++）。

  • 基于 TI Clang 的编译器现在通过 compiler id TIClang 得到支持。

命令¶

  • add_custom_command(TARGET) 签名现在支持通过 别名目标 (Alias Targets) 添加构建事件。

  • 新增了 cmake_language(EXIT) 子命令，用于以指定的退出代码终止 cmake -P 脚本。

  • 新增了 export(SETUP) 子命令，用于配置导出集。其 TARGET 选项的 XCFRAMEWORK_LOCATION 设置指定了可替代已安装目标的 .xcframework 的位置。

  • if() 命令增加了新的测试项 IS_READABLE、IS_WRITABLE 和 IS_EXECUTABLE，用于检查文件或目录权限。

  • try_compile() 和 try_run() 命令增加了 LINKER_LANGUAGE 选项，用于在生成的测试项目中指定 LINKER_LANGUAGE 目标属性。

变量¶

  • 新增了 CMAKE_INSTALL_PREFIX 环境变量，用于为 CMAKE_INSTALL_PREFIX 变量提供默认值。

  • 新增了 CMAKE_LINKER_TYPE 变量和相应的 LINKER_TYPE 目标属性，用于指定某些工具链使用的链接器。

  • 新增了 CMAKE_<LANG>_COMPILER_LINKER、CMAKE_<LANG>_COMPILER_LINKER_ID、CMAKE_<LANG>_COMPILER_LINKER_VERSION 和 CMAKE_<LANG>_COMPILER_LINKER_FRONTEND_VARIANT 变量，用于描述该语言链接步骤所使用的链接器。

  • CMAKE_PROJECT_INCLUDE、CMAKE_PROJECT_INCLUDE_BEFORE、CMAKE_PROJECT_<PROJECT-NAME>_INCLUDE 和 CMAKE_PROJECT_<PROJECT-NAME>_INCLUDE_BEFORE 变量现在支持以 分号分隔的列表 形式指定 CMake 语言文件，以便按顺序包含它们。这些变量还可以引用在 CMAKE_MODULE_PATH 中找到或 CMake 内置的模块名称。

  • 新增了 CMAKE_SKIP_TEST_ALL_DEPENDENCY 变量，用于控制 test（或 RUN_TESTS）构建系统目标是否依赖于 all（或 ALL_BUILD）目标。

  • 新增了 CMAKE_TEST_LAUNCHER 变量和相应的 TEST_LAUNCHER 目标属性，用于指定当通过 add_test() 命令添加测试时，可执行目标使用的启动器。

属性¶

  • CROSSCOMPILING_EMULATOR 目标属性现在支持 生成器表达式。

  • UNITY_BUILD 目标属性现在支持 Objective C (OBJC) 和 Objective C++ (OBJCXX) 语言。

  • 新增了 XCODE_EMBED_XPC_SERVICES 目标属性，用于告知 Xcode 生成器将哪些目标放入 Embed XPC Resources 构建阶段。

模块¶

  • CMakePackageConfigHelpers 模块增加了新的 generate_apple_platform_selection_file() 和 generate_apple_architecture_selection_file() 函数，分别可用于生成包含特定于 Apple 平台的其他文件，或包含软件包针对 Apple 平台的特定架构实现的文件。

  • FindOpenGL 模块现在可以查找与 GL 包含目录不同的 GLU 包含目录。新的 OPENGL_INCLUDE_DIRS 结果变量提供了所有包含目录。

CTest¶

  • ctest(1) 增加了 --http-header 选项，用于在向 CDash 提交时添加自定义标头。

  • ctest(1) 增加了 --tests-from-file 和 --exclude-from-file 选项，用于运行或排除文件中指定的测试。

  • ctest(1) 现在支持 POSIX 系统上的 任务服务器集成。

  • ctest -j 选项现在可以不带值使用，以让 ctest 选择默认的并行级别，或者使用 0 让 ctest 使用无限制的并行级别。相应的 CTEST_PARALLEL_LEVEL 环境变量如果设置为空字符串，现在等同于传递不带值的 -j。

  • ctest_test() 命令增加了 INCLUDE_FROM_FILE 和 EXCLUDE_FROM_FILE 选项，用于运行或排除文件中指定的测试。

CPack¶

  • CPack DEB 生成器 的 CPACK_DEBIAN_FILE_NAME 变量现在可以设置为不带任何后缀，系统将自动添加 .deb 后缀。

  • CPack RPM 生成器 的 CPACK_RPM_FILE_NAME 变量现在可以设置为不带任何后缀，系统将自动添加 .rpm 后缀。

  • CPack WIX 生成器 增加了一个新变量 CPACK_WIX_INSTALL_SCOPE，用于控制 WiX MSI 安装程序的 InstallScope 属性。

其他更改¶

  • CMake 现在能够根据链接器功能对链接行上的库进行去重。参见策略 CMP0156。

  • add_test() 命令现在仅在交叉编译时遵循 CMAKE_CROSSCOMPILING_EMULATOR。参见策略 CMP0158。

  • 在 Windows 上，当以 MSVC ABI 为目标时，find_library() 命令现在在考虑 .lib 之后也会接受 .a 文件名。这与以 GNU ABI 为目标时的现有行为对称，即该命令在考虑 .a 之后也会接受 .lib 文件名。

  • 在 Windows 上，当以 MSVC ABI 为目标时，find_library() 命令现在在 .lib 之前考虑 .dll.lib 文件名。这是 Rust 工具链为 MSVC ABI 创建的 DLL 导入库的默认后缀。

  • Ninja 和 NMake Makefiles 生成器现在从 2021.4 版本开始，在使用 IntelLLVM 时对系统包含使用 -external:I 标志。从 2022.2 版本开始，还会使用 -external:W0 标志。

  • create_test_sourcelist() 命令现在提供生成的驱动程序源文件的完整路径。

  • CPACK_PRODUCTBUILD_DOMAINS 变量现在默认为 true。参见策略 CMP0161。

更新¶

自 CMake 3.29.0 以来的变更包括以下内容。

3.29.1¶

  • CMAKE_LINKER_TYPE 变量和相应的 LINKER_TYPE 目标属性现在适用于 Swift 语言的编译器。

3.29.2, 3.29.3, 3.29.4¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.29.5¶

  • CPack WIX 生成器 的 CPACK_WIX_INSTALL_SCOPE 变量（3.29 新增）现在默认为 NONE。这恢复了与 3.28 及以下版本行为的兼容性：如果没有自定义 WiX 模板，它生成的安装程序将仅为当前用户创建开始菜单和卸载条目，即使它们是为所有用户安装的。

    在 3.29.0 到 3.29.4 版本中，CPACK_WIX_INSTALL_SCOPE 默认为 perMachine。这创建的 MSI 安装程序默认会为所有用户创建开始菜单和卸载条目。虽然这种行为本身更好，但这些安装程序无法干净地替换使用 3.28 及以下版本生成的 MSI 安装程序所创建的现有安装。3.29.5 恢复了此默认值以保持兼容性。项目可以根据自己的计划，通过设置 CPACK_WIX_INSTALL_SCOPE 自行过渡到 perMachine。

3.29.6, 3.29.7, 3.29.8, 3.29.9¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。
