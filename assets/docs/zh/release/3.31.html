自 CMake 3.30 以来的变更包括以下内容。

新功能¶

预设¶

  • cmake-presets(7) 文件现在可以在 JSON 对象的任意层级中使用 $comment 键添加注释，以提供文档说明。

  • cmake-presets(7) 文件现在可以通过配置预设中的 graphviz 键请求输出 graphviz 文件。

生成器¶

  • Ninja 生成器 和 Makefile 生成器 现在会生成一个 codegen 构建目标。参见策略 CMP0171。它会驱动构建图中足以运行 add_custom_command() 新增的 CODEGEN 选项所创建的自定义命令的子集。

命令行¶

  • cmake --workflow 模式现在接受预设名称作为第一个参数，允许使用更简洁的命令行 cmake --workflow <preset>。

  • 新增了 cmake -LR[A][H] 选项，用于列出名称与正则表达式匹配的缓存条目。

编译器¶

  • 现在支持 LFortran 编译器，其 编译器标识 为 LFortran。

命令¶

  • add_custom_command() 命令增加了 CODEGEN 选项，用于将自定义命令的输出标记为 codegen 目标的依赖项。参见策略 CMP0171。

  • 新增了 cmake_pkg_config() 命令，作为使用 CMake 原生 pkg-config 格式解析器的入口点。此版本中唯一支持的选项是 EXTRACT，它提供了对解析 pkg-config 文件所产生值的底层访问。对于大多数用户而言，这尚不足以替代 FindPkgConfig 模块。

  • file(ARCHIVE_CREATE) 命令增加了 WORKING_DIRECTORY 选项，用于指定归档过程的工作目录。

  • file(MAKE_DIRECTORY) 命令增加了 RESULT 选项，用于在结果变量中捕获错误。

  • install(FILES) 和 install(DIRECTORY) 命令的 TYPE 参数增加了对 LIBEXEC 类型的支持。

变量¶

  • 新增了 CMAKE_AIX_SHARED_LIBRARY_ARCHIVE 变量和相应的 AIX_SHARED_LIBRARY_ARCHIVE 目标属性，用于在 AIX 上创建作为共享库归档文件的共享库。

  • 新增了 CMAKE_EXPORT_BUILD_DATABASE 变量、相应的 CMAKE_EXPORT_BUILD_DATABASE 环境变量以及 EXPORT_BUILD_DATABASE 目标属性，用于启用导出 C++ 模块编译命令。此功能仅支持 Ninja 生成器。

  • 新增了 CMAKE_HOST_EXECUTABLE_SUFFIX 变量，用于提供宿主平台上可执行文件的后缀。

  • 新增了 CMAKE_<LANG>_HOST_COMPILER_ID 和 CMAKE_<LANG>_HOST_COMPILER_VERSION 变量，其中 <LANG> 为 CUDA 或 HIP。当 CMAKE_<LANG>_COMPILER_ID 为 NVIDIA 时，这些变量会被填充以标识 NVCC 的宿主编译器。

  • 新增了 CMAKE_<LANG>_STANDARD_LINK_DIRECTORIES 变量。工具链文件可以设置此变量，以控制哪些链接库目录路径始终传递给指定语言的编译器。

  • 新增了 CMAKE_LINK_LIBRARIES_STRATEGY 变量和相应的 LINK_LIBRARIES_STRATEGY 目标属性，用于可选地指定 CMake 生成链接行所使用的策略。

  • 新增了 CMAKE_CONFIG_DIR 环境变量，用于指定 cmake-file-api(7) 和 cmake-instrumentation(7) 查询的 CMake 用户级配置目录。

属性¶

  • 新增了 MACOSX_FRAMEWORK_BUNDLE_NAME 目标属性，用于在 Apple FRAMEWORK 的 Info.plist 文件中设置 CFBundleName 键。

  • 新增了 UNITY_BUILD 目标属性，现在支持 CUDA 语言。

  • 新增了 VS_FRAMEWORK_REFERENCES 目标属性，用于通知 Visual Studio 生成器 添加框架引用。

模块¶

  • 检查模块现在支持 CMAKE_REQUIRED_LINK_DIRECTORIES 变量。以下模块获得了此支持：

    • CMakePushCheckState

    • CheckCCompilerFlag

    • CheckCSourceCompiles

    • CheckCSourceRuns

    • CheckCXXCompilerFlag

    • CheckCXXSourceCompiles

    • CheckCXXSourceRuns

    • CheckCXXSymbolExists

    • CheckCompilerFlag

    • CheckFortranCompilerFlag

    • CheckFortranFunctionExists

    • CheckFortranSourceCompiles

    • CheckFortranSourceRuns

    • CheckFunctionExists

    • CheckIncludeFile

    • CheckIncludeFileCXX

    • CheckIncludeFiles

    • CheckOBJCCompilerFlag

    • CheckLibraryExists

    • CheckOBJCCompilerFlag

    • CheckOBJCSourceCompiles

    • CheckOBJCSourceRuns

    • CheckOBJCXXCompilerFlag

    • CheckOBJCXXSourceCompiles

    • CheckOBJCXXSourceRuns

    • CheckPrototypeDefinition

    • CheckSourceCompiles

    • CheckSourceRuns

    • CheckStructHasMember

    • CheckSymbolExists

    • CheckTypeSize

    • CheckVariableExists

  • CMakePackageConfigHelpers 模块的 generate_apple_platform_selection_file() 函数增加了对 iOS Mac Catalyst 的支持。

  • GoogleTest 模块的 gtest_discover_tests() 命令增加了一个新的 DISCOVERY_EXTRA_ARGS 关键字。它允许在查询测试列表时向命令行追加额外参数。

  • FindCUDAToolkit 模块现在提供了一个 CUDA::nvml_static 目标。

  • FindOpenMP 模块增加了对 CUDA 语言的支持。

CTest¶

  • ctest_submit() 命令和 ctest -T Submit 步骤现在默认验证连接到 https:// URL 的 TLS 服务器证书。详情请参见 CTEST_TLS_VERIFY 变量。

  • ctest_submit() 命令和 ctest -T Submit 步骤现在默认要求连接到 https:// URL 时使用 TLS 1.2 或更高版本。详情请参见 CTEST_TLS_VERSION 变量。

CPack¶

  • CPack DEB 生成器 增加了 CPACK_DEBIAN_PACKAGE_MULTIARCH 选项，以支持多架构软件包。

  • CPack IFW 生成器 增加了新的 CPACK_IFW_PACKAGE_PRODUCT_IMAGE_URLS 变量，用于指定与 CPACK_IFW_PACKAGE_PRODUCT_IMAGES 条目关联的图像。此功能适用于 QtIFW 4.0 及更高版本。

  • CPack RPM 生成器 增加了对 zstd 作为 CPACK_RPM_COMPRESSION_TYPE 值的支持。

  • CPack 模块在 CPack WIX 生成器 中默认启用按机器安装。参见策略 CMP0172 和 CPACK_WIX_INSTALL_SCOPE 变量。

已弃用和已删除的特性¶

  • 对旧于 3.10 版本的 CMake 的兼容性现已被弃用，并将从未来的版本中移除。调用 cmake_minimum_required() 或 cmake_policy() 并将策略版本设置为较旧值时，现在会发出弃用诊断。

  • CMakeFindFrameworks 模块已通过 CMP0173 弃用。项目应改用 find_library()。

  • Visual Studio 12 2013 生成器已被移除。

其他更改¶

  • 当链接行上的静态库被去重时（根据策略 CMP0156），现在所有平台上都会保留第一次出现的库。参见策略 CMP0179。

  • TEST_LAUNCHER 和 CROSSCOMPILING_EMULATOR 目标属性中的空列表元素现在由以下命令保留：

    • add_test() 命令。

    • ExternalData_Add_Test() 命令（来自 ExternalData 模块）。

    • gtest_add_tests() 和 gtest_discover_tests() 命令（来自 GoogleTest 模块）。这两个命令在 EXTRA_ARGS 关键字之后的空列表元素现在也被保留。

    参见策略 CMP0178。

  • execute_process() 命令的 ENCODING 选项（在 Windows 上有意义）现在默认设置为 UTF-8。参见策略 CMP0176。

  • file(DOWNLOAD) 和 file(UPLOAD) 命令现在默认验证连接到 https:// URL 的 TLS 服务器证书。详情请参见 CMAKE_TLS_VERIFY 变量。此变更未关联任何策略，以确保即使用户构建的项目未更新也能受到保护。用户可以将 CMAKE_TLS_VERIFY 环境变量设置为 0 来恢复旧的默认行为。

  • file(DOWNLOAD) 和 file(UPLOAD) 命令现在默认要求连接到 https:// URL 时使用 TLS 1.2 或更高版本。详情请参见 CMAKE_TLS_VERSION 变量。

  • file(GET_RUNTIME_DEPENDENCIES) 命令已更新，以更贴近 Linux 上动态加载器的行为。

  • install() 命令的 DESTINATION 参数现在被 标准化，但在 install(TARGETS) 中的 INCLUDES DESTINATION 参数除外。参见策略 CMP0177。

  • project() 命令现在始终将 <PROJECT-NAME>_SOURCE_DIR、<PROJECT-NAME>_BINARY_DIR 和 <PROJECT-NAME>_IS_TOP_LEVEL 设置为普通变量和缓存条目。参见策略 CMP0180。

  • cmake_parse_arguments(PARSE_ARGV) 命令现在为单值关键字后的空字符串定义一个变量。参见策略 CMP0174。

更新¶

自 CMake 3.31.0 以来的变更包括以下内容。

3.31.1, 3.31.2, 3.31.3, 3.31.4, 3.31.5, 3.31.6, 3.31.7, 3.31.8, 3.31.9¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.31.10¶

  • execute_process() 命令再次在达到 TIMEOUT 时终止子进程。这在 CMake 3.29 中被意外退化。

3.31.11, 3.31.12¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。
