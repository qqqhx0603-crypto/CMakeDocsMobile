自 CMake 3.25 以来的更改如下。

新功能¶

语言¶

  • 添加了 ASM_MARMASM 语言以支持 Microsoft ARM 汇编语言。

命令行¶

  • cmake -E copy 命令行工具现在支持 -t 参数。

  • 添加了 cmake -E copy_directory_if_different 命令行工具。

配置日志¶

  • CMake 现在将配置时检查的 YAML 日志写入构建目录顶层的 CMakeFiles/CMakeConfigureLog.yaml 中。请参阅 cmake-configure-log(7) 手册。

基于文件的 API¶

  • cmake-file-api(7) 的 "codemodel" 2 版本中的 version 字段已更新为 2.5。

  • cmake-file-api(7) "codemodel" 2 版本中的 "target" 对象新增了 fileSets 字段，并向 sources 对象添加了关联的 fileSetIndex 字段。

  • cmake-file-api(7) "codemodel" 2 版本中的 "directory" 对象 installers 字段新增了 cxxModuleBmi 安装程序类型以及关联的 cxxModuleBmiTarget 字段。

  • cmake-file-api(7) 新增了一种 "configureLog" 对象类型，允许以稳定方式访问 cmake-configure-log(7)。

命令¶

  • add_custom_command() 和 add_custom_target() 命令现在支持在其 COMMENT 选项中使用 生成器表达式。

  • message() 命令新增了 CONFIGURE_LOG 模式，用于在 cmake-configure-log(7) 中记录条目。

  • string(TIMESTAMP) 和 file(TIMESTAMP) 命令现在支持用于时区的 %z 和 %Z 说明符。

  • try_compile() 和 try_run() 命令新增了 LOG_DESCRIPTION 选项，用于指定要在 cmake-configure-log(7) 中记录的文本。

  • try_compile() 和 try_run() 命令新增了 NO_LOG 选项，用于跳过记录 cmake-configure-log(7) 条目。

变量¶

  • CMAKE_<LANG>_COMPILER_FRONTEND_VARIANT 变量现在为仅有一种前端变体的 GNU、MSVC 和 AppleClang 编译器进行设置。

  • CMAKE_VS_VERSION_BUILD_NUMBER 变量现在由 Visual Studio（集成开发环境） 生成器 为 VS 2017 及以上版本设置，用以报告四部分的 Visual Studio（集成开发环境） 版本号。

属性¶

  • <LANG>_CLANG_TIDY_EXPORT_FIXES_DIR 目标属性已被添加，以允许 clang-tidy 工具将其建议的修复导出到一组 .yaml 文件中。创建了新的 CMAKE_<LANG>_CLANG_TIDY_EXPORT_FIXES_DIR 变量来初始化此属性。

  • XCODE_EMBED_EXTENSIONKIT_EXTENSIONS 目标属性已被添加，用以告知 Xcode 生成器嵌入基于 ExtensionKit 的扩展，例如使用 Background Assets 框架的扩展。嵌入的各个方面可以通过以下属性进行自定义：

    • XCODE_EMBED_EXTENSIONKIT_EXTENSIONS_PATH

    • XCODE_EMBED_EXTENSIONKIT_EXTENSIONS_CODE_SIGN_ON_COPY

    • XCODE_EMBED_EXTENSIONKIT_EXTENSIONS_REMOVE_HEADERS_ON_COPY

模块¶

  • ExternalProject 模块的 ExternalProject_Add() 命令新增了 INSTALL_BYPRODUCTS 选项，用于指定由 install 步骤生成的文件。

  • FindCUDAToolkit 模块

    • 增加了对 sbsa-linux 交叉编译目标的支持，并且

    • 如果找到，现在提供一个用于 nvrtc_static 的导入目标。

  • FindImageMagick 模块现在提供导入目标。

  • FindPython3 和 FindPython 模块增加了对 稳定应用程序二进制接口 (Stable ABI) 的支持。

  • UseSWIG 模块增加了对 perl5 语言的支持。

生成器表达式¶

  • 添加了 $<BUILD_LOCAL_INTERFACE:...> 生成器表达式，以防止使用要求被导出到依赖项目中。

CTest¶

  • 添加了 CTEST_NO_TESTS_ACTION 环境变量，为 ctest(1) 的 --no-tests=<action> 命令行参数提供默认值。

已弃用和已删除的功能¶

  • CMakeFiles/CMakeOutput.log 和 CMakeFiles/CMakeError.log 文件不再由 CMake 的内置模块填充。cmake(1) 在发生 CMake Error 后不再建议查看这些文件。以前记录到这些文件中的信息现在记录到 cmake-configure-log(7) 中。

  • 在 CYGWIN 上，已移除了用于与早于 2.8.4 版本的 CMake 兼容的、未记录的 CMAKE_LEGACY_CYGWIN_WIN32 模式。

其他更改¶

  • 语言标准标志（例如 -std=c++11），当由于 target_compile_features() 或 CMAKE_<LANG>_STANDARD 而生成时，现在被放置在由 target_compile_options() 添加的标志之前，而不是之后。

  • 对于所有 COMPILE_DEFINITIONS 属性，条目开头的任何 -D 都会被移除，无论它是否由生成器表达式指定。

  • 由 CMAKE_EXPORT_COMPILE_COMMANDS 启用的 compile_commands.json 数据库现在在编译命令对象中提供了 output 字段。这允许诸如 Ninja（构建工具） Multi-Config 生成器之类的多配置生成器，为所有配置提供编译命令。

  • USE_FOLDERS 全局属性默认被视为 ON。请参阅策略 CMP0143。

  • 顶层的 project() 调用现在会在未遵守与 cmake_minimum_required() 相关的记录命令顺序时发出作者警告。

  • cmake --trace 选项现在会跟踪 try_compile() 和 try_run() 的调用。

更新¶

自 CMake 3.26.0 以来的更改如下。

3.26.1, 3.26.2, 3.26.3, 3.26.4, 3.26.5¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.26.6¶

  • cmake-file-api(7) "codemodel" 2 版本中的 "target" 对象 fileSets 字段由 CMake 3.26.0 引入，其 baseDirectories 成员的条目即使在顶层源目录内也被错误地表示为绝对路径。此问题已修复。客户端必须更新以期望顶层源目录下的相对路径。
