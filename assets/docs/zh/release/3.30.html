自 CMake 3.29 以来的变更包括以下内容。

新特性¶

预设 (Presets)¶

  • cmake-presets(7) 文件现在支持架构版本 9。include 字段现在会展开除 $env{} 和特定于预设的宏（即从预设定义内部字段派生的宏）之外的所有宏。

基于文件的 API¶

  • cmake-file-api(7) "cmakeFiles" 版本 1 对象的 version 字段已更新为 1.1。它新增了一个 globsDependent 字段，用于报告使用 CONFIGURE_DEPENDS 的 file(GLOB) 调用。

生成器¶

  • Visual Studio（集成开发环境） 生成器 现在向 .vcxproj 文件添加 UseDebugLibraries 指示符，以标注哪些配置属于调试配置。请参阅策略 CMP0162。

语言¶

  • 编译特性 (Compile Features) 功能现在实现了对 cxx_std_26 和 cuda_std_26 元特性的支持，用于指示编译器模式必须至少达到 C++26。这些元特性最早由 CMake 3.25 文档化，但此前未完全实现。

命令¶

  • add_library() 命令现在在不支持共享库的平台上拒绝创建共享库，而不是自动将其转换为静态库。请参阅策略 CMP0164。

  • enable_language() 命令如果在大约首次 project() 调用之前执行，现在会报错并失败。请参阅策略 CMP0165。

  • file(DOWNLOAD) 和 file(UPLOAD) 命令增加了 TLS_VERSION <min> 选项，用于指定连接到 https:// URL 时的最低 TLS 版本。

变量¶

  • CMAKE_<LANG>_STANDARD_LATEST 变量被添加，用于描述 CMake 为所选编译器支持的最新 <LANG> 语言标准。

  • CMAKE_TLS_VERIFY 环境变量被添加，作为现有 CMAKE_TLS_VERIFY 变量的回退选项。它指定默认情况下是否验证 https:// URL 的服务器证书。

  • CMAKE_TLS_VERSION 变量和 CMAKE_TLS_VERSION 环境变量被添加，用于指定 file(DOWNLOAD) 和 file(UPLOAD) 命令连接到 https:// URL 时的默认最低 TLS 版本。

  • CMAKE_VS_USE_DEBUG_LIBRARIES 变量和相应的 VS_USE_DEBUG_LIBRARIES 目标属性被添加，用于显式控制 .vcxproj 文件中的 UseDebugLibraries 指示符。

属性¶

  • GENERATED 源文件属性现在在所有目录中可见。请参阅策略 CMP0163。策略 CMP0118 的文档已修订，以描述其真实影响。

  • PROPAGATE_TOP_LEVEL_INCLUDES_TO_TRY_COMPILE 全局属性可用于将 CMAKE_PROJECT_TOP_LEVEL_INCLUDES 传播到使用 整个项目签名 (whole-project signature) 的 try_compile() 调用中。这主要旨在让依赖提供者能够在这些 try_compile() 调用中启用。

  • VS_FILTER_PROPS 目标属性被添加，用于告知 Visual Studio（集成开发环境） 生成器 使用自定义的 MSBuild 过滤器 .props 文件。

模块¶

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了 TLS_VERSION <min> 选项，并支持 CMAKE_TLS_VERSION 变量和 CMAKE_TLS_VERSION 环境变量，以指定连接到 https:// URL 时的最低 TLS 版本。

  • FindBacktrace 模块现在提供了一个导入目标。

  • FindBLAS 和 FindLAPACK 模块获得了对 libblastrampoline 的支持。

  • FindCUDAToolkit 模块现在为 libnvfatbin 和 libnvfatbin_static（如果找到）提供目标。

  • FindCUDAToolkit 模块现在即使在未启用 CUDA 语言时，也会搜索 CMAKE_CUDA_COMPILER 变量和 CUDACXX 环境变量。

  • FindOpenMP 模块增加了 OpenMP_RUNTIME_MSVC 选项，用于控制与 MSVC 一起使用的 OpenMP 运行时。

  • FindPython 和 FindPython3 模块获得了对自由线程 Python 版本的支持。

  • FindPython、FindPython2 和 FindPython3 模块在 Windows（微软桌面系统） 上现在为 Python 调试变体提供了更好的支持

    • 新变量

      • Python_EXECUTABLE_DEBUG

      • Python_INTERPRETER

      • Python_DEBUG_POSTFIX

    • 新目标

      • Python::InterpreterDebug

      • Python::InterpreterMultiConfig

    python_add_library() 命令现在根据 Python_DEBUG_POSTFIX 变量的值来管理 DEBUG_POSTFIX 目标属性。

生成器表达式¶

  • <LANG>_COMPILER_FRONTEND_VARIANT 生成器表达式系列被添加，用于访问关联的 CMAKE_<LANG>_COMPILER_FRONTEND_VARIANT 变量的值。

  • 链接特性（与 LINK_LIBRARY 生成器表达式一起使用时）获得了通过指定 CMAKE_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 或 CMAKE_<LANG>_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 变量来描述其行为的属性的能力。

  • QUOTE 生成器表达式被添加，用于求值为 "。

  • TARGET_PROPERTY 生成器表达式现在可以评估由新的 TRANSITIVE_COMPILE_PROPERTIES 和 TRANSITIVE_LINK_PROPERTIES 目标属性定义的 自定义传递属性。

  • TARGET_PROPERTY 生成器表达式现在通过遵循静态库的私有依赖关系，正确评估目标属性 INTERFACE_LINK_OPTIONS、INTERFACE_LINK_DIRECTORIES 和 INTERFACE_LINK_DEPENDS。请参阅策略 CMP0166。

CTest¶

  • ctest_submit() 命令和 ctest -T Submit 步骤增加了 TLSVersion 和 TLSVerify 选项，以控制与 https:// URL 的协商。请参阅 CTEST_TLS_VERSION 和 CTEST_TLS_VERIFY 变量。

CPack¶

  • CPack Inno Setup Generator 现在可在非 Windows（微软桌面系统） 主机上使用。

  • CPack NuGet Generator 增加了 CPACK_NUGET_PACKAGE_README、CPACK_NUGET_PACKAGE_REPOSITORY_URL、CPACK_NUGET_PACKAGE_REPOSITORY_TYPE、CPACK_NUGET_PACKAGE_REPOSITORY_BRANCH 和 CPACK_NUGET_PACKAGE_REPOSITORY_COMMIT 变量。

  • CPack NuGet Generator 现在可以为框架特定的依赖项生成依赖组。新增了 CPACK_NUGET_PACKAGE_TFMS 变量，用于指定应为其生成组的目标框架代号 (TFMs) 列表。

  • CPack WIX Generator 获得了对 WiX Toolset v4 的支持。请参阅 CPACK_WIX_VERSION 变量。

弃用和移除的功能¶

  • FindBoost 模块已被策略 CMP0167 移除。项目应迁移至使用上游 Boost 的 BoostConfig.cmake 包配置文件，find_package(Boost) 现在会搜索此文件。

  • 仅使用依赖项名称调用 FetchContent_Populate() 现已被弃用。项目应改为调用 FetchContent_MakeAvailable()。请参阅策略 CMP0169。使用完整的填充细节而非仅依赖项名称调用 FetchContent_Populate() 仍然完全支持。

  • Visual Studio 9 2008 生成器已被移除。

其他更改¶

  • ctest(1) 现在拒绝未知的命令行参数并报错。此前它们会被静默忽略。

  • cmake.org 提供的预编译 Windows（微软桌面系统） .msi 安装程序在执行全新安装时，默认现在会修改系统范围的 PATH。当替换 3.30 或更高版本的现有安装时，默认会保留对 PATH 修改的偏好设置。

  • cmake.org 提供的官方 .zip 源码存档现在使用 LF 换行符（而非 CRLF 换行符），以符合现代惯例。

  • "Configuring done" 和 "Generating done" 消息后打印的持续时间现在反映了生成器特定步骤所花费的时间，以及生成开始时之前未捕获的代码模型评估步骤的时间。打印的持续时间可能看起来比旧版本 CMake 长，但更加准确。

  • FetchContent 现在倾向于直接填充内容，而不是使用单独的子构建。这可以显着改善某些系统上的配置时间（尤其是 Windows（微软桌面系统），以及使用 Xcode 生成器时的 macOS（苹果桌面系统））。cmake --fresh 还会强制重新执行直接填充的依赖项的下载、更新和补丁步骤。策略 CMP0168 为那些仍依赖于使用子构建进行内容填充的项目提供了向后兼容性。

  • 当 FETCHCONTENT_FULLY_DISCONNECTED 设置为 true 时，FetchContent_MakeAvailable() 和 FetchContent_Populate() 的单参数形式要求依赖项的源目录必须已经填充。CMake 3.29 及更早版本未检查此要求，但现在已强制执行，受策略 CMP0170 约束。

更新¶

自 CMake 3.30.0 以来的变更包括以下内容。

3.30.1, 3.30.2¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.30.3¶

  • project(<PROJECT-NAME>) 命令现在将 <PROJECT-NAME>_SOURCE_DIR、<PROJECT-NAME>_BINARY_DIR 和 <PROJECT-NAME>_IS_TOP_LEVEL 设置为普通变量，此外还将其设置为缓存条目。这对于在策略 CMP0169 的 NEW 行为下保留对某些 FetchContent 用例的支持是必要的。

  • FindPython 和 FindPython3 模块现在在 Windows（微软桌面系统） 上分别定义了 Python_DEFINITIONS 和 Python3_DEFINITIONS 变量，以支持使用自由线程版本的 Python 进行开发。这些模块提供的各种目标还定义了 INTERFACE_COMPILE_DEFINITIONS 目标属性。

3.30.4¶

  • project(<PROJECT-NAME>) 命令现在仅当调用 project() 时它们已被设置为缓存或非缓存变量时，才将 <PROJECT-NAME>_SOURCE_DIR、<PROJECT-NAME>_BINARY_DIR 和 <PROJECT-NAME>_IS_TOP_LEVEL 设置为普通变量。同名的缓存条目始终像以前一样被设置。这改进了 3.30.3 的行为变更，以恢复嵌套目录调用具有相同项目名称的 project() 时的行为，但此版本中的实现是有缺陷的（此发布说明已追溯更新）。它可能导致第一次运行和后续运行之间的行为不同。如果您的项目包含嵌套的 project() 调用且使用相同项目名称，并且您使用了这些变量，请不要使用 CMake 3.30.4。

3.30.5¶

  • project(<PROJECT-NAME>) 命令现在仅当调用 project() 时它们已被设置为非缓存变量时，才将 <PROJECT-NAME>_SOURCE_DIR、<PROJECT-NAME>_BINARY_DIR 和 <PROJECT-NAME>_IS_TOP_LEVEL 设置为非缓存变量。同名的缓存条目始终像以前一样被设置。这改进了 3.30.3 的行为变更，以恢复嵌套目录调用具有相同项目名称的 project() 时的行为，并修复了 3.30.4 中引入的实现错误。

3.30.6, 3.30.7, 3.30.8, 3.30.9¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。
