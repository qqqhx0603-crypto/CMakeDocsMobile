自 CMake 3.21 以来的变更包括以下内容。

新功能¶

命令¶

  • cmake_host_system_information() 命令现在可以从 /etc/os-release 文件中查询 操作系统标识变量。

  • string(TIMESTAMP) 命令现在支持用于 ISO 8601 周数的 %V 说明符。

变量¶

  • 添加了 CMAKE_BUILD_TYPE 环境变量，以便为 CMAKE_BUILD_TYPE 变量提供默认值。

  • 添加了 CMAKE_CONFIGURATION_TYPES 环境变量，以便为 CMAKE_CONFIGURATION_TYPES 变量提供默认值。

  • 添加了 CMAKE_INSTALL_MODE 环境变量，用于指示 install() 规则（由 file(INSTALL) 实现）安装符号链接，而不是复制文件。

  • 添加了 CMAKE_<LANG>_LINK_WHAT_YOU_USE_FLAG 和 CMAKE_LINK_WHAT_YOU_USE_CHECK 变量，用于控制 LINK_WHAT_YOU_USE 目标属性所使用的链接器标志和检查。

  • 添加了 CMAKE_REQUIRE_FIND_PACKAGE_<PackageName> 变量，用于将非 REQUIRED 的 find_package() 调用转换为 REQUIRED 调用。

属性¶

  • <LANG>_EXTENSIONS 目标属性现在初始化为从编译器检测到的 CMAKE_<LANG>_EXTENSIONS_DEFAULT。请参阅 CMP0128。

  • VS_SETTINGS 源文件属性现在支持所有源文件类型。此前它仅适用于非构建源文件。

模块¶

  • CMakeDependentOption 模块中的 cmake_dependent_option() 命令现在支持完整的 条件语法。请参阅策略 CMP0127。

  • FetchContent 模块现在将 CMAKE_TLS_VERIFY、CMAKE_TLS_CAINFO、CMAKE_NETRC 和 CMAKE_NETRC_FILE 变量（如果定义）传递给底层的 ExternalProject 子构建。此前，这些变量会被 FetchContent 静默忽略。

  • FindBLAS 和 FindLAPACK 模块增加了 BLA_SIZEOF_INTEGER 选项，用于查找其 ABI 使用特定整数大小的 BLAS/LAPACK。

  • FindJasper 模块现在提供导入目标。

  • FindMatlab 模块现在提供导入目标。

  • FindPkgConfig 模块增加了 PKG_CONFIG_ARGN 变量，用于指定传递给 pkg-config 调用的参数。

  • GoogleTest 模块中的 gtest_discover_tests() 函数增加了 TEST_FILTER 选项，用于在测试发现期间使用 --gtest_filter 过滤测试。

  • UseSWIG 模块现在可以为 Visual Studio（集成开发环境） 生成器 使用 swig 工具来生成隐式依赖项。

CTest¶

  • ctest(1) 现在能够识别运行时附加到测试上的标签。此前，只能通过在配置时使用 LABELS 测试属性来为测试附加标签。更多信息，请参阅 额外的测试度量。

  • ctest(1) 现在可以通过 ENVIRONMENT_MODIFICATION 属性修改测试的环境。这允许根据测试运行时存在的环境更新环境变量。

  • ctest_memcheck() 命令现在还会生成一个 DynamicAnalysis-Test.xml 文件，可用于将测试结果提交给 CDash。

CPack¶

  • CPack DEB 生成器 增加了将 CPACK_DEBIAN_COMPRESSION_TYPE 设置为 zstd 的选项，从而为 deb 软件包启用 Zstandard 压缩。

  • CPack NSIS 生成器 增加了一个新的 CPACK_NSIS_IGNORE_LICENSE_PAGE 变量，用于抑制安装程序中的许可页面。

  • CPack RPM 生成器 增加了 CPACK_RPM_REQUIRES_EXCLUDE_FROM 选项，以避免扫描特定路径以查找依赖项。

已弃用和移除的功能¶

  • Visual Studio 10 2010 生成器现已弃用，并将在 CMake 的未来版本中移除。

其他更改¶

  • 编译特性 功能现在可以在未指定标准级别时正确禁用或启用编译器扩展，并避免在请求的设置与编译器默认值匹配时添加多余的语言标准标志。请参阅 CMP0128。

  • 编译特性 功能现在会忽略未启用语言的特性。

  • Ninja（构建工具） 生成器 现在使用 ccmake(1)（如果可用）实现 edit_cache 目标。

  • Ninja（构建工具） 和 NMake Makefile（构建脚本）s 生成器现在对系统包含项使用 MSVC -external:I 标志。此功能自 VS 16.10（工具链版本 14.29.30037）起可用。

  • CPack NSIS 生成器 现在要求 NSIS 3.03 或更高版本。

更新¶

自 CMake 3.22.0 以来的变更包括以下内容。

3.22.1¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。

3.22.2¶

  • CMP0128 的 OLD 行为已修复，即使在指定的标准与编译器默认值匹配时也会添加标志。

3.22.3¶

  • while() 命令再次像 CMake 3.21 及更早版本一样，忽略条件评估中的错误。此 Bug 在 3.22.0 中得到修复，但暴露了现有项目中的错误。该修复已被撤销以恢复兼容性。此修复可能会在 CMake 的未来版本中通过策略恢复。

3.22.4, 3.22.5, 3.22.6¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。
