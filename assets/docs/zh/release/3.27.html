自 CMake 3.26 以来的变更如下。

新功能¶

调试器¶

  • cmake(1) 现在支持 CMake 语言的交互式调试。请参阅 --debugger 选项。

预设 (Presets)¶

  • cmake-presets(7) 文件现在支持架构版本 7。

  • cmake-presets(7) 现在支持在 include 字段中使用 $penv{} 宏展开。

生成器¶

  • Makefile（构建脚本） 和 Ninja（构建工具） 生成器现在支持使用 --dependency-file 链接器标志（由 GNU Binutils 2.35 和 LLVM 的 LLD 12.0.0 添加），这样链接器读取的文件在发生变化（通常是修改时间戳）时将触发重新链接。请参阅 CMAKE_LINK_DEPENDS_USE_LINKER 变量。

  • 用于 VS 14 (2015) 及更高版本的 Visual Studio（集成开发环境） 生成器 现在可以通过 CMAKE_GENERATOR_PLATFORM 变量中的 version= 字段显式选择 Windows（微软桌面系统） SDK 版本。请参阅 Visual Studio（集成开发环境） 平台选择。

语言¶

  • CXX 语言现在将源文件扩展名 .ccm、.cxxm 和 .c++m 视为 C++。

基于文件的 API¶

  • cmake-file-api(7) 的 "codemodel" 版本 2 中的 version 字段已更新为 2.6。

  • cmake-file-api(7) 的 "codemodel" 版本 2 中的 "target" 对象在 "compileGroups" 对象中增加了一个新的 "frameworks" 字段。

平台¶

  • 现在可以在 macOS（苹果桌面系统） 上为共享库创建 Apple 基于文本的存根（即 .tbd 文件）。请参阅 ENABLE_EXPORTS 属性。

命令¶

  • add_custom_command() 命令增加了一个新的 DEPENDS_EXPLICIT_ONLY 选项，用于指示 Ninja（构建工具） 生成器不要添加该命令所附加的目标所隐含的任何依赖项。CMAKE_ADD_CUSTOM_COMMAND_DEPENDS_EXPLICIT_ONLY 变量也被添加，以便在所有对 add_custom_command() 命令的调用中启用 DEPENDS_EXPLICIT_ONLY。

  • 添加了 cmake_file_api() 命令，以便项目为当前的 CMake 运行添加 CMake 文件 API 查询。

  • find_package() 命令现在可以搜索由大写 <PACKAGENAME>_ROOT CMake 变量和大写 <PACKAGENAME>_ROOT 环境变量指定的前缀。请参阅策略 CMP0144。

  • install(CODE) 和 install(SCRIPT) 命令现在支持 $<INSTALL_PREFIX> 生成器表达式。

变量¶

  • 添加了 CMAKE_DLL（动态链接库）_NAME_WITH_SOVERSION 变量和相关的 DLL（动态链接库）_NAME_WITH_SOVERSION 目标属性，用于在 Windows（微软桌面系统） 上可选地将 SOVERSION 附加到共享库的 .dll 部分的文件名中。

  • 添加了变量 CMAKE_VS_DEBUGGER_COMMAND、CMAKE_VS_DEBUGGER_COMMAND_ARGUMENTS、CMAKE_VS_DEBUGGER_ENVIRONMENT 和 CMAKE_VS_DEBUGGER_WORKING_DIRECTORY，用于初始化相应的目标属性。

  • 添加了 CMAKE_VS_WINDOWS_TARGET_PLATFORM_MIN_VERSION 变量，以便在创建目标时初始化所有目标的 VS_WINDOWS_TARGET_PLATFORM_MIN_VERSION 目标属性。

属性¶

  • 向 对象库（Object Libraries） 添加了 CUDA_CUBIN_COMPILATION 目标属性，以支持编译为 .cubin 文件而非宿主对象文件。目前仅支持 NVIDIA。

  • 向 对象库 添加了 CUDA_FATBIN_COMPILATION 目标属性，以支持编译为 .fatbin 文件而非宿主对象文件。目前仅支持 NVIDIA。

  • 向 对象库 添加了 CUDA_OPTIX_COMPILATION 目标属性，以支持编译为 .optixir 文件而非宿主对象文件。目前仅支持 NVIDIA。

  • 目标属性 <LANG>_CLANG_TIDY、<LANG>_CPPCHECK、<LANG>_CPPLINT 和 <LANG>_INCLUDE_WHAT_YOU_USE 现在支持 生成器表达式。

  • <LANG>_LINKER_LAUNCHER 目标属性现在支持 生成器表达式。

  • 添加了 SKIP_LINTING 源文件属性，用于禁止对特定源文件进行目标范围的代码检查。

模块¶

  • FindCUDAToolkit 模块现在提供用于 cudla 的导入目标，以及用于 CUPTI 的 nvperf 和 pcsampling 组件的导入目标。

  • FindDoxygen 模块的 doxygen_add_docs() 命令增加了 CONFIG_FILE 选项，用于指定自定义的 Doxygen 配置文件。

  • FindOpenGL（开放图形库） 模块增加了对 GLES2 和 GLES3 组件的支持。

  • FindwxWidgets 模块现在提供一个导入目标。

生成器表达式¶

  • 添加了 COMPILE_ONLY 生成器表达式，用于指定编译使用要求而不包含任何链接要求。

  • 添加了用于 查询、转换 和 排序 列表 (lists) 操作的 $<LIST:...> 生成器表达式。

  • 用于 分解 和 转换 操作的 $<PATH:...> 生成器表达式现在可以逐个元素地处理路径 列表。

  • 添加了 TARGET_IMPORT_FILE、TARGET_IMPORT_FILE_BASE_NAME、TARGET_IMPORT_FILE_PREFIX、TARGET_IMPORT_FILE_SUFFIX、TARGET_IMPORT_FILE_NAME 和 TARGET_IMPORT_FILE_DIR 生成器表达式。这些表达式展开为有关目标的链接器导入文件的详细信息。

  • 添加了 TARGET_RUNTIME_DLL（动态链接库）_DIRS 生成器表达式。它展开为包含 TARGET_RUNTIME_DLL（动态链接库）S 中 DLL（动态链接库） 的目录列表。

Autogen¶

  • 添加了 CMAKE_AUTOMOC_EXECUTABLE、CMAKE_AUTORCC_EXECUTABLE 和 CMAKE_AUTOUIC_EXECUTABLE 变量，用于在创建目标时初始化相应的目标属性。

  • 添加了 AUTOGEN_USE_SYSTEM_INCLUDE 目标属性和相应的 CMAKE_AUTOGEN_USE_SYSTEM_INCLUDE 变量，以显式控制是否将自动生成的头文件视为系统头文件。

  • 添加了 INTERFACE_AUTOMOC_MACRO_NAMES 目标属性，以作为传递使用要求指定 moc 的宏名称。

CTest¶

  • 添加了 TIMEOUT_SIGNAL_NAME 和 TIMEOUT_SIGNAL_GRACE_PERIOD 测试属性，用于指定在测试进程超时时发送给它的 POSIX 信号。

CPack¶

  • 添加了 CPack Inno Setup 生成器，用于使用 Inno Setup 进行打包。

已弃用和已移除的功能¶

  • 与 3.5 之前版本的 CMake 的兼容性现已弃用，并将从未来版本中移除。现在，将策略版本设置为较旧值的 cmake_minimum_required() 或 cmake_policy() 调用会发出弃用诊断。

  • 额外生成器 (Extra Generators（扩展生成器）) 已弃用。IDE 可以使用 cmake-file-api(7) 查看 CMake 生成的项目构建树。

  • FindCUDA 模块（自 CMake 3.10 起已弃用）已根据策略 CMP0146 移除。请将项目移植到 CMake 的一等 CUDA 语言支持。

  • FindPythonInterp 和 FindPythonLibs 模块（自 CMake 3.12 起已弃用）已根据策略 CMP0148 移除。请将项目移植到 FindPython3、FindPython2 或 FindPython。

  • Dart 和 FindDart 模块已通过策略 CMP0145 弃用。请将项目移植到 CTest 模块。

  • Visual Studio 9 2008 生成器现已弃用，并将从未来版本的 CMake 中移除。

其他变更¶

  • cmake --build $dir --verbose 现在将打印用于执行构建的工作目录和命令行。

  • ExternalProject 和 FetchContent 模块现在将相对 GIT_REPOSITORY 路径解析为相对于父项目的远程，而不是作为本地文件系统路径。请参阅 CMP0150。

  • ExternalProject 的 configure 步骤在启用 UPDATE_DISCONNECTED 选项时不再在每次构建时重新运行。它仅在 download、update 或 patch 步骤的细节发生变化时才会重新运行。

  • ExternalProject 的 update 和 patch 步骤现在始终在细节发生变化时重新运行，即使启用了 UPDATE_DISCONNECTED 选项也是如此。如果使用 GIT 下载方法，并且 GIT_TAG 被更改为本地尚不知道的提交，现在会报错，而不是静默地使用之前的 GIT_TAG。

  • FindPython、FindPython2 和 FindPython3 模块现在支持 Windows（微软桌面系统） ARM64 平台。

  • file(GET_RUNTIME_DEPENDENCIES) 命令现在在 Windows（微软桌面系统） 上会保留报告的 DLL（动态链接库） 名称的大小写。它们仍会转换为小写以进行过滤匹配。

  • SYSTEM 目标属性现在对 Apple 框架生效。

  • VS 15.8 (2017) 及更高版本的 Visual Studio（集成开发环境） 生成器 现在可以并行构建自定义命令。请参阅策略 CMP0147。

  • 用于 VS 14 (2015) 及以上版本的 Visual Studio（集成开发环境） 生成器 现在倾向于选择最新的 Windows（微软桌面系统） SDK 版本。请参阅策略 CMP0149。

更新¶

自 CMake 3.27.0 以来的变更如下。

3.27.1¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。

3.27.2¶

  • 用于 VS 14 (2015) 及以上版本的 Visual Studio（集成开发环境） 生成器 现在倾向于选择最新的 Windows（微软桌面系统） SDK（如策略 CMP0149 所述），在针对任何版本的 Windows（微软桌面系统） 时均适用。在 CMake 3.27.[0-1] 中，该首选项仅限于针对 Windows（微软桌面系统） 10 及以上版本。

  • 用于 VS 14 (2015) 及以上版本的 Visual Studio（集成开发环境） 生成器 现在支持在 CMAKE_GENERATOR_PLATFORM 变量中使用 version=8.1 来选择 Windows（微软桌面系统） 8.1 SDK。在 CMake 3.27.[0-1] 中，version= 字段仅限于选择 Windows（微软桌面系统） 10 SDK。

3.27.3, 3.27.4, 3.27.5, 3.27.6, 3.27.7, 3.27.8¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.27.9¶

  • cmake-file-api(7) 的 "codemodel" 版本 2 "target" 对象中的 fileSets 字段由 CMake 3.26.0 引入，其 baseDirectories 成员的条目即使在顶层源目录内，也被错误地表示为绝对路径。此问题已在 CMake 3.26.6 中修复，现在也在 3.27.9 中修复。客户端必须更新，以期待顶层源目录下的相对路径。

  • Ninja（构建工具） 生成器 中的 Fortran 模块依赖扫描由 CMake 3.27.0 更新为使用精确的整理依赖关系。这旨在修复在目标之间移动模块源时出现的微妙重构失败。自那以后，发现了多个在使用 对象库 时错误计算精确整理依赖关系的情况。其中的一些情况已在 3.27.x 补丁系列中逐步修复，但此后发现了更微妙的情况。为了避免在 3.27 发布系列中进行进一步的大规模改动，原始更改已被撤回，并延期至未来的 CMake 版本。
