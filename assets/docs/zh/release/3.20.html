自 CMake 3.19 以来的变更包括以下内容。

新功能¶

预设 (Presets)¶

  • cmake-presets(7) 增加了对构建（build）和测试（test）预设的支持。

生成器¶

  • Makefile（Make 构建脚本） 生成器现在针对某些工具链，在编译源文件时使用编译器来提取隐式依赖项。

语言¶

  • 现在可以通过 CXX_STANDARD、CUDA_STANDARD 或 OBJCXX_STANDARD 目标属性，或通过 编译特性 (Compile Features) 功能的 cxx_std_23 元特性来指定 C++23 编译器模式。

  • 当 nvcc 是符号链接时（例如由于 ccache 或 colornvcc 包装脚本），CUDA 语言支持现在可以正常工作。

  • 增加了 CUDAARCHS 环境变量，用于初始化 CMAKE_CUDA_ARCHITECTURES。这在编译器默认值不适合机器 GPU 的情况下非常有用。

编译器¶

  • 现在支持 NVIDIA HPC SDK 编译器，编译器 ID 为 NVHPC。

  • 现在支持 Intel oneAPI 下一代 LLVM 编译器，编译器 ID 为 IntelLLVM

    • 截至 oneAPI 2021.1，Linux 上的 icx/icpx C/C++ 编译器以及 Windows（微软桌面系统） 上的 icx C/C++ 编译器均已得到完全支持。

    • 截至 oneAPI 2021.1，支持 Linux 上的 ifx Fortran 编译器。

    • Windows 上的 ifx Fortran 编译器尚不支持。

    Intel oneAPI 经典编译器（icc、icpc 和 ifort）继续得到支持，编译器 ID 为 Intel。

  • 增加了对 IAR STM8 编译器的支持。

平台¶

  • CMake 对 Android 交叉编译 的支持现在已与 Android NDK 的工具链文件合并。它们现在的行为相似，尽管部分变量名称有所不同。面向用户的变更包括：

    • find_* 函数现在默认会搜索 NDK ABI / API 特定的路径。

    • Android 的默认 CMAKE_BUILD_TYPE 现在为 RelWithDebInfo。

    • 增加了 CMAKE_ANDROID_NDK_VERSION 变量，用于报告 NDK 的版本。

基于文件的 API¶

  • cmake-file-api(7) 增加了一个新的“toolchains”（工具链）对象类型，用于描述每种已启用语言所使用的编译器。

命令¶

  • add_custom_command() 和 add_custom_target() 的 OUTPUT 和 BYPRODUCTS 选项现在支持 生成器表达式。

    它们的 COMMAND、WORKING_DIRECTORY 和 DEPENDS 选项增加了对新生成器表达式 $<COMMAND_CONFIG:...> 和 $<OUTPUT_CONFIG:...> 的支持，用于在使用 Ninja Multi-Config 生成器时控制跨配置处理。

  • add_custom_command() 命令在 Makefile（Make 构建脚本） 生成器 上增加了 DEPFILE 支持。

  • add_library() 命令此前在使用潜在的多架构配置时禁止导入对象库。这主要影响 Xcode 生成器，例如在针对 iOS 或其他设备平台时。此限制现已移除。

  • 增加了用于操作文件系统路径的 cmake_path() 命令。

  • configure_file() 命令增加了 USE_SOURCE_PERMISSIONS 和 FILE_PERMISSIONS 选项，分别支持复制源文件的权限以及使用指定的权限。

  • file(GENERATE) 命令增加了 NEWLINE_STYLE 选项，用于指定生成的文件的换行符处理方式。

  • file(GENERATE) 命令增加了 NO_SOURCE_PERMISSIONS、USE_SOURCE_PERMISSIONS 和 FILE_PERMISSIONS 选项，用于控制生成文件的权限。

  • install(FILES) 命令的 RENAME 选项现在支持 生成器表达式。

  • target_include_directories() 命令增加了一个新选项 AFTER。

  • target_sources() 命令现在支持由 add_custom_target() 命令创建的目标。

  • try_run() 命令增加了一个 WORKING_DIRECTORY 选项，用于设置运行已编译检查可执行文件的工作目录。

变量¶

  • 增加了 CMAKE_<LANG>_BYTE_ORDER 变量，用于提供从工具链检测到的目标架构字节序。

  • CMAKE_RUNTIME_OUTPUT_DIRECTORY、CMAKE_LIBRARY_OUTPUT_DIRECTORY 和 CMAKE_ARCHIVE_OUTPUT_DIRECTORY 变量现在支持与目标相关的生成器表达式。

属性¶

  • <LANG>_CLANG_TIDY 目标属性及相关的 CMAKE_<LANG>_CLANG_TIDY 变量现在支持 OBJC 和 OBJCXX 语言。

  • 增加了 EXPORT_COMPILE_COMMANDS 目标属性，并对应 CMAKE_EXPORT_COMPILE_COMMANDS 变量，允许按目标配置编译命令导出。

  • 生成的源文件现在可以在跨目录使用时无需手动标记。此外，GENERATED 源文件属性一旦开启就无法再关闭，也不能设置为布尔值以外的值。参见策略 CMP0118。

  • 增加了 UNITY_BUILD_UNIQUE_ID 目标属性，用于支持在统一构建（unity builds）中生成每个源文件唯一的标识符。这有助于解决匿名命名空间的符号冲突问题。

  • WIN32_EXECUTABLE 目标属性现在可以与 Windows（微软桌面系统） 上的 Clang 一起使用。

  • 增加了 XCODE_EMBED_FRAMEWORKS 目标属性，用于指示 Xcode 生成器嵌入框架。嵌入的细节可以通过 XCODE_EMBED_FRAMEWORKS_PATH、XCODE_EMBED_FRAMEWORKS_CODE_SIGN_ON_COPY 和 XCODE_EMBED_FRAMEWORKS_REMOVE_HEADERS_ON_COPY 目标属性进行定制。

模块¶

  • ExternalData 模块的 ExternalData_Add_Target() 函数增加了一个 SHOW_PROGRESS <bool> 选项，用于控制在构建期间是否显示进度输出。

  • ExternalProject 模块的 ExternalProject_Add() 函数增加了一个 CONFIGURE_HANDLED_BY_BUILD 选项。这可以用于在外部项目依赖项重新构建时，由构建步骤触发后续的配置步骤运行，而不是在这些情况下始终重新运行配置步骤。

  • FindBoost 模块增加了一个 Boost_NO_WARN_NEW_VERSIONS 选项，用于屏蔽有关新 Boost 版本未知依赖项的警告。

  • FindCUDAToolkit 模块增加了对当 nvcc 是符号链接时（例如由于 ccache 或 colornvcc 包装脚本）查找 CUDA 工具包的支持。

  • FindGDAL 模块已得到改进，记录并将其缓存变量标记为高级（advanced）。增加了一个新的 FindGDAL_SKIP_GDAL_CONFIG 变量，可用于跳过基于 gdal-config 的搜索。用户还可以设置 GDAL_ADDITIONAL_LIBRARY_VERSIONS 来向库名称搜索策略中添加其他版本。

  • FindIntl 模块现在提供了一个导入的目标。

  • FindOpenSSL 模块现在支持版本范围。

  • FindPython3、FindPython2 和 FindPython 模块增加了控制搜索无版本解释器名称方式的选项。

  • UseJava 模块的 add_jar() 命令的 GENERATE_NATIVE_HEADERS 特性增加了导出生成目标的功能选项。

  • UseSWIG 模块增加了能力，针对 Makefile（Make 构建脚本） 和 Ninja（构建工具） 生成器，使用 swig 工具来生成隐式依赖。

自动生成 (Autogen)¶

  • AUTOMOC 特性现在可与按配置（per-config）的源文件一起工作。

CTest¶

  • ctest(1) 增加了一个 --test-dir 选项，用于指定查找测试的目录。

CPack¶

  • CPack 增加了 CPACK_THREADS 变量，用于控制并行操作（例如压缩安装程序包）所使用的线程数。

  • CPack DEB 生成器增加了一个新的 CPACK_DEBIAN_PACKAGE_SHLIBDEPS_PRIVATE_DIRS 变量，用于在使用 dpkg-shlibdeps 时指定解析私有库依赖项的额外搜索目录。

  • CPack IFW 生成器增加了一个新的 CPACK_IFW_PACKAGE_WIZARD_SHOW_PAGE_LIST 变量，用于控制安装程序向导左侧列出页面的部件的可见性。此功能仅在 QtIFW 4.0 及以上版本中可用。

  • CPack NSIS 生成器增加了一些新的 CPACK_NSIS_BRANDING_TEXT 和 CPACK_NSIS_BRANDING_TEXT_TRIM_POSITION 变量，用于更改安装窗口底部的文本及其修剪位置。

  • CPack NSIS 生成器现在可以正确处理 Unicode 字符。如果您希望 CPACK_RESOURCE_FILE_LICENSE 包含 UTF-8 字符，则需要以带 BOM 的 UTF-8 格式编码。

  • CPack NuGet 生成器增加了选项：

    • CPACK_NUGET_PACKAGE_ICON 和 CPACK_NUGET_<compName>_PACKAGE_ICON 允许通过本地文件指定包图标。

    • CPACK_NUGET_PACKAGE_LICENSE_EXPRESSION 和 CPACK_NUGET_<compName>_PACKAGE_LICENSE_EXPRESSION 增加了对指定 软件打包数据交换 (SPDX) 认可的许可证的支持。

    • CPACK_NUGET_PACKAGE_LICENSE_FILE_NAME 和 CPACK_NUGET_<compName>_PACKAGE_LICENSE_FILE_NAME 允许通过本地文件指定许可证。

    • CPACK_NUGET_PACKAGE_LANGUAGE 和 CPACK_NUGET_<compName>_PACKAGE_LANGUAGE 允许指定包的区域设置，例如 en_CA。

弃用和移除的功能¶

  • cmake-server(7) 模式已被移除。客户端应改为使用 cmake-file-api(7)。

  • WriteCompilerDetectionHeader 模块已通过策略 CMP0120 弃用。项目应迁移至弃用该模块。

  • TestBigEndian 模块已被弃用，建议改为使用 CMAKE_<LANG>_BYTE_ORDER 变量。

  • AddFileDependencies 模块已被弃用。请将项目移植到直接使用 set_property()。

  • CPack NuGet 生成器弃用了一些变量，以反映 NuGet 规范的变化。

    • CPACK_NUGET_PACKAGE_ICONURL 和 CPACK_NUGET_<compName>_PACKAGE_ICONURL 已被弃用；请替换为对本地图标文件的引用。

    • CPACK_NUGET_PACKAGE_LICENSEURL 和 CPACK_NUGET_<compName>_PACKAGE_LICENSEURL 已被弃用；请替换为对项目许可证文件或 SPDX 许可证表达式的引用。

其他更改¶

  • 在运行 cmake(1) 以 生成项目构建系统 时，以连字符 (-) 开头的未知命令行参数现在会被拒绝并报错。此前它们会被静默忽略。

  • 源文件扩展名现在必须是明确的。详见策略 CMP0115。

  • LANGUAGE 源文件属性现在强制将文件编译为指定语言。参见策略 CMP0119。

  • 在 AIX 上，安装 XCOFF 可执行文件和共享库不再需要重新链接以将运行时搜索路径从构建树 RPATH 更改为安装树 RPATH。CMake 现在会在安装期间直接编辑 XCOFF 二进制文件，这与 ELF 平台上的做法一致。

  • 使用类似 MSVC 的编译器时，CMAKE_CXX_FLAGS 的值默认不再包含用于运行时类型信息的 /GR 标志。参见策略 CMP0117。

  • Ninja（构建工具） 生成器现在会转换由 add_custom_command() 生成的 DEPFILE。详见策略 CMP0116。

  • cmake.org 上提供的预编译 Linux 二进制文件的命名模式已更改为 cmake-$ver-linux-$arch，其中 $arch 为 x86_64 或 aarch64。

  • cmake.org 上提供的预编译 Windows（微软桌面系统） 二进制文件的命名模式已更改为 cmake-$ver-windows-$arch，其中 $arch 为 x86_64 或 i386。

更新¶

自 CMake 3.20.0 以来的变更包括以下内容。

3.20.1¶

  • CMake 3.20.0 中的 FindIntl 模块增加了 Intl_HAVE_GETTEXT_BUILTIN、Intl_HAVE_DCGETTEXT_BUILTIN 和 Intl_IS_BUILTIN 检查，但实现有误。这些已被移除并替换为单个 Intl_IS_BUILT_IN 检查，其命名与 FindIconv 模块保持一致。

  • -rpath 链接器标志现在在所有 Apple 平台上均被指定为受支持，而不仅仅是 macOS（苹果桌面系统）。用于 iOS、tvOS 和 watchOS 的 install_name_dir 现在应默认为 @rpath，而不是使用完整的绝对路径（这会在库或框架嵌入到应用程序包中时导致运行时失败，参见 XCODE_EMBED_<type>）。

3.20.2¶

  • Intel Classic 2021 编译器版本号现在被正确检测为主要版本 2021。CMake 3.20.1 及以下版本并未意识到 Intel 从 2021 版本开始更改了识别宏版本方案，因此将版本检测为 20.2。

  • Intel oneAPI Fortran 编译器现在被识别为 IntelLLVM。由于 oneAPI 2021.1 Fortran 编译器缺少一个识别宏，CMake 3.20.1 及以下版本将其识别为 Intel。CMake 现在有一个特殊情况来将 oneAPI 2021.1 Fortran 识别为 IntelLLVM。oneAPI 2021.2 Fortran 编译器定义了正确的识别宏，因此在所有 CMake 3.20 版本中均被识别为 IntelLLVM。

3.20.3, 3.20.4, 3.20.5, 3.20.6¶

这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。
