自 CMake 3.4 以来的变更包括以下内容。

新特性¶

GUI¶

  • cmake-gui(1) 增加了用于控制已弃用功能警告的选项。

  • cmake-gui(1) 增加了一个设置 VS IDE 和 Xcode 生成器所用工具集的选项，类似于 cmake(1) 中现有的 -T 选项。

  • cmake-gui(1) 增加了“正则表达式探索器”，可用于实时创建和评估正则表达式。该探索器窗口可通过“工具 (Tools)”菜单访问。

命令行¶

  • cmake(1) 的 -Wdev 和 -Wno-dev 选项现在默认也会启用和禁止输出关于已弃用功能的警告。

  • 开发者警告是否被视为错误的抑制功能现在可以通过新增的 -Werror=dev 和 -Wno-error=dev cmake(1) 选项进行控制。

  • cmake(1) 的 -E 命令行工具 copy、copy_if_different、copy_directory 和 make_directory 现在支持多个输入文件或目录。

命令¶

  • cmake_parse_arguments() 命令现在通过原生实现。为了兼容性，CMakeParseArguments 模块仍保留为空占位符。

  • install(DIRECTORY) 命令现在支持在目录列表中使用 生成器表达式。

变量¶

  • CMAKE_ERROR_DEPRECATED 变量现在可以使用 -Werror=deprecated 和 -Wno-error=deprecated cmake(1) 选项进行设置。

  • CMAKE_WARN_DEPRECATED 变量现在可以使用 -Wdeprecated 和 -Wno-deprecated cmake(1) 选项进行设置。

属性¶

  • VS_GLOBAL_<variable> 目标属性现已针对 VS 2010 及更高版本实现。此前它仅在 VS 2008 及更低版本中生效。

模块¶

  • ExternalProject 模块新增了 GIT_REMOTE_NAME 选项，用于控制 git clone --origin 的值。

  • FindBoost 模块现在提供导入的目标，例如 Boost::boost 和 Boost::filesystem。

  • FindFLEX 模块的 flex_target() 命令新增了 DEFINES_FILE 选项，用于指定生成的自定义输出头文件。

  • FindGTest 模块现在提供导入的目标。

  • FindGTK2 模块在启用 GTK2_USE_IMPORTED_TARGETS 时，现在会将 GTK2_LIBRARIES 设置为包含导入的目标列表，而不是库的路径。此外，它还设置了一个新的 GTK2_TARGETS 变量，其中包含所有导入的目标。

  • FindOpenMP 模块增加了对 Clang 的支持。

  • FindOpenSSL 模块增加了 OPENSSL_MSVC_STATIC_RT 选项，用于搜索使用 MSVC 静态运行时的库。

  • FindPNG 模块现在提供导入的目标。

  • FindTIFF 模块现在提供导入的目标。

  • 引入了 FindXalanC 模块，用于查找 Apache Xalan-C++ XSL 转换处理库。

  • FindXercesC 模块现在提供导入的目标。

平台¶

  • 增加了对 ARM 编译器 (arm.com) 的支持，编译器 ID 为 ARMCC。

  • 新增了一个平台文件，用于在 Cray Linux 环境中交叉编译以定位计算节点。详细使用方法请参阅 为 Cray Linux 环境进行交叉编译。

  • 编译特性功能现在能够识别 Windows 上 Clang 编译器 (MinGW) 支持的特性。

  • 在为 Apple 嵌入式平台（如 iOS）构建时，CMake 现在支持构建和安装包含设备和模拟器构建产物的组合目标。此行为可通过设置 IOS_INSTALL_COMBINED 目标属性来启用。

CPack¶

  • CPack DragNDrop 生成器增加了新变量，用于指定运行 AppleScript 文件以自定义 DragNDrop 安装程序文件夹的外观，包括使用提供的 PNG 或多分辨率 TIFF 文件设置背景图像。请参阅 CPACK_DMG_DS_STORE_SETUP_SCRIPT 和 CPACK_DMG_BACKGROUND_IMAGE 变量。

  • CPack DEB 生成器现在支持使用整体变量或按组件变量设置可选配置文件中的 Source 字段。请参阅 CPACK_DEBIAN_PACKAGE_SOURCE。

  • CPack DEB 生成器现在支持按组件设置 Package、Section 和 Priority 控制字段。请参阅变量 CPACK_DEBIAN_<COMPONENT>_PACKAGE_SECTION 和 CPACK_DEBIAN_<COMPONENT>_PACKAGE_PRIORITY。

  • CPack DragNDrop 生成器现在支持向 DMG 添加多语言 SLA，用户在挂载 DMG 时会看到这些内容。详细信息请参阅 CPACK_DMG_SLA_LANGUAGES 和 CPACK_DMG_SLA_DIR 变量。

  • CPack NSIS 生成器增加了新变量用于向安装程序添加位图。请参阅 CPACK_NSIS_MUI_WELCOMEFINISHPAGE_BITMAP 和 CPACK_NSIS_MUI_UNWELCOMEFINISHPAGE_BITMAP 变量。

  • CPack RPM 生成器现在支持按组件设置 Name 和 Group 控制字段。请参阅 CPACK_RPM_<component>_PACKAGE_NAME 和 CPACK_RPM_<component>_PACKAGE_GROUP。

其他 (Other)¶

  • 关于已弃用功能的警告现在默认开启。可以使用 -Wno-deprecated 或将 CMAKE_WARN_DEPRECATED 变量设置为 false 来抑制这些警告。

已弃用和移除的功能 (Deprecated and Removed Features)¶

  • cmake(1) 的 -E time 命令现在可以正确地将带有空格或特殊字符的参数传递给子进程。这可能会破坏之前通过添加额外引用或转义来规避此错误的脚本。

  • Xcode 生成器已被修复，以确保其字符串中反斜杠的转义方式与其他生成器一致。之前通过在 Xcode 生成器上添加额外反斜杠来规避不一致性的项目，必须进行更新以移除针对 CMake 3.5 及更高版本的这种绕过方式。

其他更改 (Other Changes)¶

  • Visual Studio 14 2015 生成器现在可以将 /debug:fastlink 链接器标志映射到 .vcxproj 文件属性。

  • FindGTK2 模块现在配置 GTK2::sigc++ 导入目标，以便在使用 sigc++ 2.5.1 或更高版本时为依赖项启用 C++11。

  • cmake.org 提供的 Windows 预编译二进制文件现在是 .msi 软件包，而不是可执行安装程序。在安装新软件包之前，可能需要手动卸载低于 3.5 版本的 CMake。
