自 CMake 3.17 以来的变更包括以下内容。

新功能¶

语言¶

  • 现在可以在非 Windows 平台上使用 Clang 编译 CUDA 语言。目前在任何平台上尚不支持可分离编译（Separable compilation）。

命令行¶

  • cmake(1) 增加了通过 --profiling-output 和 --profiling-format 参数对 CMake 脚本进行性能分析的支持。

  • cmake(1) 增加了一个 cat 命令行选项，可用于连接文件并将其打印到标准输出。

命令¶

  • add_library() 和 add_executable() 命令学会了创建引用非 GLOBAL 导入目标 (Imported Targets) 的 别名目标 (Alias Targets)。

  • 添加了 cmake_language() 命令，用于对脚本命令或内置命令进行元操作，起始模式包括用于调用其他命令的 CALL，以及用于原位执行 CMake 脚本的 EVAL CODE。

  • execute_process() 命令增加了 ECHO_OUTPUT_VARIABLE 和 ECHO_ERROR_VARIABLE 选项。

  • export() 命令现在如果多次使用相同的 FILE 而没有使用 APPEND，会引发错误。参见策略 CMP0103。

  • file() 命令增加了 ARCHIVE_CREATE 和 ARCHIVE_EXTRACT 子命令，以便在 CMake 脚本代码中公开 cmake(1) 的 -E tar 功能。

  • file(CONFIGURE) 子命令被创建，用于在不依赖磁盘上现有文件作为输入的情况下复制 configure_file() 的功能。其内容直接作为字符串传递。

  • file(UPLOAD) 命令增加了 TLS_VERIFY 和 TLS_CAINFO 选项，用于控制服务器证书验证。

  • find_program()、find_library()、find_path() 和 find_file() 命令增加了一个新的 REQUIRED 选项，如果未找到任何内容，该选项将停止处理并显示错误消息。

  • 具有 SOURCE 作用域的 get_property() 命令增加了 DIRECTORY 和 TARGET_DIRECTORY 选项，用于从提供的目录作用域获取属性。

  • get_source_file_property() 命令增加了 DIRECTORY 和 TARGET_DIRECTORY 选项，用于从提供的目录作用域获取属性。

  • list() 操作 SORT 增加了 NATURAL 排序选项，以使用自然顺序进行排序（参见 strverscmp(3) 手册）。

  • 具有 SOURCE 作用域的 set_property() 命令增加了 DIRECTORY 和 TARGET_DIRECTORY 选项，用于在提供的目录作用域中设置属性。

  • set_source_files_properties() 命令增加了 DIRECTORY 和 TARGET_DIRECTORY 选项，用于在提供的目录作用域中设置属性。

  • string() 命令学习了一个新的 HEX 子命令，它将字符串转换为其十六进制表示形式。

变量¶

  • 添加了 CMAKE_CUDA_ARCHITECTURES 变量以指定 CUDA 输出架构。鼓励用户使用此变量而不是手动指定选项，因为这种方法与编译器无关。当 CMAKE_CUDA_COMPILER_ID 为 NVIDIA 时，该变量会自动初始化。该变量用于初始化新的 CUDA_ARCHITECTURES 目标属性。参见策略 CMP0104。

  • 添加了 CMAKE_PCH_WARN_INVALID 变量以初始化 PCH_WARN_INVALID 目标属性，从而允许移除预编译头文件无效警告。

属性¶

  • 添加了 CUDA_ARCHITECTURES 目标属性以指定 CUDA 输出架构。鼓励用户使用此属性而不是手动指定选项，因为这种方法与编译器无关。该属性由新的 CMAKE_CUDA_ARCHITECTURES 变量初始化。参见策略 CMP0104。

  • 添加了 Fortran_PREPROCESS 目标属性和 Fortran_PREPROCESS 源文件属性，用于控制 Fortran 源文件的预处理。

  • 创建了 FRAMEWORK_MULTI_CONFIG_POSTFIX_<CONFIG> 目标属性和相关的 CMAKE_FRAMEWORK_MULTI_CONFIG_POSTFIX_<CONFIG> 变量，以允许在使用多配置生成器时为框架文件名添加后缀。

  • OBJECT_OUTPUTS 源文件属性现在支持 生成器表达式。

  • 添加了 PCH_WARN_INVALID 目标属性，允许移除预编译头文件无效警告。

  • 添加了 UNITY_BUILD_MODE 目标属性，用于告知生成器使用何种算法对包含的源文件进行分组。

  • 添加了 VS_SOURCE_SETTINGS_<tool> 目标属性，用于告知 Visual Studio 生成器（针对 VS 2010 及以上版本）使用 <tool> 为非构建源文件添加元数据。

  • 添加了 VS_SETTINGS 源文件属性，用于告知 Visual Studio 生成器（针对 VS 2010 及以上版本）为非构建源文件添加元数据。

  • 添加了 VS_PLATFORM_TOOLSET 目标属性，用于告知 Visual Studio 生成器（针对 VS 2010 及以上版本）覆盖平台工具集。

  • 添加了 VS_SOLUTION_DEPLOY 目标属性，用于告知 Visual Studio 生成器（针对 VS 2010 及以上版本）即使在不为 Windows Phone/Store/CE 构建时，也将目标标记为部署。

模块¶

  • 添加了 CheckLinkerFlag 模块，提供检查链接器标志有效性的功能。

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了一个新的 GIT_REMOTE_UPDATE_STRATEGY 关键字。这可用于指定如何处理 git 更新期间失败的变基操作。还添加了 CMAKE_EP_GIT_REMOTE_UPDATE_STRATEGY 变量作为全局默认值，并被 ExternalProject 和 FetchContent 模块所遵循。

  • FetchContent 模块的 FetchContent_Declare() 命令现在支持 SOURCE_SUBDIR 选项。它可以用于引导 FetchContent_MakeAvailable() 在不同位置查找 CMakeLists.txt 文件。

  • FindBLAS 模块现在提供导入目标。

  • FindCUDAToolkit 模块

    • 增加了 CUDAToolkit_LIBRARY_ROOT 变量，它是包含 nvvm 目录和 version.txt 的目录。

    • 使用在 CUDA 编译器检测期间找到的工具包和库根目录。

  • FindLAPACK 模块现在提供导入目标。

  • FindPython3、FindPython2 和 FindPython 模块

    • 获得了在 cmake-gui(1) 和 ccmake(1) 中为交互式编辑创建每个组件缓存变量的可能性。

    • 在 Development 组件下增加了子组件 Development.Module 和 Development.Embed。

    • 获得了指定要查找的 Python 实现的能力，包括 IronPython 和 PyPy。

  • FindRuby 模块的输入和输出变量已全部从 RUBY_ 重命名为 Ruby_，以与其他查找模块保持一致。如果提供了旧大小写的输入变量，仍将被遵循，且始终提供旧大小写的输出变量。

  • FindSWIG 模块现在接受目标语言作为 find_package 的 COMPONENTS 和 OPTIONAL_COMPONENTS 参数。

  • GoogleTest 模块的 gtest_discover_tests() 命令

    • 增加了一个新的 DISCOVERY_MODE 选项以控制何时运行测试发现步骤。它提供了一个新的 PRE_TEST 设置，以便在测试时而非构建时运行发现。新的 CMAKE_GTEST_DISCOVER_TESTS_DISCOVERY_MODE 变量可用于全局更改默认值。

    • 增加了一个新的可选参数 XML_OUTPUT_DIR。设置此参数后，JUnit XML 测试结果将存储在该目录中。

  • FindLibXslt 模块现在提供导入目标。

  • UseSWIG 模块现在支持 Fortran 作为目标语言，前提是 SWIG_EXECUTABLE 是 SWIG-Fortran。

生成器表达式¶

  • 添加了 $<DEVICE_LINK:...> 和 $<HOST_LINK:...> 生成器表达式，用于管理设备和主机链接步骤。

  • 添加了 $<LINK_LANGUAGE:...> 和 $<LINK_LANG_AND_ID:...> 生成器表达式。

CTest¶

  • ctest(1) 增加了一个新的 CTEST_RESOURCE_SPEC_FILE 变量，可用于指定 资源规范文件。

  • ctest(1) 增加了一个 --stop-on-failure 选项，可用于在测试失败时停止运行后续测试。

  • ctest_test() 命令增加了一个 STOP_ON_FAILURE 选项，可用于在测试失败时停止运行后续测试。

  • CTestCoverageCollectGCOV 模块的 ctest_coverage_collect_gcov() 命令增加了一个 TARBALL_COMPRESSION 选项，以控制收集结果的 tarball 压缩。

CPack¶

  • CPack 归档生成器 的 TXZ 格式引入了 CPACK_ARCHIVE_THREADS 变量以启用并行压缩。需要 CMake 使用的 liblzma 提供支持。

  • CPack NSIS 生成器 增加了一个新变量 CPACK_NSIS_MANIFEST_DPI_AWARE，用于声明安装程序是 DPI 感知的。

  • CPack RPM 生成器 增加了 CPACK_RPM_PRE_TRANS_SCRIPT_FILE 和 CPACK_RPM_POST_TRANS_SCRIPT_FILE 变量，用于指定事务前和事务后的脚本。

其他¶

  • cmake-gui(1) 现在根据 CMAKE_GENERATOR 环境变量填充其生成器选择窗口的默认值。此外，环境变量 CMAKE_GENERATOR_PLATFORM 和 CMAKE_GENERATOR_TOOLSET 被用于填充其各自窗口的默认值。

  • ccmake(1) 学会了读取 CCMAKE_COLORS 环境变量来定制颜色。

  • 编译特性 功能现在能够识别 MSVC 19.27 和 19.28 中 C11 特性的可用性，包括对 c_restrict、c_static_assert 特性和 -std:c11 标志的支持。

已弃用和已删除的特性¶

  • Documentation 模块已通过 CMP0106 被弃用。该模块本质上是 CMake 不再应该分发的 VTK 代码。

  • 为策略 CMP0070 和策略 CMP0071 增加了显式的弃用诊断（CMP0069 及以下版本已被弃用）。cmake-policies(7) 手册解释说所有策略的旧行为均已被弃用，项目应迁移到新行为。

其他更改¶

  • 在 Windows 上，Ninja 和 Ninja Multi-Config 生成器在未明确指定编译器的情况下，现在会选择 PATH 环境变量所列目录中找到的第一个（任何名称的）编译器。

  • LINK_OPTIONS 和 INTERFACE_LINK_OPTIONS 目标属性现在用于设备链接步骤。参见策略 CMP0105。

  • 创建覆盖现有目标的 ALIAS 目标现在会引发错误。参见策略 CMP0107。

  • 通过别名将目标链接到自身现在会引发错误。参见策略 CMP0108。

  • FindPackageHandleStandardArgs 模块的 REQUIRED_VARS 选项现在在指定了 HANDLE_COMPONENTS 时是可选的。

  • source_group() 命令现在也识别正斜杠作为子组分隔符，而不仅仅是反斜杠。

  • ctest(1) 现在记录它为每个测试设置的环境变量（由于 ENVIRONMENT 属性或 资源分配 功能），并将此日志提交给 CDash。它不记录在 CTest 外部设置的环境变量。

  • 当从源代码构建 CMake 本身且不使用系统提供的 libcurl 时，现在为支持通过 http(s) 进行网络通信的命令（例如 file(DOWNLOAD)、file(UPLOAD) 和 ctest_submit()）启用了 HTTP/2 支持。cmake.org 上提供的预编译二进制文件现在支持 HTTP/2。

  • cmake-file-api(7) “codemodel” 版本 2 的 version 字段已更新至 2.1。

  • cmake-file-api(7) “codemodel” 版本 2 的 “target” 对象在 compileGroups 对象中增加了一个新的 precompileHeaders 字段。

更新¶

自 CMake 3.18.0 以来的变更包括以下内容。

3.18.1¶

  • Xcode 生成器在未定义 CMAKE_OSX_ARCHITECTURES 时，现在选择 $(NATIVE_ARCH_ACTUAL) 作为默认架构（Xcode 的 ARCHS 设置）。这是 Xcode 12 选择主机架构所必需的，旧版本的 Xcode 默认即如此。

  • 在 CMake 3.18.0 中，add_test() 命令支持在测试名称中使用特殊字符。这在发行说明中被意外遗漏了。不幸的是，该修复破坏了现有的使用手动引用或转义来绕过先前限制的项目。此修复已在 3.18.1 中回滚，但可能会在未来的 CMake 版本中以兼容性策略的形式重新引入。

3.18.2¶

  • CMAKE_AUTOMOC_PATH_PREFIX 的默认值已更改为 OFF，因为此功能可能会破坏在不同包含目录中具有相同名称头文件的现有项目。这恢复了与 CMake 3.15 及以下版本行为的兼容性。该默认值在 3.16.9 和 3.17.5 中也已更改为 OFF。

  • 编译特性 功能已针对上述 MSVC 19.27 进行了更新（c_restrict）。

3.18.3¶

  • 编译特性 功能已针对上述 MSVC 19.28 进行了更新（c_static_assert）。
