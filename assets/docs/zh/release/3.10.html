自 CMake 3.9 以来的主要变更如下。

新功能¶

平台¶

  • 现已支持 flang Fortran 编译器，编译器 ID 为 Flang。

  • 添加了用于 Midipix 的最小化平台文件。

  • 增加了对 MSVC ARM64 架构的支持。Visual Studio 2017 Update 4 及更高版本提供了 ARM64 工具链。

  • 改进了对 IAR ARM 编译器的支持。

生成器¶

  • Makefile 生成器 和 Ninja 生成器现在支持为 CUDA 语言添加编译器启动工具（如 ccache），并与编译器一同使用（此前已支持 C 和 CXX）。详细信息请参阅 CMAKE_<LANG>_COMPILER_LAUNCHER 变量以及 <LANG>_COMPILER_LAUNCHER 目标属性。

  • CodeBlocks 额外生成器现在支持选择性地从生成的项目中排除项目根目录之外的文件。请参阅 CMAKE_CODEBLOCKS_EXCLUDE_EXTERNAL_FILES 变量。

命令¶

  • cmake_host_system_information() 命令增加了更多键值，用于获取关于处理器能力和宿主操作系统版本的信息。

  • configure_file() 命令现已支持缩进的 #  cmakedefine 和 #  cmakedefine01。现在能够识别并保留 # 字符与 cmakedefine/cmakedefine01 单词之间的空格和/或制表符，并将其输出。

  • execute_process() 命令增加了一个 RESULTS_VARIABLE 选项，当指定多个 COMMAND 参数时，该选项可收集进程管道中所有子进程的结果列表。

  • include_guard() 命令引入，用于防止 CMake 脚本被多次包含。该命令支持 DIRECTORY 和 GLOBAL 选项，以调整相应的包含保护范围。如果未提供选项，则该包含保护类似于基本的变量检查。

  • string() 命令增加了一个新的 PREPEND 子命令。

  • string(TIMESTAMP) 命令现在支持使用 %A 获取完整星期几名称，使用 %B 获取完整月份名称。

变量¶

  • 添加了 CMAKE_DIRECTORY_LABELS 变量，用于为目录中的所有测试指定标签。

属性¶

  • 引入了 <LANG>_CPPCHECK 目标属性以及配套的 CMAKE_<LANG>_CPPCHECK 变量，用于告知 Makefile 生成器 和 Ninja 生成器在编译 C 和 CXX 语言时运行 cppcheck。

  • 添加了 LABELS 目录属性，用于为目录中的所有目标和测试指定标签。

  • 添加了 TEST_INCLUDE_FILES 目录属性，用于列出运行 ctest(1) 测试时包含的文件列表。此属性推广了 TEST_INCLUDE_FILE 属性。

  • 添加了 VS_DOTNET_REFERENCEPROP_<refname>_TAG_<tagname> 目标属性，以支持 C# 目标中引用程序集的自定义 XML 标签。

  • 添加了源文件属性 VS_SHADER_OUTPUT_HEADER_FILE 和 VS_SHADER_VARIABLE_NAME，用于在使用 Visual Studio 生成器 时为 .hlsl 源文件指定更多细节。

模块¶

  • FindCurses 模块增加了一个 CURSES_NEED_WIDE 选项，用于请求宽字符变体。

  • FindEXPAT 模块现在提供导入目标。

  • FindFreetype 模块现在提供导入目标。

  • FindMPI 增加了一系列新特性，包括：

    • 模块中添加了特定于语言的组件。

    • 现在支持更多的 MPI 环境。

    • 改进了对 Fortran 的环境支持。

    • 用户现在可以更精细地控制 MPI 选择过程，包括向 MPI 编译器传递自定义参数。

    • 现在可以获取已实现的 MPI 标准版本。

    • 现在可以检测 MPI-2 C++ 绑定，如果需要，也可以禁用它们。

    • 现在可以检测并验证可用的 Fortran 绑定。

    • 可以请求各种 MPI-3 信息，包括库版本和各个绑定的 Fortran 功能。

    • 支持静态链接的 MPI 实现。

  • 添加了 FindOpenACC 模块，用于检测编译器对 OpenACC 的支持。目前仅支持 PGI、GNU 和 Cray 编译器。

  • FindOpenGL 模块增加了对 Linux 下 GLVND 的支持。

  • FindOpenMP 模块增加了对特定语言组件的支持。

  • 添加了 FindPatch 模块，用于查找 patch 命令行可执行文件。

  • FindProtobuf 模块中的 protobuf_generate_cpp() 命令增加了一个 DESCRIPTORS 选项，用于生成描述符文件。

  • GoogleTest 模块增加了一个新命令 gtest_discover_tests()，实现了动态（构建时）测试发现。与源解析方法不同，动态发现是在构建时执行测试（以“列出可用测试”模式），从而发现测试。这种方式对于标签测试的异常方式非常稳健，提供了对参数化测试等高级特性的更好支持，且无需重新运行 CMake 即可发现测试可执行文件中添加或删除的测试。请注意，CMake 3.10.3 中进行了一项破坏性更改，以解决 TIMEOUT 关键字的歧义（请参阅 3.10.3）。

  • InstallRequiredSystemLibraries 模块增加了对安装 Intel 编译器运行时的支持。

自动生成 (Autogen)¶

  • 在使用 AUTOMOC 或 AUTOUIC 配合多配置生成器（例如 Xcode）时，包含的 *.moc、moc_*.cpp 和 ui_*.h 文件会在 <AUTOGEN_BUILD_DIR>/include_<CONFIG> 中生成，而不是 <AUTOGEN_BUILD_DIR>/include。

  • 在使用 AUTOMOC 或 AUTOUIC 时，被标记为 GENERATED 的源文件也将被处理。在早期版本中，它们会被 AUTOMOC 和 AUTOUIC 忽略。请参阅策略 CMP0071。

  • 在使用 AUTOMOC 时，CMake 会在源文件中搜索字符串 Q_OBJECT、Q_GADGET 或 Q_NAMESPACE，以确定是否需要对其进行 moc 处理。新的 CMAKE_AUTOMOC_MACRO_NAMES 变量和 AUTOMOC_MACRO_NAMES 目标属性可用于注册需要搜索的其他字符串（宏名称）。

  • 在使用 AUTOMOC 时，新的 CMAKE_AUTOMOC_COMPILER_PREDEFINES 变量和 AUTOMOC_COMPILER_PREDEFINES 目标属性用于指定是否启用或禁用编译器预定义文件 moc_predefs.h 的生成。

CTest¶

  • 添加了 CTEST_LABELS_FOR_SUBPROJECTS CTest 模块变量和 CTest 脚本变量，用于指定应由 CDash 视为子项目的标签列表。要在 CTest 模块和 ctest 命令行 仪表板客户端 模式（例如 ctest -S）中使用此值，请在 CTestConfig.cmake 配置文件中设置它。

CPack¶

  • 为 FreeBSD pkg(8) 添加了 CPack FreeBSD 生成器。

  • CPack DEB 生成器 现已在 Windows 上启用。虽然功能不全（由于缺少外部 UNIX 工具），但这允许构建基本的跨平台 Debian 包。

  • CPack DEB 生成器 现在支持在 Version 信息属性中设置包发布版本。请参阅 CPACK_DEBIAN_PACKAGE_RELEASE 变量。

  • CPack DEB 生成器 实现了更严格的包版本检查，以符合 Debian 规则。

  • CPackIFW 模块的 cpack_ifw_configure_component() 和 cpack_ifw_configure_component_group() 命令增加了新的 REPLACES 和 CHECKABLE 选项。

  • CPack IFW 生成器 增加了新的 CPACK_IFW_PACKAGE_FILE_EXTENSION 变量，用于自定义目标二进制格式。

  • CPack IFW 生成器 增加了新的 CPACK_IFW_REPOSITORIES_DIRECTORIES 变量，用于指定用于解析和重新打包依赖组件的额外仓库目录。此功能仅在使用 QtIFW 3.1 或更高版本时可用。

  • CPack RPM 生成器 和 CPack DEB 生成器 实现了设置包纪元版本（epoch version）。请参阅 CPACK_RPM_PACKAGE_EPOCH 和 CPACK_DEBIAN_PACKAGE_EPOCH 变量。

其他¶

  • cmake(1) 的 -E 模式增加了对 sha1sum、sha224sum、sha256sum、sha384sum 和 sha512sum 的支持。

  • graphviz 输出现在可以区分不同的依赖类型（PUBLIC、PRIVATE 和 INTERFACE），并分别以实线、虚线和点线边在输出图中表示。

已弃用和已删除的功能¶

  • 删除了对使用 C++98 编译器构建 CMake 本身的支持。CMake 现在使用 C++11 实现。

  • 删除了在 HP-UX 上构建 CMake 的支持，等待更好的 C++11 支持和 libuv 的移植。请参阅 CMake 问题 17137。如需 HP-UX 支持，请使用 CMake 3.9 或更早版本。

其他更改¶

  • 在 FreeBSD 上，名为 c++ 的 C++ 编译器现在是首选的默认项。

  • file(GENERATE) 命令现在将其 OUTPUT 和 INPUT 参数中的相对路径分别相对于调用者的当前二进制目录和源目录进行解析。请参阅策略 CMP0070。

  • get_filename_component() 的 PROGRAM 模式语义已被修订，在接受参数的同时，不再容忍程序路径中存在未加引号的空格。虽然从技术上讲与旧行为不兼容，但预期在正确引用命令行的情况下，典型的用例行为不会发生变化。

更新¶

自 CMake 3.10.0 以来的变更如下。

3.10.1¶

  • 由于内存消耗过大，删除了 3.10.0 版本中 cmake-server(7) 的 codemodel 响应中的交叉引用字段。将需要另一种方法来提供回溯信息。

3.10.3¶

  • CMake 3.10.1 为 GoogleTest 模块的 gtest_discover_tests() 添加了 TIMEOUT 选项。该关键字与测试属性 TIMEOUT 冲突，后者通常通过命令的 PROPERTIES 关键字进行设置，这通常会导致合法但非预期的行为。在 CMake 3.10.3 中，该关键字被更改为 DISCOVERY_TIMEOUT 以解决此问题。3.10.1 和 3.10.2 中 gtest_discover_tests() 命令的 TIMEOUT 关键字的模糊行为未得到保留。
