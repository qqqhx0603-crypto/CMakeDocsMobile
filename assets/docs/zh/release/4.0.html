自 CMake 3.31 以来的变更包括以下内容。

新特性¶

基于文件的 API¶

  • cmake-file-api(7) 的 “codemodel” 第 2 版 version 字段已更新至 2.8。

  • cmake-file-api(7) 的 “codemodel” 第 2 版 “target” 对象新增了一个 debugger 字段。

命令行¶

  • 添加了 cmake --link-no-warning-as-error 选项，用于抑制 LINK_WARNING_AS_ERROR 目标属性和 CMAKE_LINK_WARNING_AS_ERROR 变量的效果。

  • 添加了 cmake --project-file 选项，用于为 CMakeLists.txt 文件指定替代文件名。这旨在供开发人员在增量迁移期间临时使用，而非用于发布最终产品。当项目文件不是 CMakeLists.txt 时，CMake 将始终发出警告。

命令¶

  • target_link_libraries() 命令现在支持 LINKER: 前缀。

变量¶

  • 当目标系统或宿主系统为 AIX 时，AIX 和 CMAKE_HOST_AIX 变量现在分别被设置为真。

  • 链接器标志变量现已支持 LINKER: 前缀

    • CMAKE_EXE_LINKER_FLAGS

    • CMAKE_EXE_LINKER_FLAGS_<CONFIG>

    • CMAKE_SHARED_LINKER_FLAGS

    • CMAKE_SHARED_LINKER_FLAGS_<CONFIG>

    • CMAKE_MODULE_LINKER_FLAGS

    • CMAKE_MODULE_LINKER_FLAGS_<CONFIG>

    参见策略 CMP0181。

  • 添加了 CMAKE_EXECUTE_PROCESS_COMMAND_ERROR_IS_FATAL 变量，用于指定 execute_process() 命令的默认 COMMAND_ERROR_IS_FATAL 行为。

  • 添加了 CMAKE_<LANG>_LINK_MODE 和 CMAKE_<LANG>_DEVICE_LINK_MODE 变量，以提供有关链接步骤执行方式的信息。

  • 添加了 CMAKE_LINK_WARNING_AS_ERROR 变量及相应的 LINK_WARNING_AS_ERROR 目标属性，用于使用链接器特定的标志将警告视为错误进行链接。

  • 引入了 CMAKE_MSVC_RUNTIME_CHECKS 变量和 MSVC_RUNTIME_CHECKS 目标属性，用于为以 MSVC ABI 为目标的编译器选择运行时检查。参见策略 CMP0184。

  • 添加了 CMAKE_POLICY_VERSION_MINIMUM 变量，旨在帮助打包人员和最终用户尝试配置尚未更新以支持当前 CMake 版本的现有项目。同时添加了 CMAKE_POLICY_VERSION_MINIMUM 环境变量以对其进行初始化。

  • 添加了 CMAKE_UNITY_BUILD_RELOCATABLE 变量和相应的 UNITY_BUILD_RELOCATABLE 目标属性，以使 CMAKE_UNITY_BUILD 生成的源文件尽可能使用相对路径 #include 原始源文件。

  • 添加了 CMAKE_XCODE_SCHEME_LLDB_INIT_FILE 变量和相应的 XCODE_SCHEME_LLDB_INIT_FILE 目标属性，以告诉 Xcode 生成器在方案的 “LLDB Init File” 设置中放置什么内容。

  • 添加了 CMAKE_XCODE_SCHEME_TEST_CONFIGURATION 变量和相应的 XCODE_SCHEME_TEST_CONFIGURATION 目标属性，以告诉 Xcode 生成器在方案中测试操作的 “Build Configuration” 设置中放置什么内容。

属性¶

  • 添加了 DEBUGGER_WORKING_DIRECTORY 目标属性和相应的 CMAKE_DEBUGGER_WORKING_DIRECTORY 变量，以告诉生成器应为目标设置什么调试器工作目录。

  • STATIC_LIBRARY_OPTIONS 目标属性现在支持 ARCHIVER: 前缀，以便以可移植的方式通过编译器驱动程序将选项传递给归档器。

  • Swift_MODULE_DIRECTORY 目标属性现在支持 生成器表达式。

  • 添加了 VS_CUSTOM_COMMAND_DISABLE_PARALLEL_BUILD 源文件属性，以告诉 Visual Studio 生成器不要并行运行自定义命令。

  • 添加了 VS_SOLUTION_ITEMS 目录属性，以告诉 Visual Studio 生成器将文件直接附加到解决方案 (.sln) 中。

模块¶

  • ExternalData 模块新增了 ExternalData_HTTPHEADERS 变量用于指定 HTTP 头信息。

  • ExternalProject 模块的 ExternalProject_Add() 命令新增了 INSTALL_JOB_SERVER_AWARE 选项，以便在使用 Makefile 生成器并显式指定 INSTALL_COMMAND 时启用 GNU Make 作业服务器的集成。

  • FeatureSummary 模块的 add_feature_info() 命令现在支持完整的 条件语法。参见策略 CMP0183。

  • FindBISON 模块的 bison_target() 命令新增了 OPTIONS 选项，用于指定 Bison 命令行选项。

  • FindCURL 模块现在提供了一个 CURL_VERSION 结果变量，以匹配上游 cURL 的 CMake 包。

  • FindFLEX 模块的 flex_target() 命令新增了 OPTIONS 选项，用于指定 Flex 命令行选项。

  • FindPatch 模块现在支持在 cmake -P 脚本模式下运行，通过跳过导入目标的创建来实现。

  • FindProtobuf 模块的 protobuf_generate() 命令新增了 PROTOC_EXE 选项，用于指定自定义的 protoc 可执行文件。

  • FindPython、FindPython2 和 FindPython3 模块现在可以通过分别使用变量 Python_ARTIFACTS_PREFIX、Python2_ARTIFACTS_PREFIX 和 Python3_ARTIFACTS_PREFIX 在同一目录中进行多次调用。

  • FindRuby 模块现已学会查找由 rbenv 提供的安装。

生成器表达式¶

  • $<PATH> 生成器表达式新增了 NATIVE_PATH 操作，用于将 CMake 路径转换为原生路径。

CTest¶

  • 在 Windows 上，ctest --interactive-debug-mode 选项现在默认在测试进程中启用 Windows 错误报告，允许它们创建调试弹出窗口和核心转储文件。这是在 libuv 更新使其成为可能后，恢复了 CMake 3.11 之前移除的行为。

CPack¶

  • CPack 归档生成器现已支持生成不压缩的 .tar 包。

  • CPack 归档生成器现在对所有包都遵循 CPACK_ARCHIVE_FILE_NAME 变量。此前，该变量仅适用于基于组件的包。

已弃用和移除的功能¶

  • 已移除对早于 3.5 版本的 CMake 的兼容性。调用 cmake_minimum_required() 或 cmake_policy() 并将策略版本设置为更早的值现在会引发错误。请注意，这些命令的调用仍然可以通过使用其 VERSION 参数的 <min>...<max> 语法来支持旧版本的 CMake。这仅需要 <min> 版本的 CMake，但在运行较新版本时，会将策略设置为高达 <max> 版本。

  • Visual Studio 14 2015 和 Visual Studio 15 2017 生成器不再支持将平台指定为生成器名称的一部分。参见 Visual Studio 平台选择。

  • CMAKE_<LANG>_USING_LINKER_MODE 变量不再用于确定 CMAKE_<LANG>_USING_LINKER_<TYPE> 变量的内容类型。取而代之的是由 CMake 设置的 CMAKE_<LANG>_LINK_MODE 变量。

  • FindGDAL 模块现已弃用，转而推荐使用上游 GDAL 官方的 CMake 包配置文件。通过调用 find_package(GDAL CONFIG) 将项目移植到后者。更多详情，请参见 GDAL 关于 CMake 集成的文档。

  • FindRuby 模块不再提供带有大写 RUBY_ 前缀的变量。参见策略 CMP0185。

  • CTest 的无文档说明的声明式脚本模式已被移除。此模式过去由 ctest -S 脚本触发，除非显式设置 CTEST_RUN_CURRENT_SCRIPT 为 OFF，否则该模式不会调用任何 CTest 命令。

  • ctest_run_script() 命令现在不再允许在没有任何参数的情况下调用。

其他更改¶

  • 在 macOS 上使用 Ninja 生成器和 Makefile 生成器时，当在 /usr/bin 中找到编译器时，它现在会原样使用，不再映射到 Xcode 内部对应的编译器。该映射由 CMake 3.2 引入，旨在允许构建树即使在 xcode-select 切换到不同的 Xcode 安装时也能继续使用其原始编译器。然而，Xcode 内部的编译器如果没有显式的 -isysroot 标志则无法使用，因此不适合传递给任意的第三方构建系统。此外，这种映射行为可能会覆盖用户指定的编译器路径。因此，该行为已被恢复。

  • 在以 macOS 为目标的构建中，CMAKE_OSX_SYSROOT 现在默认为空，推迟到编译器默认的 macOS SDK 选择。为了通过编译器的 -isysroot 标志传递显式的 macOS SDK，用户可以配置他们的构建树为 -DCMAKE_OSX_SYSROOT=macosx，或者在环境中 export SDKROOT="$(xcrun --sdk macosx --show-sdk-path)"。

    备注

    • macOS 上的编译器在 /usr/bin 中，当未调用 -isysroot 时，会在系统 SDK 路径之前搜索 /usr/local/include 中的头文件，这符合许多平台上的约定。对于在 /usr/local 安装了 Homebrew 的 macOS-x86_64 主机用户，如果不使用 Homebrew 工具构建，则应如上所述传递显式 SDK。

    • 某些 Clang 编译器没有默认的 macOS SDK 选择。对于这些编译器，如果 CMAKE_OSX_SYSROOT 为空，CMake 将自动传递由 xcrun --show-sdk-path 打印出的 macOS SDK 的 -isysroot。

  • 在 Windows 上，项目代码（如 CMakeLists.txt）中命名的文件路径不再自动转换为磁盘上的大写/小写。项目应使用其规范大小写一致地拼写源文件路径，例如在调用 add_library()、set_property(SOURCE) 或 set_source_files_properties() 时。这与区分大小写文件系统平台上的行为一致。

  • 在 AIX 上，SHARED 库目标现在默认产生一个共享库归档。参见策略 CMP0182。

  • cmake --preset 命令现在不再默认输出预设的 CMake 缓存和环境变量摘要。该摘要现在仅在通过 cmake --log-level 选项或 CMAKE_MESSAGE_LOG_LEVEL 缓存变量将消息日志级别设置为 VERBOSE、DEBUG 或 TRACE 时显示。

  • 预编译的 SunOS sparc64 和 x86_64 二进制文件现在可在 cmake.org 上获取。

更新¶

自 CMake 4.0.0 以来的变更包括以下内容。

4.0.1¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。

4.0.2¶

  • 当使用没有默认 macOS SDK 的 Clang 为 macOS 构建时，如果 CMAKE_OSX_SYSROOT 为空，CMake 现在会自动传递由 xcrun --show-sdk-path 打印出的 macOS SDK 的 -isysroot。这恢复了在 macOS 上使用 LLVM/Clang 而无需手动设置 CMAKE_OSX_SYSROOT 的支持，该支持此前因 CMake 4.0.0 移除了默认值而中断。

  • DEBUGGER_WORKING_DIRECTORY 目标属性现在被 Xcode 生成器用作 XCODE_SCHEME_WORKING_DIRECTORY 目标属性的后备。

4.0.3, 4.0.4¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

4.0.5¶

  • 当使用 Xcode 生成器为 macOS 构建时，CMAKE_OSX_DEPLOYMENT_TARGET 再次默认为宿主的 macOS 版本（如果其早于 macOS SDK 版本），正如在 CMake 4.0 之前的版本中所做的那样。

  • execute_process() 命令现在在达到 TIMEOUT 时再次终止子进程。这此前在 CMake 3.29 中被意外回归。

4.0.6, 4.0.7¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。
