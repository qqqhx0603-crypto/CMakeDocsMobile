自 CMake 3.3 以来的更改包括以下内容。

新特性¶

生成器 (Generators)¶

  • Visual Studio 14 2015 生成器学会了根据 CMAKE_SYSTEM_VERSION 变量的值以及主机上可用的 SDK 来选择 Windows（微软桌面系统） 10 SDK。

  • CMake 学会了对 Apple Swift 语言的基础支持。当将 Xcode 生成器与 Xcode 6.1 或更高版本配合使用时，可以通过 enable_language() 命令或 project() 命令启用 Swift 语言（在其他生成器或 Xcode 版本过旧时这会报错）。之后，便可以在目标中列出 .swift 源文件进行编译。

命令 (Commands)¶

  • find_program() 命令增加了 NAMES_PER_DIR 选项，以便在进入下一个目录之前，先在每个目录中考虑所有给定的 NAMES。

  • get_filename_component() 命令增加了一个新的 BASE_DIR 子命令。这用于在根据相对路径计算绝对路径时指定基准目录。

  • if() 命令增加了一个新的 TEST 运算符，如果给定的测试名称已由 add_test() 命令定义，则该运算符评估为真。请参阅策略 CMP0064。

  • install(DIRECTORY) 命令的 DESTINATION 选项已支持 生成器表达式。

  • install(FILES) 命令的 DESTINATION 选项已支持 生成器表达式。

  • string() 命令增加了一个新的 APPEND 子命令。

变量 (Variables)¶

  • Makefile（构建脚本） 生成器 和 Ninja（构建工具） 生成器现在可以为 C 和 CXX 语言添加编译器启动工具（如 distcc 和 ccache）。详情请参阅 CMAKE_<LANG>_COMPILER_LAUNCHER 变量和 <LANG>_COMPILER_LAUNCHER 目标属性。

  • 引入了新的 CMAKE_LINK_SEARCH_START_STATIC 和 CMAKE_LINK_SEARCH_END_STATIC 变量，分别用于初始化 LINK_SEARCH_START_STATIC 和 LINK_SEARCH_END_STATIC 目标属性。

属性 (Properties)¶

  • Visual Studio（集成开发环境） 生成器 现在支持额外的目标属性，以定制适用于 NVIDIA Nsight Tegra（NVIDIA Tegra 调试扩展） Visual Studio（集成开发环境） Edition（NVIDIA Tegra 的 Visual Studio（集成开发环境） 扩展） 的项目：

    • ANDROID_ANT_ADDITIONAL_OPTIONS

    • ANDROID_ARCH

    • ANDROID_ASSETS_DIRECTORIES

    • ANDROID_JAR_DEPENDENCIES

    • ANDROID_JAR_DIRECTORIES

    • ANDROID_JAVA_SOURCE_DIR

    • ANDROID_NATIVE_LIB_DEPENDENCIES

    • ANDROID_NATIVE_LIB_DIRECTORIES

    • ANDROID_PROCESS_MAX

    • ANDROID_PROGUARD

    • ANDROID_PROGUARD_CONFIG_PATH

    • ANDROID_SECURE_PROPS_PATH

    • ANDROID_SKIP_ANT_STEP

    • ANDROID_STL_TYPE

  • ARCHIVE_OUTPUT_DIRECTORY, LIBRARY_OUTPUT_DIRECTORY, 以及 RUNTIME_OUTPUT_DIRECTORY 目标属性已支持 生成器表达式。

  • 引入了 SOURCE_DIR 和 BINARY_DIR 目标属性，允许项目代码查询目标的定义位置。

  • OUTPUT_NAME 目标属性及其变体已支持 生成器表达式。

  • 添加了一个 TARGET_MESSAGES 全局属性，用于告知 Makefile（构建脚本） 生成器 是否在每个目标完成后生成打印输出的命令。

  • 在 Windows（微软桌面系统） 上使用 MS 兼容工具时，CMake 学会了可选地为 SHARED 库生成模块定义 (.def) 文件。请参阅 WINDOWS_EXPORT_ALL_SYMBOLS 目标属性。

模块 (Modules)¶

  • ExternalProject 模块的 ExternalProject_Add() 函数的 GIT_SUBMODULES 选项现在不仅限制更新的子模块集合，还限制初始化的子模块集合。

  • ExternalProject 模块引入了新的 USES_TERMINAL 参数，用于让步骤拥有独占的终端访问权。这在使用 Ninja（构建工具） 生成器监控 CMake 超级构建（superbuild）进度并防止 CPU 过度订阅时非常有用。

  • FindBISON 模块的 bison_target() 命令增加了一个 DEFINES_FILE 选项，用于指定生成的自定义输出头文件。

  • FindHDF5 模块增加了一个 HDF5_PREFER_PARALLEL 选项，允许用户在并行和串行 HDF5 工具均可用时指定首选并行工具。

  • FindIce 模块现在提供导入目标（imported targets（目标））。

  • FindJava 模块现在可以选择查找 idlj 和 jarsigner 工具。

  • FindOpenSSL 模块现在提供导入目标。

  • FindOpenSSL 模块增加了一个 OPENSSL_USE_STATIC_LIBS 选项，仅搜索静态库。

  • FindPkgConfig 增加了一个新的 pkg_get_variable() 命令，可用于从包中查询任意变量（例如用于相关工具或数据及插件的安装路径）。

  • FindProtobuf 模块增加了 protobuf_generate_python() 命令，用于从 .proto 文件生成 Python 源文件。

  • FindTIFF 模块学会了分别搜索调试和发布版本。

  • FindwxWidgets 模块学会了支持版本请求。

  • FindXercesC 模块学会了分别搜索调试和发布版本。

  • FindZLIB 模块学会了分别搜索调试和发布版本。

  • GNUInstallDirs 模块根据 GNU 编码标准和文件系统层次结构标准，为某些安装前缀提供了特殊的默认值。

  • UseJava 模块的 add_jar 函数已支持使用响应文件（例如 @srcs.txt）来指定源文件。

  • UseJava 模块的 install_jar 函数增加了新的 DESTINATION 和 COMPONENT 选项，用于指定对应的 install() 命令选项。

  • UseJava 模块增加了一个新的 create_javah 函数，用于从 Java 类创建 C 头文件。

生成器表达式¶

  • 添加了一个新的 $<SHELL_PATH:...> 生成器表达式。

CTest¶

  • CTest 学会了在并行测试时可选地测量 CPU 负载，并避免启动可能导致负载超过给定阈值的测试。请参阅 ctest(1) 命令的 --test-load 选项、CTest 测试步骤 的 TestLoad 设置、CTEST_TEST_LOAD 变量，以及 ctest_test() 命令的 TEST_LOAD 选项。

  • ctest(1) 增加了 --test-output-size-passed 和 --test-output-size-failed 选项，用于自定义作为 仪表板客户端 运行并提交测试结果时的输出大小限制。

CPack¶

  • CPack DEB 生成器 学会了按组件设置包依赖项。请参阅以下变量：

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_BREAKS

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_CONFLICTS

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_ENHANCES

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_PREDEPENDS

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_PROVIDES

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_RECOMMENDS

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_REPLACES

    • CPACK_DEBIAN_<COMPONENT>_PACKAGE_SUGGESTS

  • CPack 模块学会了打包空目录。

  • CPack 模块引入了一个新设置 CPACK_VERBATIM_VARIABLES，用于确保 cpack 程序接收到的设置值与所设置的内容完全一致，即使它们包含 CMake 特殊字符。为了兼容性，默认情况下此功能是关闭的。

其他 (Other)¶

  • 编译特性 功能现在能够识别 Windows（微软桌面系统） 上 GNU C 编译器所支持的特性。

  • CMake 学会了在使用 MSVC 工具时尊重 *.manifest 源文件。作为 .exe 和 .dll 目标源文件的清单文件，将与链接器生成的清单合并并嵌入到二进制文件中。

  • 现在支持 Concurrent Fortran 77 编译器。其 编译器 ID 为 CCur。

  • cmake(1) 增加了一个新的 --trace-expand 命令行选项，它类似于 --trace，但在输出中展开变量引用。

已弃用和移除的功能 (Deprecated and Removed Features)¶

  • CMakeExpandImportedTargets 模块现已标记为弃用。请参阅模块文档以获取解释。

  • CMAKE_USE_RELATIVE_PATHS 变量不再生效。之前它仅部分实现且不可靠。

其他更改 (Other Changes)¶

  • CheckFunctionExists, CheckLibraryExists, CheckSymbolExists, 以及 FindThreads 模块学会了在仅启用 CXX 的环境中工作。

  • CPack DEB 生成器 现在可以在计算包校验和时正确排除符号链接。

  • CPack DEB 生成器 不再使用 fakeroot 和系统 tar 程序进行打包。

  • CPack 模块不再在将设置用作其他设置的默认值时混淆包含 CMake 特殊字符的设置。负责此操作的宏 cpack_set_if_not_set 现已弃用。

  • 除非设置了 ENABLE_EXPORTS 目标属性，否则 CMake 不再使用导出符号的标志链接可执行文件。请参阅策略 CMP0065。

  • 使用 add_library() 命令创建的 MODULE 库不再设置 SONAME 字段。MODULE 库用于运行时显式动态加载。它们无法被链接，因此 SONAME 没有用处。

  • 内部的 CMAKE_<LANG>_COMPILE_OBJECT 规则变量现在在单独的 <INCLUDES> 占位符中替换编译器包含标志，而不是使用主要的 <FLAGS> 占位符。
