自 CMake 3.24 以来的更改如下。

新功能¶

预设¶

  • cmake-presets(7) 模式版本已提升至 6。

  • cmake-presets(7) 格式现在支持 packagePresets 字段，用于为 cpack --preset 指定预设。

  • cmake-presets(7) 格式现在支持 workflowPresets 字段，用于为 cmake --workflow 指定预设。

  • cmake-presets(7) 格式现在支持 outputJUnitFile 字段，用于在测试预设中指定 JUnit 输出。

语言¶

  • 现在可以通过 CXX_STANDARD、CUDA_STANDARD、HIP_STANDARD 或 OBJCXX_STANDARD 目标属性来指定 C++26 编译器模式。

  • CUDA 语言支持现在包括使用 nvcc 时的设备链接时优化 (device link-time optimization)。CMAKE_INTERPROCEDURAL_OPTIMIZATION 变量及相关联的 INTERPROCEDURAL_OPTIMIZATION 目标属性将激活设备 LTO。

命令行¶

  • 增加了一种 cmake --workflow --preset 模式，通过单个命令驱动配置、构建、测试和打包操作序列。

  • cmake -E capabilities 命令增加了一个新的 tls 字段，用于指示是否启用了 TLS。

  • cmake -E env 命令行工具增加了一个 --modify 标志，以支持 ENVIRONMENT_MODIFICATION 操作。

  • cmake --debug-trycompile 选项现在会打印日志消息，报告每个尝试编译 (try-compile) 检查执行的目录。

编译器¶

  • 增加了对 Tasking 编译器工具链（SmartCode、TriCore、独立版：ARM、MCS、8051）的支持，编译器 ID 为 Tasking。请参阅 CMAKE_TASKING_TOOLSET 变量。

命令¶

  • add_subdirectory() 命令增加了一个 SYSTEM 选项，用于在子目录中启用 SYSTEM 目录属性。

  • 增加了 block() 和 endblock() 命令，用于管理包含命令块的特定作用域（策略或变量）。

  • cmake_language() 命令增加了一个新的 GET_MESSAGE_LOG_LEVEL 子命令。它可用于查询当前的日志记录级别。

  • find_file()、find_path()、find_library() 和 find_program() 命令增加了一个 VALIDATOR 选项，用于指定一个函数，以便在对每个候选项进行校验时调用。

  • find_package() 命令在搜索包配置文件时，现在会考虑 <prefix>/<name>*/(cmake|CMake)/<name>*/ 格式的路径。

  • return() 命令增加了一个 PROPAGATE 选项，用于将变量传播到控制返回的作用域。请参阅策略 CMP0140。

  • try_compile() 和 try_run() 命令增加了新的签名，这些签名更一致地使用关键字调度，并且不需要指定二进制目录。此外，这些签名对每次调用使用唯一的目录，这允许在使用 cmake --debug-trycompile 时保留多个输出。

  • try_compile() 和 try_run() 命令增加了 NO_CACHE 选项，以将结果存储在普通变量中。

  • try_run() 命令增加了 RUN_OUTPUT_STDOUT_VARIABLE 和 RUN_OUTPUT_STDERR_VARIABLE 选项，用于从编译程序的输出中分别捕获 stdout 和 stderr。

变量¶

  • 当目标或宿主系统为 BSD 时，BSD 和 CMAKE_HOST_BSD 变量现在分别被设置为字符串值。

  • 当目标或宿主系统为 Linux 时，LINUX 和 CMAKE_HOST_LINUX 变量现在分别被设置为 true。

  • 引入了 CMAKE_MSVC_DEBUG_INFORMATION_FORMAT 变量和 MSVC_DEBUG_INFORMATION_FORMAT 目标属性，用于选择面向 MSVC ABI 的编译器的调试信息格式。请参阅策略 CMP0141。

  • 添加了 CMAKE_XCODE_SCHEME_ENABLE_GPU_API_VALIDATION 变量和对应的 XCODE_SCHEME_ENABLE_GPU_API_VALIDATION 目标属性，以告诉 Xcode 生成器在方案的 Metal: API Validation 设置中填入什么。

  • 添加了 CMAKE_XCODE_SCHEME_ENABLE_GPU_SHADER_VALIDATION 变量和对应的 XCODE_SCHEME_ENABLE_GPU_SHADER_VALIDATION 目标属性，以告诉 Xcode 生成器在方案的 Metal: Shader Validation 设置中填入什么。

  • 添加了 CMAKE_XCODE_SCHEME_LAUNCH_MODE 变量和对应的 XCODE_SCHEME_LAUNCH_MODE 目标属性，以告诉 Xcode 生成器在方案的“启动”(Launch) 模式设置中填入什么。

  • 添加了 CMAKE_XCODE_SCHEME_LAUNCH_CONFIGURATION 变量和对应的 XCODE_SCHEME_LAUNCH_CONFIGURATION 目标属性，以告诉 Xcode 生成器在方案的“启动”操作中填入什么配置。

属性¶

  • <LANG>_COMPILER_LAUNCHER 目标属性现在支持 生成器表达式。

  • 添加了 EXPORT_NO_SYSTEM 目标属性，用于指定 install(EXPORT) 和 export() 命令将生成一个 SYSTEM 属性为 OFF 的导入目标。

  • 添加了 SYSTEM 目标属性，用于指定目标是否应被视为系统库（即其包含目录在编译使用者时自动标记为 SYSTEM）。如果未设置，则默认为之前的行为：导入目标为开启，其他目标为关闭。

  • 添加了 SYSTEM 目录属性，用于初始化在该目录中创建的目标的 SYSTEM 目标属性。

模块¶

  • FetchContent 模块的 FetchContent_Declare() 命令增加了一个 SYSTEM 选项，该选项会设置由 FetchContent_MakeAvailable() 创建的子目录的 SYSTEM 目录属性。

  • FindCUDAToolkit 模块现在为 CUDA 10.0+ 提供 nvtx3 目标，它取代了 nvToolsExt。如果项目需要 CMake 3.25 且使用 CUDA 10.0+，则在使用 nvToolsExt 时会发出弃用警告。

  • FindDoxygen 模块的版本处理已得到改进。

    • 如果需要，现在会考虑多个候选项安装以满足版本约束。此前，仅考虑第一个遇到的候选项。

    • 支持版本范围。

    • 现在可以容忍 Doxygen 报告的版本格式变化（例如末尾的 git 提交哈希）。

  • FindOpenAL 模块现在提供了一个导入目标。

  • FindOpenSP 模块已添加，用于查找 OpenSP 库。

  • FindVulkan 模块增加了对新组件的支持：

    dxc

    DirectX 着色器编译器。

    volk

    Volk 开源 Vulkan 元加载器。

CPack¶

  • CPack 归档生成器 增加了一个新的 CPACK_ARCHIVE_FILE_EXTENSION 变量，用于控制包文件的扩展名。

  • CPack NSIS 生成器 增加了两个新变量 CPACK_NSIS_EXECUTABLE_PRE_ARGUMENTS 和 CPACK_NSIS_EXECUTABLE_POST_ARGUMENTS，用于向 NSIS 可执行文件调用提供参数。

  • CPack 模块增加了 CPACK_READELF_EXECUTABLE、CPACK_OBJCOPY_EXECUTABLE 和 CPACK_OBJDUMP_EXECUTABLE 变量，用于控制 cpack(1) 使用的 binutils 的位置。

已弃用和已删除的功能¶

  • IMPORTED_NO_SYSTEM 目标属性已被弃用，取而代之的是 SYSTEM 和 EXPORT_NO_SYSTEM。

  • Visual Studio 10 2010 生成器已被移除。

  • Visual Studio 11 2012 生成器现在已被弃用，并将在未来的 CMake 版本中移除。

其他更改¶

  • SSL_CERT_FILE 和 SSL_CERT_DIR 环境变量现在可用于覆盖查找 TLS/SSL 操作证书颁发机构的位置。

  • 如果 <LANG>_CLANG_TIDY 包含 -p 参数，则完整的编译器命令行不再附加在 -- 之后。

  • Xcode 生成器不再向库搜索路径添加每个配置的后缀 $(CONFIGURATION)$(EFFECTIVE_PLATFORM_NAME)。请参阅策略 CMP0142。

更新¶

自 CMake 3.25.0 以来的更改如下。

3.25.1¶

  • 在 Windows（微软桌面系统） 上，当针对 MSVC ABI 时，find_library() 命令不再接受 .a 文件名。此行为在 CMake 3.25.0 中添加，但由于在以前未出现的情况下找到了 GNU-ABI 库，因此已被撤销。

3.25.2¶

  • NVCC 12.0 及以上版本现在支持 CUDA 语言级别 20（对应 C++20）。

  • 在 Windows（微软桌面系统） 上，Intel oneAPI 2023.0 及以上版本提供的 icpx 编译器不再被选中，因为 CMake 尚不支持其类似 GNU 的命令行。

3.25.3¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。
