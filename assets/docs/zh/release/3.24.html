自 CMake 3.23 以来的更改包括以下内容。

新功能¶

预设¶

  • cmake-presets(7) 文件现在支持方案版本 5。

  • cmake-presets(7) 文件现在支持 ${pathListSep} 宏，该宏会根据平台扩展为 : 或 ;。

  • cmake-presets(7) 文件增加了对在测试预设中指定 testOutputTruncation 字段的支持，该字段用于指定达到最大测试输出大小时的截断模式。

生成器¶

  • Green Hills MULTI 生成器现在可以生成构建规则，以便在任何 CMake 文件更新时重新运行 CMake。

  • Visual Studio（集成开发环境） 生成器 现在在使用 VS 2019 Update 11 或更高版本时支持 SYSTEM 头文件。

命令行¶

  • cmake(1) 增加了 --fresh 命令行选项，用于在配置构建树时删除任何现有的 CMakeCache.txt 文件及相关的 CMakeFiles/ 目录，从而像刚刚创建构建树一样开始新的配置。

  • cmake(1) 增加了 --compile-no-warning-as-error 命令行选项，它会导致 COMPILE_WARNING_AS_ERROR 目标属性和 CMAKE_COMPILE_WARNING_AS_ERROR 变量的效果被忽略。

  • cmake(1) 的 --trace=json-v1 跟踪格式增加了 global_frame 和 line_end 字段。

  • cmake(1) 的 -E 命令 cat 和 env 学会了遵循双破折号 (--) 参数，该参数作为分隔符表示选项结束。后续的所有参数都将被视为操作数/位置参数，即使它们以破折号 - 开头。

  • cmake(1) 的 -E tar 命令增加了 --touch 选项，以保留当前的本地时间戳，而不是从归档文件中提取文件时间戳。

编译器¶

  • LLVM 的 flang Fortran 编译器现已在某些平台上得到支持，编译器 ID 为 LLVMFlang。

  • ADSP 编译器支持（SHARC 和 Blackfin）现在涵盖了 CCES 和 VDSP++ 安装，所需的配置现在在编译器模块本身中完成，而不是在 Generic-ADSP 平台模块中。

平台¶

  • 添加了一个专用的 ADSP 平台来替换现有的 Generic-ADSP 实现。它具有自动检测最新 CCES/VDSP++ 安装的功能，并根据 CMAKE_SYSTEM_PROCESSOR 变量选择编译器（cc21k 或 ccblkfn）。

命令¶

  • cmake_host_system_information() 命令在 Windows（微软桌面系统） 上增加了 QUERY WINDOWS_REGISTRY 模式。请参阅其 cmake_host_system_information(QUERY WINDOWS_REGISTRY) 特征。

  • cmake_language() 命令增加了一个新的 SET_DEPENDENCY_PROVIDER 子命令。当设置了依赖项提供程序时，对 find_package() 和 FetchContent_MakeAvailable() 的调用可以通过自定义命令进行重定向，该命令可以选择直接满足请求、修改请求处理方式或将其留给内置实现来处理。请参阅 依赖项提供程序。

  • file(DOWNLOAD) 命令增加了 RANGE_START 和 RANGE_END 选项，用于指定下载的字节范围。这对于下载大型二进制文件的部分内容非常有用。

  • find_file(), find_path(), find_library(), find_program() 和 find_package() 命令增加了 NO_CMAKE_INSTALL_PREFIX 选项，用于控制对 CMAKE_INSTALL_PREFIX 的搜索。

  • find_file(), find_path(), find_library(), find_program() 和 find_package() 命令获得了指定必须查询哪些 Windows（微软桌面系统） 注册表视图的能力。

  • find_package() 命令增加了一个 GLOBAL 选项，允许在 find_package() 调用期间将导入的目标提升到全局作用域。

  • if() 命令获得了使用 PATH_EQUAL 操作符比较路径的能力。请参阅策略 CMP0139。

变量¶

  • CMAKE_COLOR_DIAGNOSTICS 变量已添加，用于控制编译器生成的彩色诊断信息。该变量还通过 Makefile（构建脚本） 生成器 控制彩色构建系统消息，取代了 CMAKE_COLOR_MAKEFILE。

    CMAKE_COLOR_DIAGNOSTICS 环境变量已添加，用于设置 CMAKE_COLOR_DIAGNOSTICS 的默认值。

  • CMAKE_COMPILE_WARNING_AS_ERROR 变量和相应的 COMPILE_WARNING_AS_ERROR 目标属性已添加，用于启用通过特定编译器标志将警告视为错误的功能，例如 -Werror。

  • CMAKE_CUDA_ARCHITECTURES 变量和相关的 CUDA_ARCHITECTURES 目标属性现在支持特殊的 native 值，用于为主机 GPU 的架构进行编译。

  • CMAKE_FIND_PACKAGE_TARGETS_GLOBAL 变量已添加，用于切换 find_package() 命令新 GLOBAL 选项的行为。

  • CMAKE_FIND_USE_INSTALL_PREFIX 变量已添加，用于切换 find_file(), find_library(), find_path(), find_package() 和 find_program() 命令中新 NO_CMAKE_INSTALL_PREFIX 选项的行为。

  • CMAKE_PROJECT_TOP_LEVEL_INCLUDES 变量已添加，允许在确定主机和目标平台详细信息后，在第一个 project() 调用处注入自定义代码。

  • CMAKE_TRY_COMPILE_NO_PLATFORM_VARIABLES 变量已添加，用于告知 try_compile() 命令不要将任何平台变量传递给测试项目。

  • CMAKE_VERIFY_INTERFACE_HEADER_SETS 变量和相应的 VERIFY_INTERFACE_HEADER_SETS 目标属性已添加，用于启用验证头文件集中的所有头文件是否可以独立使用的构建规则。

  • CMAKE_VS_NO_COMPILE_BATCHING 变量和相应的 VS_NO_COMPILE_BATCHING 目标属性已添加，用于告知 Visual Studio（集成开发环境） 生成器 是否禁用编译器并行处理，并每次调用编译器处理一个源文件。

  • CMAKE_WATCOM_RUNTIME_LIBRARY 变量和 WATCOM_RUNTIME_LIBRARY 目标属性已引入，用于选择针对 Watcom ABI 的编译器所使用的运行时库。请参阅策略 CMP0136。

  • CMAKE_XCODE_XCCONFIG 变量和相应的 XCODE_XCCONFIG 目标属性已添加，用于告知 Xcode 生成器处理 xcconfig 文件。

属性¶

  • INTERFACE_LINK_LIBRARIES_DIRECT 和 INTERFACE_LINK_LIBRARIES_DIRECT_EXCLUDE 目标属性已添加，用于表达影响消费者的直接链接依赖项的使用需求。

  • INTERFACE_HEADER_SETS_TO_VERIFY 目标属性已添加，用于指定应由 VERIFY_INTERFACE_HEADER_SETS 验证的头文件集。

  • LINK_LIBRARIES 目标属性现在支持 $<LINK_ONLY:...> 生成器表达式。请参阅策略 CMP0131。

  • VS_DOTNET_STARTUP_OBJECT 目标属性已添加，用于告知 Visual Studio（集成开发环境） 生成器 在执行程序或项目时应使用哪个启动类。当托管 .NET 项目中存在多个 static void Main(string[]) 函数签名时，这是必需的。

模块¶

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了一个新的 DOWNLOAD_EXTRACT_TIMESTAMP 选项，用于控制在使用 URL 下载方法时，提取内容的最后修改时间是否与归档文件中的时间匹配。策略 CMP0135 已添加，默认启用此选项。

  • FetchContent 模块和 find_package() 命令现在支持集成功能。

    • FetchContent_MakeAvailable() 现在可以尝试通过先调用 find_package() 来满足依赖关系。新的 FETCHCONTENT_TRY_FIND_PACKAGE_MODE 变量控制这是否是所有依赖项的默认行为，还是按需选定，或者是完全禁用。

    • find_package() 可以被重定向，改为调用 FetchContent_MakeAvailable()。一个新的只读变量 CMAKE_FIND_PACKAGE_REDIRECTS_DIR 指向一个目录，其中可以放置配置文件包以促进这些重定向调用。

  • FindJNI 模块现在提供导入的目标。

  • FindMatlab 模块的 matlab_add_mex() 函数增加了一个 NO_IMPLICIT_LINK_TO_MATLAB_LIBRARIES 选项，用于禁用 MATLAB 库的自动链接。

  • FindVulkan 模块现在支持通过组件选择要查找的 VulkanSDK 工具和库，而不限于 Vulkan SDK 头文件和库。

  • FindZLIB 增加了一个新的 ZLIB_USE_STATIC_LIBS 变量，用于仅搜索静态库。

生成器表达式¶

  • LINK_LIBRARY 生成器表达式已添加，用于管理链接步骤中指定库的方式。CMAKE_<LANG>_LINK_LIBRARY_USING_<FEATURE> 和 CMAKE_LINK_LIBRARY_USING_<FEATURE> 变量用于定义 LINK_LIBRARY 生成器表达式可用的功能。此外，LINK_LIBRARY_OVERRIDE 和 LINK_LIBRARY_OVERRIDE_<LIBRARY> 目标属性可用于解决不兼容的功能。

    LINK_LIBRARY 生成器表达式在针对 Apple 平台时可以多种方式链接框架。添加了以下功能

    • FRAMEWORK

    • NEEDED_FRAMEWORK

    • REEXPORT_FRAMEWORK

    • WEAK_FRAMEWORK

    LINK_LIBRARY 生成器表达式在针对 Apple 平台时可以多种方式链接库。添加了以下功能

    • NEEDED_LIBRARY

    • REEXPORT_LIBRARY

    • WEAK_LIBRARY

    LINK_LIBRARY 生成器表达式增加了 WHOLE_ARCHIVE 功能，以强制加载静态库中的所有成员。此功能在以下目标平台上受支持

    • 所有 Apple 变体

    • Linux

    • 所有 BSD 变体

    • SunOS

    • Windows（微软桌面系统）

    • CYGWIN

    • MSYS

  • LINK_GROUP 生成器表达式已添加，用于管理链接步骤中库的分组。CMAKE_<LANG>_LINK_GROUP_USING_<FEATURE> 和 CMAKE_LINK_GROUP_USING_<FEATURE> 变量用于定义 LINK_GROUP 生成器表达式可用的功能。此版本定义了 RESCAN 功能，在使用 Linux、BSD、SunOS 工具链和 Windows（微软桌面系统） 的 GNU 工具链时，可用于处理静态库之间的循环引用。

  • PATH 生成器表达式已添加，用于管理路径。

  • PATH_EQUAL 生成器表达式已添加，用于管理路径比较。

  • TARGET_BUNDLE_DIR_NAME 生成器表达式已添加，用于评估给定包目标的包目录名称。

CTest¶

  • ctest(1) 增加了一个 --test-output-truncation 选项（以及相应的 CTEST_CUSTOM_TEST_OUTPUT_TRUNCATION 变量），用于指定达到最大测试输出大小时的截断模式。可能的值为 tail（默认）、middle 或 head。

CPack¶

  • CPack WIX 生成器 增加了一个新变量 CPACK_WIX_ARCHITECTURE，用于指定安装程序架构，以支持在 Windows（微软桌面系统） ARM 上运行的计算机。

  • 当使用 libarchive 3.6 或更高版本编译时，CPack 现在支持 zstd 压缩的 CPACK_THREADS 选项。cmake.org 上提供的官方 CMake 二进制文件支持此功能。

已弃用和已移除的功能¶

  • CPack 模块不再默认在 CPack DragNDrop 生成器 中启用 SLA。请参阅策略 CMP0133 以及 CPACK_DMG_SLA_USE_RESOURCE_FILE_LICENSE 变量。

  • 已弃用的 CPack PackageMaker 生成器 已被移除。

  • FindGLUT 模块不再提供未归档的 GLUT_LIBRARY 和 GLUT_INCLUDE_PATH 结果变量。

  • FindVulkan 模块不再静默忽略 find_package(Vulkan REQUIRED ...) 调用请求的未知组件。随着组件支持的增加，请求未知组件现在会产生错误。

其他更改¶

  • 在构建目录中首次运行 CMake 并启用相应语言时，CMake 不再设置诸如 CC, CXX 等环境变量。请参阅策略 CMP0132。

  • CheckIPOSupported 模块的 check_ipo_supported() 命令现在使用调用者的 CMAKE_<LANG>_FLAGS 和 CMAKE_<LANG>_FLAGS_<CONFIG> 值。请参阅策略 CMP0138。

  • MSYS Makefile（构建脚本）s 和 MinGW Makefile（构建脚本）s 生成器在未明确指定编译器时，现在会选择 PATH 环境变量列出的目录中找到的第一个编译器（任意名称）。

  • try_compile() 命令的 whole-project 特征现在会传播平台变量。请参阅策略 CMP0137。

  • while() 命令现在在条件评估期间诊断错误。请参阅策略 CMP0130。

  • cmake.org 上提供的预编译 macOS（苹果桌面系统） 二进制文件不再将 SLA 附加到 .dmg 包。这是因为 macOS（苹果桌面系统） 12 弃用了用于附加 .dmg 资源的工具。

  • 预编译的 Windows（微软桌面系统） arm64 二进制文件现在可以在 cmake.org 上获得。

更新¶

自 CMake 3.24.0 以来的更改包括以下内容。

3.24.1, 3.24.2¶

  • 这些版本未对已记录的功能或接口进行任何更改。为了支持生态系统更改和/或修复回归，进行了一些实现更新。

3.24.3¶

  • 3.24.0 中添加的 LLVMFlang Fortran 编译器支持已扩展：

    • 它现在支持 Fortran 与 C 或 CXX 之间的混合语言链接。

    • 它现在在 Windows（微软桌面系统） 上支持 GNU ABI (MinGW)，但尚未支持 MSVC ABI。

  • 进行了一些实现更新以支持生态系统变化和/或修复回归。

3.24.4¶

  • 此版本未对文档功能或接口进行任何更改。进行了一些实现更新以支持生态系统更改和/或修复回归。
