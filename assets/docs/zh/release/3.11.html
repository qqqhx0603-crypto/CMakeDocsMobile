自 CMake 3.10 以来的变更如下。

新功能¶

平台¶

  • Ninja（构建工具） 生成器现在支持 TI C/C++ 编译器。

生成器¶

  • CodeBlocks 额外生成器现在会检查 CMAKE_CODEBLOCKS_COMPILER_ID 变量，以获取要在项目文件中使用的自定义编译器标识值。

  • Makefile（构建脚本） 生成器 和 Ninja（构建工具） 生成器现在支持为 Fortran 语言添加编译器启动工具（之前已支持 C、CXX 和 CUDA）。详情请参阅 CMAKE_<LANG>_COMPILER_LAUNCHER 变量和 <LANG>_COMPILER_LAUNCHER 目标属性。

  • Visual Studio（集成开发环境） 生成器 现在支持在目标范围的 COMPILE_DEFINITIONS、INCLUDE_DIRECTORIES、COMPILE_OPTIONS 以及 file(GENERATE) 中使用 COMPILE_LANGUAGE 生成器表达式。有关注意事项，请参阅生成器表达式文档。

  • Xcode 生成器现在支持在目标范围的 COMPILE_DEFINITIONS 和 INCLUDE_DIRECTORIES 中使用 COMPILE_LANGUAGE 生成器表达式。之前仅支持 COMPILE_OPTIONS 和 file(GENERATE)。有关注意事项，请参阅生成器表达式文档。

命令¶

  • add_library() 和 add_executable() 命令现在可以在不指定任何源文件的情况下调用，只要稍后通过 target_sources() 命令添加源文件即可，不会报错。

  • file(DOWNLOAD) 和 file(UPLOAD) 命令增加了 NETRC 和 NETRC_FILE 选项，用于指定是否使用 .netrc 文件。

  • target_compile_definitions() 命令现在可以设置 导入目标 的 INTERFACE_COMPILE_DEFINITIONS 属性。

  • target_compile_features() 命令现在可以设置 导入目标 的 INTERFACE_COMPILE_FEATURES 属性。

  • target_compile_options() 命令现在可以设置 导入目标 的 INTERFACE_COMPILE_OPTIONS 属性。

  • target_include_directories() 命令现在可以设置 导入目标 的 INTERFACE_INCLUDE_DIRECTORIES 属性。

  • target_sources() 命令现在可以设置 导入目标 的 INTERFACE_SOURCES 属性。

  • target_link_libraries() 命令现在可以设置 导入目标 的 INTERFACE_LINK_LIBRARIES 属性。

变量¶

  • 引入了 CMAKE_GENERATOR_INSTANCE 变量，用于在存在多个原生工具实例时保存所选的生成器对应实例。这被 Visual Studio 15 2017 生成器用于持久化保存所选的 Visual Studio（集成开发环境） 实例。

  • 添加了 CMAKE_INSTALL_DEFAULT_DIRECTORY_PERMISSIONS 变量，用于启用在 install() 和 file(INSTALL)（例如在 make install 期间）隐式创建目录时设置默认权限。

  • 添加了 CMAKE_JOB_POOLS 变量，用于指定 JOB_POOLS 属性的值。这使得在使用 Ninja（构建工具） 生成器时，可以通过命令行配置参数来控制构建并行度。

  • 添加了 CMAKE_NETRC 和 CMAKE_NETRC_FILE 变量，用于指定在 file(DOWNLOAD)、file(UPLOAD) 命令和 ExternalProject 模块中使用 .netrc 文件。

  • 添加了 CMAKE_CUDA_SEPARABLE_COMPILATION 变量，用于在创建目标时初始化 CUDA_SEPARABLE_COMPILATION 目标属性。

属性¶

  • COMPILE_DEFINITIONS 源文件属性现在支持 生成器表达式。

  • 添加了 COMPILE_OPTIONS 源文件属性，用于管理传递给编译器的选项列表。

  • 添加了 IMPORTED_GLOBAL 目标属性，用于指示 导入目标 是否全局可见。对于使用 add_library() 或 add_executable() 的 GLOBAL 选项创建的目标，该属性会自动设为真值。此外，项目代码现在可以将本地导入目标升级为全局可见，方法是将此属性设置为 TRUE。

  • 添加了 INCLUDE_DIRECTORIES 源文件属性，用于指定预处理器包含文件的搜索目录列表。

  • 添加了源文件属性 VS_SHADER_DISABLE_OPTIMIZATIONS 和 VS_SHADER_ENABLE_DEBUG，用于在 Visual Studio（集成开发环境） 生成器 中为 .hlsl 源文件指定更多详细信息。

模块¶

  • CheckIncludeFiles 模块的 check_include_files() 命令增加了 LANGUAGE 选项，用于指定检查时使用 C 还是 CXX 编译器。

  • CMakePackageConfigHelpers 模块的 write_basic_package_version_file() 命令在 COMPATIBILITY 参数中新增了 SameMinorVersion 模式。

  • ExternalProject 模块现在可以在注释、命令、工作目录和副产品中替换 <DOWNLOAD_DIR>。

  • ExternalProject 模块增加了 NETRC 和 NETRC_FILE 选项，用于指定是否使用 .netrc 文件。

  • 添加了新的 FetchContent 模块，支持在配置阶段填充内容，使用 ExternalProject_Add() 支持的任何下载/更新方法。这使得内容可以在配置阶段直接使用，例如与 add_subdirectory() 等配合。它对分层项目结构支持良好，允许父项目覆盖子项目的内容细节，并确保内容不会在整个项目树中被重复填充。

  • FindBLAS 和 FindLAPACK 模块现在支持 FLAME 的 blis 和 libflame。

  • FindDoxygen 模块的 doxygen_add_docs() 命令现在支持新的 DOXYGEN_VERBATIM_VARS 列表变量。该列表中包含的任何 DOXYGEN_... 变量都将绕过自动引号逻辑，在传输到输出的 Doxyfile 时保留其内容不变。

  • 添加了 FindIconv 模块来定位 iconv 支持。

  • GenerateExportHeader 模块的 generate_export_header 命令增加了 INCLUDE_GUARD_NAME 选项，用于更改写入生成的导出头文件的包含保护符号名称。此外，它现在会在生成的导出头文件的包含保护 #endif 之后添加注释。

  • UseJava 模块的 add_jar 命令增加了 GENERATE_NATIVE_HEADERS 选项，可使用 javac -h 为 javac 1.8 或更高版本生成原生头文件。这取代了 create_javah，后者由于 JEP 313 移除了 javah 工具，已无法在 JDK 1.10 及更高版本中使用。

自动生成 (Autogen)¶

  • 使用 AUTOMOC 或 AUTOUIC 时，CMake 现在启动多个并行的 moc 或 uic 进程以缩短构建时间。新的 CMAKE_AUTOGEN_PARALLEL 变量和 AUTOGEN_PARALLEL 目标属性可用于指定启动的并行 moc 或 uic 进程数。默认值派生自宿主机 CPU 的核心数。

CTest¶

  • ctest_start() 命令不再设置 CTEST_RUN_CURRENT_SCRIPT，因为它在函数内部调用时会存在作用域问题。相反，它会在 CTest 内部设置一个变量。不过，在全局作用域将 CTEST_RUN_CURRENT_SCRIPT 设置为 0 仍然可以阻止脚本在结束时重新运行。

CPack¶

  • cpack(1) 增加了 --trace 和 --trace-expand 选项。

  • CPack IFW 生成器 增加了新的 CPACK_IFW_PACKAGE_REMOVE_TARGET_DIR 变量，用于控制在卸载时是否不删除目标目录。

  • CPack RPM 生成器 现在支持强制对程序和共享库设置执行权限。请参阅 CPACK_RPM_INSTALL_WITH_EXEC 变量。

  • 添加了 CPACK_INSTALL_DEFAULT_DIRECTORY_PERMISSIONS 变量，其用途与安装期间（如 make install）的 CMAKE_INSTALL_DEFAULT_DIRECTORY_PERMISSIONS 变量相同，但在打包期间（如 make package）生效。

其他¶

  • 别名目标 现在可以为使用 add_library() 的 GLOBAL 选项创建的 导入目标 创建别名。

  • 接口库（Interface Libraries） 现在可以设置自定义属性，只要这些属性名以底线 (_) 或小写 ASCII 字符开头。最初的意图是仅允许对 INTERFACE 库有意义的属性，但这同时也阻止了自定义属性的使用。

  • 添加了 cmake(1) 的 --open <dir> 命令行选项，用于打开生成的 IDE 项目，如 Visual Studio（集成开发环境） 解决方案或 Xcode 项目。

已弃用和已删除的特性¶

  • 为策略 CMP0037 到 CMP0054 添加了显式的弃用诊断（CMP0036 及以下版本已弃用）。cmake-policies(7) 手册说明了所有策略的 OLD 行为均已弃用，项目应迁移到 NEW 行为。

  • KDevelop3 生成器已被移除。

其他更改¶

  • 策略 CMP0037 不再预留与可选功能（如 test 和 package）相关的目标名称，除非启用了相应的功能。

  • FindOpenGL（开放图形库） 模块现在优先选择 GLVND 库（如果可用）。请参阅策略 CMP0072。

  • CMAKE_OSX_DEPLOYMENT_TARGET 变量中设置的最低部署目标过去仅应用于 macOS（苹果桌面系统），而不管选择了哪个 SDK。现在它会根据 CMAKE_OSX_SYSROOT 选择的目标平台进行正确设置。例如，如果 sysroot 变量指定了 iOS SDK，则 CMAKE_OSX_DEPLOYMENT_TARGET 中的值将被解释为最低 iOS 版本。

  • Xcode 生成器每个 project() 命令生成一个项目文件的行为，现在可以通过 CMAKE_XCODE_GENERATE_TOP_LEVEL_PROJECT_ONLY 变量来控制。这对于加速大型项目的 CMake 生成步骤以及解决 ZERO_CHECK 逻辑中的错误非常有用。

  • 由于 CMakeCache.txt 格式不支持值中包含换行符，现在包含换行符的值在写入文件前会被截断。此外，缓存文件中会写入警告注释，并在控制台上向用户显示警告消息。

更新¶

自 CMake 3.11.0 以来的变更如下。

3.11.1¶

  • CheckIncludeFile 模块的 check_include_file() 命令、CheckIncludeFileCXX 模块的 check_include_file_cxx() 命令以及 CheckIncludeFiles 模块的 check_include_files() 命令在 CMake 3.11.0 中被配置为遵守 CMAKE_REQUIRED_LIBRARIES 变量。由于这改变了现有项目的检查行为，此变更已被回退。未来可能会通过兼容性策略将其恢复。

3.11.2¶

  • 调用 add_library() 来创建非全局可见的导入目标的别名，现在再次导致错误，正如 3.11.0 之前一样。这一诊断信息因允许全局可见导入目标创建别名的变更，而在 CMake 3.11.0 和 3.11.1 中意外丢失。

  • FindQt4 模块的 qt4_wrap_cpp、qt4_wrap_ui 和 qt4_add_resources 宏现在对其生成的源文件设置 SKIP_AUTOMOC 和 SKIP_AUTOUIC。这些文件不需要被 moc 或 uic 处理，我们必须明确说明这一点以符合策略 CMP0071。

3.11.3¶

  • CMake 3.11.0 引入了在 Windows（微软桌面系统） 上解析符号链接的支持，路径处理方式通常与 UNIX 上一致。由于这会导致 subst 驱动器出现损坏，该特性已被回退。
