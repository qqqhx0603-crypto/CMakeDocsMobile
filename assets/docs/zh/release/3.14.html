自 CMake 3.13 以来的变更包括以下内容。

新特性¶

生成器¶

  • 增加了 Visual Studio 16 2019 生成器。由于该版本的 VS 尚未发布，此生成器处于实验阶段，并基于“Visual Studio（集成开发环境） 2019 Preview 4”。

    VS 2019 生成器与早期版本的生成器不同，它不在生成器名称中提供指定目标平台的变体。相反，必须使用 CMAKE_GENERATOR_PLATFORM（例如通过 -A 命令行选项）。此外，默认目标平台（架构）现在基于宿主（host）平台。VS 宿主工具集选择现在也基于宿主架构。

  • Green Hills MULTI 生成器已更新

    • 现在支持 对象库（Object Libraries）。

    • 现在会对不支持的项目类型（如共享库）发出警告。

    • 现在为每个调用 project() 命令的目录生成一个顶层的 <PROJECT-NAME>.top.gpj。不再创建顶层项目文件 default.gpj。

    • 现在遵循目标重命名和目标输出控制属性，例如 RUNTIME_OUTPUT_DIRECTORY 和 OUTPUT_NAME。这也修复了对由 install() 生成的安装规则的支持。

    • 现在遵循源文件属性 INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_OPTIONS。

    • 现在支持未通过 GHS_INTEGRITY_APP 集成文件且设置了 -dynamic 目标链接标志的“动态下载 Integrity 应用”。

    • 项目文件的内容现在按名称对源组和文件进行排序。将 GHS_NO_SOURCE_GROUP_FILE 目标属性设置为 ON，可为目标生成单个项目文件，而不是为每个源组生成一个项目文件。设置 CMAKE_GHS_NO_SOURCE_GROUP_FILE 变量可对所有目标启用此功能。

基于文件的 API¶

  • 添加了用于客户端获取语义构建系统信息的文件 API。请参阅 cmake-file-api(7) 手册。此 API 旨在取代用于 IDE 的 cmake-server(7) 模式。

平台¶

  • CMake 现在支持使用简单的工具链文件进行 iOS、tvOS 或 watchOS 的交叉编译。

命令行¶

  • cmake(1) 的 构建工具模式（cmake --build）增加了 --verbose 和 -v 选项，用于指定详细的构建输出。目前 Xcode 等某些生成器尚不支持此选项。

  • cmake(1) 的 -E compare_files 命令增加了新的 --ignore-eol 选项，用于在比较文件时忽略行尾差异（例如 LF 与 CRLF）。

  • cmake-gui(1) 对话框增加了 -S 和 -B 参数，用于明确指定源码目录和构建目录。

命令¶

  • file() 命令增加了名为 CREATE_LINK 的子命令，可用于创建硬链接或符号链接。

  • file() 命令增加了名为 READ_SYMLINK 的子命令，可用于确定符号链接指向的路径。

  • file() 命令增加了 SIZE 模式，用于获取磁盘上文件的大小。

  • find_package() 命令学会了在包配置文件路径中可选地解析符号链接。请参阅 CMAKE_FIND_PACKAGE_RESOLVE_SYMLINKS 变量。

  • get_filename_component() 命令增加了新的 LAST_EXT 和 NAME_WLE 变体，用于处理名称中最后一个 . 之后的扩展名。

  • if() 命令增加了对检查缓存变量是否已定义的语法支持：DEFINED CACHE{VAR}。

  • install(CODE) 和 install(SCRIPT) 命令学会了支持生成器表达式。请参阅策略 CMP0087。

  • install(TARGETS) 命令学会了在没有 DESTINATION 参数的情况下，根据 GNUInstallDirs 模块的变量和内置默认值，将目标安装到适当的默认目录。

  • install(FILES) 和 install(DIRECTORY) 命令学会了一组新的参数，用于按文件类型安装文件，并根据 GNUInstallDirs 和内置默认值设置目标路径，无需 DESTINATION 参数。

  • list() 操作 REMOVE_ITEM、REMOVE_DUPLICATES、SORT、REVERSE 和 FILTER 现在都接受不存在的变量作为列表，因为对空列表执行这些操作的结果仍为空列表。

  • list() 操作 REMOVE_AT 现在会指出给定的索引对于不存在的变量或空列表是无效的。

  • try_compile() 和 try_run() 命令增加了新的 LINK_OPTIONS 选项。

变量¶

  • 添加了 CMAKE_BUILD_RPATH_USE_ORIGIN 变量和相应的 BUILD_RPATH_USE_ORIGIN 目标属性，以启用相对运行时路径 (RPATH)。这有助于实现与构建目录无关的可重定位和可复现构建。

  • 添加了 CMAKE_VS_PLATFORM_NAME_DEFAULT 变量，供 Visual Studio（集成开发环境） 生成器 在未明确设置 CMAKE_GENERATOR_PLATFORM 时报告所使用的默认平台。

属性¶

  • 添加了 CMAKE_ROLE 全局属性，允许脚本确定它们是在项目模式、脚本模式、find-package 模式、CTest 还是 CPack 中运行。

  • CUDA_RESOLVE_DEVICE_SYMBOLS 目标属性现在在共享库、模块库和可执行目标上受支持。此前它仅在静态库上生效。

  • 创建了 EXCLUDE_FROM_ALL 目标属性以覆盖其目录设置。现在，如果目标的 EXCLUDE_FROM_ALL 属性被设置为 OFF，即使其所属目录被标记为 EXCLUDE_FROM_ALL，该目标也会作为“all”的一部分进行构建。

  • INTERFACE_POSITION_INDEPENDENT_CODE 目标属性现在支持 生成器表达式。

模块¶

  • 用于检查功能（如 CheckCSourceCompiles）的模块系列获得了管理 LINK_OPTIONS 的能力。

  • 添加了 CheckFortranSourceRuns 模块，提供 check_fortran_source_runs() 命令，用于检查 Fortran 源码片段是否能编译并运行。

  • CMakePackageConfigHelpers 模块的 write_basic_package_version_file() 命令增加了一个新的 ARCH_INDEPENDENT 选项，用于支持与架构无关的包。

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了 LOG_DIR 和 LOG_MERGED_STDOUTERR 选项以控制日志记录。

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了 LOG_PATCH 选项，用于可选地记录补丁步骤。

  • ExternalProject 模块的 ExternalProject_Add() 命令学会在同时使用 BUILD_IN_SOURCE 时应用 SOURCE_SUBDIR。BUILD_COMMAND 将在 SOURCE_DIR 的给定 SOURCE_SUBDIR 中运行。

  • FetchContent 模块增加了新的 FetchContent_MakeAvailable() 命令。它接受一个依赖项名称列表，并遍历该列表，使用规范模式填充每个依赖项并将其添加到主构建中。这显著减少了项目中所需的样板代码。

  • FindBISON 模块的 bison_target() 命令现在以 CMAKE_CURRENT_BINARY_DIR 作为工作目录运行 bison。请参阅策略 CMP0088。

  • FindCURL 模块增加了对将协议作为包组件进行请求的支持。

  • FindFontconfig 模块被添加，用于查找 fontconfig。

  • FindGDAL 模块现在提供导入的目标。

  • FindGIF 模块现在提供导入的目标。

  • FindGit 模块现在提供用于 Git 可执行文件的导入目标。

  • FindIce 模块学会了查找 slice2confluence 和 slice2matlab。

  • FindLibinput 模块被添加，用于查找 libinput。

  • FindLibLZMA 模块现在提供导入的目标。

  • FindMatlab 模块增加了 R2017b 和 R2018a 选项以指定要使用的 MEX API 版本；这些选项镜像了 MATLAB R2018a 中 mex 命令的新选项。不再需要 MX_LIBRARY 选项。

  • FindPostgreSQL 模块现在提供导入的目标。

  • FindPython、FindPython2 和 FindPython3 模块增加了对 NumPy 组件的支持。

  • FindPython2、FindPython3 和 FindPython 模块现在支持通过跳过创建导入目标和辅助函数来以脚本模式运行。

  • FindSQLite3 模块被添加，用于查找 SQLite v3.x 库。

  • FindX11 重命名了以下变量，以匹配其库名而非头文件名。旧变量保留以供兼容性使用：

    • X11_Xxf86misc_INCLUDE_PATH 代替 X11_xf86misc_INCLUDE_PATH

    • X11_Xxf86misc_LIB 代替 X11_xf86misc_LIB

    • X11_Xxf86misc_FOUND 代替 X11_xf86misc_FOUND

    • X11_Xxf86vm_INCLUDE_PATH 代替 X11_xf86vmode_INCLUDE_PATH

    • X11_Xxf86vm_LIB 代替 X11_xf86vmode_LIB

    • X11_Xxf86vm_FOUND 代替 X11_xf86vmode_FOUND

    • X11_xkbfile_INCLUDE_PATH 代替 X11_Xkbfile_INCLUDE_PATH

    • X11_xkbfile_LIB 代替 X11_Xkbfile_LIB

    • X11_xkbfile_FOUND 代替 X11_Xkbfile_FOUND

    • X11_Xtst_INCLUDE_PATH 代替 X11_XTest_INCLUDE_PATH

    • X11_Xtst_LIB 代替 X11_XTest_LIB

    • X11_Xtst_FOUND 代替 X11_XTest_FOUND

    • X11_Xss_INCLUDE_PATH 代替 X11_Xscreensaver_INCLUDE_PATH

    • X11_Xss_LIB 代替 X11_Xscreensaver_LIB

    • X11_Xss_FOUND 代替 X11_Xscreensaver_FOUND

    以下变量已被完全弃用，因为它们本质上是重复的：

    • X11_Xinput_INCLUDE_PATH (使用 X11_Xi_INCLUDE_PATH)

    • X11_Xinput_LIB (使用 X11_Xi_LIB)

    • X11_Xinput_FOUND (使用 X11_Xi_FOUND)

  • FindX11 现在提供 X11_Xext_INCLUDE_PATH。

  • FindX11 现在提供导入的目标。

  • UseSWIG 模块学会了在定义了文件属性 SWIG_MODULE_NAME 时，向 SWIG 编译器传递 -module <module_name>。请参阅策略 CMP0086。

  • UseSWIG 模块增加了指定 SWIG 源文件扩展名的选项。

生成器表达式¶

  • 添加了 $<Fortran_COMPILER_ID:...> 和 $<Fortran_COMPILER_VERSION:...> 生成器表达式。

  • $<IN_LIST:...> 生成器表达式现在可以正确处理空参数。详情请参阅 CMP0085。

Autogen¶

  • 添加了 AUTOMOC_EXECUTABLE、AUTORCC_EXECUTABLE 和 AUTOUIC_EXECUTABLE 目标属性。它们都接受一个可执行文件的路径，并强制 automoc/autorcc/autouic 使用该可执行文件。

    设置这些属性还将阻止配置时对这些可执行文件的测试。这主要在你自行构建这些工具时很有用。

  • 新的变量 CMAKE_GLOBAL_AUTOGEN_TARGET、CMAKE_GLOBAL_AUTOGEN_TARGET_NAME、CMAKE_GLOBAL_AUTORCC_TARGET 和 CMAKE_GLOBAL_AUTORCC_TARGET_NAME 控制全局 autogen 和 autorcc 目标的生成。

  • 可以设置新的 CMAKE_AUTOGEN_ORIGIN_DEPENDS 变量和 AUTOGEN_ORIGIN_DEPENDS 目标属性，以启用或禁用将原始目标依赖项转发到相应的 <ORIGIN>_autogen 目标。

CTest¶

  • ctest(1) 增加了 --show-only=json-v1 选项，以机器可读的 JSON 格式显示测试列表。请参阅手册中的 以 JSON 对象模型显示 部分。

  • ctest_submit() 命令增加了一个新的 Done 部分，可用于通知 CDash 构建已完成，不会再有其他部分上传。

  • CTest 学会了接受来自单个变量的仪表板服务器提交 URL。请参阅 ctest(1) 中的 SubmitURL 设置、CTEST_SUBMIT_URL 变量以及 ctest_submit() 命令的 SUBMIT_URL 参数。

弃用和移除的功能¶

  • 为策略 CMP0064 和 CMP0065 添加了明确的弃用诊断（CMP0063 及以下版本已被弃用）。cmake-policies(7) 手册解释了所有策略的 OLD 行为均已被弃用，项目应迁移到 NEW 行为。

  • Xcode 生成器弃用了对 Xcode 5 之前版本的支持。未来版本的 CMake 将取消对这些版本的支持。

  • FindQt 模块不再被 find_package() 命令作为查找模块使用。这允许 Qt 项目上游可选地提供其自己的 QtConfig.cmake 包配置文件，并让应用程序通过 find_package(Qt) 而非 find_package(Qt CONFIG) 来使用它。请参阅策略 CMP0084。

  • 对在 Windows（微软桌面系统） XP 和 Windows（微软桌面系统） Vista 上运行 CMake 的支持已被放弃。cmake.org 上提供的预编译 Windows（微软桌面系统） 二进制文件现在需要 Windows（微软桌面系统） 7 或更高版本。

  • CTest 不再支持通过 ftp、scp、cp 和 xmlrpc 进行提交。CDash 是 CTest 唯一维护的测试仪表板，它仅支持通过 http 和 https 进行提交。

  • MacroAddFileDependencies 模块已被弃用。请将项目移植为直接使用 set_property()。

其他更改¶

  • 修复了对象库链接，以将对象库的私有链接库传播到消费目标。

  • add_subdirectory() 下的安装规则现在与调用目录中的安装规则交错。详情请参阅策略 CMP0082。

  • CMake 现在强制设置最大递归深度，以防止脚本无限递归导致栈溢出。该限制可在运行时通过 CMAKE_MAXIMUM_RECURSION_DEPTH 进行调整。

  • 当通过 CMAKE_<LANG>_CPPCHECK 变量或 <LANG>_CPPCHECK 属性使用 cppcheck 时，如果 cppcheck 返回非零值（按其命令行选项配置），构建现在将失败。

  • 当设置了 POSITION_INDEPENDENT_CODE 时，现在会添加管理位置无关可执行文件所需的链接选项。项目负责使用 CheckPIESupported 模块来检查 PIE 支持，以确保 POSITION_INDEPENDENT_CODE 目标属性在可执行文件的链接时生效。此行为由策略 CMP0083 控制。

  • VS 2010 及以上版本的 Visual Studio（集成开发环境） 生成器 学会了支持通过 add_custom_target() 创建的目标上的 VS_DEBUGGER_* 属性。

  • CPack 模块不再默认使用 CPACK_DEBIAN_ARCHIVE_TYPE 变量中的 paxr 值，因为 dpkg 从未支持 PAX tar 格式。paxr 值将被映射到 gnutar 并发出弃用消息。

  • 如果 install(TARGETS) 命令中列出的目标的 EXCLUDE_FROM_ALL 属性设置为 true，CMake 不再发出警告。

更新¶

自 CMake 3.14.0 以来的变更包括以下内容。

3.14.1¶

  • 3.14.0 添加的 FindFontconfig 模块意外使用了不符合我们约定的全大写 FONTCONFIG_* 变量名。3.14.1 修订了该模块以使用 Fontconfig_* 变量名。这与 3.14.0 不兼容，但由于该模块在 3.14 系列中是新增的，其使用应该尚未普及。

3.14.3¶

  • 添加了 CMAKE_VS_PLATFORM_NAME_DEFAULT 变量，以帮助工具链文件使用 Visual Studio 16 2019 生成器，其中默认平台现在取决于宿主平台。

3.14.4¶

  • 在 CMake 3.14.0 到 3.14.3 中，调用 target_link_libraries() 为在其他目录中创建的静态库添加 PRIVATE 依赖项（在策略 CMP0079 NEW 行为下），会错误地将这些依赖项的使用要求传播给链接该静态库的从属目标。此问题已修复。该 Bug 也存在于 3.13.0 到 3.13.4 中，并已在 3.13.5 中修复。

3.14.5¶

  • CPATH 环境变量的条目不再像在 CMake 3.14.0 到 3.14.4 中那样，被显式排除在 include_directories() 和 target_include_directories() 的使用之外。

3.14.6¶

  • 在 CMake 3.14.0 到 3.14.5 中，FindBISON 模块的策略 CMP0088 NEW 行为意外地将 .y 输入的相对路径解释为相对于构建树目录，而非源码树目录。此问题已修复。

3.14.7¶

  • 在 CMake 3.14.0 到 3.14.6 中，EXCLUDE_FROM_ALL 目录属性回归到 3.14 之前的行为，导致目录内的目标甚至从其自身的“all”中被排除。此问题已修复。
