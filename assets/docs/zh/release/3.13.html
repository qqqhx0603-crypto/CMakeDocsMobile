自 CMake 3.12 以来的变更包括以下内容。

新功能¶

生成器¶

  • 用于 VS 2010 及以上版本的 Visual Studio 生成器 现已支持 INTERPROCEDURAL_OPTIMIZATION 目标属性以及配套的 CheckIPOSupported 模块。

  • Xcode 生成器现已支持配置更多的 Xcode Scheme 字段。请参阅 CMAKE_XCODE_GENERATE_SCHEME 变量。

  • Green Hills MULTI 生成器已更新

    • 增加了通过 CMAKE_GENERATOR_PLATFORM 进行架构选择的支持：例如 arm, ppc 和 86。

    • 增加了通过 CMAKE_GENERATOR_TOOLSET 进行工具集选择的支持，例如 comp_201205, comp_201510, comp_201722_beta。

    • 增加了通过 GHS_TARGET_PLATFORM 进行平台选择的支持，例如 integrity, linux, standalone 等。

    • 不再检查是否安装了基于 arm 的编译器，而是确保存在正确的 gbuild.exe。

    • 不再硬编码 ARM 文件、BSP、工具集或操作系统路径。

命令行¶

  • cmake(1) 命令增加了 -S <source_dir> 命令行选项，用于指定源目录的位置。此选项可以独立于 -B 使用。

  • cmake(1) 命令增加了 -B <build_dir> 命令行选项，用于指定构建目录的位置。此选项可以独立于 -S 使用。

  • cmake(1) 的 -E create_symlink 命令现在可以在 Windows 上使用。

命令¶

  • add_custom_command() 和 add_custom_target() 命令现已支持在 WORKING_DIRECTORY 选项中使用生成器表达式。

  • 创建了 add_link_options() 命令，用于在当前目录中添加链接选项。

  • install(TARGETS) 命令现已支持安装在当前目录之外创建的目标。

  • link_directories() 命令增加了用于控制插入位置的选项。

  • list(SORT) 命令增加了用于控制条目排序比较操作的选项。

  • math() 命令增加了十六进制相关的选项。

  • 创建了 target_link_directories() 命令，用于为目标及其依赖项指定链接目录。

  • 创建了 target_link_options() 命令，用于为目标及其依赖项指定链接选项。

  • target_link_libraries() 命令现在可以调用以修改在当前目录之外创建的目标。请参阅策略 CMP0079。

变量¶

  • 添加了 CMAKE_AUTOGEN_VERBOSE 变量，用于在 CMake 项目代码内选择性地增加 AUTOMOC、AUTOUIC 和 AUTORCC 的详细输出。

  • 添加了 CMAKE_VS_GLOBALS 变量，用于在创建目标时初始化目标的 VS_GLOBAL_<variable> 属性。

属性¶

  • 添加了 DEPLOYMENT_ADDITIONAL_FILES 目标属性，用于告知 Visual Studio 9 2008 生成器为 WinCE 设备的远程调试指定额外的部署文件。

  • 创建了 INTERFACE_LINK_DEPENDS 目标属性，用于指定文件上的传递性链接依赖。

  • LINK_DEPENDS 目标属性现已支持 生成器表达式。

  • 添加了 LINK_DIRECTORIES 和 INTERFACE_LINK_DIRECTORIES 目标属性，用于收集目标及其依赖项的链接目录。请使用 target_link_directories() 命令进行设置。

  • 添加了 LINK_OPTIONS 和 INTERFACE_LINK_OPTIONS 目标属性，用于收集目标及其依赖项的链接选项。请使用 target_link_options() 命令进行设置。

  • 添加了 LINK_OPTIONS 目录属性，用于收集在当前目录下创建的目标的链接选项。请使用 add_link_options() 命令进行设置。

  • 创建了 STATIC_LIBRARY_OPTIONS 目标属性，用于指定创建静态库时使用的归档器选项。

  • 创建了 VS_DEBUGGER_COMMAND_ARGUMENTS 目标属性，用于设置 VS 2010 及以上版本 Visual Studio 生成器 的调试命令行参数。

  • 创建了 VS_DEBUGGER_ENVIRONMENT 目标属性，用于设置 VS 2010 及以上版本 Visual Studio 生成器 的调试环境。

  • VS_DEBUGGER_COMMAND 和 VS_DEBUGGER_WORKING_DIRECTORY 目标属性现在支持生成器表达式。

模块¶

  • FindBoost 模块增加了 Boost_ARCHITECTURE 选项，用于指定 Boost 架构特定的库文件名片段。

  • FindCURL 模块现在可以分别查找调试版本和发布版本。

  • FindMatlab 模块增加了新组件 ENGINE_LIBRARY 和 DATAARRAY_LIBRARY，分别用于请求查找 Matlab C++ Engine 和 DataArray 库。

  • FindMatlab 模块现在在 Visual Studio 中显式导出 mexFunction。

  • FindMatlab 模块增加了一个新的 MCC_COMPILER 组件，用于请求查找 Matlab Compiler 插件。

  • FindPkgConfig 模块增加了在全局作用域中创建导入目标的选项。

  • FindPkgConfig 模块增加了除已支持的 >=, <= 和 = 之外，用于版本检查的 < 和 > 运算符支持。

  • FindPython3, FindPython2 和 FindPython 模块获得了控制 macOS (Framework) 和 Windows (Registry) 资源查找顺序的能力。

  • FindSubversion 模块的 Subversion_WC_INFO 命令增加了 IGNORE_SVN_FAILURE 选项，用于抑制失败报错（例如当源目录不在 Subversion 控制之下时）。

  • UseSWIG 模块现已学会管理用于 SWIG 编译的目标属性 INCLUDE_DIRECTORIES。

CTest¶

  • ctest(1) 增加了 --progress 选项，以便在输出到终端时启用实时测试进度摘要。

CPack¶

  • CPack DEB 生成器 在设置 CPACK_DEBIAN_DEBUGINFO_PACKAGE 时，现已支持将调试符号拆分为相应的 .ddeb 包。

  • CPack DEB 生成器 现已支持在打包文件时遵循 SOURCE_DATE_EPOCH 环境变量。这对生成可重现的包非常有用。

  • CPack 增加了一个新的 CPack External 生成器，用于以其他软件可以理解的格式导出 CPack 元数据。该生成器的目的是允许外部打包软件在无法使用 CPack 完成整个打包过程时，也能利用 CPack 的功能。

已弃用和已删除的功能¶

  • 为策略 CMP0055 到 CMP0063 添加了明确的弃用诊断（CMP0054 及以下版本已被弃用）。cmake-policies(7) 手册说明了所有策略的 OLD 行为均已弃用，项目应迁移到 NEW 行为。

其他更改¶

  • cmake.org 提供的预编译二进制文件现在包含 qthelp 格式的文档。

  • option() 命令现在会尊重同名的现有普通变量，并直接跳过，而不是尝试创建缓存条目（或设置其类型）并删除普通变量。请参阅策略 CMP0077。

  • Makefile 生成器 现在可以在 make clean 期间删除自定义命令和自定义目标的副产品。

  • target_sources() 命令现在将相对源文件路径解析为相对于当前源目录。这简化了从子目录逐步构建目标源文件的过程。添加了 CMP0076 策略以提供向后兼容性。

  • BundleUtilities 模块可能不再在配置时被包含。这本身一直是一个错误。请参阅策略 CMP0080。

  • UseSWIG 模块已更改了目标命名的策略。请参阅策略 CMP0078。

  • LINK_DIRECTORIES 目标属性现在要求使用绝对路径。请参阅策略 CMP0081。

  • CPack 生成器已移动到文档中单独的部分，不再位于其内部实现模块中。这些内部实现模块也不再提供给那些可能错误地引用它们的脚本，因为它们本就不应该被直接引用。

更新¶

自 CMake 3.13.0 以来的变更包括以下内容。

3.13.2¶

  • CMake 3.13.0 包含了一项更改，将编译器隐式包含目录传递给 AUTOMOC 的 moc 工具。此更改因导致现有构建回归而被撤销，在后续版本重新引入之前需要进一步调查。

3.13.3¶

  • Visual Studio 15 2017 生成器已修复，使其在安装 VS 2019 时也能正常工作。

  • CMake 现在检查在运行 CMake 时是否至少指定了源目录或二进制目录中的一个，如果两者都缺失则报错。这一直是一个既定的文档要求，但之前的实现意外地允许了在两者都不指定的情况下，只要有其他参数存在，就会静默地使用当前工作目录作为源和构建树的情况。

3.13.4¶

  • 3.13.3 版本中在未指定源目录或二进制目录时添加的错误已降级为警告。虽然这从未是预期、文档记录或支持的行为，但某些项目依赖于它。该错误在 3.13.x 版本的其余部分中被降级为警告，以允许过渡期，但它可能会在以后的版本中再次成为致命错误。依赖旧行为的脚本可以通过显式指定源目录的路径（即使只是 .）来轻松修复，并可在所有版本的 CMake 中继续工作。

3.13.5¶

  • 在 CMake 3.13.0 到 3.13.4 中，调用 target_link_libraries() 为在另一个目录中创建的静态库添加 PRIVATE 依赖项（在策略 CMP0079 NEW 行为下）会错误地将这些依赖项的用法需求传播给链接该静态库的依赖方。此问题已修复。
