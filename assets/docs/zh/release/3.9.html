自 CMake 3.8 以来的改进包括：

新功能¶

语言¶

  • Visual Studio Generators（VS 2010 及以上版本）现已支持 CUDA。这补充了 Makefile（构建脚本） Generators 和 Ninja（构建工具） 生成器现有的支持。由于早期版本在 VS 集成中存在已知错误，建议使用 CUDA 8.0.61 或更高版本。

  • 对于以下 编译器标识，CMake 现已能够识别 C++ 标准 和 C 标准 及其相关的元特性：Cray、PGI 和 XL。

生成器¶

  • Visual Studio（集成开发环境） Generators（Visual Studio（集成开发环境） 生成器）（VS 2010 及以上版本）学会了在安装了 nasm 的情况下支持 ASM_NASM 语言。

  • Xcode 生成器学会了创建 Xcode 方案（schema）文件。这是一个实验性功能，可以通过将 CMAKE_XCODE_GENERATE_SCHEME 变量设置为 TRUE 来激活。

命令¶

  • add_library() 命令的 IMPORTED 选项现已支持 对象库（Object Libraries）。

  • find_library() 命令学会了在构建目标为 x32 ABI 时搜索 libx32 路径。请参见 FIND_LIBRARY_USE_LIBX32_PATHS 全局属性。

  • include_external_msproject() 命令学会了使用 MAP_IMPORTED_CONFIG_<CONFIG> 目标属性，将当前配置映射到外部配置。

  • install(TARGETS) 命令增加了一个新的 OBJECTS 选项，用于指定安装 对象库 的位置。

  • install(EXPORT) 命令学会了如何导出 对象库。

  • project() 命令增加了一个可选的 DESCRIPTION 参数，用于设置 PROJECT_DESCRIPTION 变量。

  • separate_arguments() 命令增加了一个 NATIVE_COMMAND 模式，该模式根据主机操作系统执行参数分离。

变量¶

  • 增加了 CMAKE_ANDROID_NDK_DEPRECATED_HEADERS 变量，供在 使用 NDK 为 Android 进行交叉编译 时使用，以请求使用已弃用的头文件（即使统一头文件可用）。目前的默认行为是在可用时使用统一头文件。

  • 引入了 CMAKE_AUTOMOC_DEPEND_FILTERS 变量，允许 CMAKE_AUTOMOC 从源文件内容中为 moc 提取额外的依赖文件名。

  • 引入了 CMAKE_AUTOUIC_SEARCH_PATHS 变量，允许 CMAKE_AUTOUIC 在比包含 ui_foo.h 的文件更广泛的区域搜索 foo.ui。

  • 增加了 CMAKE_FIND_LIBRARY_CUSTOM_LIB_SUFFIX 变量，告知 find_library() 命令在搜索每个通常会检查的 lib 目录之前，先搜索 lib<suffix> 目录。

  • 增加了 CMAKE_INTERPROCEDURAL_OPTIMIZATION 变量，用于初始化所有目标上的 INTERPROCEDURAL_OPTIMIZATION 属性。

  • 增加了 CMAKE_<LANG>_COMPILER_AR 变量，用于保存 GCC/Clang 的 ar 包装器路径。

  • 增加了 CMAKE_<LANG>_COMPILER_RANLIB 变量，用于保存 GCC/Clang 的 ranlib 包装器路径。

  • 增加了 CMAKE_SYSROOT_COMPILE 和 CMAKE_SYSROOT_LINK 变量，以便在编译和链接时使用单独的 sysroot。

属性¶

  • 引入了新的 AUTOGEN_BUILD_DIR 目标属性，用于为 AUTOMOC、AUTOUIC 和 AUTORCC 设置自定义输出目录。

  • 引入了新的 AUTOMOC_DEPEND_FILTERS 目标属性，允许 AUTOMOC 从源文件内容中为 moc 提取额外的依赖文件名。

  • 引入了新的 AUTOUIC_SEARCH_PATHS 目标属性，允许 AUTOUIC 在比包含 ui_foo.h 的文件更广泛的区域搜索 foo.ui。

  • 引入了全局属性 AUTOGEN_SOURCE_GROUP、AUTOMOC_SOURCE_GROUP 和 AUTORCC_SOURCE_GROUP，允许将由 AUTOMOC 或 AUTORCC 生成的文件放入 source_group() 中。

  • 增加了 BUILD_WITH_INSTALL_NAME_DIR 目标属性和对应的 CMAKE_BUILD_WITH_INSTALL_NAME_DIR 变量，用于控制是否在构建树中使用二进制文件的 INSTALL_NAME_DIR 目标属性值。这与 macOS（苹果桌面系统） 的 install_name 的关系，正如 BUILD_WITH_INSTALL_RPATH 与 RPATH 的关系一样。

  • 增加了 CUDA_PTX_COMPILATION 目标属性，用于 对象库，以支持编译为 .ptx 文件而非主机目标文件。

  • 增加了 GENERATOR_IS_MULTI_CONFIG 全局属性，用于确定当前生成器是否为多配置生成器（例如 Visual Studio（集成开发环境） Generators（Visual Studio（集成开发环境） 生成器） 或 Xcode）。

  • INTERPROCEDURAL_OPTIMIZATION 目标属性在启用时现已强制执行。CMake 将无条件添加 IPO 标志，或者如果不知道当前编译器的标志，则产生错误。现在项目有责任使用 CheckIPOSupported 模块在启用该目标属性之前检查 IPO 支持情况。请参见策略 CMP0069。

  • WINDOWS_EXPORT_ALL_SYMBOLS 目标属性现在可以与显式的 .def 文件结合使用，以便导出目标内对象文件中的所有符号，以及链接器在依赖项中找到的显式符号列表（例如 msvcrt.lib）。

模块¶

  • 增加了 CheckIPOSupported 模块，帮助项目检查当前工具链和 CMake 版本是否支持过程间优化（IPO）。

  • CMakeFindDependencyMacro 模块的 find_dependency 宏现在会将所有参数转发给底层的 find_package() 调用。现有的使用方式将照常运行，但调用者现在可以访问 find_package() 接受的所有参数套件。

  • FeatureSummary 模块的 feature_summary() 命令现在接受新的 DEFAULT_DESCRIPTION 选项，该选项将打印选定包类型的默认标题。

  • FeatureSummary 模块增加了一个新的 FeatureSummary_<TYPE>_DESCRIPTION 变量，可以为每种 <TYPE> 定义，以便在模块使用包类型作为输出字符串时，将类型名称替换为指定的字符串。

  • FindDoxygen 模块学会了使用 CMake 变量控制 Doxygen 行为，并通过新增加的 doxygen_add_docs() 命令生成文档。Doxygen 输入文件（Doxyfile）会自动生成，并且 doxygen 将作为自定义目标的一部分运行。可以指定额外的组件来查找可选工具：dot、mscgen 和 dia。

  • FindMPI 模块现已提供导入目标。

  • FindProtobuf 模块的 protobuf_generate_cpp() 命令增加了一个 EXPORT_MACRO 选项，用于指定 DLL（动态链接库） 导出标记宏的名称。

  • FindProtobuf 模块现在通过一个新的 Protobuf_USE_STATIC_LIBS 输入变量支持在 Unix 上使用静态库。

  • FindProtobuf 模块现在在找到库时提供导入目标。

  • 新增了 GoogleTest 模块，用于独立于 FindGTest 模块提供 gtest_add_tests() 函数。该函数还进行了更新以支持关键字参数，其功能扩展为允许指定测试名称的前缀和后缀，使源文件的依赖关系成为可选，并允许将发现的测试用例列表返回给调用者。

CTest¶

  • ctest_submit() 命令增加了一个 HTTPHEADER 选项，用于指定在提交过程中发送的自定义头部信息。

  • ctest(1) 可执行文件增加了一些新选项，允许开发人员禁用自动将测试添加到测试集以满足装置依赖关系（fixture dependencies）。-FS 阻止为匹配提供的正则表达式的装置添加设置测试，-FC 阻止为匹配的装置添加清理测试，-FA 阻止为匹配的装置添加任何测试。

  • 增加了 DISABLED 测试属性，用于标记已配置但被显式禁用的测试，使它们不会运行。

CPack¶

  • CPack Archive Generator 学会了按组件修改文件名。请参见 CPACK_ARCHIVE_FILE_NAME 变量及其按组件版本 CPACK_ARCHIVE_<component>_FILE_NAME。

  • CPackComponent 模块的 cpack_add_component() 命令增加了一个新的 PLIST <filename> 选项，用于在使用 productbuild 生成器时指定 pkgbuild 的 --component-plist 参数。

  • CPackIFW 模块的 cpack_ifw_configure_component() 和 cpack_ifw_configure_component_group() 命令增加了对 DISPLAY_NAME 和 DESCRIPTION 选项的国际化支持。

  • CPack IFW Generator 学会了新的提示变量 CPACK_IFW_ROOT，用于查找安装在非标准位置的 QtIFW 工具套件。

  • CPack productbuild Generator 增加了一个新的 CPACK_PRODUCTBUILD_RESOURCES_DIR 变量，用于指定要复制到 Resources 目录中的资源。

  • CPack RPM Generator 学会了修改 debuginfo 包名称。请参见 CPACK_RPM_DEBUGINFO_FILE_NAME 变量。

  • CPack WIX Generator 的补丁系统现在有能力设置额外的属性。这可以通过在 Id 属性之后指定带有 CPackWiXFragment XML 标记的属性来完成。请参见 CPACK_WIX_PATCH_FILE 变量。

  • CPack WIX Generator 实现了一个新的 CPACK_WIX_ROOT_FOLDER_ID 变量，允许使用自定义的根文件夹 ID，而不是默认的 ProgramFilesFolder / ProgramFiles64Folder。

其他¶

  • 过程间优化（IPO）现已支持 GNU 和 Clang 编译器，使用链接时优化（LTO）标志。请参见 INTERPROCEDURAL_OPTIMIZATION 目标属性和 CheckIPOSupported 模块。

  • 生成器表达式 TARGET_OBJECTS 现在被 add_custom_command() 和 file(GENERATE) 命令所支持。

  • 增加了两个用于检索 Apple Bundle 目录的新信息性生成器表达式。第一个 $<TARGET_BUNDLE_DIR:tgt> 输出到 Bundle 目录的完整路径，另一个 $<TARGET_BUNDLE_CONTENT_DIR:tgt> 输出到 macOS（苹果桌面系统） Bundles 和 App Bundles 的 Contents 目录的完整路径。对于所有其他 bundle 类型和 SDK，它与 $<TARGET_BUNDLE_DIR:tgt> 相同。这些新表达式有助于查询与 macOS（苹果桌面系统） 和 iOS 上不同 Bundle 类型和布局无关的 Bundle 位置。

已弃用和已删除的功能¶

  • 为策略 CMP0036 及以下版本添加了显式的弃用诊断。cmake-policies(7) 手册解释了所有策略的旧（OLD）行为已被弃用，项目应尽快移植到新（NEW）行为。

  • Visual Studio 8 2005 生成器现已被弃用，并将在未来版本的 CMake 中删除。

  • Visual Studio 7 .NET 2003 生成器已被删除。

  • Xcode 生成器放弃了对旧于 3 版本的 Xcode 的支持。

  • FindDoxygen 模块已弃用多个变量。

  • 由于上游 curl 7.52 的更改，随 CMake 捆绑的 curl 版本在 Windows（微软桌面系统） 上不再接受 file://c:/... 格式的 URL。请使用 file:///c:/... 格式以兼容所有版本。

其他更改¶

  • 在使用 AUTOMOC 时，CMake 现在会扫描 Q_PLUGIN_METADATA 宏的存在，并在宏的 FILE 参数指定的文件发生更改时重新运行 moc。

  • 当 AUTOMOC 检测到 #include "moc_<basename>.cpp" 形式的包含语句时，对相应头文件的搜索现在也会查看目标的 INCLUDE_DIRECTORIES。

  • 在运行测试时，CTest 学会了将跳过的测试（使用 SKIP_RETURN_CODE 属性）视为与具有新 DISABLED 属性的测试相同。由于此更改，当所有测试要么被跳过要么通过时，CTest 将不再报告失败。

  • Ninja（构建工具） 生成器放宽了对象编译的依赖关系。对象编译现在仅依赖于与对象目标所依赖的库关联的自定义目标和自定义命令，而不再依赖于库本身。依赖目标中的源文件现在可以在不等待其目标依赖项链接的情况下进行编译。

  • 在 macOS（苹果桌面系统） 上，RPATH 设置（如 BUILD_WITH_INSTALL_RPATH）不再影响 install_name 字段。请参见策略 CMP0068。

  • Visual Studio 14 2015 生成器已被告知有关 VS 2015 更新对 v140 工具集所做的更改。VS 更改了其理解用于生成 -DEBUG 链接器标志变体的 GenerateDebugInformation 链接器设置的值集。

更新¶

自 CMake 3.9.0 以来的改进包括：

3.9.1¶

  • 由于 CMake 3.9.0 新增使用 <PackageName>_ROOT 变量导致的回归，CMake 3.9.0 添加的 find_ 命令 PACKAGE_ROOT 搜索路径组已在 3.9 系列中移除。未来可能会以更兼容的方式重新引入该行为。

3.9.2¶

  • 在 macOS（苹果桌面系统） 上，默认的应用包 Info.plist 文件不再像 3.9.0 和 3.9.1 中那样启用 Hi-DPI 支持。此更改必须撤销，因为它破坏了 iOS 应用程序。

  • Xcode 生成器不再像在 3.9.0 和 3.9.1 中那样向自定义脚本构建阶段添加 "outputPaths"。这是为了支持 Xcode 9 的新构建系统而添加的，但破坏了旧版和新版 Xcode 构建系统的增量重建。
