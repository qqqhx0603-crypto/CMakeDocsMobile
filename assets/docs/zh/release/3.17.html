自 CMake 3.16 以来的更改包括以下内容。

新功能¶

生成器¶

  • cmake(1) 增加了 Ninja Multi-Config 生成器，它类似于 Ninja 生成器，但可用于同时构建多种配置。

  • Visual Studio（集成开发环境） 生成器现在支持按配置（per-config）的源文件。此前只有命令行构建工具生成器支持此功能。

  • Visual Studio（集成开发环境） 生成器（针对 VS 2010 及更高版本）现在支持在 CMAKE_GENERATOR_TOOLSET 设置中为项目文件指定 VCTargetsPath 值。

  • Visual Studio（集成开发环境） 生成器（针对 VS 2010 及更高版本）增加了对 .NET Standard 和 .NET Core 的支持。请参阅 DOTNET_TARGET_FRAMEWORK 目标属性以及相关的 CMAKE_DOTNET_TARGET_FRAMEWORK 变量。

语言¶

  • 编译特性 (Compile Features) 功能现在为 CUDA 语言标准级别提供了元特性（例如 cuda_std_03, cuda_std_14）。请参阅 CMAKE_CUDA_KNOWN_FEATURES。

编译器¶

  • IBM XL Fortran 编译器现已受 Ninja 生成器支持。

命令行¶

  • cmake(1) 增加了 --debug-find 命令行选项，用于启用额外的可读输出，以显示 find_* 命令的搜索路径。

  • cmake(1) 增加了 --trace-format 命令行选项，可用于设置 --trace 的输出格式。目前支持旧版人类可读格式和新的 JSON 格式。与现有格式相比，新的 JSON 格式更易于自动解析。

  • cmake(1) 增加了 -E rm 命令行工具，可用于删除目录和文件。该工具取代了现有的 -E remove 和 -E remove_directory 工具，并具备更好的语义。

命令¶

  • add_custom_command() 命令现在可以解释相对于当前二进制目录指定的 DEPENDS 参数路径。

  • foreach() 命令增加了新的 ZIP_LISTS 选项，用于同时遍历多个列表。

  • load_cache(READ_WITH_PREFIX) 命令模式现在允许在使用 cmake -P 运行脚本 时调用。

  • message() 命令现在可以输出由 CMAKE_MESSAGE_CONTEXT 变量提供的上下文信息（适用于 NOTICE 及更低级别的日志）。通过新的 --log-context 命令行选项或 CMAKE_MESSAGE_CONTEXT_SHOW 变量可启用此输出。

  • message() 命令增加了新的关键字 CHECK_START, CHECK_PASS 和 CHECK_FAIL。

  • target_compile_options() 命令现在更一致地遵循 BEFORE 关键字。请参阅策略 CMP0101。

变量¶

  • 添加了 CMAKE_CTEST_ARGUMENTS 变量，用于指定当通过生成的构建系统中的 test（或 RUN_TESTS）目标运行 CTest 时传递的命令行参数列表。

  • 以下变量现在在 function() 内部定义

    • CMAKE_CURRENT_FUNCTION

    • CMAKE_CURRENT_FUNCTION_LIST_DIR

    • CMAKE_CURRENT_FUNCTION_LIST_FILE

    • CMAKE_CURRENT_FUNCTION_LIST_LINE

  • 引入了 CMAKE_CUDA_RUNTIME_LIBRARY 变量和 CUDA_RUNTIME_LIBRARY 目标属性，用于在链接使用 CUDA 的目标时选择 CUDA 运行时库。

  • 引入了 CMAKE_FIND_DEBUG_MODE 变量，用于在 cmake 运行期间向标准错误输出额外的 find_* 调用信息。该输出旨在供人类阅读，而非用于解析。

  • CMAKE_EXPORT_COMPILE_COMMANDS 变量现在在没有显式配置的情况下，将从 CMAKE_EXPORT_COMPILE_COMMANDS 环境变量中获取其初始值。

  • CMAKE_<LANG>_COMPILER_LAUNCHER 变量如果未明确设置，现在将从同名的环境变量 CMAKE_<LANG>_COMPILER_LAUNCHER 中获取初始值。

  • CMAKE_MESSAGE_LOG_LEVEL 变量现在可用于在不同 CMake 运行之间持久化记录级别，这与仅适用于特定运行的 --log-level 命令行选项不同。

  • 添加了 CMAKE_XCODE_SCHEME_ENVIRONMENT 变量，用于初始化 XCODE_SCHEME_ENVIRONMENT 目标属性。

  • 添加了 CMAKE_XCODE_SCHEME_WORKING_DIRECTORY 变量及相关的 XCODE_SCHEME_WORKING_DIRECTORY 目标属性，用于通知 Xcode 生成器设置 Custom Working Directory 方案选项的值。

属性¶

  • 创建了 AIX_EXPORT_ALL_SYMBOLS 目标属性及相关的 CMAKE_AIX_EXPORT_ALL_SYMBOLS 变量，用于选择性地显式禁用 AIX 上共享库符号的自动导出。

  • 添加了 DEPRECATION 目标属性，用于将目标标记为已弃用。如果链接的目标被标记为已弃用，则会在生成时发出带有弃用消息的警告。

  • INSTALL_NAME_DIR 目标属性现在支持 生成器表达式。特别是，可以使用 $<INSTALL_PREFIX> 生成器表达式来设置相对于安装时前缀的目录。

  • 添加了 MACHO_COMPATIBILITY_VERSION 和 MACHO_CURRENT_VERSION 目标属性，分别用于设置 Mach-O 二进制文件的 compatibility_version 和 current_version。为了向后兼容，如果未设置这些属性，则分别使用 SOVERSION 和 VERSION 作为后备方案。

  • 添加了 VS_DOTNET_DOCUMENTATION_FILE 目标属性，用于通知 Visual Studio（集成开发环境） 生成器在 .csproj 文件中生成 DocumentationFile 引用。

模块¶

  • ExternalProject 模块的 ExternalProject_Add() 命令增加了 GIT_SUBMODULES_RECURSE 选项，用于指定是否应递归更新 Git 子模块。默认为开启，以保持现有行为。

  • 添加了 FindCUDAToolkit 模块，用于在不启用 CUDA 语言的情况下查找 CUDA 工具包。

  • FindCURL 模块学会了使用 CURL 的 cmake 构建系统生成的 CURLConfig.cmake 配置文件来查找 CURL。同时增加了新的 CURL_NO_CURL_CMAKE 选项以禁用此行为。

  • FindFLEX 模块的 flex_target() 命令现在以 CMAKE_CURRENT_BINARY_DIR 作为工作目录运行 flex。请参阅策略 CMP0098。

  • FindLibArchive 模块现在为 libarchive 提供了一个导入目标。

  • FindPython 模块已学会查找由 conda 管理的活动虚拟环境中的 Python 组件。

  • FindPython3 和 FindPython 模块分别增加了 Python3_SOABI 和 Python_SOABI 变量，提供模块的标准扩展后缀。此外，命令 Python3_add_library() 和 Python_add_library() 增加了 WITH_SOABI 选项，用于在库后缀前添加 SOABI 的值。

  • FindLibXml2 模块现在为 xmllint 可执行文件提供了一个导入目标。

Autogen (自动生成)¶

  • AUTOMOC 和 AUTOUIC 学会了处理带有 .hh 扩展名的头文件。请参阅策略 CMP0100。

CTest¶

  • CTEST_CONFIGURATION_TYPE 变量现在在 ctest(1) 以 -C <cfg> 调用时从命令行设置。

  • ctest(1) 工具增加了对 Dr. Memory 进行内存检查运行的支持。

  • ctest(1) 工具增加了 --no-tests=<[error|ignore]> 选项，用于在未发现任何测试时，显式设置并统一直接调用与脚本模式之间的行为。

  • ctest(1) 工具增加了 --repeat <mode>:<n> 选项，用于指定重复测试的条件。这概括了现有的 --repeat-until-fail <n> 选项，并增加了 until-pass 和 after-timeout 模式。

  • ctest_test() 命令增加了 REPEAT <mode>:<n> 选项以指定重复测试的条件。

CPack¶

  • CPack DragNDrop 生成器学会了使用 CPACK_DMG_<component>_FILE_NAME 变量，以便在将组件打包到各自的 DMG 中时设置自定义文件名。

  • CPack DragNDrop 生成器学会了处理 RTF 格式的许可文件。当设置 CPACK_DMG_SLA_DIR 变量时，会考虑 <language>.license.rtf，但为了保持向后兼容性，这仅作为在未找到纯文本 (.txt) 文件时的后备选项。

  • CPack NSIS 生成器增加了一个新变量 CPACK_NSIS_MUI_HEADERIMAGE 来设置页眉图像。为了不破坏现有设置，如果未设置该新变量，它仍默认为 CPACK_PACKAGE_ICON。

  • CPack NSIS 生成器现在支持 CPACK_NSIS_UNINSTALL_NAME，可用于指定卸载程序的名称。

  • CPack NSIS 生成器现在支持 CPACK_NSIS_WELCOME_TITLE 和 CPACK_NSIS_WELCOME_TITLE_3LINES。它们可用于指定欢迎页面的标题并以 3 行显示。

  • CPack NSIS 生成器现在支持 CPACK_NSIS_FINISH_TITLE 和 CPACK_NSIS_FINISH_TITLE_3LINES。它们可用于指定完成页面的标题并以 3 行显示。

  • CPack productbuild 生成器增加了对 CPACK_PRODUCTBUILD_BACKGROUND 变量的支持，用于指定 macOS（苹果桌面系统） 安装程序的背景图像。

其他¶

  • ccmake(1) 现在如果终端支持颜色，会根据条目类型以不同颜色显示缓存值。

  • ccmake(1) 现在会在配置和生成期间显示消息和进度条。如果有任何错误或警告，它将保持输出显示。

已弃用和已移除的功能¶

  • 为策略 CMP0068 和 CMP0069 添加了显式的弃用诊断信息（CMP0067 及更早版本已弃用）。cmake-policies(7) 手册说明了所有策略的 OLD 行为均已弃用，项目应迁移到 NEW 行为。

  • CPack PackageMaker 生成器已被弃用，因为 Xcode 不再分发 PackageMaker 工具。未记录的 OSXX11 生成器也已被弃用。

  • cmake(1) 的命令行 -E remove 和 -E remove_directory 工具已被弃用，建议改用新的 -E rm 工具。旧工具在命名路径不存在时（即使没有 force 选项）总是返回 0，且不破坏兼容性无法修复，因此已被取代。

  • CPack NSIS 生成器现在需要 NSIS 3.0 或更高版本。

其他更改¶

  • file API 索引文件现在发出一个 multiConfig 标志，指定生成器是否支持多种输出配置。

  • 目标链接属性 INTERFACE_LINK_OPTIONS, INTERFACE_LINK_DIRECTORIES 和 INTERFACE_LINK_DEPENDS 现在对于静态库的私有依赖项具有传递性。请参阅策略 CMP0099。

  • 使用 MinGW 工具时，find_library() 命令默认不再查找 .dll 文件。相反，它期望能够获取 .dll.a 导入库。

  • MinGW Makefiles 生成器如果环境变量 PATH 中存在 sh.exe，不再发出错误。

  • Ninja 生成器现在倾向于使用 PATH 中出现的第一个 ninja 构建工具，无论其名称是 ninja-build、ninja 还是 samu。此前，这些名称中在 PATH 中任何位置最先出现的会被优先选中。

  • 使用 SDCC 时，sdar 工具现在被优先用作库管理器，取代 sdcclib。后者在 SDCC 3.2.0 中被弃用，并在 SDCC 3.8.6 中被移除。

  • 使用 SDCC 时，默认标志不再包含任何特定于目标的标志。此前默认标志是针对 8051 硬编码的。

  • CMAKE_VS_GLOBALS 变量的值现在适用于编译器标识期间以及通过 add_custom_target() 命令创建的目标。

  • Xcode 生成器不再硬编码 -Wmost, -Wno-four-char-constants 和 -Wno-unknown-pragmas 警告标志。

更新¶

自 CMake 3.17.0 以来的更改包括以下内容。

3.17.1¶

  • CMake 3.17.0 更新了 CPack NSIS 生成器，其中包含需要 NSIS 3.0 或更高版本的更改。CMake 3.17.1 现在强制使用足够新的版本。

3.17.3¶

  • 如果未设置 OBJC 或 OBJCXX 环境变量，Objective C 或 C++ 编译器的选择现在会考虑 CC 或 CXX 环境变量。

  • FindPkgConfig 模块现在将以 -isystem 为前缀的包含目录提取到 *_INCLUDE_DIRS 变量和 INTERFACE_INCLUDE_DIRECTORIES 目标属性中。此前它们会放置在 *_CFLAGS_OTHER 变量和 INTERFACE_COMPILE_OPTIONS 目标属性中。

3.17.5¶

  • CMAKE_AUTOMOC_PATH_PREFIX 的默认值已更改为 OFF，因为此功能可能会破坏在不同包含目录中具有同名头文件的现有项目。这恢复了与 CMake 3.15 及更早版本行为的兼容性。该默认值在 3.16.9 中也已更改为 OFF。
