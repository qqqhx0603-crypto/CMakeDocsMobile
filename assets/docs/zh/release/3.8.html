自 CMake 3.7 以来的变更包括以下内容。

新特性¶

语言¶

C#¶

  • CMake 已支持将 CSharp (C#) 作为一等公民语言，可以通过 project() 和 enable_language() 命令启用。目前，VS 2010 及更高版本的 Visual Studio 生成器 已提供支持。

    C# 程序集和程序可以像普通 C++ 目标一样，使用 add_library() 和 add_executable() 命令添加。同一源树中 C# 目标之间的引用可以像 C++ 一样通过 target_link_libraries() 指定。对系统或第三方程序集的引用可以通过目标属性 VS_DOTNET_REFERENCE_<refname> 和 VS_DOTNET_REFERENCES 指定。

  • 可以使用目标和源文件属性对 C# 目标进行更精细的微调。特别值得关注的是与 Visual Studio 相关的目标属性（VS_*），可用于设置工具集版本、根命名空间、程序集图标等。

CUDA¶

  • CMake 已支持将 CUDA 作为一等公民语言，可以通过 project() 和 enable_language() 命令启用。

  • CUDA 目前在 Linux、macOS 和 Windows 上由 Makefile 生成器 和 Ninja 生成器支持。对 Visual Studio IDE 的支持正在开发中，不包含在此版本中。

  • 支持 NVIDIA CUDA 工具包编译器 (nvcc)。

C & C++¶

  • 编译特性功能现在提供元特性，可为特定的语言标准级别（例如 cxx_std_11）请求编译器模式。请参见 CMAKE_C_KNOWN_FEATURES 和 CMAKE_CXX_KNOWN_FEATURES。

  • 编译特性功能现在能够识别 C++ 17。除了 cxx_std_17 元特性外，尚未枚举具体特性。

  • 编译特性功能现在可以识别 gcc 3.4 版本及以上对 C99 的支持。

平台¶

  • 添加了一个用于 Fuchsia 的新最小平台文件。

生成器¶

  • CodeBlocks 额外生成器现在可以配合 NMake Makefiles JOM 使用。

  • VS 2013 及以上版本的 Visual Studio 生成器 现在支持在 CMAKE_GENERATOR_TOOLSET 值中使用 host=x64 选项（例如通过 cmake(1) 的 -T 选项），以请求在 64 位主机上使用 64 位 VS 工具链。

  • Visual Studio 生成器 现在会将传递给 target_link_libraries() 且以 .targets 结尾的文件视为 MSBuild “targets” 文件，并导入到生成的项目文件中。

命令¶

  • add_custom_command() 和 add_custom_target() 命令增加了 COMMAND_EXPAND_LISTS 选项，该选项会展开 COMMAND 参数中的列表，包括由生成器表达式创建的列表。

  • execute_process() 命令增加了 ENCODING 选项，用于在 Windows 上指定子进程输出使用的编码。

  • math(EXPR) 命令增加了对一元 + 和 - 运算符的支持。

  • source_group() 命令增加了 TREE 和 PREFIX 选项，用于按照源树目录结构添加分组。

  • string(TIMESTAMP) 命令现在支持使用 %% 对字面字符 % 进行转义。

  • string(TIMESTAMP) 命令现在会尊重 SOURCE_DATE_EPOCH 环境变量，并使用其值代替当前时间。

  • try_compile() 命令的源文件签名增加了新选项，用于指定在生成的测试项目中使用的语言标准。

  • try_compile() 命令的源文件签名现在会尊重语言标准变量，例如 CMAKE_CXX_STANDARD。请参见策略 CMP0067。

变量¶

  • 添加了 CMAKE_CODELITE_USE_TARGETS 变量，以告知 CodeLite 额外生成器将生成的项目调整为以目标为中心的组织结构。此后在 CodeLite 内的 build、rebuild 和 clean 操作将作用于所选的目标，而不是整个工作区。（请注意，目标上的 Ninja 清理操作包含其依赖项。）

  • 添加了 CMAKE_SUBLIME_TEXT_2_ENV_SETTINGS 变量，用于告知 Sublime Text 2 额外生成器将指定的环境变量放置在生成的 .sublime-project 中。

  • 添加了 CMAKE_SUBLIME_TEXT_2_EXCLUDE_BUILD_TREE 变量，用于告知 Sublime Text 2 额外生成器，当构建树位于源树内时，是否将其从 .sublime-project 中排除。

  • 添加了 CMAKE_VS_INCLUDE_PACKAGE_TO_DEFAULT_BUILD 变量，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 将 PACKAGE 目标包含在默认构建中，类似于现有的 CMAKE_VS_INCLUDE_INSTALL_TO_DEFAULT_BUILD 变量（用于 INSTALL 目标）。

属性¶

  • 添加了 BUILD_RPATH 目标属性及相应的 CMAKE_BUILD_RPATH 变量，以支持向构建树中的二进制文件添加自定义的 RPATH 位置。

  • COMPILE_FLAGS 源文件属性现在支持 生成器表达式。

  • FRAMEWORK 目标属性现在也可以应用于 Apple 目标上的静态库。它将生成一个标准的 Framework，但内部包含的是静态库。

  • 导入的 接口库 (Interface Libraries) 增加了新的 IMPORTED_LIBNAME 和 IMPORTED_LIBNAME_<CONFIG> 目标属性，用于指定链接库名称，因为接口库不构建自己的库文件。

  • 引入了 <LANG>_CPPLINT 目标属性及支持的 CMAKE_<LANG>_CPPLINT 变量，以告知 Makefile 生成器 和 Ninja 生成器在编译 C 和 CXX 语言时同时运行 cpplint 风格检查器。

  • 添加了 MANUALLY_ADDED_DEPENDENCIES 目标属性。它提供了一个只读列表，列出了使用 add_dependencies() 命令添加的依赖项。

  • MAP_IMPORTED_CONFIG_<CONFIG> 目标属性现在可以将空列表元素解释为引用 IMPORTED_LOCATION 指定的无配置导入位置。

  • NO_SYSTEM_FROM_IMPORTED 目标属性现在在 导入的 接口库 (Interface Libraries) 上受到支持。

  • 添加了新的源文件属性 SKIP_AUTOMOC、SKIP_AUTOUIC、SKIP_AUTORCC 和 SKIP_AUTOGEN，允许源文件排除在 AUTOMOC、AUTOUIC 和 AUTORCC 目标属性的处理范围之外。

  • 添加了 VS_COPY_TO_OUT_DIR 源文件属性，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 是否应将文件复制到输出目录。

  • 添加了 VS_DEBUGGER_WORKING_DIRECTORY 目标属性，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 应为目标设置什么调试器工作目录。

  • 添加了 VS_DOTNET_REFERENCES_COPY_LOCAL 目标属性，用于指定是否将引用的程序集复制到输出目录。

  • 添加了 VS_DOTNET_REFERENCE_<refname> 目标属性，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 添加带有给定提示路径的 .NET 引用。

  • 添加了 VS_INCLUDE_IN_VSIX 源文件属性，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 是否将文件包含在 Visual Studio 扩展包中。

  • 添加了 VS_RESOURCE_GENERATOR 源文件属性，为 VS 2010 及以上版本的 Visual Studio 生成器 提供资源生成器设置（仅限 C#）。

  • 添加了 VS_USER_PROPS 目标属性，用于告知 VS 2010 及以上版本的 Visual Studio 生成器 使用自定义的 MSBuild 用户 .props 文件。

  • 添加了 XCODE_EMIT_EFFECTIVE_PLATFORM_NAME 全局属性，用于告知 Xcode 生成器是否发出 EFFECTIVE_PLATFORM_NAME 变量。这在并行构建多个 SDK（如 macosx 和 iphoneos）时非常有用。

  • 创建了新的 XCODE_PRODUCT_TYPE 和 XCODE_EXPLICIT_FILE_TYPE 目标属性，用于告知 Xcode 生成器在生成的 Xcode 项目中为目标使用相应的自定义值。

模块¶

  • 添加了 CSharpUtilities 模块，以辅助参数化 Visual Studio C# 目标。它提供了一些函数，允许自动设置源文件属性，以根据需要支持 Windows Forms、WPF/XAML 或其他技术。

  • ExternalData 模块现在支持为同一个数据文件使用不同的哈希值（例如 img.png.sha256 和 img.png.sha1）来提供多个内容链接。这允许从由不同哈希算法索引的源获取对象。

  • ExternalProject 模块增加了 GIT_PROGRESS 选项，用于强制 Git 在克隆存储库时显示进度。

  • ExternalProject 模块增加了 GIT_CONFIG 选项，以便在克隆存储库时向 Git 传递 --config 选项。

  • FeatureSummary 模块的 feature_summary() 命令现在接受一个新的 QUIET_ON_EMPTY 选项，该选项可在属于所选类别的包列表为空时禁止输出。

  • FeatureSummary 模块的 add_feature_info() 命令现在接受依赖列表，用于决定特性是否启用。

  • FeatureSummary 模块接受的包类型现在可以通过更改 FeatureSummary_PKG_TYPES、FeatureSummary_REQUIRED_PKG_TYPES 和 FeatureSummary_DEFAULT_PKG_TYPE 全局属性来进行调整。

  • FindOpenGL 模块现在在找到相关库时提供导入的目标 OpenGL::GL 和 OpenGL::GLU。

  • UseSWIG 模块增加了 swig_add_library 命令，相比旧的 swig_add_module 命令提供了更大的灵活性。

  • UseSWIG 模块的 swig_add_source_to_module 命令增加了新的 SWIG_OUTFILE_DIR 选项，用于控制输出文件位置（swig -o）。

  • WriteCompilerDetectionHeader 模块增加了 ALLOW_UNKNOWN_COMPILERS 和 ALLOW_UNKNOWN_COMPILER_VERSIONS 选项，允许创建即使在未知或旧编译器上也能正常工作的头文件（通过直接假设它们不支持任何所请求的特性）。

CTest¶

  • ctest_memcheck() 命令增加了 DEFECT_COUNT <var> 选项，用于捕获检测到的内存缺陷数量。

  • ctest_memcheck() 命令现在可以从 CTEST_MEMORYCHECK_SUPPRESSIONS_FILE 变量中读取 sanitizers 的抑制文件位置。

  • ctest_memcheck() 命令现在支持独立于 AddressSanitizer 使用 LeakSanitizer。

  • ctest_update() 命令的 CDASH_UPLOAD 签名现在会尊重 RETRY_COUNT、RETRY_DELAY 和 QUIET 选项。

CPack¶

  • CPackIFWConfigureFile 模块已添加，定义了一个新的 cpack_ifw_configure_file() 命令，用于配置以 QtIFW/SDK/Creator 风格准备的文件模板。

  • CPackIFW 模块的 cpack_ifw_configure_component() 和 cpack_ifw_configure_component_group() 命令增加了新的 DEFAULT、VIRTUAL、FORCED_INSTALLATION、REQUIRES_ADMIN_RIGHTS、DISPLAY_NAME、UPDATE_TEXT、DESCRIPTION、RELEASE_DATE、AUTO_DEPEND_ON 和 TRANSLATIONS 选项，以实现更具体的配置。

  • CPackIFW 模块的 cpack_ifw_configure_component() 命令为 DEPENDS 选项增加了一个新的 DEPENDENCIES 别名。

  • CPackIFW 模块的 cpack_ifw_configure_component_group() 命令增加了 DEPENDS 选项。同时也添加了 DEPENDENCIES 别名。

  • CPackIFW 模块的 cpack_ifw_configure_component() 和 cpack_ifw_configure_component_group() 命令的 PRIORITY 选项现已弃用，并将于未来版本的 CMake 中移除。请改用新的 SORTING_PRIORITY 选项。

  • CPack IFW 生成器增加了新的 CPACK_IFW_PACKAGE_WATERMARK、CPACK_IFW_PACKAGE_BANNER、CPACK_IFW_PACKAGE_BACKGROUND、CPACK_IFW_PACKAGE_WIZARD_STYLE、CPACK_IFW_PACKAGE_WIZARD_DEFAULT_WIDTH、CPACK_IFW_PACKAGE_WIZARD_DEFAULT_HEIGHT 和 CPACK_IFW_PACKAGE_TITLE_COLOR 变量，用于定制 QtIFW 安装程序的外观。

  • CPack productbuild 生成器增加了对包进行签名的选项。请参见变量 CPACK_PRODUCTBUILD_IDENTITY_NAME、CPACK_PRODUCTBUILD_KEYCHAIN_PATH、CPACK_PKGBUILD_IDENTITY_NAME 和 CPACK_PKGBUILD_KEYCHAIN_PATH。

  • CPack RPM 生成器现在可以省略所提供的 rpmbuild 工具不支持的标记。如果设置了不支持的标记，它们将被忽略，并会打印开发者警告。

  • CPack RPM 生成器现在可以生成主要组件包，该包强制为定义的组件生成一个 rpm，且文件名和包名中不带组件后缀。请参见 CPACK_RPM_MAIN_COMPONENT 变量。

  • CPack RPM 生成器即使在使用组件打包时，也能根据需求生成单个 debuginfo 包。请参见 CPACK_RPM_DEBUGINFO_SINGLE_PACKAGE 变量。

  • CPack RPM 生成器在使用 CPACK_RPM_USER_FILELIST 变量时，支持每个文件使用多个指令。

其他¶

  • 使用加密哈希的 CMake 功能现在支持 SHA-3 算法。

  • 添加了一个新的生成器表达式 $<IF:cond,true-value,false-value>。如果条件为 1，它将解析为 true-value；如果条件为 0，它将解析为 false-value。

已弃用和已删除的特性¶

  • FeatureSummary 模块命令 set_package_info()、set_feature_info()、print_enabled_features() 和 print_disabled_features() 现已弃用。

  • UseSWIG 模块的 swig_add_module 命令现已弃用，建议使用 swig_add_library。

其他更改¶

  • 如果 <LANG>_CLANG_TIDY 目标属性指定的命令在构建时返回非零值，现在这将被视为错误，而不是静默忽略。

  • ctest_memcheck() 命令不再自动为 AddressSanitizer 使用的选项添加 leak_check=1。除非显式设置 leak_check=0，否则 AddressSanitizer 的默认行为是运行 LeakSanitizer 来检查泄漏。

  • ctest_memcheck() 命令已修复，现在可以正确地将从 CTEST_MEMORYCHECK_SANITIZER_OPTIONS 变量读取的额外 sanitizer 选项追加到 sanitizers 内部使用的环境变量中。

  • FeatureSummary 模块的 set_package_properties() 命令在未显式设置类型时，不再强制将包类型设置为 OPTIONAL。

  • 编译特性功能现在可以识别 UNIX 和 Windows 平台上 Intel C++ 编译器版本 12.1 到 17.0 所支持的特性。

  • 在成功调用 FindPkgConfig 模块的 pkg_check_modules() 命令后，如果该前缀的命令参数发生更改，现在会重新评估该前缀的缓存值。

  • 当使用 AUTOMOC 或 AUTOUIC 时，生成的 moc_*、*.moc 和 ui_* 文件会被放置在 <CMAKE_CURRENT_BINARY_DIR>/<TARGETNAME>_autogen/include 目录中，该目录会自动添加到目标的 INCLUDE_DIRECTORIES 中。因此，无需再将 CMAKE_CURRENT_BINARY_DIR 包含在目标的 INCLUDE_DIRECTORIES 中。

  • Sublime Text 2 生成器不再在启用详细构建输出的情况下运行本机构建命令（如 ninja 或 make）。

  • try_compile() 命令的源文件签名现在会在生成的测试项目中尊重 CMAKE_WARN_DEPRECATED 变量的值。

  • VS 2010 及以上版本的 Visual Studio 生成器 现在将每个源文件的标志置于目标级标志之后（当它们被分类为原始标志且无项目文件设置（AdditionalOptions）时）。此行为与其他生成器产生的标志顺序更加一致，并允许更具体的属性（每个源文件）上的标志覆盖更通用的（每个目标）属性上的标志。

  • cmake.org 提供的预编译 Windows 二进制 MSI 包现在会在 Windows 注册表的 HKLM\Software\Kitware\CMake 键下记录安装目录，其值名为 InstallDir。
