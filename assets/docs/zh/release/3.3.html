自 CMake 3.2 以来的变更如下。

新特性¶

生成器 (Generators)¶

  • Makefile 生成器现在向包含磁盘文件实际构建规则的 Makefile 中添加了 .DELETE_ON_ERROR。这会告知 GNU make 在规则执行中修改输出但失败时，删除对应的输出文件。

  • Visual Studio 生成器现在支持 .xaml 源文件，并能将它们自动与对应的 .h 和 .cpp 源文件关联。

  • Windows 平台新增了一个实验性的 Green Hills MULTI 生成器。Green Hills MULTI 是一款用于嵌入式实时系统的 IDE。

命令 (Commands)¶

  • add_dependencies() 命令现在允许将依赖项添加到 接口库。添加到接口库的依赖项会以传递方式被追踪，因为目标本身并不进行构建。

  • execute_process() 命令现在支持为 OUTPUT_FILE 和 ERROR_FILE 指定同一个文件。

  • file(GLOB) 和 file(GLOB_RECURSE) 命令新增了 LIST_DIRECTORIES <bool> 选项，用于指定 glob 结果中是否应包含目录。

  • find_library()、find_path() 和 find_file() 命令现在会从 PATH 环境变量派生的安装前缀中进行搜索。

  • if() 命令新增了一个 IN_LIST 运算符，如果给定的元素包含在命名列表中，则计算结果为 true。

  • install(EXPORT) 和 export() 命令现在可以导出填充了 INTERFACE_SOURCES 目标属性的目标。

  • install(TARGETS) 命令现在支持在 DESTINATION 值中使用生成器表达式。

变量 (Variables)¶

  • 部分 Fortran 编译器的版本现在可以被检测并存储在 CMAKE_Fortran_COMPILER_VERSION 变量中。

  • Visual Studio 生成器新增了 CMAKE_VS_INCLUDE_INSTALL_TO_DEFAULT_BUILD 选项，可将 INSTALL 目标放入解决方案（.sln）文件的默认构建中。

属性 (Properties)¶

  • 引入了 CROSSCOMPILING_EMULATOR 目标属性和对应的 CMAKE_CROSSCOMPILING_EMULATOR 变量，允许在交叉编译时于宿主机上运行目标平台二进制文件。

  • 引入了 <LANG>_INCLUDE_WHAT_YOU_USE 目标属性和对应的 CMAKE_<LANG>_INCLUDE_WHAT_YOU_USE 变量，用于告知 Makefile 生成器和 Ninja 生成器在编译 C 和 CXX 语言时，调用 include-what-you-use 工具。

  • <LANG>_VISIBILITY_PRESET 和 VISIBILITY_INLINES_HIDDEN 目标属性现在会影响所有目标类型源文件的编译。参见策略 CMP0063。

  • XCODE_ATTRIBUTE_<an-attribute> 目标属性现在支持生成器表达式。

模块 (Modules)¶

  • 引入了 CheckFortranCompilerFlag 模块，用于检查 Fortran 编译器标志，类似于 CheckCCompilerFlag 模块对 C 语言的作用。

  • ExternalData 模块新增了 ExternalData_NO_SYMLINKS 选项，用于禁用符号链接并改用文件拷贝来填充实际数据文件。

  • ExternalData 模块在 DATA{} 引用中增加了 RECURSE: 选项以指定目录。这允许匹配关联文件的整个目录树。

  • ExternalData 模块新增了 URL 模板占位符 %(algo:<key>)，通过配置新的 ExternalData_URL_ALGO_<algo>_<key> 变量，允许自定义算法名称到 URL 组件的映射。这为远程 URL 提供了更大的灵活性。

  • ExternalProject 模块现在可以在每个步骤的 BYPRODUCTS 中替换诸如 <BINARY_DIR> 之类的标记。

  • ExternalProject 模块 API 在使用 LOG_* 选项以及 CMake 初始缓存选项时，现在支持 生成器表达式。

  • FindBoost 模块现在分别为 RELEASE 和 DEBUG 配置跟踪包含库的目录。

  • FindCUDA 模块现在默认优先使用静态 CUDA 运行时库（如果可用）。提供了一个新的 CUDA_USE_STATIC_CUDA_RUNTIME 选项来控制此行为。

  • FindMatlab 模块已完全重写。它现在能够识别版本和组件，并以更精确、跨平台的方式查找 Matlab。该模块现在提供了用于创建 mex 扩展、文档和单元测试的 API。

  • FindPackageHandleStandardArgs 模块的 find_package_handle_standard_args() 命令现在总是同时填充 <PackageName>_FOUND 和 <UPPERCASE_NAME>_FOUND 变量（后者是为了向后兼容）。FOUND_VAR 选项现已被忽略，仅用于强制检查其允许的值。

  • InstallRequiredSystemLibraries 模块新增了 CMAKE_INSTALL_SYSTEM_RUNTIME_COMPONENT 选项，用于指定安装组件。

生成器表达式¶

  • 引入了新的 COMPILE_LANGUAGE 生成器表达式，允许根据每个源文件的 LANGUAGE 为目标文件指定编译选项。由于底层原生构建工具的限制，此功能在不同生成器间的支持程度不同。详见 cmake-generator-expressions(7) 手册。

CTest¶

  • ctest(1) 工具新增了 --repeat-until-fail <n> 选项，有助于发现偶发性的测试失败。

  • CTestCoverageCollectGCOV 模块现在支持与 ctest_coverage() 命令相同的 CTEST_CUSTOM_COVERAGE_EXCLUDE 选项。

CPack¶

  • CPack IFW 生成器现在支持 Qt Framework Installer 2.0 工具。

  • CPack DEB 生成器新增了 CPACK_DEBIAN_<COMPONENT>_PACKAGE_SHLIBDEPS 变量，用于指定分组件使用 dpkg-shlibdeps。

  • CPack DEB 生成器新增了 CPACK_DEBIAN_<COMPONENT>_PACKAGE_DEPENDS 选项，用于指定分组件依赖项。

  • CPack RPM 生成器现在能够更整洁地打包符号链接，并支持新版 rpmbuild 的目录符号链接。

  • CPack RPM 生成器新增了 CPACK_RPM_ADDITIONAL_MAN_DIRS 变量，用于指定包含 brp-compress RPM 宏所需帮助手册页的目录。

  • CPack RPM 生成器新增了 CPACK_RPM_<component>_PACKAGE_ARCHITECTURE 变量，用于指定组件特定的包架构。

  • CPack WIX 生成器新增了 CPACK_START_MENU_SHORTCUTS、CPACK_DESKTOP_SHORTCUTS 和 CPACK_STARTUP_SHORTCUTS 安装文件属性，可分别用于在“开始”菜单、桌面和启动文件夹中安装快捷方式。

其他 (Other)¶

  • 编译特性功能现在支持 Windows 上的 GNU 编译器 4.4 至 5.0 版本。

  • cmake(1) 的 -E tar 命令新增了 --format<format> 选项，用于指定要写入的归档格式。

  • 在 OS X 上，CMake 现在可以在 Xcode 中创建用于测试框架和 App Bundles 的 XCTest 包。FindXCTest 模块提供了方便的函数来处理 XCTEST 包。

已弃用和移除的功能 (Deprecated and Removed Features)¶

  • 在 OS X 上，cmake-gui(1) 不再提供“安装以供命令行使用”菜单项。取而代之的是一个“如何安装以供命令行使用”菜单项，它会弹出一个信息对话框，说明如何使命令行工具可用。例如：

    /Applications/CMake.app/Contents/bin/cmake-gui --install

  • ctest_build() 和 build_command() 命令不再通过 -i 选项告知 make 工具忽略错误。以前这是针对 Makefile 生成器做的，而非其他生成器。参见策略 CMP0061。

  • Visual Studio 10 2010 生成器不再检查 VS IDE 是否已运行并打开了项目，也不会要求它们重新加载。这最初是为 VS 10 做的（之前 VS 7 到 9 也这样做），以避免对解决方案中的每个项目都进行提示。由于 VS >= 10 允许整个解决方案一次性重新加载，因此它们不再需要 CMake 的帮助。

  • Visual Studio 7 生成器（.NET 2002）现已弃用，并将在 CMake 的未来版本中移除。

  • Visual Studio 6 生成器现已弃用，并将在 CMake 的未来版本中移除。

  • find_package() 命令不再考虑最近在 cmake-gui(1) 中配置的项目构建树。这以前只在 Windows 上执行，现在则完全取消。NO_CMAKE_BUILDS_PATH 选项如果被提供，现在会被忽略，且功能上相当于始终处于开启状态。项目可以填充 用户包注册表，以辅助用户依次构建多个依赖项目。

  • add_definitions() 命令不再导致 DEFINITIONS 目录属性被填充。参见策略 CMP0059。

  • 在使用 Visual Studio 7、8 和 9 生成器时，$(OutDir) 占位符的值不再解析为配置名称。项目应改用 $(ConfigurationName) 来实现该目的。

  • 不再允许将 export() 的输出与 install(FILES) 命令一起使用。详情参见策略 CMP0062。

其他更改 (Other Changes)¶

  • Ninja 生成器现在要求对 add_custom_command() 和 add_custom_target() 命令的调用必须使用 BYPRODUCTS 选项，以明确指定由自定义命令生成但未列为输出的文件（例如因为允许其时间戳早于输入）。参见策略 CMP0058。

  • Makefile 生成器的构建进度输出得到了改进。它在并行构建时不再混淆进度信息和构建规则消息。链接规则消息现在包含进度条，并显示为粗绿色而不是粗红色（因为红色通常与错误消息相关联）。

  • 对于 Visual Studio 7、8 和 9，CMAKE_CFG_INTDIR 变量的值现在是 $(ConfigurationName)，而不是 $(OutDir)。这对变量的预期用例不应产生影响。

  • 通过隐式链接器搜索目录中的完整路径链接库文件（例如 /usr/lib/libfoo.a）时，不再要求链接器搜索该库（例如 -lfoo），而是直接通过完整路径进行链接。参见策略 CMP0060。
