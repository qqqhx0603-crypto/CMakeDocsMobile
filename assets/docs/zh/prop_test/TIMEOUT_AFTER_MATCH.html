TIMEOUT_AFTER_MATCH¶

3.6 版本新增。

在测试的输出中遇到匹配行后，更改测试的超时时长。

用法¶

add_test(mytest ...)
set_property(TEST mytest PROPERTY TIMEOUT_AFTER_MATCH "${seconds}" "${regex}")

代码说明

  • add_test(mytest ...)：调用 add_test() 命令，登记一个测试用例。

  • set_property(TEST mytest PROPERTY TIMEOUT_AFTER_MATCH "${seconds}" "${regex}")：调用 set_property() 命令，设置目标、目录、源文件等对象的属性；变量说明：seconds：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；regex：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

描述¶

允许测试在输出中遇到 regex 后完成 seconds 秒。

当测试输出与 regex 匹配的行时，其开始时间将重置为当前时间，并且其超时时长将更改为 seconds。在此之前，超时时长由 TIMEOUT 属性或 CTEST_TEST_TIMEOUT 变量（如果已设置）确定。 由于测试的开始时间被重置，其执行时间将不包括花费在等待匹配输出上的任何时间。

TIMEOUT_AFTER_MATCH 对于避免虚假超时非常有用，尤其是在测试必须等待某个系统资源可用后才能执行时。 将 TIMEOUT 设置为更长的持续时间，以考虑资源获取和使用，并使用 TIMEOUT_AFTER_MATCH 来控制允许实际测试运行多长时间。

如果 CTest 可以控制所需的资源，则应使用 RESOURCE_LOCK 而不是 TIMEOUT_AFTER_MATCH。 仅当测试本身才能确定其所需资源何时可用时，才应使用此属性。

另请参阅 TIMEOUT_SIGNAL_NAME。
