SKIP_RETURN_CODE¶

标记测试为跳过的返回码。

有时只有测试本身才能确定是否满足测试的所有要求。如果这种情况不应被视为硬性失败，则可以指定进程的返回码，如果遇到该返回码，则将测试标记为 未 运 行。有效值范围为 0 到 255（含）。

超过由 TIMEOUT 指定的超时的测试，无论 SKIP_RETURN_CODE 如何，仍然会失败。包括段错误、信号中止或堆错误在内的系统级测试失败，即使返回码匹配，也可能导致测试失败。

# cmake (1) defines this to return code 1
add_test(NAME r1 COMMAND ${CMAKE_COMMAND} -E false)

set_tests_properties(r1 PROPERTIES SKIP_RETURN_CODE 1)

要运行可能存在系统级故障但仍应在 SKIP_RETURN_CODE 匹配时跳过的测试，请使用 CMake 命令来包装可执行文件的运行。请注意，这将阻止自动处理 CROSSCOMPILING_EMULATOR 和 TEST_LAUNCHER 目标属性。

add_executable(main main.c)

# cmake -E env <command> returns 1 if the command fails in any way
add_test(NAME sigabrt COMMAND ${CMAKE_COMMAND} -E env $<TARGET_FILE:main>)

set_property(TEST sigabrt PROPERTY SKIP_RETURN_CODE 1)

#include <signal.h>

int main(void){ raise(SIGABRT); return 0; }

要处理可能需要跳过的多种情况，请考虑 SKIP_REGULAR_EXPRESSION 属性。
