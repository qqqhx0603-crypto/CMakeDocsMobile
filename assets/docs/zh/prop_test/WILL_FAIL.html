WILL_FAIL¶

如果为 true，则反转通过/失败的测试标准。对于 WILL_FAIL 为 true 的测试，返回码为 0 时失败，而非零返回码时通过。 即使设置了 WILL_FAIL，超过由 TIMEOUT 指定的超时时间的测试仍然会失败。 即使 WILL_FAIL 为 true，系统级测试失败（包括段错误、信号中止或堆错误）也可能导致测试失败。

一个原本会通过的测试示例，但由于 WILL_FAIL 为 true 而失败

add_test(NAME failed COMMAND ${CMAKE_COMMAND} -E true)
set_property(TEST failed PROPERTY WILL_FAIL true)

要运行可能存在系统级错误的测试，但如果设置了 WILL_FAIL 仍然可以通过，请使用 CMake 命令来包装可执行文件的运行。 请注意，这将阻止自动处理 CROSSCOMPILING_EMULATOR 和 TEST_LAUNCHER 目标属性。

add_executable(main main.c)

add_test(NAME sigabrt COMMAND ${CMAKE_COMMAND} -E env $<TARGET_FILE:main>)

set_property(TEST sigabrt PROPERTY WILL_FAIL TRUE)

#include <signal.h>

int main(void){ raise(SIGABRT); return 0; }
