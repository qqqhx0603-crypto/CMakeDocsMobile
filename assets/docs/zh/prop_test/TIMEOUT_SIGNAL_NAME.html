TIMEOUT_SIGNAL_NAME¶

在 3.27 版本中新增。

指定在测试进程超时时发送给它的自定义信号。这仅在支持 POSIX 信号的平台上可用。在 Windows（微软桌面系统） 上不可用。

名称必须是以下之一：

SIGINT

中断。

SIGQUIT

退出。

SIGTERM

终止。

SIGUSR1

用户自定义信号 1。

SIGUSR2

用户自定义信号 2。

自定义信号被发送到测试进程，以便在宽限期内使其有机会优雅地退出

  • 如果测试进程创建了任何子进程，它有责任也终止它们。

  • 宽限期长度由 TIMEOUT_SIGNAL_GRACE_PERIOD 测试属性确定。

  • 如果测试进程在宽限期结束前没有终止，ctest(1) 将通过 SIGSTOP 和 SIGKILL 强制终止其整个进程树。

另请参阅 CTEST_TEST_TIMEOUT、TIMEOUT 和 TIMEOUT_AFTER_MATCH。
