SKIP_REGULAR_EXPRESSION¶

3.16 版新增。

如果测试输出（stderr 或 stdout）与此正则表达式匹配，则无论进程退出代码如何，该测试都将被标记为跳过。 即使超过由 TIMEOUT 指定的超时时间，测试仍然会失败，而与 SKIP_REGULAR_EXPRESSION 无关。 即使正则表达式匹配，系统级测试失败（包括段错误、信号中止或堆错误）也可能导致测试失败。

示例

add_test(NAME mytest COMMAND ${CMAKE_COMMAND} -E echo "Skipped this test")

set_property(TEST mytest PROPERTY
  SKIP_REGULAR_EXPRESSION "[^a-z]Skip" "SKIP" "Skipped"
)

SKIP_REGULAR_EXPRESSION 期望一个正则表达式列表。

要运行可能存在系统级错误的测试，但如果 SKIP_REGULAR_EXPRESSION 匹配则仍然跳过，请使用 CMake 命令来包装可执行文件的运行。 请注意，这将阻止自动处理 CROSSCOMPILING_EMULATOR 和 TEST_LAUNCHER 目标属性。

add_executable(main main.c)

add_test(NAME sigabrt COMMAND ${CMAKE_COMMAND} -E env $<TARGET_FILE:main>)

set_property(TEST sigabrt PROPERTY SKIP_REGULAR_EXPRESSION "SIGABRT;[aA]bort")

#include <signal.h>

int main(void){ raise(SIGABRT); return 0; }

另请参阅 SKIP_RETURN_CODE、PASS_REGULAR_EXPRESSION 和 FAIL_REGULAR_EXPRESSION 测试属性。
