CMAKE_CXX_KNOWN_FEATURES¶

版本 3.1 中新增。

此版本的 CMake 所知的 C++ 特性列表。

此全局属性中列出的特性可能为 C++ 编译器所支持。如果该特性在 C++ 编译器中可用，它将被列在 CMAKE_CXX_COMPILE_FEATURES 变量中。

此处列出的功能可与 target_compile_features() 命令一起使用。有关编译功能和支持的编译器列表的信息，请参阅 cmake-compile-features(7) 手册。

此版本的 CMake 所知的特性列出如下。

指示 C++ 标准支持的高级元特性¶

版本 3.8 新增。

以下元特性指示了对相关语言标准的通用支持。它反映了编译器声称的语言支持情况，但这并不一定意味着完全符合该标准。

cxx_std_98

编译器模式至少为 C++ 98。

cxx_std_11

编译器模式至少为 C++ 11。

cxx_std_14

编译器模式至少为 C++ 14。

cxx_std_17

编译器模式至少为 C++ 17。

cxx_std_20

3.12 版本新增。

编译器模式至少为 C++ 20。

cxx_std_23

在 3.20 版本中添加。

编译器模式至少为 C++ 23。

cxx_std_26

3.30 版本新增。

编译器模式至少为 C++ 26。

注意

如果编译器的默认标准级别至少等于所请求功能的标准级别，CMake 可能会省略 -std= 标志。如果编译器的默认扩展模式与 <LANG>_EXTENSIONS 目标属性不匹配，或者 <LANG>_STANDARD 目标属性已设置，则仍可能添加该标志。

低级独立编译特性¶

对于 C++ 11 和 C++ 14，编译器实现某些语言特性的速度有时较慢。CMake 提供了一些独立的编译特性，以帮助项目确定特定特性是否可用。这些独立特性现在的重要性已降低，项目通常应优先使用高级元特性。C++ 17 或更高版本不再提供独立编译特性。

有关使用独立编译特性的更多讨论，请参阅 cmake-compile-features(7) 手册。
