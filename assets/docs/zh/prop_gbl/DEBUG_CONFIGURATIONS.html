DEBUG_CONFIGURATIONS¶

此属性指定哪些 构建配置 用于调试。

该值必须是 分号分隔的列表，包含构建配置名称。目前，此属性仅由 target_link_libraries() 命令使用。未来可能会定义其他用途。

此属性必须在项目顶层设置，并且在第一个 target_link_libraries() 命令调用之前设置。如果列表中的任何条目与项目的有效配置不匹配，则行为未定义。

默认情况下，此属性未设置。

示例¶

以下示例在保留任何现有配置的同时，为非优化的调试配置添加自定义配置。如果项目使用默认 Debug 配置，也应将其包含在内。

set_property(GLOBAL APPEND PROPERTY DEBUG_CONFIGURATIONS Debug CustomBuild)

代码说明

  • set_property(GLOBAL APPEND PROPERTY DEBUG_CONFIGURATIONS Debug CustomBuild)：调用 `set_property()` 命令，设置目标、目录、源文件等对象的属性。

另请参阅¶

  • MAP_IMPORTED_CONFIG_<CONFIG> 目标属性，它在链接到具有 IMPORTED_CONFIGURATIONS 属性设置的 导入目标 时，映射构建配置。
