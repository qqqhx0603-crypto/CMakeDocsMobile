简介¶

CMake 是一款用于管理源代码构建过程的工具。最初，CMake 被设计为各种 Makefile 方言的生成器，如今 CMake 可以生成现代构建系统（如 Ninja），以及用于 IDE（如 Visual Studio（集成开发环境） 和 Xcode）的项目文件。

CMake 被广泛用于 C 和 C++ 语言，但它也可以用于构建其他语言的源代码。

初次接触 CMake 的用户可能有不同的初始目标。若要学习如何构建从互联网下载的源代码包，请从 用户交互指南 (User Interaction Guide) 开始。该指南将详细介绍运行 cmake(1) 或 cmake-gui(1) 可执行程序所需的步骤，以及如何选择生成器并完成构建。

使用依赖项指南 (Using Dependencies Guide) 旨在帮助希望开始使用第三方库的开发人员。

对于开始使用 CMake 进行项目的开发者，CMake 教程 (CMake Tutorial) 是一个合适的起点。cmake-buildsystem(7) 手册旨在帮助开发者扩展有关维护构建系统的知识，并熟悉 CMake 中可呈现的构建目标。cmake-packages(7) 手册则解释了如何创建可供第三方基于 CMake 的构建系统轻松调用的软件包。

命令行工具¶

  • cmake(1)：CMake 命令行工具：配置、生成、构建和安装项目。
  • ctest(1)：CTest 命令行工具：运行测试并提交测试结果。
  • cpack(1)：CPack 命令行工具：生成安装包和归档包。

交互式对话框¶

  • cmake-gui(1)：CMake 图形界面：交互设置缓存变量并生成构建系统。
  • ccmake(1)：CMake 终端界面：在字符界面中交互配置项目。

参考手册¶

  • cmake-buildsystem(7)：CMake 构建系统：目标、属性、传递用法要求及构建规范。
  • cmake-commands(7)：CMake 命令索引：脚本中可调用的全部内置命令。
  • cmake-compile-features(7)：编译特性：声明项目所需的 C、C++ 编译器特性。
  • cmake-configure-log(7)：配置日志：记录配置阶段的系统检查事件。
  • cmake-cxxmodules(7)：C++ 模块：CMake 对 C++20 模块的扫描与构建支持。
  • cmake-developer(7)：开发者手册：编写和维护 CMake 模块及上游代码的规则。
  • cmake-env-variables(7)：环境变量索引：影响 CMake、CTest 和 CPack 行为的环境变量。
  • cmake-file-api(7)：文件 API：供 IDE 和工具读取构建系统语义信息。
  • cmake-generator-expressions(7)：生成器表达式：在生成和构建阶段按上下文计算值。
  • cmake-generators(7)：生成器索引：选择 Ninja、Makefile 或 IDE 构建系统。
  • cmake-instrumentation(7)：插桩接口：采集 CMake 命令执行和构建过程数据。
  • cmake-language(7)：CMake 语言：语法、变量、列表、控制结构和作用域。
  • cmake-modules(7)：CMake 模块索引：查找依赖和提供辅助功能的模块。
  • cmake-packages(7)：软件包：查找、导出和使用 CMake 包配置。
  • cmake-policies(7)：策略索引：管理不同 CMake 版本之间的行为兼容性。
  • cmake-presets(7)：预设：用 JSON 文件保存配置、构建、测试和打包参数。
  • cmake-properties(7)：属性索引：全局、目录、目标、源文件和测试属性。
  • cmake-qt(7)：Qt 支持：自动处理 Qt 工具、资源、界面和元对象代码。
  • cmake-server(7)：旧服务器模式：已移除接口及其迁移说明。
  • cmake-toolchains(7)：工具链：编译器、链接器、交叉编译和平台配置。
  • cmake-variables(7)：变量索引：CMake 提供或识别的全部变量。
  • cpack-generators(7)：CPack 生成器索引：各类安装包和归档格式。
