CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION¶

3.4 版本新增。

Visual Studio（集成开发环境） Windows（微软桌面系统） 目标平台版本。

当目标平台为 Windows（微软桌面系统） 10 及更高版本时，VS 2015 及更高版本的 Visual Studio（集成开发环境） 生成器 支持指定 Windows（微软桌面系统） SDK 版本

  • 如果 CMAKE_GENERATOR_PLATFORM 指定了 version= 字段，如 Visual Studio（集成开发环境） 平台选择 中所述，则会选择该 SDK 版本。

  • 否则，如果 Windows（微软桌面系统）SDKVersion 环境变量设置为可用的 SDK 版本，则会选择该版本。这适用于由 vcvarsall.bat 或类似脚本建立的环境。

    3.27 版本新增：这由策略 CMP0149 启用。

  • 否则，如果 CMAKE_SYSTEM_VERSION 设置为可用的 SDK 版本，则会选择该版本。

    3.27 版本更改：这由策略 CMP0149 禁用。

  • 否则，CMake 将使用可用的最新 Windows（微软桌面系统） SDK 版本。

所选的 Windows（微软桌面系统） 目标版本号在 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION 中提供。如果不可用 Windows（微软桌面系统） 10 SDK，则此值将为空。

可以设置 CMAKE_WINDOWS_KITS_10_DIR 环境变量 为指向 Windows（微软桌面系统） 10 SDK 的绝对路径，以告知 CMake 在自定义位置查找 Windows（微软桌面系统） 10 SDK。指定的目录预计包含 Include/10.0.* 目录。

另请参阅 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION_MAXIMUM。
