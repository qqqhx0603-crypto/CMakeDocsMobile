CMAKE_CFG_INTDIR¶

变量说明

CMAKE_CFG_INTDIR：自 3.21 版本弃用： 此变量对 Ninja Multi-Config 的支持较差，并且早于 $<CONFIG> 生成器表达式的出现。请使用 $<CONFIG> 代替。

自 3.21 版本弃用： 此变量对 Ninja Multi-Config 的支持较差，并且早于 $<CONFIG> 生成器表达式的出现。请使用 $<CONFIG> 代替。

构建时引用每个配置的输出子目录。

对于支持在构建树中进行多个配置的本机构建系统（例如 Visual Studio（集成开发环境） 生成器 和 Xcode），其值是对构建时变量的引用，该变量指定每个配置的输出子目录的名称。在 Makefile（Make 构建脚本） 生成器 上，它评估为 .，因为构建树中只有一个配置。示例值

$(Configuration)

Visual Studio（集成开发环境）

$(CONFIGURATION)

Xcode

.

基于 Makefile（Make 构建脚本） 的工具

.

Ninja（构建工具）

${CONFIGURATION}

Ninja（构建工具） 多配置

由于这些值由本机构建系统评估，因此此变量仅适用于将在构建时评估的命令行。预期用途示例

add_executable(mytool mytool.c)
add_custom_command(
  OUTPUT out.txt
  COMMAND ${CMAKE_CURRENT_BINARY_DIR}/${CMAKE_CFG_INTDIR}/mytool
          ${CMAKE_CURRENT_SOURCE_DIR}/in.txt out.txt
  DEPENDS mytool in.txt
  )
add_custom_target(drive ALL DEPENDS out.txt)

代码说明

  • add_executable(mytool mytool.c)：调用 add_executable() 命令，定义一个可执行目标。

  • add_custom_command(：这一行继续补充当前命令或表达式的内容。

  • OUTPUT out.txt：这一行继续补充当前命令或表达式的内容。

  • COMMAND ${CMAKE_CURRENT_BINARY_DIR}/${CMAKE_CFG_INTDIR}/mytool：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_BINARY_DIR：当前源目录所对应的构建目录；CMAKE_CFG_INTDIR：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；条件运算符：COMMAND 表示判断给定名称是否为可调用命令。

  • ${CMAKE_CURRENT_SOURCE_DIR}/in.txt out.txt：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • DEPENDS mytool in.txt：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

  • add_custom_target(drive ALL DEPENDS out.txt)：调用 add_custom_target() 命令，定义一个没有直接输出文件的自定义目标。

请注意，CMAKE_CFG_INTDIR 不再需要此目的，但为了与现有项目兼容而保留。相反，add_custom_command() 在其 COMMAND 选项中识别可执行目标名称，因此 ${CMAKE_CURRENT_BINARY_DIR}/${CMAKE_CFG_INTDIR}/mytool 可以简单地替换为 mytool。

此变量是只读的。设置它会导致未定义行为。在多配置构建系统中，此变量的值作为预处理器符号 CMAKE_INTDIR 的值传递给所有源文件的编译。
