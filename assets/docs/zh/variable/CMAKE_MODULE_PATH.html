CMAKE_MODULE_PATH¶

变量说明

CMAKE_MODULE_PATH：分号分隔的列表，使用正斜杠表示，指定 CMake 模块的搜索路径，这些模块将被 include() 或 find_package() 命令在检查随 CMake 提供的默认模块之前加载。默认情况下它是空的。它旨在由项目设置。

分号分隔的列表，使用正斜杠表示，指定 CMake 模块的搜索路径，这些模块将被 include() 或 find_package() 命令在检查随 CMake 提供的默认模块之前加载。默认情况下它是空的。它旨在由项目设置。

一个项目通常会有一个目录，其中包含各种 *.cmake 文件来辅助开发。将该目录添加到 CMAKE_MODULE_PATH 可以简化加载它们。例如，一个项目的顶级 CMakeLists.txt 文件可能包含

list(APPEND CMAKE_MODULE_PATH "${CMAKE_CURRENT_SOURCE_DIR}/cmake")

include(Foo) # Loads ${CMAKE_CURRENT_SOURCE_DIR}/cmake/Foo.cmake

find_package(Bar) # Loads ${CMAKE_CURRENT_SOURCE_DIR}/cmake/FindBar.cmake

代码说明

  • list(APPEND CMAKE_MODULE_PATH "${CMAKE_CURRENT_SOURCE_DIR}/cmake")：调用 list() 命令，对列表变量执行追加、删除、查询等操作；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • include(Foo) # Loads ${CMAKE_CURRENT_SOURCE_DIR}/cmake/Foo.cmake：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • find_package(Bar) # Loads ${CMAKE_CURRENT_SOURCE_DIR}/cmake/FindBar.cmake：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。
