CMAKE_ANDROID_STL_TYPE¶

变量说明

CMAKE_ANDROID_STL_TYPE：在使用 使用 NVIDIA Nsight Tegra（NVIDIA Tegra 调试扩展） Visual Studio（集成开发环境） Edition（NVIDIA Tegra 的 Visual Studio（集成开发环境） 扩展） 进行 Android 交叉编译 时，可以设置此变量来指定 ANDROID_STL_TYPE 目标属性的默认值。有关更多信息，请参阅该目标属性。

3.4 版本新增。

在使用 使用 NVIDIA Nsight Tegra（NVIDIA Tegra 调试扩展） Visual Studio（集成开发环境） Edition（NVIDIA Tegra 的 Visual Studio（集成开发环境） 扩展） 进行 Android 交叉编译 时，可以设置此变量来指定 ANDROID_STL_TYPE 目标属性的默认值。有关更多信息，请参阅该目标属性。

在使用 使用 NDK 进行 Android 交叉编译 时，可以设置此变量来指定要使用的 STL 变体。该值可以是以下之一：

none

无 C++ 支持

system

无 STL 的最小 C++

gabi++_static

GAbi++ 静态

gabi++_shared

GAbi++ 共享

gnustl_static

GNU libstdc++ 静态

gnustl_shared

GNU libstdc++ 共享

c++_static

LLVM libc++ 静态

c++_shared

LLVM libc++ 共享

stlport_static

STLport 静态

stlport_shared

STLport 共享

默认值为在提供它的 NDK 版本上为 gnustl_static，否则为 c++_static。请注意，此默认值与原生 NDK 构建系统不同，因为 CMake 可用于构建并非原生为其实现的 Android 项目，并使用 C++ 标准库。
