CMAKE_CROSSCOMPILING¶

CMake 会设置此变量以指示是否正在交叉编译，但请注意下面讨论的限制。

如果手动设置了 CMAKE_SYSTEM_NAME 变量（即在工具链文件或从 cmake 命令行中作为缓存条目设置），CMake 将此变量设置为 true。在大多数情况下，只有在交叉编译时才会手动设置 CMAKE_SYSTEM_NAME，因为如果未手动设置，它将获得与 CMAKE_HOST_SYSTEM_NAME 相同的值，这对于非交叉编译的情况是正确的。如果将 CMAKE_SYSTEM_NAME 手动设置为与 CMAKE_HOST_SYSTEM_NAME 相同的值，则 CMAKE_CROSSCOMPILING 仍然会设置为 true。

需要注意的另一个情况是，针对 macOS 以外的 Apple 平台的构建与其他交叉编译场景的处理方式不同。Apple 设备构建不依赖于 CMAKE_SYSTEM_NAME 来选择目标平台，而是使用 CMAKE_OSX_SYSROOT 来选择适当的 SDK，这间接确定了目标平台。此外，在使用 Xcode 生成器时，开发人员可以在构建时在设备和模拟器构建之间切换，而不是在配置时进行单一选择，因此构建是交叉编译还是不是的的概念更加复杂。因此，不建议在针对 Apple 设备的工程中使用 CMAKE_CROSSCOMPILING。
