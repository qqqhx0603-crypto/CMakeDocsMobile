CMAKE_OSX_SYSROOT¶

指定要使用的 macOS 平台 SDK 的位置或名称。

如果未显式设置，则该值将由环境变量 SDKROOT 初始化（如果已设置）。否则，该值默认为空，编译器预计将自行选择默认的 macOS SDK。

版本 4.0 中更改：默认值现在为空。 之前，默认值是根据 CMAKE_OSX_DEPLOYMENT_TARGET 或主机平台计算得出的。

为了通过编译器的 -isysroot 标志传递显式的 macOS SDK，用户可以使用 -DCMAKE_OSX_SYSROOT=macosx 配置他们的构建树，或者在他们的环境中执行 export SDKROOT="$(xcrun --sdk macosx --show-sdk-path)"。

备注

  • 在 /usr/bin 中的 macOS 编译器，在没有使用 -isysroot 调用时，会在系统 SDK 路径之前搜索 /usr/local/include 中的头文件，这符合许多平台上的约定。 在安装了 Homebrew 的 macOS-x86_64 主机上的用户，如果未使用 Homebrew 工具构建，应如上所述传递显式 SDK。

  • 某些 Clang 编译器没有默认的 macOS SDK 选择。 对于这些编译器，如果 CMAKE_OSX_SYSROOT 为空，CMake 将自动使用 xcrun --show-sdk-path 打印的 macOS SDK 传递 -isysroot。

此变量的值应在第一个 project() 或 enable_language() 命令调用之前设置，因为它可能会影响工具链和标志的配置。 它旨在由创建构建树的用户本地设置。 此变量应设置为 CACHE 条目（否则 CMake 可能会在初始化同名缓存条目时将其删除），除非策略 CMP0126 设置为 NEW。

尽管变量名中包含 OSX，但它们也适用于 macOS 以外的其他 SDK，例如 iOS、tvOS、visionOS 或 watchOS。

此变量在非 Apple 平台上被忽略。
