CMAKE_NINJA_OUTPUT_PATH_PREFIX¶

3.6 版本新增。

告知 Ninja（构建工具） 生成器 在 build.ninja 中的每个输出路径前添加一个前缀。如果前缀缺少尾部斜杠，则会附加一个斜杠。

当生成的 ninja 文件旨在作为 subninja 文件嵌入到 *超级* ninja 项目中时，这很有用。例如，命令

cd super-build-dir &&
cmake -G Ninja -S /path/to/src -B sub -DCMAKE_NINJA_OUTPUT_PATH_PREFIX=sub/
#                                 ^^^---------- these match -----------^^^

代码说明

  • cd super-build-dir && cmake -G Ninja -S /path/to/src -B sub -DCMAKE_NINJA_OUTPUT_PATH_PREFIX=sub/ # ^^^---------- these match -----------^^^：这是一条命令行输入，切换当前工作目录。

生成一个构建目录，其顶级目录 (CMAKE_BINARY_DIR) 位于 super-build-dir/sub 中。构建目录的路径以输出路径前缀结尾。这使其适合在单独编写的 super-build-dir/build.ninja 文件中使用如下指令

subninja sub/build.ninja

代码说明

  • subninja sub/build.ninja：这一行给出示例文本或输出，用来说明：subninja sub/build.ninja

在 super-build-dir/build.ninja 中的 auto-regeneration 规则必须对 sub/build.ninja 具有仅顺序依赖关系。

3.27 版本新增：Ninja（构建工具） Multi-Config 生成器支持此变量。

注意

当设置 CMAKE_NINJA_OUTPUT_PATH_PREFIX 时，CMake 生成的项目不能用作独立项目。未指定任何默认目标。

CMAKE_NINJA_OUTPUT_PATH_PREFIX 的值必须匹配 CMAKE_BINARY_DIR 末尾的一个或多个路径组件，否则行为未定义。但是，此要求不会自动检查。
