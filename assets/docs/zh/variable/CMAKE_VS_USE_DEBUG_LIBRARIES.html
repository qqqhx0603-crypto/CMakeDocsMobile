CMAKE_VS_USE_DEBUG_LIBRARIES¶

3.30 版本新增。

向 Visual Studio 生成器 指示哪些配置被视为调试（Debug）配置。这会控制 .vcxproj 文件中每个配置下的 UseDebugLibraries 设置。

Visual Studio 项目中的“使用调试库（Use Debug Libraries）”设置，尽管名称听起来很具体，但它实际上是用于标识哪些配置属于调试配置的通用指标。在独立项目中，这可能会影响 MSBuild 对 MSVC 运行时库、优化标志、运行时检查及类似设置的默认选择。在 CMake 项目中，这些设置通常是根据项目规范显式生成的，例如，MSVC 运行时库由 CMAKE_MSVC_RUNTIME_LIBRARY 控制。不过，UseDebugLibraries 指标对于人类和工具的参考仍然很有用，并且也可能影响特定平台 SDK 的行为。

将 CMAKE_VS_USE_DEBUG_LIBRARIES 设置为 true 或 false 值，以指示每个配置是否被视为调试配置。该值也可以为空字符串（""），在这种情况下，CMake 不会显式添加 UseDebugLibraries，而 MSBuild 将使用其默认值 false。

使用 生成器表达式 进行逐配置规范。例如，以下代码

set(CMAKE_VS_USE_DEBUG_LIBRARIES "$<CONFIG:Debug,Custom>")

表明所有后续目标都将它们的“Debug”和“Custom”配置视为调试配置，而其他配置则视为非调试配置。

该变量用于在创建所有目标时初始化其 VS_USE_DEBUG_LIBRARIES 属性。它还会通过调用 try_compile() 命令传播到其测试项目中。

如果未设置此变量，则不会自动设置 VS_USE_DEBUG_LIBRARIES 属性。如果该属性未设置，CMake 将使用启发式方法来确定哪些配置是调试配置。请参阅策略 CMP0162。
