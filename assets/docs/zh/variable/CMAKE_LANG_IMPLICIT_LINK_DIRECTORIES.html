CMAKE_<LANG>_IMPLICIT_LINK_DIRECTORIES¶

检测到的语言 <LANG> 的隐式链接器搜索路径。

编译器在调用链接器时，通常会传递包含语言运行时库的目录以及默认的库搜索路径。这些路径就是该编译器所用语言的隐式链接器搜索目录。

对于通过 project() 或 enable_language() 命令启用的每种语言，CMake 都会自动检测这些目录并将结果存入此变量。可以通过设置环境变量 CMAKE_<LANG>_IMPLICIT_LINK_DIRECTORIES_EXCLUDE 来从自动检测的结果中排除特定目录。

当链接到静态库时，CMake 会为静态库中使用的每种语言（不包括用于驱动链接的编译器所对应的语言）添加此变量中的隐式链接目录。对于导入的静态库，IMPORTED_LINK_INTERFACE_LANGUAGES 目标属性列出了需要其隐式链接信息的语言。如果其中任何语言未被启用，项目可以自行提供其 CMAKE_<LANG>_IMPLICIT_LINK_DIRECTORIES 变量的值。或者，工具链文件 可以将该变量设置为特定工具链已知的值。当语言启用时，该值要么会被覆盖，要么会被用作备用值。

某些工具链会从 LIBRARY_PATH 等环境变量中读取隐式目录。如果使用了此类环境变量，请在给定的构建树中操作时保持其值一致，因为 CMake 会在首次创建构建树时保存检测到的值。

在 CMake 4.0 之前的版本中，如果策略 CMP0060 未设置为 NEW，那么当通过完整路径向 target_link_libraries() 提供这些目录中的库时，出于历史原因，CMake 会在链接行上生成 -l<name> 形式的命令。

另请参阅 CMAKE_<LANG>_IMPLICIT_LINK_LIBRARIES 变量。
