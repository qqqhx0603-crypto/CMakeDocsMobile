CMAKE_Swift_COMPILATION_MODE¶

在版本 3.29 中添加。

指定 Swift 编译目标的方式。此变量用于初始化在创建目标时，目标上的 Swift_COMPILATION_MODE 属性。

允许的值为

incremental

将模块中的每个 Swift 源代码文件单独编译，从而在构建过程中获得更好的并行性。编译器会将额外信息输出到构建目录，从而在对源代码进行小幅更改后重新构建时提高重建性能。这是在项目更改期间迭代的最佳选项。

wholemodule

整个模块优化编译速度最慢，但会生成最优化过的库。整个上下文加载到编译器的单个实例中，因此模块中的源代码文件之间没有并行性。

singlefile

将 Swift 模块中的每个源代码文件单独编译，从而获得更好的并行性。与 incremental 构建模式不同，编译器在构建过程中不会输出任何额外信息，因此在对源代码文件进行小幅更改后重新构建将不会更快。除非是为了解决编译器错误，否则应谨慎使用此选项，优先使用 incremental 构建。

使用 生成器表达式 来支持按配置指定。例如，以下代码

set(CMAKE_Swift_COMPILATION_MODE
  "$<IF:$<CONFIG:Release>,wholemodule,incremental>")

在构建发布配置时，将默认 Swift 编译模式设置为 wholemodule 模式，在其他配置中设置为 incremental 模式。

如果未设置此变量，则 Swift_COMPILATION_MODE 目标属性将不会自动设置。如果该属性未设置，则 CMake 将使用默认值 incremental 来构建 Swift 源代码文件。

注意

只有当策略 CMP0157 在第一个 project() 或 enable_language() 命令（该命令启用 Swift 语言）之前设置为 NEW 时，此属性才有效。
