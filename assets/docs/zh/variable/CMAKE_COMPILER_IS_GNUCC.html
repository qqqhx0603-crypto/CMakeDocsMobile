CMAKE_COMPILER_IS_GNUCC¶

自版本 3.24 起已弃用：请使用 CMAKE_C_COMPILER_ID 变量代替。

在 CMake 2.6 中引入 CMAKE_<LANG>_COMPILER_ID 变量之前，在早期 CMake 版本中使用了 CMAKE_COMPILER_IS_* 变量。

如果 C 编译器是 GNU，则将 CMAKE_COMPILER_IS_GNUCC 变量设置为布尔值 true。

示例¶

在 CMake 的早期版本中，CMAKE_COMPILER_IS_GNUCC 变量用于检查 C 编译器是否为 GNU

if(CMAKE_COMPILER_IS_GNUCC)
  # GNU C compiler-specific logic.
endif()

代码说明

  • if(CMAKE_COMPILER_IS_GNUCC) # GNU C compiler-specific logic. endif()：开始一个 `if()` 条件判断，根据条件结果选择是否执行后续语句。

从 CMake 2.6 开始，应使用 CMAKE_C_COMPILER_ID 变量代替

if(CMAKE_C_COMPILER_ID STREQUAL "GNU")
  # GNU C compiler-specific logic.
endif()

代码说明

  • if(CMAKE_C_COMPILER_ID STREQUAL "GNU") # GNU C compiler-specific logic. endif()：开始一个 `if()` 条件判断，根据条件结果选择是否执行后续语句。

升级代码时，请考虑是否应该将除了 GNU 之外的其他 C 编译器作为目标。 过去，LCC 和 QCC 编译器也设置了此变量（有关详细信息，请参阅 CMP0047 和 CMP0129 策略）。 为了考虑到这一点，可以使用以下方法

# Targeting GNU, LCC, and QCC compilers for C:
if(CMAKE_C_COMPILER_ID MATCHES "^(GNU|LCC|QCC)$")
  # ...
endif()

代码说明

  • # Targeting GNU, LCC, and QCC compilers for C: if(CMAKE_C_COMPILER_ID MATCHES "^(GNU|LCC|QCC)$") # ... endif()：这是注释，用来说明：Targeting GNU, LCC, and QCC compilers for C: if(CMAKE_C_COMPILER_ID MATCHES "^(GNU|LCC|QCC)$") # ... endif()。
