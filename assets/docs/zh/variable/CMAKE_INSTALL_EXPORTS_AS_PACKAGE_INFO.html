CMAKE_INSTALL_EXPORTS_AS_PACKAGE_INFO¶

Added in version 4.3.

注意

此变量仅在通过 CMAKE_EXPERIMENTAL_FIXME 门控启用实验性支持时才具有意义。

这是一个指令列表，用于指示 CMake 在通过 install(EXPORT) 安装导出的目标信息时，同时安装“通用包规范”（Common Package Specification）包信息。该值被视为一个列表，每个指令的格式为

<export-name>:<package-name>[/[l][a<appendix-name>][/<destination>]]

斜杠用于分隔指令的不同组件。

请注意，此功能旨在供包分发者使用，应仅在无法编辑项目的 CMake 脚本时使用。开发人员应直接使用 install(PACKAGE_INFO)。

此外，由于 CMAKE_INSTALL_EXPORTS_AS_PACKAGE_INFO 的工作原理是模拟对 install(PACKAGE_INFO) 的调用，因此在已经直接调用 install(PACKAGE_INFO) 的项目中使用它可能会导致安装指令冲突，这通常会导致项目的配置阶段失败。

各项含义如下：

<export-name>

需要安装包信息的导出名称。

<package-name>

要生成包信息的包名称。这也是用户在 find_package() 调用中使用的名称。

l

可选。指定磁盘上的包信息文件名应为小写。请参阅 install(PACKAGE_INFO) 的 LOWER_CASE_FILE 选项。

a<appendix-name>

可选。指定应创建一个附录 <appendix-name> 而不是根包描述。请参阅 install(PACKAGE_INFO) 的 APPENDIX 选项。请注意，不能向附录中添加额外信息（见下文）。

<destination>

可选。指定安装包信息文件的目标路径。请参阅 install(PACKAGE_INFO) 的 DESTINATION 选项。请注意，默认值是大多数情况下适用于通用包规范文件的平台特定位置，而不是指令所匹配的 install(EXPORT) 的 DESTINATION。

对于非附录，CMake 还将从多个 <export-name>_EXPORT_PACKAGE_INFO_<var> 格式的 CMake 变量中推断额外信息。这些变量的值首先会通过带 @ONLY 选项的 string(CONFIGURE) 进行处理。这些变量是可选的，其效果等同于将它们的值传递给 install(PACKAGE_INFO) 命令的 <var> 选项。

额外的变量包括：

  • <export-name>_EXPORT_PACKAGE_INFO_VERSION

  • <export-name>_EXPORT_PACKAGE_INFO_COMPAT_VERSION

  • <export-name>_EXPORT_PACKAGE_INFO_VERSION_SCHEMA

  • <export-name>_EXPORT_PACKAGE_INFO_LICENSE

  • <export-name>_EXPORT_PACKAGE_INFO_DEFAULT_LICENSE

  • <export-name>_EXPORT_PACKAGE_INFO_DEFAULT_CONFIGURATIONS

理想情况下，版本号应设置为 @PROJECT_VERSION@。但是，某些项目可能不使用 project() 命令的 VERSION 选项。

示例¶

考虑以下（简化）项目

project(Example VERSION 1.2.3 SPDX_LICENSE "BSD-3-Clause")

add_library(foo ...)
add_library(bar ...)

install(TARGETS foo EXPORT required-targets)
install(TARGETS bar EXPORT optional-targets)

install(EXPORT required-targets FILE example-targets.cmake ...)
install(EXPORT optional-targets FILE example-optional-targets.cmake ...)

在此示例中，假设 example-targets.cmake 是始终安装的文件，而 example-optional-targets.cmake 是可选安装的文件（例如作为依赖于“基础”包的单独包的一部分进行分发）。

现在，假设我们是一位分发者，想要向用户提供“通用包规范”包信息文件，但我们不想修改项目的构建文件。我们可以通过将以下参数传递给 CMake 来实现这一点：

-DCMAKE_EXPERIMENTAL_FIXME=<elided>
-DCMAKE_INSTALL_EXPORTS_AS_PACKAGE_INFO=\
  required-targets:Example/l;\
  optional-targets:Example/laoptional
-Drequired-targets_EXPORT_PACKAGE_INFO_VERSION=@Example_VERSION@
-Drequired-targets_EXPORT_PACKAGE_INFO_LICENSE=@Example_SPDX_LICENSE@

（为了可读性而添加的空格和续行符，在实际使用时应删除。参数可能需要加引号以防止被命令行 shell 重新解析。）

这将导致 CMake 同时创建并安装描述 Example 包的 example.cps 和 example-optional.cps 文件。如果项目没有通过 project() 命令提供此信息，我们还可以使用包构建系统（例如 rpm、dpkg）提供的替换项来指定版本和许可证信息。
