CMAKE_POLICY_WARNING_CMP<NNNN>¶

变量说明

CMAKE_POLICY_WARNING_CMP<NNNN>：当 CMake 策略 CMP<NNNN> 未通过 cmake_policy() 显式设置，也未通过 cmake_minimum_required() 隐式设置时，显式启用或禁用相应警告。这仅对默认情况下不发出警告的策略有意义。

当 CMake 策略 CMP<NNNN> 未通过 cmake_policy() 显式设置，也未通过 cmake_minimum_required() 隐式设置时，显式启用或禁用相应警告。这仅对默认情况下不发出警告的策略有意义。

  • CMAKE_POLICY_WARNING_CMP0025 控制 CMake 4.0 版本之前策略 CMP0025 的警告。

  • CMAKE_POLICY_WARNING_CMP0047 控制 CMake 4.0 版本之前策略 CMP0047 的警告。

  • CMAKE_POLICY_WARNING_CMP0056 控制 CMake 4.0 版本之前策略 CMP0056 的警告。

  • CMAKE_POLICY_WARNING_CMP0060 控制 CMake 4.0 版本之前策略 CMP0060 的警告。

  • CMAKE_POLICY_WARNING_CMP0065 控制 CMake 4.0 版本之前策略 CMP0065 的警告。

  • CMAKE_POLICY_WARNING_CMP0066 控制策略 CMP0066 的警告。

  • CMAKE_POLICY_WARNING_CMP0067 控制策略 CMP0067 的警告。

  • CMAKE_POLICY_WARNING_CMP0082 控制策略 CMP0082 的警告。

  • CMAKE_POLICY_WARNING_CMP0089 控制策略 CMP0089 的警告。

  • CMAKE_POLICY_WARNING_CMP0102 控制策略 CMP0102 的警告。

  • CMAKE_POLICY_WARNING_CMP0112 控制策略 CMP0112 的警告。

  • CMAKE_POLICY_WARNING_CMP0116 控制策略 CMP0116 的警告。

  • CMAKE_POLICY_WARNING_CMP0126 控制策略 CMP0126 的警告。

  • CMAKE_POLICY_WARNING_CMP0128 控制策略 CMP0128 的警告。

  • CMAKE_POLICY_WARNING_CMP0129 控制策略 CMP0129 的警告。

  • CMAKE_POLICY_WARNING_CMP0133 控制策略 CMP0133 的警告。

  • CMAKE_POLICY_WARNING_CMP0172 控制策略 CMP0172 的警告。

  • CMAKE_POLICY_WARNING_CMP0206 控制策略 CMP0206 的警告。

项目不应在 CMake 代码中设置此变量。运行 CMake 的项目开发人员可以在其缓存中设置此变量以启用警告（例如 -DCMAKE_POLICY_WARNING_CMP<NNNN>=ON）。或者，在运行 cmake(1) 时使用 --debug-output、--trace 或 --trace-expand 选项也会启用该警告。
