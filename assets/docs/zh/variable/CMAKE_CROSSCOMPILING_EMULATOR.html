CMAKE_CROSSCOMPILING_EMULATOR¶

3.3 版本中新增。

只有当 CMAKE_CROSSCOMPILING 开启时，此变量才会被使用。它应该指向主机系统上一个可以运行为目标系统构建的可执行文件的命令。

3.15 版本新增： 如果此变量包含一个 分号分隔的列表，那么第一个值是命令，其余值是它的参数。

3.28 版本新增： 可以通过 CMAKE_CROSSCOMPILING_EMULATOR 环境变量初始化此变量。

该命令将被用于运行 try_run() 生成的可执行文件，从而避免手动填充 TryRunResults.cmake 文件。

此变量也被用作可执行文件的 CROSSCOMPILING_EMULATOR 目标属性的默认值。但是，虽然 生成器 表达式 受目标属性支持（自 CMake 3.29 起），但它们不受此变量的 try_run() 功能支持。
