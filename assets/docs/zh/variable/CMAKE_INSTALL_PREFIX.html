CMAKE_INSTALL_PREFIX¶

install() 命令所使用的安装目录。

当执行 make install 或构建 INSTALL 目标时，该目录会被预置到所有安装目录路径之前。

此变量的默认值如下：

  • 3.29 版本新增：如果设置了 CMAKE_INSTALL_PREFIX 环境变量，其值将作为此变量的默认值。

  • Windows 上为 c:/Program Files/${PROJECT_NAME}。

  • UNIX 平台上为 /usr/local。

请参阅 CMAKE_INSTALL_PREFIX_INITIALIZED_TO_DEFAULT，了解项目如何选择自己的默认值。

在 UNIX 上，可以使用 DESTDIR 机制将整个安装过程重定位到暂存区。更多信息请参阅 DESTDIR 环境变量。

安装前缀也会被添加到 CMAKE_SYSTEM_PREFIX_PATH 中，以便 find_package()、find_program()、find_library()、find_path() 和 find_file() 在查找其他软件时会搜索该前缀。在首次调用 project() 之前，将 CMAKE_FIND_NO_INSTALL_PREFIX 设置为 TRUE 可以禁用此行为。

注意

使用 GNUInstallDirs 模块可提供符合 GNU 风格的安装目录布局选项。

在配置构建树时，可以定义 CMAKE_INSTALL_PREFIX 以设置其安装前缀。或者，在使用 cmake(1) 命令行工具的 --install 模式时，可以使用 --prefix 选项指定不同的前缀。

cmake --install . --prefix /my/install/prefix

注意

当项目使用 GNUInstallDirs 模块时，存在一些特殊情况，这些情况是在配置阶段基于 CMAKE_INSTALL_PREFIX 变量的值进行评估的。即使在安装期间使用了替代的 --prefix 选项，其结果依然有效。

另请参阅¶

  • --install-prefix 选项。

  • $<INSTALL_PREFIX> 生成器表达式。

  • CMake 配置预设中的 installDir 字段。

  • CPACK_PACKAGING_INSTALL_PREFIX 变量。
