CMAKE_GENERATOR_INSTANCE¶

变量说明

CMAKE_GENERATOR_INSTANCE：用户提供的生成器特定实例规范。

3.11 版本新增。

用户提供的生成器特定实例规范。

当存在多个原生构建系统实例时，某些 CMake 生成器支持选择特定的实例。如果用户指定了某个实例（例如通过设置此缓存条目，或通过 CMAKE_GENERATOR_INSTANCE 环境变量），或者在首次配置构建树时选择了默认实例后，该值将在此变量中可用。

此变量的值绝不应由项目代码修改。由 CMAKE_TOOLCHAIN_FILE 变量指定的工具链文件可以将 CMAKE_GENERATOR_INSTANCE 初始化为缓存条目。一旦给定的构建树使用该变量的特定值进行了初始化，修改该值将产生未定义的行为。

实例规范仅在特定生成器上受支持。

Visual Studio（集成开发环境） 实例选择¶

Visual Studio（集成开发环境） 生成器 支持对 Visual Studio 2017（微软集成开发环境）（微软集成开发环境） 及以上版本进行实例规范。可以将 CMAKE_GENERATOR_INSTANCE 变量设置为缓存条目，通过以下形式之一选择 Visual Studio（集成开发环境） 实例：

  • location

  • location[,key=value]*

  • key=value[,key=value]*

location 指定 VS 安装顶级目录的绝对路径。

key=value 对是逗号分隔的选项列表，用于指定实例选择的详细信息。支持的键值对为：

version=<major>.<minor>.<date>.<build>

在版本 3.23 中添加。

指定 4 分量 VS 构建版本，也称为构建编号 (Build Number)。

各分量含义如下：

<major>.<minor>

VS 主版本号和次版本号。这些与发布版本号相同。

<date>

格式为 MMMDD 的构建日期，其中 MMM 是自微软所用纪元以来的月份索引，DD 是当月的天数。

<build>

由 <date> 表示日期当天的构建索引。

该构建编号由 vswhere 以 installationVersion 的形式报告。例如，VS 16.11.10 的构建编号为 16.11.32126.315。

3.23 版本新增：可以通过同时提供 location 和 version= 来指定一个 Visual Studio（集成开发环境） 安装程序所未知的便携式 VS 实例。

如果用户或工具链文件未显式指定 CMAKE_GENERATOR_INSTANCE 的值，CMake 会查询 Visual Studio（集成开发环境） 安装程序以定位 VS 实例，选择一个，并将其设置为缓存条目以持久化保存该值。如果设置了 VS##0COMNTOOLS 格式的环境变量（其中 ## 是 Visual Studio（集成开发环境） 主版本号），且该变量指向某个 VS 实例内的 Common7/Tools 目录，则会使用该实例。否则，如果安装了多个 VS 实例，CMake 不会定义默认选择哪一个。

所选 VS 实例的 VS 版本构建号存储在 CMAKE_VS_VERSION_BUILD_NUMBER 变量中。
