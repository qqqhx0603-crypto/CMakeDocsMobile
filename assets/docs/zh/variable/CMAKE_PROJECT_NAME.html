CMAKE_PROJECT_NAME¶

顶级项目的名称。

此变量保存了在顶级 CMakeLists.txt 文件中由 project() 命令指定的项目名称。如果顶级 CMakeLists.txt 文件包含多个 project() 调用，则来自该顶级 CMakeLists.txt 的最近一次调用将决定 CMAKE_PROJECT_NAME 包含的名称。例如，考虑以下顶级 CMakeLists.txt

cmake_minimum_required(VERSION 3.0)
project(First)
project(Second)
add_subdirectory(sub)
project(Third)

代码说明

  • cmake_minimum_required(VERSION 3.0) project(First) project(Second) add_subdirectory(sub) project(Third)：调用 `cmake_minimum_required()` 命令，声明工程所需的最低 CMake 版本。

以及以下内容的 sub/CMakeLists.txt：

project(SubProj)
message("CMAKE_PROJECT_NAME = ${CMAKE_PROJECT_NAME}")

代码说明

  • project(SubProj) message("CMAKE_PROJECT_NAME = ${CMAKE_PROJECT_NAME}")：调用 `project()` 命令，定义项目名称及语言等元数据。

从顶级 CMakeLists.txt 中看到的最近一次 project() 命令将是 project(Second)，因此这将打印

CMAKE_PROJECT_NAME = Second

代码说明

  • CMAKE_PROJECT_NAME = Second：这一行把某个名称与给定值关联起来，常见于配置或键值示例。

要获取当前目录作用域或更高作用域中对 project() 的最近一次调用的名称，请参阅 PROJECT_NAME 变量。
