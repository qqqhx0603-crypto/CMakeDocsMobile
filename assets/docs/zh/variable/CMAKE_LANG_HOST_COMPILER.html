CMAKE_<LANG>_HOST_COMPILER¶

3.10 版本新增: CMAKE_CUDA_HOST_COMPILER

3.28 版本新增: CMAKE_HIP_HOST_COMPILER

当 <LANG> 为 CUDA 或 HIP 时，此变量可用。

当 CMAKE_<LANG>_COMPILER_ID 为 NVIDIA 时，CMAKE_<LANG>_HOST_COMPILER 用于指定编译 CUDA 或 HIP 语言文件时所使用的宿主（host）代码编译器可执行文件。这对应于 nvcc -ccbin 选项。

CMAKE_<LANG>_HOST_COMPILER 变量可以在通过 project() 或 enable_language() 命令首次启用 CUDA 或 HIP 之前进行显式设置。可以通过命令行参数 -DCMAKE_<LANG>_HOST_COMPILER=... 或在 工具链文件 中进行设置。此外，也可以设置 CUDAHOSTCXX 或 HIPHOSTCXX 环境变量来提供默认值。

一旦启用了 CUDA 或 HIP 语言，CMAKE_<LANG>_HOST_COMPILER 变量将变为只读，对其进行修改属于未定义行为。

注意

由于 CMAKE_<LANG>_HOST_COMPILER 仅在 CMAKE_<LANG>_COMPILER_ID 为 NVIDIA 时才有意义，因此在未将 CMAKE_<LANG>_COMPILER 设置为 NVCC 的情况下设置 CMAKE_<LANG>_HOST_COMPILER 是没有意义的。

注意

项目不应尝试自行设置 CMAKE_<LANG>_HOST_COMPILER 以匹配 CMAKE_CXX_COMPILER。确保 NVCC 与 C++ 编译器使用相同的 ABI 是最终用户的责任，而非项目的责任。

注意

在使用 Visual Studio 生成器 时将被忽略。

有关 nvcc 所使用的宿主编译器的信息（无论是默认值还是通过 CMAKE_<LANG>_HOST_COMPILER 指定的值），请参阅 CMAKE_<LANG>_HOST_COMPILER_ID 和 CMAKE_<LANG>_HOST_COMPILER_VERSION 变量。
