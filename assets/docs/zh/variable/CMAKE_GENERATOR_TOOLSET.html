CMAKE_GENERATOR_TOOLSET¶

用户提供的原生构建系统工具集规范。

某些 CMake 生成器支持工具集规范，以告知原生构建系统如何选择编译器。如果用户指定了工具集（例如通过 cmake -T 选项或通过 CMAKE_GENERATOR_TOOLSET 环境变量），则该值将在此变量中可用。

此变量的值绝不应由项目代码修改。由 CMAKE_TOOLCHAIN_FILE 变量指定的工具链文件可以初始化 CMAKE_GENERATOR_TOOLSET。一旦给定的构建树使用该变量的特定值进行了初始化，修改其值将导致未定义的行为。

工具集规范仅在特定的生成器上受支持

  • Visual Studio 生成器（适用于 VS 2010 及以上版本）

  • 适用于 Xcode 3.0 及以上版本的 Xcode 生成器

  • Green Hills MULTI 生成器

有关允许的工具集名称，请参阅原生构建系统的文档。

Visual Studio 工具集选择¶

Visual Studio 生成器支持使用以下形式之一进行工具集规范

  • toolset

  • toolset[,key=value]*

  • key=value[,key=value]*

toolset 指定工具集名称。所选工具集名称在 CMAKE_VS_PLATFORM_TOOLSET 变量中提供。

key=value 对形成一个逗号分隔的选项列表，用于指定工具集选择的生成器特定细节。支持的键值对包括

cuda=<version>|<path>

指定要使用的 CUDA 工具包版本或独立 CUDA 工具包目录的路径。VS 2010 及以上版本支持此项。版本号仅适用于已全局安装 CUDA 工具包 VS 集成的情况。请参阅 CMAKE_VS_PLATFORM_TOOLSET_CUDA 和 CMAKE_VS_PLATFORM_TOOLSET_CUDA_CUSTOM_DIR 变量。

fortran=<compiler>

在版本 3.29 中添加。

指定要使用的 Fortran 编译器，前提是已安装所需的 Visual Studio 集成功能。该值可以是以下之一

ifort

Intel 经典 Fortran 编译器。

ifx

Intel oneAPI Fortran 编译器。

请参阅 CMAKE_VS_PLATFORM_TOOLSET_FORTRAN 变量。

host=<arch>

指定宿主工具架构为 x64 或 x86。VS 2013 及以上版本支持此项。请参阅 CMAKE_VS_PLATFORM_TOOLSET_HOST_ARCHITECTURE 变量。

version=<version>

指定要使用的工具集版本。VS 2017 及以上版本在安装了指定工具集的情况下支持此项。请参阅 CMAKE_VS_PLATFORM_TOOLSET_VERSION 变量。

VCTargetsPath=<path>

为 Visual Studio 项目文件指定替代的 VCTargetsPath 值。这允许使用未随 VS 一起安装的 VS 平台扩展配置文件（.props 和 .targets）。

Visual Studio 工具集定制¶

这些是不稳定的接口，没有兼容性保证，因为它们钩入了未文档化的内部 CMake 实现细节。机构可以使用这些接口来内部维护对非公开 Visual Studio 平台和工具集的支持，但必须承担在 CMake 发生变更时进行更新的责任。

此外还提供以下 key=value 对

customFlagTableDir=<path>

3.21 版本新增。

指定一个目录的绝对路径，从中加载存储为 JSON 文档的自定义标志表。这些文件的命名格式为 <platform>_<toolset>_<tool>.json 或 <platform>_<tool>.json，其中 <platform> 为 CMAKE_VS_PLATFORM_NAME，<toolset> 为 CMAKE_VS_PLATFORM_TOOLSET，而 <tool> 是该标志表所针对的工具。此命名模式是 CMake 的内部实现细节。 <tool> 名称未文档化。.json 标志表文件的格式也未文档化。
