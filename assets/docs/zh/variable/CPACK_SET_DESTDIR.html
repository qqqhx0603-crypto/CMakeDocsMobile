CPACK_SET_DESTDIR¶

变量说明

CPACK_SET_DESTDIR：一个布尔开关，用于使 CPack 在打包时使用 DESTDIR 机制。

一个布尔开关，用于使 CPack 在打包时使用 DESTDIR 机制。

DESTDIR 代表目标目录 (DESTination DIRectory)。它通常被 Makefile（Make 构建脚本） 用户用来以非默认位置安装软件。这是一种基本的重定位机制，不应在 Windows（微软桌面系统） 上使用（请参阅 CMAKE_INSTALL_PREFIX 文档）。通常以如下方式调用

make DESTDIR=/home/john install

代码说明

  • make DESTDIR=/home/john install：这是一条命令行输入，调用 Make 构建工具；运算符：= 表示赋值或键值关联。

这将使用安装前缀（例如 /usr/local）安装相关的软件，该前缀将与 DESTDIR 值连接，最终得到 /home/john/usr/local。在准备软件包时，CPack 首先使用相同的 DESTDIR 机制将要打包的项目安装到本地（相对于构建树）目录中。然而，如果设置了 CPACK_SET_DESTDIR，CPack 将在执行本地安装之前设置 DESTDIR。最明显的区别是，如果没有设置 CPACK_SET_DESTDIR，CPack 将使用 CPACK_PACKAGING_INSTALL_PREFIX 作为前缀，而设置了 CPACK_SET_DESTDIR 后，CPack 将使用 CMAKE_INSTALL_PREFIX 作为前缀。

手动设置 CPACK_SET_DESTDIR 可能会有所帮助（或者只是必要的），如果某些安装规则使用绝对 DESTINATION（请参阅 CMake install() 命令）。但是，从 CPack/CMake 2.8.3 开始，RPM 和 DEB 安装程序尝试自动处理 DESTDIR，因此用户很少需要设置它。
