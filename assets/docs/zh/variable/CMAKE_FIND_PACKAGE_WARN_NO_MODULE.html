CMAKE_FIND_PACKAGE_WARN_NO_MODULE¶

变量说明

CMAKE_FIND_PACKAGE_WARN_NO_MODULE：告知 find_package() 在没有显式模式的情况下调用时发出警告。

告知 find_package() 在没有显式模式的情况下调用时发出警告。

如果 find_package() 在没有显式模式选项（MODULE、CONFIG 或 NO_MODULE）的情况下被调用，并且 CMAKE_MODULE_PATH 中没有 Find<pkg>.cmake 模块，那么 CMake 会隐式地假设调用者打算搜索包配置文件。如果未找到包配置文件，则失败消息的措辞必须同时考虑包确实缺失的情况以及项目存在错误且未能提供预期的 Find 模块的情况。如果调用者指定了显式模式选项，则失败消息可以更具体。

将 CMAKE_FIND_PACKAGE_WARN_NO_MODULE 设置为 TRUE，以告知 find_package() 在它隐式地假设 Config 模式时发出警告。这有助于开发人员强制在项目中的所有对 find_package() 的调用中使用显式模式。

如果 CMAKE_FIND_PACKAGE_PREFER_CONFIG 设置为 TRUE，则此变量无效。
