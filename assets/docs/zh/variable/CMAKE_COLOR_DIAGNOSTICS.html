CMAKE_COLOR_DIAGNOSTICS¶

在 3.24 版本中添加。

启用生成的构建系统中的彩色诊断信息。

此变量有三种状态：ON、OFF 和未定义。

当未定义时

  • Makefile 生成器 将 CMAKE_COLOR_MAKEFILE 变量初始化为 ON。它控制彩色构建系统消息。

  • GNU/Clang 编译器不会使用任何彩色诊断标志被调用。

当 ON 时

  • Makefile 生成器 默认生成彩色构建系统消息。 CMAKE_COLOR_MAKEFILE 未初始化，但可以显式设置为 OFF 以禁用彩色构建系统消息。

  • GNU/Clang 编译器将使用一个启用彩色诊断信息的标志被调用 (-fcolor-diagnostics)。

当 OFF 时

  • Makefile 生成器 默认不生成彩色构建系统消息。 CMAKE_COLOR_MAKEFILE 未初始化，但可以显式设置为 ON 以启用彩色构建系统消息。

  • GNU/Clang 编译器将使用一个禁用彩色诊断信息的标志被调用 (-fno-color-diagnostics)。

如果设置了 CMAKE_COLOR_DIAGNOSTICS 环境变量，则使用其值。否则，CMAKE_COLOR_DIAGNOSTICS 默认情况下未定义。

请参阅 CLICOLOR 和 CLICOLOR_FORCE 环境变量以控制 CMake 命令行工具的彩色输出。
