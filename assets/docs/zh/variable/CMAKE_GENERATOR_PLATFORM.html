CMAKE_GENERATOR_PLATFORM¶

版本 3.1 中新增。

由用户提供的特定于生成器的目标平台规范。

某些 CMake 生成器支持将目标平台名称提供给原生构建系统，以选择编译器工具链。如果用户指定了平台名称（例如通过 cmake -A 选项或通过 CMAKE_GENERATOR_PLATFORM 环境变量），则该值将在此变量中可用。

此变量的值不应由项目代码修改。由 CMAKE_TOOLCHAIN_FILE 变量指定的工具链文件可能会初始化 CMAKE_GENERATOR_PLATFORM。一旦给定的构建树已使用此变量的特定值进行初始化，更改该值将导致未定义的行为。

平台规范仅在特定的生成器上受支持

  • 对于 VS 2005 及更高版本的 Visual Studio（集成开发环境） 生成器，此变量指定目标架构。

  • 对于 Green Hills MULTI，此变量指定目标架构。

有关允许的平台名称，请参阅原生构建系统文档。

Visual Studio（集成开发环境） 平台选择¶

Visual Studio（集成开发环境） 生成器支持使用以下格式之一指定平台

  • platform

  • platform[,key=value]*

  • key=value[,key=value]*

platform 指定目标平台（VS 目标架构），例如 x64、ARM64 或 Win32。所选平台名称在 CMAKE_VS_PLATFORM_NAME 变量中提供。

key=value 对构成了以逗号分隔的选项列表，用于指定平台选择中特定于生成器的详细信息。支持的键值对包括

version=<version>

在 3.27 版本中新增。

指定要使用的 Windows（微软桌面系统） SDK 版本。VS 2015 及更高版本在针对 Windows（微软桌面系统） 或 Windows（微软桌面系统） Store 进行开发时支持此项。CMake 会将 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION 变量设置为所选的 SDK 版本。

<version> 可以是以下值之一

10.0

指定可以使用任何 10.0 SDK 版本，并由 Visual Studio（集成开发环境） 进行选择。VS 2019 及更高版本支持此项。

10.0.<build>.<increment>

指定精确的 4 部分 SDK 版本，例如 10.0.19041.0。必须已安装指定的 SDK 版本。如果设置了 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION_MAXIMUM 变量，则指定版本不得超过该变量的值。

8.1

指定 8.1 SDK 版本。VS 2015 始终支持此项。在 VS 2017 及更高版本上，必须安装 8.1 SDK。

如果未指定 version 字段，CMake 将按照 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION 变量文档中所述的方式选择版本。
