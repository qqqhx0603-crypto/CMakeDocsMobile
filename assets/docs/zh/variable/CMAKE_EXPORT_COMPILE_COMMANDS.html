CMAKE_EXPORT_COMPILE_COMMANDS¶

变量说明

CMAKE_EXPORT_COMPILE_COMMANDS：启用/禁用生成期间的编译命令输出。

3.5 版本新增。

启用/禁用生成期间的编译命令输出。

如果启用，将生成一个 compile_commands.json 文件，其中包含项目所有翻译单元的精确编译器调用，以机器可读的形式呈现。JSON 文件的格式如下

[
  {
    "directory": "/home/user/development/project",
    "command": "/usr/bin/c++ ... -c ../foo/foo.cc",
    "file": "../foo/foo.cc",
    "output": "../foo.dir/foo.cc.o"
  },

  ...

  {
    "directory": "/home/user/development/project",
    "command": "/usr/bin/c++ ... -c ../foo/bar.cc",
    "file": "../foo/bar.cc",
    "output": "../foo.dir/bar.cc.o"
  }
]

代码说明

  • [：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • {：这一行用于界定当前代码块、结构体或函数体的开始或结束。

  • "directory": "/home/user/development/project",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "command": "/usr/bin/c++ ... -c ../foo/foo.cc",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "file": "../foo/foo.cc",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "output": "../foo.dir/foo.cc.o"：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • },：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • ...：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • {：这一行用于界定当前代码块、结构体或函数体的开始或结束。

  • "directory": "/home/user/development/project",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "command": "/usr/bin/c++ ... -c ../foo/bar.cc",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "file": "../foo/bar.cc",：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • "output": "../foo.dir/bar.cc.o"：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • }：这一行用于界定当前代码块、结构体或函数体的开始或结束。

  • ]：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

该选项由 CMAKE_EXPORT_COMPILE_COMMANDS 环境变量初始化，并为所有目标初始化 EXPORT_COMPILE_COMMANDS 目标属性。

注意

此选项仅由 Makefile（Make 构建脚本） 生成器 和 Ninja（构建工具） 生成器 实现。在其他生成器上会被忽略。

此选项目前与 UNITY_BUILD 目标属性或 CMAKE_UNITY_BUILD 变量结合使用效果不佳。
