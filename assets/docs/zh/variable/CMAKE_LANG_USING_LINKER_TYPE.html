CMAKE_<LANG>_USING_LINKER_<TYPE>¶

在版本 3.29 中添加。

此变量定义了如何指定链接步骤的 <TYPE> 链接器，由 CMAKE_LINKER_TYPE 变量或 LINKER_TYPE 目标属性控制。根据 CMAKE_<LANG>_LINK_MODE 变量的值，CMAKE_<LANG>_USING_LINKER_<TYPE> 可以包含链接步骤的编译器标志，或者链接器工具的路径。

版本 4.0 中更改。

此变量中存储的信息类型现在由 CMAKE_<LANG>_LINK_MODE 变量确定，而不是由 CMAKE_<LANG>_USING_LINKER_MODE 变量确定。

注意

通常期望指定的链接器工具可以通过 PATH 环境变量访问。

例如，对于 GNU 编译器的 LLD 链接器定义如下

# CMAKE_C_LINK_MODE holds value "DRIVER"
set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld")

代码说明

  • # CMAKE_C_LINK_MODE holds value "DRIVER" set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld")：这是注释，用来说明：CMAKE_C_LINK_MODE holds value "DRIVER" set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld")。

在 Windows（微软桌面系统） 平台上，使用 Clang 编译器模拟 MSVC 并使用 GNU 前端

# CMAKE_C_LINK_MODE holds value "DRIVER"
set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld-link")

代码说明

  • # CMAKE_C_LINK_MODE holds value "DRIVER" set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld-link")：这是注释，用来说明：CMAKE_C_LINK_MODE holds value "DRIVER" set(CMAKE_C_USING_LINKER_LLD "-fuse-ld=lld-link")。

对于 MSVC 编译器或模拟 MSVC 的 Clang 编译器（使用 MSVC 前端），链接器将直接调用，不通过编译器前端

# CMAKE_C_LINK_MODE holds value "LINKER"
set(CMAKE_C_USING_LINKER_LLD "/path/to/lld-link.exe")

代码说明

  • # CMAKE_C_LINK_MODE holds value "LINKER" set(CMAKE_C_USING_LINKER_LLD "/path/to/lld-link.exe")：这是注释，用来说明：CMAKE_C_LINK_MODE holds value "LINKER" set(CMAKE_C_USING_LINKER_LLD "/path/to/lld-link.exe")。

也可以定义自定义链接器类型，通常在工具链文件中

set(CMAKE_LINKER_TYPE lld_launcher)
set(CMAKE_C_USING_LINKER_lld_launcher "-fuse-ld=/path/to/lld-launcher.sh")

代码说明

  • set(CMAKE_LINKER_TYPE lld_launcher) set(CMAKE_C_USING_LINKER_lld_launcher "-fuse-ld=/path/to/lld-launcher.sh")：调用 `set()` 命令，把变量 `CMAKE_LINKER_TYPE` 设为给定的值。
