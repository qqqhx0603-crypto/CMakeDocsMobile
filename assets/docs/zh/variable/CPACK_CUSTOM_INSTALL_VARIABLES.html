CPACK_CUSTOM_INSTALL_VARIABLES¶

3.21 版本新增。

CPack 变量（通过例如 cpack -D、CPackConfig.cmake 或 CPACK_PROJECT_CONFIG_FILE 脚本设置）不能直接在安装脚本中可见。相反，可以将一个 varName=value 对的列表传递到 CPACK_CUSTOM_INSTALL_VARIABLES 变量中。在安装时，列表中的每个条目都会导致一个指定名称 (varName) 的变量被设置为给定的 value。可以省略 = 以获得一个空的 value。

CPACK_CUSTOM_INSTALL_VARIABLES 允许在 CPack 运行时由用户或驱动脚本影响打包安装，而无需重新生成安装脚本。

示例¶

install(FILES large.txt DESTINATION data)

install(CODE [[
  if(ENABLE_COMPRESSION)
    # "run-compressor" is a fictional tool that produces
    # large.txt.xz from large.txt and then removes the input file
    execute_process(COMMAND run-compressor $ENV{DESTDIR}${CMAKE_INSTALL_PREFIX}/large.txt)
  endif()
]])

代码说明

  • install(FILES large.txt DESTINATION data) install(CODE [[ if(ENABLE_COMPRESSION) # "run-compressor" is a fictional tool that produces # large.txt.xz from large.txt and then removes the input file execute_process(COMMAND run-compressor $ENV{DESTDIR}${CMAKE_INSTALL_PREFIX}/large.txt) endif() ]])：调用 `install()` 命令，声明安装规则。

根据上面的示例片段，cpack 默认情况下将使用未设置 ENABLE_COMPRESSION 的安装脚本运行，从而生成包含未压缩的 large.txt 的软件包。可以通过以下方式在调用 cpack 时覆盖此设置

cpack -D "CPACK_CUSTOM_INSTALL_VARIABLES=ENABLE_COMPRESSION=TRUE"

代码说明

  • cpack -D "CPACK_CUSTOM_INSTALL_VARIABLES=ENABLE_COMPRESSION=TRUE"：这是一条命令行输入，执行打包流程。

然后安装脚本将使用将 ENABLE_COMPRESSION 设置为 TRUE 运行，从而生成包含压缩的 large.txt.xz 的软件包。
