CMAKE_LINKER_TYPE¶

在版本 3.29 中添加。

指定链接步骤中使用的链接器。

此变量用于初始化通过调用 add_library() 或 add_executable() 创建的每个目标上的 LINKER_TYPE 属性。它仅对具有链接步骤的目标有意义。如果已设置，其值也会被 try_compile() 命令使用。

注意

假设所指定的链接器与编译器通常调用的默认链接器完全兼容。CMake 不会执行任何选项转换。

链接器类型区分大小写，且只能包含字母、数字和下划线。全大写的链接器类型保留给 CMake 自身的内置类型。预定义的链接器类型为：

DEFAULT

此类型对应于标准链接，本质上等同于完全未设置 LINKER_TYPE 目标属性。

SYSTEM

使用平台或工具链提供的标准链接器。例如，这意味着对于所有兼容 MSVC 的编译器，都使用 Microsoft 链接器。此类型支持以下平台-编译器组合：

  • Linux：GNU, Clang, LLVMFlang, NVIDIA 和 Swift 编译器。

  • Apple 平台：AppleClang, Clang, GNU 和 Swift 编译器。

  • Windows（微软桌面系统）：MSVC, GNU, Clang, NVIDIA 和 Swift 编译器。

LLD

使用 LLVM 链接器。此类型支持以下平台-编译器组合：

  • Linux：GNU, Clang, LLVMFlang, NVIDIA 和 Swift 编译器。

  • Apple 平台：Clang, AppleClang 和 Swift 编译器。

  • Windows（微软桌面系统）：GNU, 带有 MSVC 风格前端的 Clang, 带有 GNU 风格前端的 Clang, MSVC, 带有 MSVC 风格前端的 NVIDIA 和 Swift。

BFD

使用 GNU 链接器。此类型支持以下平台-编译器组合：

  • Linux：GNU, Clang, LLVMFlang 和 NVIDIA 编译器。

  • Windows（微软桌面系统）：GNU, 带有 GNU 风格前端的 Clang。

GOLD

在带有 GNU, Clang, LLVMFlang, NVIDIA 和 Swift 编译器的 Linux 平台上受支持。

MOLD

使用 mold 链接器。此类型支持以下平台-编译器组合：

  • Linux：GNU, Clang, LLVMFlang 和 NVIDIA 编译器。

  • Apple 平台：Clang 和 AppleClang 编译器（用作 sold 链接器 的别名）。

SOLD

使用 sold 链接器。此类型仅在带有 Clang 和 AppleClang 编译器的 Apple 平台上受支持。

APPLE_CLASSIC

使用旧版行为的 Apple 链接器（即 Xcode 15.0 之前的版本）。此类型仅在带有 GNU, Clang, AppleClang 和 Swift 编译器的 Apple 平台上受支持。

MSVC

使用 Microsoft 链接器。此类型仅在带有 MSVC, 带有 MSVC 风格前端的 Clang 和 Swift 编译器的 Windows（微软桌面系统） 平台上受支持。
