CMAKE_MAXIMUM_RECURSION_DEPTH¶

变量说明

CMAKE_MAXIMUM_RECURSION_DEPTH：3.14 版新增。

3.14 版新增。

CMake 脚本的最大递归深度。 建议通过命令行使用 -DCMAKE_MAXIMUM_RECURSION_DEPTH=<x> 设置，或者在 CMakeLists.txt 中由需要较大递归深度的项目设置。 设置此变量的项目应提供让用户覆盖它的方法。 例如

# About to perform deeply recursive actions
if(NOT CMAKE_MAXIMUM_RECURSION_DEPTH)
  set(CMAKE_MAXIMUM_RECURSION_DEPTH 2000)
endif()

代码说明

  • # About to perform deeply recursive actions：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • if(NOT CMAKE_MAXIMUM_RECURSION_DEPTH)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值。

  • set(CMAKE_MAXIMUM_RECURSION_DEPTH 2000)：调用 set() 命令，把变量 CMAKE_MAXIMUM_RECURSION_DEPTH 设为给定的值；变量说明：CMAKE_MAXIMUM_RECURSION_DEPTH：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • endif()：这一行结束或切换上一段控制结构的分支。

如果未设置，或设置为非整数值，则会使用合理的默认限制。 如果达到递归限制，脚本将立即以致命错误终止。

调用以下任何命令都会增加递归深度

  • include()

  • find_package()

  • add_subdirectory()

  • try_compile()

  • ctest_read_custom_files()

  • ctest_run_script() (除非指定 NEW_PROCESS)

  • 用户定义的 function() 和 macro() (请注意，function() 和 macro() 本身不会增加递归深度)

  • 读取或写入被 variable_watch() 监视的变量

另请参阅 CMAKE_MAXIMUM_RECURSION_DEPTH 环境变量。
