CMAKE_LINK_LIBRARY_<FEATURE>_ATTRIBUTES¶

变量说明

CMAKE_LINK_LIBRARY_<FEATURE>_ATTRIBUTES：此变量定义了指定链接库 <FEATURE> 的行为。它规定了 <FEATURE> 如何与其他特性交互、何时应用该 <FEATURE>，以及在 CMake 生成最终链接器命令行时（例如去重）该如何处理 <FEATURE> 的相关方面。

3.30 版本新增。

此变量定义了指定链接库 <FEATURE> 的行为。它规定了 <FEATURE> 如何与其他特性交互、何时应用该 <FEATURE>，以及在 CMake 生成最终链接器命令行时（例如去重）该如何处理 <FEATURE> 的相关方面。

针对 <FEATURE> 的链接器标志语法由 CMAKE_<LANG>_LINK_LIBRARY_USING_<FEATURE> 和 CMAKE_LINK_LIBRARY_USING_<FEATURE> 变量控制。CMAKE_<LANG>_LINK_LIBRARY_USING_<FEATURE>_SUPPORTED 和 CMAKE_LINK_LIBRARY_USING_<FEATURE>_SUPPORTED 变量用于控制该 <FEATURE> 是否可用。

当针对特定语言 <LANG> 进行链接时，如果同一个 <FEATURE> 也定义了 CMAKE_<LANG>_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 变量，则 CMAKE_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 将被忽略。

在定义目标的目录作用域末尾，CMAKE_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 和 CMAKE_<LANG>_LINK_LIBRARY_<FEATURE>_ATTRIBUTES 的值才是生效的值。

特性属性定义¶

特性属性定义是一个分号分隔的列表，包含多个 attribute=value(s) 项。如果一个属性有多个值，这些值必须用逗号分隔。

支持以下属性：

LIBRARY_TYPE=<library-type-list>

指定该特性支持的库类型。支持的值有：STATIC、SHARED、MODULE 和 EXECUTABLE。

如果未指定此属性，默认值为 LIBRARY_TYPE=STATIC,SHARED,MODULE,EXECUTABLE。

如果该特性与不支持的库类型一起使用，CMake 将发出开发者警告并忽略该特性。

OVERRIDE=<feature-list>

指定发生冲突时该特性所替换的其他特性。如果定义了 LINK_LIBRARY_OVERRIDE 或 LINK_LIBRARY_OVERRIDE_<LIBRARY> 目标属性，此覆盖机制将被这些定义所取代。

如果未指定此属性，则默认为空列表。

DEDUPLICATION=YES|NO|DEFAULT

指定使用此特性的库的去重策略。

是

库总是会被去重。CMake 通常应用的默认策略将被忽略。

否

库永远不会被去重。CMake 通常应用的默认策略将被忽略。

DEFAULT

让 CMake 自动决定去重策略。

如果未指定此属性，将使用 DEFAULT。

示例¶

一个常见的需求是在创建共享库时加载完整的归档文件。内置的 WHOLE_ARCHIVE 特性可用于此目的。该内置特性的实现设置了以下链接库特性属性：

set(CMAKE_LINK_LIBRARY_WHOLE_ARCHIVE_ATTRIBUTES
  LIBRARY_TYPE=STATIC
  OVERRIDE=DEFAULT
  DEDUPLICATION=YES
)

  • set(CMAKE_LINK_LIBRARY_WHOLE_ARCHIVE_ATTRIBUTES：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_LINK_LIBRARY_WHOLE_ARCHIVE_ATTRIBUTES：CMake 链接、库相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • LIBRARY_TYPE=STATIC：这一行继续补充当前命令或表达式的内容；变量说明：LIBRARY_TYPE：库、类型相关变量，其精确取值和作用域以当前页面或脚本上下文为准；运算符：= 表示赋值或键值关联。

  • OVERRIDE=DEFAULT：这一行继续补充当前命令或表达式的内容；变量说明：OVERRIDE：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；运算符：= 表示赋值或键值关联。

  • DEDUPLICATION=YES：这一行继续补充当前命令或表达式的内容；变量说明：DEDUPLICATION：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；运算符：= 表示赋值或键值关联。

  • )：这一行继续补充当前命令或表达式的内容。

LIBRARY_TYPE=STATIC

此特性仅对静态库有意义。

OVERRIDE=DEFAULT

DEFAULT 特性将被 WHOLE_ARCHIVE 特性覆盖，因为它们是兼容的且能提升用户体验：可以自由使用标准库规范和 $<LINK_LIBRARY:WHOLE_ARCHIVE>。

DEDUPLICATION=YES

当使用此特性时，链接器会加载静态库中的所有符号，因此无需在链接器命令行上重复该库。

WHOLE_ARCHIVE 特性可以像这样使用：

add_library(A STATIC ...)
add_library(B STATIC ...)

target_link_libraries(B PUBLIC A)
target_link_libraries(A PUBLIC B)

add_library(global SHARED ...)
target_link_libraries(global PRIVATE $<LINK_LIBRARY:WHOLE_ARCHIVE,A>)

代码说明

  • add_library(A STATIC ...)：调用 add_library() 命令，定义一个库目标。

  • add_library(B STATIC ...)：调用 add_library() 命令，定义一个库目标。

  • target_link_libraries(B PUBLIC A)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • target_link_libraries(A PUBLIC B)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • add_library(global SHARED ...)：调用 add_library() 命令，定义一个库目标。

  • target_link_libraries(global PRIVATE $<LINK_LIBRARY:WHOLE_ARCHIVE,A>)：调用 target_link_libraries() 命令，为目标链接依赖库；生成器表达式：$<LINK_LIBRARY:...> 在生成构建系统时按当前目标和配置计算结果。

最终生成的链接命令中只会指定一次 A 库，并且链接器标志将确保从 A 库中加载所有符号。
