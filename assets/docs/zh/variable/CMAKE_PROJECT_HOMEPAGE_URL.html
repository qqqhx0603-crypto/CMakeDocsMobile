CMAKE_PROJECT_HOMEPAGE_URL¶

3.12 版本新增。

顶级项目的首页 URL。

此变量包含在顶级 CMakeLists.txt 文件中由 project() 命令指定的项目的首页 URL。如果顶级 CMakeLists.txt 文件包含多个 project() 调用，则来自该顶级 CMakeLists.txt 的最近一次调用将决定 CMAKE_PROJECT_HOMEPAGE_URL 包含的值。例如，考虑以下顶级 CMakeLists.txt

cmake_minimum_required(VERSION 3.0)
project(First HOMEPAGE_URL "https://first.example.com")
project(Second HOMEPAGE_URL "https://second.example.com")
add_subdirectory(sub)
project(Third HOMEPAGE_URL "https://third.example.com")

代码说明

  • cmake_minimum_required(VERSION 3.0) project(First HOMEPAGE_URL "https://first.example.com") project(Second HOMEPAGE_URL "https://second.example.com") add_subdirectory(sub) project(Third HOMEPAGE_URL "https://third.example.com")：调用 `cmake_minimum_required()` 命令，声明工程所需的最低 CMake 版本。

以及以下内容的 sub/CMakeLists.txt：

project(SubProj HOMEPAGE_URL "https://subproj.example.com")
message("CMAKE_PROJECT_HOMEPAGE_URL = ${CMAKE_PROJECT_HOMEPAGE_URL}")

代码说明

  • project(SubProj HOMEPAGE_URL "https://subproj.example.com") message("CMAKE_PROJECT_HOMEPAGE_URL = ${CMAKE_PROJECT_HOMEPAGE_URL}")：调用 `project()` 命令，定义项目名称及语言等元数据。

来自顶级 `CMakeLists.txt` 的最后看到的 `project()` 命令将是 `project(Second ...)`，因此这将打印：

CMAKE_PROJECT_HOMEPAGE_URL = https://second.example.com

代码说明

  • CMAKE_PROJECT_HOMEPAGE_URL = https://second.example.com：这一行把某个名称与给定值关联起来，常见于配置或键值示例。

要从当前目录范围或更高范围内的 project() 的最近一次调用中获取首页 URL，请参阅 PROJECT_HOMEPAGE_URL 变量。
