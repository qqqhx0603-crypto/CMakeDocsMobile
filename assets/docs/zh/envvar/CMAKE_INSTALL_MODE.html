CMAKE_INSTALL_MODE¶

3.22 版本新增。

这是一个 CMake 环境变量。其初始值取自调用进程环境。

CMAKE_INSTALL_MODE 环境变量允许用户以替代模式运行 CMake 的 file(INSTALL) 和 install() 命令。

安装的默认行为是将源文件从源目录复制到目标目录。然而，此环境变量允许用户覆盖此行为，从而使 CMake 改为创建符号链接。

使用场景¶

安装符号链接而非复制文件在以下方面有所帮助：

  • 节省存储空间，因为文件不必在磁盘上重复存储。

  • 对符号链接源文件的更改会立即反映在安装目标中，无需重新运行安装步骤。

  • 通过安装目标处的链接进行编辑将修改链接的源文件。在处理 CMake 项目层级结构（例如使用 ExternalProject）并希望在各项目间保持一致的源码导航和重构时，这非常有用。

允许的值¶

CMAKE_INSTALL_MODE 允许使用以下值：

COPY，空值或未设置

在目标位置复制文件。这是默认行为。

ABS_SYMLINK

在目标位置创建指向源文件的绝对符号链接。如果无法创建链接，则报错停止。

ABS_SYMLINK_OR_COPY

类似于 ABS_SYMLINK，但如果无法创建符号链接，则静默回退为复制文件。

REL_SYMLINK

在目标位置创建指向源文件的相对符号链接。如果无法创建链接，则报错停止。

REL_SYMLINK_OR_COPY

类似于 REL_SYMLINK，但如果无法创建符号链接，则静默回退为复制文件。

SYMLINK

尝试通过 REL_SYMLINK 执行，如果无法使用相对路径表示引用的文件，则回退到 ABS_SYMLINK。如果无法创建链接，则报错停止。

SYMLINK_OR_COPY

类似于 SYMLINK，但如果无法创建符号链接，则静默回退为复制文件。

注意

符号链接本身只包含引用的文件路径，而不包含内容本身，因此有两种表达关系的方式：相对路径或绝对路径。

何时设置环境变量¶

为了使环境变量生效，必须在正确的构建阶段设置它。

  • 如果项目直接调用 file(INSTALL)，则必须在配置阶段设置环境变量。

  • 为了应用于 install()，环境变量必须在安装期间设置。这可能是在使用 install 或 package 构建目标进行构建时，或者在构建之外独立调用安装，亦或在命令行运行 cpack 时。

  • 当使用 ExternalProject 时，可能需要在构建阶段设置，因为外部项目自身的配置、构建和安装步骤是在主项目的构建阶段执行的。

鉴于上述情况，建议在所有阶段（配置、构建和安装）中一致地设置该环境变量。

注意事项¶

请谨慎使用此环境变量。以下几点需要考虑：

  • CMAKE_INSTALL_MODE 仅影响文件，不影响目录。

  • 并非所有平台都支持符号链接。

  • 此环境变量与 ExternalProject 安装步骤的交互方式更为复杂。有关详细信息，请参阅该模块的文档。

  • 符号链接以持久的方式将目标与源关联起来。写入两者中的任何一个都会影响两个文件系统对象。这与常规的安装行为形成对比，常规安装仅复制安装时存在的文件，源和目标之间没有持久的关系。

  • 不支持将 CMAKE_INSTALL_MODE 与 IOS_INSTALL_COMBINED 结合使用。

  • 更改之前运行所使用的 CMAKE_INSTALL_MODE 可能会导致意外结果。从非符号链接模式切换到符号链接模式将丢弃目标处之前的任何文件，但反之则不然。一旦目标处存在符号链接，即使切换回非符号链接模式，该符号链接仍会保留在目标处，而不会被实际文件替换。
