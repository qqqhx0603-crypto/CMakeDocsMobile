CMP0092¶

版本 3.15 新增。

默认情况下，MSVC 警告标志不会包含在 CMAKE_<LANG>_FLAGS 中。

在 CMake 3.14 及以下版本中使用类 MSVC 的编译器时，像 /W3 这样的警告标志默认会被添加到 CMAKE_<LANG>_FLAGS 中。对于那些希望以编程方式选择不同警告级别的项目来说，这会带来麻烦。特别是，它要求开发人员了解 CMake 的内置默认值，以便对 CMAKE_<LANG>_FLAGS 变量进行字符串编辑，从而替换掉这些默认值。

CMake 3.15 及以上版本倾向于默认不在 CMAKE_<LANG>_FLAGS 的值中包含警告标志。

此策略为尚未更新以适应“无警告标志”预期的项目提供了兼容性。该策略设置在第一个初始化给定语言 <LANG> 的 CMAKE_<LANG>_FLAGS 的 project() 或 enable_language() 命令执行时生效。

注意

一旦策略在项目顶层对给定语言生效，该选择就必须在整个树中对该语言使用。在具有嵌套项目的子目录中，请确保一起转换所有内容。

此策略的 OLD 行为是将 MSVC 警告标志放入默认的 CMAKE_<LANG>_FLAGS 缓存条目中。此策略的 NEW 行为是不将 MSVC 警告标志放入默认缓存条目中。

此策略在 CMake 版本 3.15 中引入。它可以由 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake **不** 会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
