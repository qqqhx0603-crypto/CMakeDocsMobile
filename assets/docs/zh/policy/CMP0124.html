CMP0124¶

3.21 版本新增。

foreach() 循环变量仅在循环范围内可用。

CMake 3.20 及更早版本始终在循环结束时将循环变量设置为其值，如果存在，则设置为循环之前的原始值；否则设置为空字符串。CMake 3.21 及更高版本更倾向于将循环变量恢复到循环开始时的状态，即已设置或未设置。此策略为期望循环变量始终保持已设置状态的项目提供兼容性。

此策略的 OLD 行为是在循环结束时设置循环变量，要么设置为其原始值，要么设置为一个空值。此策略的 NEW 行为是恢复循环变量到循环开始时的状态，即已设置或未设置。

例如

set(items a b c)

set(var1 "value")
unset(var2)

foreach(var1 IN LISTS items)
endforeach()

foreach(var2 IN LISTS items)
endforeach()

if(DEFINED var1)
  message("var1: ${var1}")
endif()
if(DEFINED var2)
  message("var2: ${var2}")
endif()

在 OLD 行为下，此代码会打印 var1: value 和 var2:。在 NEW 行为下，此代码只会打印 var1: value。

此策略在 CMake 3.21 版本中引入。可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 不会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
