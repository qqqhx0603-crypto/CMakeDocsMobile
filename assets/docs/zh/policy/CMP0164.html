CMP0164¶

3.30 版本新增。

add_library() 在平台不支持时拒绝 SHARED 库。

在 CMake 3.29 及更早版本中，对于不支持共享库的平台（TARGET_SUPPORTS_SHARED_LIBS 为 false），add_library() 命令会自动将 SHARED 库转换为 STATIC 库，以帮助用户在这些平台上构建项目。然而，共享库和静态库的语义差异很大，这种自动转换通常无法正常工作。使用共享库的项目需要针对这些平台进行逐个案例的移植。

在 CMake 3.30 及更高版本中，add_library() 倾向于拒绝在不支持共享库的平台上创建共享库，并以致命错误消息失败。此策略为那些曾经通过自动转换为静态库而正常工作，但尚未更新以进行显式移植的项目提供了兼容性。

此策略的 OLD 行为是隐式创建一个静态库并附带开发者警告。此策略的 NEW 行为是失败。

此策略是在 CMake 3.30 版本中引入的。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 不会警告行为更改，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
