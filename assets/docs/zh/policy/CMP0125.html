CMP0125¶

3.21 版本新增。

find_file()、find_path()、find_library() 和 find_program() 命令将其结果缓存于第一个参数指定的变量中。在 CMake 3.21 之前，如果调用前已存在同名的缓存变量但该缓存变量没有类型，则同名的非缓存变量会被丢弃，且始终使用该缓存变量（参见 CMP0126 以了解另一种类似的行为）。这与“非缓存变量应优先于同名缓存变量”的约定相矛盾。如果用户在命令行设置缓存变量时未指定类型（例如使用 cmake -DMYVAR=blah ... 而非 cmake -DMYVAR:FILEPATH=blah），就可能出现这种情况。

与上述相关的情况是，如果指定的缓存变量名已存在且确实拥有类型，各种 find_...() 命令会直接返回该值而不做修改。特别地，如果该变量包含相对路径，在此种情况下它不会被转换为绝对路径。

当策略 CMP0125 设置为 OLD 或未设置时，行为如上所述。当设置为 NEW 时，行为如下：

  • 如果在调用 find_...() 命令时存在指定名称的非缓存变量，则无论是否存在同名的缓存变量，都将使用该非缓存变量的值。如果之前不存在此类缓存变量，则此时不会创建缓存变量。如果之前已存在该名称的缓存变量，则缓存将被更新以匹配该非缓存变量。

  • 各种 find...() 命令总会在结果变量中提供一个绝对路径，除非缓存变量或非缓存变量提供的相对路径无法解析为现有路径。

此策略在 CMake 3.21 版本中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果不设置，CMake 不会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
