CMP0100¶

在 3.17 版本中添加。

让 AUTOMOC 和 AUTOUIC 处理以 .hh 后缀结尾的头文件。

从 3.17 版本开始，CMake 在 AUTOMOC 和 AUTOUIC 中处理以 .hh 后缀结尾的头文件。在更早的 CMake 版本中，这些头文件会被 AUTOMOC 和 AUTOUIC 忽略。

此策略影响在 AUTOMOC 和 AUTOUIC 中如何处理以 .hh 后缀结尾的头文件。

此策略的 OLD 行为是在 AUTOMOC 和 AUTOUIC 中忽略 .hh 头文件。

此策略的 NEW 行为是在 AUTOMOC 和 AUTOUIC 中像处理其他头文件一样处理 .hh 头文件。

注意

为了消除 CMP0100 警告，可以通过设置源文件属性 SKIP_AUTOMOC、SKIP_AUTOUIC 或 SKIP_AUTOGEN，将源文件从 AUTOMOC 和 AUTOUIC 处理中排除。

# Source skip example:
set_property(SOURCE /path/to/file1.hh PROPERTY SKIP_AUTOMOC ON)
set_property(SOURCE /path/to/file2.hh PROPERTY SKIP_AUTOUIC ON)
set_property(SOURCE /path/to/file3.hh PROPERTY SKIP_AUTOGEN ON)

代码说明

  • # Source skip example:：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • set_property(SOURCE /path/to/file1.hh PROPERTY SKIP_AUTOMOC ON)：调用 set_property() 命令，设置目标、目录、源文件等对象的属性。

  • set_property(SOURCE /path/to/file2.hh PROPERTY SKIP_AUTOUIC ON)：调用 set_property() 命令，设置目标、目录、源文件等对象的属性。

  • set_property(SOURCE /path/to/file3.hh PROPERTY SKIP_AUTOGEN ON)：调用 set_property() 命令，设置目标、目录、源文件等对象的属性。

此策略在 CMake 3.17.0 版本中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
