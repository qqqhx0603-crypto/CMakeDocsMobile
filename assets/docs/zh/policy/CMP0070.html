CMP0070¶

3.10 版本新增。

定义 file(GENERATE) 命令处理相对路径的行为。

CMake 3.10 及更高版本规定，传递给 INPUT 和 OUTPUT 参数的相对路径 file(GENERATE) 命令，分别相对于当前源码目录和构建目录进行解释。CMake 3.9 及更低版本未定义相对路径的行为，但也没有诊断它们，而是意外地将它们视为相对于进程的工作目录。策略 CMP0070 为使用旧的未定义行为的项目提供兼容性。

此策略影响传递给 file(GENERATE) 命令的相对路径的行为。此策略的 OLD 行为是将其视为相对于 CMake 的工作目录。此策略的 NEW 行为是相对于调用者的当前源码或构建目录解释相对路径。

此策略是在 CMake 3.10 版本中引入的。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
