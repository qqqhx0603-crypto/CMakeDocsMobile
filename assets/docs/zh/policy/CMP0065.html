CMP0065¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

3.4 版本新增。

不要在没有设置 ENABLE_EXPORTS 目标属性的情况下，向可执行文件添加导出符号的标志。

出于历史原因，CMake 3.3 及更早版本在某些平台上始终使用诸如 -rdynamic 之类的标志来链接可执行文件，以便通过 dlopen 加载的任何插件使用其中的符号。CMake 3.4 及更高版本更倾向于仅对显式设置了 ENABLE_EXPORTS 目标属性的可执行文件执行此操作。

此策略的 OLD 行为是，无论 ENABLE_EXPORTS 目标属性的值如何，始终在链接可执行文件时使用额外的链接标志。

此策略的 NEW 行为是，仅当 ENABLE_EXPORTS 目标属性设置为 True 时，才在链接可执行文件时使用额外的链接标志。

此策略是在 CMake 3.4 版本中引入的。在 CMake 4.0 版本中移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 默认情况下不发出警告，并使用 OLD 行为。

请参阅 CMAKE_POLICY_WARNING_CMP0065 变量的文档，以控制 CMake 4.0 版本之前的警告。
