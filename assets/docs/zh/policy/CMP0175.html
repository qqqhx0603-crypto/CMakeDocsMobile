CMP0175¶

在版本 3.31 中添加。

add_custom_command() 拒绝无效参数。

CMake 3.30 及更早版本会静默忽略 add_custom_command() 命令不同形式中不支持的关键字以及缺失或无效的参数。CMake 3.31 实施了更严格的参数检查，并将无效或缺失的参数标记为错误。

此策略的 OLD 行为将接受与 CMake 3.30 及更早版本相同的无效关键字或参数。NEW 行为会将以下此前未报告的情况标记为错误：

  • OUTPUT 形式不接受 PRE_BUILD、PRE_LINK 或 POST_BUILD 关键字。

  • 当给定 APPEND 关键字时，OUTPUT 形式也不接受 BYPRODUCTS、COMMAND_EXPAND_LISTS、DEPENDS_EXPLICIT_ONLY、DEPFILE、JOB_POOL、JOB_SERVER_AWARE、USES_TERMINAL 或 VERBATIM 关键字。

  • TARGET 形式要求必须且只能给定 PRE_BUILD、PRE_LINK 或 POST_BUILD 中的一个。此前，如果不提供任何关键字，则假定为 POST_BUILD；如果提供了多个关键字，则使用最后一个。

  • TARGET 形式不接受 DEPENDS、DEPENDS_EXPLICIT_ONLY、DEPFILE、IMPLICIT_DEPENDS、MAIN_DEPENDENCY、JOB_POOL 或 JOB_SERVER_AWARE 关键字。

  • TARGET 形式现在要求至少给定一个 COMMAND。

  • 如果某个关键字在其后期望一个值，但未提供任何值，此前会被视为完全没有给出该关键字。

  • COMMENT 关键字在其后期望且仅期望一个值。如果给出了多个值，或者多次给出了 COMMENT 关键字，则会报错。

此策略在 CMake 版本 3.31 中引入。它可以由 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
