CMP0037¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

目标名称不应使用保留名称，且应符合有效性模式。

CMake 2.8.12 及更低版本允许在使用 add_library()、add_executable() 和 add_custom_target() 创建目标时，对目标名称不做任何限制。诸如 cmake-generator-expressions(7) 等较新的 CMake 特性以及某些诊断工具要求目标名称必须匹配特定的限制模式。

目标名称可以包含大写和小写字母、数字、下划线（_）、点（.）、加号（+）和减号（-）。作为特殊情况，ALIAS 和 IMPORTED 目标可以包含两个连续的冒号。

不允许使用被一个或多个 CMake 生成器保留的目标名称。这些名称包括但不限于 all、clean、help 和 install。

与可选功能关联的目标名称（如 test 和 package）也可能是保留名称。CMake 3.10 及以下版本始终保留这些名称。CMake 3.11 及以上版本仅在启用相应功能时（例如通过包含 CTest 或 CPack 模块）才保留它们。

此策略的 OLD 行为是允许创建使用保留名称或不符合有效性模式的目标。此策略的 NEW 行为是，如果 add_* 命令使用了无效的目标名称，则报告错误。

此策略是在 CMake 版本 3.0 中引入的。在 CMake 版本 4.0 中移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。
