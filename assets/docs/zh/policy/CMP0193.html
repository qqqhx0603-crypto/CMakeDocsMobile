CMP0193¶

在 4.1 版本中新增。

GNUInstallDirs 在安装前缀为 / 时，缓存带有前缀 usr/ 的 CMAKE_INSTALL_* 变量。

当 CMAKE_INSTALL_PREFIX 为 / 时，对于 BINDIR、SBINDIR、LIBEXECDIR、SHAREDSTATEDIR、INCLUDEDIR、OLDINCLUDEDIR、DATAROOTDIR 和 LIBDIR 这些 <dir>，其对应的 CMAKE_INSTALL_<dir> 变量会根据 特殊情况 (special cases) 的文档，在路径前添加 usr/。在 CMake 4.0 及更低版本中，这些 CMAKE_INSTALL_<dir> 变量在缓存时并未包含前导的 usr/，该前缀是在覆盖缓存条目的普通变量中添加的。CMake 4.1 及更高版本倾向于将带有前导 usr/ 的路径缓存到 CMAKE_INSTALL_<dir> 中。因此，GNUInstallDirs_get_absolute_install_dir() 命令不再修改相对输入变量参数 <input-var>。此策略为尚未更新以适应新行为的项目提供了兼容性。

此策略的 OLD 行为是将 CMAKE_INSTALL_<dir> 变量缓存且不包含前导的 usr/。此策略的 NEW 行为是将 CMAKE_INSTALL_<dir> 变量缓存且包含前导的 usr/。

此策略是在 CMake 版本 4.1 中引入的。它可以由 cmake_policy() 或 cmake_minimum_required() 命令设置。如果未设置，CMake **不会** 警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
