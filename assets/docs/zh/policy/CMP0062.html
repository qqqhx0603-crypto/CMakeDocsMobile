CMP0062¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

3.3 版本中新增。

禁止 install() 对 export() 结果的操作。

export() 命令生成一个包含 导入目标 (Imported Targets) 的文件，该文件适用于从构建目录中使用。它不适合安装，因为它包含指向构建系统位置的绝对路径，并且特定于单个构建配置。

install(EXPORT) 生成并安装包含 导入目标 (Imported Targets) 的文件。这些文件生成时使用相对路径（除非用户指定了绝对路径），并且专为多配置使用而设计。更多信息请参阅 创建包 (Creating Packages)。

CMake 3.3 不再允许将 install(FILES) 命令用于 export() 命令的结果。

此策略的 OLD 行为是允许安装 export() 命令的结果。此策略的 NEW 行为是不允许安装 export() 命令的结果。

此策略引入于 CMake 版本 3.3。在 CMake 版本 4.0 移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 会发出警告并使用 OLD 行为。
