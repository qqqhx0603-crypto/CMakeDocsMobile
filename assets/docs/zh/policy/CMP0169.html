CMP0169¶

3.30 版本新增。

以单个参数（即已声明依赖项的名称）调用 FetchContent_Populate() 的做法已被弃用。

在引入 FetchContent_MakeAvailable() 之前，项目通常使用以下模式来填充（populate）先前声明的内容（使用 FetchContent_Declare()）：

FetchContent_GetProperties(depname)
if(NOT depname_POPULATED)
  FetchContent_Populate(depname)
  add_subdirectory(${depname_SOURCE_DIR} ${depname_BINARY_DIR})
endif()

代码说明

  • FetchContent_GetProperties(depname) if(NOT depname_POPULATED) FetchContent_Populate(depname) add_subdirectory(${depname_SOURCE_DIR} ${depname_BINARY_DIR}) endif()：调用 `FetchContent_GetProperties()` 命令，具体行为由这一行传入的参数决定。

上述模式不支持随着时间推移添加到 FetchContent 中的许多功能。它忽略了诸如 SYSTEM 和 EXCLUDE_FROM_ALL 等选项，这些选项可以传递给 FetchContent_Declare()，但无法让上述项目代码感知到。它也不支持 依赖项提供程序（dependency providers）。项目应调用 FetchContent_MakeAvailable()，而不是使用上述模式。

CMake 3.30 及更高版本倾向于拒绝以已声明依赖项的名称调用 FetchContent_Populate()。此策略为尚未更新为改用 FetchContent_MakeAvailable() 的项目提供了兼容性。

此策略的 OLD 行为允许以已声明依赖项的名称调用 FetchContent_Populate()。而 NEW 行为在此类情况下会停止执行并抛出致命错误。

注意

以完整的填充细节作为命令参数（而非仅使用依赖项名称）来调用 FetchContent_Populate() 仍然完全受支持。本策略仅废弃了以单个参数（即先前声明的依赖项名称）调用 FetchContent_Populate() 的形式。

此策略引入于 CMake 版本 3.30。它可以由 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
