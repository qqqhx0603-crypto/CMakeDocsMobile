CMP0185¶

4.0 版本新增。

FindRuby 不再提供大写 RUBY_* 变量。

FindRuby 在 CMake 3.31 及更早版本中提供了以大写 RUBY_ 前缀命名的结果变量。CMake 3.18 将它们重命名为 Ruby_ 前缀以匹配包的名称，但为了兼容性，继续提供大写 RUBY_ 变量。

CMake 4.0 及更高版本更倾向于只提供 Ruby_ 变量，不再提供 RUBY_ 变量。此策略为尚未更新的项目提供兼容性。

此策略的 OLD 行为是提供 Ruby_ 和 RUBY_ 变量。此策略的 NEW 行为是只提供 Ruby_ 变量，不提供 RUBY_ 变量。

此策略引入于 CMake 版本 4.0。可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake *不会* 警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
