CMP0209¶

4.3 版本新增。

验证接口头集检查不导出符号的可执行文件。

当 VERIFY_INTERFACE_HEADER_SETS 在一个可执行目标上设置为 true 时，CMake 4.2 及更早版本仅当目标的 ENABLE_EXPORTS 属性为 true 时才会执行这些检查。 排除的原因是，如果可执行目标不导出符号，则没有其他目标可以消费该可执行目标的头文件。 从那时起，验证头文件集出现了其他用例，其中导出符号不再是必需的。 因此，CMake 4.3 及更高版本更倾向于验证可执行目标的接口文件集，无论它们是否导出符号。

此策略的 OLD 行为仅当其 ENABLE_EXPORTS 属性设置为 true 时，才验证可执行目标的接口文件集。 NEW 行为在 VERIFY_INTERFACE_HEADER_SETS 设置为 true 时，始终验证可执行目标的接口文件集。

此策略是在 CMake 4.3 版本中引入的。可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
