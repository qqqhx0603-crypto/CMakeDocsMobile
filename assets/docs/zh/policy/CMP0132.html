CMP0132¶

在 3.24 版本中添加。

首次运行时不要设置编译器环境变量。

除了使用Xcode 生成器和一些 Visual Studio（集成开发环境） 生成器 之外，CMake 3.23 及更早版本会在启用相应语言时设置诸如 CC、CXX 等环境变量。这仅发生在 CMake 在构建目录中首次运行时，并且环境变量仅在配置时定义，而不是构建时定义。在后续的 CMake 运行中，这些环境变量不会被设置，从而导致首次运行和后续运行之间的行为差异。CMake 3.24 及更高版本更倾向于在启用语言时不要设置这些环境变量，即使是在构建目录中首次运行时也是如此。

此策略的 OLD 行为是在启用语言时首次运行时设置相关的环境变量。此策略的 NEW 行为是不设置任何此类环境变量。

此策略已在 CMake 版本 3.24 中引入。它可以由 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake **不会** 警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
