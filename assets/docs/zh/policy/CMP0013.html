CMP0013¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

不允许重复的二进制目录。

CMake 2.6.3 及更早版本允许 `add_subdirectory()` 调用多次创建相同的二进制目录。 在构建系统生成文件时，文件会被写入构建树并被覆盖，这可能导致奇怪的行为。 CMake 2.6.4 及更高版本会显式检测重复的二进制目录。 CMake 2.6.4 始终将这种情况视为错误。 在 CMake 2.8.0 及更高版本中，此策略决定了是否将这种情况视为错误。 此策略的 OLD 行为是允许重复的二进制目录。 此策略的 NEW 行为是不允许重复的二进制目录，并报错。

此策略是在 CMake 2.8.0 版本中引入的。 在 CMake 4.0 版本中移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。 如果未设置，CMake 会发出警告，并使用 OLD 行为。
