CMP0120¶

在 3.20 版本中添加。

WriteCompilerDetectionHeader 模块已被移除。

CMake 3.1 到 3.19 版本提供此模块，用于通过重用 CMake 预处理器检查表中的信息来生成 C++ 兼容层，用于 cmake-compile-features(7)。但是

  • 这些细粒度的特性已被用于 要求语言标准 的元特性所取代，例如 cxx_std_11。因此，将不会添加新的细粒度特性检查，项目需要使用其他方式来有条件地使用新的 C++ 特性。

  • 该模块将 CMake 的一些实现细节直接暴露给 C++ 翻译单元。

  • 该模块的方法有效地提供了一个与 CMake 捆绑的头文件，从而将头文件的版本与 CMake 的版本绑定。许多项目发现 WriteCompilerDetectionHeader 最好的使用方式是手动使用最新版本的 CMake 局部生成其头文件，然后将其与项目源代码捆绑在一起，以便可以使用较旧的 CMake 版本。

出于包括上述原因，CMake 3.20 及更高版本更倾向于不提供 WriteCompilerDetectionHeader 模块。此策略为尚未移植到其他方案的项目提供兼容性。使用该模块的项目应更新以停止使用它。替代方案包括：

  • 将生成的头文件的副本捆绑在项目的源代码中。

  • 使用第三方替代方案，例如 CC0 许可的 Hedley。

  • 放弃对功能原生提供的编译器太旧的支持。

此策略的 OLD 行为是允许包含已弃用的 WriteCompilerDetectionHeader 模块。 NEW 行为是包含该模块将失败，就像该模块不存在一样。

此策略是在 CMake 3.20 版本中引入的。 它可以由 cmake_policy() 或 cmake_minimum_required() 设置。 如果未设置，CMake 会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
