CMP0042¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

MACOSX_RPATH 默认启用。

CMake 2.8.12 及更高版本支持在目标的安装名称中使用 @rpath。 这通过设置目标属性 MACOSX_RPATH 来启用。 在安装名称中使用 @rpath 比使用 @executable_path 或 @loader_path 查找共享库更灵活、功能更强大。

CMake 3.0 及更高版本默认倾向于将此属性设置为 ON。 希望在目标安装名称中使用 @rpath 的项目可以删除任何设置 INSTALL_NAME_DIR 和 CMAKE_INSTALL_NAME_DIR 变量。

此策略是在 CMake 版本 3.0 中引入的。在 CMake 版本 4.0 中移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。
