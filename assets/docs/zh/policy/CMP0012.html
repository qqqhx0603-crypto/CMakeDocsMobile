CMP0012¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

if() 识别数字和布尔常量。

在 CMake 2.6.4 及更低版本中，if() 命令会隐式解引用与变量对应的参数，即使这些变量的名称类似于数字或布尔常量，除了 0 和 1。诸如 true、false、yes、no、on、off、y、n、notfound、ignore（均不区分大小写）的数字和布尔常量在某些情况下会被识别，但并非全部。例如，代码 if(TRUE) 可能会被评估为 false。像 2 这样的数字仅在布尔表达式中被识别，例如 if(NOT 2)（结果为 false），而不能作为单个参数，例如 if(2)（同样结果为 false）。CMake 的后续版本更倾向于将数字和布尔常量按字面意义处理，因此不应将其用作变量名。

此策略的 OLD 行为是隐式解引用名称类似于数字和布尔常量的变量。此策略的 NEW 行为是识别数字和布尔常量，而不解引用具有此类名称的变量。

此策略是在 CMake 2.8.0 版本中引入的。在 CMake 4.0 版本中移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 会发出警告，并使用 OLD 行为。
