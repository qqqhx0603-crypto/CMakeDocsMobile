CMP0149¶

在 3.27 版本中新增。

Visual Studio 生成器默认选择最新的 Windows SDK。

Visual Studio 生成器会选择一个 Windows SDK 版本，并将其填入 .vcxproj 文件中的 WindowsTargetPlatformVersion 设置。CMake 会将 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION 变量设置为所选的 SDK 版本。

在 CMake 3.27 之前，SDK 版本始终根据 CMAKE_SYSTEM_VERSION 变量的值来选择。用户或工具链文件可以将该变量设置为宿主系统上可用的某个精确 Windows SDK 版本。由于 CMAKE_SYSTEM_VERSION 默认值为 CMAKE_HOST_SYSTEM_VERSION，且无法保证一定存在匹配的 Windows SDK 版本，因此如果找不到精确匹配版本，CMake 不得不回退使用最新的 Windows SDK 版本。这种做法存在问题：

  • 是否选择最新的 Windows SDK 取决于宿主 Windows 版本是否恰好与可用的 SDK 版本匹配，这具有不确定性。

  • 可能会选到一个未针对 C11 等较新语言标准进行更新的旧版 Windows SDK。

CMake 3.27 及更高版本倾向于忽略 CMAKE_SYSTEM_VERSION 的精确值，并默认选择当前可用的最新 SDK 版本。如需指定精确的 SDK 版本，可以在 CMAKE_GENERATOR_PLATFORM 变量中使用 version= 字段显式指定。请参阅 Visual Studio 平台选择。

此策略为那些尚未迁移、仍使用 CMAKE_SYSTEM_VERSION 来指定精确 SDK 版本的项目、工具链文件和构建脚本提供兼容性。

注意

此策略必须在项目顶层的第一个 project() 或 enable_language() 命令调用之前设置。因为这是 Visual Studio 生成器选择 Windows SDK 的时刻。

此策略的 OLD 行为是尽可能使用 CMAKE_SYSTEM_VERSION 的精确值。NEW 行为则是忽略该变量。

此策略是在 CMake 版本 3.27 中引入的。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake *不会* 发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
