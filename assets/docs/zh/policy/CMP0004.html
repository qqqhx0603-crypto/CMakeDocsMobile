CMP0004¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

链接的库名称前后不能包含空格。

CMake 2.4 及更早版本会悄无声息地移除使用类似代码链接的库名称前后的空格：

target_link_libraries(myexe " A ")

代码说明

  • target_link_libraries(myexe " A ")：调用 target_link_libraries() 命令，为目标链接依赖库。

这可能导致用户项目中的细微错误。

此策略的 OLD 行为是悄无声息地移除库名称前后的空格。此策略的 NEW 行为是诊断此类空格的存在并将其视为错误。在检查库名称时，此策略的设置是在使用 add_executable() 或 add_library() 命令创建目标时生效的设置。

此策略在 CMake 2.6.0 版本中引入。在 CMake 4.0 版本移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 会发出警告并使用 OLD 行为。
