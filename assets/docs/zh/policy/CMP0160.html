CMP0160¶

在版本 3.29 中添加。

更多只读目标属性在尝试设置时会报错。

set_target_properties() 和 set_property() 命令本应在所有只读属性上报错。然而，CMake 3.28 及以下版本仅针对以下属性执行此操作：

  • HEADER_SETS

  • INTERFACE_HEADER_SETS

  • IMPORTED_GLOBAL

  • MANUALLY_ADDED_DEPENDENCIES

  • NAME

  • TYPE

此策略强制执行以下目标属性的只读性质：

  • ALIAS_GLOBAL

  • BINARY_DIR

  • CXX_MODULE_SETS

  • IMPORTED

  • INTERFACE_CXX_MODULE_SETS

  • LOCATION

  • LOCATION_<CONFIG>

  • SOURCE_DIR

此策略的 OLD 行为是仅针对属性 MANUALLY_ADDED_DEPENDENCIES、NAME 和 TYPE 报错。此策略的 NEW 行为是针对所有被记录为只读的目标属性报错。

此策略于 CMake 3.29 版本中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 将发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
