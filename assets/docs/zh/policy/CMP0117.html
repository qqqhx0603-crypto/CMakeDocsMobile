CMP0117¶

在 3.20 版本中添加。

MSVC RTTI 标志 /GR 不再默认添加到 CMAKE_CXX_FLAGS 中。

在使用 CMake 3.19 及以下版本的 MSVC 类编译器时，RTTI 标志 /GR 会默认添加到 CMAKE_CXX_FLAGS 中。这种行为源于对 Visual Studio（集成开发环境） 2003 及更早版本的 MSVC 支持，因为那些版本默认未启用 RTTI。现在已不再需要此做法。此外，对于希望以编程方式更改为 /GR- 的项目来说，这会造成问题。具体而言，它要求在知晓 CMake 内置默认值的前提下，对 CMAKE_CXX_FLAGS 变量进行字符串编辑，以便进行替换。

CMake 3.20 及以上版本倾向于默认不在 CMAKE_CXX_FLAGS 的值中包含 /GR。

此策略为那些尚未更新以适应缺少 /GR 标志的项目提供了兼容性。策略设置在第一个初始化 CMAKE_CXX_FLAGS 的 project() 或 enable_language() 命令调用时生效。

注意

一旦策略在项目顶层对给定语言生效，该选择就必须在整个树中对该语言使用。在具有嵌套项目的子目录中，请确保一起转换所有内容。

此策略的 OLD 行为是将 MSVC 的 /GR 标志放入默认的 CMAKE_CXX_FLAGS 缓存条目中。此策略的 NEW 行为是不在默认缓存条目中放入 MSVC 的 /GR 标志。

此策略于 CMake 3.20 版本引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 不会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
