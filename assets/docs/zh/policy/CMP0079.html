CMP0079¶

3.13 版本新增。

target_link_libraries() 允许用于其他目录中的目标。

在 CMake 3.13 之前，target_link_libraries() 命令不允许将非当前目录创建的目标作为第一个参数，用于更新目标自身 LINK_LIBRARIES 的调用。它意外地接受了仅更新 INTERFACE_LINK_LIBRARIES 的调用中来自其他目录的目标，但仅仅是将条目添加到属性中，就像在原始目录中进行调用一样。因此，通过这种方式指定的链接接口库始终由生成器在原始目标的作用域中查找，而不是在调用 target_link_libraries() 的作用域中查找。

CMake 3.13 现在允许从任何目录调用 target_link_libraries() 命令，以便向在其他目录中创建的目标添加链接依赖项和链接接口库。这些条目被添加到 LINK_LIBRARIES 和 INTERFACE_LINK_LIBRARIES 中，并使用特殊的（内部）后缀来告知生成器在调用作用域中查找名称，而不是在创建该目标的作用域中查找。

此策略为那些已经对其他目录中的目标使用带有 INTERFACE 关键字的 target_link_libraries() 的项目提供了兼容性，以便在目标所在目录中查找 INTERFACE_LINK_LIBRARIES 条目。此类项目应进行更新，以了解在这种情况下新的作用域规则。

此策略的 OLD 行为是不允许 target_link_libraries() 调用命名来自其他目录的目标，但先前意外允许的仅使用 INTERFACE 关键字的情况除外。此策略的 NEW 行为是允许所有此类调用，但使用新的作用域规则。

此策略在 CMake 版本 3.13 中引入。它可以由 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 将发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
