CMP0129¶

在版本 3.23 中添加。

MCST LCC 编译器的编译器 ID 现在是 LCC，而不是 GNU。

CMake 3.23 及以上版本将 MCST LCC 编译器识别为与 GNU 不同的编译器，具有自己的命令行和功能集。CMake 现在更倾向于通过将 CMAKE_<LANG>_COMPILER_ID 变量设置为 LCC 而不是 GNU 来向项目呈现此信息。但是，现有项目可能假定 LCC 的编译器 ID 是 GNU，就像在 CMake 3.23 之前的版本中一样。因此，此策略确定 MCST LCC 编译器在语言 <LANG> 被 project() 或 enable_language() 命令启用后，在 CMAKE_<LANG>_COMPILER_ID 变量中报告哪个编译器 ID。必须在调用任一命令之前设置此策略。

此策略的 OLD 行为是使用编译器 ID GNU（并将 CMAKE_<LANG>_COMPILER_VERSION 设置为支持的 GNU 编译器版本。）此策略的 NEW 行为是使用编译器 ID LCC，并将 CMAKE_<LANG>_SIMULATE_ID 设置为 GNU，并将 CMAKE_<LANG>_SIMULATE_VERSION 设置为支持的 GNU 编译器版本。

此策略是在 CMake 3.23 版本中引入的。可以通过 cmake_policy() 或 cmake_minimum_required() 来设置。如果未设置，CMake 默认情况下不发出警告，并使用 OLD 行为。

请参阅 CMAKE_POLICY_WARNING_CMP0129 变量的文档以控制警告。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
