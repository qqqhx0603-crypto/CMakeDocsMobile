CMP0084¶

3.14 版新增。

FindQt 模块对 find_package() 不存在。

FindQt 的存在意味着，为了让 Qt 上游提供的包配置文件能被 find_package(Qt) 找到，使用方项目必须显式指定 find_package(Qt CONFIG)。移除该模块为 Qt 提供了一条途径，使其能够导出自己的配置文件，并能被使用方项目轻松找到。

此策略假定 CMake 内置的 FindQt 模块对 find_package() 不存在。如果项目确实需要使用 Qt 3 或 4，可以调用 find_package(Qt[34])、include(FindQt)，或者将 FindQt 添加到它们的 CMAKE_MODULE_PATH 中。

此策略的 OLD 行为是使 FindQt 对 find_package() 可用。NEW 行为则是假定它对 find_package() 不存在。

此策略在 CMake 版本 3.14 中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
