CMP0095¶

3.16 版新增。

在中间的 CMake 安装脚本中，RPATH 条目会正确转义。

在 CMake 3.15 及更早版本中，通过 CMAKE_INSTALL_RPATH 或 INSTALL_RPATH 设置的 RPATH 条目在插入到 cmake_install.cmake 脚本之前未被转义。基于 ELF 的系统（例如 Linux 和 FreeBSD）上的动态链接器允许在 RPATH 条目中使用某些关键字，例如 ${ORIGIN} （更多详细信息可在这些系统上的 ld.so 手册页中找到）。这些关键字的语法可能与 CMake 的变量语法匹配。为了避免被中间的 cmake_install.cmake 脚本替换（通常替换为空字符串），用户必须双重转义这些 RPATH 关键字，例如 set(CMAKE_INSTALL_RPATH "\\\${ORIGIN}/../lib")。由于中间的 cmake_install.cmake 脚本是 CMake 的一个实现细节，CMake 3.16 及更高版本将确保通过转义任何巧合的 CMake 语法来字面插入 RPATH 条目。

此策略的 OLD 行为是不转义中间 cmake_install.cmake 脚本中的 RPATH 条目。NEW 行为是在生成中间 cmake_install.cmake 脚本时，正确转义 RPATH 条目中巧合的 CMake 语法。

此策略在 CMake 3.16 版本中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置此策略，当 CMake 检测到使用类似 CMake 的语法时会发出警告，并采用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
