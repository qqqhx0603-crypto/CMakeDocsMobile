CMP0068¶

版本 3.9 中添加。

macOS（苹果桌面系统） 上的 RPATH 设置不影响 install_name。

CMake 3.9 及更高版本移除了以下设置对 macOS（苹果桌面系统） 上目标 install_name 的影响：

  • BUILD_WITH_INSTALL_RPATH 目标属性

  • SKIP_BUILD_RPATH 目标属性

  • CMAKE_SKIP_RPATH 变量

  • CMAKE_SKIP_INSTALL_RPATH 变量

此前，设置 BUILD_WITH_INSTALL_RPATH 会同时将目标的 install_name 设置为 INSTALL_NAME_DIR，并将 RPATH 设置为 INSTALL_RPATH。在 CMake 3.9 中，它仅影响 RPATH 的设置。然而，如果希望 INSTALL_NAME_DIR 应用于构建树中的目标，可以设置 BUILD_WITH_INSTALL_NAME_DIR。

如果此前使用 SKIP_BUILD_RPATH、CMAKE_SKIP_RPATH 或 CMAKE_SKIP_INSTALL_RPATH 来剔除目标 install_name 的目录部分，现在可以改用设置 INSTALL_NAME_DIR=""。

此策略的 OLD 行为是在 macOS（苹果桌面系统） 上将 RPATH 设置用于 install_name。此策略的 NEW 行为是在 macOS（苹果桌面系统） 上忽略用于 install_name 的 RPATH 设置。

此策略于 CMake 3.9 版本引入。可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
