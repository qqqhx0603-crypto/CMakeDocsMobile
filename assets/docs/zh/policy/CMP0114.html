CMP0114¶

3.19 版本新增。

ExternalProject 步骤目标完全采用其步骤。

ExternalProject_Add() 的 STEP_TARGETS 选项以及 ExternalProject_Add_StepTargets() 函数可用于为外部项目的各个步骤创建构建目标。

在 CMake 3.18 及更低版本中，步骤目标存在一些局限性。

  • 步骤目标总是依赖于由 ExternalProject_Add() 的 DEPENDS 选项所指定的目标，即使并非所有步骤都需要它们。为了允许在没有这些依赖项的情况下创建步骤目标，可以使用 ExternalProject_Add() 的 INDEPENDENT_STEP_TARGETS 选项，或使用 ExternalProject_Add_StepTargets() 的 NO_DEPENDS 选项。然而，添加此类“独立”步骤目标仅对特定步骤（如 download、update 和 patch）才有意义，因为它们不需要任何外部项目的构建依赖项。此外，为依赖于非独立步骤的步骤创建独立步骤目标是没有意义的。此类规则并未强制执行，如果不遵守这些规则，项目可能会生成行为令人困惑且依赖于生成器的构建系统。

  • 步骤目标持有其实现步骤的自定义命令的副本，这些副本与 ExternalProject_Add() 所创建的主目标中的副本是相互独立的，且主目标并不依赖于这些步骤目标。在同时驱动主目标和步骤目标的并行构建中，步骤命令的多个副本可能会同时运行并发生竞争。

    此外，在策略 CMP0113 之前，由 Makefile（Make 构建脚本） 生成器 生成的步骤目标还包含其步骤所依赖的所有自定义命令。即使在串行构建中，这也可能导致这些步骤被重复执行。

在 CMake 3.19 及更高版本中，ExternalProject 模块倾向于采用改进后的设计来解决这些问题。

  • 如果某个步骤不依赖于由 ExternalProject_Add() 的 DEPENDS 指定的其他目标，则该步骤被归类为“独立”步骤。预定义的步骤默认会自动进行分类。

    • download、update 和 patch 步骤是独立的。

    • configure、build、test 和 install 步骤则不是。

    对于自定义步骤，ExternalProject_Add_Step() 命令提供了一个 INDEPENDENT 选项来将其标记为独立。如果一个步骤依赖于其他非独立步骤，却将其标记为独立，则是错误的。注意，“独立”一词的这种用法仅指独立于外部目标，与步骤对其他步骤的依赖关系无关。

  • 由 ExternalProject_Add() 的 STEP_TARGETS 选项或 ExternalProject_Add_Step() 函数创建的步骤目标，当且仅当其步骤被标记为独立时，它们才是独立的。不再允许使用 ExternalProject_Add() 的 INDEPENDENT_STEP_TARGETS 选项和 ExternalProject_Add_StepTargets() 的 NO_DEPENDS 选项。

  • 步骤目标一旦创建，就完全负责持有实现其步骤的自定义命令。ExternalProject_Add() 创建的主目标依赖于这些步骤目标，且步骤目标之间也相互依赖。目标级的依赖关系与每个步骤的自定义命令所使用的文件级依赖关系相匹配。

    当使用 ExternalProject_Add() 的 UPDATE_DISCONNECTED 或 TEST_EXCLUDE_FROM_MAIN 选项时，或者对自定义步骤使用 ExternalProject_Add_Step() 的 EXCLUDE_FROM_MAIN 选项时，可能会自动创建一些步骤目标。这些目标是必需的，用于存放主目标和断开连接的步骤目标通常依赖的步骤。

策略 CMP0114 为尚未更新以适配新行为的项目提供了兼容性。OLD 行为指使用上述 3.18 及更低版本中记录的行为。NEW 行为指使用上述 3.19 及更高版本所采用的行为。

此策略在 CMake 版本 3.19 中引入。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
