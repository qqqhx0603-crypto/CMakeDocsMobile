CMP0061¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

3.3 版本中新增。

CTest 默认不再通知 make 忽略错误 (-i)。

ctest_build() 和 build_command() 命令不再为 Makefile 生成器 生成带有 -i 选项的构建命令。此前这样做是为了尽可能多地构建被测试的项目。然而，这种行为与其他生成器不一致，并且会导致 make 工具的返回代码失去意义。

当然，用户仍然可以通过设置 CTEST_BUILD_COMMAND 或 MAKECOMMAND 缓存条目来手动添加此选项。有关其作用的影响，请参阅 CTest 构建步骤 中关于 MakeCommand 设置的文档。

此策略的 OLD（旧）行为是在 CTest 的 make 调用中添加 -i。此策略的 NEW（新）行为是不添加 -i。

此策略于 CMake 3.3 版本引入。在 CMake 4.0 版本移除之前，它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake 不会发出警告，并使用 OLD 行为。
