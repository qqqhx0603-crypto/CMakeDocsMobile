CMP0119¶

在 3.20 版本中添加。

LANGUAGE 源文件属性明确指定以指定语言编译。

LANGUAGE 源文件属性的文档说明是，该源文件是用指定的语言编写的。在 CMake 3.19 及更早版本中，设置此属性会导致 CMake 使用指定语言的编译器来编译源文件。但是，它仅传递一个明确的标志，告诉编译器将源视为指定的语言，适用于 MSVC-like、XL 和 Embarcadero 编译器以及 CXX 语言。CMake 3.20 及更高版本更倾向于使用类似于 -x c 的标志，在所有已知此类标志的编译器上明确告诉编译器使用指定的语言。

此策略为尚未更新以期望此行为的项目提供兼容性。例如，一些项目将 LANGUAGE 属性设置为 C，用于汇编语言 .S 源文件，以便使用 C 编译器编译它们。此类项目应更新为使用 enable_language(ASM)，CMake 通常会在相关平台上将 C 编译器选为汇编器。

此策略的 OLD 行为是使用其未记录的含义来解释 LANGUAGE <LANG> 属性，即“使用 <LANG> 编译器”。此策略的 NEW 行为是使用其记录的含义来解释 LANGUAGE <LANG> 属性，即“编译为 <LANG> 源”。

此策略是在 CMake 版本 3.20 中引入的。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 不会发出警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
