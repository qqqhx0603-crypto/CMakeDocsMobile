CMP0162¶

3.30 版本新增。

Visual Studio 生成器 默认添加 UseDebugLibraries 指示符。

Visual Studio 项目中的“使用调试库 (Use Debug Libraries)”设置用于指示哪些配置被视为调试配置。在独立项目中，这可能会影响 MSBuild 对 MSVC 运行时库、优化标志、运行时检查及类似设置的默认选择。CMake 通常会根据项目规范显式生成所有这些设置，因此 CMake 3.29 及以下版本不会向 .vcxproj 文件写入任何 UseDebugLibraries 指示符。

CMake 3.30 及以上版本倾向于写入 UseDebugLibraries 指示符，因为它们不仅方便人类和工具查阅，还可能影响特定平台 SDK 的行为。目标的每个配置的指示符确定方式如下：

  • 如果目标针对已知的 MSVC 运行时库编译源文件（例如通过 MSVC_RUNTIME_LIBRARY 指定的库），则对于编译为“调试 (Debug)”运行时库的配置，UseDebugLibraries 为 true，对于其他配置则为 false。

  • 否则（例如由 add_custom_target() 创建的目标），UseDebugLibraries 对于 Debug 配置为 true，对于其他配置则为 false。

此策略为尚未更新以支持这些指示符的项目提供了兼容性。策略设置会在每个目标创建时记录，并用于确定该目标 .vcxproj 文件的默认行为。

此策略的 OLD 行为是默认不生成 UseDebugLibraries 指示符。此策略的 NEW 行为是默认生成 UseDebugLibraries 指示符。

如果设置了 CMAKE_VS_USE_DEBUG_LIBRARIES 变量和/或 VS_USE_DEBUG_LIBRARIES 目标属性，它将明确控制 UseDebugLibraries 的生成，而无需考虑此策略。

此策略引入于 CMake 版本 3.30。它可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake *不会* 警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
