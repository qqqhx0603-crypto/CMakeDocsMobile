CMP0023¶

警告

此策略的 OLD 行为已在 CMake 4.0 版本中移除。此策略必须通过调用 cmake_minimum_required() 或 cmake_policy() 来设置为 NEW。

普通格式与关键字格式的 target_link_libraries() 签名不能混用。

CMake 2.8.12 引入了使用 PUBLIC、PRIVATE 和 INTERFACE 关键字的 target_link_libraries() 签名，旨在推广 CMake 2.8.7 中引入的 LINK_PUBLIC 和 LINK_PRIVATE 关键字。使用任何这些关键字的签名都会显式地设置目标的链接接口（link interface），即使该接口为空。当它与普通 target_link_libraries() 签名的历史行为结合使用时，会产生令人困惑的行为。例如，考虑以下代码：

target_link_libraries(mylib A)
target_link_libraries(mylib PRIVATE B)

代码说明

  • target_link_libraries(mylib A)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • target_link_libraries(mylib PRIVATE B)：调用 target_link_libraries() 命令，为目标链接依赖库。

在第一行代码执行后，链接接口尚未显式设置，因此 CMake 会使用链接实现（A）作为链接接口。然而，第二行代码将链接接口设置为空。为了避免这种隐晦的行为，CMake 现在倾向于禁止在单个目标中混用 target_link_libraries() 的普通签名和关键字签名。

此策略的 OLD 行为是允许混合使用关键字格式和普通格式的 target_link_libraries() 签名。此策略的 NEW 行为是不允许混用关键字签名和普通签名。

此策略在 CMake 版本 2.8.12 中引入。在 CMake 版本 4.0 移除之前，可以通过 cmake_policy() 或 cmake_minimum_required() 设置。如果未设置，CMake 会发出警告并使用 OLD 行为。
