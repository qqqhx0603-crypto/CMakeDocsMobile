CMP0201¶

版本 4.2 中添加。

Python::NumPy 不依赖于 Python::Development.Module。

FindPython3、FindPython2 和 FindPython 提供 Python{3,2,}::NumPy 和 Python{3,2,}::Development.Module 目标，当请求相应的组件时。

在 CMake 4.1 及更早版本中，请求 NumPy 组件意味着请求 Development.Module 组件，并且提供的 Python{3,2,}::NumPy 目标依赖于提供的 Python{3,2,}::Development.Module 目标。这种依赖是不必要的。在 CMake 4.2 及更高版本中，请求 NumPy 组件并不意味着请求 Development.Module 组件，并且提供的 Python{3,2,}::NumPy 目标不依赖于提供的 Python{3,2,}::Development.Module 目标。此策略为期望旧行为的项目提供兼容性。

此策略的 OLD 行为使 NumPy 依赖于 Development.Module。此策略的 NEW 行为则不。

此策略是在 CMake 版本 4.2 中引入的。它可以通过 cmake_policy() 或 cmake_minimum_required() 进行设置。如果未设置，CMake **不会** 警告，并使用 OLD 行为。

注意

策略的 OLD 行为根据定义已被弃用，并可能在未来版本的 CMake 中被移除。
