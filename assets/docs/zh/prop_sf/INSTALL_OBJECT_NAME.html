INSTALL_OBJECT_NAME¶

版本 4.2 中添加。

设置源文件的安装对象名称（不包含对象扩展名）。 空字符串值会禁用自定义对象命名。 该值必须是相对路径，并且不能包含特殊目录组件（例如 ..）。

请注意，在某些 INSTALL_OBJECT_NAME_STRATEGY 策略中，对象名称可能不会按原样使用。 策略可能根据其目标要求对其进行更改。

此属性支持 生成器表达式。

注意

CMake 不会对目标内部进行冲突检测。 使用此属性时，必须在项目代码中避免冲突。 CMake 有一些生成的源文件，它们也会创建可能与给定自定义名称冲突的对象文件。 这些包括

  • 生成的 PCH 源文件 (cmake_pch)

  • 生成的 Unity 编译文件 (unity_...)

  • Qt 自动生成源文件 (moc_compilations.cpp)
