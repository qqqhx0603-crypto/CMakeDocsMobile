SKIP_LINTING¶

在 3.27 版本中新增。

此属性允许您从代码检查（linting）过程中排除特定的源文件。代码检查过程包括在源文件上运行诸如 <LANG>_CPPLINT、<LANG>_CLANG_TIDY、<LANG>_CPPCHECK、<LANG>_ICSTAT、<LANG>_PVS_STUDIO 和 <LANG>_INCLUDE_WHAT_YOU_USE 等工具，以及作为 VERIFY_INTERFACE_HEADER_SETS 一部分对头文件进行编译。通过在源文件上设置 SKIP_LINTING，上述代码检查工具将不会对该特定文件执行。

示例¶

考虑一个包含多个源文件的 C++ 项目，例如 main.cpp、things.cpp 和 generatedBindings.cpp。在此示例中，您希望将 generatedBindings.cpp 文件从代码检查过程中排除。为此，您可以利用 SKIP_LINTING 属性与 set_source_files_properties() 命令，如下所示：

add_executable(MyApp main.cpp things.cpp generatedBindings.cpp)

set_source_files_properties(generatedBindings.cpp PROPERTIES
    SKIP_LINTING ON
)

在提供的代码片段中，SKIP_LINTING 属性被设置为 true 以用于 generatedBindings.cpp 源文件。因此，当执行 <LANG>_CPPLINT、<LANG>_CLANG_TIDY、<LANG>_CPPCHECK、<LANG>_ICSTAT 或 <LANG>_INCLUDE_WHAT_YOU_USE 指定的工具时，它们将跳过对 generatedBindings.cpp 文件的分析。

通过使用 SKIP_LINTING 属性，您可以选择性地从代码检查过程中排除特定的源文件。这使您可以将代码检查工具集中在项目的相关部分，从而提高代码检查工作流程的效率和效果。
