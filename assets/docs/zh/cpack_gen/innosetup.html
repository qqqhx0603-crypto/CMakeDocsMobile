CPack Inno Setup 生成器¶

在 3.27 版本中新增。

Inno Setup 是由 Jordan Russell 和 Martijn Laan 开发的一款免费的 Windows（微软桌面系统） 程序安装程序制作工具（https://jrsoftware.org/isinfo.php）。

本文档介绍了 Inno Setup 生成器特有的选项。

该生成器提供了许多选项（如组件）。遗憾的是，目前 Inno Setup 并不支持所有功能（例如组件依赖），因此生成器暂时忽略了这些功能。

CPack 要求使用 Inno Setup 6 或更高版本。

版本 3.30 新增: 该生成器现在可在非 Windows（微软桌面系统） 主机上使用，但需要 Wine 来运行 Inno Setup 工具。

CPack Inno Setup 生成器特有的变量¶

您可以使用以下变量来更改 CPack INNOSETUP 生成器的行为

常规¶

Inno Setup 生成器工作时无需设置以下任何变量。如果下文中某个变量被标记为强制性，但未进行设置，则会使用其默认值。

这些变量也可以包含 Inno Setup 常量，例如 {app}。请参考 Inno Setup 文档以获取更多信息。

如果要求提供任何文件的路径，您可以始终给出绝对路径，或者在大多数情况下提供相对于安装所有文件的顶层目录的相对路径，该路径由 install() 指令所指定。

CPack 会尝试为您转义引号和其他特殊字符。但是，使用特殊字符可能会导致问题。

以下变量简化了 CMake 中 Inno Setup 的使用

CPACK_INNOSETUP_USE_CMAKE_BOOL_FORMAT¶

Inno Setup 仅使用 yes 或 no 作为布尔格式，而 CMake 使用许多替代格式，如 ON 或 OFF。开启此选项可启用自动转换。

考虑以下示例

set(CMAKE_INNOSETUP_SETUP_AllowNoIcons OFF)

代码说明

  • set(CMAKE_INNOSETUP_SETUP_AllowNoIcons OFF)：调用 `set()` 命令，把变量 `CMAKE_INNOSETUP_SETUP_AllowNoIcons` 设为给定的值。

如果开启此选项，输出脚本中将生成以下行：AllowNoIcons=no。否则，将生成以下错误行：AllowNoIcons=OFF

转换在每个 Inno Setup 特有变量中均已启用。

强制项:

是

默认值：

ON

安装程序特定变量¶

CPACK_INNOSETUP_ARCHITECTURE¶

x86、x64、arm64 或 ia64 之一。此变量指定安装程序的目标架构。这也影响所使用的 Program Files 文件夹或注册表项。

CPack 会尝试通过试编译（参见 CMAKE_SIZEOF_VOID_P）来确定正确的值，但此选项也可以手动指定（特别是在使用 ia64 或交叉编译时）。

强制项:

是

默认值：

根据试编译结果，为 x86 或 x64

CPACK_INNOSETUP_INSTALL_ROOT¶

如果您不希望安装程序在 Program Files 下创建安装目录，则必须在此处指定安装根目录。

安装的完整目录将为：${CPACK_INNOSETUP_INSTALL_ROOT}/${CPACK_PACKAGE_INSTALL_DIRECTORY}。

强制项:

是

默认值：

{autopf}

CPACK_INNOSETUP_ALLOW_CUSTOM_DIRECTORY¶

如果开启，安装程序将允许用户通过提供额外的向导页面来更改安装目录。

强制项:

是

默认值：

ON

CPACK_INNOSETUP_PROGRAM_MENU_FOLDER¶

创建的开始菜单文件夹的初始名称。

如果此变量设置为 .，则不会创建单独的文件夹，应用程序快捷方式将出现在开始菜单的顶级文件夹中。

强制项:

是

默认值：

CPACK_PACKAGE_NAME 的值

CPACK_INNOSETUP_LANGUAGES¶

您希望 Inno Setup 包含的语言的 分号分隔列表。

当前可用：armenian, brazilianPortuguese, bulgarian, catalan, corsican, czech, danish, dutch, english, finnish, french, german, hebrew, icelandic, italian, japanese, norwegian, polish, portuguese, russian, slovak, slovenian, spanish, turkish 和 ukrainian。此列表可能因 Inno Setup 的版本而异。

强制项:

是

默认值：

english

CPACK_INNOSETUP_IGNORE_LICENSE_PAGE¶

如果您没有使用 CPACK_RESOURCE_FILE_LICENSE 指定许可证文件，CPack 将使用一个演示文件。如果您希望安装程序完全忽略许可证文件，可以启用此选项。

强制项:

是

默认值：

关闭

CPACK_INNOSETUP_IGNORE_README_PAGE¶

如果您没有使用 CPACK_RESOURCE_FILE_README 指定自述文件，CPack 将使用一个演示文件。如果您希望安装程序完全忽略自述文件，可以启用此选项。使用自定义自述文件时，请确保禁用此选项。

强制项:

是

默认值：

ON

CPACK_INNOSETUP_PASSWORD¶

启用密码保护和使用给定密码进行文件加密。

强制项:

否

CPACK_INNOSETUP_USE_MODERN_WIZARD¶

启用 Inno Setup 提供的现代外观。如果关闭此选项，则使用经典样式。图像和图标文件也会受到影响。

强制项:

是

默认值：

出于兼容性原因，默认为 OFF

CPACK_INNOSETUP_ICON_FILE¶

自定义安装程序 .ico 文件的路径。

使用 CPACK_PACKAGE_ICON 自定义向导中显示的位图文件。

强制项:

否

CPACK_INNOSETUP_SETUP_<directive>¶

此组允许适配 Inno Setup 提供的任何 [Setup] 部分指令，其中 directive 为其名称。

以下是一些示例

set(CPACK_INNOSETUP_SETUP_WizardSmallImageFile "my_bitmap.bmp")
set(CPACK_INNOSETUP_SETUP_AllowNoIcons OFF) # This requires CPACK_INNOSETUP_USE_CMAKE_BOOL_FORMAT to be on

代码说明

  • set(CPACK_INNOSETUP_SETUP_WizardSmallImageFile "my_bitmap.bmp") set(CPACK_INNOSETUP_SETUP_AllowNoIcons OFF) # This requires CPACK_INNOSETUP_USE_CMAKE_BOOL_FORMAT to be on：这一行继续补充当前命令或表达式的内容。

所有这些变量的优先级均高于其他变量。请考虑以下示例

set(CPACK_INNOSETUP_SETUP_Password "admin")
set(CPACK_INNOSETUP_PASSWORD "secret")

代码说明

  • set(CPACK_INNOSETUP_SETUP_Password "admin") set(CPACK_INNOSETUP_PASSWORD "secret")：调用 `set()` 命令，把变量 `CPACK_INNOSETUP_SETUP_Password` 设为给定的值。

最终密码将为 admin，因为 CPACK_INNOSETUP_PASSWORD 的优先级低于 CPACK_INNOSETUP_SETUP_Password。

强制项:

否

文件特定变量¶

尽管由 install() 指令安装的所有文件都会被自动处理并添加到安装程序中，但仍有一些变量可用于自定义安装过程。

在使用快捷方式（如 CPACK_CREATE_DESKTOP_LINKS）或 [Run] 条目中的可执行文件（仅限 .exe 或 .com）之前，您必须将原始文件名（无路径和扩展名）添加到 CPACK_PACKAGE_EXECUTABLES 中，并为其创建开始菜单快捷方式。

如果您有两个同名的原始文件（例如 a/executable.exe 和 b/executable.com），则该部分中的条目会被创建两次。这将导致未定义的行为，不建议这样做。

CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS¶

此变量应包含一个 分号分隔的 路径, 指令 对列表，可用于自定义为每个文件或目录自动创建的安装命令。

CPack 为每个文件创建以下 Inno Setup 指令...

Source: "absolute\path\to\my_file.txt"; DestDir: "{app}"; Flags: ignoreversion

代码说明

  • Source: "absolute\path\to\my_file.txt"; DestDir: "{app}"; Flags: ignoreversion：这一行给出示例文本或输出，用来说明：Source: "absolute\path\to\my_file.txt"; DestDir: "{app}"; Flags: ignoreversion

...并为每个目录创建以下行

Name: "{app}\my_folder"

代码说明

  • Name: "{app}\my_folder"：这一行给出示例文本或输出，用来说明：Name: "{app}\my_folder"

您可能希望更改 my_file.txt 的目标目录或标志。由于我们也可以提供相对路径，您想要的行如下

Source: "my_file.txt"; DestDir: "{userdocs}"; Flags: ignoreversion uninsneveruninstall

代码说明

  • Source: "my_file.txt"; DestDir: "{userdocs}"; Flags: ignoreversion uninsneveruninstall：这一行给出示例文本或输出，用来说明：Source: "my_file.txt"; DestDir: "{userdocs}"; Flags: ignoreversion uninsneveruninstall

您可以通过将 my_file.txt 用作 路径，将 Source: "my_file.txt"; DestDir: "{userdocs}"; Flags: ignoreversion uninsneveruninstall 用作 指令 来实现。

您必须处理 转义问题。因此 CMake 命令将是

set(CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS "my_file.txt;Source: \\\"my_file.txt\\\"\\; DestDir: \\\"{userdocs}\\\"\\; Flags: ignoreversion uninsneveruninstall")

代码说明

  • set(CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS "my_file.txt;Source: \\\"my_file.txt\\\"\\; DestDir: \\\"{userdocs}\\\"\\; Flags: ignoreversion uninsneveruninstall")：调用 `set()` 命令，把变量 `CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS` 设为给定的值。

为了提高可读性，您应该通过使用 CPACK_VERBATIM_VARIABLES 或将指令放入单独的 CPack 项目配置文件中来绕过转义问题。

如果您自定义了特定文件的安装指令，则会丢失与其组件的连接。为绕过此问题，请手动添加 Components: <component>。您还需要在自定义部分中自行添加其快捷方式和 [Run] 条目，因为 CPACK_PACKAGE_EXECUTABLES 将无法再找到该可执行文件。

这是另一个示例（注意：您必须绕过转义问题才能使示例正常工作）

set(CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS
    "component1/my_folder" "Name: \"{userdocs}\\my_folder\"\; Components: component1"
    "component2/my_folder2/my_file.txt" "Source: \"component2\\my_folder2\\my_file.txt\"\; DestDir: \"{app}\\my_folder2\\my_file.txt\"\; Flags: ignoreversion uninsneveruninstall\; Components: component2")

代码说明

  • set(CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS "component1/my_folder" "Name: \"{userdocs}\\my_folder\"\; Components: component1" "component2/my_folder2/my_file.txt" "Source: \"component2\\my_folder2\\my_file.txt\"\; DestDir: \"{app}\\my_folder2\\my_file.txt\"\; Flags: ignoreversion uninsneveruninstall\; Components: component2")：调用 `set()` 命令，把变量 `CPACK_INNOSETUP_CUSTOM_INSTALL_INSTRUCTIONS` 设为给定的值。

强制项:

否

CPACK_INNOSETUP_MENU_LINKS¶

此变量应包含一个 分号分隔的 链接, 链接名称 对列表，可用于在开始菜单文件夹中添加除可执行文件之外的快捷方式（参见 CPACK_PACKAGE_EXECUTABLES）。虽然 链接名称 是标签，但 链接 可以是 URL 或相对于安装目录的路径。

这是一个示例

set(CPACK_INNOSETUP_MENU_LINKS
    "doc/cmake-@CMake_VERSION_MAJOR@.@CMake_VERSION_MINOR@/cmake.html"
    "CMake Help" "https://cmake.com.cn" "CMake Web Site")

代码说明

  • set(CPACK_INNOSETUP_MENU_LINKS "doc/cmake-@CMake_VERSION_MAJOR@.@CMake_VERSION_MINOR@/cmake.html" "CMake Help" "https://cmake.com.cn" "CMake Web Site")：调用 `set()` 命令，把变量 `CPACK_INNOSETUP_MENU_LINKS` 设为给定的值。

强制项:

否

CPACK_INNOSETUP_CREATE_UNINSTALL_LINK¶

如果开启此选项，应用程序卸载程序的快捷方式会自动添加到开始菜单文件夹中。

强制项:

是

默认值：

关闭

CPACK_INNOSETUP_RUN_EXECUTABLES¶

CPACK_PACKAGE_EXECUTABLES 中指定的可执行文件的 分号分隔列表，用户可以在安装程序完成时运行这些文件。

它们会在内部添加到 [Run] 部分。

强制项:

否

组件特定变量¶

生成器支持组件以及下载的组件。但是，有些组件功能尚不支持（特别是组件依赖）。这些变量暂时会被忽略。

出于技术原因，如果组件有父组，CPack 将在 Inno Setup 中更改其名称。如果您拥有组件 component 及其父组 group，请考虑在 Inno Setup 脚本中将 group\component 用作组件名称。

以下是一些额外的组件变量

CPACK_INNOSETUP_<compName>_INSTALL_DIRECTORY¶

如果您不希望组件 compName 安装在 {app} 下，则必须在此处指定其安装目录。

强制项:

否

CPACK_INNOSETUP_VERIFY_DOWNLOADS¶

此选项仅影响已下载的组件。

如果开启此选项，则在编译和下载时计算已下载归档文件的哈希值。仅在匹配时，安装程序才会继续。

强制项:

是

默认值：

ON

编译和脚本编写特定变量¶

CPACK_INNOSETUP_EXECUTABLE¶

Inno Setup 脚本编译器命令的文件名。

强制项:

是

默认值：

ISCC

CPACK_INNOSETUP_EXECUTABLE_ARGUMENTS¶

Inno Setup 脚本编译器命令的额外命令行选项的 分号分隔列表。

例如：/Qp;/Smysigntool=$p

请处理 转义问题。

强制项:

否

CPACK_INNOSETUP_DEFINE_<macro>¶

此组允许将自定义定义指令作为命令行选项添加到 Inno Setup 预处理器命令中。每个条目模拟一个 #define public <macro> 指令。其宏可在任何地方访问（public），因此也可以在额外的脚本文件中使用。

宏名称不能包含任何特殊字符。有关详细规则，请参阅 Inno Setup 预处理器文档。

考虑以下示例

# The following line emulates: #define public MyMacro "Hello, World!"
set(CPACK_INNOSETUP_DEFINE_MyMacro "Hello, World!")

代码说明

  • # The following line emulates: #define public MyMacro "Hello, World!" set(CPACK_INNOSETUP_DEFINE_MyMacro "Hello, World!")：这是注释，用来说明：The following line emulates: #define public MyMacro "Hello, World!" set(CPACK_INNOSETUP_DEFINE_MyMacro "Hello, World!")。

此时，您可以在任何地方使用 MyMacro。例如在以下额外脚本中

AppComments={#emit "'My Macro' has the value: " + MyMacro}

代码说明

  • AppComments={#emit "'My Macro' has the value: " + MyMacro}：这一行把某个名称与给定值关联起来，常见于配置或键值示例。

请处理 转义问题。

强制项:

否

CPACK_INNOSETUP_EXTRA_SCRIPTS¶

要处理的额外 .iss 脚本文件的路径的 分号分隔列表。

它们会在内部使用 #include 指令包含在输出脚本文件的顶部。

您可以在文件中添加任何部分以扩展安装程序（例如添加额外的任务或注册表项）。在扩展 [Setup] 部分时，首选使用 CPACK_INNOSETUP_SETUP_<directive>。

强制项:

否

CPACK_INNOSETUP_CODE_FILES¶

要处理的额外 Pascal 文件的路径的 分号分隔列表。

此变量实际上与 CPACK_INNOSETUP_EXTRA_SCRIPTS 相同，只是您不必在文件顶部添加 [Code]。切勿在代码文件中更改当前部分。这将导致未定义的行为！请将它们视为普通的 Pascal 脚本。

代码文件包含在输出脚本的最底部。

强制项:

否
