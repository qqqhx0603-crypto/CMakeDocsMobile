CPack DEB 生成器¶

内置的（二进制）CPack DEB 生成器（仅限 Unix）

CPack Debian (DEB) 生成器特有的变量¶

CPack DEB 生成器可用于使用 CPack 创建 DEB 软件包。CPack DEB 生成器是一个 CPack 生成器，因此它使用由 CPack 使用的 CPACK_XXX 变量。

CPack DEB 生成器应该可以在任何 Linux 主机上工作，但当构建系统上可以使用 Debian 特定工具 dpkg-xxx 时，它将产生更好的 deb 软件包。

CPack DEB 生成器具有由特定的 CPACK_DEBIAN_XXX 变量控制的特定功能。

CPACK_DEBIAN_<COMPONENT>_XXXX 变量可用于获取组件特定的值。但请注意，<COMPONENT> 指的是以大写字母书写的分组名称。它可以是组件名称或组件组名称。

以下是一些 CPack DEB 生成器的 wiki 资源，出于历史原因保留在此，不再维护，但可能仍有参考价值：

  • https://gitlab.kitware.com/cmake/community/-/wikis/doc/cpack/Configuration

  • https://gitlab.kitware.com/cmake/community/-/wikis/doc/cpack/PackageGenerators#deb-unix-only

CPack DEB 生成器特有变量列表

CPACK_DEB_COMPONENT_INSTALL¶

为 CPackDEB 启用组件打包

强制项:

否

默认值：

关闭

如果启用（ON），则生成多个包。默认情况下，生成一个包含所有组件文件的单个包。

CPACK_DEBIAN_PACKAGE_NAME¶
CPACK_DEBIAN_<COMPONENT>_PACKAGE_NAME¶

设置软件包控制字段（变量会自动转换为小写）。

强制项:

是

默认值：

  • CPACK_PACKAGE_NAME 用于非基于组件的安装

  • CPACK_DEBIAN_PACKAGE_NAME，对于基于组件的安装，后缀为 -<COMPONENT>。

3.5 版本新增： 每个组件的 CPACK_DEBIAN_<COMPONENT>_PACKAGE_NAME 变量。

参见 https://www.debian.org/doc/debian-policy/ch-controlfields.html#s-f-source

CPACK_DEBIAN_FILE_NAME¶
CPACK_DEBIAN_<COMPONENT>_FILE_NAME¶

3.6 版本新增。

软件包文件名。

强制项:

是

默认值：

<CPACK_PACKAGE_FILE_NAME>[-<component>].deb

可以设置为

DEB-DEFAULT

告诉 CPack 自动以 deb 格式生成软件包文件名

<PackageName>_<VersionNumber>-<DebianRevisionNumber>_<DebianArchitecture>.deb

此设置被推荐为首选行为，但为了与 3.6 版本之前的 CMake 中的 CPack DEB 生成器保持向后兼容，这不是默认设置。如果没有此设置，可能会出现重复的文件名。重复的文件会被覆盖，打包者需要以防止此类错误的方式设置变量。

<file-name>[.deb]

使用给定的文件名。

3.29 版本变更： 如果文件名不以 .deb 或 .ipk 结尾，则会自动添加 .deb 后缀。之前必须显式包含该后缀。

<file-name>.ipk

3.10 版本新增。

使用给定的文件名。.ipk 后缀由 OPKG 打包系统使用。

3.10 版本新增。

Debian 软件包纪元（Epoch）

强制项:

否

默认值：

无

可选数字，当更改版本方案或修复旧版本软件包中的版本号错误时，应增加此数字。

CPACK_DEBIAN_PACKAGE_VERSION¶

Debian 软件包版本

强制项:

是

默认值：

CPACK_PACKAGE_VERSION

此变量只能包含字母数字（A-Za-z0-9）和字符 . + - ~（句点、加号、连字符、波浪号），并且应以数字开头。如果未设置 CPACK_DEBIAN_PACKAGE_RELEASE，则不允许使用连字符。

注意

为了与 CMake 3.9 及更低版本保持向后兼容，当未设置 CPACK_DEBIAN_PACKAGE_RELEASE 和 CPACK_DEBIAN_PACKAGE_EPOCH 变量时，对此变量内容的失败测试不是硬错误。而是报告作者警告。

CPACK_DEBIAN_PACKAGE_RELEASE¶

3.6 版本新增。

Debian 软件包发布版本 - Debian 修订号。

强制项:

否

默认值：

无

这是 DEB 软件包本身的编号，即打包的版本，而不是内容版本（参见 CPACK_DEBIAN_PACKAGE_VERSION）。如果之前的打包有缺陷，或者您想在此处放入花哨的 Linux 发行版特定编号，则可以更改默认值。

CPACK_DEBIAN_PACKAGE_DEPENDS¶
CPACK_DEBIAN_<COMPONENT>_PACKAGE_DEPENDS¶

设置此软件包的 Debian 依赖项。

强制项:

否

默认值：

  • 对于非基于组件的安装，为空字符串

  • 对于基于组件的安装，则为 CPACK_DEBIAN_PACKAGE_DEPENDS。

3.3 版本新增： 每个组件的 CPACK_DEBIAN_<COMPONENT>_PACKAGE_DEPENDS 变量。

注意

如果为此组件设置了 CPACK_DEBIAN_PACKAGE_SHLIBDEPS，或者更具体地说是 CPACK_DEBIAN_<COMPONENT>_PACKAGE_SHLIBDEPS，则发现的依赖项将附加到 CPACK_DEBIAN_<COMPONENT>_PACKAGE_DEPENDS，而不是 CPACK_DEBIAN_PACKAGE_DEPENDS。如果 CPACK_DEBIAN_<COMPONENT>_PACKAGE_DEPENDS 是一个空字符串，则只会为此组件设置自动发现的依赖项。

3.31 版本变更： 变量总是作为一个列表展开。在此之前，它仅在与 CPACK_DEB_COMPONENT_INSTALL、CPACK_DEBIAN_PACKAGE_SHLIBDEPS 或 CPACK_DEBIAN_<COMPONENT>_PACKAGE_SHLIBDEPS 配合使用时才会展开。这意味着如果一个组件没有发现共享库（例如仅由脚本组成的软件包），您必须自己连接列表以获得有效的 Depends 字段。

示例

set(CPACK_DEBIAN_PACKAGE_DEPENDS "libc6 (>= 2.3.1-6), libc6 (< 2.4)")
list(APPEND CPACK_DEBIAN_PACKAGE_DEPENDS cmake)

CPACK_DEBIAN_COMPRESSION_LEVEL¶

Added in version 4.3.

用于创建 Debian 软件包的压缩级别。

强制项:

否

默认值：

CPACK_COMPRESSION_LEVEL 的值

此变量允许对 Debian 软件包归档的压缩比和速度进行微调。它控制传递给由 CPACK_DEBIAN_COMPRESSION_TYPE 定义的压缩器的数字压缩级别。如果未设置 CPACK_DEBIAN_COMPRESSION_LEVEL，或者设置为 0，则将使用默认值。

有效范围和解释取决于所选的压缩类型

  • gzip – 级别 1–9

  • bzip2 – 级别 1–9

  • xz – 级别 1–9

  • lzma – 级别 1–9

  • zstd – 级别 1–19

用法示例

set(CPACK_DEBIAN_COMPRESSION_TYPE "xz")
set(CPACK_DEBIAN_COMPRESSION_LEVEL 9)

include(CPack)

CPACK_DEBIAN_PACKAGE_SHLIBDEPS¶
CPACK_DEBIAN_<COMPONENT>_PACKAGE_SHLIBDEPS¶

可以设置为 ON 以使用 dpkg-shlibdeps 生成更好的软件包依赖项列表。

强制项:

否

默认值：

  • 如果已设置，则为 CPACK_DEBIAN_PACKAGE_SHLIBDEPS，或者

  • 关闭

注意

如果您使用此功能，可能需要将 CMAKE_INSTALL_RPATH 设置为适当的值，因为如果您不这样做，dpkg-shlibdeps 可能会无法找到您自己的共享库。参见 https://gitlab.kitware.com/cmake/community/-/wikis/doc/cmake/RPATH-handling

注意

如果您使用此功能，也可以将 CPACK_DEBIAN_PACKAGE_SHLIBDEPS_PRIVATE_DIRS 设置为适当的值，以满足 dpkg-shlibdeps。但是，您应该仅对无法以其他方式解析的私有共享库执行此操作。

3.3 版本新增： 每个组件的 CPACK_DEBIAN_<COMPONENT>_PACKAGE_SHLIBDEPS 变量。

3.6 版本新增： CMAKE_INSTALL_RPATH 中 $ORIGIN 的正确处理。

CPACK_DEBIAN_PACKAGE_SHLIBDEPS_PRIVATE_DIRS¶

在 3.20 版本中添加。

可以设置为目录列表，这些目录将通过其 -l 选项提供给 dpkg-shlibdeps。dpkg-shlibdeps 将搜索这些目录以查找私有共享库依赖项。

强制项:

否

默认值：

无

注意

如果您使用 dpkg-shlibdeps，应该优先将 CMAKE_INSTALL_RPATH 设置为适当的值。当前的选项仅在需要私有共享库依赖项时才真正需要。

强制项:

  • 对于非基于组件的安装，为空字符串

这比 Breaks 具有更强的限制，Breaks 防止被破坏的软件包在破坏软件包处于“已解包”状态时被配置，但允许两个软件包同时解包。

CPACK_DEBIAN_<COMPONENT>_PACKAGE_SOURCE¶

3.5 版本新增。

设置二进制 Debian 软件包的 Source 字段。当二进制软件包名称与源码软件包名称不同时（特别是当从一个源码生成多个组件/二进制文件时），应使用字段 Source 指明二进制文件是从哪个源码生成的。

强制项:

否

默认值：

  • 对于非基于组件的安装，为空字符串

  • 对于基于组件的安装，为 CPACK_DEBIAN_PACKAGE_SOURCE。

参见 https://www.debian.org/doc/debian-policy/ch-controlfields.html#s-f-source

注意

此值不会被解释。也可以传递所引用的源码软件包的可选修订号。

CPACK_DEBIAN_PACKAGE_MULTIARCH¶
CPACK_DEBIAN_<COMPONENT>_PACKAGE_MULTIARCH¶

设置 Debian 软件包的 Multi-Arch 字段。软件包可以在其控制文件中声明它们应该如何处理不同架构的软件包被安装在同一台机器上的情况。

强制项:

否

默认值：

  • 对于非基于组件的安装，为空字符串

  • 对于基于组件的安装，为 CPACK_DEBIAN_PACKAGE_MULTIARCH。

3.31 版本新增： 每个组件的 CPACK_DEBIAN_<COMPONENT>_PACKAGE_MULTIARCH 变量。

参见 https://wiki.debian.org/MultiArch/Hints

注意

此值经过验证。它必须是以下值之一： same, foreign, allowed。

调试信息的打包¶

3.13 版本新增。

Dbgsym 软件包包含用于调试打包二进制文件的调试符号。

Dbgsym 打包有其自己的一组变量

CPACK_DEBIAN_DEBUGINFO_PACKAGE¶
CPACK_DEBIAN_<component>_DEBUGINFO_PACKAGE¶

启用 dbgsym .ddeb 软件包的生成。

强制项:

否

默认值：

关闭

注意

设置此项也会剥离生成的非 dbgsym 软件包中的 ELF 文件，这会导致调试信息仅在 dbgsym 软件包中可用。

注意

二进制文件在打包前必须包含调试符号，因此对于 CMAKE_BUILD_TYPE 变量值，请使用 Debug 或 RelWithDebInfo。

此外，如果设置了 CPACK_STRIP_FILES，文件将在进入 DEB 生成器之前被剥离，因此不会包含调试符号，也不会构建 dbgsym 软件包。不要与 CPACK_STRIP_FILES 一起使用。

在 Windows 上构建 Debian 软件包¶

3.10 版本新增。

为了将 UNIX 文件权限从安装阶段传达给 CPack DEB 生成器，使用了 cmake_mode_t NTFS 备用数据流 (ADT)。

当使用没有 ADT 支持的文件系统时，只能保留所有者的读/写权限。

可复现的软件包¶

3.13 版本新增。

环境变量 SOURCE_DATE_EPOCH 可以设置为一个 UNIX 时间戳，定义为自 1970 年 1 月 1 日 00:00:00 UTC 以来的秒数（不包括闰秒）。如果设置了，CPack DEB 生成器将在软件包中的时间戳使用其值。
