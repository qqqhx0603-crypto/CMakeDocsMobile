CPack 归档生成器¶

用于将文件打包进归档文件的 CPack 生成器，它可以采用以下任一格式

  • 7Z - 7zip - (.7z) - LZMA 压缩

    版本 3.1 中新增。

    这是 7Z_LZMA 的别名

  • 7Z_BZ2 - 7zip - (.7z) - BZip2 压缩

    4.3 版本新增。

  • 7Z_DEFLATE - 7zip - (.7z) - Deflate 压缩

    4.3 版本新增。

  • 7Z_LZMA - 7zip - (.7z) - LZMA 压缩

    4.3 版本新增。

  • 7Z_LZMA2 - 7zip - (.7z) - LZMA2 压缩

    4.3 版本新增。

  • 7Z_PPMD - 7zip - (.7z) - PPMd 压缩

    4.3 版本新增。

  • 7Z_STORE - 7zip - (.7z) - 不使用压缩

    4.3 版本新增。

  • 7Z_ZSTD - 7zip - (.7z) - Zstandard 压缩

    4.3 版本新增。

  • TAR (.tar) - 不使用压缩

    4.0 版本新增。

  • TBZ2 (.tar.bz2) - BZip2 压缩

  • TGZ (.tar.gz) - Deflate 压缩

  • TXZ (.tar.xz) - LZMA2 压缩

    版本 3.1 中新增。

  • TZ (.tar.Z) - LZW 压缩

  • TZST (.tar.zst) - Zstandard 压缩

    3.16 版新增。

  • ZIP (.zip) - Deflate 压缩

    这是 ZIP_DEFLATE 的别名

  • ZIP_BZ2 (.zip) - BZip2 压缩

    4.3 版本新增。

  • ZIP_DEFLATE (.zip) - Deflate 压缩

    4.3 版本新增。

  • ZIP_LZMA (.zip) - LZMA 压缩

    4.3 版本新增。

  • ZIP_LZMA2 (.zip) - LZMA2 压缩

    4.3 版本新增。

  • ZIP_STORE (.zip) - 不使用压缩

    4.3 版本新增。

  • ZIP_ZSTD (.zip) - Zstandard 压缩

    4.3 版本新增。

当此生成器从 CPackSourceConfig.cmake（或通过 package_source 目标）调用时，生成的归档文件将包含项目目录中的所有文件，但 CPACK_SOURCE_IGNORE_FILES 中指定的文件除外。以下是打包项目所有源文件的一个示例

set(CPACK_SOURCE_GENERATOR "TGZ")
set(CPACK_SOURCE_IGNORE_FILES
  \\.git/
  build/
  ".*~$"
)
set(CPACK_VERBATIM_VARIABLES YES)
include(CPack)

代码说明

  • set(CPACK_SOURCE_GENERATOR "TGZ") set(CPACK_SOURCE_IGNORE_FILES \\.git/ build/ ".*~$" ) set(CPACK_VERBATIM_VARIABLES YES) include(CPack)：调用 `set()` 命令，把变量 `CPACK_SOURCE_GENERATOR` 设为给定的值。

当此生成器从 CPackConfig.cmake（或通过 package 目标）调用时，生成的归档文件将包含通过 CMake 的 install() 命令（以及已弃用的命令 install_files()、install_programs() 和 install_targets（目标）()）安装的所有文件。

CPack 归档生成器专用变量¶

CPACK_ARCHIVE_FILE_NAME¶

版本 3.9 中添加。

基于组件的包的归档名称，不带扩展名。

默认值：

CPACK_PACKAGE_FILE_NAME

扩展名会自动附加。

如果 CPACK_COMPONENTS_GROUPING 设置为 ALL_COMPONENTS_IN_ONE，这将是单一输出归档文件的名称。

版本 4.0 中更改：此变量也适用于非组件包。

CPACK_ARCHIVE_<component>_FILE_NAME¶

版本 3.9 中添加。

不带扩展名的组件归档名称。

默认值：

<CPACK_ARCHIVE_FILE_NAME>-<component>，并将空格替换为 '-'。

扩展名会自动附加。请注意，变量名中的 <component> 为全大写。

CPACK_ARCHIVE_FILE_EXTENSION¶

在 3.25 版本中新增。

归档文件扩展名。

默认值：

默认值见上表。

CPACK_ARCHIVE_COMPONENT_INSTALL¶

启用组件打包。

默认值：

关闭

如果启用（ON），则生成多个包。默认情况下，生成一个包含所有组件文件的单个包。

CPACK_ARCHIVE_UID¶

设置归档中包含条目的 UID。指定 -1 以使用当前用户的 UID。

默认值：

0（参阅策略 CMP0206）

CPACK_ARCHIVE_GID¶

设置归档中包含条目的 GID。指定 -1 以使用当前用户的 GID。

默认值：

0（参阅策略 CMP0206）

CPack 归档生成器使用的变量¶

这些变量由归档生成器使用，但也适用于本质上是归档的 CPack 生成器。这些包括

  • CPack Cygwin 生成器

  • CPack FreeBSD 生成器

CPACK_ARCHIVE_THREADS¶

在 3.18 版本中新增。

执行压缩时使用的线程数。

默认值：

CPACK_THREADS 的值

如果设置为 0，则将使用机器上可用的核心数。请注意，并非所有压缩模式在所有环境下都支持线程处理。

版本 3.21 中新增：cmake.org 上提供的官方 CMake 二进制文件现在附带了支持并行压缩的 liblzma。旧版本不支持。

CPACK_ARCHIVE_COMPRESSION_LEVEL¶

4.3 版本新增。

压缩归档文件时使用的压缩级别。

默认值：

CPACK_COMPRESSION_LEVEL 的值

压缩级别应介于 0 和 9 之间。

除 ZIP_ZSTD 模式外，基于 Zstandard 的算法的压缩级别可以在 0 到 19 之间设置。

值 0 用于指定默认压缩级别。它由归档库后端自动选择，而不是直接由 CMake 本身设置。默认压缩级别可能因归档格式、平台等而异。
