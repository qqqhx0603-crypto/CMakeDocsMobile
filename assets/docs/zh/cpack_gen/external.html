CPack 外部生成器¶

3.13 版本新增。

CPack 提供了许多生成器，用于为各种平台和打包系统创建安装包。其目的是让 CMake/CPack 成为构建和打包软件项目的完整端到端解决方案。然而，由于技术限制或需要使用特定工具的策略，有时可能无法将 CPack 用于整个打包过程。因此，CPack 提供了“外部”(External) 生成器，允许外部打包软件利用 CPack 提供的一些功能，例如组件安装和依赖关系图。

与外部打包工具的集成¶

CPack 外部生成器会生成一个包含 CPack 内部元数据的 .json 文件，该文件为外部软件提供了关于如何打包该软件的信息。外部打包软件可以自行调用 CPack，读取生成的元数据，并根据需要安装和打包文件。

或者，CPack 也可以通过 CPACK_EXTERNAL_PACKAGE_SCRIPT 中的可选自定义 CMake 脚本来调用外部打包软件。

当通过 CPACK_EXTERNAL_ENABLE_STAGING 变量启用时，安装文件的暂存（Staging）也可以选择由生成器来处理。

JSON 格式¶

JSON 元数据文件包含 CPack 组件和组件组的列表、传递给 cpack_add_component() 和 cpack_add_component_group() 的各种选项、组件和组件组之间的依赖关系，以及传递给 CPack 的其他各种选项。

JSON 的根对象将始终提供两个字段：formatVersionMajor 和 formatVersionMinor，它们始终是描述生成器输出格式的整数。对输出格式的向后兼容更改（例如，添加以前不存在的新字段）会导致次版本号递增；而不兼容的更改（例如，删除字段或更改其含义）会导致主版本号递增，并将次版本号重置为 0。格式版本始终采用 major.minor 格式。换句话说，它始终由两个部分组成，中间用句点分隔。

您可以如下所述通过 CPACK_EXTERNAL_REQUESTED_VERSIONS 请求一个或多个特定版本的输出格式。输出格式的主版本号将与请求的主版本号完全匹配，且次版本号将大于或等于请求的次版本号。如果未通过 CPACK_EXTERNAL_REQUESTED_VERSIONS 请求任何版本，则默认使用已知的最新主版本。目前，唯一支持的格式是 1.0，如下所述。

版本 1.0¶

除了标准格式字段外，格式版本 1.0 在根目录下还提供以下字段：

组件

components 字段是一个对象，其中组件名称作为键，描述组件的对象作为值。组件对象具有以下字段：

名称

组件名称。这始终与 components 对象中的键相同。

displayName

传递给 cpack_add_component() 的 DISPLAY_NAME 字段的值。

描述

传递给 cpack_add_component() 的 DESCRIPTION 字段的值。

isHidden

如果传递了 HIDDEN 给 cpack_add_component()，则为 true；否则为 false。

isRequired

如果传递了 REQUIRED 给 cpack_add_component()，则为 true；否则为 false。

isDisabledByDefault

如果传递了 DISABLED 给 cpack_add_component()，则为 true；否则为 false。

group

仅当传递了 GROUP 给 cpack_add_component() 时才会出现。如果是，该字段是一个包含组件组名称的字符串值。

dependencies

一个数组，包含该组件所依赖的其他组件。它包含在传递给 cpack_add_component() 的 DEPENDS 参数中的值。如果没有传递 DEPENDS 参数，则这是一个空列表。

installationTypes

一个数组，包含该组件所属的安装类型。它包含在传递给 cpack_add_component() 的 INSTALL_TYPES 参数中的值。如果没有传递 INSTALL_TYPES 参数，则这是一个空列表。

isDownloaded

如果传递了 DOWNLOADED 给 cpack_add_component()，则为 true；否则为 false。

archiveFile

传递给 cpack_add_component() 的 ARCHIVE_FILE 参数的归档文件名。如果没有传递 ARCHIVE_FILE 参数，则这是一个空字符串。

componentGroups

componentGroups 字段是一个对象，其中组件组名称作为键，描述组件组的对象作为值。组件组对象具有以下字段：

名称

组件组名称。这始终与 componentGroups 对象中的键相同。

displayName

传递给 cpack_add_component_group() 的 DISPLAY_NAME 字段的值。

描述

传递给 cpack_add_component_group() 的 DESCRIPTION 字段的值。

parentGroup

仅当传递了 PARENT_GROUP 给 cpack_add_component_group() 时才会出现。如果是，该字段是一个包含组件组的父组名称的字符串值。

isExpandedByDefault

如果传递了 EXPANDED 给 cpack_add_component_group()，则为 true；否则为 false。

isBold

如果传递了 BOLD_TITLE 给 cpack_add_component_group()，则为 true；否则为 false。

组件

一个数组，包含作为该组直接成员的组件名称（即将此组作为其 GROUP 的组件）。子组的组件不包含在内。

subgroups

一个数组，包含作为该组子组的组件组名称（即将此组作为其 PARENT_GROUP 的组）。

installationTypes

installationTypes 字段是一个对象，其中安装类型名称作为键，描述安装类型的对象作为值。安装类型对象具有以下字段：

名称

安装类型名称。这始终与 installationTypes 对象中的键相同。

displayName

传递给 cpack_add_install_type() 的 DISPLAY_NAME 字段的值。

索引

安装类型在列表中的整数索引。

projects

projects 字段是一个描述构成 CPack 项目的 CMake 项目的对象数组。此字段中的值源自 CPACK_INSTALL_CMAKE_PROJECTS。在大多数情况下，这仅是一个单一项目。项目对象具有以下字段：

projectName

传递给 CPACK_INSTALL_CMAKE_PROJECTS 的项目名称。

component

构成该项目的组件或组件集的名称。

directory

CMake 项目的构建目录。这是包含 cmake_install.cmake 脚本的目录。

subDirectory

将项目安装到 CPack 包中的子目录。

packageName

CPACK_PACKAGE_NAME 中给出的包名称。仅当设置此选项时才会出现。

packageVersion

CPACK_PACKAGE_VERSION 中给出的包版本。仅当设置此选项时才会出现。

packageDescriptionFile

CPACK_PACKAGE_DESCRIPTION_FILE 中给出的包描述文件。仅当设置此选项时才会出现。

packageDescriptionSummary

CPACK_PACKAGE_DESCRIPTION_SUMMARY 中给出的包描述摘要。仅当设置此选项时才会出现。

buildConfig

通过 cpack -C 选项提供给 CPack 的构建配置。仅当设置此选项时才会出现。

defaultDirectoryPermissions

CPACK_INSTALL_DEFAULT_DIRECTORY_PERMISSIONS 中给出的默认目录权限。仅当设置此选项时才会出现。

setDestdir

如果 CPACK_SET_DESTDIR 为 true，则为 true；否则为 false。

packagingInstallPrefix

CPACK_PACKAGING_INSTALL_PREFIX 中给出的安装前缀。仅当 CPACK_SET_DESTDIR 为 true 时才会出现。

stripFiles

如果 CPACK_STRIP_FILES 为 true，则为 true；否则为 false。

warnOnAbsoluteInstallDestination

如果 CPACK_WARN_ON_ABSOLUTE_INSTALL_DESTINATION 为 true，则为 true；否则为 false。

errorOnAbsoluteInstallDestination

如果 CPACK_ERROR_ON_ABSOLUTE_INSTALL_DESTINATION 为 true，则为 true；否则为 false。

CPack 外部生成器特有的变量¶

CPACK_EXTERNAL_REQUESTED_VERSIONS¶

此变量用于请求特定版本的 CPack 外部生成器。它是一个由分号分隔的 major.minor 值列表。

如果将此变量设置为非空值，CPack 外部生成器将遍历列表中的每一项，以搜索它知道如何生成的版本。所请求的版本应按客户端软件的偏好降序排列，因为将生成列表中第一个匹配的版本。

如果生成器拥有一个主版本号与请求的主版本号完全匹配，且次版本号大于或等于请求的次版本号的版本化生成器，则该生成器即知道如何生成该版本。例如，如果 CPACK_EXTERNAL_REQUESTED_VERSIONS 包含 1.0，而 CPack 外部生成器知道如何生成 1.1，它将生成 1.1。如果生成器不知道如何生成列表中的某个版本，它会跳过该版本并查看下一个。如果它不知道如何生成任何请求的版本，则会抛出错误。

如果未设置此变量或该变量为空，CPack 外部生成器将生成它所知道的最高主版本号和次版本号。

如果在 CPACK_EXTERNAL_REQUESTED_VERSIONS 中遇到无效版本（即不匹配 major.minor，其中 major 和 minor 为整数），则该版本将被忽略。

CPACK_EXTERNAL_ENABLE_STAGING¶

可将此变量设置为 true 以启用向临时暂存区的可选安装，随后外部打包工具可以拾取并打包这些文件。CPack 用于当前打包任务的顶层目录包含在 CPACK_TOPLEVEL_DIRECTORY 中。它在每次运行开始打包前都会自动清理，并可用于外部打包工具所需的自定义临时文件。它还包含启用暂存时 CPack 执行安装的暂存区 CPACK_TEMPORARY_DIRECTORY。

CPACK_EXTERNAL_PACKAGE_SCRIPT¶

此变量可以可选地指定作为 CPack 调用的一部分运行的 CMake 脚本文件的完整路径。它在（可选）暂存完成后调用，并可能运行一个外部打包工具。该脚本可以访问由 CPack 配置文件定义的变量。

CPACK_EXTERNAL_BUILT_PACKAGES¶

3.19 版本新增。

CPACK_EXTERNAL_PACKAGE_SCRIPT 脚本可以将此列表变量设置为生成的包文件的完整路径。如果设置了 CPACK_PACKAGE_CHECKSUM，CPack 会将这些文件从暂存目录复制回顶级构建目录，并可能生成校验和文件。
