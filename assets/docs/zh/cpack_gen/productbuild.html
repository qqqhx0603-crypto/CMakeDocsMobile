CPack productbuild 生成器¶

3.7 版本中新增。

productbuild CPack 生成器（macOS（苹果桌面系统））。

CPack productbuild 生成器特有变量¶

以下变量专用于在 macOS（苹果桌面系统） 上使用 ProductBuild 构建的安装程序：

CPACK_COMMAND_PRODUCTBUILD¶

用于为 macOS（苹果桌面系统） 安装程序或 Mac App Store 生成产品归档的 productbuild(1) 命令的路径。此变量可用于覆盖自动检测到的命令（或在自动检测失败时指定其位置）。

CPACK_PRODUCTBUILD_IDENTIFIER¶

在版本 3.23 中添加。

设置与产品关联的唯一（非本地化）产品标识符（例如 com.kitware.cmake）。任何组件产品名称都将附加到此值后。

CPACK_PRODUCTBUILD_IDENTITY_NAME¶

版本 3.8 新增。

为生成的包添加数字签名。

CPACK_PRODUCTBUILD_KEYCHAIN_PATH¶

版本 3.8 新增。

指定用于搜索签名标识的特定钥匙串（keychain）。

CPACK_COMMAND_PKGBUILD¶

用于在 macOS（苹果桌面系统） 上生成组件包的 pkgbuild(1) 命令的路径。此变量可用于覆盖自动检测到的命令（或在自动检测失败时指定其位置）。

CPACK_PKGBUILD_IDENTITY_NAME¶

版本 3.8 新增。

为生成的包添加数字签名。

CPACK_PKGBUILD_KEYCHAIN_PATH¶

版本 3.8 新增。

指定用于搜索签名标识的特定钥匙串（keychain）。

CPACK_PREFLIGHT_<COMP>_SCRIPT¶

用作指定 <COMP> 组件包的 preinstall 脚本的文件的完整路径，其中 <COMP> 是大写的组件名称。如果未为给定组件定义此变量，则不会添加 preinstall 脚本。

CPACK_POSTFLIGHT_<COMP>_SCRIPT¶

用作指定 <COMP> 组件包的 postinstall 脚本的文件的完整路径，其中 <COMP> 是大写的组件名称。如果未为给定组件定义此变量，则不会添加 postinstall 脚本。

CPACK_PRODUCTBUILD_RESOURCES_DIR¶

版本 3.9 中添加。

如果指定，productbuild 生成器会将此目录中的文件（包括子目录）复制到 Resources 目录。此操作在 CPACK_RESOURCE_FILE_WELCOME、CPACK_RESOURCE_FILE_README 和 CPACK_RESOURCE_FILE_LICENSE 文件被复制之前执行。

CPACK_PRODUCTBUILD_DOMAINS¶

在版本 3.23 中添加。

此选项允许更精细地控制产品的安装位置。当设置为 true 时（参阅策略 CMP0161），以下形式的 domains 元素将被添加到 productbuild 分发 XML 中。

<domains enable_anywhere="true" enable_currentUserHome="false" enable_localSystem="true"/>

代码说明

  • <domains enable_anywhere="true" enable_currentUserHome="false" enable_localSystem="true"/>：这一行给出一个 XML 标签或其内容，用于描述结构化节点。

默认值如上所示，但可以使用 CPACK_PRODUCTBUILD_DOMAINS_ANYWHERE、CPACK_PRODUCTBUILD_DOMAINS_USER 和 CPACK_PRODUCTBUILD_DOMAINS_ROOT 进行覆盖。

CPACK_PRODUCTBUILD_DOMAINS_ANYWHERE¶

在版本 3.23 中添加。

可用于覆盖分发 XML 中 domains 元素里的 enable_anywhere 属性。设置为 true 时，产品可以安装在任何卷的根目录下，包括非系统卷。

CPACK_PRODUCTBUILD_DOMAINS 必须设置为 true，此变量才会生效。

CPACK_PRODUCTBUILD_DOMAINS_USER¶

在版本 3.23 中添加。

可用于覆盖分发 XML 中 domains 元素里的 enable_currentUserHome 属性。设置为 true 时，产品可以安装到当前用户的主目录下。请注意，当安装到用户主目录时，适用以下额外要求：

  • 安装程序不得在用户主目录之外进行写入。

  • 安装将以当前用户身份执行，而不是以 root 身份。这可能会对 CPACK_PREFLIGHT_<COMP>_SCRIPT 和 CPACK_POSTFLIGHT_<COMP>_SCRIPT 产生影响。

  • 执行安装不需要管理员权限。

CPACK_PRODUCTBUILD_DOMAINS 必须设置为 true，此变量才会生效。

CPACK_PRODUCTBUILD_DOMAINS_ROOT¶

在版本 3.23 中添加。

可用于覆盖分发 XML 中 domains 元素里的 enable_localSystem 属性。设置为 true 时，产品可以安装在根目录下。除非产品应仅安装到用户的主目录，否则通常应设置为 true。

CPACK_PRODUCTBUILD_DOMAINS 必须设置为 true，此变量才会生效。

背景图片¶

在 3.17 版本中添加。

此变量组用于控制生成安装程序的背景图片。

CPACK_PRODUCTBUILD_BACKGROUND¶

如果指定，则为分发 XML 添加背景。该值包含 Resources 目录中图像的路径。

CPACK_PRODUCTBUILD_BACKGROUND_ALIGNMENT¶

为分发 XML 中的背景添加 alignment 属性。请参阅 Apple 文档以获取有效值。

CPACK_PRODUCTBUILD_BACKGROUND_SCALING¶

为分发 XML 中的背景添加 scaling 属性。请参阅 Apple 文档以获取有效值。

CPACK_PRODUCTBUILD_BACKGROUND_MIME_TYPE¶

为分发 XML 中的背景添加 mime-type 属性。此选项包含图像的 MIME 类型。

CPACK_PRODUCTBUILD_BACKGROUND_UTI¶

为分发 XML 中的背景添加 uti 属性。此选项包含图像的 UTI 类型。

CPACK_PRODUCTBUILD_BACKGROUND_DARKAQUA¶

如果指定，则为分发 XML 添加深色水纹（Dark Aqua）主题的背景。该值包含 Resources 目录中图像的路径。

CPACK_PRODUCTBUILD_BACKGROUND_DARKAQUA_ALIGNMENT¶

与 CPACK_PRODUCTBUILD_BACKGROUND_ALIGNMENT 选项功能相同，但用于深色主题。

CPACK_PRODUCTBUILD_BACKGROUND_DARKAQUA_SCALING¶

与 CPACK_PRODUCTBUILD_BACKGROUND_SCALING 选项功能相同，但用于深色主题。

CPACK_PRODUCTBUILD_BACKGROUND_DARKAQUA_MIME_TYPE¶

与 CPACK_PRODUCTBUILD_BACKGROUND_MIME_TYPE 选项功能相同，但用于深色主题。

CPACK_PRODUCTBUILD_BACKGROUND_DARKAQUA_UTI¶

与 CPACK_PRODUCTBUILD_BACKGROUND_UTI 选项功能相同，但用于深色主题。

分发 XML 模板¶

CPack 使用模板文件来生成此包生成器内部使用的 distribution.dist 文件。通常 CMake 会提供该模板文件，但项目可以通过将名为 CPack.distribution.dist.in 的文件放置在 CMAKE_MODULE_PATH 变量列出的目录中来提供自己的模板。随后，CPack 将选用项目的模板文件，而不是使用自己的。

distribution.dist 文件是通过执行类似于 configure_file() 命令的替换操作来生成的。CPack 运行时设置的任何变量都可以使用通常的 @...@ 形式进行替换。以下变量也会在内部设置并可用于替换：

CPACK_RESOURCE_FILE_LICENSE_NOPATH

与 CPACK_RESOURCE_FILE_LICENSE 相同，但不包含路径。指定的文件将在与生成的 distribution.dist 文件相同的目录下可用。

CPACK_RESOURCE_FILE_README_NOPATH

与 CPACK_RESOURCE_FILE_README 相同，但不包含路径。指定的文件将在与生成的 distribution.dist 文件相同的目录下可用。

CPACK_RESOURCE_FILE_WELCOME_NOPATH

与 CPACK_RESOURCE_FILE_WELCOME 相同，但不包含路径。指定的文件将在与生成的 distribution.dist 文件相同的目录下可用。

CPACK_APPLE_PKG_INSTALLER_CONTENT

在版本 3.23 中添加。

这包含了指定安装程序范围选项（包括域详情）、默认背景和选项大纲的所有 XML 元素。

CPACK_PACKAGEMAKER_CHOICES

自 3.23 版本起弃用。

这仅包含指定默认背景和选项大纲的 XML 元素。它不包括安装程序范围的选项或任何域详情。请改用 CPACK_APPLE_PKG_INSTALLER_CONTENT。
