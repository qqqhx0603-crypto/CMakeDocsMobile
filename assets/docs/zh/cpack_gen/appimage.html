CPack AppImage 生成器¶

版本 4.2 中添加。

CPack AppImage 生成器支持将应用程序打包为 AppImage 格式。它使用 appimagetool 来打包应用程序，并使用 patchelf 将应用程序的 RPATH 设置为基于 AppImage 挂载位置的相对路径。

appimagetool 不会扫描库依赖项。它仅打包已安装的内容，并检查所提供的 .desktop 文件是否创建正确。为了获得最佳兼容性，建议在旧版本的 LTS 发行版上进行构建，并将任何依赖项包含在生成的文件中。

下面的代码片段可以添加到您的 CMakeLists.txt 文件中。将 my_application_target 替换为您的应用程序目标。该示例将尽最大努力识别您的应用程序链接到的库，并将它们复制到安装位置。

install(CODE [[
    file(GET_RUNTIME_DEPENDENCIES
        EXECUTABLES $<TARGET_FILE:my_application_target>
        RESOLVED_DEPENDENCIES_VAR resolved_deps
    )

    foreach(dep ${resolved_deps})
        # copy the symlink
        file(COPY ${dep} DESTINATION ${CMAKE_INSTALL_PREFIX}/lib)

        # Resolve the real path of the dependency (follows symlinks)
        file(REAL_PATH ${dep} resolved_dep_path)

        # Copy the resolved file to the destination
        file(COPY ${resolved_dep_path} DESTINATION ${CMAKE_INSTALL_PREFIX}/lib)
    endforeach()
]])

CPack AppImage 生成器将根据提供的 .desktop 条目生成一个默认的 AppRun。或者，如果安装了自定义的 AppRun（例如通过 install() 命令），它将被使用，而不是生成的文件。

对于基于 Qt 的项目，建议调用 qt_generate_deploy_app_script() 或 qt_generate_deploy_qml_app_script() 并安装脚本生成的文件。这将安装所需的 Qt 插件。

您还必须设置 CPACK_PACKAGE_ICON，其值必须与 Desktop 文件中列出的值相同。

CPack AppImage 生成器特有的变量¶

CPACK_APPIMAGE_TOOL_EXECUTABLE¶

appimagetool 可执行文件的名称。如果未提供绝对路径，则内部将基于 find_program() 的逻辑使用此值来查找可执行文件。

默认值：

appimagetool

CPACK_APPIMAGE_PATCHELF_EXECUTABLE¶

patchelf 可执行文件的名称。如果未提供绝对路径，则内部将基于 find_program() 的逻辑使用此值来查找可执行文件。

默认值：

patchelf

CPACK_APPIMAGE_DESKTOP_FILE¶

要安装的 freedesktop.org 桌面文件的名称。如果未指定，将使用在要安装的文件列表中找到的第一个 .desktop 文件。包必须包含一个有效的 .desktop 文件，并且它必须包含一个与 CPACK_PACKAGE_ICON（不带文件扩展名）匹配的 Icon 条目。图标的实际安装位置应遵循 freedesktop.org 规范。

默认值：

未设置

CPACK_APPIMAGE_UPDATE_INFORMATION¶

将此变量的值作为更新信息嵌入。有关 --updateinformation 选项支持的值和格式，请参阅 appimagetool 源代码。如果使用 zsync 更新信息，强烈建议安装 zsyncmake 工具。

默认值：

未设置

CPACK_APPIMAGE_GUESS_UPDATE_INFORMATION¶

当此变量为 true 时，在调用 appimagetool 时添加 --guess 选项。这会指导该工具尝试根据 GitHub 或 GitLab 环境变量猜测适当的更新信息。

默认值：

未设置

CPACK_APPIMAGE_COMPRESSOR¶

覆盖 appimagetool 的默认 squashfs 压缩类型（zstd）。这对应于 appimagetool --comp 选项。

默认值：

未设置

CPACK_APPIMAGE_MKSQUASHFS_OPTIONS¶

传递给 mksquashfs 的参数列表。在 appimagetool 命令行中，每个参数前面都会加上 --mksquashfs-opt。

默认值：

未设置

CPACK_APPIMAGE_NO_APPSTREAM¶

如果设置为 true，则不检查 AppStream 元数据。这会将 --no-appstream 选项传递给 appimagetool。

默认值：

未设置

CPACK_APPIMAGE_EXCLUDE_FILE¶

除了 .appimageignore 之外，使用指定的文件作为 mksquashfs 的排除文件。这使用了 appimagetool 的 --exclude-file 选项。

默认值：

未设置

CPACK_APPIMAGE_RUNTIME_FILE¶

指定一个运行时文件以使用，而不是让 appimagetool 下载一个运行时来嵌入生成的 AppImage 中。

默认值：

未设置

CPACK_APPIMAGE_SIGN¶

当设置为 true 时，使用 gpg[2] 对生成的 AppImage 进行签名。如果使用此选项，还应指定 CPACK_APPIMAGE_SIGN_KEY。

默认值：

未设置

CPACK_APPIMAGE_SIGN_KEY¶

当使用 CPACK_APPIMAGE_SIGN 启用签名时，用于 gpg[2] 签名的密钥 ID。

默认值：

未设置
