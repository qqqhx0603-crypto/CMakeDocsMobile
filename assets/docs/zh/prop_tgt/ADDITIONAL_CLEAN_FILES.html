ADDITIONAL_CLEAN_FILES¶

版本 3.15 新增。

一个 分号分隔的列表，包含在全局 clean 目标中将被删除的文件或目录。它可以用来指定作为构建目标的一部分生成的文件和目录，或者以某种方式与目标直接相关的文件和目录（例如，作为运行目标的结果而创建）。

对于自定义目标，如果这些文件可以作为输出或副产品捕获，那么应该优先于将其添加到此属性。如果额外的清理文件被多个目标使用或不是目标特定的，那么 ADDITIONAL_CLEAN_FILES 目录属性可能是更合适的属性。

允许使用相对路径，并且这些路径将相对于当前二进制目录进行解释。

ADDITIONAL_CLEAN_FILES 的内容可以使用 生成器 表达式。

此属性仅适用于 Ninja（构建工具） 和 Makefile（构建脚本） 生成器。它会被其他生成器忽略。
