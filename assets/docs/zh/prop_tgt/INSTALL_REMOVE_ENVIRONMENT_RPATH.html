INSTALL_REMOVE_ENVIRONMENT_RPATH¶

3.16 版新增。

控制在安装过程中是否移除工具链定义的 rpath。

当目标正在被安装时，CMake 可能需要重写其 rpath 信息。当安装 rpath（如 INSTALL_RPATH 目标属性所指定）的内容与目标构建时使用的 rpath 不同时，就会发生这种情况。某些工具链会在构建过程中将自己的 rpath 内容插入到二进制文件中。默认情况下，CMake 会在安装 rpath 中保留这些额外插入的内容。对于需要丢弃此类工具链插入条目在安装期间的场景，请将 INSTALL_REMOVE_ENVIRONMENT_RPATH 目标属性设置为 true。

当目标创建时，此属性由 CMAKE_INSTALL_REMOVE_ENVIRONMENT_RPATH 的值初始化。
