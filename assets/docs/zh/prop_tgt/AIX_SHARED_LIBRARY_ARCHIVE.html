AIX_SHARED_LIBRARY_ARCHIVE¶

在版本 3.31 中添加。

在 AIX 上，启用或禁用为 SHARED 库目标创建共享库存档。

  • 如果启用，共享对象 .so 文件将被放置在一个存档 .a 文件中。这是 AIX 上的首选约定。

    存档中的共享对象名称会编码来自 SOVERSION 目标属性的版本信息（如果已设置），否则会编码来自 VERSION 目标属性的版本信息（如果已设置）。

  • 如果禁用，将生成一个纯粹的共享对象 .so 文件。这与其他 UNIX 平台一致。

此属性默认为 CMAKE_AIX_SHARED_LIBRARY_ARCHIVE，如果该变量在通过 add_library() 创建非导入的 SHARED 库目标时已设置。导入的目标必须显式启用 AIX_SHARED_LIBRARY_ARCHIVE，如果它们导入了 AIX 共享库存档。

版本 4.0 中更改：对于非导入的目标，如果未设置此属性，则默认值为 *启用*。请参阅策略 CMP0182。

在 CMake 3.31 中，策略 CMP0182 不存在，因此默认值为 *禁用*。

在 CMake 3.30 及更低版本中，AIX_SHARED_LIBRARY_ARCHIVE 不存在，因此默认值为 *禁用*。
