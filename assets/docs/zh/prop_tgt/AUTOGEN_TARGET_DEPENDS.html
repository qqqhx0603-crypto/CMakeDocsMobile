AUTOGEN_TARGET_DEPENDS¶

对应 <ORIGIN>_autogen 目标的额外目标依赖项。

注意

如果使用 Qt 5.15 或更高版本，且生成器为 Ninja 或 Makefile（Make 构建脚本） 生成器，则额外的目标依赖项会被添加到 <ORIGIN>_autogen_timestamp_deps 目标中，而不是 <ORIGIN>_autogen 目标中。

将 AUTOMOC 或 AUTOUIC 属性设置为 ON 的目标，都有一个对应的用于生成 moc 和 uic 文件的 <ORIGIN>_autogen 目标。由于此 <ORIGIN>_autogen 目标是在生成时（generate-time）创建的，因此无法使用例如 add_dependencies() 来定义其依赖关系。取而代之的是，可以将 AUTOGEN_TARGET_DEPENDS 目标属性设置为一个以分号分隔的列表，包含 <ORIGIN>_autogen 目标的额外依赖项。依赖项可以是目标名称或文件名。

总之，<ORIGIN>_autogen 目标的依赖项由以下部分组成：

  • 转发的源目标依赖项（默认通过 AUTOGEN_ORIGIN_DEPENDS 启用）

  • 来自 AUTOGEN_TARGET_DEPENDS 的用户自定义额外依赖项

有关将 CMake 与 Qt 结合使用的更多信息，请参阅 cmake-qt(7) 手册。

使用场景¶

如果 AUTOMOC 或 AUTOUIC 依赖于以下文件：

  • 一个 GENERATED 的非 C++ 文件（例如 GENERATED 的 .json 或 .ui 文件），或者

  • 一个 GENERATED 的 C++ 文件，但该文件未被 AUTOMOC 和 AUTOUIC 识别，因为它被 SKIP_AUTOMOC、SKIP_AUTOUIC、SKIP_AUTOGEN 或 CMP0071 跳过，或者

  • 一个不在源目标源代码列表中的文件

则必须将其添加到 AUTOGEN_TARGET_DEPENDS 中。
