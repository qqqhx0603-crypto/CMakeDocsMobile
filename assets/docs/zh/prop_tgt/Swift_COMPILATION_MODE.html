Swift_COMPILATION_MODE¶

在版本 3.29 中添加。

指定 Swift 编译目标的方式。

允许的值为

incremental

将模块中的每个 Swift 源代码文件单独编译，从而在构建过程中实现更好的并行性。编译器会将额外信息输出到构建目录，从而在源代码在重建之间进行小幅更改时提高重建性能。这是在项目更改期间迭代的最佳选项。

wholemodule

全模块优化编译速度最慢，但会生成最优化过的库。整个上下文加载到编译器的单个实例中，因此模块中的源代码文件之间没有并行性。

singlefile

将 Swift 模块中的每个源代码文件单独编译，从而实现更好的并行性。与 incremental 构建模式不同，编译器在构建过程中不会输出任何额外信息，因此在对源代码文件进行小幅更改后的重建速度不会更快。除非是为了解决编译器错误，否则应谨慎使用此选项，优先使用 incremental 构建。

使用 生成器表达式 来支持按配置指定。例如，以下代码

add_library(foo foo.swift)
set_property(TARGET foo PROPERTY
  Swift_COMPILATION_MODE "$<IF:$<CONFIG:Release>,wholemodule,incremental>")

在发布配置中将 Swift 编译模式设置为 wholemodule 模式，并在其他配置中将属性设置为 incremental 模式。

如果已设置，则该属性将从 CMAKE_Swift_COMPILATION_MODE 变量的值进行初始化。如果未设置该属性或为空，则 CMake 将默认值 incremental 用于指定 swift 编译模式。

注意

只有当策略 CMP0157 在第一个启用 Swift 语言的 project() 或 enable_language() 命令之前设置为 NEW 时，此属性才有效。
