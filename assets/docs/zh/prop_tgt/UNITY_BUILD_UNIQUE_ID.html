UNITY_BUILD_UNIQUE_ID¶

在 3.20 版本中添加。

在 unity 构建期间设置为每个文件唯一的有效 C 标识符的名称。

当此属性被填充并且当 UNITY_BUILD 为 true 时，该属性值用于定义指定名称的编译器定义。 定义的符号的值未指定，但对于每个文件路径都是唯一的。

给定

set_target_properties(myTarget PROPERTIES
  UNITY_BUILD "ON"
  UNITY_BUILD_UNIQUE_ID "MY_UNITY_ID"
)

代码说明

  • set_target_properties(myTarget PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • UNITY_BUILD "ON"：这一行继续补充当前命令或表达式的内容。

  • UNITY_BUILD_UNIQUE_ID "MY_UNITY_ID"：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

符号 MY_UNITY_ID 被定义为每个文件唯一的数值。

此标识符的一个已知用例是在有限范围内消除匿名命名空间中的变量歧义。 匿名命名空间为 unity 构建带来问题，因为它们用于确保某些变量和声明限定于一个翻译单元，这近似于单个源文件。 当源文件在 unity 构建文件中组合时，来自不同文件的这些变量被组合到一个翻译单元中，并且名称会冲突。 可以使用此属性通过以下代码避免这种情况

// Needed for when unity builds are disabled
#ifndef MY_UNITY_ID
#define MY_UNITY_ID
#endif

namespace { namespace MY_UNITY_ID {
  // The name 'i' clashes (or could clash) with other
  // variables in other anonymous namespaces
  int i = 42;
}}

int use_var()
{
  return MY_UNITY_ID::i;
}

代码说明

  • #ifndef MY_UNITY_ID：这一行开始预处理条件分支，按宏条件决定后续代码是否参与编译。

  • #define MY_UNITY_ID：这一行定义预处理宏，供后续代码在编译前展开使用。

  • #endif：这一行切换或结束预处理条件分支。

  • namespace { namespace MY_UNITY_ID {：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • // The name 'i' clashes (or could clash) with other：这是代码注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • // variables in other anonymous namespaces：这是代码注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • int i = 42;：这一行完成赋值或初始化，把右侧结果写入左侧变量；变量说明：i：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；运算符：= 表示赋值或键值关联。

  • }}：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

伪匿名命名空间在真正的匿名命名空间中使用。 在许多平台上，这保持了内部符号在执行 unity 构建时不具有外部链接的不变性。
