XCODE_LINK_BUILD_PHASE_MODE¶

3.19 版本新增。

当使用 Xcode 生成器时，需要链接的库将在 Xcode 项目文件中通过“Link Binary With Libraries”（使用库链接二进制文件）构建阶段或直接作为链接器标志来指定。前者允许 Xcode 管理构建路径，这在创建 Xcode 归档（archive）时可能是必要的，因为它可能使用与常规构建不同的构建路径。

此属性控制针对属于应用程序包、可执行文件、共享库、共享框架或模块库的目标，是否使用“Link Binary With Libraries”构建阶段。

可能的值为

  • NONE 库将通过直接指定链接器标志的方式进行链接。

  • BUILT_ONLY 在满足以下条件时，“Link Binary With Libraries”构建阶段将用于链接到另一个目标：

    • 被链接的目标是一个常规的非导入（non-imported）、非接口（non-interface）库目标。

    • 正在构建的目标的输出目录未更改其默认值（参见 RUNTIME_OUTPUT_DIRECTORY 和 LIBRARY_OUTPUT_DIRECTORY）。

  • KNOWN_LOCATION 在满足 BUILT_ONLY 的相同条件，以及以下额外条件时，“Link Binary With Libraries”构建阶段将用于链接到另一个目标：

    • 除 UNKNOWN 类型之外的导入库目标。

    • 任何直接通过路径指定的非目标库。

对于所有其他情况，库将通过直接指定链接器标志的方式进行链接。

警告

使用“Link Binary With Libraries”链接的库，其链接顺序排在通过常规链接器标志链接的库之后。当不同的静态库包含同名符号时，应考虑此顺序，因为前者将优先于后者。

警告

如果两个或多个目录包含文件名相同的库，且从这些目录中链接了部分库，则库搜索路径查找最终将链接来自第一个目录的库。这是 Xcode 的已知限制。

如果在创建目标时设置了 CMAKE_XCODE_LINK_BUILD_PHASE_MODE 变量，则此属性将由该变量的值进行初始化。
