资源¶

在 FRAMEWORK 或 BUNDLE 中指定资源文件。

带有 FRAMEWORK 或 BUNDLE 属性的目标会生成框架或应用程序包（macOS（苹果桌面系统） 和 iOS 均受支持），或者在其他平台上生成普通的共享库。可以将此属性设置为要放置在相应目录（例如 macOS（苹果桌面系统） 的 Resources 目录）内的文件列表。在非 Apple 平台上，可以使用 install(TARGETS) 命令的 RESOURCE 选项来安装这些文件。

应用程序包示例

add_executable(ExecutableTarget
  addDemo.c
  resourcefile.txt
  appresourcedir/appres.txt)

target_link_libraries(ExecutableTarget heymath mul)

set(RESOURCE_FILES
  resourcefile.txt
  appresourcedir/appres.txt)

set_target_properties(ExecutableTarget PROPERTIES
  MACOSX_BUNDLE TRUE
  MACOSX_FRAMEWORK_IDENTIFIER org.cmake.ExecutableTarget
  RESOURCE "${RESOURCE_FILES}")

代码说明

  • add_executable(ExecutableTarget：这一行继续补充当前命令或表达式的内容。

  • addDemo.c：这一行继续补充当前命令或表达式的内容。

  • resourcefile.txt：这一行继续补充当前命令或表达式的内容。

  • appresourcedir/appres.txt)：这一行继续补充当前命令或表达式的内容。

  • target_link_libraries(ExecutableTarget heymath mul)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • set(RESOURCE_FILES：这一行继续补充当前命令或表达式的内容；变量说明：RESOURCE_FILES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • resourcefile.txt：这一行继续补充当前命令或表达式的内容。

  • appresourcedir/appres.txt)：这一行继续补充当前命令或表达式的内容。

  • set_target_properties(ExecutableTarget PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • MACOSX_BUNDLE TRUE：这一行继续补充当前命令或表达式的内容。

  • MACOSX_FRAMEWORK_IDENTIFIER org.cmake.ExecutableTarget：这一行继续补充当前命令或表达式的内容。

  • RESOURCE "${RESOURCE_FILES}")：这一行继续补充当前命令或表达式的内容；变量说明：RESOURCE_FILES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。
