LINK_LIBRARIES_STRATEGY¶

在版本 3.31 中添加。

指定目标在链接器命令行上对其直接链接依赖项进行排序的策略。如果在创建目标时设置了 CMAKE_LINK_LIBRARIES_STRATEGY 变量，则此属性将由该变量的值初始化。

CMake 使用其 目标链接属性 生成目标的链接行。特别是，LINK_LIBRARIES 目标属性记录了目标的直接链接依赖项，通常通过调用 target_link_libraries() 来填充。间接链接依赖项是通过跟踪 INTERFACE_LINK_LIBRARIES 属性的传递闭包，从 LINK_LIBRARIES 中命名库目标的条目传播而来的。CMake 支持多种策略来对直接和间接链接依赖项进行名义排序，随后会针对 工具链特定行为 进行过滤。

请参考以下策略示例

add_library(A STATIC ...)
add_library(B STATIC ...)
add_library(C STATIC ...)
add_executable(main ...)
target_link_libraries(B PRIVATE A)
target_link_libraries(C PRIVATE A)
target_link_libraries(main PRIVATE A B C)

代码说明

  • add_library(A STATIC ...)：调用 add_library() 命令，定义一个库目标。

  • add_library(B STATIC ...)：调用 add_library() 命令，定义一个库目标。

  • add_library(C STATIC ...)：调用 add_library() 命令，定义一个库目标。

  • add_executable(main ...)：调用 add_executable() 命令，定义一个可执行目标。

  • target_link_libraries(B PRIVATE A)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • target_link_libraries(C PRIVATE A)：调用 target_link_libraries() 命令，为目标链接依赖库。

  • target_link_libraries(main PRIVATE A B C)：调用 target_link_libraries() 命令，为目标链接依赖库。

支持的策略包括

REORDER_MINIMALLY

LINK_LIBRARIES 的条目始终首先出现，并保持其原始顺序。未被原始条目满足的间接链接依赖项可能会在彼此之间进行重新排序和去重，但始终会附加在原始条目之后。这可能会导致链接行效率较低，但允许项目控制独立条目之间的排序。当混合使用链接标志与库，或者当多个库提供给定符号时，这种控制可能非常重要。

这是默认策略。

在上面的示例中，此策略通过从原始条目 A B C 开始，然后附加另一个 A 来满足 B 和 C 对 A 的依赖，从而为 main 计算链接行。此策略产生的名义顺序为 A B C A。

请注意，针对 工具链特定行为 的额外过滤可能会在生成的构建系统中的实际链接器调用中对 A 进行去重，最终结果为 A B C 或 B C A。

REORDER_FREELY

LINK_LIBRARIES 的条目可能会被重新排序、去重，并与间接链接依赖项混合。这可能会产生更高效的链接行，但不会赋予项目任何对独立条目之间排序的控制权。

在上面的示例中，此策略通过重新排列其原始条目 A B C 以满足 B 和 C 对 A 的依赖，从而为 main 计算链接行。此策略产生的名义顺序为 B C A。

工具链特定行为¶

在上述策略之一生成直接和间接链接依赖项之间的名义顺序后，生成的构建系统中的实际链接器调用可能会根据平台特定要求和链接器功能对条目进行去重。请参阅策略 CMP0156。

例如，如果 REORDER_MINIMALLY 策略产生 A B C A，则实际链接行可能会按如下方式对 A 进行去重

  • 如果 A 是一个静态库且链接器会自动重新扫描，则保留第一次出现的位置，结果为 A B C。请参阅策略 CMP0179

  • 如果 A 是 Windows（微软桌面系统） 上的共享库，则保留第一次出现的位置，结果为 A B C。

  • 如果 A 是 macOS（苹果桌面系统） 或 UNIX 平台上的共享库，则保留最后一次出现的位置，结果为 B C A。
