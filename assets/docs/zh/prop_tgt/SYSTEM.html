SYSTEM¶

在 3.25 版本中新增。

指定目标为系统目标。这会产生以下影响：

  • INTERFACE_INCLUDE_DIRECTORIES 的条目在编译使用者时会被视为系统包含目录。INTERFACE_SYSTEM_INCLUDE_DIRECTORIES 的条目不受影响，并将始终被视为系统包含目录。

  • 在 Apple 平台上，如果 FRAMEWORK 目标属性为真，则框架目录将被视为系统目录。

对于导入目标（imported targets（目标）），此属性默认为真，这意味着默认情况下，它们的 INTERFACE_INCLUDE_DIRECTORIES 以及（如果 FRAMEWORK 目标属性为真时的）框架目录，都会被视为系统目录。如果它们的 SYSTEM 属性为假，则它们的 INTERFACE_INCLUDE_DIRECTORIES 以及框架目录将不会被视为系统目录。使用 EXPORT_NO_SYSTEM 属性可以更改目标在安装时如何设置其 SYSTEM 属性。

对于非导入目标（non-imported targets（目标）），此目标属性在创建目标时由 SYSTEM 目录属性初始化。
