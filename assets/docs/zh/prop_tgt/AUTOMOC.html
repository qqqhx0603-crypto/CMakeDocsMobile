AUTOMOC¶

是否应使用自动 moc 处理目标（针对 Qt 项目）。

AUTOMOC 是一个布尔值，指定 CMake 是否会自动处理 Qt moc 预处理器，即无需使用诸如 qt4_wrap_cpp()、qt5_wrap_cpp() 等命令。目前支持 Qt 4 到 6 版本。

如果创建目标时设置了 CMAKE_AUTOMOC 变量，则此属性将由该变量的值进行初始化。

当此属性设置为 ON 时，CMake 会在构建时扫描头文件和源文件，并相应地调用 moc。

头文件处理¶

在配置时，根据目标的源文件计算应由 AUTOMOC 扫描的头文件列表。

  • 目标源文件中的所有头文件都会添加到扫描列表中。

  • 对于目标源文件中的所有 C++ 源文件 <source_base>.<source_extension>，CMake 会搜索：

    • 具有相同基本名称的常规头文件（<source_base>.<header_extension>），以及

    • 具有相同基本名称且带有 _p 后缀的私有头文件（<source_base>_p.<header_extension>），

    并将它们添加到扫描列表中。

  • AUTOMOC_INCLUDE_DIRECTORIES 目标属性可以显式告知 moc 要搜索哪些包含目录。如果未设置，默认使用目标及其依赖项传递闭包中的包含目录。

在构建时，CMake 会扫描列表中每个未知或修改过的头文件，并搜索：

  • 来自 AUTOMOC_MACRO_NAMES 的 Qt 宏，

  • Q_PLUGIN_METADATA 宏的 FILE 参数产生的附加文件依赖项，以及

  • 由 AUTOMOC_DEPEND_FILTERS 中定义的过滤器检测到的附加文件依赖项。

如果找到了 Qt 宏，则头文件将通过 moc 编译为输出文件 moc_<base_name>.cpp。完整的输出文件路径在 输出文件位置 一节中描述。

如果附加文件依赖项中的文件发生更改，该头文件将被 moc 重新编译。

头文件的 moc 输出文件 moc_<base_name>.cpp 可以包含在源文件中。在 在源文件中包含头文件的 moc 文件 一节中有关于该主题的更多信息。

源文件处理¶

在构建时，CMake 会扫描目标源文件中每个未知或已修改的 C++ 源文件，以寻找：

  • 来自 AUTOMOC_MACRO_NAMES 的 Qt 宏，

  • 对头文件 moc 文件的包含（参见 在源文件中包含头文件的 moc 文件），

  • Q_PLUGIN_METADATA 宏的 FILE 参数产生的附加文件依赖项，以及

  • 由 AUTOMOC_DEPEND_FILTERS 中定义的过滤器检测到的附加文件依赖项。

如果找到了 Qt 宏，则期望 C++ 源文件 <base>.<source_extension> 也包含一条 include 语句

#include <<base>.moc> // or
#include "<base>.moc"

然后该源文件将通过 moc 编译为输出文件 <base>.moc。关于完整输出文件路径的描述请见 输出文件位置 一节。

如果附加文件依赖项中的文件发生更改，该源文件将被 moc 重新编译。

在源文件中包含头文件的 moc 文件¶

源文件可以通过使用以下形式的 include 语句来包含头文件 <header_base>.<header_extension> 的 moc 输出文件：

#include <moc_<header_base>.cpp> // or
#include "moc_<header_base>.cpp"

如果源文件包含了头文件的 moc 输出文件，它将生成在与不包含时不同的位置。这在 输出文件位置 一节中有所描述。

输出文件位置¶

被包含的 moc 输出文件¶

由源文件包含的 moc 输出文件将生成在

  • 单配置生成器的 <AUTOGEN_BUILD_DIR>/include 目录下，或者

  • 多配置 生成器的 <AUTOGEN_BUILD_DIR>/include_<CONFIG> 目录下。

其中 <AUTOGEN_BUILD_DIR> 是目标属性 AUTOGEN_BUILD_DIR 的值。

该包含目录会自动添加到目标的 INCLUDE_DIRECTORIES 中。

未被包含的 moc 输出文件¶

未在源文件中包含的 moc 输出文件将生成在

  • 单配置生成器的 <AUTOGEN_BUILD_DIR>/<SOURCE_DIR_CHECKSUM> 目录下，或者

  • 多配置 生成器的 <AUTOGEN_BUILD_DIR>/include_<CONFIG>/<SOURCE_DIR_CHECKSUM> 目录下。

其中 <SOURCE_DIR_CHECKSUM> 是从 moc 输入文件的相对父目录路径计算出的校验和。此方案允许在不同目录中存在同名的 moc 输入文件。

所有未被包含的 moc 输出文件将自动由 CMake 生成的文件包含：

  • <AUTOGEN_BUILD_DIR>/mocs_compilation.cpp，或

  • <AUTOGEN_BUILD_DIR>/mocs_compilation_$<CONFIG>.cpp,

该文件会被添加到目标的源文件中。

Qt 版本检测¶

启用 AUTOMOC 的目标需要知道它们正在使用的 Qt 主版本号和次版本号。主版本号通常由目标链接的 Qt[456]Core 库的 INTERFACE_QT_MAJOR_VERSION 属性提供。为了查找次版本号，CMake 会从以下位置构建可用 Qt 版本列表：

  • Qt6Core_VERSION_MAJOR 和 Qt6Core_VERSION_MINOR 变量（通常由 find_package(Qt6...) 设置）

  • Qt6Core_VERSION_MAJOR 和 Qt6Core_VERSION_MINOR 目录属性

  • Qt5Core_VERSION_MAJOR 和 Qt5Core_VERSION_MINOR 变量（通常由 find_package(Qt5...) 设置）

  • Qt5Core_VERSION_MAJOR 和 Qt5Core_VERSION_MINOR 目录属性

  • QT_VERSION_MAJOR 和 QT_VERSION_MINOR 变量（通常由 find_package(Qt4...) 设置）

  • QT_VERSION_MAJOR 和 QT_VERSION_MINOR 目录属性

在 add_executable() 或 add_library() 调用的上下文中。

假设 INTERFACE_QT_MAJOR_VERSION 是一个有效数字，则取列表中第一个匹配主版本号的条目。如果没有找到匹配的主版本号，则会产生错误。如果 INTERFACE_QT_MAJOR_VERSION 不是有效数字，则取列表中的第一个条目。

find_package(Qt[456]...) 调用会设置 QT/Qt[56]Core_VERSION_MAJOR/MINOR 变量。如果该调用位于与 add_executable() 或 add_library() 调用不同的上下文中（例如在函数内），则版本变量可能对启用了 AUTOMOC 的目标不可用。在这种情况下，版本变量可以作为目录属性从 find_package(Qt[456]...) 的调用上下文转发到 add_executable() 或 add_library() 的调用上下文中。以下 Qt5 示例展示了该过程。

function (add_qt5_client)
  find_package(Qt5 REQUIRED QUIET COMPONENTS Core Widgets)
  ...
  set_property(DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}"
    PROPERTY Qt5Core_VERSION_MAJOR "${Qt5Core_VERSION_MAJOR}")
  set_property(DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}"
    PROPERTY Qt5Core_VERSION_MINOR "${Qt5Core_VERSION_MAJOR}")
  ...
endfunction ()
...
add_qt5_client()
add_executable(myTarget main.cpp)
target_link_libraries(myTarget Qt5::QtWidgets)
set_property(TARGET myTarget PROPERTY AUTOMOC ON)

修饰符¶

AUTOMOC_EXECUTABLE: moc 可执行文件将自动检测，但可以使用此目标属性强制指定为特定的二进制文件。

AUTOMOC_MOC_OPTIONS: 可在此目标属性中设置 moc 的其他命令行选项。

AUTOMOC_MACRO_NAMES: 此 Qt 宏名称列表可以扩展，以便在头文件和源文件中搜索其他宏。

AUTOMOC_DEPEND_FILTERS: 通过在此目标属性中定义文件名过滤器，可以从头文件或源文件中提取 moc 依赖文件名。

AUTOMOC_COMPILER_PREDEFINES: moc 的编译器预定义会被写入 moc_predefs.h 文件。此文件的生成可以在此目标属性中启用或禁用。

AUTOMOC_INCLUDE_DIRECTORIES: 指定一个或多个包含目录，供 AUTOMOC 显式传递给 moc，而不是自动发现目标的包含目录。

SKIP_AUTOMOC: 通过设置此源文件属性，可以从 AUTOMOC 处理中排除源文件和头文件。

SKIP_AUTOGEN: 通过设置此源文件属性，可以从 AUTOMOC、AUTOUIC 和 AUTORCC 处理中排除源文件。

AUTOGEN_SOURCE_GROUP: 此全局属性可用于在 IDE（如 MSVS）中将由 AUTOMOC 或 AUTORCC 生成的文件归为一组。

AUTOGEN_TARGETS_FOLDER: 此全局属性可用于在 IDE（如 MSVS）中将 AUTOMOC、AUTOUIC 和 AUTORCC 目标归为一组。

CMAKE_GLOBAL_AUTOGEN_TARGET: 当此变量为 ON 时，将生成一个全局 autogen 目标，该目标依赖于项目中所有 AUTOMOC 或 AUTOUIC 生成的 <ORIGIN>_autogen 目标。

AUTOGEN_PARALLEL: 此目标属性控制构建期间并行启动的 moc 或 uic 进程的数量。

AUTOGEN_COMMAND_LINE_LENGTH_MAX: 此目标属性控制在 Windows 上何时对 moc 或 uic 进程使用响应文件的限制。

有关将 CMake 与 Qt 结合使用的更多信息，请参阅 cmake-qt(7) 手册。
