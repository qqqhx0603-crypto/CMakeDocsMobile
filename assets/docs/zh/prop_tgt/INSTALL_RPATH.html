INSTALL_RPATH¶

用于已安装目标的 rpath。

默认情况下，安装 rpath 为空。可以使用此属性进行设置，它是一个分号分隔的列表，用于指定安装目标中使用的 rpath（针对支持它的平台）。当创建目标时，如果设置了 CMAKE_INSTALL_RPATH 变量，此属性将由该变量的值进行初始化。除了手动设置安装 rpath 外，还可以通过 INSTALL_RPATH_USE_LINK_PATH 目标属性让 CMake 自动生成它。

通常，CMake 在使用 RPATH 的系统上构建可执行文件等内容时会使用构建树的 RPATH，请参阅 BUILD_RPATH 目标属性。当安装软件时，CMake 会编辑（或重新链接）目标（参见 CMAKE_NO_BUILTIN_CHRPATH），使其具有安装 RPATH。可以通过 BUILD_WITH_INSTALL_RPATH 目标属性来避免这种安装过程中的编辑。

对于处理依赖于工具链的 RPATH 条目，可以使用 INSTALL_REMOVE_ENVIRONMENT_RPATH。可以通过 CMAKE_SKIP_RPATH 变量完全禁用运行时路径。

由于 rpath 可能包含与 CMake 语法冲突的 ${ORIGIN}，因此 INSTALL_RPATH 的内容会在 cmake_install.cmake 脚本中进行适当的转义（参见策略 CMP0095）。

此属性支持 生成器表达式。

在 Apple 平台上，请参考 INSTALL_NAME_DIR 目标属性。在 Windows（微软桌面系统） 下，与此相关的是 TARGET_RUNTIME_DLL（动态链接库）S 生成器表达式。

上一主题

INSTALL_REMOVE_ENVIRONMENT_RPATH

下一主题

INSTALL_RPATH_USE_LINK_PATH
