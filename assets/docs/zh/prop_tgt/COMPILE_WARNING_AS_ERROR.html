COMPILE_WARNING_AS_ERROR¶

在 3.24 版本中添加。

指定是否将编译时的警告视为错误。如果启用，则添加一个标志，将编译时的警告视为错误。如果在使用 cmake --compile-no-warning-as-error 选项的 cmake(1) 命令时提供了该选项，则此属性将被忽略。

此属性并非对所有编译器都已实现。如果所用编译器没有实现，则会静默忽略此属性。当前已实现的 编译器 ID 包括

  • GNU

  • Clang

  • AppleClang

  • Fujitsu

  • FujitsuClang

  • IBMClang

  • Intel

  • IntelLLVM

  • LCC

  • MSVC

  • NVHPC

  • NVIDIA (CUDA)

  • QCC

  • SunPro

  • Tasking

  • TI

  • VisualAge

  • XL

  • XLClang

此属性由变量 CMAKE_COMPILE_WARNING_AS_ERROR 的值初始化，如果该变量在创建目标时已设置。
