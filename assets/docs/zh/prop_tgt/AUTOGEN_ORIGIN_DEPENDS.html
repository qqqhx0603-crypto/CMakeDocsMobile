AUTOGEN_ORIGIN_DEPENDS¶

3.14 版新增。

用于将原始目标（origin target）的依赖项转发至相应 <ORIGIN>_autogen 目标的开关。

注意

如果使用 Qt 5.15 或更高版本，且生成器为 Ninja 或 Makefile（Make 构建脚本） 生成器，原始目标依赖项将转发至 <ORIGIN>_autogen_timestamp_deps 目标，而不是 <ORIGIN>_autogen。

将 AUTOMOC 或 AUTOUIC 属性设置为 ON 的目标，都有一个对应的 <ORIGIN>_autogen 目标，用于生成 moc 和 uic 文件。由于此 <ORIGIN>_autogen 目标是在生成时（generate-time）创建的，因此无法使用例如 add_dependencies() 来定义其依赖项。相反，AUTOGEN_ORIGIN_DEPENDS 目标属性决定了是否应将原始目标依赖项转发至 <ORIGIN>_autogen 目标。

默认情况下，AUTOGEN_ORIGIN_DEPENDS 由 CMAKE_AUTOGEN_ORIGIN_DEPENDS 初始化，后者默认值为 ON。

<ORIGIN>_autogen 目标的依赖项总共由以下部分组成：

  • 转发的原始目标依赖项（默认通过 AUTOGEN_ORIGIN_DEPENDS 启用）

  • 来自 AUTOGEN_TARGET_DEPENDS 的额外用户定义依赖项

有关将 CMake 与 Qt 结合使用的更多信息，请参阅 cmake-qt(7) 手册。

注意

禁用 AUTOGEN_ORIGIN_DEPENDS 有助于在仅构建 <ORIGIN>_autogen 目标时避免构建原始目标依赖项。当启用了 全局 autogen 目标 时，这一点尤其重要。

当 <ORIGIN>_autogen 目标不需要原始目标的所有依赖项，且 AUTOGEN_ORIGIN_DEPENDS 被禁用时，可能需要扩展 AUTOGEN_TARGET_DEPENDS 以添加缺失的依赖项。
