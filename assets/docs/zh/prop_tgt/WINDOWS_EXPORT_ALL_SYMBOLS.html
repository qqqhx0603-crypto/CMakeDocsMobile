WINDOWS_EXPORT_ALL_SYMBOLS¶

3.4 版本新增。

此属性仅在 Windows（微软桌面系统） 上的 MS 兼容工具中实现。

启用此布尔属性，将自动为 SHARED 库（或启用 ENABLE_EXPORTS 的可执行文件）的输入 .obj 文件中找到的所有全局符号创建一个模块定义 (.def) 文件。该模块定义文件将被传递给链接器，导致所有符号从 .dll 导出。

这简化了将项目移植到 Windows（微软桌面系统） 的过程，减少了对显式 dllexport 标记的需求，甚至在 C++ 类中也是如此。函数符号将自动导出，并可由调用者链接。但是，在消费者中编译代码时，在某些情况下可能需要显式 dllimport 标记

  • 全局数据符号必须使用 __declspec(dllimport) 显式标记，才能链接到 .dll 中的数据。

  • 在编译器生成对虚拟函数表的引用时，例如在具有虚拟函数的类的委托构造函数中，必须使用 __declspec(dllimport) 标记整个类，才能链接到 .dll 中的 vftable。

  • 有关数据符号的更多信息，请参阅 MSVC 链接器 /EXPORT 选项。

当此属性启用时，零个或多个 .def 文件也可以指定为目标的源文件。这些文件命名的导出将与从对象文件中检测到的导出合并，以生成传递给链接器的单个模块定义文件。这可用于导出 .dll 中不在其任何对象文件中的符号，但由链接器从依赖项（例如 msvcrt.lib）添加的符号。

当创建目标时，如果设置了 CMAKE_WINDOWS_EXPORT_ALL_SYMBOLS 变量，则此属性将由该变量的值初始化。
