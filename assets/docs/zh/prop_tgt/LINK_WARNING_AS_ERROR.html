LINK_WARNING_AS_ERROR¶

4.0 版本新增。

指定是否将链接时的警告视为错误。如果启用，则添加一个标志，将链接时的警告视为错误。如果在使用 cmake --link-no-warning-as-error 选项的 cmake(1) 命令时指定了该选项，则此属性将被忽略。

此属性接受以下值的分号分隔列表：分号分隔列表

  • LINKER：将链接器警告视为错误。

  • DRIVER：当用于驱动链接步骤时，将编译器警告视为错误。有关更多信息，请参阅 COMPILE_WARNING_AS_ERROR 目标属性。

此外，为了与 COMPILE_WARNING_AS_ERROR 目标属性保持一致，可以指定一个布尔值

  • True 值：这等效于 LINKER 和 DRIVER 值。

  • False 值：为目标禁用此功能。

此属性并非对所有链接器都已实现。如果所使用的链接器没有实现，则会静默地忽略此属性。当前已实现的 编译器 链接器 ID 是

  • AIX

  • AppleClang

  • GNU

  • GNUgold

  • LLD

  • MOLD

  • MSVC

  • Solaris

当创建目标时，如果设置了变量 CMAKE_LINK_WARNING_AS_ERROR，则此属性将由该变量的值初始化。
