UNITY_BUILD_CODE_AFTER_INCLUDE¶

3.16 版新增。

代码片段，由 UNITY_BUILD 功能在生成的统一源文件中的每个 #include 语句之后逐字包含。例如

set(after [[
#if defined(NOMINMAX)
#undef NOMINMAX
#endif
]])
set_target_properties(myTarget PROPERTIES
  UNITY_BUILD_CODE_AFTER_INCLUDE "${after}"
)

代码说明

  • set(after [[：这一行继续补充当前命令或表达式的内容；变量说明：after：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • #if defined(NOMINMAX)：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • #undef NOMINMAX：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • #endif：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • ]])：这一行继续补充当前命令或表达式的内容。

  • set_target_properties(myTarget PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • UNITY_BUILD_CODE_AFTER_INCLUDE "${after}"：这一行继续补充当前命令或表达式的内容；变量说明：after：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • )：这一行继续补充当前命令或表达式的内容。

另请参阅 UNITY_BUILD_CODE_BEFORE_INCLUDE。
