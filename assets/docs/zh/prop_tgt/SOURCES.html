SOURCES¶

此属性指定目标（target）源文件路径的列表。以下命令均会设置或添加到 SOURCES 目标属性，这也是操作该属性的常用方式：

  • add_executable()

  • add_library()

  • add_custom_target()

  • target_sources()

SOURCES 的内容可以使用 生成器表达式。如果路径以生成器表达式开头，则预期其求值结果应为绝对路径。若未达到此要求，将被视为未定义行为。

如果路径尚未指定为绝对路径，则构建过程中生成的文件的路径将被视为相对于目标的构建目录。请注意，文件是否被视为“生成的文件”可能会受到策略 CMP0118 和 CMP0163 的影响。

如果路径不是以生成器表达式开头、不是绝对路径，且不是生成的文件，它将被视为相对于以下匹配项中首个符合条件的位置：

  • 如果指定路径的文件相对于目标的源目录存在，则使用该文件。

  • 如果策略 CMP0115 未设置为 NEW，则尝试将每个已知的源文件扩展名附加到路径后，并检查该路径相对于目标的源目录是否存在。

  • 重复上述两个步骤，但此次是相对于目标的二进制目录进行检查。

请注意，上述决策是在生成阶段（generation time）而非构建阶段（build time）做出的。

有关定义构建系统属性的更多信息，请参阅 cmake-buildsystem(7) 手册。
