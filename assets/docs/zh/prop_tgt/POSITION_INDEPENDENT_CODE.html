POSITION_INDEPENDENT_CODE¶

这是一个目标属性，用于指定是否创建启用了与位置无关代码（Position-Independent Code）的目标。

POSITION_INDEPENDENT_CODE 目标属性决定了是否创建与位置无关的可执行文件或库。对于 SHARED（共享库）和 MODULE（模块库）目标，该属性默认值为 True。对于其他目标，该属性在创建时由 CMAKE_POSITION_INDEPENDENT_CODE 变量的值进行初始化（如果已设置）；否则默认为 False。

注意

对于可执行文件目标，链接步骤受 CMP0083 策略和 CheckPIESupported 模块控制。

与位置无关的代码（PIC）是指无论其绝对内存地址如何，都能正常执行的机器码。这对于共享库尤为重要，因为共享库通常会被不同的程序加载到不同的内存地址中。生成与位置无关的代码可以确保这些库能够在多个进程之间安全高效地共享，而不会导致地址冲突。在某些平台（尤其是类 UNIX 系统）上，生成 PIC 也是创建共享库的必要条件。

在以下情况下，建议或要求使用与位置无关的代码：

  • 构建共享库或模块库时（例如使用 add_library(... SHARED) 或 add_library(... MODULE)），PIC 允许在运行时进行动态重定位。

  • 构建与位置无关的可执行文件（PIE）时，这可以通过启用地址空间布局随机化（ASLR）来增强安全性。

  • 在某些特定类型的链接或沙盒环境中要求使用 PIC 的平台或工具链上。

在某些架构上，启用 PIC 可能会导致代码体积略微增大或运行速度稍慢，但这种性能损耗通常会被其带来的灵活性和安全性优势所抵消。

示例¶

为静态库目标启用 PIC

add_library(foo STATIC foo.c)
set_target_properties(foo PROPERTIES POSITION_INDEPENDENT_CODE TRUE)

代码说明

  • add_library(foo STATIC foo.c) set_target_properties(foo PROPERTIES POSITION_INDEPENDENT_CODE TRUE)：调用 `add_library()` 命令，定义一个库目标。

为可执行文件目标启用 PIC

add_executable(app app.c)

set_target_properties(app PROPERTIES POSITION_INDEPENDENT_CODE TRUE)

# Additionally, pass PIE-related link-time options to executable(s).
include(CheckPIESupported)
check_pie_supported()

代码说明

  • add_executable(app app.c) set_target_properties(app PROPERTIES POSITION_INDEPENDENT_CODE TRUE) # Additionally, pass PIE-related link-time options to executable(s). include(CheckPIESupported) check_pie_supported()：调用 `add_executable()` 命令，定义一个可执行目标。

另请参阅¶

  • 使用 CheckPIESupported 模块，以便为可执行文件向链接器传递与 PIE 相关的选项。
