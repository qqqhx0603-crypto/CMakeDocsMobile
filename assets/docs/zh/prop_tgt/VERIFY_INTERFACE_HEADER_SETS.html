VERIFY_INTERFACE_HEADER_SETS¶

在 3.24 版本中添加。

用于验证目标（target）中所有 PUBLIC 和 INTERFACE 头文件集中的头文件是否可以独立包含。

当此属性设置为 true，且目标是对象库、静态库、共享库、接口库或可执行文件（受策略 CMP0209 约束），且该目标包含一个或多个 PUBLIC 或 INTERFACE 头文件集时，系统会创建一个名为 <target_name>_verify_interface_header_sets 的对象库目标。此验证目标为 PUBLIC 和 INTERFACE 头文件集中的每个头文件生成一个源文件。每个源文件仅包含其对应的头文件。该验证目标链接到原始目标，以获取其所有的使用需求。

该验证目标的 EXCLUDE_FROM_ALL 和 DISABLE_PRECOMPILE_HEADERS 属性被设置为 true，而其 AUTOMOC、AUTORCC、AUTOUIC、UNITY_BUILD 和 CXX_SCAN_FOR_MODULES 属性被设置为 false。

如果头文件的 LANGUAGE 属性已设置，则该属性的值将用于确定编译头文件所使用的语言。否则，如果目标包含任何 C++ 源文件，则头文件将按 C++ 编译。否则，如果目标包含任何 C 源文件，则头文件将按 C 编译。否则，如果全局启用了 C++，则头文件将按 C++ 编译。否则，如果全局启用了 C，则头文件将按 C 编译。否则，该头文件将不会被编译。

如果头文件的 SKIP_LINTING 属性设置为 true，则该文件不会被编译。

如果 VERIFY_INTERFACE_HEADER_SETS 和 VERIFY_PRIVATE_HEADER_SETS 均设置为 true，则 PUBLIC 文件集中的头文件将被验证两次，但条件不同。私有上下文和接口上下文所使用的编译器标志可能不同，从而导致编译器对头文件内容的解释方式也不同。

如果创建了任何接口文件集验证目标，则会创建一个名为 all_verify_interface_header_sets 的顶级目标，它依赖于所有接口验证目标。此外，还会创建一个名为 all_verify_header_sets 的目标，它依赖于 all_verify_interface_header_sets，以及（如果存在的话）all_verify_private_header_sets（参见 VERIFY_PRIVATE_HEADER_SETS）。

如果在创建目标时设置了 CMAKE_VERIFY_INTERFACE_HEADER_SETS 变量，则此属性将由该变量的值进行初始化。

如果项目希望控制哪些头文件集由该属性进行验证，可以设置 INTERFACE_HEADER_SETS_TO_VERIFY。
