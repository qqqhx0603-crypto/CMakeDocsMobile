<LANG>_CLANG_TIDY_EXPORT_FIXES_DIR¶

3.26 版新增。

只有当 <LANG> 是 C、CXX、OBJC 或 OBJCXX 时，此属性才会被实现，并且只有当 <LANG>_CLANG_TIDY 被设置时才生效。

指定 clang-tidy 工具存放包含其建议更改的 .yaml 文件的目录。这可用于通过 clang-tidy 进行自动的大规模重构。每个被编译的对象文件都会在此目录中有一个对应的 .yaml 文件。在构建完成后，您可以对该目录运行 clang-apply-replacements 以同时将所有建议的更改应用于代码库。如果此属性不是绝对目录，则假定它相对于目标的二进制目录。应优先使用此属性，而不是直接将 --export-fixes 或 --fix 参数添加到 <LANG>_CLANG_TIDY 属性中。

当设置此属性时，CMake 将拥有指定的目录的所有权，并且可以在配置或构建期间的任何时间创建、修改或删除目录内的文件和目录。用户应使用一个专用的目录来导出 clang-tidy 修复，以避免 CMake 删除或覆盖文件。用户不应在此目录中创建、修改或删除文件。

如果当创建目标时设置了 CMAKE_<LANG>_CLANG_TIDY_EXPORT_FIXES_DIR 变量，则此属性将被初始化为该变量的值。
