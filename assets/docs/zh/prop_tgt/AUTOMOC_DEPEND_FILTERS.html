AUTOMOC_DEPEND_FILTERS¶

版本 3.9 中添加。

由 AUTOMOC 使用的过滤器定义，用于从源文件中提取文件名，并将其注册为该源文件的 moc 文件的额外依赖项。

过滤器被定义为 KEYWORD;REGULAR_EXPRESSION 对。首先会在文件内容中搜索 KEYWORD。如果至少找到一次，则通过连续搜索 REGULAR_EXPRESSION 并提取第一个匹配组来获取文件名。

在第一个匹配组中找到的文件名将按以下顺序进行搜索：

  • 首先在源文件附近搜索

  • 然后在其目标的 INCLUDE_DIRECTORIES 中搜索。

如果任何提取的文件发生更改，即使源文件本身未更改，该源文件的 moc 文件也会被重新构建。

如果任何提取的文件是 GENERATED 的，或者它不在目标的源文件中，则可能需要将其添加到 <ORIGIN>_autogen 目标依赖项中。有关参考，请参阅 AUTOGEN_TARGET_DEPENDS。

默认情况下，AUTOMOC_DEPEND_FILTERS 由 CMAKE_AUTOMOC_DEPEND_FILTERS 初始化，后者默认为空。

从 Qt 5.15.0 开始，该变量被忽略，因为 moc 能够输出正确的依赖项。

有关将 CMake 与 Qt 结合使用的更多信息，请参阅 cmake-qt(7) 手册。

示例 1¶

头文件 my_class.hpp 使用了一个自定义宏 JSON_FILE_MACRO，该宏定义在另一个头文件 macros.hpp 中。我们希望 my_class.hpp 的 moc 文件依赖于 JSON_FILE_MACRO 的文件名参数。

// my_class.hpp
class My_Class : public QObject
{
  Q_OBJECT
  JSON_FILE_MACRO ( "info.json" )
...
};

在 CMakeLists.txt 中，我们向 CMAKE_AUTOMOC_DEPEND_FILTERS 添加一个过滤器，如下所示：

list(APPEND CMAKE_AUTOMOC_DEPEND_FILTERS
  "JSON_FILE_MACRO"
  "[\n][ \t]*JSON_FILE_MACRO[ \t]*\\([ \t]*\"([^\"]+)\""
)

  • "[\n][ \t]*JSON_FILE_MACRO[ \t]*\\([ \t]*\"([^\"]+)\""：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • )：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

我们假设 info.json 是一个普通文件（非 GENERATED），并已列在目标的源文件中。因此，我们不需要将其添加到 AUTOGEN_TARGET_DEPENDS 中。

示例 2¶

在目标 my_target 中，头文件 complex_class.hpp 使用了一个自定义宏 JSON_BASED_CLASS，该宏定义在另一个头文件 macros.hpp 中。

// macros.hpp
...
#define JSON_BASED_CLASS(name, json) \
class name : public QObject \
{ \
  Q_OBJECT \
  Q_PLUGIN_METADATA(IID "demo" FILE json) \
  name() {} \

...

  • // end of file：这是代码注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

由于 complex_class.hpp 不包含 Q_OBJECT 宏，它会被 AUTOMOC 忽略。我们通过将 JSON_BASED_CLASS 添加到 CMAKE_AUTOMOC_MACRO_NAMES 中来改变这一点。

list(APPEND CMAKE_AUTOMOC_MACRO_NAMES "JSON_BASED_CLASS")

代码说明

  • list(APPEND CMAKE_AUTOMOC_MACRO_NAMES "JSON_BASED_CLASS")：调用 list() 命令，对列表变量执行追加、删除、查询等操作。

我们希望 complex_class.hpp 的 moc 文件依赖于 meta.json。因此，我们向 CMAKE_AUTOMOC_DEPEND_FILTERS 添加一个过滤器。

list(APPEND CMAKE_AUTOMOC_DEPEND_FILTERS
  "JSON_BASED_CLASS"
  "[\n^][ \t]*JSON_BASED_CLASS[ \t]*\\([^,]*,[ \t]*\"([^\"]+)\""
)

代码说明

  • list(APPEND CMAKE_AUTOMOC_DEPEND_FILTERS：这一行继续补充当前命令或表达式的内容。

  • "JSON_BASED_CLASS"：这一行继续补充当前命令或表达式的内容。

  • "[\n^][ \t]*JSON_BASED_CLASS[ \t]*\\([^,]*,[ \t]*\"([^\"]+)\""：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

此外，我们假设 meta.json 是 GENERATED 的，这就是为什么我们必须将其添加到 AUTOGEN_TARGET_DEPENDS 中的原因。

set_property(TARGET my_target APPEND PROPERTY AUTOGEN_TARGET_DEPENDS "meta.json")

代码说明

  • set_property(TARGET my_target APPEND PROPERTY AUTOGEN_TARGET_DEPENDS "meta.json")：调用 set_property() 命令，设置目标、目录、源文件等对象的属性。
