DEFINE_SYMBOL¶

在编译此目标的源文件时定义一个预处理器符号。

当编译 SHARED 库、MODULE 库，或者启用了 ENABLE_EXPORTS 的 EXECUTABLE（可执行文件）的源文件时，CMake 会添加此定义。如果未设置 DEFINE_SYMBOL，则默认定义的格式为 <target>_EXPORTS（如果目标名称不是有效的 C 标识符，则会进行一些替换）。

该符号仅在编译目标自身时定义，不会传播到依赖的目标。

在 POSIX 平台上，此选项可用于控制符号的可见性。

CMake 通过 GenerateExportHeader 模块为此类修饰提供支持。

另请参阅 目标编译属性 下的 COMPILE_DEFINITIONS。
