AUTOMOC_MACRO_NAMES¶

3.10 版本新增。

一个 分号分隔的列表，包含由 AUTOMOC 使用的宏名称，用于确定 C++ 文件是否需要由 moc 处理。

只有当此目标的 AUTOMOC 属性为 ON 时，此属性才会被使用。

当运行 AUTOMOC 时，CMake 会在 C++ 源代码和头文件中搜索 AUTOMOC_MACRO_NAMES 列表中列出的字符串。如果找到任何一个字符串

  • 作为新行上的第一个非空格字符串，或者

  • 作为新行上 { 之后的第一个非空格字符串，

那么该文件将被 moc 处理。

默认情况下，AUTOMOC_MACRO_NAMES 从 CMAKE_AUTOMOC_MACRO_NAMES 初始化。

另请参阅目标属性 INTERFACE_AUTOMOC_MACRO_NAMES。

有关将 CMake 与 Qt 结合使用的更多信息，请参阅 cmake-qt(7) 手册。

示例¶

在这种情况下，Q_OBJECT 宏隐藏在另一个名为 CUSTOM_MACRO 的宏中。为了让 CMake 知道包含 CUSTOM_MACRO 的源文件需要被 moc 处理，我们调用

set_property(TARGET tgt APPEND PROPERTY AUTOMOC_MACRO_NAMES "CUSTOM_MACRO")
