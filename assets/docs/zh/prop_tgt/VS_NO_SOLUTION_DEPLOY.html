VS_NO_SOLUTION_DEPLOY¶

版本 3.15 新增。

指定目标不应在生成的 Visual Studio（集成开发环境） 解决方案中标记为部署到 Windows（微软桌面系统） CE 或 Windows（微软桌面系统） Phone 设备。

默认情况下，所有 EXE 和共享库 (DLL（动态链接库）) 目标都会在生成的 Visual Studio（集成开发环境） 解决方案中标记为部署到目标设备。

支持生成器表达式。

可能需要从部署中排除目标/生成项目的理由有很多

  • 库或可执行文件可能在主要的部署/调试场景中不是必需的，并且从部署中排除可以节省开发/下载/调试周期的时长。

  • 目标设备上可能没有足够的空间来容纳所有构建产物。

  • Visual Studio 2013（微软集成开发环境）（微软集成开发环境） 需要为每个标记为部署的目标输入目标设备 IP 地址。对于大量目标，这可能很繁琐。注意：Visual Studio（集成开发环境） 会将标记为部署的项目的所有项目依赖项部署到为此项目配置的 IP 地址，即使这些依赖项未标记为部署。

示例 1¶

这显示了为目标 foo 设置变量。

add_library(foo SHARED foo.cpp)
set_property(TARGET foo PROPERTY VS_NO_SOLUTION_DEPLOY ON)

代码说明

  • add_library(foo SHARED foo.cpp)：调用 add_library() 命令，定义一个库目标。

  • set_property(TARGET foo PROPERTY VS_NO_SOLUTION_DEPLOY ON)：调用 set_property() 命令，设置目标、目录、源文件等对象的属性。

示例 2¶

这显示了仅为 Release 配置设置变量。

add_library(foo SHARED foo.cpp)
set_property(TARGET foo PROPERTY VS_NO_SOLUTION_DEPLOY "$<CONFIG:Release>")

代码说明

  • add_library(foo SHARED foo.cpp)：调用 add_library() 命令，定义一个库目标。

  • set_property(TARGET foo PROPERTY VS_NO_SOLUTION_DEPLOY "$<CONFIG:Release>")：调用 set_property() 命令，设置目标、目录、源文件等对象的属性；生成器表达式：$<CONFIG:...> 在生成构建系统时按当前目标和配置计算结果。
