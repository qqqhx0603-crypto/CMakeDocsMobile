Fortran_MODULE_DIRECTORY¶

指定目标提供的 Fortran 模块的输出目录。

如果目标包含提供模块的 Fortran 源代码文件，并且编译器支持模块输出目录，则此属性指定模块将被放置的目录。 如果未设置此属性，模块将被放置在与目标源代码目录对应的构建目录中。 如果在创建目标时设置了变量 CMAKE_Fortran_MODULE_DIRECTORY，则其值将用于初始化此属性。

在使用安装在 Visual Studio（集成开发环境） 中的 Intel Fortran 插件的 Visual Studio（集成开发环境） 生成器之一时，一个以配置命名的子目录将被附加到创建模块的路径中。 例如，如果 Fortran_MODULE_DIRECTORY 设置为 C:/some/path，则模块将最终位于 C:/some/path/Debug（或 C:/some/path/Release 等）中，当生成 Intel Fortran .vfproj 文件时，而在使用任何其他生成器时，模块将位于 C:/some/path 中。

请注意，某些编译器将自动在模块输出目录中搜索编译期间 USE 的模块，而另一些编译器则不会。 如果您的源代码 USE 模块，无论此属性如何，都必须通过 INCLUDE_DIRECTORIES 指定它们的位置。
