<LANG>_PVS_STUDIO¶

4.3 版本新增。

此属性仅在 <LANG> 为 C 或 CXX 时实现。

指定一个分号分隔的列表，其中包含用于 pvs-studio-analyzer 工具（在 Windows（微软桌面系统） 上命名为 CompilerCommandsAnalyzer）的命令行。Makefile（Make 构建脚本） 生成器 和 Ninja（构建工具） 生成器 将在运行编译器时一并运行此工具，如果该工具报告任何问题，则会发出警告。

指定的 pvs-studio-analyzer 命令行将在调用时附加以下参数：

  • --source-file：源文件。

  • --output-file：目标文件附近的路径，用于写入 PVS 日志。

  • --cl-params：编译选项。

  • --preprocessor：预处理器，基于 CMAKE_<LANG>_COMPILER_ID，如果确定为以下之一：visualcpp, clang, gcc, bcc, iar。

  • --platform：目标平台，如果确定为以下之一：arm, win32, x64, linux32, linux64, macOS。

有关这些选项及其他可用选项的详细信息，请参阅 PVS-Studio 文档。

CMake 会在所提供的 pvs-studio-analyzer 所在的同一目录下查找 plog-converter 工具；如果该目录下不存在，则会在用户的 PATH 路径中查找。plog-converter 将自动以 Txt 输出类型（在 Windows（微软桌面系统） 上）或 errorfile 输出类型（在其他平台上）运行，文件内容将被发送至 stderr。转换器运行后，PVS 日志文件将被删除。

如果在创建目标时设置了 CMAKE_<LANG>_PVS_STUDIO 变量，则此属性将由该变量的值进行初始化。
