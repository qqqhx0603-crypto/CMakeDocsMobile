UNITY_BUILD_BATCH_SIZE¶

3.16 版新增。

指定在启用 unity 构建时，可以合并到任何一个 unity 源代码文件中的最大源文件数量。此限制由 UNITY_BUILD 目标属性启用时生效。原始源代码文件将分布在尽可能多的 unity 源代码文件中，以满足此限制。

此属性的初始值取自目标创建时 CMAKE_UNITY_BUILD_BATCH_SIZE 变量。如果该变量未设置，则初始值为 8。

需要仔细选择批处理大小。如果设置得太高，合并后的源代码文件的大小可能会导致编译器使用过多的内存或达到其他类似的限制。在极端情况下，这甚至可能导致构建失败。另一方面，如果批处理大小设置得太低，则构建性能的提升将微乎其微。

尽管强烈不建议这样做，但可以将批处理大小设置为 0，以将目标的全部源代码合并到一个 unity 文件中，而不管涉及多少个源代码文件。这有创建过大的 unity 源代码文件并对构建性能产生负面影响的风险，因此通常不建议将值设置为 0。
