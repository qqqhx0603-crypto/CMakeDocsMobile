INSTALL_OBJECT_ONLY_USE_DESTINATION¶

版本 4.2 中添加。

控制用于对象库的 install(DESTINATION) 是否按原样使用，或者是否补充冲突避免的子目录。

在安装目标文件时，CMake 会自动将 objects[-<CONFIG>]/<TARGET_NAME> 组件添加到目标路径，以避免冲突。使用此属性可以抑制这些组件。请注意，当为多个配置使用单个安装前缀（无论是通过多配置生成器还是单独的构建树）时，目标路径必须使用 $<CONFIG> 以避免冲突。或者，可以使用 INSTALL_OBJECT_NAME 来避免基于配置的冲突。

此属性由目标创建时 CMAKE_INSTALL_OBJECT_ONLY_USE_DESTINATION 的值初始化。
