IMPORTED_NO_SYSTEM¶

在版本 3.23 中添加。

自版本 3.25 起已弃用：IMPORTED_NO_SYSTEM 已弃用。请使用以下替代方案

  • 如果不想将目标的包含目录在编译使用者时视为系统目录，请将 SYSTEM 设置为 false。

  • 如果不想将由 install(EXPORT) 和 export() 命令生成的导入目标的包含目录在编译使用者时视为系统目录，请将 EXPORT_NO_SYSTEM 设置为 true。

在 导入目标 上将 IMPORTED_NO_SYSTEM 设置为 true，表示它不是系统目标。这将产生以下效果

  • INTERFACE_INCLUDE_DIRECTORIES 的条目在编译使用者时不会被视为系统包含目录（无论被消耗目标的 SYSTEM 属性的值如何），默认情况下它们会被视为系统包含目录。 INTERFACE_SYSTEM_INCLUDE_DIRECTORIES 的条目不受影响，并且始终被视为系统包含目录。

  • 在 Apple 平台上，如果目标是框架，则不会将其视为系统。

此属性也可以在非导入目标上启用。这样做不会影响构建系统，但会告知 install(EXPORT) 和 export() 命令在它们生成的导入目标上启用它。

请参阅 NO_SYSTEM_FROM_IMPORTED 目标属性，以在消耗包含目录的目标上而不是提供它们的的目标上设置此行为。
