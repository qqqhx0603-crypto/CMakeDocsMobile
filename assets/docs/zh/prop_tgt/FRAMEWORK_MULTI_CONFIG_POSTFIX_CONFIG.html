FRAMEWORK_MULTI_CONFIG_POSTFIX_<CONFIG>¶

在 3.18 版本中新增。

当使用多配置生成器（如 Xcode 和 Ninja Multi-Config）时，要附加到框架文件名的后缀，用于配置 <CONFIG>。

使用配置 <CONFIG> 构建时，此属性的值将附加到磁盘上构建的框架文件名。

例如，给定一个名为 my_fw 的框架，FRAMEWORK_MULTI_CONFIG_POSTFIX_DEBUG 属性的值为 _debug，并且 CMAKE_CONFIGURATION_TYPES 中为 Debug;Release，则将为 Debug 和 Release 配置创建以下相关文件：

  • Release/my_fw.framework/my_fw

  • Release/my_fw.framework/Versions/A/my_fw

  • Debug/my_fw.framework/my_fw_debug

  • Debug/my_fw.framework/Versions/A/my_fw_debug

对于框架目标，如果目标创建时设置了该变量，则此属性由 CMAKE_FRAMEWORK_MULTI_CONFIG_POSTFIX_<CONFIG> 变量的值初始化。

此属性对于非框架目标以及使用单配置生成器时将被忽略。
