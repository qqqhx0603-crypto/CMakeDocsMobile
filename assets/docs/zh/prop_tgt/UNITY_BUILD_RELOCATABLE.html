UNITY_BUILD_RELOCATABLE¶

4.0 版本新增。

默认情况下，当启用 UNITY_BUILD 时生成的 unity 文件使用绝对路径来引用原始源文件。 这会导致 unity 文件根据源文件位置产生不同的输出。

当此属性设置为 true 时，生成的 unity 源文件中的 #include 行将尝试使用相对路径来引用原始源文件（如果可能），以便标准化 unity 文件的输出。

unity 文件到原始源文件的路径优先级如下：

  • 如果源文件直接存在于 CMAKE_BINARY_DIR 或其子文件夹下，则使用相对于生成的 unity 文件的路径。

  • 如果源文件直接存在于 CMAKE_SOURCE_DIR 或其子文件夹下，则使用相对于 CMAKE_SOURCE_DIR 的路径。

  • 使用原始源文件的绝对路径。

此目标属性不能保证在不同环境中 unity 文件的一致性，因为最终优先级是绝对路径。

用法示例

add_library(example_library
            source1.cxx
            source2.cxx
            source3.cxx)

set_target_properties(example_library PROPERTIES
                      UNITY_BUILD True
                      UNITY_BUILD_RELOCATABLE TRUE)

代码说明

  • add_library(example_library source1.cxx source2.cxx source3.cxx) set_target_properties(example_library PROPERTIES UNITY_BUILD True UNITY_BUILD_RELOCATABLE TRUE)：调用 `add_library()` 命令，定义一个库目标。
