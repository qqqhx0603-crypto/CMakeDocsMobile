OPTIMIZE_DEPENDENCIES¶

3.19 版本新增。

激活静态库和目标文件的依赖优化。

当此属性设置为 true 时，如果静态库或目标文件的某些依赖项对于构建库来说不是必需的，则可以在生成时删除这些依赖项，因为静态库和目标文件实际上不会链接到任何东西。

如果静态库或目标文件启用了依赖优化，它首先会丢弃所有依赖项。然后，它会检查它最初拥有的所有直接和间接依赖项，并在满足以下任何条件时将它们添加回来：

  • 该依赖项是通过 add_dependencies() 添加到库中的。

  • 该依赖项是通过库中由使用该依赖项的自定义命令生成的文件添加到的库。

  • 该依赖项具有任何 PRE_BUILD、PRE_LINK 或 POST_BUILD 自定义命令与之关联。

  • 该依赖项包含任何由自定义命令生成的文件。但是，如果 CMP0154 对于该依赖项为 NEW 并且它使用 文件集，则仅考虑作用域为 PUBLIC、INTERFACE 或类型为 CXX_MODULES 的源文件。

  • 该依赖项包含任何产生与库相关的副作用的语言。目前，假设所有语言（C、C++、Objective-C、Objective-C++、汇编和 CUDA 除外）都会产生副作用。但是，假设来自一种语言的副作用与另一种语言无关（例如，假设 Fortran 库不会产生与 Swift 库相关的任何副作用）。

例如，假设你有一个静态 Fortran 库，它依赖于一个静态 C 库，而该 C 库又依赖于一个静态 Fortran 库。顶级 Fortran 库启用了优化，但中间 C 库没有启用。如果你构建顶层 Fortran 库，则底层 Fortran 库也会构建，但中间 C 库不会构建，因为 C 库不会产生与 Fortran 库相关的任何副作用。但是，如果你构建中间 C 库，则底层 Fortran 库也会构建，即使它不会产生与 C 库相关的任何副作用，因为 C 库没有启用优化。

此属性由目标创建时 CMAKE_OPTIMIZE_DEPENDENCIES 变量的值初始化。
