VS_NO_COMPILE_BATCHING¶

在 3.24 版本中添加。

为目标关闭编译批处理。通常，MSBuild 会使用多个 c/cpp 文件调用编译器，并且编译器为每个文件启动子进程以实现并行构建。如果您希望编译器一次被调用一个文件，请将 VS_NO_COMPILE_BATCHING 设置为 ON。如果设置此标志，MSBuild 将使用一个 c/cpp 文件一次调用编译器。当您想要使用替换编译器的工具（例如某些构建缓存工具）时，此功能很有用。

当创建目标时，如果设置了 CMAKE_VS_NO_COMPILE_BATCHING 变量，则此属性将由其初始化。

示例¶

这显示了为目标 foo 设置属性。

add_library(foo SHARED foo.cpp)
set_property(TARGET foo PROPERTY VS_NO_COMPILE_BATCHING ON)

代码说明

  • add_library(foo SHARED foo.cpp) set_property(TARGET foo PROPERTY VS_NO_COMPILE_BATCHING ON)：调用 `add_library()` 命令，定义一个库目标。
