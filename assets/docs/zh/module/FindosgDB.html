FindosgDB¶

查找 OpenSceneGraph 工具包中的 osgDB 库。

注意

在大多数情况下，建议改用 FindOpenSceneGraph 模块并将 osgDB 列为组件。这将自动处理诸如 OpenThreads 和核心 osg 库之类的依赖关系。

find_package(OpenSceneGraph COMPONENTS osgDB)

此模块由 FindOpenSceneGraph 在内部用于查找 osgDB 库。在常规使用 find_package() 命令时，不建议直接包含此模块。但是，对于需要更精细控制检测的高级用例，它作为独立模块提供。例如，显式查找 osgDB 或绕过自动组件检测。

find_package(osgDB)

OpenSceneGraph 和 osgDB 头文件旨在按以下方式包含在 C++ 项目源代码中：

example.cxx¶
#include <osg/PositionAttitudeTransform>
#include <osgDB/DatabasePager>
// ...

使用 OpenSceneGraph 工具包时，可能还需要其他库，例如 OpenGL。

结果变量¶

此模块定义了以下变量

osgDB_FOUND

3.3 版本中新增。

布尔值，指示是否找到了 OpenSceneGraph 工具包的 osgDB 库。

OSGDB_LIBRARIES

使用 osgDB 进行链接时所需的库。

OSGDB_LIBRARY

结果变量，其值与 OSGDB_LIBRARIES 变量相同。

缓存变量¶

以下缓存变量也可以设置

OSGDB_INCLUDE_DIR

包含使用 osgDB 所需头文件的目录。

OSGDB_LIBRARY_DEBUG

osgDB 调试库的路径。

提示¶

此模块接受以下变量

OSGDIR

当 OpenSceneGraph 工具包（包括其 osgDB 库）安装在自定义位置时，可以设置此环境变量以帮助定位它。它应指向配置、构建和安装时使用的 OpenSceneGraph 安装前缀：./configure --prefix=$OSGDIR。

已弃用变量¶

以下变量提供用于向后兼容性

OSGDB_FOUND

自 4.2 版本起弃用：请使用 osgDB_FOUND，其值相同。

布尔值，指示是否找到了 OpenSceneGraph 工具包的 osgDB 库。

示例¶

使用此模块显式查找 osgDB，并创建一个接口 导入目标 (imported target)，该目标封装了将其链接到项目目标时的使用需求。

find_package(osgDB)

if(osgDB_FOUND AND NOT TARGET osgDB::osgDB)
  add_library(osgDB::osgDB INTERFACE IMPORTED)
  set_target_properties(
    osgDB::osgDB
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${OSGDB_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${OSGDB_LIBRARIES}"
  )
endif()

target_link_libraries(example PRIVATE osgDB::osgDB)
