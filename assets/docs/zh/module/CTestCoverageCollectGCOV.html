CTestCoverageCollectGCOV¶

版本 3.2 中新增。

此模块旨在用于 CTest 仪表板脚本，并提供一个命令来生成包含代码覆盖率报告的压缩包。

在 CTest 脚本中加载此模块，使用

include(CTestCoverageCollectGCOV)

代码说明

  • include(CTestCoverageCollectGCOV)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

ctest_coverage_collect_gcov¶

运行 gcov 并为 CDash 打包一个 tar 文件

ctest_coverage_collect_gcov(
  TARBALL <tar-file>
  [TARBALL_COMPRESSION <compression>]
  [SOURCE <source-dir>]
  [BUILD <build-dir>]
  [GCOV_COMMAND <gcov-command>]
  [GCOV_OPTIONS <options>...]
  [GLOB]
  [DELETE]
  [QUIET]
)

代码说明

  • ctest_coverage_collect_gcov( TARBALL <tar-file> [TARBALL_COMPRESSION <compression>] [SOURCE <source-dir>] [BUILD <build-dir>] [GCOV_COMMAND <gcov-command>] [GCOV_OPTIONS <options>...] [GLOB] [DELETE] [QUIET] )：调用 `ctest_coverage_collect_gcov()` 命令，具体行为由这一行传入的参数决定。

此命令会对构建树中找到的所有 .gcda 文件运行 gcov，并将生成的 .gcov 文件打包成一个 tar 文件，同时包含以下内容：

  • data.json 文件，定义了供 CDash 使用的源目录和构建目录。

  • Labels.json 文件，指示在源文件上设置的任何 LABELS。

  • uncovered 目录，包含由 CTEST_EXTRA_COVERAGE_GLOB 找到的任何未覆盖文件。

生成的 tar 文件可以使用 ctest_submit(CDASH_UPLOAD) 命令提交到 CDash 进行显示。

参数为

TARBALL <tar-file>

指定要创建的 .tar 文件的位置，以便稍后上传到 CDash。相对路径将相对于顶级构建目录进行解释。

TARBALL_COMPRESSION <compression>

在 3.18 版本中新增。

指定 TARBALL 数据文件的压缩算法。使用此选项可以在提交到 CDash 之前减小数据文件的大小。<compression> 应为以下值之一：

  • GZIP

  • BZIP2

  • LZMA

    4.3 版本新增。

  • LZMA2

    4.3 版本新增。

    这是 XZ 的别名。

  • XZ

  • ZSTD

  • FROM_EXT

  • CMake 计算结果为 FALSE 的表达式。

默认值为 BZIP2。

如果指定了 FROM_EXT，结果文件将根据 <tar-file> 的文件扩展名进行压缩（例如，.tar.gz 将使用 GZIP 压缩）。会产生压缩输出的文件扩展名包括：

  • .tar.gz

  • .tgz

  • .tar.bzip2

  • .tbz

  • .tar.xz

  • .txz

  • .tar.lzma

    4.3 版本新增。

  • .tlzma

    4.3 版本新增。

  • .tar.zst

    4.3 版本新增。

  • .tzst

    4.3 版本新增。

SOURCE <source-dir>

指定构建的顶级源目录。默认值为 CTEST_SOURCE_DIRECTORY 的值。

BUILD <build-dir>

指定构建的顶级构建目录。默认值为 CTEST_BINARY_DIRECTORY 的值。

GCOV_COMMAND <gcov-command>

指定机器上 gcov 命令的完整路径。默认值为 CTEST_COVERAGE_COMMAND 的值。

GCOV_OPTIONS <options>...

指定传递给 gcov 的选项。gcov 命令的运行方式为 gcov <options>... -o <gcov-dir> <file>.gcda。如果未指定，默认选项仅为 -b -x。

GLOB

3.6 版本新增。

递归搜索 <build-dir> 中的 .gcda 文件，而不是通过读取 CMakeFiles/TargetDirectories.txt（由 CMake 在生成阶段生成的文件）来确定搜索位置。

DELETE

3.6 版本新增。

在覆盖率文件被打包进 .tar 后将其删除。

QUIET

禁止显示此命令本应打印出的非错误消息。

在 3.3 版本加入: 增加了对 CTEST_CUSTOM_COVERAGE_EXCLUDE 变量的支持。

示例¶

在 ctest -S 脚本中生成打包为 .tar.gz 文件的代码覆盖率数据

script.cmake¶
include(CTestCoverageCollectGCOV)

ctest_coverage_collect_gcov(
  TARBALL "${CTEST_BINARY_DIRECTORY}/gcov.tar.gz"
  TARBALL_COMPRESSION "GZIP"
)

代码说明

  • include(CTestCoverageCollectGCOV) ctest_coverage_collect_gcov( TARBALL "${CTEST_BINARY_DIRECTORY}/gcov.tar.gz" TARBALL_COMPRESSION "GZIP" )：调用 `include()` 命令，加载并执行另一个 CMake 脚本。
