FindIce¶

版本 3.1 中新增。

查找 Internet Communication Engine (Ice) 程序、库和数据文件

find_package(Ice [<version>] COMPONENTS <components>... [...])

代码说明

  • find_package(Ice [<version>] COMPONENTS <components>... [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

Ice 是由 ZeroC 开发的开源远程过程调用 (RPC) 框架，为开发网络应用程序的各种语言提供了 SDK。

版本 3.4 新增: 组件的导入目标以及许多新的 *_EXECUTABLE 变量。

3.7 版本新增： 调试和发布库变体分别查找。

版本 3.10 新增: Ice 3.7 支持，包括新组件、程序和 Nuget 包。

组件¶

Ice 由多个库和程序（可执行文件）组成。此查找模块支持组件（可使用 find_package() 命令指定），以便在 CMake 项目中选择特定的 Ice 库进行使用。无论指定了哪些组件，Ice 安装提供的可执行文件总是会自动搜索。

可用组件列表取决于所使用的 Ice 版本。要成功找到 Ice，必须至少指定一个组件

find_package(Ice COMPONENTS <components>...)

代码说明

  • find_package(Ice COMPONENTS <components>...)：调用 `find_package()` 命令，查找并加载外部依赖包。

支持的组件包括

  • Freeze

  • Glacier2

  • Ice

  • IceBox

  • IceDB

  • IceDiscovery

  • IceGrid

  • IceLocatorDiscovery

  • IcePatch

  • IceSSL

  • IceStorm

  • IceUtil

  • IceXML

  • Slice

Ice 3.7 及更高版本还包括 C++11 特有的组件

  • Glacier2++11

  • Ice++11

  • IceBox++11

  • IceDiscovery++11

  • IceGrid++11

  • IceLocatorDiscovery++11

  • IceSSL++11

  • IceStorm++11

导入的目标¶

此模块提供以下 导入目标

Ice::<component>

3.4 版本新增。

封装指定 Ice 组件（库）使用需求的目标，如果找到该组件则可用。<component> 的大小写应与上述列表一致。例如，使用 Ice::Glacier2 表示 Ice Glacier2 库，使用 Ice::Ice++11 表示 Ice++11 库，等等。

结果变量¶

此模块定义了以下变量

Ice_FOUND

布尔值，指示是否找到了（所请求版本的）Ice 主要程序、库以及所有请求的组件。

Ice_VERSION

所找到的 Ice 发行版本。

Ice_INCLUDE_DIRS

使用 Ice 所需的包含头文件的目录。

Ice_LIBRARIES

使用 Ice 时需要链接的组件库。

Ice_SLICE_DIRS

包含 Slice（Ice 规范语言）接口定义（*.ice 文件）的数据目录。

Ice 组件库存储在

Ice_<COMPONENT>_FOUND

布尔值，指示是否找到了指定的 Ice 组件。<COMPONENT> 应以大写形式书写。

Ice_<COMPONENT>_LIBRARIES

由指定 Ice 组件提供的库。<COMPONENT> 应以大写形式书写。

缓存变量¶

以下缓存变量也可以设置

Ice_<PROGRAM>_EXECUTABLE

指定 <PROGRAM> 可执行文件的路径；<PROGRAM> 是上述可执行文件结果变量中列出的 Ice 程序的大写名称。

Ice_INCLUDE_DIR

包含 Ice 头文件的目录。

Ice_SLICE_DIR

包含 Slice 接口定义的数据目录。

Ice_<COMPONENT>_LIBRARY

指定组件的库路径。<COMPONENT> 应以大写形式书写。

提示¶

此模块接受以下变量

Ice_HOME

将此 CMake 变量设置为 Ice 安装的根目录，以便在自定义位置搜索 Ice。

注意

在 Windows（微软桌面系统） 上，Ice 3.7.0 及更高版本通过 NuGet 包管理器提供库。将使用 CMAKE_PREFIX_PATH 搜索合适的 NuGet 包，或者可以将 Ice_HOME 设置为特定 NuGet 包的位置以限制搜索。

ICE_HOME

环境变量（大写）也可以设置为 Ice 安装的根目录；Ice_HOME CMake 变量具有优先权。

Ice_DEBUG

将此变量设置为布尔值 true 以启用此模块的调试输出。

注意

在大多数情况下，无需设置上述任何变量，除非安装了多个 Ice 版本且需要特定版本。在 Windows（微软桌面系统） 上，通常使用注册表找到最新版本。在类 Unix 系统上，程序、头文件和库通常在标准位置找到，尽管 Ice_SLICE_DIRS 可能无法自动检测到（会搜索通常已知的位置）。所有其他变量均基于 Ice_HOME（如果已设置）的值作为默认值。

也可以设置 Ice_HOME 同时有选择地覆盖单个组件的特定位置；例如，在较新版本的 Visual Studio（集成开发环境） 中，如果启发式方法不足以识别特定 Visual Studio（集成开发环境） 版本所需的正确程序和库，则可能需要这样做。

示例¶

查找 Ice 核心库并将其链接到项目目标

find_package(Ice COMPONENTS Ice)
target_link_libraries(project_target PRIVATE Ice::Ice)

代码说明

  • find_package(Ice COMPONENTS Ice) target_link_libraries(project_target PRIVATE Ice::Ice)：调用 `find_package()` 命令，查找并加载外部依赖包。

查找 Ice 核心库和 IceSSL 库，并将其链接到项目目标

find_package(Ice COMPONENTS Ice IceSSL)
target_link_libraries(project_target PRIVATE Ice::Ice Ice::IceSSL)

代码说明

  • find_package(Ice COMPONENTS Ice IceSSL) target_link_libraries(project_target PRIVATE Ice::Ice Ice::IceSSL)：调用 `find_package()` 命令，查找并加载外部依赖包。

将 Ice 核心库作为必需组件查找，将 Ice Freeze 库作为可选组件查找

find_package(Ice COMPONENTS Ice OPTIONAL_COMPONENTS Freeze)

代码说明

  • find_package(Ice COMPONENTS Ice OPTIONAL_COMPONENTS Freeze)：调用 `find_package()` 命令，查找并加载外部依赖包。
