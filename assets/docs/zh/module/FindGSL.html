FindGSL¶

版本 3.2 中新增。

查找本机 GNU 科学库 (GSL) 的头文件和库

find_package(GSL [<version>] [...])

代码说明

  • find_package(GSL [<version>] [...])：调用 find_package() 命令，查找并加载外部依赖包。

GNU 科学库 (GSL) 是一个为 C 和 C++ 程序员提供的数值库。它在 GNU 通用公共许可证下是自由软件。

导入的目标¶

此模块提供以下 导入目标

GSL::gsl

封装主 GSL 库和所有使用要求的 Target，仅在找到 GSL 时可用。

GSL::gslcblas

封装 GSL 使用的 CBLAS 支持库的使用要求。如果找到 GSL，则此 Target 可用。它也链接到 GSL::gsl Target，但单独提供以提高粒度。

结果变量¶

此模块定义了以下变量

GSL_FOUND

布尔值，指示是否在本地系统上找到了（请求版本的）GSL。

GSL_VERSION

发现的 GSL 安装的版本。

GSL_INCLUDE_DIRS

包含 GSL 头文件的目录。

GSL_LIBRARIES

GSL 库。

缓存变量¶

此模块可以根据平台和发现的 GSL 安装类型设置以下变量。这些变量可以可选地设置为帮助此模块找到正确的文件

GSL_CBLAS_LIBRARY

GSL CBLAS 库的位置。

GSL_CBLAS_LIBRARY_DEBUG

调试 GSL CBLAS 库的位置（如果有）。

GSL_CONFIG_EXECUTABLE

gsl-config 脚本的位置（如果有）。

GSL_LIBRARY

GSL 库的位置。

GSL_LIBRARY_DEBUG

调试 GSL 库的位置（如果有）。

提示¶

此模块接受以下变量

GSL_ROOT_DIR

将此变量设置为包含 GSL 安装的目录。

如果未设置此变量，则此模块将使用 pkg-config 和默认路径来查找 GSL。如果提供了此变量，则此模块期望在 ${GSL_ROOT_DIR}/lib 处找到库，并在 ${GSL_ROOT_DIR}/include/gsl 处找到 GSL 头文件。

库目录可以选择提供 Release 和 Debug 文件夹。如果可用，则名为 gsld、gslblasd 或 cblasd 的库将被识别为调试库。对于类 Unix 系统，此模块还将使用 gsl-config（如果找到）来帮助发现 GSL。

示例¶

查找 GSL 并将其链接到项目 Target

find_package(GSL)
target_link_libraries(project_target PRIVATE GSL::gsl)

代码说明

  • find_package(GSL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE GSL::gsl)：调用 target_link_libraries() 命令，为目标链接依赖库。
