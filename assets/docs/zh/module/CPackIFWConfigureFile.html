CPackIFWConfigureFile¶

版本 3.8 新增。

该模块提供了一个类似于 configure_file() 的命令，用于配置以 QtIFW/SDK/Creator 风格准备的文件模板。

在 CMake 项目中加载此模块，使用

include(CPackIFWConfigureFile)

命令¶

此模块提供以下命令

cpack_ifw_configure_file¶

将文件模板复制到输出文件，并替换输入文件模板内容中引用为 %{VAR} 或 %VAR% 的变量值。

cpack_ifw_configure_file(<input> <output>)
<input>

输入文件模板。如果给定为相对路径，则将其解释为相对于当前源目录（CMAKE_CURRENT_SOURCE_DIR）。

<output>

输出文件。如果给定为相对路径，则将其解释为相对于当前二进制目录（CMAKE_CURRENT_BINARY_DIR）。

Qt 安装程序框架（QtIFW）使用 @ 字符在 Qt 安装程序脚本中嵌入预定义变量（TargetDir、StartMenuDir 等）。

example.qs¶
component.addOperation(
  "CreateShortcut",
  "@TargetDir@/example.com.html",
  "@StartMenuDir@/Example Web Site.lnk"
);

此命令的目的是保留包含 @ 字符的 QtIFW 预定义变量（@VAR@），并改用 % 字符作为 Qt/IFW/SDK/Creator 模板中的模板占位符（%VAR%, %{VAR}）。否则，configure_file() 命令会替换所有包含 @ 字符的变量引用。

每个变量引用都将替换为该变量的当前值；如果变量未定义，则替换为空字符串。

示例¶

在以下示例中，该模块用于从给定的模板文件 qt.tools.foo.qs.in 创建 IFW 组件脚本，其中 %FOO_DOC_DIR% 变量引用将被 FOO_DOC_DIR CMake 变量的值替换。

CMakeLists.txt¶
cmake_minimum_required(VERSION 3.8)

project(Foo)

# ...

include(CPackIFWConfigureFile)

set(FOO_DOC_DIR "doc/foo")

cpack_ifw_configure_file(qt.tools.foo.qs.in qt.tools.foo.qs)
qt.tools.foo.qs.in¶
function Component()
{
}

Component.prototype.createOperations = function()
{
  if (installer.value("os") === "win") {
    component.addOperation(
      "CreateShortcut",
      "@TargetDir@/%FOO_DOC_DIR%/example.com.html",
      "@StartMenuDir@/Example Web Site.lnk"
    );
  }

  component.createOperations();
}

// ...
