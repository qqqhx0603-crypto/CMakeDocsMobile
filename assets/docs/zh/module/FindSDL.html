FindSDL¶

查找 SDL (Simple DirectMedia Layer) 库

find_package(SDL [<version>] [...])

代码说明

  • find_package(SDL [<version>] [...])：调用 find_package() 命令，查找并加载外部依赖包。

SDL 是一个跨平台库，用于开发多媒体软件，例如游戏和模拟器。

注意

此模块专门用于 SDL 1 版本。从 2 版本开始，SDL 在通过 CMake 构建时提供了一个 CMake 软件包配置文件，应使用 find_package(SDL2) 来查找。同样，SDL 3 版本可以使用 find_package(SDL3) 来查找。这些较新的版本提供了单独的 导入目标 (Imported Targets)，封装了使用要求。有关更多信息，请参考官方 SDL 文档。

请注意，SDL 头文件的包含路径在近期的 SDL 1 版本中已从 SDL/SDL.h 更改为简单的 SDL.h。此更改符合 SDL 使用 #include "SDL.h" 以实现可移植性的约定，因为并非所有系统都将头文件安装在 SDL/ 子目录中（例如 FreeBSD）。

当针对 macOS（苹果桌面系统） 并使用 SDL 框架时，请确保在项目中同时包含 SDLmain.h 和 SDLmain.m。对于其他平台，SDLmain 库通常使用 -lSDLmain 进行链接，此模块将尝试自动定位它。此外，对于 macOS（苹果桌面系统），此模块将根据需要添加 -framework Cocoa 标志。

导入的目标¶

此模块提供以下 导入目标

SDL::SDL

3.19 版本新增。

封装 SDL 库使用要求的目标，如果找到 SDL，则可用。

结果变量¶

此模块定义了以下变量

SDL_FOUND

布尔值，指示是否找到了（所请求版本的）SDL。

SDL_VERSION

3.19 版本新增。

包含所找到 SDL 版本的易读字符串。

SDL_VERSION_MAJOR

3.19 版本新增。

所找到 SDL 的主版本号。

SDL_VERSION_MINOR

3.19 版本新增。

所找到 SDL 的次版本号。

SDL_VERSION_PATCH

3.19 版本新增。

所找到 SDL 的修订版本号。

SDL_INCLUDE_DIRS

3.19 版本新增。

使用 SDL 所需的包含目录。

SDL_LIBRARIES

3.19 版本新增。

使用 SDL 时需要链接的库。

缓存变量¶

可以有选择地设置这些变量以帮助此模块查找正确的文件

SDL_INCLUDE_DIR

包含 SDL.h 头文件的目录。

SDL_LIBRARY

包含 SDL 库路径以及使用 SDL 时需要链接的库的列表。

提示¶

此模块接受以下变量

SDL_BUILDING_LIBRARY

当设置为布尔值 true 时，SDL_main 库将从链接中排除，因为它在构建 SDL 库本身时不需要（只有应用程序需要 main() 函数）。如果未设置，该模块会假定正在构建应用程序，并尝试定位并在返回的 SDL_LIBRARY 变量中包含适当的 SDL_main 链接标志。

SDLDIR

环境变量，可用于帮助定位安装在自定义位置的 SDL 库。它应指向配置、构建和安装 SDL 库时使用的安装目标：./configure --prefix=$SDLDIR。

在 macOS（苹果桌面系统） 上，设置此变量将优先选择框架版本（如果找到）。在这种情况下，需要手动更改 SDL_LIBRARY 的缓存值以覆盖此选择，或者设置 CMAKE_INCLUDE_PATH 变量来修改搜索路径。

故障排查¶

如果未能自动找到 SDL 库，则 SDL_LIBRARY_TEMP 变量可能为空，且 SDL_LIBRARY 将不会被设置。这通常意味着 CMake 无法定位 SDL 库（例如 SDL.dll, libSDL.so, SDL.framework 等）。要解决此问题，请手动将 SDL_LIBRARY_TEMP 设置为正确路径并重新配置项目。同样，如果 SDLMAIN_LIBRARY 未设置，也可能需要手动指定。这些变量用于构建最终的 SDL_LIBRARY 值。如果它们未设置，SDL_LIBRARY 将保持未定义状态。

已弃用变量¶

以下变量提供用于向后兼容性

SDL_VERSION_STRING

3.19 版本弃用：已被同值的 SDL_VERSION 取代。

如果找到，包含 SDL 版本的易读字符串。

示例¶

查找 SDL 库并将其链接到项目目标

find_package(SDL)
target_link_libraries(project_target PRIVATE SDL::SDL)

代码说明

  • find_package(SDL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE SDL::SDL)：调用 target_link_libraries() 命令，为目标链接依赖库。

使用 SDL 2 版本时，上游包直接提供 SDL2::SDL2 导入目标。它可以在项目中直接使用，而无需使用此模块。

find_package(SDL2)
target_link_libraries(project_target PRIVATE SDL2::SDL2)

代码说明

  • find_package(SDL2)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE SDL2::SDL2)：调用 target_link_libraries() 命令，为目标链接依赖库。

同样，对于 SDL 3 版本：

find_package(SDL3)
target_link_libraries(project_target PRIVATE SDL3::SDL3)

代码说明

  • find_package(SDL3)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE SDL3::SDL3)：调用 target_link_libraries() 命令，为目标链接依赖库。
