GetPrerequisites¶

3.16 版本后已弃用：请改用 file(GET_RUNTIME_DEPENDENCIES)。

此模块提供了用于分析和列出可执行文件或共享库文件的依赖项（先决条件）的命令。这些命令会列出可执行文件或共享库所需的共享库（.dll、.dylib 或 .so 文件）。

使用 CMake 加载此模块

include(GetPrerequisites)

代码说明

  • include(GetPrerequisites)：调用 include() 命令，加载并执行另一个 CMake 脚本。

此模块使用以下特定于平台的工具来确定依赖项：

  • dumpbin (Windows（微软桌面系统）)

  • objdump (Windows（微软桌面系统） 上的 MinGW)

  • ldd (Linux/Unix)

  • otool (Apple 操作系统)

3.16 版本变更：如果设置了 CMAKE_OBJDUMP 变量，将使用该变量指定的工具。

命令¶

本模块提供以下命令

  • get_prerequisites()

  • list_prerequisites()

  • list_prerequisites_by_glob()

  • gp_append_unique()

  • is_file_executable()

  • gp_item_default_embedded_path() (项目可以使用 gp_item_default_embedded_path_override() 来覆盖它)

  • gp_resolve_item() (项目可以使用 gp_resolve_item_override() 来覆盖它)

  • gp_resolved_file_type() (项目可以使用 gp_resolved_file_type_override() 来覆盖它)

  • gp_file_type()

get_prerequisites¶

获取指定目标所需的共享库文件列表。

get_prerequisites(<target> <prerequisites-var> <exclude-system> <recurse>
                  <exepath> <dirs> [<rpaths>])

代码说明

  • get_prerequisites(<target> <prerequisites-var> <exclude-system> <recurse>：这一行继续补充当前命令或表达式的内容。

  • <exepath> <dirs> [<rpaths>])：这一行继续补充当前命令或表达式的内容。

名为 <prerequisites-var> 的变量在首次进入此命令时应为空。退出时，<prerequisites-var> 将包含所需共享库文件的列表。

参数为

<target>

可执行文件或共享库文件的完整路径。

<prerequisites-var>

用于保存结果的 CMake 变量名。

<exclude-system>

如果设置为 1，则排除系统先决条件；如果设置为 0，则包含它们。

<recurse>

如果设置为 1，则递归查找所有先决条件；如果设置为 0，则仅列出直接先决条件。

<exepath>

在 Apple 操作系统上用于 @executable_path 替换的顶级可执行文件路径。

<dirs>

可能找到库的路径列表：当给定目标不包含任何路径信息时，优先搜索这些路径。然后还会搜索标准系统位置：PATH、框架位置、/usr/lib 等。

<rpaths>

可执行文件或库的可选运行时搜索路径，有助于查找文件。

3.14 版本新增：在调用此命令之前，可以将 GET_PREREQUISITES_VERBOSE 变量设置为 true 以启用详细输出。

list_prerequisites¶

打印一条消息，列出指定目标的先决条件。

list_prerequisites(<target> [<recurse> [<exclude-system> [<verbose>]]])

代码说明

  • list_prerequisites(<target> [<recurse> [<exclude-system> [<verbose>]]])：调用 list_prerequisites() 命令，具体行为由这一行传入的参数决定。

参数为

<target>

共享库或可执行文件的名称，或者是共享库或可执行文件的完整路径。

<recurse>

如果设置为 1，则递归查找所有先决条件；如果设置为 0，则仅列出直接先决条件。

<exclude-system>

如果设置为 1，则排除系统先决条件；如果设置为 0，则包含它们。

<verbose>

如果设置为 0，则仅打印先决条件的完整路径名。如果设置为 1，则显示额外信息。

list_prerequisites_by_glob¶

打印匹配 glob 模式的共享库和可执行文件的先决条件。

list_prerequisites_by_glob(<GLOB|GLOB_RECURSE>
                           <glob-exp>
                           [<optional-args>...])

代码说明

  • list_prerequisites_by_glob(<GLOB|GLOB_RECURSE>：这一行继续补充当前命令或表达式的内容。

  • <glob-exp>：这一行继续补充当前命令或表达式的内容。

  • [<optional-args>...])：这一行继续补充当前命令或表达式的内容。

参数为

GLOB 或 GLOB_RECURSE

Globbing 模式，决定是仅遍历匹配项还是递归遍历其子目录。

<glob-exp>

与 file(GLOB) 或 file(GLOB_RECURSE) 一起使用的 globbing 表达式，用于检索匹配文件列表。如果匹配文件是可执行文件，则列出其先决条件。

<optional-args>...

提供的任何其他（可选）参数将作为可选参数传递给 list_prerequisite() 调用。

gp_append_unique¶

仅当值不在列表中时，才将其追加到列表中。

gp_append_unique(<list-var> <value>)

代码说明

  • gp_append_unique(<list-var> <value>)：调用 gp_append_unique() 命令，具体行为由这一行传入的参数决定。

参数为

<value>

要追加到列表中的值。

<list-var>

列表变量名称，仅当值不在列表中时，该值才会被追加。

is_file_executable¶

检查给定文件是否为二进制可执行文件。

is_file_executable(<file> <result-var>)

代码说明

  • is_file_executable(<file> <result-var>)：调用 is_file_executable() 命令，具体行为由这一行传入的参数决定。

如果 <file> 是二进制可执行文件，则此命令将 <result-var> 设置为 1；否则设置为 0。

gp_item_default_embedded_path¶

确定指定项的参考路径。

gp_item_default_embedded_path(<item> <default-embedded-path-var>)

代码说明

  • gp_item_default_embedded_path(<item> <default-embedded-path-var>)：调用 gp_item_default_embedded_path() 命令，具体行为由这一行传入的参数决定。

此命令在 <item> 嵌入在包内时确定其参考路径，并将其存储到变量 <default-embedded-path-var> 中。

项目可以通过定义自定义 gp_item_default_embedded_path_override() 命令来覆盖此命令。

gp_resolve_item¶

将给定项解析为现有的完整路径文件，并将其存储到变量中。

gp_resolve_item(<context> <item> <exepath> <dirs> <resolved-item-var>
                [<rpaths>])

代码说明

  • gp_resolve_item(<context> <item> <exepath> <dirs> <resolved-item-var>：这一行继续补充当前命令或表达式的内容。

  • [<rpaths>])：这一行继续补充当前命令或表达式的内容。

参数为

<context>

在 Apple 操作系统上用于 @loader_path 替换的顶级加载路径。解析项时，@loader_path 引用将相对于给定上下文值的目录（推测为另一个库）进行解析。

<item>

要解析的项。

<exepath>

请参阅 get_prerequisites() 中的参数说明。

<dirs>

请参阅 get_prerequisites() 中的参数说明。

<resolved-item-var>

用于存储已解析项的结果变量。

<rpaths>

请参阅 get_prerequisites() 中的参数说明。

项目可以通过定义自定义 gp_resolve_item_override() 命令来覆盖此命令。

gp_resolved_file_type¶

确定给定文件的类型。

gp_resolved_file_type(<original-file> <file> <exepath> <dirs> <type-var>
                      [<rpaths>])

代码说明

  • gp_resolved_file_type(<original-file> <file> <exepath> <dirs> <type-var>：这一行继续补充当前命令或表达式的内容。

  • [<rpaths>])：这一行继续补充当前命令或表达式的内容。

此命令相对于 <original-file> 确定 <file> 的类型。先决条件的最终类型存储在 <type-var> 变量中。

如有必要，使用 <exepath> 和 <dirs> 来解析非绝对路径的 <file> 值——但仅限于非嵌入项。

<rpaths>

请参阅 get_prerequisites() 中的参数说明。

<type-var> 变量将被设置为以下值之一：

  • system

  • local

  • embedded (嵌入)

  • other (其他)

项目可以通过定义自定义 gp_resolved_file_type_override() 命令来覆盖此命令。

gp_file_type¶

gp_file_type(<original-file> <file> <type-var>)

代码说明

  • gp_file_type(<original-file> <file> <type-var>)：调用 gp_file_type() 命令，具体行为由这一行传入的参数决定。

此命令相对于 <original-file> 确定 <file> 的类型。先决条件的最终类型存储在 <type-var> 变量中。

示例¶

示例：基本用法¶

打印共享库的所有依赖项（包括系统库），并显示详细输出。

include(GetPrerequisites)
list_prerequisites("path/to/libfoo.dylib" 1 0 1)

代码说明

  • include(GetPrerequisites)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • list_prerequisites("path/to/libfoo.dylib" 1 0 1)：调用 list_prerequisites() 命令，具体行为由这一行传入的参数决定。

示例：升级代码¶

例如

include(GetPrerequisites)
# ...
gp_append_unique(keys "${key}")

代码说明

  • include(GetPrerequisites)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • # ...：这是 CMake 注释。此处省略了与当前要点无关的代码。

  • gp_append_unique(keys "${key}")：调用 gp_append_unique() 命令，具体行为由这一行传入的参数决定；变量说明：key：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

在新代码中，可以将 gp_append_unique() 替换为

if(NOT key IN_LIST keys)
  list(APPEND keys "${key}")
endif()

代码说明

  • if(NOT key IN_LIST keys)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；IN_LIST 表示判断左侧值是否包含在右侧列表变量中。

  • list(APPEND keys "${key}")：调用 list() 命令，对列表变量执行追加、删除、查询等操作；变量说明：key：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • endif()：这一行结束或切换上一段控制结构的分支。
