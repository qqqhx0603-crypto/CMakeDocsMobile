CheckCXXSymbolExists¶

此模块提供了一个用于检查 C++ 符号是否存在的命令。

在 CMake 项目中加载此模块，使用

include(CheckCXXSymbolExists)

代码说明

  • include(CheckCXXSymbolExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_cxx_symbol_exists¶

检查符号是否作为函数、变量或预处理器宏在 C++ 中存在

check_cxx_symbol_exists(<symbol> <headers> <variable>)

代码说明

  • check_cxx_symbol_exists(<symbol> <headers> <variable>)：调用 check_cxx_symbol_exists() 命令，具体行为由这一行传入的参数决定。

该命令用于检查在包含指定的头文件 <headers> 后，<symbol> 是否可用，并将结果存储在内部缓存变量 <variable> 中。多个头文件可以在一个参数中指定，作为一个使用 分号分隔列表 的字符串。

如果头文件将该符号定义为宏，则视为可用并假设其正常工作。如果符号被声明为函数或变量，该检查还会确保其链接成功（即，该符号必须存在于链接的库或目标文件中）。

类型、枚举值或 C++ 模板形式的符号无法被识别。对于这些情况，请考虑使用 CheckTypeSize 或 CheckSourceCompiles 模块。

此命令旨在检查 C++ 中出现的符号。对于 C 符号，请改用 CheckSymbolExists 模块。

注意

该命令对于（潜在的）重载函数符号是不可靠的。由于无法可靠地预测系统环境中的给定函数是否可能被定义为重载函数，或者在其他系统上是否可能是重载函数，亦或未来是否会成为重载函数，因此通常建议使用 CheckSourceCompiles 模块来检查任何函数符号（除非确信该函数在其他系统上不会被重载，且未来也不会被重载）。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

一个以空格分隔的字符串，用于传递给编译器的额外标志。分号分隔列表 在此无效。CMAKE_<LANG>_FLAGS 及其关联的配置特定变量 CMAKE_<LANG>_FLAGS_<CONFIG> 的内容会自动在这些变量之前添加到编译器命令中。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

以下示例展示了如何使用此模块检查预处理器宏 SEEK_SET 和来自 <cstdio> 头文件的 C++ 函数 std::fopen() 是否存在。

include(CheckCXXSymbolExists)

# Check for macro SEEK_SET
check_cxx_symbol_exists(SEEK_SET "cstdio" HAVE_SEEK_SET)

# Check for function std::fopen
check_cxx_symbol_exists(std::fopen "cstdio" HAVE_STD_FOPEN)

代码说明

  • include(CheckCXXSymbolExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • # Check for macro SEEK_SET：这是 CMake 注释。检查 SEEK_SET 宏是否可用。

  • check_cxx_symbol_exists(SEEK_SET "cstdio" HAVE_SEEK_SET)：调用 check_cxx_symbol_exists() 命令，具体行为由这一行传入的参数决定。

  • # Check for function std::fopen：这是 CMake 注释。检查注释所指的条件或功能是否可用。

  • check_cxx_symbol_exists(std::fopen "cstdio" HAVE_STD_FOPEN)：调用 check_cxx_symbol_exists() 命令，具体行为由这一行传入的参数决定。
