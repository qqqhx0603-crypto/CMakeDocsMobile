CheckFunctionExists¶

此模块提供了一个用于检查 C 函数是否存在的命令。

在 CMake 项目中加载此模块，使用

include(CheckFunctionExists)

代码说明

  • include(CheckFunctionExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_function_exists¶

检查 C 函数是否可以从系统库中链接，仅执行一次

check_function_exists(<function> <variable>)

代码说明

  • check_function_exists(<function> <variable>)：调用 check_function_exists() 命令，具体行为由这一行传入的参数决定。

该命令检查系统库中是否提供了 <function>，并将结果存储在内部缓存变量 <variable> 中。

注意

建议优先使用 CheckSymbolExists 或 CheckSourceCompiles 而非此命令，原因如下：

  • check_function_exists() 无法检测内联在头文件中或定义为预处理器宏的函数。

  • 由于调用约定不匹配，check_function_exists() 无法检测 32 位 Win32 API 中的任何函数。

  • check_function_exists() 仅验证链接，不验证该函数是否在系统头文件中声明。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

这是一个以空格分隔的字符串，用于传递给编译器的额外标志。以分号分隔的列表将无法工作。CMAKE_<LANG>_FLAGS 及其关联的配置专用 CMAKE_<LANG>_FLAGS_<CONFIG> 变量的内容会自动置于此变量内容之前并传递给编译器命令。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

示例：基本用法¶

在以下示例中，执行检查以确定链接器是否能识别 C 函数 fopen()，并将结果存储在 HAVE_FOPEN 内部缓存变量中。

include(CheckFunctionExists)

check_function_exists(fopen HAVE_FOPEN)

代码说明

  • include(CheckFunctionExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_function_exists(fopen HAVE_FOPEN)：调用 check_function_exists() 命令，具体行为由这一行传入的参数决定。

示例：缺少声明¶

如上所述，CheckSymbolExists 模块是检查 C 函数的首选，因为它还会验证函数是否已被声明或定义为宏。在以下示例中，该模块被用于检查函数可能未在系统头文件中声明的边缘情况。例如，在 macOS（苹果桌面系统） 上，fdatasync() 函数可能存在于 C 库中，但其声明未在 unistd.h 系统头文件中提供。

CMakeLists.txt¶
include(CheckFunctionExists)
include(CheckSymbolExists)

check_symbol_exists(fdatasync "unistd.h" HAVE_FDATASYNC)

# Check if fdatasync() is available in the C library.
if(NOT HAVE_FDATASYNC)
  check_function_exists(fdatasync HAVE_FDATASYNC_WITHOUT_DECL)
endif()

代码说明

  • include(CheckFunctionExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • include(CheckSymbolExists)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_symbol_exists(fdatasync "unistd.h" HAVE_FDATASYNC)：调用 check_symbol_exists() 命令，具体行为由这一行传入的参数决定。

  • # Check if fdatasync() is available in the C library.：这是 CMake 注释。检查注释所指的条件或功能是否可用。

  • if(NOT HAVE_FDATASYNC)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值。

  • check_function_exists(fdatasync HAVE_FDATASYNC_WITHOUT_DECL)：调用 check_function_exists() 命令，具体行为由这一行传入的参数决定。

  • endif()：这一行结束或切换上一段控制结构的分支。

在这种情况下，如果缺少声明，项目可以提供自己的声明。

example.c¶
#ifdef HAVE_FDATASYNC_WITHOUT_DECL
  extern int fdatasync(int);
#endif

代码说明

  • #ifdef HAVE_FDATASYNC_WITHOUT_DECL：这一行开始预处理条件分支，按宏条件决定后续代码是否参与编译。

  • extern int fdatasync(int);：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文；变量说明：fdatasync：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • #endif：这一行切换或结束预处理条件分支。
