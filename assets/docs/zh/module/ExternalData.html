本模块提供了管理存储在源代码树之外的数据文件的命令。

在 CMake 项目中加载此模块，使用

include(ExternalData)

代码说明

  • include(ExternalData)：调用 include() 命令，加载并执行另一个 CMake 脚本。

简介¶

使用此模块可以明确地引用存储在源代码树之外的数据文件，并在构建时从任意本地或远程的内容寻址位置获取它们。该模块提供的函数会识别语法为 DATA{<name>} 的参数，将其视为对外部数据的引用，并将其替换为这些数据在本地副本的完整路径，同时创建构建规则以获取和更新这些本地副本。

例如

include(ExternalData)
set(ExternalData_URL_TEMPLATES "file:///local/%(algo)/%(hash)"
                               "file:////host/share/%(algo)/%(hash)"
                               "http://data.org/%(algo)/%(hash)")
ExternalData_Add_Test(MyData
  NAME MyTest
  COMMAND MyExe DATA{MyInput.png}
  )
ExternalData_Add_Target(MyData)

代码说明

  • include(ExternalData)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • set(ExternalData_URL_TEMPLATES "file:///local/%(algo)/%(hash)"：这一行继续补充当前命令或表达式的内容；变量说明：ExternalData_URL_TEMPLATES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • "file:////host/share/%(algo)/%(hash)"：这一行继续补充当前命令或表达式的内容。

  • "http://data.org/%(algo)/%(hash)")：这一行继续补充当前命令或表达式的内容。

  • ExternalData_Add_Test(MyData：这一行继续补充当前命令或表达式的内容。

  • NAME MyTest：这一行继续补充当前命令或表达式的内容。

  • COMMAND MyExe DATA{MyInput.png}：这一行继续补充当前命令或表达式的内容；条件运算符：COMMAND 表示判断给定名称是否为可调用命令。

  • )：这一行继续补充当前命令或表达式的内容。

  • ExternalData_Add_Target(MyData)：调用 ExternalData_Add_Target() 命令，具体行为由这一行传入的参数决定。

当测试 MyTest 运行时，DATA{MyInput.png} 参数将被替换为磁盘上数据文件 MyInput.png 的真实实例的完整路径。如果源代码树包含一个内容链接，如 MyInput.png.md5，那么 MyData 目标会在构建树中创建一个真实的 MyInput.png。

模块函数¶

ExternalData_Expand_Arguments¶

ExternalData_Expand_Arguments 函数评估其参数中的 DATA{} 引用，并构建一个新的参数列表。

ExternalData_Expand_Arguments(
  <target>   # Name of data management target
  <outVar>   # Output variable
  [args...]  # Input arguments, DATA{} allowed
  )

代码说明

  • ExternalData_Expand_Arguments(：这一行继续补充当前命令或表达式的内容。

  • <target> # Name of data management target：这一行继续补充当前命令或表达式的内容。

  • <outVar> # Output variable：这一行继续补充当前命令或表达式的内容。

  • [args...] # Input arguments, DATA{} allowed：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

它将每个参数中的 DATA{} 引用替换为在 <target> 构建后将存在的磁盘上真实数据文件的完整路径。

ExternalData_Add_Test¶

ExternalData_Add_Test 函数是对 CMake add_test() 命令的包装，但支持在参数中使用 DATA{} 引用。

ExternalData_Add_Test(
  <target>   # Name of data management target
  ...        # Arguments of add_test(), DATA{} allowed
  )

代码说明

  • ExternalData_Add_Test(：这一行继续补充当前命令或表达式的内容。

  • <target> # Name of data management target：这一行继续补充当前命令或表达式的内容。

  • ... # Arguments of add_test(), DATA{} allowed：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

它通过 ExternalData_Expand_Arguments 传递其参数，然后使用结果调用 add_test() 命令。

3.31 版本变更：如果 <target> 之后的参数定义了一个带有可执行文件的测试，且该文件是一个 CMake 目标，则该目标 TEST_LAUNCHER 和 CROSSCOMPILING_EMULATOR 属性中的空值将被保留。参见策略 CMP0178。

ExternalData_Add_Target¶

ExternalData_Add_Target 函数创建一个自定义目标，用于管理存储在外部的数据文件的本地实例。

ExternalData_Add_Target(
  <target>                  # Name of data management target
  [SHOW_PROGRESS <ON|OFF>]  # Show progress during the download
  )

代码说明

  • ExternalData_Add_Target(：这一行继续补充当前命令或表达式的内容。

  • <target> # Name of data management target：这一行继续补充当前命令或表达式的内容。

  • [SHOW_PROGRESS <ON|OFF>] # Show progress during the download：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

它根据需要创建自定义命令，以使数据文件可供本模块提供的其他函数先前评估的每个 DATA{} 引用使用。数据文件可以从 ExternalData_URL_TEMPLATES 变量中指定的 URL 模板之一获取，也可以在 ExternalData_OBJECT_STORES 变量中指定的路径之一本地找到。

3.20 版本新增：可以传递 SHOW_PROGRESS 参数以抑制下载对象期间的进度信息。如果未提供，对于 Ninja 和 Ninja Multi-Config 生成器，它默认为 OFF，否则为 ON。

通常只需要一个目标来管理项目内的所有外部数据。在配置结束时，所有数据引用都被处理后调用此函数一次。

模块变量¶

以下变量用于配置行为。它们应该在调用本模块提供的任何函数之前设置。

ExternalData_BINARY_ROOT¶

ExternalData_BINARY_ROOT 变量可设置为保存由扩展后的 DATA{} 引用命名的数据文件的目录。默认值为 CMAKE_BINARY_DIR。目录布局将镜像 ExternalData_SOURCE_ROOT 下的内容链接布局。

ExternalData_CUSTOM_SCRIPT_<key>¶

版本 3.2 中新增。

指定由 ExternalData_URL_TEMPLATES 列表条目中 <key> 标识的 .cmake 自定义获取脚本的完整路径。参见 自定义获取脚本。

ExternalData_HTTPHEADERS¶

4.0 版本新增。

ExternalData_HTTPHEADERS 变量可用于提供 HTTP 标头列表，每个元素包含一个格式为 Key: Value 的标头。参见 file(DOWNLOAD) 命令的 HTTPHEADER 选项。

ExternalData_LINK_CONTENT¶

ExternalData_LINK_CONTENT 变量可设置为支持的哈希算法名称，以启用将 DATA{} 语法引用的真实数据文件自动转换为内容链接。对于每个这样的 <file>，都会创建一个名为 <file><ext> 的内容链接。原始文件被重命名为 .ExternalData_<algo>_<hash> 的格式，以便为将来将其传输到 URL 模板列表中的某个位置做好准备（传输方式超出本模块范围）。为内容链接创建的数据获取规则将在无法使用任何 URL 模板找到该对象时，使用暂存的对象。

ExternalData_NO_SYMLINKS¶

3.3 版本中新增。

由扩展后的 DATA{} 引用命名的真实数据文件可以在某些平台上通过符号链接在 ExternalData_BINARY_ROOT 下提供。ExternalData_NO_SYMLINKS 变量可设置为禁用符号链接的使用，并启用副本的使用。

ExternalData_OBJECT_STORES¶

ExternalData_OBJECT_STORES 变量可设置为一组本地目录列表，这些目录使用 <dir>/%(algo)/%(hash) 的布局来存储对象。这些目录将被优先搜索以查找所需对象。如果对象在任何存储中都不可用，它将使用 URL 模板远程获取，并添加到列出的第一个本地存储中。如果没有指定存储，默认位置是构建树内的一个位置。

ExternalData_SERIES_PARSE¶
ExternalData_SERIES_PARSE_PREFIX¶
ExternalData_SERIES_PARSE_NUMBER¶
ExternalData_SERIES_PARSE_SUFFIX¶
ExternalData_SERIES_MATCH¶

参见 引用文件序列。

ExternalData_SOURCE_ROOT¶

ExternalData_SOURCE_ROOT 变量可设置为包含任何由 DATA{} 引用命名的路径的最高级源代码目录。默认值为 CMAKE_SOURCE_DIR。ExternalData_SOURCE_ROOT 和 CMAKE_SOURCE_DIR 必须指向同一个源代码分发包内的目录（例如，它们在同一个 tarball 中）。

ExternalData_TIMEOUT_ABSOLUTE¶

ExternalData_TIMEOUT_ABSOLUTE 变量设置下载的绝对超时时间（秒），默认值为 300 秒。设置为 0 以禁用强制执行。

ExternalData_TIMEOUT_INACTIVITY¶

ExternalData_TIMEOUT_INACTIVITY 变量设置下载的非活动超时时间（秒），默认值为 60 秒。设置为 0 以禁用强制执行。

ExternalData_URL_ALGO_<algo>_<key>¶

3.3 版本中新增。

当获取通过哈希算法 <algo> 引用的对象时，指定一个自定义 URL 组件，用于替换 %(algo:<key>) 形式的 URL 模板占位符，其中 <key> 是有效的 C 标识符。如果未定义，对于任何 <key>，默认的 URL 组件仅为 <algo>。

ExternalData_URL_TEMPLATES¶

ExternalData_URL_TEMPLATES 可设置为提供 URL 模板列表，并在每个模板中使用 %(algo) 和 %(hash) 占位符。数据获取规则通过将哈希算法名称替换为 %(algo)，将哈希值替换为 %(hash)，依次尝试每个 URL 模板。或者，可以使用 %(algo:<key>) 结合 ExternalData_URL_ALGO_<algo>_<key> 变量，以在远程 URL 中获得更大的灵活性。

引用文件¶

引用单个文件¶

DATA{} 语法是字面意义上的，<name> 是源代码树内的完整或相对路径。源代码树必须在 <name> 处包含一个真实的数据文件，或者在 <name><ext> 处包含一个“内容链接”，该链接包含使用对应于 <ext> 的哈希算法的真实文件哈希值。例如，参数 DATA{img.png} 可以由当前源代码目录中的真实 img.png 文件，或者包含其 MD5 和的 img.png.md5 文件满足。

3.8 版本新增：支持同名但使用不同哈希算法的多个内容链接（例如 img.png.sha256 和 img.png.sha1），只要它们都对应于同一个真实文件。这允许从由不同哈希算法索引的源中获取对象。

引用文件序列¶

DATA{} 语法可以通过 DATA{<name>,:} 形式告知获取文件序列，其中 : 是字面意义上的。如果源代码树包含一组类似序列命名的文件或内容链接，则引用其中一个成员将添加获取所有成员的规则。虽然获取了序列的所有成员，但只有 DATA{} 参数最初命名的文件会被替换。默认配置识别以 #.ext、_#.ext、.#.ext 或 -#.ext 结尾的文件序列名，其中 # 是一串十进制数字，.ext 是任何单个扩展名。使用正则表达式配置它，该正则表达式解析 <name> 末尾的 <number> 和 <suffix> 部分。

ExternalData_SERIES_PARSE - 格式为 (<number>)(<suffix>)$ 的正则表达式。

对于更复杂的情况，请设置：

  • ExternalData_SERIES_PARSE - 至少包含两个 () 组的正则表达式。

  • ExternalData_SERIES_PARSE_PREFIX - <prefix> 的正则表达式组号（如果有）。

  • ExternalData_SERIES_PARSE_NUMBER - <number> 的正则表达式组号。

  • ExternalData_SERIES_PARSE_SUFFIX - <suffix> 的正则表达式组号。

配置序列编号匹配，使用匹配序列成员 <prefix><number><suffix> 中 <number> 部分的正则表达式。

ExternalData_SERIES_MATCH - 匹配所有序列成员中 <number> 的正则表达式。

请注意，序列的 <suffix> 不包括哈希算法扩展名。

引用关联文件¶

DATA{} 语法也可以匹配与命名文件相关联且包含在同一目录中的文件。关联文件可以使用语法 DATA{<name>,<opt1>,<opt2>,...} 的选项指定。每个选项可以通过名称指定一个文件，或者使用 REGEX:<regex> 语法使用正则表达式匹配文件名。例如，参数：

DATA{MyData/MyInput.mhd,MyInput.img}                   # File pair
DATA{MyData/MyFrames00.png,REGEX:MyFrames[0-9]+\\.png} # Series

代码说明

  • DATA{MyData/MyInput.mhd,MyInput.img} # File pair：这一行给出示例文本或输出，用来说明：DATA{MyData/MyInput.mhd,MyInput.img} # File pair

  • DATA{MyData/MyFrames00.png,REGEX:MyFrames[0-9]+\\.png} # Series：这一行给出示例文本或输出，用来说明：DATA{MyData/MyFrames00.png,REGEX:MyFrames[0-9]+\\.png} # Series

将在命令行上传递 MyInput.mha 和 MyFrames00.png，但确保关联文件存在于它们旁边。

引用目录¶

DATA{} 语法可以使用尾部斜杠和关联文件列表来引用目录。形式 DATA{<name>/,<opt1>,<opt2>,...} 添加获取目录中匹配关联文件选项之一的任何文件的规则。例如，参数 DATA{MyDataDir/,REGEX:.*} 将在命令行上传递 MyDataDir 目录的完整路径，并确保该目录包含与 MyDataDir 源代码目录中每个文件或内容链接相对应的文件。

3.3 版本新增：为了匹配子目录中的关联文件，请指定 RECURSE: 选项，例如 DATA{MyDataDir/,RECURSE:,REGEX:.*}。

自定义获取脚本¶

版本 3.2 中新增。

当必须从 ExternalData_URL_TEMPLATES 变量中指定的 URL 模板之一获取数据文件时，它通常使用 file(DOWNLOAD) 命令下载。可以通过使用 ExternalDataCustomScript://<key>/<loc> 形式的 URL 模板来指定使用自定义获取脚本。<key> 必须是 C 标识符，<loc> 必须包含 %(algo) 和 %(hash) 占位符。必须将与键对应的变量 ExternalData_CUSTOM_SCRIPT_<key> 设置为 .cmake 脚本文件的完整路径。该脚本将被包含以执行实际的获取操作，并提供以下变量：

ExternalData_CUSTOM_LOCATION¶

加载自定义获取脚本时，此变量设置为 URL 的位置部分，其中将包含替换后的哈希算法名称和内容哈希值。

ExternalData_CUSTOM_FILE¶

加载自定义获取脚本时，此变量设置为脚本必须将获取的内容存储在其中的文件的完整路径。文件的名称未指定，不应以任何方式进行解释。

自定义获取脚本应将获取的内容存储在文件中，或设置变量：

ExternalData_CUSTOM_ERROR¶

当自定义获取脚本无法获取请求的内容时，它必须将此变量设置为简短的单行消息，描述失败原因。
