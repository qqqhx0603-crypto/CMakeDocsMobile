FindMPI¶

查找消息传递接口（MPI）实现

find_package(MPI [<version>] [COMPONENTS <langs>...] [...])

消息传递接口（MPI）是一个用于编写高性能分布式内存并行应用程序的库，通常部署在集群上。MPI 是一个由 MPI 论坛定义的标准接口，目前有许多可用的实现版本。

3.10 版本新增: 该模块进行了重大重构：增加了许多新变量、按语言划分的组件，并支持更广泛的运行时环境。

组件¶

本模块支持可选组件，可以通过 find_package() 命令指定，以控制要搜索的 MPI 语言

find_package(MPI [COMPONENTS <langs>...])

支持的组件包括

C

3.10 版本新增。

查找 MPI C API。

CXX

3.10 版本新增。

查找可用于 C++ 的 MPI C API。

MPICXX

3.10 版本新增。

查找 MPI-3 中已移除的 MPI-2 C++ API。

Fortran

3.10 版本新增。

查找 MPI Fortran API。

如果不指定任何组件，模块将根据项目中启用的语言，自动搜索 C、CXX 和 Fortran 组件。

导入的目标¶

此模块提供以下 导入目标

MPI::MPI_<lang>

版本 3.9 中添加。

封装了使用语言 <lang> 编写 MPI 程序所需的使用需求的目标（如果已找到 MPI）。<lang> 是上述列表中指定的组件名称。

结果变量¶

此模块定义了以下变量

MPI_FOUND

布尔值，表示是否找到了所有请求组件（语言）的 MPI 设置。如果不指定组件，则在该变量表示所有已启用语言的 MPI 设置均被检测到时设为真（true）。注意，MPICXX 组件不会影响此变量。

MPI_VERSION

在所有请求的语言中检测到的 MPI 最低版本；如果未指定组件，则为所有已启用语言中的最低版本。

本模块将在 CMake 项目中为每种语言设置以下变量，其中 <lang> 为 C、CXX 或 Fortran 之一

MPI_<lang>_FOUND

布尔值，表示已找到 <lang> 的 MPI 设置，并且简单的 MPI 测试程序可以使用提供的设置进行编译。

MPI_<lang>_COMPILER

如果存在，则为 <lang> 的 MPI 编译器。

MPI_<lang>_COMPILE_OPTIONS

<lang> 中 MPI 程序的编译选项，以 分号分隔的列表 给出。

MPI_<lang>_COMPILE_DEFINITIONS

<lang> 中 MPI 程序的编译定义，以 分号分隔的列表 给出。

MPI_<lang>_INCLUDE_DIRS

MPI 头文件的包含路径。

MPI_<lang>_LINK_FLAGS

MPI 程序的链接器标志。

MPI_<lang>_LIBRARIES

链接 MPI 程序所需的所有库。

将定义以下指示存在哪些绑定的变量

MPI_MPICXX_FOUND

布尔值，表示 MPI-2 C++ 绑定是否存在（在 MPI-2 中引入，在 MPI-3 中移除）。

MPI_Fortran_HAVE_F77_HEADER

如果 Fortran 77 头文件 <mpif.h> 可用，则为真。

MPI_Fortran_HAVE_F90_MODULE

如果 Fortran 90 模块 mpi 可用于访问 MPI，则为真（仅限 MPI-2 及更高版本）。

MPI_Fortran_HAVE_F08_MODULE

如果 Fortran 2008 模块 mpi_f08 可用于 MPI 程序，则为真（仅限 MPI-3 及更高版本）。

如果可能，本模块将确定 MPI 版本。检测 MPI 版本的设施是在 MPI-1.2 中引入的，因此无法在更旧的 MPI 版本中找到。

MPI_<lang>_VERSION

MPI 发行版为 <lang> 实现的 MPI 版本。

MPI_<lang>_VERSION_MAJOR

MPI 发行版为 <lang> 实现的 MPI 主版本号。

MPI_<lang>_VERSION_MINOR

MPI 发行版为 <lang> 实现的 MPI 次版本号。

请注意，没有针对通过 <mpi.h> 访问的 C 绑定的变量，因为 MPI 标准始终要求此绑定在 C 和 C++ 代码中均能正常工作。

对于运行 MPI 程序，本模块设置以下变量

MPIEXEC_EXECUTABLE

用于运行 MPI 程序的可执行文件（如果存在）。

MPIEXEC_NUMPROC_FLAG

在提供要运行的处理器数量之前，传递给 mpiexec 的标志。

MPIEXEC_MAX_NUMPROCS

要使用的 MPI 处理器数量。默认为在宿主系统上检测到的处理器数量。

MPIEXEC_PREFLAGS

在要运行的可执行文件之前直接传递给 mpiexec 的标志。

MPIEXEC_POSTFLAGS

在其他标志之后传递给 mpiexec 的标志。

用于定位 MPI 的变量¶

本模块对 MPI 实现执行四步搜索

  1. 搜索 MPIEXEC_EXECUTABLE，如果找到，则使用其基础目录。

  2. 检查编译器是否内置了 MPI 支持。如果用户将编译器包装器作为 CMAKE_<LANG>_COMPILER 传递，或者使用 Cray 系统编译器包装器，则属于这种情况。

  3. 尝试寻找 MPI 编译器包装器，并从中确定编译器信息。

  4. 尝试通过猜测设置来寻找不提供此类包装器的 MPI 实现。目前仅支持 Windows 上的 Microsoft MPI 和 MPICH2。

为了控制 MPIEXEC_EXECUTABLE 步骤，可以设置以下变量

MPIEXEC_EXECUTABLE

手动指定 mpiexec 的位置。

MPI_HOME

指定 MPI 安装的基础目录。

ENV{MPI_HOME}

用于指定 MPI 安装基础目录的环境变量。

ENV{I_MPI_ROOT}

用于指定 MPI 安装基础目录的环境变量。

为了控制编译器包装器步骤，可以设置以下变量

MPI_<lang>_COMPILER

搜索指定的编译器包装器并使用它。

MPI_<lang>_COMPILER_FLAGS

在侦测期间传递给 MPI 编译器包装器的标志。某些编译器包装器支持在传递特定标志时链接调试或跟踪库，此变量可用于获取它们。

MPI_COMPILER_FLAGS

如果未给出语言特定的标志，则用于初始化 MPI_<lang>_COMPILER_FLAGS。默认为空。

MPI_EXECUTABLE_SUFFIX

追加到所有正在查找的名称后的后缀。例如，在 Debian 及其衍生版上，可以将其设置为 .mpich 或 .openmpi 以优先选择其中之一。

为了控制猜测步骤，可以设置以下变量

MPI_GUESS_LIBRARY_NAME

有效值为 MSMPI 和 MPICH2。如果设置，则仅搜索给定的库。默认情况下，如果两者都可用，则 MSMPI 优先于 MPICH2。这也将 MPI_SKIP_COMPILER_WRAPPER 变量设置为 true，该设置可以被覆盖。

可以使用以下控制变量跳过每个搜索步骤

MPI_ASSUME_NO_BUILTIN_MPI

如果为真，模块假设编译器本身不提供 MPI 实现，并跳转到第 2 步。

MPI_SKIP_COMPILER_WRAPPER

如果为真，则不会搜索编译器包装器。

MPI_SKIP_GUESSING

如果为真，将跳过猜测步骤。

此外，可以使用以下控制变量来更改搜索行为

MPI_CXX_SKIP_MPICXX

添加一些禁用 MPI-2 C++ 绑定的定义。目前支持 MPICH、Open MPI、Platform MPI 及其衍生版本（例如 MVAPICH 或 Intel MPI）。

如果模块内部变量 MPI_<lang>_WORKS 的查找过程失败，则说明检测到的或传递给模块的设置无效，甚至连简单的 MPI 测试程序都无法编译。

如果所有这些参数仍不足以找到正确的 MPI 实现，用户可以通过在 MPI_<lang>_LIBRARIES 中指定库列表，并在 MPI_<lang>_ADDITIONAL_INCLUDE_DIRS 中指定包含目录列表来禁用整个自动检测过程。除了这两个变量外，还可以设置任何其他变量。模块将验证 MPI 设置并将其存储在缓存中。

缓存变量¶

变量 MPI_<lang>_INCLUDE_DIRS 将从以下变量中组装。

对于 C 和 CXX

MPI_<lang>_HEADER_DIR

磁盘上 <mpi.h> 头文件的位置。

对于 Fortran

MPI_Fortran_F77_HEADER_DIR

Fortran 77 头文件 <mpif.h> 的位置（如果存在）。

MPI_Fortran_MODULE_DIR

mpi 或 mpi_f08 模块的位置（如果可用）。

对于所有语言，还将额外考虑以下变量

MPI_<lang>_ADDITIONAL_INCLUDE_DIRS

除了普通包含目录之外，所需的路径的 分号分隔列表。

MPI_<include-name>_INCLUDE_DIR

<include-name> 所指的包含文件夹的路径变量。

MPI_<lang>_ADDITIONAL_INCLUDE_VARS

将添加到 <lang> 包含位置的 <include-name> 的 分号分隔列表。

变量 MPI_<lang>_LIBRARIES 将从以下变量中组装

MPI_<lib-name>_LIBRARY

名为 <lib-name> 的库的位置，供 MPI 使用。

MPI_<lang>_LIB_NAMES

将添加到 <lang> 包含位置的 <lib-name> 的 分号分隔列表。

使用 MPI 的高级变量¶

模块可以根据显式请求执行一些高级功能检测。

注意

如果不运行 MPI 测试程序，则无法执行以下检查。考虑在交叉编译期间 try_run() 行为的特殊注意事项。此外，运行 MPI 程序可能会导致其他问题，例如在某些系统上的防火墙通知。仅在绝对需要信息时才应启用这些检测。

如果将以下变量设置为真，将执行相应的搜索

MPI_DETERMINE_Fortran_CAPABILITIES

确定所有可用 Fortran 绑定的 MPI_SUBARRAYS_SUPPORTED 和 MPI_ASYNC_PROTECTS_NONBLOCKING 的值，并将其分别作为 MPI_Fortran_<binding>_SUBARRAYS 和 MPI_Fortran_<binding>_ASYNCPROT 提供，其中 <binding> 为 F77_HEADER、F90_MODULE 和 F08_MODULE 之一。

MPI_DETERMINE_LIBRARY_VERSION

对于每种语言，找到 MPI_Get_library_version 的输出并将其作为 MPI_<lang>_LIBRARY_VERSION_STRING 提供。此信息通常与 MPI 实现的运行时组件相关联，并可能因 <lang> 而异。注意，返回值完全由实现定义。此信息可用于标识 MPI 供应商，例如在多个第三方二进制文件中选择与 MPI 供应商匹配的正确版本。

已弃用变量¶

以下变量提供用于向后兼容性

MPI_COMPILER

2.8.5 版本后弃用: 使用 MPI_<lang>_COMPILER 代替。

MPI_LIBRARY

2.8.5 版本后弃用: 使用 MPI_<lang>_LIBRARIES 代替。

MPI_EXTRA_LIBRARY

2.8.5 版本后弃用: 使用 MPI_<lang>_LIBRARIES 代替。

MPI_COMPILE_FLAGS

2.8.5 版本后弃用: 使用 MPI_<lang>_COMPILE_OPTIONS 和 MPI_<lang>_COMPILE_DEFINITIONS 代替。

MPI_INCLUDE_PATH

2.8.5 版本后弃用: 使用 MPI_<lang>_INCLUDE_DIRS 代替。

MPI_LINK_FLAGS

2.8.5 版本后弃用: 使用 MPI_<lang>_LINK_FLAGS 代替。

MPI_LIBRARIES

2.8.5 版本后弃用: 使用 MPI_<lang>_LIBRARIES 代替。

MPI_<lang>_COMPILE_FLAGS

3.10 版本后弃用: 使用 MPI_<lang>_COMPILE_OPTIONS 和 MPI_<lang>_COMPILE_DEFINITIONS 代替。

MPI_<lang>_INCLUDE_PATH

3.10 版本后弃用: 使用 MPI_<lang>_INCLUDE_DIRS 获取内容，使用 MPI_<lang>_ADDITIONAL_INCLUDE_DIRS 指定文件夹。

MPIEXEC

3.10 版本后弃用: 使用 MPIEXEC_EXECUTABLE 代替。

示例¶

示例：基本用法¶

查找 MPI 并将导入的目标链接到项目目标

find_package(MPI)
target_link_libraries(example PRIVATE MPI::MPI_C)

示例：mpiexec 的使用¶

使用 MPIEXEC_EXECUTABLE 运行 MPI 应用程序时，通常应按如下方式使用所有 MPIEXEC_EXECUTABLE 标志。

在以下示例中，命令在进程中执行。<executable> 应替换为 MPI 程序，<args> 替换为传递给 MPI 程序的参数。

find_package(MPI)

if(MPI_FOUND)
  execute_process(
    COMMAND
      ${MPIEXEC_EXECUTABLE}
      ${MPIEXEC_NUMPROC_FLAG}
      ${MPIEXEC_MAX_NUMPROCS}
      ${MPIEXEC_PREFLAGS}
      <executable>
      ${MPIEXEC_POSTFLAGS}
      <args>
  )
endif()
