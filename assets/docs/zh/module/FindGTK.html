FindGTK¶

注意

此模块仅在类 Unix 系统上工作，并且最初是为不再维护的 GTK 1.x 分支设计的。请使用最新支持的 GTK 版本和 FindPkgConfig 模块来代替此模块查找 GTK。例如

find_package(PkgConfig REQUIRED)
pkg_check_modules(GTK REQUIRED IMPORTED_TARGET gtk4>=4.14)
target_link_libraries(example PRIVATE PkgConfig::GTK)

代码说明

  • find_package(PkgConfig REQUIRED)：调用 find_package() 命令，查找并加载外部依赖包。

  • pkg_check_modules(GTK REQUIRED IMPORTED_TARGET gtk4>=4.14)：调用 pkg_check_modules() 命令，具体行为由这一行传入的参数决定；运算符：>= 表示大于或等于比较。

  • target_link_libraries(example PRIVATE PkgConfig::GTK)：调用 target_link_libraries() 命令，为目标链接依赖库。

查找 GTK、glib 和 GTKGLArea

find_package(GTK [...])

代码说明

  • find_package(GTK [...])：调用 find_package() 命令，查找并加载外部依赖包。

GTK 是一个跨平台工具包，用于创建图形用户界面。

结果变量¶

此模块定义了以下变量

GTK_FOUND

布尔值，指示是否找到了 GTK。

GTK_GL_FOUND

布尔值，指示是否找到了 GTK 的 GL 特性。

GTK_INCLUDE_DIR

包含使用 GTK 所需头文件的包含目录。

GTK_LIBRARIES

使用 GTK 所需的链接库。

示例¶

查找 GTK 1.x 并创建一个 导入目标，该目标封装了其用于链接到项目目标的使用要求

find_package(GTK)

if(GTK_FOUND)
  add_library(GTK::GTK INTERFACE IMPORTED)
  set_target_properties(
    GTK::GTK
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${GTK_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${GTK_LIBRARIES}"
  )
endif()

target_link_libraries(example PRIVATE GTK::GTK)

代码说明

  • find_package(GTK)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(GTK_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • add_library(GTK::GTK INTERFACE IMPORTED)：调用 add_library() 命令，定义一个库目标。

  • set_target_properties(：这一行继续补充当前命令或表达式的内容。

  • GTK::GTK：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • INTERFACE_INCLUDE_DIRECTORIES "${GTK_INCLUDE_DIR}"：这一行继续补充当前命令或表达式的内容；变量说明：GTK_INCLUDE_DIR：包含目录、目录相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • INTERFACE_LINK_LIBRARIES "${GTK_LIBRARIES}"：这一行继续补充当前命令或表达式的内容；变量说明：GTK_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

  • target_link_libraries(example PRIVATE GTK::GTK)：调用 target_link_libraries() 命令，为目标链接依赖库。
