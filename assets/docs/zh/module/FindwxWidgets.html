FindwxWidgets¶

查找 wxWidgets 安装并提供在项目中使用的使用要求

find_package(wxWidgets [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(wxWidgets [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

wxWidgets（原名 wxWindows）是一个用于创建跨平台应用程序图形用户界面（GUI）的窗口工具包和工具库。

3.4 版本新增： 支持 find_package() 版本参数。

3.14 版本新增： 支持 OPTIONAL_COMPONENTS。

组件¶

wxWidgets 是一个模块化库。本模块支持通过组件指定要使用的模块。组件可以通过 find_package() 命令进行指定。

find_package(
  wxWidgets
  [COMPONENTS <components>...]
  [OPTIONAL_COMPONENTS <components>...]
)

代码说明

  • find_package(：这一行继续补充当前命令或表达式的内容。

  • wxWidgets：这一行继续补充当前命令或表达式的内容。

  • [COMPONENTS <components>...]：这一行继续补充当前命令或表达式的内容。

  • [OPTIONAL_COMPONENTS <components>...]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

支持的组件包括

base

查找提供任何 wxWidgets 代码所依赖的强制性类的库。实现 wxWidgets 的应用程序始终需要此组件。

core

查找提供基本 GUI 类（如 GDI 类或控件）的库。

gl

查找 OpenGL（开放图形库） 支持。

mono

查找 wxWidgets 单体（monolithic）库。

aui

查找高级用户界面（Advanced User Interface）停靠库。

net

查找提供网络访问的库。

webview

3.4 版本新增。

查找提供 Web 文档（HTML/CSS/JavaScript）渲染的库。

有关受支持 wxWidgets 组件的完整列表，请参阅上游文档。

如果未指定组件，本模块默认搜索 core 和 base 组件。

导入的目标¶

此模块提供以下 导入目标

wxWidgets::wxWidgets

在 3.27 版本中新增。

一个导入的接口目标，封装了所找到组件的 wxWidgets 使用要求，在找到 wxWidgets 时可用。

结果变量¶

此模块定义了以下变量

wxWidgets_FOUND

布尔值，表示是否找到（所请求版本的）wxWidgets 及其所有请求的组件。

wxWidgets_VERSION

版本 4.2 中添加。

找到的 wxWidgets 版本。

wxWidgets_INCLUDE_DIRS

WIN32 的包含目录，即在哪里查找 <wx/wx.h> 和 <wx/setup.h>；对于类 Unix 系统，此变量可能为空。

wxWidgets_LIBRARIES

wxWidgets 库的路径。

wxWidgets_LIBRARY_DIRS

编译时链接目录，对于在类 Unix 系统上设置 rpath 很有用。在 WIN32 环境中通常为空字符串。

wxWidgets_DEFINITIONS

包含编译/链接 WX 所需的编译定义，例如 WXUSINGDLL。

wxWidgets_DEFINITIONS_DEBUG

包含编译/链接 WX 调试版本所需的编译定义，例如 __WXDEBUG__。

wxWidgets_CXX_FLAGS

类 Unix 系统的包含目录和编译器标志，在 Windows（微软桌面系统） 上为空。本质上是 wx-config --cxxflags 的输出。

提示¶

本模块在调用 find_package(wxWidgets) 之前接受以下变量

WX_CONFIG

3.11 版本新增。

环境变量，用于手动指定除默认名称 wx-config 之外还要搜索的 wxWidgets 库配置提供程序可执行文件的名称。

WXRC_CMD

3.11 版本新增。

环境变量，用于手动指定除默认名称 wxrc 之外还要搜索的 wxWidgets 资源文件编译器可执行文件的名称。

有两种搜索分支：Windows（微软桌面系统） 风格和 Unix 风格。对于 Windows（微软桌面系统），将搜索以下变量，并在有多种选择时设置为默认值。如果默认值不符合预期，请更改它们（即，这些是选择配置时唯一应该更改的变量）。

wxWidgets_ROOT_DIR

wxWidgets 根目录（例如 C:/wxWidgets-3.2.0）。

wxWidgets_LIB_DIR

wxWidgets 库路径（例如 C:/wxWidgets-3.2.0/lib/vc_x64_lib）。

wxWidgets_CONFIGURATION

要使用的配置（例如 msw, mswd, mswu, mswunivud 等）

wxWidgets_EXCLUDE_COMMON_LIBRARIES

设置为 TRUE 以排除链接通常需要的库（例如 png, tiff, jpeg, zlib, webp, regex, expat, scintilla, lexilla 等）。

对于 Unix 风格，本模块使用 wx-config 实用程序。可以通过在 QtDialog 或 ccmake 界面中开启/关闭以下变量，在调试/发布、Unicode/ANSI、通用/非通用以及静态/共享之间进行选择：

wxWidgets_USE_DEBUG

如果启用，将搜索 wxWidgets 调试版本。

wxWidgets_USE_UNICODE

如果启用，将搜索 wxWidgets Unicode 版本。

wxWidgets_USE_UNIVERSAL

如果启用，将搜索 wxWidgets 通用（Universal）版本。

wxWidgets_USE_STATIC

如果启用，将链接静态 wxWidgets 库。

wxWidgets_CONFIG_OPTIONS

此变量可用于所有其他需要传递给 wx-config 实用程序的选项。例如，要使用在系统 /usr 安装前缀下找到的 base 工具包，请按如下方式设置变量（在调用 find_package() 命令之前）：

set(wxWidgets_CONFIG_OPTIONS --toolkit=base --prefix=/usr)

代码说明

  • set(wxWidgets_CONFIG_OPTIONS --toolkit=base --prefix=/usr)：调用 set() 命令，把变量 wxWidgets_CONFIG_OPTIONS 设为给定的值；变量说明：wxWidgets_CONFIG_OPTIONS：配置、选项相关变量，其精确取值和作用域以当前页面或脚本上下文为准；运算符：= 表示赋值或键值关联。

已弃用变量¶

以下变量提供用于向后兼容性

wxWidgets_VERSION_STRING

4.2 版本后已弃用： 请使用值相同的 wxWidgets_VERSION。

3.4 版本新增。

找到的 wxWidgets 版本。

wxWidgets_USE_FILE

4.2 版本后已弃用： 不要使用此变量，请直接包含 UsewxWidgets 模块。

include(UsewxWidgets)

代码说明

  • include(UsewxWidgets)：调用 include() 命令，加载并执行另一个 CMake 脚本。

用于在当前目录中使用 wxWidgets 的 UsewxWidgets 模块的路径。例如

find_package(wxWidgets)
if(wxWidgets_FOUND)
  include(${wxWidgets_USE_FILE})
endif()

代码说明

  • find_package(wxWidgets)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(wxWidgets_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • include(${wxWidgets_USE_FILE})：调用 include() 命令，加载并执行另一个 CMake 脚本；变量说明：wxWidgets_USE_FILE：文件相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • endif()：这一行结束或切换上一段控制结构的分支。

示例¶

示例：查找 wxWidgets¶

查找 wxWidgets 并将其设为必需（如果找不到 wxWidgets，处理将停止并报错）

find_package(wxWidgets REQUIRED)

代码说明

  • find_package(wxWidgets REQUIRED)：调用 find_package() 命令，查找并加载外部依赖包。

示例：使用导入的目标¶

查找 wxWidgets 并在项目中使用导入的目标

find_package(wxWidgets)
target_link_libraries(example PRIVATE wxWidgets::wxWidgets)

代码说明

  • find_package(wxWidgets)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(example PRIVATE wxWidgets::wxWidgets)：调用 target_link_libraries() 命令，为目标链接依赖库。

示例：使用组件¶

查找 wxWidgets 并指定组件

find_package(wxWidgets COMPONENTS gl core base OPTIONAL_COMPONENTS net)
target_link_libraries(example PRIVATE wxWidgets::wxWidgets)

代码说明

  • find_package(wxWidgets COMPONENTS gl core base OPTIONAL_COMPONENTS net)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(example PRIVATE wxWidgets::wxWidgets)：调用 target_link_libraries() 命令，为目标链接依赖库。

示例：单体 wxWidgets 构建¶

单体 wxWidgets 构建的示例用法

find_package(wxWidgets COMPONENTS mono)
target_link_libraries(example PRIVATE wxWidgets::wxWidgets)

代码说明

  • find_package(wxWidgets COMPONENTS mono)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(example PRIVATE wxWidgets::wxWidgets)：调用 target_link_libraries() 命令，为目标链接依赖库。

示例：使用变量¶

在 3.27 之前的 CMake 版本中，在导入目标尚不可用时查找和使用 wxWidgets

# Note that for MinGW users the order of libs is important.
find_package(wxWidgets COMPONENTS gl core base OPTIONAL_COMPONENTS net)

if(wxWidgets_FOUND)
  include(UsewxWidgets)
  # and for each of the project dependent executable/library targets:
  target_link_libraries(example ${wxWidgets_LIBRARIES})
endif()

  • find_package(wxWidgets COMPONENTS gl core base OPTIONAL_COMPONENTS net)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(wxWidgets_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • include(UsewxWidgets)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • # and for each of the project dependent executable/library targets:：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • target_link_libraries(example ${wxWidgets_LIBRARIES})：调用 target_link_libraries() 命令，为目标链接依赖库；变量说明：wxWidgets_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。
