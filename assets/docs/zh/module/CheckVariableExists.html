CheckVariableExists¶

此模块提供了一个用于检查 C 变量是否存在的命令。

在 CMake 项目中加载此模块，使用

include(CheckVariableExists)

代码说明

  • include(CheckVariableExists)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_variable_exists¶

一次性检查 C 变量是否存在

check_variable_exists(<var> <variable>)

代码说明

  • check_variable_exists(<var> <variable>)：调用 `check_variable_exists()` 命令，具体行为由这一行传入的参数决定。

此命令尝试编译并链接一个引用了指定 C 变量 <var> 的测试 C 程序。检查是否成功的布尔结果将存储在内部缓存变量 <variable> 中。

注意

建议优先使用 CheckSymbolExists 或 CheckSourceCompiles 来代替此命令，以获得更稳健的检测。此命令仅执行链接检查，无法检测变量是否在系统或库头文件中声明。它也无法检测可能定义为预处理器宏的变量。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

这是一个由空格分隔的字符串，包含要传递给编译器的额外标志。分号分隔的列表在此处无效。CMAKE_<LANG>_FLAGS 及其相关的特定配置变量 CMAKE_<LANG>_FLAGS_<CONFIG> 的内容会自动添加到该变量内容之前的编译器命令中。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

示例：基本用法¶

在以下示例中，将检查链接器是否能识别 C 变量 tzname，并将检查结果存储在内部缓存变量 PROJECT_HAVE_TZNAME 中

include(CheckVariableExists)

check_variable_exists(tzname PROJECT_HAVE_TZNAME)

代码说明

  • include(CheckVariableExists) check_variable_exists(tzname PROJECT_HAVE_TZNAME)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

示例：带链接库的隔离检查¶

在以下示例中，此模块与 CMakePushCheckState 模块结合使用，通过 CMAKE_REQUIRED_LIBRARIES 变量来链接所需的额外库。例如，在查找模块（find module）中，检查 Net-SNMP 库是否包含 usmHMAC192SHA256AuthProtocol 数组

include(CheckVariableExists)
include(CMakePushCheckState)

find_library(SNMP_LIBRARY NAMES netsnmp)

if(SNMP_LIBRARY)
  cmake_push_check_state(RESET)

  set(CMAKE_REQUIRED_LIBRARIES ${SNMP_LIBRARY})

  check_variable_exists(usmHMAC192SHA256AuthProtocol SNMP_HAVE_SHA256)

  cmake_pop_check_state()
endif()

代码说明

  • include(CheckVariableExists) include(CMakePushCheckState) find_library(SNMP_LIBRARY NAMES netsnmp) if(SNMP_LIBRARY) cmake_push_check_state(RESET) set(CMAKE_REQUIRED_LIBRARIES ${SNMP_LIBRARY}) check_variable_exists(usmHMAC192SHA256AuthProtocol SNMP_HAVE_SHA256) cmake_pop_check_state() endif()：调用 `include()` 命令，加载并执行另一个 CMake 脚本。
