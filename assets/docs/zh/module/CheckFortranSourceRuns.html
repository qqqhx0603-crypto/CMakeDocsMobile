CheckFortranSourceRuns¶

3.14 版新增。

该模块提供了一个用于检查 Fortran 源代码是否可以构建并运行的命令。

在 CMake 项目中加载此模块，使用

include(CheckFortranSourceRuns)

代码说明

  • include(CheckFortranSourceRuns)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_fortran_source_runs¶

检查给定的 Fortran 源代码是否能编译并链接成可执行文件，且随后能够成功运行。

check_fortran_source_runs(<code> <variable> [SRC_EXT <extension>])

代码说明

  • check_fortran_source_runs(<code> <variable> [SRC_EXT <extension>])：调用 `check_fortran_source_runs()` 命令，具体行为由这一行传入的参数决定。

在 <code> 中提供的 Fortran 源代码必须包含一个 Fortran program 单元。检查结果存储在由 <variable> 指定的内部缓存变量中。如果代码构建成功且运行时的退出代码为 0，则通过布尔真值表示成功。构建或运行失败则由布尔假值表示，例如空字符串或错误信息。

选项包括

SRC_EXT <extension>

默认情况下，用于检查的内部测试源文件将使用 .F90 文件扩展名。此选项可用于将扩展名更改为 .<extension>。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

一个以空格分隔的字符串，用于传递给编译器的额外标志。使用 分号分隔的列表 将无效。CMAKE_<LANG>_FLAGS 的内容及其相关的配置特定变量 CMAKE_<LANG>_FLAGS_<CONFIG> 会在编译器命令中自动添加在该变量内容之前。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

以下示例展示了如何使用该模块来检查 Fortran 源代码是否能够运行，并将检查结果存储在内部缓存变量 HAVE_COARRAY 中。

include(CheckFortranSourceRuns)

check_fortran_source_runs([[
  program test
  real :: x[*]
  call co_sum(x)
  end program
]] HAVE_COARRAY)

代码说明

  • include(CheckFortranSourceRuns) check_fortran_source_runs([[ program test real :: x[*] call co_sum(x) end program ]] HAVE_COARRAY)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。
