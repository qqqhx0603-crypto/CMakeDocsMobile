CSharpUtilities¶

版本 3.8 新增。

此工具模块旨在简化 CSharp/.NET 目标的配置，并提供了一系列命令，用于管理 Visual Studio（集成开发环境） 生成器（2010 及更新版本）下的 CSharp 目标。

在 CMake 项目中加载此模块，使用

include(CSharpUtilities)

代码说明

  • include(CSharpUtilities)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

本模块提供以下命令

主要命令

  • csharp_set_windows_forms_properties()

  • csharp_set_designer_cs_properties()

  • csharp_set_xaml_cs_properties()

辅助命令

  • csharp_get_filename_keys()

  • csharp_get_filename_key_base()

  • csharp_get_dependentupon_name()

主要命令¶

csharp_set_windows_forms_properties¶

为使用 Windows（微软桌面系统） Forms 设置源文件属性

csharp_set_windows_forms_properties([<files>...])

代码说明

  • csharp_set_windows_forms_properties([<files>...])：调用 `csharp_set_windows_forms_properties()` 命令，具体行为由这一行传入的参数决定。

<files>...

零个或多个相关源文件的列表，用于设置 VS_CSHARP_<tagname> 源文件属性。这通常包括后缀为 .cs、.resx 和 .Designer.cs 的文件。

当项目中的 CSharp 目标使用 Windows（微软桌面系统） Forms 时，请使用此命令。

该命令会在提供的文件列表中搜索以 .Designer.cs（设计器文件）或 .resx（资源文件）结尾的相关文件对。对于每一个此类文件，都会搜索对应的基础 .cs 文件（具有相同的文件名基）。一旦找到，将按下述方式设置 VS_CSHARP_<tagname> 源文件属性：

对于 .cs 文件：

  • VS_CSHARP_SubType "Form"

对于 .Designer.cs 文件（如果存在）：

  • VS_CSHARP_DependentUpon <cs-filename>

  • VS_CSHARP_DesignTime ""（如果先前已定义，则移除该标签）

  • VS_CSHARP_AutoGen ""（如果先前已定义，则移除该标签）

对于 .resx 文件（如果存在）：

  • VS_RESOURCE_GENERATOR ""（如果先前已定义，则移除该标签）

  • VS_CSHARP_DependentUpon <cs-filename>

  • VS_CSHARP_SubType "Designer"

csharp_set_designer_cs_properties¶

根据同级文件名，为 .Designer.cs 文件设置源文件属性

csharp_set_designer_cs_properties([<files>...])

代码说明

  • csharp_set_designer_cs_properties([<files>...])：调用 `csharp_set_designer_cs_properties()` 命令，具体行为由这一行传入的参数决定。

<files>...

零个或多个相关源文件的列表，用于设置 VS_CSHARP_<tagname> 源文件属性。这通常包括后缀为 .resx、.settings 和 .Designer.cs 的文件。

如果 CSharp 目标不使用 Windows（微软桌面系统） Forms（对于 Windows（微软桌面系统） Forms，请改用 csharp_set_windows_forms_properties()），则使用此命令。

该命令会在提供的列表中搜索以 .Designer.cs 结尾的文件（设计器文件）。对于每一个此类文件，它会查找具有相同文件名基但后缀不同的同级文件。如果找到匹配文件，则会根据匹配的后缀，在对应的 .Designer.cs 文件上设置相应的源文件属性。

如果匹配的是 .resx 文件：

  • VS_CSHARP_AutoGen "True"

  • VS_CSHARP_DesignTime "True"

  • VS_CSHARP_DependentUpon <resx-filename>

如果匹配的是 .cs 文件：

  • VS_CSHARP_DependentUpon <cs-filename>

如果匹配的是 .settings 文件：

  • VS_CSHARP_AutoGen "True"

  • VS_CSHARP_DesignTimeSharedInput "True"

  • VS_CSHARP_DependentUpon <settings-filename>

注意

由于 .Designer.cs 文件的源文件属性是根据找到的匹配项设置的，并且每个匹配项都会设置 VS_CSHARP_DependentUpon 源文件属性，因此每个 Designer.cs 文件应该只有一个匹配项。

csharp_set_xaml_cs_properties¶

为使用 Windows（微软桌面系统） Presentation Foundation (WPF) 和 XAML 设置源文件属性

csharp_set_xaml_cs_properties([<files>...])

代码说明

  • csharp_set_xaml_cs_properties([<files>...])：调用 `csharp_set_xaml_cs_properties()` 命令，具体行为由这一行传入的参数决定。

如果 CSharp 目标使用 WPF/XAML，请使用此命令。

<files>...

零个或多个相关源文件的列表，用于设置 VS_CSHARP_<tagname> 源文件属性。这通常包括后缀为 .cs、.xaml 和 .xaml.cs 的文件。

该命令会在提供的文件列表中搜索以 .xaml.cs 结尾的文件。对于每一个此类 XAML 后台代码文件，都会搜索一个具有相同文件名基的对应 .xaml 文件。如果找到，则会在该 .xaml.cs 文件上设置以下源文件属性：

  • VS_CSHARP_DependentUpon <xaml-filename>

辅助命令¶

这些命令由上述主命令调用，通常不直接使用。

csharp_get_filename_keys¶

计算一组规范化的键值，以便独立于 CMake 中给出的相对或绝对路径来识别源文件，并消除大小写敏感性的影响。

csharp_get_filename_keys(<variable> [<files>...])

代码说明

  • csharp_get_filename_keys(<variable> [<files>...])：调用 `csharp_get_filename_keys()` 命令，具体行为由这一行传入的参数决定。

<variable>

存储计算出的键列表的变量名。

<files>...

传递给 CSharp 目标的零个或多个源文件路径，使用如 add_library() 或 add_executable() 等命令。

该命令会对文件路径进行规范化，以确保一致地识别源文件。当源文件通过不同的路径形式添加到目标时，这非常有用。如果不进行规范化，CMake 可能会将 myfile.Designer.cs 和 ${CMAKE_CURRENT_SOURCE_DIR}/myfile.Designer.cs 视为不同的文件，这在设置源文件属性时可能会导致问题。

例如，以下代码将无法设置属性，因为文件路径不完全匹配：

add_library(lib myfile.cs ${CMAKE_CURRENT_SOURCE_DIR}/myfile.Designer.cs)

set_source_files_properties(
  myfile.Designer.cs
  PROPERTIES VS_CSHARP_DependentUpon myfile.cs
)

代码说明

  • add_library(lib myfile.cs ${CMAKE_CURRENT_SOURCE_DIR}/myfile.Designer.cs) set_source_files_properties( myfile.Designer.cs PROPERTIES VS_CSHARP_DependentUpon myfile.cs )：调用 `add_library()` 命令，定义一个库目标。

csharp_get_filename_key_base¶

返回键的完整文件路径和名称（不含扩展名）。

csharp_get_filename_key_base(<base> <key>)

代码说明

  • csharp_get_filename_key_base(<base> <key>)：调用 `csharp_get_filename_key_base()` 命令，具体行为由这一行传入的参数决定。

<base>

包含计算出的 <key> 基值的变量名，不含文件扩展名。

<key>

要计算基值的键。预期为来自 csharp_get_filename_keys() 的大写完整文件名。

csharp_get_dependentupon_name¶

计算一个可用于源文件属性 VS_CSHARP_<tagname> 的值，其中 <tagname> 为 DependentUpon。

csharp_get_dependentupon_name(<variable> <file>)

代码说明

  • csharp_get_dependentupon_name(<variable> <file>)：调用 `csharp_get_dependentupon_name()` 命令，具体行为由这一行传入的参数决定。

<variable>

包含结果值的变量名。该值包含 <file> 的名称（不含目录）。

<file>

要转换以用于 VS_CSHARP_DependentUpon 源文件属性值的文件名。
