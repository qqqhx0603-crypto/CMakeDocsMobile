FindPythonLibs¶

3.27 版本变更：仅当策略 CMP0148 未设置为 NEW 时，此模块才可用。

3.12 版本弃用：请改用 FindPython3、FindPython2 或 FindPython。

查找 Python 安装路径并确定其包含目录（include directories）和库的位置，以及用于链接的 Python 库名称。

find_package(PythonLibs [<version>] [...])

注意

当同时使用本模块和 FindPythonInterp 模块时，请在 find_package(PythonLibs) 之前调用 find_package(PythonInterp)。这能确保使用检测到的解释器版本来指导兼容库的选择，从而获得一致的 PYTHON_LIBRARIES 值。

结果变量¶

此模块定义了以下变量

PythonLibs_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）Python 库。

PYTHONLIBS_VERSION_STRING

找到的 Python 库版本。

PYTHON_LIBRARIES

使用 Python 时链接所需的库。

PYTHON_INCLUDE_DIRS

使用 Python 时所需的包含目录。

缓存变量¶

以下缓存变量也可以通过设置来指定要使用的 Python 安装路径：

PYTHON_LIBRARY

Python 库的路径。

PYTHON_INCLUDE_DIR

包含 Python.h 头文件的目录。

提示¶

在调用 find_package(PythonLibs) 之前，该模块接受以下变量：

Python_ADDITIONAL_VERSIONS

此变量可用于指定搜索 Python 时应考虑的版本号列表。

已弃用变量¶

以下变量提供用于向后兼容性

PYTHONLIBS_FOUND

3.12 版本弃用：请改用 PythonLibs_FOUND，其值相同。

布尔值，指示是否找到了（所请求版本的）Python 库。

PYTHON_DEBUG_LIBRARIES

2.8.8 版本弃用：请改用 PYTHON_LIBRARIES。

保存调试库路径的结果变量。

PYTHON_INCLUDE_PATH

2.8.0 版本弃用：请改用 PYTHON_INCLUDE_DIR 或 PYTHON_INCLUDE_DIRS。

保存包含 Python.h 头文件的目录路径的结果变量。

示例¶

在早期版本的 CMake 中，项目中查找和使用 Python 库的方式如下：

find_package(PythonLibs)
target_link_libraries(app PRIVATE ${PYTHON_LIBRARIES})
target_include_directories(app PRIVATE ${PYTHON_INCLUDE_DIRS})

从 CMake 3.12 开始，可以使用 FindPython 模块来查找 Python 库。使用现代方法及导入目标（imported target）的等效示例如下：

find_package(Python COMPONENTS Development)
target_link_libraries(app PRIVATE Python::Python)
