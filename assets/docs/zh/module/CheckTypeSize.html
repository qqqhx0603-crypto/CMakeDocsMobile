CheckTypeSize¶

此模块提供了一个用于检查 C/C++ 类型或表达式大小的命令。

在 CMake 项目中加载此模块，使用

include(CheckTypeSize)

代码说明

  • include(CheckTypeSize)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_type_size¶

检查 C/C++ 类型或表达式是否已存在，并确定其大小。

check_type_size(
  <type>
  <size-var>
  [RESULT_VARIABLE <result-var>]
  [BUILTIN_TYPES_ONLY]
  [LANGUAGE <language>]
)

代码说明

  • check_type_size( <type> <size-var> [RESULT_VARIABLE <result-var>] [BUILTIN_TYPES_ONLY] [LANGUAGE <language>] )：调用 `check_type_size()` 命令，具体行为由这一行传入的参数决定。

参数为

<type>

要检查的类型或表达式。

<size-var>

用于存储类型或表达式 <type> 大小的内部缓存变量名。该名称也用作下述变量的前缀。

RESULT_VARIABLE <result-var>

版本 4.2 中添加。

用于存储布尔值的内部缓存变量名，该布尔值指示类型或表达式 <type> 是否存在。如果未给出此参数，则命令默认定义一个名为 HAVE_<size-var> 的内部缓存变量。

BUILTIN_TYPES_ONLY

如果给出此项，则检查中仅支持编译器内置类型。如果未给出，命令会检查常见的头文件 <sys/types.h>、<stdint.h> 和 <stddef.h>，并将结果保存在内部缓存变量 HAVE_SYS_TYPES_H、HAVE_STDINT_H 和 HAVE_STDDEF_H 中。对于 C++ 的 std:: 类型，也会分别检查 <cstdint> 和 <cstddef>，并定义 HAVE_CSTDINT 和 HAVE_CSTDDEF。该命令会自动将可用的头文件包含在类型大小检查中，从而支持对头文件中定义的类型进行检查。

LANGUAGE <language>

使用 <language> 编译器执行检查。可选值为 C 和 CXX。如果未指定，默认为 C。

结果变量

结果在以下变量中报告

<size-var>

内部缓存变量，包含以下值之一

<size>

如果类型或表达式 <type> 存在，它将具有一个非零的字节大小 <size>。

0

当类型的大小取决于架构时（例如当 CMAKE_OSX_ARCHITECTURES 包含多个架构时），该值为 0。在这种情况下，还会定义 <size-var>_KEYS 变量，且 <size-var>_CODE 变量包含如下所述的预处理器测试映射。

"" (空字符串)

当类型或表达式 <type> 不存在时。

HAVE_<size-var>

内部缓存变量，保存一个布尔值，指示类型或表达式 <type> 是否存在。当未使用 RESULT_VARIABLE 参数时，定义此变量。

<result-var>

版本 4.2 中添加。

当使用 RESULT_VARIABLE 参数时定义的内部缓存变量。它保存一个布尔值，指示类型或表达式 <type> 是否存在（与 HAVE_<size-var> 的值相同）。在这种情况下，不会定义 HAVE_<size-var> 变量。

<size-var>_CODE

CMake 变量，保存用于定义宏 <size-var> 为类型大小的预处理器代码；如果类型不存在，则宏保持未定义状态。

当类型具有架构相关大小时（<size-var> 值为 0），此变量包含从每个架构宏到相应类型大小的预处理器测试映射。

<size-var>_KEYS

仅当类型具有架构相关大小（<size-var> 值为 0）时定义的 CMake 变量，包含架构宏的列表。每个键的值存储在 <size-var>-<key> 变量中。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

以空格分隔的字符串，用于传递给编译器的额外标志。分号分隔的列表在此处无效。CMAKE_<LANG>_FLAGS 及其关联的配置特定变量 CMAKE_<LANG>_FLAGS_<CONFIG> 的内容会自动在这些变量之前添加到编译器命令中。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

CMAKE_EXTRA_INCLUDE_FILES

执行检查时要包含的额外头文件的分号分隔列表。

示例¶

考虑以下代码

include(CheckTypeSize)

# Check for size of long.
check_type_size(long SIZEOF_LONG)

message("HAVE_SIZEOF_LONG: ${HAVE_SIZEOF_LONG}")
message("SIZEOF_LONG: ${SIZEOF_LONG}")
message("SIZEOF_LONG_CODE: ${SIZEOF_LONG_CODE}")

代码说明

  • include(CheckTypeSize) # Check for size of long. check_type_size(long SIZEOF_LONG) message("HAVE_SIZEOF_LONG: ${HAVE_SIZEOF_LONG}") message("SIZEOF_LONG: ${SIZEOF_LONG}") message("SIZEOF_LONG_CODE: ${SIZEOF_LONG_CODE}")：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

在 64 位架构上，输出可能如下所示

HAVE_SIZEOF_LONG: TRUE
SIZEOF_LONG: 8
SIZEOF_LONG_CODE: #define SIZEOF_LONG 8

代码说明

  • HAVE_SIZEOF_LONG: TRUE SIZEOF_LONG: 8 SIZEOF_LONG_CODE: #define SIZEOF_LONG 8：这一行给出示例文本或输出，用来说明：HAVE_SIZEOF_LONG: TRUE SIZEOF_LONG: 8 SIZEOF_LONG_CODE: #define SIZEOF_LONG 8

在 Apple 平台上，当 CMAKE_OSX_ARCHITECTURES 包含多个架构时，类型可能具有架构相关的大小。例如，使用以下代码

include(CheckTypeSize)

check_type_size(long SIZEOF_LONG)

message("HAVE_SIZEOF_LONG: ${HAVE_SIZEOF_LONG}")
message("SIZEOF_LONG: ${SIZEOF_LONG}")
foreach(key IN LISTS SIZEOF_LONG_KEYS)
  message("key: ${key}")
  message("value: ${SIZEOF_LONG-${key}}")
endforeach()
message("SIZEOF_LONG_CODE:\n${SIZEOF_LONG_CODE}")

代码说明

  • include(CheckTypeSize) check_type_size(long SIZEOF_LONG) message("HAVE_SIZEOF_LONG: ${HAVE_SIZEOF_LONG}") message("SIZEOF_LONG: ${SIZEOF_LONG}") foreach(key IN LISTS SIZEOF_LONG_KEYS) message("key: ${key}") message("value: ${SIZEOF_LONG-${key}}") endforeach() message("SIZEOF_LONG_CODE:\n${SIZEOF_LONG_CODE}")：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

结果可能是

HAVE_SIZEOF_LONG: TRUE
SIZEOF_LONG: 0
key: __i386
value: 4
key: __x86_64
value: 8
SIZEOF_LONG_CODE:
#if defined(__i386)
# define SIZEOF_LONG 4
#elif defined(__x86_64)
# define SIZEOF_LONG 8
#else
# error SIZEOF_LONG unknown
#endif

代码说明

  • HAVE_SIZEOF_LONG: TRUE SIZEOF_LONG: 0 key: __i386 value: 4 key: __x86_64 value: 8 SIZEOF_LONG_CODE: #if defined(__i386) # define SIZEOF_LONG 4 #elif defined(__x86_64) # define SIZEOF_LONG 8 #else # error SIZEOF_LONG unknown #endif：这一行给出示例文本或输出，用来说明：HAVE_SIZEOF_LONG: TRUE SIZEOF_LONG: 0 key: __i386 value: 4 key: __x86_64 value: 8 SIZEOF_LONG_CODE: #if defined(__i386) # define SIZEOF_LONG 4 #elif defined(__x86_64) # define SIZEOF_LONG 8 #else # error SIZEOF_LONG unknown #endif

示例：配置头文件¶

下一个示例演示了如何在配置头文件中使用结果变量

include(CheckTypeSize)
check_type_size(long SIZEOF_LONG)

configure_file(config.h.in config.h @ONLY)

代码说明

  • include(CheckTypeSize) check_type_size(long SIZEOF_LONG) configure_file(config.h.in config.h @ONLY)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

config.h.in¶
/* Define whether the type 'long' exists. */
#cmakedefine HAVE_SIZEOF_LONG

/* The size of 'long', as computed by sizeof. */
@SIZEOF_LONG_CODE@

代码说明

  • /* Define whether the type 'long' exists. */ #cmakedefine HAVE_SIZEOF_LONG /* The size of 'long', as computed by sizeof. */ @SIZEOF_LONG_CODE@：这是注释，用来说明：Define whether the type 'long' exists. */ #cmakedefine HAVE_SIZEOF_LONG /* The size of 'long', as computed by sizeof. */ @SIZEOF_LONG_CODE@。

示例：检查复杂表达式¶

尽管该模块名称如此，但它也可以用于确定更复杂表达式的大小。例如，要检查结构体成员的大小

include(CheckTypeSize)
check_type_size("((struct something*)0)->member" SIZEOF_MEMBER)

代码说明

  • include(CheckTypeSize) check_type_size("((struct something*)0)->member" SIZEOF_MEMBER)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

示例：独立检查¶

在以下示例中，检查是使用 CMAKE_EXTRA_INCLUDE_FILES 变量和 CMakePushCheckState 模块临时修改额外头文件后执行的。检查结果存储在 HAVE_SIZEOF_UNION_SEMUN 中，大小存储在 SIZEOF_UNION_SEMUN 内部缓存变量中。

include(CheckTypeSize)
include(CMakePushCheckState)

cmake_push_check_state(RESET)
  set(CMAKE_EXTRA_INCLUDE_FILES sys/types.h sys/ipc.h sys/sem.h)
  check_type_size("union semun" SIZEOF_UNION_SEMUN)
cmake_pop_check_state()

代码说明

  • include(CheckTypeSize) include(CMakePushCheckState) cmake_push_check_state(RESET) set(CMAKE_EXTRA_INCLUDE_FILES sys/types.h sys/ipc.h sys/sem.h) check_type_size("union semun" SIZEOF_UNION_SEMUN) cmake_pop_check_state()：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

示例：自定义结果变量¶

自 CMake 4.2 起，可以使用 RESULT_VARIABLE 参数自定义 HAVE_<size-var> 变量名称。在以下示例中，此模块用于检查 struct flock 是否存在，并将结果存储在 MyProj_HAVE_STRUCT_FLOCK 内部缓存变量中

cmake_minimum_required(VERSION 4.2)

# ...

include(CheckTypeSize)
include(CMakePushCheckState)

cmake_push_check_state(RESET)
  set(CMAKE_EXTRA_INCLUDE_FILES "fcntl.h")

  check_type_size(
    "struct flock"
    MyProj_SIZEOF_STRUCT_FLOCK
    RESULT_VARIABLE MyProj_HAVE_STRUCT_FLOCK
  )
cmake_pop_check_state()

代码说明

  • cmake_minimum_required(VERSION 4.2) # ... include(CheckTypeSize) include(CMakePushCheckState) cmake_push_check_state(RESET) set(CMAKE_EXTRA_INCLUDE_FILES "fcntl.h") check_type_size( "struct flock" MyProj_SIZEOF_STRUCT_FLOCK RESULT_VARIABLE MyProj_HAVE_STRUCT_FLOCK ) cmake_pop_check_state()：调用 `cmake_minimum_required()` 命令，声明工程所需的最低 CMake 版本。
