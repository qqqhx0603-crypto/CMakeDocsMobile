UseSWIG¶

此模块提供对 SWIG 的支持。假设 FindSWIG 模块已经被加载。

在 CMake 项目中加载此模块，使用

include(UseSWIG)

CMake 命令¶

定义了以下命令以配合 SWIG 使用

swig_add_library¶

版本 3.8 新增。

使用给定名称和指定语言定义 swig 模块

swig_add_library(<name>
                 [TYPE <SHARED|MODULE|STATIC|USE_BUILD_SHARED_LIBS>]
                 LANGUAGE <language>
                 [NO_PROXY]
                 [DEBUG_POSTFIX <postfix>]
                 [OUTPUT_DIR <directory>]
                 [OUTFILE_DIR <directory>]
                 SOURCES <file>...
                )

使用 swig_add_library 命令创建的目标具有与使用 add_library() 命令创建的目标相同的功能，因此这些目标可以与任何期望接收目标的命令（例如 target_link_libraries()）一起使用。

3.13 版本变更: 当策略 CMP0078 设置为 NEW 时，此命令会创建一个具有指定 <name> 的目标。否则，旧版行为将选择不同的目标名称，并将其存储在 SWIG_MODULE_<name>_REAL_NAME 变量中。

3.15 版本变更: 替代库名称（例如通过 OUTPUT_NAME 属性设置的名称）将传递给 Python 和 CSharp 包装库。

3.21 版本变更: 当策略 CMP0122 设置为 NEW 时，生成的库将为 CSharp 语言使用标准命名约定。否则，将应用旧版行为。

注意

对于多配置生成器，此模块不支持由 SWIG 生成的特定配置的文件。所有构建配置必须产生相同的生成源文件。

注意

对于 Makefile 生成器，如果某些源文件的 USE_SWIG_DEPENDENCIES 属性为 FALSE，则 swig_add_library 不会跟踪文件依赖关系，因此对于需要 swig 生成的文件存在的任何目标，必须依赖于 <name>_swig_compilation 自定义目标。其他生成器可能会直接依赖于 SWIG 将生成的源文件。

类型

SHARED、MODULE 和 STATIC 的语义与 add_library() 命令相同。如果指定了 USE_BUILD_SHARED_LIBS，则库类型将根据当前 BUILD_SHARED_LIBS 变量的值是 ON 还是 OFF，而设为 STATIC 或 SHARED。如果没有指定类型，则将使用 MODULE。

语言

指定目标语言。

3.1 版本新增: Go 和 Lua 语言支持。

3.2 版本新增: R 语言支持。

3.18 版本新增: Fortran 语言支持。

NO_PROXY

3.12 版本新增。

禁止生成包装层（swig -noproxy 选项）。

DEBUG_POSTFIX

版本 4.2 中添加。

管理 DEBUG_POSTFIX 目标属性。目前仅对 python 语言有意义。

如果定义了 DEBUG_CONFIGURATIONS 全局属性，则为每个调试配置定义 <CONFIG>_POSTFIX 目标属性。

OUTPUT_DIR

3.12 版本新增。

指定写入特定语言文件的位置（swig -outdir 选项）。如果未提供，将使用 CMAKE_SWIG_OUTDIR 变量。如果两者均未指定，则默认值取决于 UseSWIG_MODULE_VERSION 变量的值，如下所示：

  • 如果 UseSWIG_MODULE_VERSION 为 1 或未定义，输出将写入 CMAKE_CURRENT_BINARY_DIR 目录。

  • 如果 UseSWIG_MODULE_VERSION 为 2，将使用专用目录。此目录的路径可以从 SWIG_SUPPORT_FILES_DIRECTORY 目标属性中检索。

OUTFILE_DIR

3.12 版本新增。

指定生成源文件放置的输出目录名称（swig -o 选项）。如果未指定，将使用 SWIG_OUTFILE_DIR 变量。如果两者均未指定，则改为使用 OUTPUT_DIR 或 CMAKE_SWIG_OUTDIR。

SOURCES

库的源文件列表。扩展名为 .i 的文件将被识别为 SWIG 工具的源文件。其他文件将按标准方式处理。

3.14 版本新增: 可以通过指定变量 SWIG_SOURCE_FILE_EXTENSIONS 来覆盖此行为。

注意

如果 UseSWIG_MODULE_VERSION 设置为 2，强烈建议在指定 OUTPUT_DIR 选项或 CMAKE_SWIG_OUTDIR 变量时，使用对目标唯一的专用目录。输出目录内容会作为目标构建过程的一部分被擦除，因此为了防止目标之间的干扰或丢失其他重要文件，每个目标都应有自己的专用输出目录。

源文件属性¶

模块文件的源文件属性必须在调用 swig_add_library 命令之前设置，以指定 SWIG 的特殊行为并确保生成的文件获得所需的设置。

CPLUSPLUS

以 c++ 模式调用 SWIG。例如：

set_property(SOURCE mymod.i PROPERTY CPLUSPLUS ON)
swig_add_library(mymod LANGUAGE python SOURCES mymod.i)
SWIG_FLAGS

3.12 版本弃用: 被后续的细粒度属性所取代。

将自定义标志传递给 SWIG 可执行文件。

INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_OPTIONS

3.12 版本新增。

向 SWIG 编译器添加自定义标志，其语义与 INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_OPTIONS 属性相同。

USE_TARGET_INCLUDE_DIRECTORIES

3.13 版本新增。

如果设置为 TRUE，则目标属性 INCLUDE_DIRECTORIES 的内容将被转发给 SWIG 编译器。如果设置为 FALSE，则目标属性 INCLUDE_DIRECTORIES 将被忽略。如果未设置，则会考虑目标属性 SWIG_USE_TARGET_INCLUDE_DIRECTORIES。

GENERATED_INCLUDE_DIRECTORIES、GENERATED_COMPILE_DEFINITIONS 和 GENERATED_COMPILE_OPTIONS

3.12 版本新增。

向生成的 C/C++ 源文件添加自定义标志。它们将分别填充生成的 C/C++ 文件的 INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_OPTIONS 属性。

DEPENDS

3.12 版本新增。

为源文件指定额外的依赖项。

USE_SWIG_DEPENDENCIES

在 3.20 版本中添加。

如果设置为 TRUE，则隐式依赖项将由 swig 工具本身生成。此属性仅对 Makefile、Ninja、Xcode 和 Visual Studio 生成器有意义。默认值为 FALSE。

3.21 版本新增: 增加了对 Xcode 生成器的支持。

3.22 版本新增: 增加了对 Visual Studio 生成器 的支持。

SWIG_MODULE_NAME

指定模块在目标语言中的实际导入名称。如果无法自动从源文件扫描，或者名称与模块文件的基本名称不同，则需要此项。例如：

set_property(SOURCE mymod.i PROPERTY SWIG_MODULE_NAME mymod_realname)

3.14 版本变更: 如果策略 CMP0086 设置为 NEW，则 -module <module_name> 将被传递给 SWIG 编译器。

OUTPUT_DIR

3.19 版本新增。

指定为考虑中的源文件写入特定语言文件的位置（swig -outdir 选项）。如果未指定，则应用定义输出目录的其他方式（参见 swig_add_library() 命令的 OUTPUT_DIR 选项）。

OUTFILE_DIR

3.19 版本新增。

指定生成源文件为考虑中的源文件放置的输出目录（swig -o 选项）。如果未指定，将使用 OUTPUT_DIR 源属性。如果两者均未指定，则应用定义输出文件目录的其他方式（参见 swig_add_library() 命令的 OUTFILE_DIR 选项）。

目标属性¶

可以设置目标库属性，将相同的配置应用于所有 SWIG 输入文件。

SWIG_INCLUDE_DIRECTORIES、SWIG_COMPILE_DEFINITIONS 和 SWIG_COMPILE_OPTIONS

3.12 版本新增。

这些属性将应用于所有 SWIG 输入文件，其语义与目标属性 INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_OPTIONS 相同。

set (UseSWIG_TARGET_NAME_PREFERENCE STANDARD)
swig_add_library(mymod LANGUAGE python SOURCES mymod.i)
set_property(TARGET mymod PROPERTY SWIG_COMPILE_DEFINITIONS MY_DEF1 MY_DEF2)
set_property(TARGET mymod PROPERTY SWIG_COMPILE_OPTIONS -bla -blb)
SWIG_USE_TARGET_INCLUDE_DIRECTORIES

3.13 版本新增。

如果设置为 TRUE，则目标属性 INCLUDE_DIRECTORIES 的内容将被转发给 SWIG 编译器。如果设置为 FALSE 或未定义，则目标属性 INCLUDE_DIRECTORIES 将被忽略。此行为可以通过指定源属性 USE_TARGET_INCLUDE_DIRECTORIES 来覆盖。

SWIG_GENERATED_INCLUDE_DIRECTORIES、SWIG_GENERATED_COMPILE_DEFINITIONS 和 SWIG_GENERATED_COMPILE_OPTIONS

3.12 版本新增。

这些属性将分别填充所有生成的 C/C++ 文件的 INCLUDE_DIRECTORIES、COMPILE_DEFINITIONS 和 COMPILE_FLAGS 属性。

SWIG_DEPENDS

3.12 版本新增。

向所有 SWIG 输入文件添加依赖项。

只读目标属性¶

以下目标属性是输出属性，可用于获取有关由 SWIG 接口编译生成的支持文件的信息。

SWIG_SUPPORT_FILES

3.12 版本新增。

此输出属性是 SWIG 编译过程中生成的包装文件列表。

set (UseSWIG_TARGET_NAME_PREFERENCE STANDARD)
swig_add_library(mymod LANGUAGE python SOURCES mymod.i)
get_property(support_files TARGET mymod PROPERTY SWIG_SUPPORT_FILES)

注意

仅列出最主要的支持文件。如果使用了 SWIG 的某些高级功能（例如 %template），则相关支持文件可能不会被列出。建议优先使用 SWIG_SUPPORT_FILES_DIRECTORY 属性来处理支持文件。

SWIG_SUPPORT_FILES_DIRECTORY

3.12 版本新增。

此输出属性指定生成支持文件的目录。

注意

当定义了源属性 OUTPUT_DIR 时，可以指定多个目录作为 SWIG_SUPPORT_FILES_DIRECTORY 的一部分。

CMake 变量¶

可以设置一些变量来自定义 swig_add_library 以及 SWIG 的行为。

UseSWIG_MODULE_VERSION

3.12 版本新增。

为 UseSWIG 模块指定不同的行为。

  • 设置为 1 或未定义：应用旧版行为。

  • 设置为 2：针对支持文件应用一种新的策略：支持文件的输出目录在 SWIG 接口编译之前会被擦除。

CMAKE_SWIG_FLAGS

向所有 swig 调用添加标志。

CMAKE_SWIG_OUTDIR

指定写入特定语言文件的位置（swig -outdir 选项）。

SWIG_OUTFILE_DIR

版本 3.8 新增。

指定生成源文件放置的输出目录名称。如果未指定，则使用 CMAKE_SWIG_OUTDIR。

SWIG_MODULE_<name>_EXTRA_DEPS

为 <name> 生成的模块指定额外的依赖项。

SWIG_SOURCE_FILE_EXTENSIONS

3.14 版新增。

指定源文件扩展名列表，以覆盖仅将 .i 文件视为 SWIG 工具源文件的默认行为。例如：

set(SWIG_SOURCE_FILE_EXTENSIONS ".i" ".swg")
SWIG_USE_SWIG_DEPENDENCIES

在 3.20 版本中添加。

如果设置为 TRUE，则隐式依赖项将由 swig 工具本身生成。此变量仅对 Makefile、Ninja、Xcode 和 Visual Studio 生成器有意义。默认值为 FALSE。

如果未定义，源文件属性 USE_SWIG_DEPENDENCIES 将使用此变量的值进行初始化。

3.21 版本新增: 增加了对 Xcode 生成器的支持。

3.22 版本新增: 增加了对 Visual Studio 生成器 的支持。

弃用的命令¶

swig_link_libraries¶

3.13 版本弃用: 使用带有标准目标名称的 target_link_libraries()，或者对于旧版目标命名，使用 ${SWIG_MODULE_<name>_REAL_NAME}。

将库链接到 swig 模块

swig_link_libraries(<name> <item>...)

此命令具有与 target_link_libraries() 命令相同的功能。

注意

当策略 CMP0078 设置为 NEW 时，swig_add_library() 会创建一个具有指定 <name> 的标准目标，并且必须使用 target_link_libraries() 而不是此命令。

在使用旧版行为（当 CMP0078 设置为 OLD 且 UseSWIG_TARGET_NAME_PREFERENCE 变量设置为 "LEGACY" 时，或在 3.12 之前的 CMake 版本中），更倾向于使用 target_link_libraries(${SWIG_MODULE_<name>_REAL_NAME} ...) 而不是此命令。
