CMakeAddFortranSubdirectory¶

该模块提供了一个用于添加位于子目录中的 Fortran 项目的命令。

在 CMake 项目中加载此模块，使用

include(CMakeAddFortranSubdirectory)

命令¶

此模块提供以下命令

cmake_add_fortran_subdirectory¶

将一个仅包含 Fortran 的子项目从子目录添加到当前项目中

cmake_add_fortran_subdirectory(
  <subdir>
  PROJECT <project-name>
  ARCHIVE_DIR <dir>
  RUNTIME_DIR <dir>
  LIBRARIES <libs>...
  LINK_LIBRARIES
    [LINK_LIBS <lib> <deps>...]...
  [CMAKE_COMMAND_LINE <flags>...]
  NO_EXTERNAL_INSTALL
)

该命令会检查当前编译器是否支持 Fortran，或者尝试定位 Fortran 编译器。如果找到了兼容的 Fortran 编译器，则位于 <subdir> 中的 Fortran 项目将作为子目录添加到当前项目中。

如果没有找到 Fortran 编译器且当前编译器是 MSVC，它会搜索 MinGW 的 gfortran 编译器。在这种情况下，Fortran 项目将作为外部项目使用 MinGW 工具进行构建，并创建与 Fortran 相关的导入目标（imported targets）。此设置仅在 Fortran 代码被构建为共享 DLL 库时有效，因此外部项目中的 BUILD_SHARED_LIBS 变量必须处于启用状态。此外，CMAKE_GNUtoMS 变量会被设置为 ON，以确保创建与 Microsoft 兼容的 .lib 文件。

选项包括

PROJECT

Fortran 项目的名称，定义在位于 <subdir> 中的顶层 CMakeLists.txt 中。

ARCHIVE_DIR

项目放置 .lib 归档文件的目录。相对路径将被解释为相对于 CMAKE_CURRENT_BINARY_DIR。

RUNTIME_DIR

项目放置 .dll 运行时文件的目录。相对路径将被解释为相对于 CMAKE_CURRENT_BINARY_DIR。

LIBRARIES

要在当前项目中创建或导入的库目标名称。

LINK_LIBRARIES

为 LIBRARIES 指定链接接口库。此选项需要一个 LINK_LIBS <lib> <deps>... 项目列表，其中

  • LINK_LIBS 标记新配对的开始

  • <lib> 是一个库目标。

  • <deps>... 表示 <lib> 所需的一个或多个依赖项。

CMAKE_COMMAND_LINE

在配置 Fortran 子项目时，传递给 cmake(1) 命令的附加命令行标志。

NO_EXTERNAL_INSTALL

防止安装外部项目。

注意

NO_EXTERNAL_INSTALL 选项是向后兼容所必需的，以便未来版本支持在执行 make install 时安装外部项目的二进制文件。

示例¶

将 Fortran 子目录添加到项目中可以通过包含此模块并调用 cmake_add_fortran_subdirectory() 命令来实现。在以下示例中，一个 Fortran 项目提供了 hello 库及其依赖的 world 库

include(CMakeAddFortranSubdirectory)

cmake_add_fortran_subdirectory(
  fortran-subdir
  PROJECT FortranHelloWorld
  ARCHIVE_DIR lib
  RUNTIME_DIR bin
  LIBRARIES hello world
  LINK_LIBRARIES
    LINK_LIBS hello world # hello library depends on the world library
  NO_EXTERNAL_INSTALL
)

# The Fortran target can be then linked to the main project target.
add_executable(main main.c)
target_link_libraries(main PRIVATE hello)

另请参阅¶

集成 Fortran 库有多种方式。其他可选方法包括：

  • add_subdirectory() 命令，直接将子目录添加到构建中。

  • 在子项目中使用 export() 命令，以提供 导入目标 (Imported Targets) 或类似功能，以便与其他项目集成。

  • 当处理外部依赖时，使用 FetchContent 或 ExternalProject 模块。
