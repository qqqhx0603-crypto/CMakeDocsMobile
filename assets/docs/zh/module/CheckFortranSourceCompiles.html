CheckFortranSourceCompiles¶

版本 3.1 中新增。

该模块提供了一个用于检查 Fortran 源代码是否可以构建的命令。

在 CMake 项目中加载此模块，使用

include(CheckFortranSourceCompiles)

代码说明

  • include(CheckFortranSourceCompiles)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_fortran_source_compiles¶

检查给定的 Fortran 源代码是否可以构建

check_fortran_source_compiles(
  <code>
  <variable>
  [FAIL_REGEX <regexes>...]
  [SRC_EXT <extension>]
)

代码说明

  • check_fortran_source_compiles(：这一行继续补充当前命令或表达式的内容。

  • <code>：这一行继续补充当前命令或表达式的内容。

  • <variable>：这一行继续补充当前命令或表达式的内容。

  • [FAIL_REGEX <regexes>...]：这一行继续补充当前命令或表达式的内容。

  • [SRC_EXT <extension>]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

此命令会检查一次 <code> 中提供的源代码是否可以编译（并链接成可执行文件）。检查结果将存储在由 <variable> 指定的内部缓存变量中。

参数为

<code>

要检查的 Fortran 源代码。这必须是一个完整的程序，就像写在包含主体块的文件中一样。源代码中使用的所有符号都应按通常方式在其对应的头文件中声明。

<variable>

内部缓存变量的变量名，用于存储检查结果，成功为布尔真，失败为布尔假。

FAIL_REGEX <regexes>...

如果此选项提供一个或多个正则表达式，则通过检查编译器输出中是否有任何内容与指定的任何正则表达式匹配来确定失败。

SRC_EXT <extension>

3.7 版本中新增。

默认情况下，用于检查的测试源文件将被赋予 .F 文件扩展名。此选项可用于将其覆盖为 .<extension> —— .F90 是一个典型的选择。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

一个以空格分隔的字符串，包含要传递给编译器的附加标志。使用 分号分隔的列表 将无法工作。CMAKE_<LANG>_FLAGS 的内容及其相关的特定配置变量 CMAKE_<LANG>_FLAGS_<CONFIG> 会在编译器命令中自动添加在该变量内容之前。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

CMAKE_TRY_COMPILE_TARGET_TYPE

内部使用 `try_compile() 命令来执行检查，此变量控制它创建的目标类型。如果此变量设置为 `EXECUTABLE`（默认值），则检查会将测试源代码编译并链接成可执行程序。如果设置为 `STATIC_LIBRARY`，则测试源代码将被编译但不会被链接。

示例¶

检查 Fortran 编译器是否支持 pure 过程属性

include(CheckFortranSourceCompiles)

check_fortran_source_compiles("
  pure subroutine foo()
  end subroutine
  program test
    call foo()
  end
" HAVE_PURE SRC_EXT "F90")

代码说明

  • include(CheckFortranSourceCompiles)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_fortran_source_compiles("：这一行继续补充当前命令或表达式的内容。

  • pure subroutine foo()：这一行继续补充当前命令或表达式的内容。

  • end subroutine：这一行继续补充当前命令或表达式的内容。

  • program test：这一行继续补充当前命令或表达式的内容。

  • call foo()：这一行继续补充当前命令或表达式的内容。

  • end：这一行继续补充当前命令或表达式的内容。

  • " HAVE_PURE SRC_EXT "F90")：这一行继续补充当前命令或表达式的内容。
