FindPythonInterp¶

3.27 版本变更: 仅当策略 CMP0148 未设置为 NEW 时，此模块才可用。

3.12 版本弃用: 请改用 FindPython3、FindPython2 或 FindPython。

查找 Python 解释器并确定其可执行文件的位置

find_package(PythonInterp [<version>] [...])

代码说明

  • find_package(PythonInterp [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

注意

当同时使用本模块和 FindPythonLibs 模块时，请在 find_package(PythonLibs) 之前调用 find_package(PythonInterp)。这可以确保检测到的解释器版本被用于指导兼容库的选择，从而得到一致的 PYTHON_LIBRARIES 值。

注意

对 Python 版本 V 调用 find_package(PythonInterp ${V}) 时，可能会找到没有版本后缀的 python 可执行文件。在这种情况下，不会尝试排除其他版本的 Python 可执行文件。请改用 FindPython3、FindPython2 或 FindPython。

结果变量¶

此模块定义了以下变量

PythonInterp_FOUND

3.3 版本中新增。

布尔值，表示是否找到了（所请求版本的）Python 可执行文件。

PYTHON_VERSION_STRING

找到的 Python 版本（例如 2.5.2）。

PYTHON_VERSION_MAJOR

找到的 Python 主版本号（例如 2）。

PYTHON_VERSION_MINOR

找到的 Python 次版本号（例如 5）。

PYTHON_VERSION_PATCH

找到的 Python 修订版本号（例如 2）。

缓存变量¶

以下缓存变量也可以设置

PYTHON_EXECUTABLE

Python 解释器的路径。

提示¶

该模块在调用 find_package(PythonInterp) 之前接受以下变量：

Python_ADDITIONAL_VERSIONS

此变量可用于指定在搜索 Python 时应考虑的版本号列表。

已弃用变量¶

以下变量提供用于向后兼容性

PYTHONINTERP_FOUND

3.12 版本弃用: 使用 PythonInterp_FOUND，其值相同。

布尔值，表示是否找到了（所请求版本的）Python 可执行文件。

示例¶

在较早版本的 CMake 中查找 Python 解释器

find_package(PythonInterp)
execute_process(COMMAND ${PYTHON_EXECUTABLE} --help)

代码说明

  • find_package(PythonInterp) execute_process(COMMAND ${PYTHON_EXECUTABLE} --help)：调用 `find_package()` 命令，查找并加载外部依赖包。

从 CMake 3.12 开始，可以使用 FindPython 模块来查找 Python 解释器。使用现代方法的等效示例如下：

find_package(Python)
execute_process(COMMAND ${Python_EXECUTABLE} --help)

代码说明

  • find_package(Python) execute_process(COMMAND ${Python_EXECUTABLE} --help)：调用 `find_package()` 命令，查找并加载外部依赖包。
