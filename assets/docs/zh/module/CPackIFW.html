CPackIFW¶

版本 3.1 中新增。

该模块用于查找 Qt Installer Framework（Qt 安装程序框架） (QtIFW) 所提供的命令行工具的位置。

在 CMake 项目中加载此模块，使用

include(CPackIFW)

代码说明

  • include(CPackIFW)：调用 include() 命令，加载并执行另一个 CMake 脚本。

该模块还定义了几个命令，用于控制 CPack IFW 生成器 的行为。

命令¶

该模块定义了以下命令

cpack_ifw_configure_component¶

设置 CPack IFW 生成器特有的参数。

cpack_ifw_configure_component(<compname> [COMMON] [ESSENTIAL] [VIRTUAL]
                    [FORCED_INSTALLATION] [REQUIRES_ADMIN_RIGHTS]
                    [NAME <name>]
                    [DISPLAY_NAME <display_name>] # Note: Internationalization supported
                    [DESCRIPTION <description>] # Note: Internationalization supported
                    [UPDATE_TEXT <update_text>]
                    [VERSION <version>]
                    [RELEASE_DATE <release_date>]
                    [SCRIPT <script>]
                    [PRIORITY|SORTING_PRIORITY <sorting_priority>] # Note: PRIORITY is deprecated
                    [DEPENDS|DEPENDENCIES <com_id> ...]
                    [AUTO_DEPEND_ON <comp_id> ...]
                    [LICENSES <display_name> <file_path> ...]
                    [DEFAULT <value>]
                    [USER_INTERFACES <file_path> <file_path> ...]
                    [TRANSLATIONS <file_path> <file_path> ...]
                    [REPLACES <comp_id> ...]
                    [CHECKABLE <value>])

代码说明

  • cpack_ifw_configure_component(<compname> [COMMON] [ESSENTIAL] [VIRTUAL]：这一行继续补充当前命令或表达式的内容。

  • [FORCED_INSTALLATION] [REQUIRES_ADMIN_RIGHTS]：这一行继续补充当前命令或表达式的内容。

  • [NAME <name>]：这一行继续补充当前命令或表达式的内容。

  • [DISPLAY_NAME <display_name>] # Note: Internationalization supported：这一行继续补充当前命令或表达式的内容。

  • [DESCRIPTION <description>] # Note: Internationalization supported：这一行继续补充当前命令或表达式的内容。

  • [UPDATE_TEXT <update_text>]：这一行继续补充当前命令或表达式的内容。

  • [VERSION <version>]：这一行继续补充当前命令或表达式的内容。

  • [RELEASE_DATE <release_date>]：这一行继续补充当前命令或表达式的内容。

  • [SCRIPT <script>]：这一行继续补充当前命令或表达式的内容。

  • [PRIORITY|SORTING_PRIORITY <sorting_priority>] # Note: PRIORITY is deprecated：这一行继续补充当前命令或表达式的内容。

  • [DEPENDS|DEPENDENCIES <com_id> ...]：这一行继续补充当前命令或表达式的内容。

  • [AUTO_DEPEND_ON <comp_id> ...]：这一行继续补充当前命令或表达式的内容。

  • [LICENSES <display_name> <file_path> ...]：这一行继续补充当前命令或表达式的内容。

  • [DEFAULT <value>]：这一行继续补充当前命令或表达式的内容。

  • [USER_INTERFACES <file_path> <file_path> ...]：这一行继续补充当前命令或表达式的内容。

  • [TRANSLATIONS <file_path> <file_path> ...]：这一行继续补充当前命令或表达式的内容。

  • [REPLACES <comp_id> ...]：这一行继续补充当前命令或表达式的内容。

  • [CHECKABLE <value>])：这一行继续补充当前命令或表达式的内容。

该命令应在 cpack_add_component() 命令之后调用。

COMMON

如果设置，则该组件将作为其所属组的一部分进行打包和安装。

ESSENTIAL

3.6 版本新增。

如果设置，则在更新该组件之前，包管理器将保持禁用状态。

VIRTUAL

版本 3.8 新增。

如果设置，则该组件将在安装程序中隐藏。这等同于 cpack_add_component() 命令中的 HIDDEN 选项。

FORCED_INSTALLATION

版本 3.8 新增。

如果设置，则该组件必须始终安装。这等同于 cpack_add_component() 命令中的 REQUIRED 选项。

REQUIRES_ADMIN_RIGHTS

版本 3.8 新增。

如果组件需要以提升的权限安装，请设置此项。

名称

用于为此组件创建类似域的标识符。默认情况下使用原始组件名称。

DISPLAY_NAME

版本 3.8 新增。

设置以重写由 cpack_add_component() 命令配置的原始名称。

DESCRIPTION

版本 3.8 新增。

设置以重写由 cpack_add_component() 命令配置的原始描述。

UPDATE_TEXT

版本 3.8 新增。

如果这是对组件的更新，则将添加到组件描述中。

VERSION

是组件的版本。默认情况下使用 CPACK_PACKAGE_VERSION。

RELEASE_DATE

版本 3.8 新增。

保持为空以自动生成。

SCRIPT

是该组件操作脚本的相对或绝对路径。

SORTING_PRIORITY

版本 3.8 新增。

是组件在树中的优先级。

PRIORITY

自 3.8 版本弃用： SORTING_PRIORITY 的旧名称。

DEPENDS, DEPENDENCIES

版本 3.8 新增。

QtIFW 风格的依赖组件或组件组标识符列表。

3.21 版本新增。

列为依赖项的组件或组名称可能包含连字符。这需要 QtIFW 3.1 或更高版本。

AUTO_DEPEND_ON

版本 3.8 新增。

该组件自动依赖的 QtIFW 风格的组件或组件组标识符列表。

LICENSES

该组件的许可证文本的 <显示名称> 和 <文件路径> 对。您可以指定多个许可证。

DEFAULT

版本 3.8 新增。

可能的值为：TRUE、FALSE 和 SCRIPT。设置为 FALSE 可在安装程序中禁用该组件，或设置为 SCRIPT 以在运行时解析（别忘了添加脚本文件作为 SCRIPT 选项的值）。

USER_INTERFACES

3.7 版本中新增。

是要加载的页面列表（'.ui' 文件路径）。

TRANSLATIONS

版本 3.8 新增。

是要加载的翻译列表（'.qm' 文件路径）。

REPLACES

3.10 版本新增。

要替换的组件或组件组的标识符列表。

CHECKABLE

3.10 版本新增。

可能的值为：TRUE、FALSE。如果您想隐藏项目的复选框，请设置为 FALSE。这在只需要选择少数几个子组件而不是全部时非常有用。

cpack_ifw_add_repository¶

将 QtIFW 特有的远程仓库添加到二进制安装程序中。

cpack_ifw_add_repository(<reponame> [DISABLED]
                    URL <url>
                    [USERNAME <username>]
                    [PASSWORD <password>]
                    [DISPLAY_NAME <display_name>])

代码说明

  • cpack_ifw_add_repository(<reponame> [DISABLED]：这一行继续补充当前命令或表达式的内容。

  • URL <url>：这一行继续补充当前命令或表达式的内容。

  • [USERNAME <username>]：这一行继续补充当前命令或表达式的内容。

  • [PASSWORD <password>]：这一行继续补充当前命令或表达式的内容。

  • [DISPLAY_NAME <display_name>])：这一行继续补充当前命令或表达式的内容。

该命令还会将 <reponame> 仓库添加到变量 CPACK_IFW_REPOSITORIES_ALL 中。

DISABLED

如果设置，则该仓库默认将被禁用。

URL

指向可用组件列表的地址。

USERNAME

用作受保护仓库的用户名。

PASSWORD

受保护仓库使用的密码。

DISPLAY_NAME

是代替 URL 显示的字符串。

cpack_ifw_update_repository¶

3.6 版本新增。

从远程仓库更新 QtIFW 特有的仓库。

cpack_ifw_update_repository(<reponame>
                    [[ADD|REMOVE] URL <url>]|
                     [REPLACE OLD_URL <old_url> NEW_URL <new_url>]]
                    [USERNAME <username>]
                    [PASSWORD <password>]
                    [DISPLAY_NAME <display_name>])

代码说明

  • cpack_ifw_update_repository(<reponame>：这一行继续补充当前命令或表达式的内容。

  • [[ADD|REMOVE] URL <url>]|：这一行继续补充当前命令或表达式的内容。

  • [REPLACE OLD_URL <old_url> NEW_URL <new_url>]]：这一行继续补充当前命令或表达式的内容。

  • [USERNAME <username>]：这一行继续补充当前命令或表达式的内容。

  • [PASSWORD <password>]：这一行继续补充当前命令或表达式的内容。

  • [DISPLAY_NAME <display_name>])：这一行继续补充当前命令或表达式的内容。

该命令还会将 <reponame> 仓库添加到变量 CPACK_IFW_REPOSITORIES_ALL 中。

URL

指向可用组件列表的地址。

OLD_URL

指向将被替换的列表地址。

NEW_URL

指向将要替换成的列表地址。

USERNAME

用作受保护仓库的用户名。

PASSWORD

受保护仓库使用的密码。

DISPLAY_NAME

是代替 URL 显示的字符串。

cpack_ifw_add_package_resources¶

3.7 版本中新增。

在安装程序二进制文件中添加额外资源。

cpack_ifw_add_package_resources(<file_path> <file_path> ...)

代码说明

  • cpack_ifw_add_package_resources(<file_path> <file_path> ...)：调用 cpack_ifw_add_package_resources() 命令，具体行为由这一行传入的参数决定。

该命令还会将指定的文件添加到变量 CPACK_IFW_PACKAGE_RESOURCES 中。
