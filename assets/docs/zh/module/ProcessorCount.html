ProcessorCount¶

此模块提供了一个命令来确定处理器/核心的数量。

使用 CMake 加载此模块

include(ProcessorCount)

代码说明

  • include(ProcessorCount)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

ProcessorCount¶

确定机器上可用的逻辑 CPU 核心数量

ProcessorCount(<variable>)

代码说明

  • ProcessorCount(<variable>)：调用 ProcessorCount() 命令，具体行为由这一行传入的参数决定。

此命令将一个名为 <variable> 的变量设置为机器上可用的逻辑 CPU 核心数量，如果可以确定该信息的话。如果成功，变量保证设置为一个正整数（>=1）。如果无法确定处理器数量，则设置为 0。

目前，此功能已为 AIX、Cygwin、FreeBSD、Haiku、HPUX、Linux、macOS（苹果桌面系统）、QNX、Sun 和 Windows（微软桌面系统） 实现。

此命令提供当前机器上可用计算核心数量的近似值，使其对于并行构建和测试很有用。它的目的是帮助利用机器的合理容量，但用户在使用其全部容量进行并行任务之前，应考虑机器上运行的其他工作负载。

版本 3.15 中的更改：在 Linux 上，返回容器 CPU 数量而不是主机 CPU 数量。

注意

此模块依赖于系统相关的命令来确定处理器数量，这些命令在某些环境中可能无法始终提供准确的信息。也可以使用 cmake_host_system_information() 来获得更普遍准确的逻辑 CPU 数量。

cmake_host_system_information(RESULT n QUERY NUMBER_OF_LOGICAL_CORES)

代码说明

  • cmake_host_system_information(RESULT n QUERY NUMBER_OF_LOGICAL_CORES)：调用 cmake_host_system_information() 命令，具体行为由这一行传入的参数决定。

示例¶

在以下示例中，此模块在 ctest -S 仪表板脚本中使用，以确定用于并行 CTest 构建步骤 的核心数量

include(ProcessorCount)
ProcessorCount(n)
if(NOT n EQUAL 0)
  set(CTEST_BUILD_FLAGS -j${n})
  set(ctest_test_args ${ctest_test_args} PARALLEL_LEVEL ${n})
endif()

代码说明

  • include(ProcessorCount)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • ProcessorCount(n)：调用 ProcessorCount() 命令，具体行为由这一行传入的参数决定。

  • if(NOT n EQUAL 0)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；EQUAL 表示按数值比较两侧是否相等。

  • set(CTEST_BUILD_FLAGS -j${n})：调用 set() 命令，把变量 CTEST_BUILD_FLAGS 设为给定的值；变量说明：n：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；CTEST_BUILD_FLAGS：CTest 构建、选项相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • set(ctest_test_args ${ctest_test_args} PARALLEL_LEVEL ${n})：调用 set() 命令，把变量 ctest_test_args 设为给定的值；变量说明：ctest_test_args：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供；n：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • endif()：这一行结束或切换上一段控制结构的分支。
