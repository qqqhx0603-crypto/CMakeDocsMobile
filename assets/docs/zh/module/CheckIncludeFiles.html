CheckIncludeFiles¶

该模块提供了一个用于检查一个或多个 C/C++ 头文件的命令。

在 CMake 项目中加载此模块，使用

include(CheckIncludeFiles)

代码说明

  • include(CheckIncludeFiles)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_include_files¶

检查一个或多个头文件是否存在，以及它们是否可以在 C 或 C++ 代码中被同时包含。

check_include_files(<includes> <variable> [LANGUAGE <language>])

代码说明

  • check_include_files(<includes> <variable> [LANGUAGE <language>])：调用 check_include_files() 命令，具体行为由这一行传入的参数决定。

参数为

<includes>

一个由分号分隔的列表，包含需要检查的头文件。

<variable>

用于存储检查结果的变量名。此变量将被创建为内部缓存变量。

LANGUAGE <language>

3.11 版本新增。

如果设置，将使用指定的 <language> 编译器来执行检查。可接受的值为 C 和 CXX。如果未提供此选项，且已启用 C 编译器，则使用 C 编译器。如果未启用 C 编译器，且已启用 C++ 编译器，则使用 C++ 编译器。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

一个由空格分隔的字符串，包含要传递给编译器的附加标志。使用分号分隔的列表是无效的。CMAKE_<LANG>_FLAGS 及其关联的特定配置变量 CMAKE_<LANG>_FLAGS_<CONFIG> 的内容会自动预置到该变量内容之前的编译器命令中。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

3.12 版本新增: 当策略 CMP0075 设置为 NEW 时，新增了 CMAKE_REQUIRED_LIBRARIES 变量。

示例¶

检查一个或多个 C 头文件并将检查结果存储在缓存变量中

include(CheckIncludeFiles)

check_include_files(sys/socket.h HAVE_SYS_SOCKET_H)

if(HAVE_SYS_SOCKET_H)
  # The <net/if.h> header on Darwin and BSD-like systems is not self-contained
  # and also requires <sys/socket.h>
  check_include_files("sys/socket.h;net/if.h" HAVE_NET_IF_H)
else()
  check_include_files(net/if.h HAVE_NET_IF_H)
endif()

代码说明

  • include(CheckIncludeFiles)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_include_files(sys/socket.h HAVE_SYS_SOCKET_H)：调用 check_include_files() 命令，具体行为由这一行传入的参数决定。

  • if(HAVE_SYS_SOCKET_H)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • # The <net/if.h> header on Darwin and BSD-like systems is not self-contained：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • # and also requires <sys/socket.h>：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • check_include_files("sys/socket.h;net/if.h" HAVE_NET_IF_H)：调用 check_include_files() 命令，具体行为由这一行传入的参数决定。

  • else()：这一行结束或切换上一段控制结构的分支。

  • check_include_files(net/if.h HAVE_NET_IF_H)：调用 check_include_files() 命令，具体行为由这一行传入的参数决定。

  • endif()：这一行结束或切换上一段控制结构的分支。

LANGUAGE 选项可用于指定要使用的编译器。例如，当项目中同时启用了 C 和 CXX 语言时，检查多个 C++ 头文件。

include(CheckIncludeFiles)

check_include_files("header_1.hpp;header_2.hpp" HAVE_HEADERS LANGUAGE CXX)

代码说明

  • include(CheckIncludeFiles)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_include_files("header_1.hpp;header_2.hpp" HAVE_HEADERS LANGUAGE CXX)：调用 check_include_files() 命令，具体行为由这一行传入的参数决定。
