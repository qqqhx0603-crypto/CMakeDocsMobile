CheckOBJCCompilerFlag¶

3.16 版新增。

该模块提供了一个用于检查 Objective-C 编译器是否支持给定标志的命令。

在 CMake 项目中加载此模块，使用

include(CheckOBJCCompilerFlag)

代码说明

  • include(CheckOBJCCompilerFlag)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_objc_compiler_flag¶

检查 Objective-C 编译器是否支持给定标志（仅检查一次）

check_objc_compiler_flag(<flag> <variable>)

代码说明

  • check_objc_compiler_flag(<flag> <variable>)：调用 check_objc_compiler_flag() 命令，具体行为由这一行传入的参数决定。

该命令检查一次 <flag> 是否被 OBJC 编译器接受且不产生任何诊断消息。可以使用以分号分隔的列表形式，在单个参数中指定多个标志。

检查结果存储在由 <variable> 指定的内部缓存变量中，成功为布尔真，失败为布尔假。

成功结果仅表示编译器在给定该标志时未报告错误。该标志是否有任何效果或是否达到预期效果，不在此模块的范围之内。

注意

由于底层的 try_compile() 命令也使用 CMAKE_<LANG>_FLAGS 等变量中的标志，这些变量中未知或不支持的标志可能导致此检查出现误报（假阴性）。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

这是一个以空格分隔的字符串，包含要传递给编译器的额外标志。以分号分隔的列表在此处无效。CMAKE_<LANG>_FLAGS 及其关联的特定于配置的 CMAKE_<LANG>_FLAGS_<CONFIG> 变量的内容会自动置于此变量内容之前，传递给编译器命令。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

一个 分号分隔的列表，包含要添加到链接命令的选项（有关详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

一个 分号分隔的列表，包含要添加到链接命令的库。这些可以是系统库的名称，也可以是 导入的目标（有关详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

一个 分号分隔的列表，包含要传递给链接器的库搜索路径（有关详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

CMAKE_TRY_COMPILE_TARGET_TYPE

在内部，使用 try_compile() 命令执行检查，而此变量控制它创建的目标类型。如果此变量设置为 EXECUTABLE（默认值），则检查会将测试源代码编译并链接为一个可执行程序。如果设置为 STATIC_LIBRARY，则测试源代码将被编译但不会链接。

示例¶

以下示例演示了如何使用此模块检查 Objective-C 编译器标志 -fobjc-arc。检查结果存储在内部缓存变量 HAVE_OBJC_ARC 中，并使用 target_compile_options() 命令有条件地启用该标志。$<COMPILE_LANGUAGE:...> 生成器表达式确保该标志仅添加到 OBJC 源文件中。

include(CheckOBJCCompilerFlag)
check_objc_compiler_flag(-fobjc-arc HAVE_OBJC_ARC)

if(HAVE_OBJC_ARC)
  target_compile_options(
    example
    PRIVATE $<$<COMPILE_LANGUAGE:OBJC>:-fobjc-arc>
  )
endif()

代码说明

  • include(CheckOBJCCompilerFlag)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_objc_compiler_flag(-fobjc-arc HAVE_OBJC_ARC)：调用 check_objc_compiler_flag() 命令，具体行为由这一行传入的参数决定。

  • if(HAVE_OBJC_ARC)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • target_compile_options(：这一行继续补充当前命令或表达式的内容。

  • example：这一行继续补充当前命令或表达式的内容。

  • PRIVATE $<$<COMPILE_LANGUAGE:OBJC>:-fobjc-arc>：这一行继续补充当前命令或表达式的内容；生成器表达式：$<COMPILE_LANGUAGE:...> 在生成构建系统时按当前目标和配置计算结果；运算符：> 表示大于比较。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。
