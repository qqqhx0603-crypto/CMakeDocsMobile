FindThreads¶

查找并确定用于多线程支持的系统线程库

find_package(Threads [...])

代码说明

  • find_package(Threads [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

多线程可以在单个程序内实现并发执行，通常通过创建多个执行线程来完成。最常见的方法是使用诸如类 Unix 系统上的 POSIX 线程（pthreads）或 Windows（微软桌面系统） 上的 Windows（微软桌面系统） 线程之类的库。

该模块抽象了特定平台的差异，并检测如何启用线程支持——无论是需要链接到特定库、添加编译器标志（如 -pthread），还是两者兼有。在某些平台上，线程功能在默认库中隐式可用，无需使用额外的标志或库。

该模块适用于依赖系统级线程 API 的 C 和 C++ 项目（以及其他偶尔使用的编译语言）。

使用此模块可通过以可移植的方式处理线程支持的检测和设置，从而确保项目能够在不同平台上正确构建。

C 和 C++ 语言标准¶

C11 标准通过 <threads.h> 头文件引入了一个最小化的跨平台线程 API，而 C++11 在标准库中增加了 <thread> 头文件，提供了高级多线程支持。这些标准头文件允许在语言层面编写可移植的线程代码，而无需直接使用诸如 pthreads 或 Windows（微软桌面系统） 线程等特定平台的 API。

然而，即使可以使用标准的 C11 或 C++11 线程，某些应用程序可能仍然需要特定平台的编译器或链接器标志（例如类 Unix 系统上的 -pthread）。这就是 FindThreads 的用武之地——它确保这些标志和任何必要的库被正确设置，即使代码中没有显式使用系统 API。

简而言之

  • 在源代码中使用 <thread>（C++11 及更高版本）或 <threads.h>（C11）以实现可移植性和更简单的语法。

  • 当应用程序需要传统的线程支持并确保代码能够在不同平台上正确编译和链接时，在 CMake 项目中使用 find_package(Threads)。

导入的目标¶

此模块提供以下 导入目标

Threads::Threads

版本 3.1 中新增。

该目标封装了通过标志或线程库启用线程的使用需求（如果找到的话）。如果检测到支持线程，则该目标可用。

结果变量¶

此模块定义了以下变量

Threads_FOUND

布尔值，指示是否支持线程（通过单独的库或标准库）。

CMAKE_THREAD_LIBS_INIT

要使用的线程库。如果线程函数由系统库提供且无需特殊标志即可使用，则此变量可能为空。

CMAKE_USE_WIN32_THREADS_INIT

如果找到的线程库是 win32 线程库。

CMAKE_USE_PTHREADS_INIT

如果找到的线程库是 pthread 兼容的。

CMAKE_HP_PTHREADS_INIT

如果找到的线程库是 HP 线程库。

影响行为的变量¶

该模块在调用 find_package(Threads) 之前接受以下变量

THREADS_PREFER_PTHREAD_FLAG

版本 3.1 中新增。

如果偏好使用 -pthread 编译器和链接器标志，调用者可以将此变量设置为布尔值 true。该编译器标志只能与导入的目标一起使用。对于新代码，强烈建议同时使用导入的目标以及此开关。

如果系统库提供了线程函数（即 CMAKE_THREAD_LIBS_INIT 为空），则此变量无效。

示例¶

查找线程并将导入的目标链接到项目目标

set(THREADS_PREFER_PTHREAD_FLAG TRUE)
find_package(Threads)
target_link_libraries(example PRIVATE Threads::Threads)

代码说明

  • set(THREADS_PREFER_PTHREAD_FLAG TRUE) find_package(Threads) target_link_libraries(example PRIVATE Threads::Threads)：调用 `set()` 命令，把变量 `THREADS_PREFER_PTHREAD_FLAG` 设为给定的值。
