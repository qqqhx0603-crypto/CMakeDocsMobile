FindOpenThreads¶

查找 OpenThreads C++ 线程库

find_package(OpenThreads [...])

代码说明

  • find_package(OpenThreads [...])：调用 find_package() 命令，查找并加载外部依赖包。

OpenThreads 头文件应当按下述方式包含：

example.cxx¶
#include <OpenThreads/Thread>

代码说明

  • #include <OpenThreads/Thread>：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

结果变量¶

此模块定义了以下变量

OpenThreads_FOUND

3.3 版本中新增。

指示是否找到 OpenThreads 库的布尔值。

OPENTHREADS_LIBRARY

使用 OpenThreads 时需要链接的库。该变量提供发布版（优化）或调试版库变体，具体取决于项目的 构建配置 (Build Configurations)，它们会被分别查找。

缓存变量¶

以下缓存变量也可以设置

OPENTHREADS_INCLUDE_DIR

包含使用 OpenThreads 所需头文件的目录。

提示¶

此模块接受以下变量

OPENTHREADS_DIR

可以设置的环境变量或 CMake 变量，用于帮助查找安装在自定义位置的 OpenThreads 库。它应指向配置、构建和安装 OpenThreads 库时所使用的安装目标路径：./configure --prefix=$OPENTHREADS_DIR。

该模块最初是为了支持 FindOpenSceneGraph 模块及其组件而引入的。为了简化 OpenSceneGraph 软件包在进行上游开发和分发时的一步式自动化配置与构建，该模块支持额外的环境变量以在特定位置查找依赖项。这种方法优于在命令行中指定 -DVAR=value，因为它能更好地隔离模块内部变更，并且在独立于 CMAKE_*_PATH 变量指定各个 OSG 组件时提供了更大的灵活性。如果需要，显式的 -DVAR=value 参数仍然可以覆盖这些设置。由于 OpenThreads 是 OpenSceneGraph 的一个可选独立依赖项，为了方便起见，本模块也遵循以下变量：

OSG_DIR

可作为环境变量或 CMake 变量设置。处理方式与 OPENTHREADS_DIR 相同。

OSGDIR

处理方式与 OPENTHREADS_DIR 相同的环境变量。

已弃用变量¶

以下变量提供用于向后兼容性

OPENTHREADS_FOUND

自 4.2 版本起废弃：请使用 OpenThreads_FOUND，其值相同。

指示是否找到 OpenThreads 库的布尔值。

示例¶

查找 OpenThreads 库并创建一个 导入目标 (imported target) 接口，该接口封装了将其链接到项目目标所需的各项使用要求。

find_package(OpenThreads)

if(OpenThreads_FOUND AND NOT TARGET OpenThreads::OpenThreads)
  add_library(OpenThreads::OpenThreads INTERFACE IMPORTED)
  set_target_properties(
    OpenThreads::OpenThreads
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${OPENTHREADS_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${OPENTHREADS_LIBRARY}"
  )
endif()

target_link_libraries(example PRIVATE OpenThreads::OpenThreads)

代码说明

  • find_package(OpenThreads)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(OpenThreads_FOUND AND NOT TARGET OpenThreads::OpenThreads)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；AND 表示逻辑与：两侧条件都为真时结果为真；TARGET 表示判断给定构建目标是否已定义。

  • add_library(OpenThreads::OpenThreads INTERFACE IMPORTED)：调用 add_library() 命令，定义一个库目标。

  • set_target_properties(：这一行继续补充当前命令或表达式的内容。

  • OpenThreads::OpenThreads：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • INTERFACE_INCLUDE_DIRECTORIES "${OPENTHREADS_INCLUDE_DIR}"：这一行继续补充当前命令或表达式的内容；变量说明：OPENTHREADS_INCLUDE_DIR：包含目录、目录相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • INTERFACE_LINK_LIBRARIES "${OPENTHREADS_LIBRARY}"：这一行继续补充当前命令或表达式的内容；变量说明：OPENTHREADS_LIBRARY：库相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

  • target_link_libraries(example PRIVATE OpenThreads::OpenThreads)：调用 target_link_libraries() 命令，为目标链接依赖库。
