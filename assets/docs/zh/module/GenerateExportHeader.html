GenerateExportHeader¶

此模块提供了用于生成头文件的命令，该头文件包含用于控制 C/C++ 符号可见性的预处理器宏定义。

在 CMake 项目中加载此模块，使用

include(GenerateExportHeader)

代码说明

  • include(GenerateExportHeader)：调用 include() 命令，加载并执行另一个 CMake 脚本。

3.12 版本新增： 支持 C 项目。旧版本仅支持 C++ 项目。

在开发 C 或 C++ 项目（尤其是跨平台项目）时，符号可见性决定了哪些函数、类、全局变量、模板和其他符号对库的使用者可见。

例如，在 Windows（微软桌面系统） 上，构建共享库时必须显式标记符号为 __declspec(dllexport)，而在使用它时标记为 __declspec(dllimport)。其他平台可能会使用如 __attribute__((visibility("default"))) 等属性。

此模块简化了用于管理这些需求的预处理器宏的创建和使用，避免了源代码中重复且易错的 #ifdef 代码块。

某些符号可见性也可以通过编译器选项来控制。在 CMake 中，目标属性（如 <LANG>_VISIBILITY_PRESET 和 VISIBILITY_INLINES_HIDDEN）可在适当时启用编译器可见性标志。另请参阅相关的便捷变量 CMAKE_<LANG>_VISIBILITY_PRESET 和 CMAKE_VISIBILITY_INLINES_HIDDEN，以便在当前作用域内为所有目标启用这些设置。它们通常与此模块结合使用，以进一步简化 C/C++ 代码，从而无需在源代码中使用某些预处理器宏。

命令¶

本模块提供以下命令

生成导出头文件¶

generate_export_header¶

生成一个适合包含在源代码中的头文件，其中包含用于控制符号可见性的预处理器导出宏。

generate_export_header(
  <target>
  [BASE_NAME <base-name>]
  [EXPORT_FILE_NAME <export-file-name>]
  [EXPORT_MACRO_NAME <export-macro-name>]
  [NO_EXPORT_MACRO_NAME <no-export-macro-name>]
  [DEPRECATED_MACRO_NAME <deprecated-macro-name>]
  [DEFINE_NO_DEPRECATED]
  [NO_DEPRECATED_MACRO_NAME <no-deprecated-macro-name>]
  [STATIC_DEFINE <static-define>]
  [PREFIX_NAME <prefix>]
  [CUSTOM_CONTENT_FROM_VARIABLE <variable>]
  [INCLUDE_GUARD_NAME <include-guard-name>]
)

代码说明

  • generate_export_header(：这一行继续补充当前命令或表达式的内容。

  • <target>：这一行继续补充当前命令或表达式的内容。

  • [BASE_NAME <base-name>]：这一行继续补充当前命令或表达式的内容。

  • [EXPORT_FILE_NAME <export-file-name>]：这一行继续补充当前命令或表达式的内容。

  • [EXPORT_MACRO_NAME <export-macro-name>]：这一行继续补充当前命令或表达式的内容。

  • [NO_EXPORT_MACRO_NAME <no-export-macro-name>]：这一行继续补充当前命令或表达式的内容。

  • [DEPRECATED_MACRO_NAME <deprecated-macro-name>]：这一行继续补充当前命令或表达式的内容。

  • [DEFINE_NO_DEPRECATED]：这一行继续补充当前命令或表达式的内容。

  • [NO_DEPRECATED_MACRO_NAME <no-deprecated-macro-name>]：这一行继续补充当前命令或表达式的内容。

  • [STATIC_DEFINE <static-define>]：这一行继续补充当前命令或表达式的内容。

  • [PREFIX_NAME <prefix>]：这一行继续补充当前命令或表达式的内容。

  • [CUSTOM_CONTENT_FROM_VARIABLE <variable>]：这一行继续补充当前命令或表达式的内容。

  • [INCLUDE_GUARD_NAME <include-guard-name>]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

默认情况下，此命令会在当前二进制目录（CMAKE_CURRENT_BINARY_DIR）中生成一个名为 <target-name-lowercase>_export.h 的头文件。此头文件定义了一组预处理器宏，用于在不同平台和构建类型（例如静态或共享构建）中将 API 符号标记为导出、隐藏或已弃用。由于它会影响公共 API 声明，因此该文件应与库的公共头文件一起安装。

  • <MACRO>_EXPORT：标记要导出或导入的符号，使其在构建或使用共享库时作为公共 API 的一部分可见。

  • <MACRO>_NO_EXPORT：标记不应导出的符号。如果 <LANG>_VISIBILITY_PRESET 目标属性设置为 hidden，则在源代码中使用此宏通常是冗余的。

  • <MACRO>_DEPRECATED：将符号标记为已弃用。当使用此类符号时，编译器会在编译时发出警告。

  • <MACRO>_DEPRECATED_EXPORT：结合了导出/导入和弃用标记，适用于既是公共 API 一部分又已弃用的符号。

  • <MACRO>_DEPRECATED_NO_EXPORT：标记不应导出的已弃用符号（内部且已弃用）。

  • <MACRO>_NO_DEPRECATED：一个可在源代码中使用的宏，用于通过预处理器逻辑从构建中有条件地排除弃用的代码部分。

<MACRO> 部分默认派生自目标的大写名称或显式提供的 <base-name>。所有宏名称都可以使用可选参数进行自定义。

参数为

<target>

要为其生成导出头文件的目标的名称。支持的目标类型：

  • STATIC 库（在这种情况下，导出相关的宏定义为空值）

  • SHARED 库

  • MODULE 库

  • 3.1 版本新增： OBJECT 库

BASE_NAME <base-name>

如果指定，它将覆盖默认的文件名和宏名称。

EXPORT_FILE_NAME <export-file-name>

如果指定，它将生成的导出头文件的完整路径和名称（默认为 <base-name-lowercase>_export.h）覆盖为 <export-file-name>。如果给定时为相对路径，它将被解释为相对于当前二进制目录（CMAKE_CURRENT_BINARY_DIR）。

EXPORT_MACRO_NAME <export-macro-name>

如果指定，它将覆盖导出指令的默认宏名称。

NO_EXPORT_MACRO_NAME <no-export-macro-name>

如果指定，<no-export-macro-name> 将用于表示不应导出项目的属性的宏名称。

DEPRECATED_MACRO_NAME <deprecated-macro-name>

如果指定，将使用以下名称：

  • <deprecated-macro-name> （用于标记已弃用符号的宏）

  • <deprecated-macro-name>_EXPORT （用于带有导出标记的已弃用符号的宏）

  • <deprecated-macro-name>_NO_EXPORT （用于带有禁止导出标记的已弃用符号的宏）

代替默认名称格式 <MACRO>_DEPRECATED{,_EXPORT,_NO_EXPORT}。

DEFINE_NO_DEPRECATED

如果指定，这将定义一个名为 <MACRO>_NO_DEPRECATED 的宏。

NO_DEPRECATED_MACRO_NAME <no-deprecated-macro-name>

与 DEFINE_NO_DEPRECATED 选项结合使用。如果指定，则使用名为 <no-deprecated-macro-name> 的宏，而不是默认的 <MACRO>_NO_DEPRECATED。

STATIC_DEFINE <static-define>

如果指定，将使用 <static-define> 宏名称而不是默认的 <MACRO>_STATIC_DEFINE。此宏控制生成头文件中静态库的符号导出行为。它通常用于通过单个生成的导出头文件从相同源构建库的共享和静态变体时。当为静态库定义此宏时，导出相关的宏将扩展为空。这在 Windows（微软桌面系统） 上也很重要，因为只有共享库才需要符号修饰，而静态库则不需要。

PREFIX_NAME <prefix>

如果指定，则在所有生成的宏名称前加上附加的 <prefix>。

CUSTOM_CONTENT_FROM_VARIABLE <variable>

3.7 版本中新增。

如果指定，来自 <variable> 的值将被附加到生成头文件的预处理器宏定义之后。

INCLUDE_GUARD_NAME <include-guard-name>

3.11 版本新增。

如果指定，<include-guard-name> 将用作预处理器宏名称，以防止多次包含生成的头文件，取代默认名称 <export-macro-name>_H。

<base-name-lowercase>_export.h¶
#ifndef <include-guard-name>
#define <include-guard-name>
// ...
#endif /* <include-guard-name> */

代码说明

  • #ifndef <include-guard-name>：这一行开始预处理条件分支，按宏条件决定后续代码是否参与编译。

  • #define <include-guard-name>：这一行定义预处理宏，供后续代码在编译前展开使用。

  • // ...：这是代码注释。此处省略了与当前要点无关的代码。

  • #endif /* <include-guard-name> */：这一行切换或结束预处理条件分支。

已弃用的命令¶

add_compiler_export_flags¶

3.0 版本弃用： 请改用目标属性 CXX_VISIBILITY_PRESET 和 VISIBILITY_INLINES_HIDDEN。

添加 C++ 编译器选项 -fvisibility=hidden （如果支持，还包括 -fvisibility-inlines-hidden），以默认隐藏所有符号，该选项将被添加到 CMAKE_CXX_FLAGS 变量或指定的变量中。

add_compiler_export_flags([<output_variable>])

代码说明

  • add_compiler_export_flags([<output_variable>])：调用 add_compiler_export_flags() 命令，具体行为由这一行传入的参数决定。

此命令在 Windows（微软桌面系统） 上是空操作，因为 Windows（微软桌面系统） 不需要额外的编译器标志来支持导出。

<output-variable>

可选变量名，它将填充为一串空格分隔的 C++ 编译选项，这些选项是启用当前编译器/架构的可见性支持所必需的。如果指定了此参数，则不会修改 CMAKE_CXX_FLAGS 变量。

示例：生成导出头文件¶

以下示例演示了如何使用此模块在当前二进制目录中生成导出头文件（example_export.h），并在名为 example 的 C++ 库中使用它来控制符号可见性。生成的头文件定义了预处理器宏 EXAMPLE_EXPORT、EXAMPLE_NO_EXPORT、EXAMPLE_DEPRECATED、EXAMPLE_DEPRECATED_EXPORT 和 EXAMPLE_DEPRECATED_NO_EXPORT，并与库的其他公共头文件一起安装。

CMakeLists.txt¶
cmake_minimum_required(VERSION 3.24)
project(GenerateExportHeaderExample)

# Set default visibility of all symbols to hidden
set(CMAKE_CXX_VISIBILITY_PRESET "hidden")
set(CMAKE_VISIBILITY_INLINES_HIDDEN TRUE)

add_library(example)

include(GenerateExportHeader)
generate_export_header(example)

target_sources(
  example
  PRIVATE example.cxx
  PUBLIC
    FILE_SET HEADERS
      FILES example.h
    FILE_SET generated_headers
      TYPE HEADERS
      BASE_DIRS ${CMAKE_CURRENT_BINARY_DIR}
      FILES ${CMAKE_CURRENT_BINARY_DIR}/example_export.h
)

install(
  TARGETS example
  FILE_SET HEADERS
  FILE_SET generated_headers
)

示例：自定义生成的头文件¶

BASE_NAME 参数可用于覆盖生成的文件名和宏名称。以下内容将生成一个名为 other_name_export.h 的文件，其中包含导出相关的宏，如 OTHER_NAME_EXPORT、OTHER_NAME_NO_EXPORT、OTHER_NAME_DEPRECATED 等。

add_library(example example.cxx)
include(GenerateExportHeader)
generate_export_header(example BASE_NAME "other_name")

代码说明

  • add_library(example example.cxx)：调用 add_library() 命令，定义一个库目标。

  • include(GenerateExportHeader)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • generate_export_header(example BASE_NAME "other_name")：调用 generate_export_header() 命令，具体行为由这一行传入的参数决定。

BASE_NAME 可以通过指定其他命令选项来覆盖。例如，以下代码创建了一个宏 OTHER_NAME_EXPORT，而不是 EXAMPLE_EXPORT，但其他宏和生成的头文件名设置为默认值。

add_library(example example.cxx)
include(GenerateExportHeader)
generate_export_header(example EXPORT_MACRO_NAME "OTHER_NAME_EXPORT")

代码说明

  • add_library(example example.cxx)：调用 add_library() 命令，定义一个库目标。

  • include(GenerateExportHeader)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • generate_export_header(example EXPORT_MACRO_NAME "OTHER_NAME_EXPORT")：调用 generate_export_header() 命令，具体行为由这一行传入的参数决定。

以下示例创建了 KDE_DEPRECATED 宏，而不是默认的 EXAMPLE_DEPRECATED。

add_library(example example.cxx)
include(GenerateExportHeader)
generate_export_header(example DEPRECATED_MACRO_NAME "KDE_DEPRECATED")
