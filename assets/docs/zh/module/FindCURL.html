FindCURL¶

查找用于 URL 数据传输的原生 curl 安装（包含目录和库）

find_package(CURL [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(CURL [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

3.17 版本新增: 如果 curl 是使用其基于 CMake 的构建系统构建的，它将提供自己的 CMake 包配置文件 (CURLConfig.cmake)，以便在配置模式 (config mode) 下通过 find_package() 命令使用。默认情况下，该模块会搜索此文件，如果找到，则直接返回结果，无需进一步操作。如果未找到上游配置文件，该模块将回退到模块模式 (module mode) 并搜索标准位置。

3.13 版本新增: Debug 和 Release 版本的库会分开查找。

有关支持的协议和特性的更多信息，请参考 curl 文档。组件名称区分大小写，并遵循上游 curl 的命名规范。

导入的目标¶

此模块提供以下 导入目标

CURL::libcurl

3.12 版本新增。

封装 curl 使用需求的导入目标，在找到 curl 时可用。

结果变量¶

此模块定义了以下变量

CURL_FOUND

布尔值，指示是否找到了（所请求版本的）curl 以及所有必需的组件。

CURL_VERSION

4.0 版本新增。

找到的 curl 版本。此变量取代了 CURL_VERSION_STRING。

CURL_<component>_FOUND

3.14 版新增。

布尔值，指示是否找到了指定的组件（curl 协议或特性）。

CURL_INCLUDE_DIRS

包含 curl/curl.h 及使用 curl 所需的其他头文件的目录。

注意

当通过配置模式找到 curl 时，此变量仅在 curl 8.9 或更高版本中可用。

CURL_LIBRARIES

使用 curl 进行链接时所需的库列表。

注意

当通过模块模式找到 curl 时，这是一个库文件路径列表。在配置模式下，此变量仅在 curl 8.9 或更高版本中可用，且包含一个导入目标列表。

提示¶

此模块接受以下变量

CURL_NO_CURL_CMAKE

在 3.17 版本中添加。

将此变量设置为 TRUE 可禁用通过配置模式搜索 curl。

CURL_USE_STATIC_LIBS

版本 3.28 新增。

将此变量设置为 TRUE 以使用静态库。这仅在 curl 未通过配置模式找到时有效。

已弃用变量¶

以下变量提供用于向后兼容性

CURL_VERSION_STRING

4.0 版本弃用: 由 CURL_VERSION 取代。

找到的 curl 版本。

示例¶

查找 curl 库并指定所需的最低版本

find_package(CURL 7.61.0)

代码说明

  • find_package(CURL 7.61.0)：调用 find_package() 命令，查找并加载外部依赖包。

查找 curl 库并将其链接到项目目标

find_package(CURL)
target_link_libraries(project_target PRIVATE CURL::libcurl)

代码说明

  • find_package(CURL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE CURL::libcurl)：调用 target_link_libraries() 命令，为目标链接依赖库。

使用组件检查已找到的 curl 是否支持特定协议或特性

find_package(CURL OPTIONAL_COMPONENTS HTTPS SSL)

if(CURL_HTTPS_FOUND)
  # curl supports the HTTPS protocol
endif()

if(CURL_SSL_FOUND)
  # curl has SSL feature enabled
endif()
