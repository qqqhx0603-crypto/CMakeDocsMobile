FindJNI¶

查找 Java 本地接口 (JNI) 的包含目录和库

find_package(JNI [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(JNI [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

JNI 使得在 Java 虚拟机 (JVM) 或 Android 上的 Dalvik 虚拟机 (DVM) 中运行的 Java 代码能够调用其他语言（如 C 和 C++）编写的本地应用程序和库，并被其调用。

此模块用于查找是否安装了 Java，并确定包含文件和库的位置。它还会确定库的名称。

版本 3.24 新增: 导入的目标、组件和 Android NDK 支持。

使用 Android NDK 时，会报告相应的包版本，并可以请求特定的发行版。在 Android API 31 及以上级别，可以请求额外的 NativeHelper 组件。NativeHelper 也作为 JVM 组件的隐式依赖项公开（前提是不会导致冲突），它提供了对 JVM 函数的统一访问。

组件¶

此模块支持可选组件，这些组件可以通过 find_package() 命令指定

find_package(JNI [COMPONENTS <components>...])

代码说明

  • find_package(JNI [COMPONENTS <components>...])：调用 find_package() 命令，查找并加载外部依赖包。

支持的组件包括

AWT

在 3.24 版本中添加。

查找 Java 抽象窗口工具包 (AWT)。

JVM

在 3.24 版本中添加。

查找 Java 虚拟机 (JVM)。

NativeHelper

在 3.24 版本中添加。

查找 Android 上的 NativeHelper 库 (libnativehelper.so)，该库公开了诸如 JNI_CreateJavaVM() 之类的 JVM 函数。

如果未指定组件，模块默认值为：

  • 针对 Android API 31 及以上级别时：模块查找 NativeHelper 组件。对于其他 Android API 级别，默认不设置组件。

  • 针对其他系统时：模块查找 AWT 和 JVM 组件。

导入的目标¶

此模块提供以下 导入目标

JNI::JNI

在 3.24 版本中添加。

封装了所有 JNI 使用要求的主目标，如果找到 jni.h 则可用。

JNI::AWT

在 3.24 版本中添加。

封装了 Java AWT 本地接口 (JAWT) 库使用要求的目标，如果找到 AWT 组件则可用。

JNI::JVM

在 3.24 版本中添加。

封装了 Java 虚拟机 (JVM) 库使用要求的目标，如果找到 JVM 组件则可用。

JNI::NativeHelper

在 3.24 版本中添加。

封装了 NativeHelper 库使用要求的目标，在针对 Android API 31 及以上级别且找到该库时可用。

结果变量¶

此模块定义了以下变量

JNI_FOUND

布尔值，指示是否找到了（所请求版本的）JNI。

JNI_<component>_FOUND

在 3.24 版本中添加。

布尔值，指示是否找到 <component>。

JNI_VERSION

完整的 Android NDK 包版本（包括后缀，如 -beta3 和 -rc1），否则未定义。

JNI_VERSION_MAJOR

在 3.24 版本中添加。

Android NDK 主版本号，否则未定义。

JNI_VERSION_MINOR

在 3.24 版本中添加。

Android NDK 次版本号，否则未定义。

JNI_VERSION_PATCH

在 3.24 版本中添加。

Android NDK 修补版本号，否则未定义。

JNI_INCLUDE_DIRS

使用 JNI 所需的包含目录。

JNI_LIBRARIES

使用 JNI 进行链接时所需的库（JAWT 和 JVM）。

缓存变量¶

以下缓存变量也可用于设置或使用

JAVA_INCLUDE_PATH

包含 jni.h 头文件的目录。

JAVA_INCLUDE_PATH2

包含与机器相关的头文件 jni_md.h 和 jniport.h 的目录。此变量仅在 jni.h 依赖于其中一个头文件时定义。相比之下，Android NDK 的 jni.h 通常可以独立使用。

JAVA_AWT_INCLUDE_PATH

包含 jawt.h 头文件的目录。

JAVA_AWT_LIBRARY

Java AWT 本地接口 (JAWT) 库的路径。

JAVA_JVM_LIBRARY

Java 虚拟机 (JVM) 库的路径。

提示¶

此模块接受以下变量

JAVA_HOME

调用者可以设置此变量来显式指定 Java 的安装目录。

示例¶

查找 JNI 并将其链接到项目目标

find_package(JNI)
target_link_libraries(project_target PRIVATE JNI::JNI)

代码说明

  • find_package(JNI)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE JNI::JNI)：调用 target_link_libraries() 命令，为目标链接依赖库。

在指定 AWT 组件的情况下查找 JNI 并将其链接到项目目标

find_package(JNI COMPONENTS AWT)
target_link_libraries(project_target PRIVATE JNI::JNI JNI::AWT)

代码说明

  • find_package(JNI COMPONENTS AWT)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE JNI::JNI JNI::AWT)：调用 target_link_libraries() 命令，为目标链接依赖库。

在 CMake 中使用 Java 和 JNI 的更常见方式是使用专用的 UseJava 模块

find_package(Java)
find_package(JNI)
include(UseJava)

代码说明

  • find_package(Java)：调用 find_package() 命令，查找并加载外部依赖包。

  • find_package(JNI)：调用 find_package() 命令，查找并加载外部依赖包。

  • include(UseJava)：调用 include() 命令，加载并执行另一个 CMake 脚本。
