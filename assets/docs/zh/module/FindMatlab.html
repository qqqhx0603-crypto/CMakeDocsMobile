FindMatlab¶

查找 MATLAB 或 MATLAB 编译器运行时 (MCR)，并将其工具、库和编译器提供给 CMake

find_package(Matlab [<version>] [COMPONENTS <components>...] [REGISTRY_VIEW <view>] [...])

代码说明

  • find_package(Matlab [<version>] [COMPONENTS <components>...] [REGISTRY_VIEW <view>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

该模块的主要目的是查找与 MATLAB 或 MCR 关联的库，以便能够构建 MATLAB 扩展（MEX 文件）。它还可以用于：

  • 在 MATLAB 可用时，在 MATLAB 中运行特定命令

  • 声明 MATLAB 单元测试

  • 从 MATLAB 检索各种信息（MEX 扩展、版本和发行版查询等）

3.12 版本新增：MATLAB 编译器运行时 (MCR) 支持。

3.30 版本新增：支持为 find_package() 指定版本范围；支持为 find_package() 指定 REGISTRY_VIEW 参数；新增 matlab_extract_all_installed_versions_from_registry() 和 matlab_get_all_valid_matlab_roots_from_registry() 命令。默认行为保持不变，仍使用注册表视图 TARGET。

注意

传递给 find_package() 的版本是 MATLAB 的“版本号”，不要将其与 MATLAB 的“发行版名称”（例如 R2023b）混淆。matlab_get_version_from_release_name() 和 matlab_get_release_name_from_version() 命令提供了发行版名称与版本号之间的映射。

组件¶

该模块支持可选组件，可以使用 find_package() 命令指定：

find_package(Matlab [COMPONENTS <components>...])

代码说明

  • find_package(Matlab [COMPONENTS <components>...])：调用 `find_package()` 命令，查找并加载外部依赖包。

支持的组件包括

查找 MATLAB 的 MAT 库。

输入变量¶

用户或项目可以在调用 find_package(Matlab) 之前设置以下变量来配置模块行为：

Matlab_ROOT

在 3.25 版本中新增。

Matlab_ROOT_DIR 变量的默认值，即 MATLAB 安装的根目录。

Matlab_ROOT_DIR

3.3 版本中新增。

MATLAB 安装的根文件夹。如果在此之前设置了调用 find_package()，模块将在该路径中查找组件。如果未设置，则执行 MATLAB 的自动搜索。如果设置，它应指向一个有效的 MATLAB 版本。

可以指定此变量以提供所需 MATLAB 版本的路径。否则，行为取决于平台：

  • Windows（微软桌面系统）：已安装的 MATLAB/MCR 版本从 Windows（微软桌面系统） 注册表中检索。可以可选地指定 REGISTRY_VIEW 参数以手动控制应搜索 32 位还是 64 位版本。

  • macOS（苹果桌面系统）：已安装的 MATLAB/MCR 版本由 $HOME/Applications 和 /Applications 下的 MATLAB 默认安装路径提供。如果未找到此类应用程序，它将回退到可能从 PATH 访问的路径。

  • Unix：所需的 MATLAB 应可从 PATH 中访问。这不适用于 MCR 安装，在此平台上应指定 Matlab_ROOT_DIR。

MATLAB_FIND_DEBUG

3.3 版本中新增。

如果设置为布尔值真，则会将额外的调试信息输出到控制台，例如 MATLAB 的查找过程和中间配置步骤。

MATLAB_ADDITIONAL_VERSIONS

3.3 版本中新增。

如果设置，它将指定额外的 MATLAB 版本，用于自动检索可能被处理和查找的已安装版本。

发行版名称与 MATLAB 版本的映射通过定义对（名称，版本）来执行。该变量应为字符串列表，按发行版名称和版本的对进行组织，如下所示：

set(
  MATLAB_ADDITIONAL_VERSIONS
    "release_name1=corresponding_version1"
    "release_name2=corresponding_version2"
    # ...
)

find_package(Matlab)

代码说明

  • set( MATLAB_ADDITIONAL_VERSIONS "release_name1=corresponding_version1" "release_name2=corresponding_version2" # ... ) find_package(Matlab)：调用 `set()` 命令，把变量 `MATLAB_ADDITIONAL_VERSIONS` 设为给定的值。

示例

set(
  MATLAB_ADDITIONAL_VERSIONS
    "R2013b=8.2"
    "R2013a=8.1"
    "R2012b=8.0"
    # ...
)

find_package(Matlab)

代码说明

  • set( MATLAB_ADDITIONAL_VERSIONS "R2013b=8.2" "R2013a=8.1" "R2012b=8.0" # ... ) find_package(Matlab)：调用 `set()` 命令，把变量 `MATLAB_ADDITIONAL_VERSIONS` 设为给定的值。

当安装了多个 MATLAB 版本时，此列表中条目的顺序很重要。优先级根据此列表中的顺序设置。

matlab_extract_all_installed_versions_from_registry¶

3.3 版本中新增。

解析 Windows（微软桌面系统） 注册表并查找所有已安装的 MATLAB 版本。

matlab_extract_all_installed_versions_from_registry(<versions-var> [REGISTRY_VIEW <view>])¶

参数为

<versions-var>

存储找到的所有 MATLAB 版本列表的变量名称。

REGISTRY_VIEW <view>

3.30 版本新增。

可选的注册表视图参数，提供了与 Windows（微软桌面系统） 注册表交互的更精确接口。该参数被传递（或省略）给 cmake_host_system_information()，无需进一步检查或修改。例如，<view> 值可以是 64、32、64_32、32_64、HOST、TARGET、BOTH 之一。

旧的签名是：

matlab_extract_all_installed_versions_from_registry(<win64> <versions-var>)¶

参数为

<win64>

布尔值，是否搜索 64 位版本的 MATLAB。如果设置为布尔值真，将搜索 64 位注册表视图。如果设置为布尔值假，将搜索 32 位注册表视图。为了更精细的控制，请使用上面的签名。

<versions-var>

存储找到的所有 MATLAB 版本列表的变量名称。

此命令仅在 Windows（微软桌面系统） 上可用。解析的注册表部分取决于主机处理器。

返回的列表包含 HKLM\SOFTWARE\Mathworks\MATLAB、HKLM\SOFTWARE\Mathworks\MATLAB Runtime 和 HKLM\SOFTWARE\Mathworks\MATLAB Compiler Runtime 下的所有版本，如果发生错误（或未找到任何内容），则返回空列表。

注意

仅提供版本号。不对注册表中引用的安装的存在性进行检查。

matlab_add_mex¶

3.3 版本中新增。

添加一个编译 MATLAB MEX 目标文件的目标。

matlab_add_mex(
  NAME <name>
  [EXECUTABLE | MODULE | SHARED]
  SRC <sources>...
  [OUTPUT_NAME <output-name>]
  [DOCUMENTATION <file>]
  [LINK_TO <targets>...]
  [R2017b | R2018a]
  [EXCLUDE_FROM_ALL]
  [NO_IMPLICIT_LINK_TO_MATLAB_LIBRARIES]
  [...]
)

代码说明

  • matlab_add_mex( NAME <name> [EXECUTABLE | MODULE | SHARED] SRC <sources>... [OUTPUT_NAME <output-name>] [DOCUMENTATION <file>] [LINK_TO <targets>...] [R2017b | R2018a] [EXCLUDE_FROM_ALL] [NO_IMPLICIT_LINK_TO_MATLAB_LIBRARIES] [...] )：调用 `matlab_add_mex()` 命令，具体行为由这一行传入的参数决定。

该命令使用当前工具链编译给定的源文件，以生成 MEX 文件。可以指定生成输出的最终名称、额外的链接库以及 MEX 文件的文档条目。调用的其余参数将传递给 add_library() 或 add_executable() 命令。

参数为

NAME <name>

目标名称。

SRC <sources>...

要编译的一个或多个源文件。

LINK_TO <targets（目标）>...

额外链接依赖项的列表。除非传递了 NO_IMPLICIT_LINK_TO_MATLAB_LIBRARIES 选项，否则目标默认链接到 libmex 和 libmx。

OUTPUT_NAME <output-name>

如果给定，将覆盖默认名称。默认名称是目标名称（无前缀），并带有 Matlab_MEX_EXTENSION 后缀。

DOCUMENTATION <file>

如果给定，<file> 将被视为 MEX 文件的文档文件。此文件会被原样复制到同一文件夹中，名称与最终 MEX 文件相同，后缀为 .m。在这种情况下，在 MATLAB 中键入 help <name> 会打印该文件中包含的文档。

文档文件不会被处理，应采用以下格式：

% This is the documentation
function ret = mex_target_output_name(input1)

代码说明

  • % This is the documentation function ret = mex_target_output_name(input1)：这一行给出示例文本或输出，用来说明：% This is the documentation function ret = mex_target_output_name(input1)

R2017b 或 R2018a

3.14 版新增。

可以给定以指定要使用的 C API 版本：R2017b 指定传统的（单独复杂数据）C API，对应于 mex 命令的 -R2017b 标志。R2018a 指定新的交错复数 C API，对应于 mex 命令的 -R2018a 标志。对于 R2018a 之前的 MATLAB 版本，此选项将被忽略。默认为 R2017b。

MODULE 或 SHARED

3.7 版本中新增。

可以给定以指定要创建的库类型。

EXECUTABLE

3.7 版本中新增。

可以给定以创建可执行文件而不是库。如果未显式给定类型，则默认类型为 SHARED。

EXCLUDE_FROM_ALL

此选项与 EXCLUDE_FROM_ALL 目标属性含义相同，并被转发给 add_library() 或 add_executable() 命令。

NO_IMPLICIT_LINK_TO_MATLAB_LIBRARIES

在 3.24 版本中添加。

此选项允许禁用 MATLAB 库的自动链接，以便仅通过 LINK_TO 选项链接实际需要的库。

matlab_add_unit_test¶

3.3 版本中新增。

将 MATLAB 单元测试文件添加到项目的 CMake/CTest 测试集中。

matlab_add_unit_test(
  NAME <name>
  UNITTEST_FILE <matlab-file-containing-unittest.m>
  [CUSTOM_TEST_COMMAND <matlab-command-to-run-as-test>]
  [UNITTEST_PRECOMMAND <matlab-command-to-run>]
  [TIMEOUT <timeout>]
  [ADDITIONAL_PATH <paths>...]
  [MATLAB_ADDITIONAL_STARTUP_OPTIONS <options>...]
  [TEST_ARGS <args>...]
  [NO_UNITTEST_FRAMEWORK]
  [WORKING_DIRECTORY <dir>]
)

代码说明

  • matlab_add_unit_test( NAME <name> UNITTEST_FILE <matlab-file-containing-unittest.m> [CUSTOM_TEST_COMMAND <matlab-command-to-run-as-test>] [UNITTEST_PRECOMMAND <matlab-command-to-run>] [TIMEOUT <timeout>] [ADDITIONAL_PATH <paths>...] [MATLAB_ADDITIONAL_STARTUP_OPTIONS <options>...] [TEST_ARGS <args>...] [NO_UNITTEST_FRAMEWORK] [WORKING_DIRECTORY <dir>] )：调用 `matlab_add_unit_test()` 命令，具体行为由这一行传入的参数决定。

默认情况下，将使用 MATLAB 单元测试框架（>= 2013a）来运行添加的 MATLAB 脚本，但也可以使用返回退出代码的常规 .m 文件（0 表示成功）。

此命令需要 MAIN_PROGRAM 组件，因此不适用于 MCR 安装。

该单元测试使用 MATLAB 单元测试框架（默认，适用于 MATLAB 2013b+），除非给出了 NO_UNITTEST_FRAMEWORK 选项。

该命令要求提供一个 MATLAB 测试脚本文件。如果给出了 NO_UNITTEST_FRAMEWORK，则单元测试脚本文件应包含要运行的脚本，以及带有退出值的退出命令。此退出值将传递给 CTest 框架（0 为成功，非 0 为失败）。可以通过 TEST_ARGS 传递 add_test() 接受的其他参数（例如 CONFIGURATIONS <config> ...）。

参数为

NAME <name>

CTest 中单元测试的名称。

UNITTEST_FILE <matlab-file-containing-unittest.m>

MATLAB 单元测试文件。其路径将自动添加到 MATLAB 路径中。

CUSTOM_TEST_COMMAND <matlab-command-to-run-as-test>

作为测试运行的 MATLAB 脚本命令。如果未设置，则运行以下内容：runtests('matlab_file_name'), exit(max([ans(1,:).Failed]))，其中 matlab_file_name 是不带扩展名的 UNITTEST_FILE。

UNITTEST_PRECOMMAND <matlab-command-to-run>

在包含测试的文件之前运行的 MATLAB 脚本命令（例如，基于 CMake 变量的 GPU 设备初始化）。

TIMEOUT <timeout>

测试超时（秒）。默认为 180 秒，因为 MATLAB 单元测试可能会挂起。

ADDITIONAL_PATH <paths>...

在运行单元测试之前添加到 MATLAB 路径的路径列表。

MATLAB_ADDITIONAL_STARTUP_OPTIONS <options>

从命令行运行 MATLAB 的其他选项列表。-nosplash、-nodesktop 和 -nodisplay 选项总是自动添加。

TEST_ARGS <args>...

提供给 add_test 命令的其他选项。这些选项将添加到默认选项中（例如 CONFIGURATIONS Release）。

NO_UNITTEST_FRAMEWORK

设置后，表示测试不应使用 MATLAB 的单元测试框架（适用于 >= R2013a 的版本）。

WORKING_DIRECTORY <dir>

这将是测试的工作目录。如果指定，它也将是测试运行日志文件使用的输出目录。如果未指定，则使用临时目录 ${CMAKE_BINARY_DIR}/Matlab 作为工作目录和日志位置。

已知问题¶

MEX 目标中的符号冲突

默认情况下，使用命令 matlab_add_mex() 定义的 MEX 文件中的每个符号都具有隐藏可见性，入口点除外。这是 MEX 编译器的默认行为，降低了 MATLAB 随附库与 MEX 文件所链接库之间发生符号冲突的风险。这在 Windows（微软桌面系统） 平台上也是默认设置。

然而，在某些情况下，例如 MEX 文件链接到 MATLAB 已经加载的库时，这可能还不够，即使这些库具有不同的 SONAME。一种可能的解决方案是隐藏 MEX 目标链接的库的符号。这可以通过 GNU GCC 编译器使用链接器选项 -Wl,--exclude-libs,ALL 来实现。

使用 GPU 资源的测试

如果 MEX 文件正在使用 GPU，并且为了能够在此 MEX 文件上运行单元测试，GPU 资源应由 MATLAB 正确释放。一种可能的解决方案是让 MATLAB 在会话中意识到 GPU 资源的使用，这可以通过在测试脚本开头（或通过固定装置）执行命令（如 D = gpuDevice()）来实现。
