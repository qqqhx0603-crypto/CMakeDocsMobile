FindOpenMP¶

在编译器中查找 Open Multi-Processing (OpenMP) 支持

find_package(OpenMP [<version>] [COMPONENTS <langs>...] [...])

代码说明

  • find_package(OpenMP [<version>] [COMPONENTS <langs>...] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

如果编译器支持 OpenMP，则编译所需的 OpenMP 支持标志将以变量形式返回给对应的语言。如果编译器不需要特殊标志来支持 OpenMP，这些变量可能为空。

3.5 版本新增：Clang 支持。

组件¶

本模块支持使用标准语法指定的组件

find_package(OpenMP [COMPONENTS <langs>...])

代码说明

  • find_package(OpenMP [COMPONENTS <langs>...])：调用 `find_package()` 命令，查找并加载外部依赖包。

这些组件中的每一个都控制着搜索 OpenMP 支持的各种语言。以下是公开的组件：

C

3.10 版本新增。

CXX

3.10 版本新增。

Fortran

3.10 版本新增。

CUDA

在版本 3.31 中添加。

当使用支持主机端 OpenMP 的 CUDA 编译器时，支持 CUDA 语言组件。

如果未指定组件，模块将根据项目中启用的语言自动检查所有组件。

导入的目标¶

此模块提供以下 导入目标

OpenMP::OpenMP_<lang>

版本 3.9 中添加。

封装了语言 <lang> 的 OpenMP 使用要求的导入目标（Target）。

输入变量¶

可以设置以下变量来影响此模块的行为：

OpenMP_RUNTIME_MSVC

3.30 版本新增。

使用 MSVC 编译时指定 OpenMP 运行时。如果设置为非空值（例如 experimental 或 llvm），它将作为 -openmp: 标志的值传递。

结果变量¶

此模块定义了以下变量

OpenMP_FOUND

3.10 版本新增。

布尔值，指示是否找到了所有请求语言的 OpenMP 标志。如果未指定组件，则在检测到所有启用语言的 OpenMP 设置时，此变量设置为真（true）。

OpenMP_VERSION

版本 4.2 中添加。

在所请求语言中检测到的 OpenMP 标准最低版本；如果未指定组件，则为所有启用语言中的最低版本。

本模块将为项目中的每种语言设置以下变量，其中 <lang> 为 C、CXX、CUDA 或 Fortran 之一：

OpenMP_<lang>_FOUND

版本 3.9 中添加。

布尔值，指示是否检测到（请求版本）<lang> 的 OpenMP 支持。

OpenMP_<lang>_VERSION

版本 3.9 中添加。

如果可用，由 <lang> 编译器实现的 OpenMP 版本。

OpenMP_<lang>_VERSION_MAJOR

版本 3.9 中添加。

如果可用，由 <lang> 编译器实现的 OpenMP 主版本号。

OpenMP_<lang>_VERSION_MINOR

版本 3.9 中添加。

如果可用，由 <lang> 编译器实现的 OpenMP 次版本号。

OpenMP_<lang>_SPEC_DATE

3.7 版本中新增。

如果可用，由 <lang> 编译器实现的 OpenMP 规范日期。

规范日期格式遵循 OpenMP 标准：yyyymm，其中 yyyy 和 mm 分别表示 <lang> 编译器实现的 OpenMP 规范的年份和月份。

OpenMP_<lang>_FLAGS

<lang> 的 OpenMP 编译器标志，以空格分隔。

OpenMP_<lang>_INCLUDE_DIRS

3.16 版新增。

使用 OpenMP 时，必须添加到 <lang> 头文件搜索路径中的目录。

对于链接使用 <lang> 编写的 OpenMP 代码，提供以下变量：

OpenMP_<lang>_LIB_NAMES

版本 3.9 中添加。

一个分号分隔的列表，包含 <lang> 的 OpenMP 程序库名称。

OpenMP_<libname>_LIBRARY

版本 3.9 中添加。

<lang> 中 OpenMP 支持所需的各个库的位置。<libname> 名称存储在 OpenMP_<lang>_LIB_NAMES 列表中。

OpenMP_<lang>_LIBRARIES

版本 3.9 中添加。

链接使用 <lang> 编写的 OpenMP 代码所需的库列表。

专门针对 Fortran，模块设置以下变量：

OpenMP_Fortran_HAVE_OMPLIB_HEADER

版本 3.9 中添加。

布尔值，指示是否可以通过 omp_lib.h Fortran 头文件访问 OpenMP。

OpenMP_Fortran_HAVE_OMPLIB_MODULE

版本 3.9 中添加。

布尔值，指示是否可以通过 omp_lib Fortran 模块访问 OpenMP。

对于某些编译器，可能需要添加头文件搜索路径才能找到相关的 OpenMP 头文件。此位置可能与语言有关。在需要时，模块可能会尝试查找该位置，但也支持通过设置 OpenMP_<lang>_INCLUDE_DIR 缓存变量直接提供。请注意，该变量是本模块的*输入*控制项。项目代码应使用 OpenMP_<lang>_INCLUDE_DIRS *输出*变量来了解所需的包含目录，或者最好使用 OpenMP::OpenMP_<lang> 导入目标。

示例¶

使用 C 语言组件查找 OpenMP 支持并将导入目标链接到项目目标

find_package(OpenMP)
target_link_libraries(project_target PRIVATE OpenMP::OpenMP_C)

代码说明

  • find_package(OpenMP) target_link_libraries(project_target PRIVATE OpenMP::OpenMP_C)：调用 `find_package()` 命令，查找并加载外部依赖包。
