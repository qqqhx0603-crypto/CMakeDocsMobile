FindSQLite3¶

3.14 版新增。

查找 SQLite 3 库

find_package(SQLite3 [<version>] [...])

代码说明

  • find_package(SQLite3 [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

SQLite 是一个小型、快速、自包含、高可靠性且功能齐全的 SQL 数据库引擎，用 C 语言编写，旨在嵌入到应用程序中。

导入的目标¶

此模块提供以下 导入目标

SQLite3::SQLite3

封装 SQLite 库使用需求的 target。只有在找到 SQLite 时才可用。

4.3 版本新增。

SQLite::SQLite3

已弃用。与 SQLite3::SQLite3 相同。

如果您的项目需要支持 CMake < 4.3，请在调用 find_package(SQLite3 ...) 后，向您的项目添加以下内容：

if(NOT TARGET SQLite3::SQLite3) # CMake < 4.3
  add_library(SQLite3::SQLite3 ALIAS SQLite::SQLite3)
endif()

代码说明

  • if(NOT TARGET SQLite3::SQLite3) # CMake < 4.3 add_library(SQLite3::SQLite3 ALIAS SQLite::SQLite3) endif()：开始一个 `if()` 条件判断，根据条件结果选择是否执行后续语句。

这将允许您的项目使用新的名称，同时仍然允许它与旧版本的 CMake 编译。

结果变量¶

此模块定义了以下变量

SQLite3_FOUND

布尔值，指示是否找到了（请求版本的）SQLite 库。

SQLite3_VERSION

找到的 SQLite 库的版本。

SQLite3_INCLUDE_DIRS

包含 <sqlite3.h> 和相关头文件以使用 SQLite 的包含目录。

SQLite3_LIBRARIES

需要链接以使用 SQLite 的库。

示例¶

查找 SQLite 库并将其链接到项目 target

find_package(SQLite3)
target_link_libraries(project_target PRIVATE SQLite3::SQLite3)

代码说明

  • find_package(SQLite3) target_link_libraries(project_target PRIVATE SQLite3::SQLite3)：调用 `find_package()` 命令，查找并加载外部依赖包。
