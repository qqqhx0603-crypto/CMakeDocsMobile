FindDoxygen¶

查找源代码文档生成器 Doxygen 以及一些可选的支持工具，并提供用于将基于 Doxygen 的文档集成到 CMake 项目中的命令。

find_package(Doxygen [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(Doxygen [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

组件¶

可以通过 find_package() 命令将额外的 Doxygen 支持工具指定为组件

find_package(Doxygen [COMPONENTS <components>...])

代码说明

  • find_package(Doxygen [COMPONENTS <components>...])：调用 find_package() 命令，查找并加载外部依赖包。

支持的组件包括

doxygen

版本 3.9 中添加。

查找 doxygen 可执行文件。该组件即使未显式请求也会自动包含在内。

dot

版本 3.9 中添加。

查找 Graphviz 的 dot 工具，用于在文档中渲染图表和流程图。

mscgen

版本 3.9 中添加。

查找 Doxygen 的 \msc 和 \mscfile 命令所使用的 消息图生成器 (Message Chart Generator) 工具。

dia

版本 3.9 中添加。

查找 Doxygen 的 \diafile 命令所使用的 Dia 图表编辑器。

导入的目标¶

该模块提供以下 导入的目标 (Imported Targets)。如果请求了相应的组件且找到了关联的可执行文件，则会定义这些目标：

Doxygen::doxygen

版本 3.9 中添加。

封装了 doxygen 可执行文件使用要求的导入可执行目标，在找到 Doxygen 时可用。

Doxygen::dot

版本 3.9 中添加。

封装了 dot 可执行文件使用要求的导入可执行目标，在找到上述 dot 组件时可用。

Doxygen::mscgen

版本 3.9 中添加。

封装了 mscgen 可执行文件使用要求的导入可执行目标，在找到上述 mscgen 组件时可用。

Doxygen::dia

版本 3.9 中添加。

封装了 dia 可执行文件使用要求的导入可执行目标，在找到上述 dia 组件时可用。

这些目标可用于 add_custom_command() 等命令，并且优先于较旧且现已弃用的变量（如 DOXYGEN_EXECUTABLE）。

结果变量¶

此模块定义了以下变量

Doxygen_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）doxygen 可执行文件以及所有请求的必需组件。

Doxygen_VERSION

版本 4.2 中添加。

找到的 Doxygen 版本（由 doxygen --version 报告）。

命令¶

此模块提供以下命令

doxygen_add_docs¶

版本 3.9 中添加。

添加一个自定义目标，用于在构建阶段使用 Doxygen 生成文档

doxygen_add_docs(
  <target-name>
  [<files-or-dirs>...]
  [ALL]
  [USE_STAMP_FILE]
  [WORKING_DIRECTORY <dir>]
  [COMMENT <comment>]
  [CONFIG_FILE <file>]
)

代码说明

  • doxygen_add_docs(：这一行继续补充当前命令或表达式的内容。

  • <target-name>：这一行继续补充当前命令或表达式的内容。

  • [<files-or-dirs>...]：这一行继续补充当前命令或表达式的内容。

  • [ALL]：这一行继续补充当前命令或表达式的内容。

  • [USE_STAMP_FILE]：这一行继续补充当前命令或表达式的内容。

  • [WORKING_DIRECTORY <dir>]：这一行继续补充当前命令或表达式的内容。

  • [COMMENT <comment>]：这一行继续补充当前命令或表达式的内容。

  • [CONFIG_FILE <file>]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

默认情况下，此便捷命令也会在 CMake 配置阶段于当前二进制目录中生成名为 Doxyfile.<target-name> 的配置文件。它提供了合理的默认设置，因此大多数项目只需指定输入文件或目录。可以使用以下章节中描述的变量来自定义附加行为和配置。

参数为

<target-name>

为使用 Doxygen 生成文档而创建的目标名称。

<files-or-dirs>...

一个或多个作为文档输入源的路径（文件或目录）。

这些路径会被传递给生成的 Doxyfile.<target-name> 中的 INPUT Doxygen 配置标签。此处列出的文件也会作为底层 add_custom_target() 命令的 SOURCES 参数添加，以便它们出现在 IDE 项目的源列表中。

使用 USE_STAMP_FILE 选项时，仅允许使用文件（不允许使用目录、符号链接或通配符），且每个文件在调用此命令时都必须存在。

ALL

3.12 版本新增。

将创建的文档目标添加到默认构建目标中，以便在构建阶段自动运行。

USE_STAMP_FILE

3.16 版新增。

启用使用时间戳文件 (stamp file)，以避免在源文件未更改的情况下重新生成文档。

由底层自定义命令在当前二进制目录中创建名为 <target-name>.stamp 的时间戳文件。

存在此选项时，调用此命令时 <files-or-dirs> 中的所有条目必须是已存在的文件（即不能是目录、符号链接或通配符）。如果任何列出的路径无效，则会引发错误。

如果不使用此选项，无论 <files-or-dirs> 中列出的任何输入源文件是否已更改，CMake 都会在每次构建 <target-name> 目标时重新运行 Doxygen。

WORKING_DIRECTORY <dir>

默认情况下，Doxygen 的工作目录是当前源目录（CMAKE_CURRENT_SOURCE_DIR）。这与使用相对输入路径的要求一致。

使用此选项可以更改和覆盖 Doxygen 运行所在的目录。此时，绝对路径 <dir> 将用作相对路径的基准点。

另请注意，Doxygen 的默认行为是从生成的文档中的相对路径中剥离工作目录。详情请参阅 Doxygen 手册 中的 STRIP_FROM_PATH 配置标签。

COMMENT <comment>

如果提供了此选项，<comment> 字符串将作为 COMMENT 参数传递给内部创建自定义目标的底层 add_custom_target() 命令。当构建该目标时，此信息会显示在构建系统的输出中。

CONFIG_FILE <file>

在 3.27 版本中新增。

如果指定，给定的全路径文件将用作 Doxygen 配置文件，而不是默认的 Doxyfile.<target-name>。

在 CMake 中，可以通过设置 DOXYGEN_<tag> 格式的输入变量来修改这些标签，其中 <tag> 是 Doxygen 手册 中列出的配置标签之一。

例如，要修改 GENERATE_HTML 和 GENERATE_MAN 配置标签，可以在调用 doxygen_add_docs() 之前设置以下变量：

CMakeLists.txt¶
find_package(Doxygen)

if(Doxygen_FOUND)
  set(DOXYGEN_GENERATE_HTML NO)
  set(DOXYGEN_GENERATE_MAN YES)

  doxygen_add_docs(project_docs ${PROJECT_SOURCE_DIR})
endif()

代码说明

  • find_package(Doxygen)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(Doxygen_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • set(DOXYGEN_GENERATE_HTML NO)：调用 set() 命令，把变量 DOXYGEN_GENERATE_HTML 设为给定的值；变量说明：DOXYGEN_GENERATE_HTML：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • set(DOXYGEN_GENERATE_MAN YES)：调用 set() 命令，把变量 DOXYGEN_GENERATE_MAN 设为给定的值；变量说明：DOXYGEN_GENERATE_MAN：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • doxygen_add_docs(project_docs ${PROJECT_SOURCE_DIR})：调用 doxygen_add_docs() 命令，具体行为由这一行传入的参数决定；变量说明：PROJECT_SOURCE_DIR：最近一次 project() 调用所定义项目的源代码根目录。

  • endif()：这一行结束或切换上一段控制结构的分支。

默认配置

默认情况下，doxygen_add_docs() 会覆盖 Doxygen 的多项设置，以更符合典型的 CMake 项目。除非在调用 doxygen_add_docs() 之前已定义，否则将显式设置以下每个变量（下文标注的少数例外情况除外）：

DOXYGEN_HAVE_DOT

如果请求并找到了 dot 组件，则设置为 YES，否则为 NO。忽略 DOXYGEN_HAVE_DOT 的任何现有值。

DOXYGEN_DOT_MULTI_TARGETS

由本模块设置为 YES（注意，这需要版本高于 1.8.10 的 dot）。此选项仅在 DOXYGEN_HAVE_DOT 也设置为 YES 时才有意义。

DOXYGEN_GENERATE_LATEX

由本模块设置为 NO。

DOXYGEN_WARN_FORMAT

对于基于 Visual Studio（集成开发环境） 的生成器，此项设置为 Visual Studio（集成开发环境） IDE 可识别的格式：$file($line) : $text。对于所有其他生成器，不覆盖 Doxygen 的默认值。

DOXYGEN_PROJECT_NAME

使用当前项目的名称填充（即 PROJECT_NAME）。

DOXYGEN_PROJECT_NUMBER

使用当前项目的版本填充（即 PROJECT_VERSION）。

DOXYGEN_PROJECT_BRIEF

使用当前项目的描述填充（即 PROJECT_DESCRIPTION）。

DOXYGEN_INPUT

此变量会自动填充为传递给 doxygen_add_docs() 的文件和目录列表。为了与其他内置命令（如 add_executable()、add_library() 和 add_custom_target()）保持行为一致，项目不应手动设置此变量。如果项目设置了名为 DOXYGEN_INPUT 的变量，它将被忽略并会发出警告。

DOXYGEN_RECURSIVE

由本模块设置为 YES。

DOXYGEN_EXCLUDE_PATTERNS

如果 doxygen_add_docs() 的 <files-or-dirs> 参数包含目录，此变量将指定用于从中排除文件的模式。默认添加以下模式以确保 CMake 特定的文件和目录不被包含在输入中。如果项目设置了此变量，这些内容将与这些额外模式合并，而不是覆盖它们。

DOXYGEN_VERBATIM_VARS

3.11 版本新增。

一个 CMake 输入变量，由 doxygen_add_docs() 使用，用于指定 Doxygen 输入变量列表（包括它们的前导 DOXYGEN_ 前缀），这些变量的值应在不进行自动引用的情况下传递给生成的 Doxyfile 配置。

使用此变量时，项目负责确保这些变量的值在直接放置到生成的 Doxyfile 配置中时是有意义的。对于列表变量，项在输出中仍以空格分隔，但不对单个项应用引用。

对于某些 Doxygen 标签（如 ALIASES），doxygen_add_docs() 执行的自动引用可能会干扰正确的语法（例如，嵌入的引号）。

例如，以下内容将引用 DOXYGEN_PROJECT_BRIEF，但跳过 DOXYGEN_ALIASES 列表中的每一项（使用 方括号语法 可以更轻松地处理嵌入的引号）：

CMakeLists.txt¶
find_package(Doxygen)

if(Doxygen_FOUND)
  set(DOXYGEN_PROJECT_BRIEF "String with spaces")
  set(
    DOXYGEN_ALIASES
    [[somealias="@some_command param"]]
    "anotherAlias=@foobar"
  )
  set(DOXYGEN_VERBATIM_VARS DOXYGEN_ALIASES)

  add_doxygen_docs(project_docs ${PROJECT_SOURCE_DIR})
endif()

代码说明

  • find_package(Doxygen)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(Doxygen_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • set(DOXYGEN_PROJECT_BRIEF "String with spaces")：调用 set() 命令，把变量 DOXYGEN_PROJECT_BRIEF 设为给定的值；变量说明：DOXYGEN_PROJECT_BRIEF：项目相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • set(：这一行继续补充当前命令或表达式的内容。

已弃用变量¶

为了与之前版本的 CMake 兼容，还定义了以下变量，但它们已被弃用，不应再使用。

DOXYGEN_FOUND

自 4.2 版本起弃用：使用 Doxygen_FOUND。

布尔值，指示是否找到了（所请求版本的）doxygen 可执行文件以及所有请求的必需组件。它的布尔值为 YES 或 NO。

DOXYGEN_EXECUTABLE

自 3.9 版本起弃用：使用 Doxygen::doxygen 导入的目标，而不是直接引用 doxygen 可执行文件。

包含 doxygen 命令路径的缓存变量。

DOXYGEN_DOT_FOUND

自 3.9 版本起弃用：使用 Doxygen_dot_FOUND。

布尔结果变量，指示是否找到了 dot 可执行文件。

DOXYGEN_DOT_EXECUTABLE

自 3.9 版本起弃用：使用 Doxygen::dot 导入的目标，而不是直接引用 dot 可执行文件。

包含 dot 命令行可执行文件路径的缓存变量。

DOXYGEN_DOT_PATH

自 3.9 版本起弃用。

结果变量，包含 DOXYGEN_DOT_EXECUTABLE 中报告的 dot 可执行文件所在目录的路径。即使在 Windows（微软桌面系统） 上，该路径也可能具有正斜杠，不适合直接替换到 Doxyfile.in 模板中。如果需要此值，请获取 Doxygen::dot 目标的 IMPORTED_LOCATION 属性，并使用 get_filename_component() 提取该路径的目录部分。同时考虑使用 file(TO_NATIVE_PATH) 为 Doxygen 配置文件准备路径。

DOXYGEN_SKIP_DOT

自 3.9 版本起弃用。

此提示变量在 find_package(Doxygen COMPONENTS ...) 中指定组件时无效。在向后兼容模式下（即不指定组件），它会阻止此查找模块搜索 Graphviz 的 dot 工具。

DOXYGEN_VERSION

自 4.2 版本起弃用：已被 Doxygen_VERSION 取代。

找到的 Doxygen 版本。

示例：使用变量配置 Doxygen¶

在以下示例中，使用 CMake 变量自定义 Doxygen 配置。该配置设置了在使用目录作为源输入（CMAKE_CURRENT_SOURCE_DIR）时的文件模式，启用了用于在浅色和深色模式之间切换的主题切换，在构建阶段抑制了 Doxygen 的标准输出，将 Markdown 文件指定为首页，并禁用了有关未记录代码的警告。

find_package(Doxygen)

if(Doxygen_FOUND)
  set(DOXYGEN_FILE_PATTERNS *.c *.cxx *.md)
  set(DOXYGEN_HTML_COLORSTYLE "TOGGLE")
  set(DOXYGEN_QUIET YES)

  doxygen_add_docs(example_docs ${CMAKE_CURRENT_SOURCE_DIR} ALL)
