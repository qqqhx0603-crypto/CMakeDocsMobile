DeployQt4¶

注意

此模块适用于 Qt 4 版本。新代码应遵循 cmake-qt(7)，而不是使用此模块。

此模块提供了一组 CMake 实用程序命令，用于组装和部署独立的 Qt4 可执行文件。

在 CMake 项目中加载此模块，使用

include(DeployQt4)

命令¶

本模块提供以下命令

  • write_qt4_conf()

  • resolve_qt4_paths()

  • fixup_qt4_executable()

  • install_qt4_plugin_path()

  • install_qt4_plugin()

  • install_qt4_executable()

write_qt4_conf¶

创建 Qt 配置文件

write_qt4_conf(<qt-conf-dir> <qt-conf-contents>)

此命令将带有 <qt-conf-contents> 内容的 qt.conf 文件写入 <qt-conf-dir> 目录。

resolve_qt4_paths¶

将相对路径解析为绝对路径

resolve_qt4_paths(<paths-var> [<executable-path>])

此命令循环遍历 <paths-var> 列表；如果路径不存在，它将相对于 <executable-path>（如果已提供）或 CMAKE_INSTALL_PREFIX 进行解析。

fixup_qt4_executable¶

修复 Qt4 可执行文件

fixup_qt4_executable(
  <executable>
  [<qtplugins> <libs> <dirs> <plugins-dir> <request-qt-conf>]
)

此命令复制 Qt 插件，写入 Qt 配置文件（如果需要），并使用 BundleUtilities 模块修复 Qt4 可执行文件，使其成为独立的程序。只要所有系统库兼容，就可以将其直接复制到其他机器上运行。

<executable> 应指向需要修复的可执行文件。

<qtplugins> 应包含要安装的任何 Qt 插件的名称或路径列表。

<libs> 将被传递给 BundleUtilities 模块，应为任何已安装的插件、库或需要修复的可执行文件的列表。

<dirs> 将被传递给 BundleUtilities 模块，应包含用于搜索库依赖项的目录。

<plugins-dir> 允许使用自定义的插件目录。

<request-qt-conf> 将强制写入 qt.conf 文件，即使不需要该文件。

install_qt4_plugin_path¶

安装已解析的 Qt4 插件

install_qt4_plugin_path(
  <plugin>
  <executable>
  <copy>
  <installed-plugin-path-var>
  [<plugins-dir> <component> <configurations>]
)

此命令将已解析的 <plugin> 安装（或复制）到相对于 <executable> 的默认插件目录（或 <plugins-dir>），并将结果存储在变量 <installed-plugin-path-var> 中。

如果 <copy> 设置为 TRUE，则插件将被复制而不是安装。这是为了允许在 CMake 执行时（而非安装时）使用此模块。

如果设置了 <component>，则安装的任何内容都将使用此 COMPONENT。

install_qt4_plugin¶

安装未解析的 Qt4 插件

install_qt4_plugin(
  <plugin>
  <executable>
  <copy>
  <installed-plugin-path-var>
  [<plugins-dir> <component>]
)

此命令将未解析的 <plugin> 安装（或复制）到相对于 <executable> 的默认插件目录（或 <plugins-dir>），并将结果存储在变量 <installed-plugin-path-var> 中。有关其他参数，请参阅 install_qt4_plugin_path()。

install_qt4_executable¶

安装 Qt 插件，写入 Qt 配置文件（如果需要），并修复 Qt4 可执行文件

install_qt4_executable(
  <executable>
  [<qtplugins> <libs> <dirs> <plugins-dir> <request-qt-conf> <component>]
)

此命令使用 BundleUtilities 模块，使可执行文件能够独立运行，并且只要所有系统库兼容，就可以将其直接复制到其他机器上使用。可执行文件将在安装时进行修复。<component> 是用于捆绑包修复和插件安装的 COMPONENT。有关其他参数，请参阅 fixup_qt4_executable()。
