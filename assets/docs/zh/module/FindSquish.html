FindSquish¶

查找 Squish，这是一个用于各种 GUI 技术构建的应用程序的跨平台自动化 GUI 测试框架。

find_package(Squish [<version>] [...])

代码说明

  • find_package(Squish [<version>] [...])：调用 find_package() 命令，查找并加载外部依赖包。

Squish 支持测试原生和跨平台工具包，例如 Qt、Java 和 Tk。

结果变量¶

此模块定义了以下变量

Squish_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）Squish。

Squish_VERSION

版本 4.2 中添加。

找到的 Squish 的完整版本号。

SQUISH_INSTALL_DIR_FOUND

布尔值，指示是否找到了 Squish 安装目录。

SQUISH_SERVER_EXECUTABLE_FOUND

布尔值，指示是否找到了 Squish 服务器可执行文件。

SQUISH_CLIENT_EXECUTABLE_FOUND

布尔值，指示是否找到了 Squish 客户端可执行文件。

缓存变量¶

以下缓存变量也可以设置

SQUISH_INSTALL_DIR

包含 bin、lib 等的 Squish 安装目录。

SQUISH_SERVER_EXECUTABLE

squishserver 可执行文件的路径。

SQUISH_CLIENT_EXECUTABLE

squishrunner 可执行文件的路径。

命令¶

如果找到了 Squish，此模块将提供以下命令：

squish_add_test¶

向项目中添加 Squish 测试

squish_add_test(
  <name>
  AUT <target>
  SUITE <suite-name>
  TEST <squish-test-case-name>
  [PRE_COMMAND <command>]
  [POST_COMMAND <command>]
  [SETTINGSGROUP <group>]
)

代码说明

  • squish_add_test(：这一行继续补充当前命令或表达式的内容。

  • <name>：这一行继续补充当前命令或表达式的内容。

  • AUT <target>：这一行继续补充当前命令或表达式的内容。

  • SUITE <suite-name>：这一行继续补充当前命令或表达式的内容。

  • TEST <squish-test-case-name>：这一行继续补充当前命令或表达式的内容；条件运算符：TEST 表示判断给定测试是否已定义。

  • [PRE_COMMAND <command>]：这一行继续补充当前命令或表达式的内容。

  • [POST_COMMAND <command>]：这一行继续补充当前命令或表达式的内容。

  • [SETTINGSGROUP <group>]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

此命令基于 add_test() 命令构建，并将名为 <name> 的 Squish 测试添加到 CMake 项目中。它支持 Squish 4 及更新版本。

在 CMake 测试阶段，Squish 服务器会启动，测试在客户端执行，测试完成后服务器停止。如果这些步骤中的任何一个失败（包括测试本身失败），则会引发致命错误，指示测试未通过。

参数为

<name>

测试的名称。这会作为第一个参数传递给 add_test() 命令。

AUT <target>

用作 AUT（被测应用程序）的 CMake 目标名称，即将被测试的可执行文件。

SUITE <suite-name>

Squish 测试套件的完整路径，或者仅套件名称（即套件的最后一个目录名）。在后一种情况下，调用 squish_add_test() 的 CMakeLists.txt 必须位于套件的父目录中。

TEST <squish-test-case-name>

Squish 测试的名称，对应于套件目录中测试的子目录。

PRE_COMMAND <command>

在启动 Squish 测试之前执行的可选命令。请以字符串形式传递。这可以是一个单一命令，或者是命令和参数的分号分隔列表。

POST_COMMAND <command>

在 Squish 测试完成后执行的可选命令。请以字符串形式传递。这可以是一个单一命令，或者是命令和参数的分号分隔列表。

SETTINGSGROUP <group>

自 3.18 版本起弃用：此参数现在被忽略。它以前用于指定执行测试时的设置组名称，而不是默认值 CTest_<username>。

3.18 版本变更：在以前的 CMake 版本中，此命令名为 squish_v4_add_test()。

squish_v3_add_test¶

在使用 Squish 3.x 版本时，向项目中添加 Squish 测试

squish_v3_add_test(
  <test-name>
  <application-under-test>
  <squish-test-case-name>
  <environment-variables>
  <test-wrapper>
)

代码说明

  • squish_v3_add_test(：这一行继续补充当前命令或表达式的内容。

  • <test-name>：这一行继续补充当前命令或表达式的内容。

  • <application-under-test>：这一行继续补充当前命令或表达式的内容。

  • <squish-test-case-name>：这一行继续补充当前命令或表达式的内容。

  • <environment-variables>：这一行继续补充当前命令或表达式的内容。

  • <test-wrapper>：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

注意

此命令适用于不再维护的 Squish 3 版本。请使用较新的 Squish 版本和 squish_add_test() 命令。

参数为

<name>

测试的名称。

<application-under-test>

用作 AUT（被测应用程序）的可执行文件路径，即将被测试的可执行文件。

<squish-test-case-name>

Squish 测试的名称，对应于套件目录中测试的子目录。

<environment-variables>

由分号分隔的环境变量及其值列表（VAR=VALUE）。

<test-wrapper>

测试用例所需的一个或多个（分号分隔列表）测试包装器的字符串。

已弃用变量¶

以下变量提供用于向后兼容性

SQUISH_FOUND

自 4.2 版本起弃用：请使用 Squish_FOUND，其值相同。

布尔值，指示是否找到了（所请求版本的）Squish。

SQUISH_VERSION

自 4.2 版本起弃用：已被 Squish_VERSION 取代。

找到的 Squish 的完整版本号。

SQUISH_VERSION_MAJOR

自 4.2 版本起弃用：已被 Squish_VERSION 取代。

找到的 Squish 的主版本号。

SQUISH_VERSION_MINOR

自 4.2 版本起弃用：已被 Squish_VERSION 取代。

找到的 Squish 的次版本号。

SQUISH_VERSION_PATCH

自 4.2 版本起弃用：已被 Squish_VERSION 取代。

找到的 Squish 的补丁版本号。

示例¶

查找 Squish 并指定最低版本要求

find_package(Squish 6.5)

代码说明

  • find_package(Squish 6.5)：调用 find_package() 命令，查找并加载外部依赖包。

添加 Squish 测试

enable_testing()

find_package(Squish 6.5)
if(Squish_FOUND)
  squish_add_test(
    projectTestName
    AUT projectApp
    SUITE ${CMAKE_CURRENT_SOURCE_DIR}/tests/projectSuite
    TEST someSquishTest
  )
endif()

代码说明

  • enable_testing()：调用 enable_testing() 命令，开启测试支持。

  • find_package(Squish 6.5)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(Squish_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • squish_add_test(：这一行继续补充当前命令或表达式的内容。

  • projectTestName：这一行继续补充当前命令或表达式的内容。

  • AUT projectApp：这一行继续补充当前命令或表达式的内容。

  • SUITE ${CMAKE_CURRENT_SOURCE_DIR}/tests/projectSuite：这一行继续补充当前命令或表达式的内容；变量说明：CMAKE_CURRENT_SOURCE_DIR：当前正在处理的源代码目录。

  • TEST someSquishTest：这一行继续补充当前命令或表达式的内容；条件运算符：TEST 表示判断给定测试是否已定义。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

关于如何使用 squish_v3_add_test() 命令的示例

enable_testing()

find_package(Squish 3.0)
if(Squish_FOUND)
  squish_v3_add_test(
    projectTestName
    $<TARGET_FILE:projectApp>
    someSquishTest
    "FOO=1;BAR=2"
    testWrapper
  )
endif()

代码说明

  • enable_testing()：调用 enable_testing() 命令，开启测试支持。

  • find_package(Squish 3.0)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(Squish_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • squish_v3_add_test(：这一行继续补充当前命令或表达式的内容。

  • projectTestName：这一行继续补充当前命令或表达式的内容。

  • $<TARGET_FILE:projectApp>：这一行继续补充当前命令或表达式的内容；生成器表达式：$<TARGET_FILE:...> 在生成构建系统时按当前目标和配置计算结果。

  • someSquishTest：这一行继续补充当前命令或表达式的内容。

  • "FOO=1;BAR=2"：这一行继续补充当前命令或表达式的内容；运算符：= 表示赋值或键值关联。

  • testWrapper：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。
