UsewxWidgets¶

此模块作为使用 wxWidgets 库（以前称为 wxWindows）的便捷包装器，并将其实用需求（例如库目录、包含目录和编译器标志）传播到当前目录的作用域，供目标使用。

在 CMake 项目中加载此模块，使用

include(UsewxWidgets)

代码说明

  • include(UsewxWidgets)：调用 include() 命令，加载并执行另一个 CMake 脚本。

此模块调用 include_directories() 和 link_directories()，设置当前目录的编译定义，并在调用 find_package(wxWidgets) 后将一些编译标志附加到使用 wxWidgets 库。

示例¶

在找到 wxWidgets 后，将此模块包含在项目中以配置其实用需求

CMakeLists.txt¶
# Note that for MinGW users the order of libraries is important.
find_package(wxWidgets COMPONENTS net gl core base)

add_library(example example.cxx)

if(wxWidgets_FOUND)
  include(UsewxWidgets)

  # Link wxWidgets libraries for each dependent executable/library target.
  target_link_libraries(example PRIVATE ${wxWidgets_LIBRARIES})
endif()

代码说明

  • # Note that for MinGW users the order of libraries is important.：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • find_package(wxWidgets COMPONENTS net gl core base)：调用 find_package() 命令，查找并加载外部依赖包。

  • add_library(example example.cxx)：调用 add_library() 命令，定义一个库目标。

  • if(wxWidgets_FOUND)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句。

  • include(UsewxWidgets)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • # Link wxWidgets libraries for each dependent executable/library target.：这是 CMake 注释。这是代码作者保留的说明，描述当前步骤的目的、输入输出或使用限制；英文原文保留在代码中。

  • target_link_libraries(example PRIVATE ${wxWidgets_LIBRARIES})：调用 target_link_libraries() 命令，为目标链接依赖库；变量说明：wxWidgets_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • endif()：这一行结束或切换上一段控制结构的分支。

从 CMake 3.27 开始，更好的方法是仅将 wxWidgets::wxWidgets 导入目标链接到需要它的特定目标，而不是包含此模块。导入目标通过仅将其链接到的目标应用它们，从而更好地控制包的使用属性（例如包含目录和编译标志），避免将不必要的传播到当前目录中的所有目标。
