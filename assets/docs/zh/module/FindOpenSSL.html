FindOpenSSL¶

查找已安装的 OpenSSL 加密库并确定其版本

find_package(OpenSSL [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(OpenSSL [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

3.20 版本新增： 支持在调用 find_package() 命令时指定版本范围。当请求版本时，可以像以前一样指定单个值，现在也可以使用版本范围。有关版本范围用法和功能的详细描述，请参阅 find_package() 命令。

3.18 版本新增： 支持 OpenSSL 3.0。

组件¶

此模块支持以下可选组件：

Crypto

3.12 版本新增。

确保找到 OpenSSL crypto 库。

SSL

3.12 版本新增。

确保找到 OpenSSL ssl 库。

可以使用标准语法选择性地指定组件。

find_package(OpenSSL [COMPONENTS <components>...])

代码说明

  • find_package(OpenSSL [COMPONENTS <components>...])：调用 find_package() 命令，查找并加载外部依赖包。

如果不请求任何组件，模块默认会将 Crypto 作为必需组件查找，将 SSL 作为可选组件查找。

导入的目标¶

此模块提供以下 导入目标

OpenSSL::Crypto

3.4 版本新增。

封装 OpenSSL crypto 库使用要求的目标，仅在找到 crypto 库时可用。

OpenSSL::SSL

3.4 版本新增。

封装 OpenSSL ssl 库使用要求的目标，仅在找到 ssl 库时可用。为方便起见，该目标还会链接 OpenSSL::Crypto，因为 ssl 库依赖于 crypto 库。

OpenSSL::applink

在 3.18 版本中新增。

封装 OpenSSL 应用程序端接口（openssl/applink.c）使用要求的目标，仅在找到 OpenSSL 且版本至少为 0.9.8 时可用。

该接口提供了 OpenSSL BIO 层与 Windows（微软桌面系统） 编译器运行时环境之间的粘合剂，在使用 MSVC 时可能需要编译到项目中。通过链接此目标，即使项目使用不同的 MSVC 运行时配置，也可以链接其他 OpenSSL 导入目标。在非 MSVC 平台上链接此目标无效。

注意

接口文件通过 INTERFACE_SOURCES 目标属性添加。由于 CMake 中传播接口源的方式，建议将 OpenSSL::applink 目标作为 PRIVATE 链接，以确保它在任何库或可执行文件的整个依赖关系图中只被链接一次。

target_link_libraries(project_target PRIVATE OpenSSL::applink)

代码说明

  • target_link_libraries(project_target PRIVATE OpenSSL::applink)：调用 target_link_libraries() 命令，为目标链接依赖库。

为此目标使用其他作用域可能会导致构建或链接过程中出现意外问题，因为 ISO C 和 ISO C++ 标准对链接行为的规定非常少。

结果变量¶

此模块定义了以下变量

OpenSSL_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）OpenSSL 库。

OpenSSL_VERSION

版本 4.2 中添加。

找到的 OpenSSL 版本。设置为 <major>.<minor>.<revision><patch> 格式（例如 0.9.8s）。

OPENSSL_INCLUDE_DIR

OpenSSL 头文件目录。

OPENSSL_CRYPTO_LIBRARY

OpenSSL crypto 库。

OPENSSL_CRYPTO_LIBRARIES

OpenSSL crypto 库及其依赖项。

OPENSSL_SSL_LIBRARY

OpenSSL ssl 库。

OPENSSL_SSL_LIBRARIES

OpenSSL ssl 库及其依赖项。

OPENSSL_LIBRARIES

所有 OpenSSL 库及其依赖项。

OPENSSL_APPLINK_SOURCE

上述目标 OpenSSL::applink 中的源文件。此变量仅在找到的 OpenSSL 版本至少为 0.9.8 且平台为 MSVC 时定义。

提示¶

此模块接受以下变量来控制搜索行为：

OPENSSL_ROOT_DIR

设置为 OpenSSL 安装的根目录，以便在自定义位置搜索 OpenSSL 库。

OPENSSL_USE_STATIC_LIBS

3.4 版本新增。

设置为 TRUE 以优先使用静态 OpenSSL 库而非共享库。

OPENSSL_MSVC_STATIC_RT

3.5 版本新增。

设置为 TRUE 以搜索使用 MSVC 静态运行时 (MT) 构建的 OpenSSL 库。

ENV{PKG_CONFIG_PATH}

在类 UNIX 系统上，使用 pkg-config 来定位 OpenSSL。设置 PKG_CONFIG_PATH 环境变量以指定替代位置，这在安装了多个库的系统上非常有用。

已弃用变量¶

以下变量提供用于向后兼容性

OPENSSL_FOUND

4.2 版本弃用： 请使用 OpenSSL_FOUND，其值相同。

布尔值，指示是否找到了（所请求版本的）OpenSSL 库。

OPENSSL_VERSION

4.2 版本弃用： 已被 OpenSSL_VERSION 取代。

找到的 OpenSSL 版本。

示例¶

查找 OpenSSL crypto 库并将其链接到项目目标

find_package(OpenSSL)
target_link_libraries(project_target PRIVATE OpenSSL::Crypto)

代码说明

  • find_package(OpenSSL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE OpenSSL::Crypto)：调用 target_link_libraries() 命令，为目标链接依赖库。

以下示例展示了如何查找 OpenSSL crypto 和 ssl 库并将它们链接到项目目标。显式指定了 SSL 组件，以确保如果未找到 ssl 库，查找模块会报错。

find_package(OpenSSL COMPONENTS SSL)
target_link_libraries(project_target PRIVATE OpenSSL::SSL)

代码说明

  • find_package(OpenSSL COMPONENTS SSL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE OpenSSL::SSL)：调用 target_link_libraries() 命令，为目标链接依赖库。
