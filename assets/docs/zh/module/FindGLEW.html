FindGLEW¶

查找 OpenGL（开放图形库） 扩展包库 (GLEW)

find_package(GLEW [<version>] [...])

代码说明

  • find_package(GLEW [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

GLEW 是一个跨平台的 C/C++ 库，它通过提供高效的运行时机制来查询和加载核心规范之外的 OpenGL（开放图形库） 功能，从而帮助管理 OpenGL（开放图形库） 扩展。

3.7 版本新增： 调试和发布库变体分别查找。

3.15 版本新增: 如果 GLEW 是使用其基于 CMake 的构建系统构建的，它会提供一个 CMake 软件包配置文件 (GLEWConfig.cmake)。本模块现已将其纳入考量，并首先尝试以配置模式 (config mode) 查找 GLEW。如果配置文件不可用，它将回退到模块模式 (module mode) 并搜索标准路径。

导入的目标¶

此模块提供以下 导入目标

GLEW::GLEW

版本 3.1 中新增。

封装了 GLEW 使用需求的主导入目标，在找到 GLEW 时可用。它根据可用性映射到 GLEW::glew 或 GLEW::glew_s 目标的使用需求。

GLEW::glew

版本 3.15 新增。

封装共享 GLEW 库使用需求的目标。如果找到 GLEW 且未通过 GLEW_USE_STATIC_LIBS 提示变量（见下文）请求静态库，则此目标可用。

GLEW::glew_s

版本 3.15 新增。

封装静态 GLEW 库使用需求的目标。如果找到 GLEW 且 GLEW_USE_STATIC_LIBS 提示变量设置为布尔值 true，则此目标可用。

结果变量¶

此模块定义了以下变量

GLEW_FOUND

布尔值，指示是否找到了 GLEW（及其请求的版本）。

GLEW_VERSION

版本 3.15 新增。

找到的 GLEW 版本。

GLEW_VERSION_MAJOR

版本 3.15 新增。

找到的 GLEW 主版本号。

GLEW_VERSION_MINOR

版本 3.15 新增。

找到的 GLEW 次版本号。

GLEW_VERSION_MICRO

版本 3.15 新增。

找到的 GLEW 微版本号。

GLEW_INCLUDE_DIRS

使用 GLEW 库所需的包含目录。

GLEW_LIBRARIES

使用 GLEW 库进行链接所需的库文件（根据配置的不同为共享库或静态库）。

GLEW_SHARED_LIBRARIES

版本 3.15 新增。

使用共享 GLEW 库进行链接所需的库文件。

GLEW_STATIC_LIBRARIES

版本 3.15 新增。

使用静态 GLEW 库进行链接所需的库文件。

提示¶

在调用 find_package(GLEW) 之前，本模块接受以下变量来影响其行为

GLEW_USE_STATIC_LIBS

版本 3.15 新增。

设置为布尔值 true 以查找静态 GLEW 库，并创建用于静态链接的 GLEW::glew_s 导入目标。

GLEW_VERBOSE

版本 3.15 新增。

设置为布尔值 true 以输出本模块的详细日志。例如，可用于调试。

示例¶

查找 GLEW 并将其链接到项目目标

find_package(GLEW)
target_link_libraries(project_target PRIVATE GLEW::GLEW)

代码说明

  • find_package(GLEW) target_link_libraries(project_target PRIVATE GLEW::GLEW)：调用 `find_package()` 命令，查找并加载外部依赖包。

如果找到，则使用静态 GLEW 库

set(GLEW_USE_STATIC_LIBS TRUE)
find_package(GLEW)
target_link_libraries(project_target PRIVATE GLEW::GLEW)

代码说明

  • set(GLEW_USE_STATIC_LIBS TRUE) find_package(GLEW) target_link_libraries(project_target PRIVATE GLEW::GLEW)：调用 `set()` 命令，把变量 `GLEW_USE_STATIC_LIBS` 设为给定的值。
