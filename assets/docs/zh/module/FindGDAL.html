FindGDAL¶

自版本 4.0 起已弃用：GDAL 3.5 及更高版本提供一个 GDALConfig.cmake 包配置文件。调用 find_package(GDAL CONFIG) 直接查找它，并避免使用此查找模块。有关更多详细信息，请参阅 GDAL 关于 CMake 集成的文档。

查找地理空间数据抽象库 (GDAL)

find_package(GDAL [<version>] [...])

导入的目标¶

此模块提供以下 导入目标

GDAL::GDAL

3.14 版新增。

封装 GDAL 使用需求的 target，仅在找到 GDAL 时可用。

结果变量¶

此模块定义了以下变量

GDAL_FOUND

布尔值，指示是否找到（请求版本的）GDAL。

GDAL_VERSION

3.14 版新增。

找到的 GDAL 版本。

GDAL_INCLUDE_DIRS

GDAL 头文件的包含目录。

GDAL_LIBRARIES

要链接到 GDAL 的库。

缓存变量¶

以下缓存变量也可以设置

GDAL_INCLUDE_DIR

包含 <gdal.h> 的目录。

GDAL_LIBRARY

libgdal 库文件。

提示¶

以下变量可以设置以修改搜索策略

GDAL_DIR 或 GDAL_ROOT

设置其中一个环境变量以指定 GDAL 安装前缀。

FindGDAL_SKIP_GDAL_CONFIG

如果设置，将不会使用 gdal-config。如果环境中同时存在使用 autotools 构建的 GDAL 库（提供该工具）和使用 CMake 构建的 GDAL 库（不提供该工具），则此设置很有用。

GDAL_ADDITIONAL_LIBRARY_VERSIONS

要搜索的额外库名称版本。

示例¶

在配置模式下不使用此模块查找 GDAL 并将其导入的 target 链接到项目 target

find_package(GDAL CONFIG)
target_link_libraries(example PRIVATE GDAL::GDAL)
