FindFLEX¶

查找 Fast Lexical Analyzer (Flex) 命令行生成器及其库，并提供用于使用 Flex 创建自定义构建规则的 CMake 命令

find_package(FLEX [<version>] [...])

代码说明

  • find_package(FLEX [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

Flex 用于生成词法分析器，也称为扫描器 (scanners) 或 词法分析器 (lexers)。它还包含一个运行时库 (fl)，为生成的扫描器提供支持函数，例如输入处理、缓冲区管理和错误报告。

结果变量¶

此模块定义了以下变量

FLEX_FOUND

布尔值，指示是否找到了（所请求版本的）Flex。

FLEX_VERSION

找到的 Flex 版本。

FLEX_INCLUDE_DIRS

包含使用 Flex 库所需头文件的包含目录。

FLEX_LIBRARIES

使用 Flex 库时需要链接的库。

缓存变量¶

以下缓存变量也可以设置

FLEX_EXECUTABLE

flex 可执行文件的路径。

命令¶

如果找到 flex，则此模块提供以下命令

生成扫描器¶

flex_target¶

创建一个自定义构建规则，使用 Flex 从 lex 文件生成扫描器文件

flex_target(
  <name>
  <input-lex-file>
  <output-scanner-file>
  [DEFINES_FILE <header>]
  [OPTIONS <options>...]
  [COMPILE_FLAGS <string>] # Deprecated
)

代码说明

  • flex_target( <name> <input-lex-file> <output-scanner-file> [DEFINES_FILE <header>] [OPTIONS <options>...] [COMPILE_FLAGS <string>] # Deprecated )：调用 `flex_target()` 命令，具体行为由这一行传入的参数决定。

3.17 版本中变更: 当策略 CMP0098 设置为 NEW 时，flex 将在 CMAKE_CURRENT_BINARY_DIR 目录中运行。

<name>

用作此命令调用的标识符的字符串。

<input-lex-file>

输入 Flex 源文件 (.l) 的路径。如果作为相对路径给出，它将被解释为相对于当前源目录 (CMAKE_CURRENT_SOURCE_DIR)。

<output-scanner-file>

由 Flex 生成的输出文件的路径。如果作为相对路径给出，它将被解释为相对于当前 Flex 工作目录。

DEFINES_FILE <header>

3.5 版本新增。

如果 Flex 被配置为输出头文件，此选项可用于指定其名称。如果作为相对路径给出，它将被解释为相对于当前 Flex 工作目录。

OPTIONS <options>...

4.0 版本新增。

添加到 flex 命令行的额外选项的 分号分隔列表。

COMPILE_FLAGS <string>

4.0 版本弃用: 被 OPTIONS <options>... 取代。

添加到 flex 命令行的由空格分隔的额外选项字符串。使用 分号分隔列表 将不起作用。

命令变量

此命令还定义了以下变量

FLEX_<name>_DEFINED

布尔值，指示此命令是否成功调用。

FLEX_<name>_INPUT

Flex 源文件，即 <input-lex-file> 的别名。

FLEX_<name>_OUTPUT_HEADER

3.5 版本新增。

由 flex 生成的头文件（如果有）。

FLEX_<name>_OUTPUTS

由 flex 生成的文件列表，包括输出的扫描器文件和头文件。

FLEX_<name>_OPTIONS

4.0 版本新增。

flex 命令所使用的命令行选项列表。

在扫描器和解析器之间添加依赖项¶

add_flex_bison_dependency¶

添加扫描器和解析器之间所需的依赖项

add_flex_bison_dependency(<flex-name> <bison-name>)

代码说明

  • add_flex_bison_dependency(<flex-name> <bison-name>)：调用 `add_flex_bison_dependency()` 命令，具体行为由这一行传入的参数决定。

Flex 扫描器通常依赖于 Bison 生成的记号定义，这意味着 Flex 产生的代码依赖于 Bison 创建的头文件。

此命令添加扫描器和解析器之间所需的依赖关系，其中 <flex-name> 和 <bison-name> 分别是 flex_target(<name> ...) 和 bison_target(<name> ...) 命令的第一个参数。

查找 Flex

find_package(FLEX)

代码说明

  • find_package(FLEX)：调用 `find_package()` 命令，查找并加载外部依赖包。

代码说明

  • find_package(FLEX 2.5.13)：调用 `find_package()` 命令，查找并加载外部依赖包。

查找 Flex 并使其成为必需（如果未找到 Flex，处理将停止并报错）

代码说明

示例：生成扫描器¶

查找 Flex，并将当前源目录中的 lex 源文件生成的扫描器源文件放置在当前二进制目录中

find_package(FLEX)

if(FLEX_FOUND)
  flex_target(MyScanner lexer.l lexer.cpp)
endif()

add_executable(foo foo.cc ${FLEX_MyScanner_OUTPUTS})

代码说明

  • find_package(FLEX) if(FLEX_FOUND) flex_target(MyScanner lexer.l lexer.cpp) endif() add_executable(foo foo.cc ${FLEX_MyScanner_OUTPUTS})：调用 `find_package()` 命令，查找并加载外部依赖包。

示例：命令行选项¶

添加到 flex 可执行文件的额外命令行选项可以作为列表传递。例如，添加 --warn 选项以报告警告，以及添加 --noline (-L) 以不生成 #line 指令。

find_package(FLEX)

if(FLEX_FOUND)
  flex_target(MyScanner lexer.l lexer.cpp OPTIONS --warn --noline)
endif()

代码说明

  • find_package(FLEX) if(FLEX_FOUND) flex_target(MyScanner lexer.l lexer.cpp OPTIONS --warn --noline) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。

生成器表达式 可以用于 OPTIONS <options>... 参数中。例如，仅对 Debug 构建类型添加 --debug (-d) 选项

find_package(FLEX)

if(FLEX_FOUND)
  flex_target(MyScanner lexer.l lexer.cpp OPTIONS $<$<CONFIG:Debug>:--debug>)
endif()

代码说明

  • find_package(FLEX) if(FLEX_FOUND) flex_target(MyScanner lexer.l lexer.cpp OPTIONS $<$<CONFIG:Debug>:--debug>) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。

示例：使用 Flex 库¶

查找 Flex 并创建一个 导入的目标 接口，该接口封装了其链接到项目目标时的库使用要求

find_package(FLEX)

if(FLEX_FOUND AND NOT TARGET FLEX::fl)
  add_library(FLEX::fl INTERFACE IMPORTED)
  set_target_properties(
    FLEX::fl
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${FLEX_INCLUDE_DIRS}"
      INTERFACE_LINK_LIBRARIES "${FLEX_LIBRARIES}"
  )
endif()

if(FLEX_FOUND)
  flex_target(MyScanner lexer.l lexer.cpp)
endif()

add_executable(Foo foo.cc ${FLEX_MyScanner_OUTPUTS})
target_link_libraries(Foo PRIVATE FLEX::fl)

代码说明

  • find_package(FLEX) if(FLEX_FOUND AND NOT TARGET FLEX::fl) add_library(FLEX::fl INTERFACE IMPORTED) set_target_properties( FLEX::fl PROPERTIES INTERFACE_INCLUDE_DIRECTORIES "${FLEX_INCLUDE_DIRS}" INTERFACE_LINK_LIBRARIES "${FLEX_LIBRARIES}" ) endif() if(FLEX_FOUND) flex_target(MyScanner lexer.l lexer.cpp) endif() add_executable(Foo foo.cc ${FLEX_MyScanner_OUTPUTS}) target_link_libraries(Foo PRIVATE FLEX::fl)：调用 `find_package()` 命令，查找并加载外部依赖包。

示例：使用 Flex 和 Bison¶

以下示例演示了如何在 CMake 中使用 Flex 和 Bison

find_package(BISON)
find_package(FLEX)

if(BISON_FOUND AND FLEX_FOUND)
  bison_target(MyParser parser.y parser.cpp)
  flex_target(MyScanner lexer.l lexer.cpp)
  add_flex_bison_dependency(MyScanner MyParser)
endif()

add_executable(Foo foo.cc ${BISON_MyParser_OUTPUTS} ${FLEX_MyScanner_OUTPUTS})

# ...

代码说明

  • find_package(BISON) find_package(FLEX) if(BISON_FOUND AND FLEX_FOUND) bison_target(MyParser parser.y parser.cpp) flex_target(MyScanner lexer.l lexer.cpp) add_flex_bison_dependency(MyScanner MyParser) endif() add_executable(Foo foo.cc ${BISON_MyParser_OUTPUTS} ${FLEX_MyScanner_OUTPUTS}) # ...：这一行继续补充当前命令或表达式的内容。
