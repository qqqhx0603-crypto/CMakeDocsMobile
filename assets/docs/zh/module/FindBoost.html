FindBoost¶

3.30 版本变更: 仅当策略 CMP0167 未设置为 NEW 时，此模块才可用。请将项目迁移到上游 Boost 的 BoostConfig.cmake 软件包配置文件，find_package(Boost) 现在会搜索该文件。

查找 Boost 头文件和库

通过调用 find_package() 并使用以下格式来使用此模块：

find_package(Boost
  [version] [EXACT]      # Minimum or EXACT version e.g. 1.67.0
  [REQUIRED]             # Fail with error if Boost is not found
  [COMPONENTS <libs>...] # Boost libraries by their canonical name
                         # e.g. "date_time" for "libboost_date_time"
  [OPTIONAL_COMPONENTS <libs>...]
                         # Optional Boost libraries by their canonical name)
  )                      # e.g. "date_time" for "libboost_date_time"

  • [version] [EXACT] # Minimum or EXACT version e.g. 1.67.0：这一行继续补充当前命令或表达式的内容。

  • [REQUIRED] # Fail with error if Boost is not found：这一行继续补充当前命令或表达式的内容。

  • [COMPONENTS <libs>...] # Boost libraries by their canonical name：这一行继续补充当前命令或表达式的内容。

此模块用于查找头文件和请求的组件库，或者查找由“Boost CMake”构建提供的 CMake 软件包配置文件。对于后者，请直接跳至下方的 Boost CMake 部分。

3.7 版本新增: bzip2 和 zlib 组件（仅限 Windows（微软桌面系统））。

3.11 版本新增: OPTIONAL_COMPONENTS 选项。

3.13 版本新增: stacktrace_* 组件。

3.19 版本新增: 所有平台上的 bzip2 和 zlib 组件。

结果变量¶

此模块定义了以下变量

Boost_FOUND

布尔值，指示是否找到了头文件和所请求的库。

Boost_INCLUDE_DIRS

Boost 头文件目录。

Boost_LIBRARY_DIRS

Boost 库的链接目录。

Boost_LIBRARIES

需要链接的 Boost 组件库。

Boost_<COMPONENT>_FOUND

布尔值，指示是否找到了组件 <COMPONENT>（<COMPONENT> 名称为大写）。

Boost_<COMPONENT>_LIBRARY

要为组件 <COMPONENT> 链接的库（可能包含 target_link_libraries() 的 debug/optimized 关键字）。

Boost_VERSION_MACRO

来自 boost/version.hpp 的 BOOST_VERSION 值。

Boost_VERSION_STRING

X.Y.Z 格式的 Boost 版本号。

Boost_VERSION

X.Y.Z 格式的 Boost 版本号（与 Boost_VERSION_STRING 相同）。

3.15 版本变更: 在旧版本 CMake 中，此变量使用来自 Boost 头文件的原始版本字符串（与 Boost_VERSION_MACRO 相同）。请参阅策略 CMP0093。

Boost_LIB_VERSION

附加到库文件名的版本字符串。

Boost_VERSION_MAJOR, Boost_MAJOR_VERSION

Boost 主版本号（X.Y.Z 中的 X）。

Boost_VERSION_MINOR, Boost_MINOR_VERSION

Boost 次版本号（X.Y.Z 中的 Y）。

Boost_VERSION_PATCH, Boost_SUBMINOR_VERSION

Boost 修订版本号（X.Y.Z 中的 Z）。

Boost_VERSION_COUNT

版本组件数量 (3)。

Boost_LIB_DIAGNOSTIC_DEFINITIONS (Windows（微软桌面系统） 专用)

传递给 add_definitions() 以便在编译期间显示有关 Boost 自动链接的诊断信息。

3.15 版本新增: Boost_VERSION_<PART> 变量。

缓存变量¶

搜索结果将持久保存在 CMake 缓存条目中

Boost_INCLUDE_DIR

包含 Boost 头文件的目录。

Boost_LIBRARY_DIR_RELEASE

包含 Boost Release 版库的目录。

Boost_LIBRARY_DIR_DEBUG

包含 Boost Debug 版库的目录。

Boost_<COMPONENT>_LIBRARY_DEBUG

组件 <COMPONENT> 的 Debug 版库变体。

Boost_<COMPONENT>_LIBRARY_RELEASE

组件 <COMPONENT> 的 Release 版库变体。

3.3 版本新增: 各配置变量 Boost_LIBRARY_DIR_RELEASE 和 Boost_LIBRARY_DIR_DEBUG。

提示¶

此模块从以下变量中读取有关搜索位置的提示

BOOST_ROOT, BOOSTROOT

首选安装前缀。

BOOST_INCLUDEDIR

首选头文件目录，例如 <prefix>/include。

BOOST_LIBRARYDIR

首选库目录，例如 <prefix>/lib。

Boost_NO_SYSTEM_PATHS

设置为 ON 可禁止在非这些提示变量指定的位置进行搜索。默认值为 OFF。

Boost_ADDITIONAL_VERSIONS

此模块未知但希望检查的 Boost 版本列表。（Boost 安装位置可能包含版本号）。

用户可以将这些提示或结果设置为 CACHE 条目。项目不应直接读取这些条目，而应使用上述结果变量。请注意，某些提示名称以大写 BOOST 开头。如果未将它们指定为 CMake 变量或缓存条目，则可以将它们指定为环境变量。

此模块首先使用上述提示变量（不包括 BOOST_LIBRARYDIR）搜索 Boost 头文件，并将结果保存在 Boost_INCLUDE_DIR 中。然后，它使用上述提示（不包括 BOOST_INCLUDEDIR 和 Boost_ADDITIONAL_VERSIONS）、Boost_INCLUDE_DIR 附近的 "lib" 目录以及下方的库名称配置设置来搜索请求的组件库。它将库目录保存在 Boost_LIBRARY_DIR_DEBUG 和 Boost_LIBRARY_DIR_RELEASE 中，并将各个库的位置保存在 Boost_<COMPONENT>_LIBRARY_DEBUG 和 Boost_<COMPONENT>_LIBRARY_RELEASE 中。当用户更改同一构建树中之前搜索所使用的设置（不包括环境变量）时，此模块会丢弃受影响的旧搜索结果并重新搜索。

导入的目标¶

3.5 版本新增。

此模块提供以下 导入目标

Boost::boost

仅头文件依赖的目标。（Boost 头文件目录）。

Boost::headers

3.15 版本新增: Boost::boost 的别名。

Boost::<component>

特定组件依赖的目标（共享或静态库）；<component> 名称为小写。

Boost::diagnostic_definitions

接口目标，用于在编译期间启用有关 Boost 自动链接的诊断信息（添加 -DBOOST_LIB_DIAGNOSTIC）。

Boost::disable_autolinking

接口目标，用于禁用 MSVC 的自动链接（添加 -DBOOST_ALL_NO_LIB）。

Boost::dynamic_linking

接口目标，用于启用 MSVC 的动态链接（添加 -DBOOST_ALL_DYN_LINK）。

隐式依赖项（例如 Boost::filesystem 需要 Boost::system）将被自动检测并满足，即使在使用 find_package() 时未指定 system，或者如果未将 Boost::system 添加到 target_link_libraries() 中。如果使用 Boost::thread，则会自动添加 Threads::Threads。

需要注意的是，导入的目标与此模块创建的变量表现不同：在同一目录或子目录中，使用不同选项（例如静态或共享）多次调用 find_package(Boost) 不会覆盖第一次调用所创建目标的值。

其他变量¶

Boost 库有多种变体，并在其文件名中进行编码。用户或项目可以通过设置以下变量来告知此模块要查找哪些变体：

Boost_USE_DEBUG_LIBS

3.10 版本新增。

设置为 ON 或 OFF 以指定是否搜索并使用 Debug 版库。默认值为 ON。

Boost_USE_RELEASE_LIBS

3.10 版本新增。

设置为 ON 或 OFF 以指定是否搜索并使用 Release 版库。默认值为 ON。

Boost_USE_MULTITHREADED

设置为 OFF 以使用非多线程库（“mt”标签）。默认值为 ON。

Boost_USE_STATIC_LIBS

设置为 ON 以强制使用静态库。默认值为 OFF。

Boost_USE_STATIC_RUNTIME

设置为 ON 或 OFF 以指定是否使用静态链接到 C++ 运行时的库（“s”标签）。默认值取决于平台。

Boost_USE_DEBUG_RUNTIME

设置为 ON 或 OFF 以指定是否使用链接到 MS Debug C++ 运行时的库（“g”标签）。默认值为 ON。

Boost_USE_DEBUG_PYTHON

设置为 ON 以使用使用 Debug Python 构建编译的库（“y”标签）。默认值为 OFF。

Boost_USE_STLPORT

设置为 ON 以使用使用 STLPort 编译的库（“p”标签）。默认值为 OFF。

Boost_USE_STLPORT_DEPRECATED_NATIVE_IOSTREAMS

设置为 ON 以使用使用 STLPort 已弃用的 “native iostreams” 编译的库（“n”标签）。默认值为 OFF。

Boost_COMPILER

设置为特定于编译器的库后缀（例如 -gcc43）。默认值是为所使用的 C++ 编译器自动计算的。

3.9 版本变更: 如果需要测试多个兼容的后缀，可以使用列表，按优先级降序排列。

Boost_LIB_PREFIX

在 3.18 版本中新增。

设置为 Boost 静态库使用的平台特定库名称前缀（例如 lib）。仅在 CMake 默认不知道前缀的平台上才需要此设置。

Boost_ARCHITECTURE

3.13 版本新增。

设置为特定于体系结构的库后缀（例如 -x64）。默认值是为所使用的 C++ 编译器自动计算的。

Boost_THREADAPI

thread 组件库名称的后缀，例如 pthread 或 win32。将尝试带此后缀和不带此后缀的名称。

Boost_NAMESPACE

构建 boost 时使用的替代命名空间，例如，如果设置为 myboost，则会搜索 myboost_thread 而不是 boost_thread。

用户可以设置来控制此模块的其他变量包括：

Boost_DEBUG

设置为 ON 以启用 FindBoost 的调试输出。提交任何错误报告前请启用此项。

Boost_REALPATH

设置为 ON 以解析所发现库的符号链接，从而辅助打包。例如，"system" 组件库可能会解析为 /usr/lib/libboost_system.so.1.67.0 而不是 /usr/lib/libboost_system.so。这不会影响链接，除非用户需要此信息，否则不应启用。

Boost_LIBRARY_DIR

Boost_LIBRARY_DIR_RELEASE 和 Boost_LIBRARY_DIR_DEBUG 的默认值。

Boost_NO_WARN_NEW_VERSIONS

在 3.20 版本中添加。

设置为 ON 以抑制关于新 Boost 版本未知依赖项的警告。

在 Visual Studio（集成开发环境） 和 Borland 编译器上，Boost 头文件会请求自动链接到相应的库。这需要显式链接匹配的库或使其在链接库搜索路径中可用。在这种情况下，将 Boost_USE_STATIC_LIBS 设置为 OFF 可能无法实现动态链接。Boost 自动链接通常请求静态库，极少数情况除外（例如 Boost.Python）。使用

add_definitions(${Boost_LIB_DIAGNOSTIC_DEFINITIONS})

代码说明

  • add_definitions(${Boost_LIB_DIAGNOSTIC_DEFINITIONS})：调用 add_definitions() 命令，具体行为由这一行传入的参数决定；变量说明：Boost_LIB_DIAGNOSTIC_DEFINITIONS：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

来请求 Boost 报告有关自动链接请求的信息。

Boost CMake¶

如果 Boost 是使用 boost-cmake 项目构建的，或者从 Boost 1.70.0 版本开始，它提供了用于 find_package 配置模式的软件包配置文件。此模块查找名为 BoostConfig.cmake 或 boost-config.cmake 的软件包配置文件，并将结果存储在 CACHE 条目 Boost_DIR 中。如果找到，将加载该软件包配置文件，且此模块不再执行进一步操作。有关其提供内容的详细信息，请参阅 Boost CMake 软件包配置文档。

将 Boost_NO_BOOST_CMAKE 设置为 ON 可禁用对 boost-cmake 的搜索。
