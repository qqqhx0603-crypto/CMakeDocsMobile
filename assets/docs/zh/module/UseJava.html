UseJava¶

此文件提供对 Java 的支持。假定 FindJava 模块已被加载。有关如何将 Java 加载到您的 CMake 项目中的信息，请参阅 FindJava。

在 CMake 项目中加载此模块，使用

include(UseJava)

代码说明

  • include(UseJava)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

创建与安装 JAR¶

add_jar¶

创建一个包含 Java 对象以及可选资源的 jar 文件

add_jar(<target_name>
        [SOURCES] <source1> [<source2>...] [<resource1>...]
        [RESOURCES NAMESPACE <ns1> <resource1>... [NAMESPACE <nsX> <resourceX>...]... ]
        [INCLUDE_JARS <jar1> [<jar2>...]]
        [INCLUDE_MODULES <jar1> [<jar2>...]]
        [ENTRY_POINT <entry>]
        [VERSION <version>]
        [MANIFEST <manifest>]
        [OUTPUT_NAME <name>]
        [OUTPUT_DIR <dir>]
        [GENERATE_NATIVE_HEADERS <target>
                                 [DESTINATION (<dir>|INSTALL <dir> [BUILD <dir>])]]
        )

代码说明

  • add_jar(<target_name> [SOURCES] <source1> [<source2>...] [<resource1>...] [RESOURCES NAMESPACE <ns1> <resource1>... [NAMESPACE <nsX> <resourceX>...]... ] [INCLUDE_JARS <jar1> [<jar2>...]] [INCLUDE_MODULES <jar1> [<jar2>...]] [ENTRY_POINT <entry>] [VERSION <version>] [MANIFEST <manifest>] [OUTPUT_NAME <name>] [OUTPUT_DIR <dir>] [GENERATE_NATIVE_HEADERS <target> [DESTINATION (<dir>|INSTALL <dir> [BUILD <dir>])]] )：调用 `add_jar()` 命令，具体行为由这一行传入的参数决定。

此命令创建一个 <target_name>.jar。它会编译给定的 <source> 文件，并将给定的 <resource> 文件添加到该 jar 文件中。源文件可以是 Java 文件或列表文件（以 @ 为前缀）。如果仅提供了资源文件，则只创建一个 jar 文件。

SOURCES

编译指定的源文件并将结果添加到 jar 文件中。

3.4 版本新增：支持以 @ 为前缀的响应文件。

RESOURCES

3.21 版本新增。

将命名的 <resource> 文件添加到 jar 中，方法是剥离源文件路径并将其置于 jar 内的 <ns> 路径之下。

例如

RESOURCES NAMESPACE "/com/my/namespace" "a/path/to/resource.txt"

代码说明

  • RESOURCES NAMESPACE "/com/my/namespace" "a/path/to/resource.txt"：这一行给出示例文本或输出，用来说明：RESOURCES NAMESPACE "/com/my/namespace" "a/path/to/resource.txt"

最终产生一个可在 jar 内通过 /com/my/namespace/resource.txt 访问的资源。

可以通过将资源添加到 SOURCES 列表中来添加资源，而不调整其命名空间（这是原始行为），在这种情况下，资源路径必须相对于 CMAKE_CURRENT_SOURCE_DIR。在源码外构建（out of source builds）时，如果不使用 RESOURCES 参数而添加资源，几乎肯定会导致混淆。

注意

通过 SOURCES 参数添加资源依赖于一个硬编码的文件扩展名列表，这些扩展名会被测试以确定它们是否需要编译（例如 File.java）。符合扩展名的 SOURCES 文件会被编译。不符合的文件则被视为资源。要包含符合这些文件扩展名但不需要编译的资源，请使用 RESOURCES 参数。

INCLUDE_JARS

在编译 Java 源文件时，列表中的 jar 文件会被添加到类路径（classpath）中，同时也会被添加到目标的依赖项中。INCLUDE_JARS 也接受由 add_jar() 创建的其他目标名称。为了向后兼容，作为源文件列出的 jar 文件会被忽略（自此模块的第一个版本以来一直如此）。

INCLUDE_MODULES

4.3 版本新增。

在构建 Java 源文件时，列表中的 jar 文件会被添加到模块路径（module path）中，同时也会被添加到目标的依赖项中。INCLUDE_MODULES 也接受由 add_jar() 创建的其他目标名称。

ENTRY_POINT

定义 jar 文件中的入口点。

VERSION

向目标输出名称添加版本号。

以下示例将创建一个名为 shibboleet-1.2.0.jar 的 jar 文件，并创建一个指向带有版本信息 jar 的符号链接 shibboleet.jar。

add_jar(shibboleet shibbotleet.java VERSION 1.2.0)

代码说明

  • add_jar(shibboleet shibbotleet.java VERSION 1.2.0)：调用 `add_jar()` 命令，具体行为由这一行传入的参数决定。

MANIFEST

为 jar 定义自定义清单文件（manifest）。

OUTPUT_NAME

为目标指定不同的输出名称。

OUTPUT_DIR

设置生成 jar 文件的目录。如果未指定，则使用 CMAKE_CURRENT_BINARY_DIR 作为输出目录。

GENERATE_NATIVE_HEADERS

3.11 版本新增。

为声明为 native 的方法生成原生头文件。这些文件提供了允许 Java 和 C 代码交互的连接胶水。系统将创建一个 INTERFACE 目标以便于使用这些生成的文件。子选项 DESTINATION 可用于指定生成头文件的输出目录。

此选项至少需要 JDK 1.8 版本。

为了优化使用此选项，建议在调用 add_jar() 之前包含 JNI 模块。之后，生成的原生头文件目标可用于使用 target_link_libraries() 命令编译 C/C++ 源文件。

find_package(JNI)
add_jar(foo foo.java GENERATE_NATIVE_HEADERS foo-native)
add_library(bar bar.cpp)
target_link_libraries(bar PRIVATE foo-native)

代码说明

  • find_package(JNI) add_jar(foo foo.java GENERATE_NATIVE_HEADERS foo-native) add_library(bar bar.cpp) target_link_libraries(bar PRIVATE foo-native)：调用 `find_package()` 命令，查找并加载外部依赖包。

3.20 版本新增：DESTINATION 子选项现在支持为 BUILD 和 INSTALL 步骤指定不同的输出目录。如果未指定 BUILD 目录，将使用默认目录。

要导出由 GENERATE_NATIVE_HEADERS 选项生成出的接口目标，需要使用 DESTINATION 的 INSTALL 子选项。

add_jar(foo foo.java GENERATE_NATIVE_HEADERS foo-native
                     DESTINATION INSTALL include)
install(TARGETS foo-native EXPORT native)
install(DIRECTORY "$<TARGET_PROPERTY:foo-native,NATIVE_HEADERS_DIRECTORY>/"
        DESTINATION include)
install(EXPORT native DESTINATION /to/export NAMESPACE foo)

代码说明

  • add_jar(foo foo.java GENERATE_NATIVE_HEADERS foo-native DESTINATION INSTALL include) install(TARGETS foo-native EXPORT native) install(DIRECTORY "$<TARGET_PROPERTY:foo-native,NATIVE_HEADERS_DIRECTORY>/" DESTINATION include) install(EXPORT native DESTINATION /to/export NAMESPACE foo)：调用 `add_jar()` 命令，具体行为由这一行传入的参数决定。

可以设置一些变量来自定义 add_jar() 的行为以及 Java 编译器。

CMAKE_JAVA_COMPILE_FLAGS

指定 Java 编译器的额外标志。

CMAKE_JAVA_INCLUDE_PATH

指定类路径的额外路径。

CMAKE_JNI_TARGET

如果目标是 JNI 库，请将此布尔变量设置为 TRUE 以启用 JNI 符号链接的创建（另请参阅 install_jni_symlink()）。

CMAKE_JAR_CLASSES_PREFIX

如果应从同一个 Java 源文件树生成多个 jar 文件，为防止后续 jar 文件中出现重复类文件的堆积，请在调用 add_jar() 之前设置/重置 CMAKE_JAR_CLASSES_PREFIX。

set(CMAKE_JAR_CLASSES_PREFIX com/redhat/foo)
add_jar(foo foo.java)

set(CMAKE_JAR_CLASSES_PREFIX com/redhat/bar)
add_jar(bar bar.java)

代码说明

  • set(CMAKE_JAR_CLASSES_PREFIX com/redhat/foo) add_jar(foo foo.java) set(CMAKE_JAR_CLASSES_PREFIX com/redhat/bar) add_jar(bar bar.java)：调用 `set()` 命令，把变量 `CMAKE_JAR_CLASSES_PREFIX` 设为给定的值。

add_jar() 函数在 <target_name> 上设置以下目标属性：

INSTALL_FILES

应安装的文件。由 install_jar() 使用。

JNI_SYMLINK

应安装的 JNI 符号链接。由 install_jni_symlink() 使用。

JAR_FILE

jar 文件的位置，以便您可以包含它。

CLASSDIR

可以找到类文件的目录。例如，用于配合 javah 使用。

NATIVE_HEADERS_DIRECTORY

在 3.20 版本中添加。

原生头文件的生成目录。当指定 GENERATE_NATIVE_HEADERS 选项时定义。

install_jar¶

此命令将 jar 文件安装到给定的目标位置。

install_jar(<target_name> <destination>)
install_jar(<target_name> DESTINATION <destination> [COMPONENT <component>])

代码说明

  • install_jar(<target_name> <destination>) install_jar(<target_name> DESTINATION <destination> [COMPONENT <component>])：调用 `install_jar()` 命令，具体行为由这一行传入的参数决定。

COMPONENT

install_jar() 命令在 <target_name> 上设置以下目标属性：

INSTALL_DESTINATION

头文件生成¶

create_javah¶

3.4 版本新增。

为 Java 类生成 C 头文件。

create_javah(TARGET <target> | GENERATED_FILES <VAR>
             CLASSES <class>...
             [CLASSPATH <classpath>...]
             [DEPENDS <depend>...]
             [OUTPUT_NAME <path>|OUTPUT_DIR <path>]
             )

代码说明

  • create_javah(TARGET <target> | GENERATED_FILES <VAR> CLASSES <class>... [CLASSPATH <classpath>...] [DEPENDS <depend>...] [OUTPUT_NAME <path>|OUTPUT_DIR <path>] )：调用 `create_javah()` 命令，具体行为由这一行传入的参数决定。

3.11 版本弃用：由于 javah 工具的废除，从 JDK 10 开始将不再支持此命令。应改用 add_jar(GENERATE_NATIVE_HEADERS) 命令。

从 Java 类创建 C 头文件。这些文件提供了允许 Java 和 C 代码交互的连接胶水。

create_javah() 有两个主要签名。第一个签名通过 GENERATED_FILES 选项指定的变量返回生成的文件。例如：

create_javah(GENERATED_FILES files_headers
  CLASSES org.cmake.HelloWorld
  CLASSPATH hello.jar
)

代码说明

  • create_javah(GENERATED_FILES files_headers CLASSES org.cmake.HelloWorld CLASSPATH hello.jar )：调用 `create_javah()` 命令，具体行为由这一行传入的参数决定。

create_javah() 的第二个签名创建一个封装了头文件生成的目标。例如：

create_javah(TARGET target_headers
  CLASSES org.cmake.HelloWorld
  CLASSPATH hello.jar
)

代码说明

  • create_javah(TARGET target_headers CLASSES org.cmake.HelloWorld CLASSPATH hello.jar )：调用 `create_javah()` 命令，具体行为由这一行传入的参数决定。

两个签名共享相同的选项。

CLASSES

指定用于生成头文件的 Java 类。

CLASSPATH

指定查找类的各种路径。此处可以使用 .class 文件、jar 文件或由 add_jar 命令创建的目标。

DEPENDS

javah 目标所依赖的目标。

OUTPUT_NAME

将选项 CLASSES 列出的所有类的结果头文件连接到 <path> 中。行为与 javah 工具的 -o 选项相同。

OUTPUT_DIR

设置生成头文件的目录。行为与 javah 工具的 -d 选项相同。如果未指定，则使用 CMAKE_CURRENT_BINARY_DIR 作为输出目录。

查找 JARs¶

find_jar¶

查找指定的 jar 文件。

find_jar(<VAR>
         <name> | NAMES <name1> [<name2>...]
         [PATHS <path1> [<path2>... ENV <var>]]
         [VERSIONS <version1> [<version2>]]
         [DOC "cache documentation string"]
        )

代码说明

  • find_jar(<VAR> <name> | NAMES <name1> [<name2>...] [PATHS <path1> [<path2>... ENV <var>]] [VERSIONS <version1> [<version2>]] [DOC "cache documentation string"] )：调用 `find_jar()` 命令，具体行为由这一行传入的参数决定。

此命令用于查找指定 jar 的完整路径。创建一个以 <VAR> 命名的缓存条目来存储此命令的结果。如果找到 jar 的完整路径，结果将存储在变量中，除非清除该变量，否则不会重复搜索。如果未找到，结果将为 <VAR>-NOTFOUND，且下次使用相同变量调用 find_jar() 时将再次尝试搜索。

NAMES

指定一个或多个 jar 文件的可能名称。

PATHS

指定除默认位置外还要搜索的目录。ENV var 子选项从系统环境变量中读取路径。

VERSIONS

指定 jar 版本。

DOC

为 <VAR> 缓存条目指定文档字符串。

创建 Java 文档¶

create_javadoc¶

基于文件和包创建 Java 文档。

create_javadoc(<VAR>
               (PACKAGES <pkg1> [<pkg2>...] | FILES <file1> [<file2>...])
               [SOURCEPATH <sourcepath>]
               [CLASSPATH <classpath>]
               [INSTALLPATH <install path>]
               [DOCTITLE <the documentation title>]
               [WINDOWTITLE <the title of the document>]
               [AUTHOR (TRUE|FALSE)]
               [USE (TRUE|FALSE)]
               [VERSION (TRUE|FALSE)]
               )

代码说明

  • create_javadoc(<VAR> (PACKAGES <pkg1> [<pkg2>...] | FILES <file1> [<file2>...]) [SOURCEPATH <sourcepath>] [CLASSPATH <classpath>] [INSTALLPATH <install path>] [DOCTITLE <the documentation title>] [WINDOWTITLE <the title of the document>] [AUTHOR (TRUE|FALSE)] [USE (TRUE|FALSE)] [VERSION (TRUE|FALSE)] )：调用 `create_javadoc()` 命令，具体行为由这一行传入的参数决定。

create_javadoc() 命令可用于创建 Java 文档。create_javadoc() 有两个主要签名。

第一个签名使用带有源文件的路径上的包名。

create_javadoc(my_example_doc
               PACKAGES com.example.foo com.example.bar
               SOURCEPATH "${CMAKE_CURRENT_SOURCE_DIR}"
               CLASSPATH ${CMAKE_JAVA_INCLUDE_PATH}
               WINDOWTITLE "My example"
               DOCTITLE "<h1>My example</h1>"
               AUTHOR TRUE
               USE TRUE
               VERSION TRUE
              )

代码说明

  • create_javadoc(my_example_doc PACKAGES com.example.foo com.example.bar SOURCEPATH "${CMAKE_CURRENT_SOURCE_DIR}" CLASSPATH ${CMAKE_JAVA_INCLUDE_PATH} WINDOWTITLE "My example" DOCTITLE "<h1>My example</h1>" AUTHOR TRUE USE TRUE VERSION TRUE )：调用 `create_javadoc()` 命令，具体行为由这一行传入的参数决定。

create_javadoc() 的第二个签名作用于给定的文件列表。

create_javadoc(my_example_doc
               FILES java/A.java java/B.java
               CLASSPATH ${CMAKE_JAVA_INCLUDE_PATH}
               WINDOWTITLE "My example"
               DOCTITLE "<h1>My example</h1>"
               AUTHOR TRUE
               USE TRUE
               VERSION TRUE
              )

代码说明

  • create_javadoc(my_example_doc FILES java/A.java java/B.java CLASSPATH ${CMAKE_JAVA_INCLUDE_PATH} WINDOWTITLE "My example" DOCTITLE "<h1>My example</h1>" AUTHOR TRUE USE TRUE VERSION TRUE )：调用 `create_javadoc()` 命令，具体行为由这一行传入的参数决定。

两个签名共享大部分选项。有关详细信息，请阅读 javadoc 手册页。

PACKAGES

指定 Java 包。

FILES

指定 Java 源文件。如果指定了相对路径，它们将相对于 CMAKE_CURRENT_SOURCE_DIR。

SOURCEPATH

指定查找包的目录。默认使用 CMAKE_CURRENT_SOURCE_DIR 目录。

CLASSPATH

指定查找用户类文件的位置。行为与 javadoc 工具的 -classpath 选项相同。

INSTALLPATH

指定安装 Java 文档的位置。如果指定了，文档将被安装到 ${CMAKE_INSTALL_PREFIX}/share/javadoc/<VAR>。

DOCTITLE

指定放置在概览摘要文件顶部附近的标题。行为与 javadoc 工具的 -doctitle 选项相同。

WINDOWTITLE

指定放置在 HTML <title> 标签中的标题。行为与 javadoc 工具的 -windowtitle 选项相同。

AUTHOR

当指定值 TRUE 时，在生成的文档中包含 @author 文本。行为与 javadoc 工具的 -author 选项相同。

USE

当指定值 TRUE 时，创建类和包的使用页面。为每个记录的类和包包含一个 Use 页面。行为与 javadoc 工具的 -use 选项相同。

VERSION

当指定值 TRUE 时，在生成的文档中包含版本文本。行为与 javadoc 工具的 -version 选项相同。
