FindASPELL¶

查找 GNU Aspell 拼写检查库

find_package(ASPELL [<version>] [COMPONENTS <components>] [...])

代码说明

  • find_package(ASPELL [<version>] [COMPONENTS <components>] [...])：调用 find_package() 命令，查找并加载外部依赖包。

组件¶

本模块支持可选组件，可以使用 find_package() 命令指定。

find_package(ASPELL [COMPONENTS <components>...])

代码说明

  • find_package(ASPELL [COMPONENTS <components>...])：调用 find_package() 命令，查找并加载外部依赖包。

支持的组件包括

ASPELL

在 4.1 版本中新增。

查找 Aspell 库及其包含路径。

Executable

在 4.1 版本中新增。

查找 Aspell 命令行交互式拼写检查器可执行文件。

如果未指定组件，则模块默认搜索 ASPELL 和 Executable 两个组件。

导入的目标¶

当 CMAKE_ROLE 为 PROJECT 时，本模块提供以下 导入的目标 (Imported Targets)

ASPELL::ASPELL

在 4.1 版本中新增。

封装 Aspell 库使用需求的目标。仅在找到 ASPELL 组件时可用。

ASPELL::Executable

在 4.1 版本中新增。

封装 Aspell 命令行拼写检查器可执行目标。仅在找到 Executable 组件时可用。

结果变量¶

此模块定义了以下变量

ASPELL_FOUND

布尔值，指示是否找到了（所请求版本的）Aspell 以及所有请求的组件。

ASPELL_VERSION

在 4.1 版本中新增。

所找到的 Aspell 的版本字符串（如果有）。只有在找到 Executable 组件时才能确定。如果无法确定版本，则不设置版本值。

ASPELL_INCLUDE_DIRS

在 4.1 版本中新增。

使用 Aspell 所需的包含目录。在找到 ASPELL 组件时可用。

Aspell 库可能还通过 pspell.h 头文件提供与 Pspell 向后兼容的接口。如果找到了此类接口，它也会被添加到包含目录列表中。

ASPELL_LIBRARIES

链接到 Aspell 所需的库。在找到 ASPELL 组件时可用。

4.1 版本变更：该变量现在设置为常规结果变量，不再是缓存变量。

缓存变量¶

以下缓存变量也可以设置

ASPELL_INCLUDE_DIR

使用 Executable 组件时，包含 aspell.h 头文件的目录。

ASPELL_LIBRARY

在 4.1 版本中新增。

使用 ASPELL 组件时，Aspell 库的路径。

ASPELL_EXECUTABLE

使用 Executable 组件时，aspell 命令行拼写检查器程序的路径。

示例¶

使用 CMake 4.1 或更高版本查找 Aspell 库并将其链接到项目目标

find_package(ASPELL COMPONENTS ASPELL)
target_link_libraries(project_target PRIVATE ASPELL::ASPELL)

代码说明

  • find_package(ASPELL COMPONENTS ASPELL)：调用 find_package() 命令，查找并加载外部依赖包。

  • target_link_libraries(project_target PRIVATE ASPELL::ASPELL)：调用 target_link_libraries() 命令，为目标链接依赖库。

在编写支持 CMake 4.0 及更早版本的向后兼容代码时，可以直接在项目中定义本地导入目标

find_package(ASPELL COMPONENTS ASPELL)
if(ASPELL_FOUND AND NOT TARGET ASPELL::ASPELL)
  add_library(ASPELL::ASPELL INTERFACE IMPORTED)
  set_target_properties(
    ASPELL::ASPELL
    PROPERTIES
      INTERFACE_LINK_LIBRARIES "${ASPELL_LIBRARIES}"
      INTERFACE_INCLUDE_DIRECTORIES "${ASPELL_INCLUDE_DIR}"
  )
endif()
target_link_libraries(project_target PRIVATE ASPELL::ASPELL)

代码说明

  • find_package(ASPELL COMPONENTS ASPELL)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(ASPELL_FOUND AND NOT TARGET ASPELL::ASPELL)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；AND 表示逻辑与：两侧条件都为真时结果为真；TARGET 表示判断给定构建目标是否已定义。

  • add_library(ASPELL::ASPELL INTERFACE IMPORTED)：调用 add_library() 命令，定义一个库目标。

  • set_target_properties(：这一行继续补充当前命令或表达式的内容。

  • ASPELL::ASPELL：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • INTERFACE_LINK_LIBRARIES "${ASPELL_LIBRARIES}"：这一行继续补充当前命令或表达式的内容；变量说明：ASPELL_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • INTERFACE_INCLUDE_DIRECTORIES "${ASPELL_INCLUDE_DIR}"：这一行继续补充当前命令或表达式的内容；变量说明：ASPELL_INCLUDE_DIR：包含目录、目录相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

  • target_link_libraries(project_target PRIVATE ASPELL::ASPELL)：调用 target_link_libraries() 命令，为目标链接依赖库。

示例：如何在项目中执行 aspell 命令行拼写检查器

find_package(ASPELL COMPONENTS Executable)
execute_process(COMMAND ${ASPELL_EXECUTABLE} --help)

代码说明

  • find_package(ASPELL COMPONENTS Executable)：调用 find_package() 命令，查找并加载外部依赖包。

  • execute_process(COMMAND ${ASPELL_EXECUTABLE} --help)：调用 execute_process() 命令，在配置阶段执行外部进程；变量说明：ASPELL_EXECUTABLE：可执行文件相关变量，其精确取值和作用域以当前页面或脚本上下文为准。
