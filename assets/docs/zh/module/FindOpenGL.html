FindOpenGL（开放图形库）¶

查找 OpenGL（开放图形库） 和 OpenGL（开放图形库） 实用程序库 (GLU)，以便在 CMake 项目中使用 OpenGL（开放图形库）

find_package(OpenGL [COMPONENTS <components>...] [...])

代码说明

  • find_package(OpenGL [COMPONENTS <components>...] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

OpenGL（开放图形库） (Open Graphics Library) 是一种用于渲染 2D 和 3D 图形的跨平台 API。它广泛应用于 CAD、游戏和可视化软件中。

  • GL 指核心 OpenGL（开放图形库） 库，它提供了基础的图形渲染 API。

  • GLU (OpenGL（开放图形库） Utility Library) 是一个配套库，提供了构建于 OpenGL（开放图形库） 之上的实用函数，例如曲面细分和更复杂的形状绘制。

3.2 版本变更：在 Unix/Linux 系统上不再将 X11 添加为依赖项。

3.10 版本新增：支持 Linux 上的 GLVND（GL 供应商中立调度库）。请参阅下文的 Linux 特有 部分。

组件¶

此模块支持可选组件，这些组件可以使用 find_package() 命令指定。

find_package(OpenGL [COMPONENTS <components>...])

代码说明

  • find_package(OpenGL [COMPONENTS <components>...])：调用 `find_package()` 命令，查找并加载外部依赖包。

支持的组件包括：

EGL

3.10 版本新增。

OpenGL（开放图形库）、OpenGL（开放图形库） ES 与底层窗口系统之间的 EGL 接口。

GLX

3.10 版本新增。

X 的扩展，用于连接 OpenGL（开放图形库）、OpenGL（开放图形库） ES 与 X 窗口系统。

OpenGL（开放图形库）

3.10 版本新增。

用于 3D 图形的跨平台 API。

GLES2

在 3.27 版本中新增。

用于功能受限的嵌入式系统的 OpenGL（开放图形库） API 子集。

GLES3

在 3.27 版本中新增。

用于功能更丰富的嵌入式系统的 OpenGL（开放图形库） API 子集。

导入的目标¶

此模块提供以下 导入目标

OpenGL（开放图形库）::GL

版本 3.8 新增。

封装了特定平台 OpenGL（开放图形库） 库使用需求的目标，若找到 OpenGL（开放图形库） 则可用。

OpenGL（开放图形库）::GLU

版本 3.8 新增。

封装了 OpenGL（开放图形库） 实用程序库 (GLU) 使用需求的目标，若找到 GLU 则可用。

此外，还提供了以下 GLVND 特有的导入库目标：

OpenGL（开放图形库）::OpenGL（开放图形库）

3.10 版本新增。

封装了 libOpenGL（开放图形库） 使用需求的目标，若系统基于 GLVND 且找到 OpenGL（开放图形库） 则可用。

OpenGL（开放图形库）::GLX

3.10 版本新增。

封装了 X 窗口系统 OpenGL（开放图形库） 扩展 (GLX) 使用需求的目标，若找到 OpenGL（开放图形库） 和 GLX 则可用。

OpenGL（开放图形库）::EGL

3.10 版本新增。

封装了 EGL 使用需求的目标，若找到 OpenGL（开放图形库） 和 EGL 则可用。

OpenGL（开放图形库）::GLES2

在 3.27 版本中新增。

封装了 GLES2 使用需求的目标，若找到 OpenGL（开放图形库） 和 GLES2 则可用。

OpenGL（开放图形库）::GLES3

在 3.27 版本中新增。

封装了 GLES3 使用需求的目标，若找到 OpenGL（开放图形库） 和 GLES3 则可用。

结果变量¶

此模块定义了以下变量

3.3 版本中新增。

布尔值，指示是否找到了 OpenGL（开放图形库） 及所有请求的组件。

OPENGL_XMESA_FOUND

布尔值，指示是否找到了 OpenGL（开放图形库） XMESA。

OPENGL_GLU_FOUND

布尔值，指示是否找到了 GLU。

OpenGL（开放图形库）_OpenGL（开放图形库）_FOUND

3.10 版本新增。

布尔值，指示是否找到了 GLVND OpenGL（开放图形库） 库。

OpenGL（开放图形库）_GLX_FOUND

3.10 版本新增。

布尔值，指示是否找到了 GLVND GLX。

OpenGL（开放图形库）_EGL_FOUND

3.10 版本新增。

布尔值，指示是否找到了 GLVND EGL。

OpenGL（开放图形库）_GLES2_FOUND

在 3.27 版本中新增。

布尔值，指示是否找到了 GLES2。

OpenGL（开放图形库）_GLES3_FOUND

在 3.27 版本中新增。

布尔值，指示是否找到了 GLES3。

OPENGL_INCLUDE_DIRS

在版本 3.29 中添加。

OpenGL（开放图形库） 头文件目录路径。

OPENGL_EGL_INCLUDE_DIRS

3.10 版本新增。

EGL 头文件目录路径。

OPENGL_LIBRARIES

OpenGL（开放图形库） 库、窗口系统库和 GLU 库的路径。在 Linux 上，这假定使用了 GLX，对于基于 EGL 的目标永远是不正确的。建议客户改用 OpenGL（开放图形库）::* 导入目标。

提示¶

此模块接受以下变量

OpenGL（开放图形库）_GL_PREFERENCE

3.10 版本新增。

此变量在 Linux 系统上受支持，用于在有多种选择时指定提供传统 GL 接口的首选方式。其值可以是以下之一：

GLVND

如果 GLVND OpenGL（开放图形库） 和 GLX 库可用，则优先使用它们。这会强制 OPENGL_gl_LIBRARY 为空。

3.11 版本变更：这是默认设置，除非策略 CMP0072 设置为 OLD 且未请求任何组件（因为组件对应于 GLVND 库）。

LEGACY

如果可用，优先使用传统的 libGL 库。这使得 OPENGL_opengl_LIBRARY 和 OPENGL_glx_LIBRARY 指向 OPENGL_gl_LIBRARY。

Linux 特有¶

一些 Linux 系统利用 GLVND 作为 OpenGL（开放图形库） 的新 ABI。GLVND 将上下文库与 OpenGL（开放图形库） 本身分离开来；OpenGL（开放图形库） 位于“libOpenGL（开放图形库）”中，而上下文定义在“libGLX”或“libEGL”中。GLVND 目前是唯一能够以跨供应商可移植的方式通过 EGL 获取 OpenGL（开放图形库） 3+ 功能的方法。项目可以使用目标 OpenGL（开放图形库）::OpenGL（开放图形库） 以及 OpenGL（开放图形库）::GLX 或 OpenGL（开放图形库）::EGL 来显式使用 GLVND。

项目可以使用 OpenGL（开放图形库）::GL 目标（或 OPENGL_LIBRARIES 变量）来使用传统的 GL 接口。根据 OpenGL（开放图形库）_GL_PREFERENCE 的设置，这些接口将使用传统 GL 库或 GLVND 的 OpenGL（开放图形库）::OpenGL（开放图形库） 和 OpenGL（开放图形库）::GLX。因此，对于非基于 EGL 的 Linux 目标，OpenGL（开放图形库）::GL 目标具有最好的可移植性。

可以设置 OpenGL（开放图形库）_GL_PREFERENCE 变量来指定在有多种选择时提供传统 GL 接口的首选方式。

对于 EGL 目标，客户端必须依赖用户系统上的 GLVND 支持。链接应使用 OpenGL（开放图形库）::OpenGL（开放图形库） OpenGL（开放图形库）::EGL 目标。理论上可以使用 GLES* 库来代替 OpenGL（开放图形库）::OpenGL（开放图形库），但本模块目前不支持此操作；欢迎提交贡献。

在 GLVND 的情况下，OPENGL_egl_LIBRARY 和 OPENGL_EGL_INCLUDE_DIRS 会被定义。对于非 GLVND 的 Linux 和其他系统，这些保持未定义状态。

macOS（苹果桌面系统） 特有¶

在 macOS（苹果桌面系统） 上，此模块默认使用 macOS（苹果桌面系统） 原生的 OpenGL（开放图形库） 框架版本。若要在 macOS（苹果桌面系统） 上使用 X11 版本的 OpenGL（开放图形库），可以使用 CMAKE_FIND_FRAMEWORK 变量禁用对框架的搜索。例如：

find_package(X11)
if(APPLE AND X11_FOUND)
  set(CMAKE_FIND_FRAMEWORK NEVER)
  find_package(OpenGL)
  unset(CMAKE_FIND_FRAMEWORK)
else()
  find_package(OpenGL)
endif()

代码说明

  • find_package(X11) if(APPLE AND X11_FOUND) set(CMAKE_FIND_FRAMEWORK NEVER) find_package(OpenGL) unset(CMAKE_FIND_FRAMEWORK) else() find_package(OpenGL) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。

构建此项目的最终用户可能需要通过 -DOpenGL（开放图形库）_ROOT=/opt/X11 等方式将 CMake 指向其 X11 安装路径。

已弃用变量¶

OPENGL_FOUND

4.2 版本弃用：请使用 OpenGL（开放图形库）_FOUND，其值相同。

布尔值，指示是否找到了 OpenGL（开放图形库） 及所有请求的组件。
