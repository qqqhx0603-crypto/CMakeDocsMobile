TestForANSIForScope¶

此模块检查 CXX 编译器是否将 for-init-statement 中声明的变量的作用域限制在循环体内部。

在 CMake 项目中加载此模块，使用

include(TestForANSIForScope)

在早期的 C++ (C++98 之前)，在 for(<init-statement> ...) 中声明的变量在循环体 (for() { <body> }) 之后仍然可以在循环外部访问。

此模块定义了以下缓存变量

CMAKE_NO_ANSI_FOR_SCOPE

一个缓存变量，包含检查结果。如果 for-init-statement 具有受限的作用域 (C++ 98 及更高版本)，则其值为 0，如果不是 (ANSI C++)，则其值为 1。

注意

自 C++ 98 标准以来，在 for-init-statement 中声明的变量被限制在循环体内部，这使得这种行为已过时。

示例¶

包含此模块将检查 for() 循环的作用域行为，并定义 CMAKE_NO_ANSI_FOR_SCOPE 缓存变量

CMakeLists.txt¶
include(TestForANSIForScope)
file(
  CONFIGURE
  OUTPUT config.h
  CONTENT "#cmakedefine CMAKE_NO_ANSI_FOR_SCOPE"
)

然后可以在 C++ 程序中使用它

example.cxx¶
#include "config.h"

#ifdef CMAKE_NO_ANSI_FOR_SCOPE
#  define for if(false) {} else for
#endif
