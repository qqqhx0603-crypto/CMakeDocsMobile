FindGettext¶

查找 GNU gettext 工具并提供用于生成多语言消息的命令

find_package(Gettext [<version>] [...])

代码说明

  • find_package(Gettext [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

GNU gettext 是一个用于国际化 (i18n) 和本地化 (l10n) 的系统，由命令行工具和运行时库 (libintl) 组成。此模块用于查找 gettext 工具（如 msgmerge 和 msgfmt），而运行时库可以通过独立的 FindIntl 模块进行查找，该模块抽象了各种实现中的 libintl 处理方式。

gettext 使用的常用文件类型

POT 文件

便携式对象模板 (.pot) 文件，用作翻译的源模板。

PO 文件

便携式对象 (.po) 文件，包含人类可读的翻译内容。

MO 文件

机器对象 (.mo) 文件，由 .po 文件编译而来，供运行时使用。

结果变量¶

此模块定义了以下变量

Gettext_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）gettext。

Gettext_VERSION

版本 4.2 中添加。

找到的 gettext 版本。

缓存变量¶

以下缓存变量也可以设置

GETTEXT_MSGMERGE_EXECUTABLE

用于合并消息目录和模板的 msgmerge 工具的完整路径。

GETTEXT_MSGFMT_EXECUTABLE

用于将消息目录编译为二进制格式的 msgfmt 工具的完整路径。

命令¶

如果找到了 gettext，此模块提供以下命令以在 CMake 中使用 gettext

gettext_process_po_files¶

创建一个构建规则，将一个或多个 .po 翻译文件处理为指定可翻译语言区域的二进制 .mo 文件

gettext_process_po_files(
  <language>
  [ALL]
  [INSTALL_DESTINATION <destdir>]
  PO_FILES <po-files>...
)

代码说明

  • gettext_process_po_files( <language> [ALL] [INSTALL_DESTINATION <destdir>] PO_FILES <po-files>... )：调用 `gettext_process_po_files()` 命令，具体行为由这一行传入的参数决定。

此命令定义了一个自定义目标，将给定的 <po-files> 编译为指定 <language> 的 .mo 文件。在首次调用时，它还会创建一个名为 pofiles 的全局自定义目标，后续所有的调用都会贡献于此。该目标可用于集体构建所有翻译文件，或在其他 CMake 命令中引用。

此命令应针对每个语言区域分别调用，以便为每个区域生成相应的 .mo 文件。

参数为

<language>

PO 文件对应的目标可翻译语言区域。

此字符串通常格式化为区域标识符（例如，德国德语使用 de_DE，奥地利德语使用 de_AT 等）。下划线之前的部分指定语言，下划线之后的部分指定国家或地区变体。在某些情况下，也可以使用仅包含语言代码的缩写形式（例如 de）。

ALL

此选项将生成的目标添加到默认的 CMake 构建目标中，以便默认构建翻译。

INSTALL_DESTINATION <destdir>

指定安装阶段生成的 .mo 文件的安装目录。如果指定，文件将安装到：<destdir>/<language>/LC_MESSAGES/*.mo。如果未指定，则不安装文件。

PO_FILES <po-files>...

一个或多个 .po 翻译文件的列表，这些文件将在构建阶段被编译为指定 <language> 的 .mo 文件。相对路径将被解释为相对于当前源目录（CMAKE_CURRENT_SOURCE_DIR）。

gettext_process_pot_file¶

创建一个构建规则，将 gettext 便携式对象模板 (.pot) 文件和关联的 .po 文件处理为编译后的 gettext 机器对象 (.mo) 文件

gettext_process_pot_file(
  <pot-file>
  [ALL]
  [INSTALL_DESTINATION <destdir>]
  LANGUAGES <languages>...
)

代码说明

  • gettext_process_pot_file( <pot-file> [ALL] [INSTALL_DESTINATION <destdir>] LANGUAGES <languages>... )：调用 `gettext_process_pot_file()` 命令，具体行为由这一行传入的参数决定。

此命令定义了一个名为 potfiles 的自定义目标，它将给定的 <pot-file> 和特定语言的 .po 文件编译为每个指定语言的二进制 .mo 文件。在调用此命令之前，相应的 <language>.po 文件必须已存在于当前二进制目录（CMAKE_CURRENT_BINARY_DIR）中。

参数为

<pot-file>

用作翻译源的 gettext 便携式对象模板文件 (.pot) 的路径。如果给出相对路径，它将被解释为相对于当前源目录（CMAKE_CURRENT_SOURCE_DIR）。

ALL

将生成的目标添加到默认的 CMake 构建目标，以便默认构建这些文件。

INSTALL_DESTINATION <destdir>

指定安装阶段生成的 .mo 文件的安装目录。如果指定，文件将安装到：<destdir>/<language>/LC_MESSAGES/<pot-base-filename>.mo。如果未指定，则不安装文件。

LANGUAGES <languages>...

一个或多个可翻译语言区域的列表（例如 en_US, fr, de_DE, zh_CN 等）。

gettext_create_translations¶

创建一个构建规则，将给定的 .pot 模板文件和关联的 .po 翻译文件处理为编译后的机器对象 (.mo) 文件

gettext_create_translations(<pot-file> [ALL] <po-files>...)

代码说明

  • gettext_create_translations(<pot-file> [ALL] <po-files>...)：调用 `gettext_create_translations()` 命令，具体行为由这一行传入的参数决定。

此命令定义了一个名为 translations 的自定义目标，将指定的 <pot-file> 和 <po-files> 编译为二进制 .mo 文件。它还会自动为生成的 .mo 文件添加一个安装规则，在安装阶段将其安装到默认的 share/locale/<language>/LC_MESSAGES/<pot-base-filename>.mo 路径中。

参数为

<pot-file>

用作翻译源的 gettext 便携式对象模板文件 (.pot) 的路径。如果给出相对路径，它将被解释为相对于当前源目录（CMAKE_CURRENT_SOURCE_DIR）。

ALL

将生成的目标添加到默认的 CMake 构建目标，以便在构建时默认创建翻译。

<po-files>...

一个或多个 .po 格式的翻译源文件列表，其文件名必须遵循 <language>.po 格式。相对路径将被解释为相对于当前源目录（CMAKE_CURRENT_SOURCE_DIR）。

注意

为了更好地控制构建和安装行为，请改用 gettext_process_po_files()。

已弃用变量¶

以下变量提供用于向后兼容性

GETTEXT_FOUND

4.2 版本弃用：请使用 Gettext_FOUND，它具有相同的值。

布尔值，指示是否找到了（所请求版本的）gettext。

GETTEXT_VERSION_STRING

4.2 版本弃用：已被 Gettext_VERSION 取代。

找到的 gettext 版本。

示例：查找 gettext¶

查找 GNU gettext 工具

find_package(Gettext)

代码说明

  • find_package(Gettext)：调用 `find_package()` 命令，查找并加载外部依赖包。

或者，查找 gettext 并指定最低要求版本

find_package(Gettext 0.21)

代码说明

  • find_package(Gettext 0.21)：调用 `find_package()` 命令，查找并加载外部依赖包。

或者，查找 gettext 并将其设为必须（如果未找到，处理将停止并报错）

find_package(Gettext REQUIRED)

代码说明

示例：在 CMake 中使用 gettext¶

刚开始使用 gettext 时，.pot 文件通常被视为手动创建的。例如，在提供的 main.cxx 源代码文件上使用 xgettext 工具

main.cxx¶
#include <iostream>
#include <libintl.h>
#include <locale.h>

int main()
{
  // Set locale from environment
  setlocale(LC_ALL, "");

  // Bind the text domain
  const char* dir = std::getenv("TEXTDOMAINDIR");
  if (!dir) {
    dir = "/usr/local/share/locale";
  }
  bindtextdomain("MyApp", dir);
  textdomain("MyApp");

  std::cout << gettext("Hello, World") << std::endl;

  return 0;
}

代码说明

  • #include <iostream> #include <libintl.h> #include <locale.h> int main() { // Set locale from environment setlocale(LC_ALL, ""); // Bind the text domain const char* dir = std::getenv("TEXTDOMAINDIR"); if (!dir) { dir = "/usr/local/share/locale"; } bindtextdomain("MyApp", dir); textdomain("MyApp"); std::cout << gettext("Hello, World") << std::endl; return 0; }：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

xgettext 工具会从提供的源代码中所有 gettext() 调用中提取字符串，并创建可翻译字符串

$ xgettext -o MyApp.pot main.cxx

代码说明

  • $ xgettext -o MyApp.pot main.cxx：这是一条命令行输入，用来演示终端中实际执行的命令。

项目可以使用 msginit 工具手动初始化可翻译文件

$ mkdir -p locale/de_DE
$ msginit -l de_DE.UTF8 -o locale/de_DE/MyApp.po -i MyApp.pot --no-translator

代码说明

  • $ mkdir -p locale/de_DE $ msginit -l de_DE.UTF8 -o locale/de_DE/MyApp.po -i MyApp.pot --no-translator：这是一条命令行输入，创建目录。

它会创建一个人类可读的文件，可以将其翻译成所需的语言（根据需要进行调整）

locale/de_DE/MyApp.po¶
msgid ""
msgstr ""
"Language: de\n"
"Content-Type: text/plain; charset=UTF-8\n"
"Content-Transfer-Encoding: 8bit\n"
"Plural-Forms: nplurals=2; plural=(n != 1);\n"

msgid "Hello, World"
msgstr "Hallo, Welt"

代码说明

  • msgid "" msgstr "" "Language: de\n" "Content-Type: text/plain; charset=UTF-8\n" "Content-Transfer-Encoding: 8bit\n" "Plural-Forms: nplurals=2; plural=(n != 1);\n" msgid "Hello, World" msgstr "Hallo, Welt"：这一行给出示例文本或输出，用来说明：msgid "" msgstr "" "Language: de\n" "Content-Type: text/plain; charset=UTF-8\n" "Content-Transfer-Encoding: 8bit\n" "Plural-Forms: nplurals=2; plural=(n != 1);\n" msgid "Hello, World" msgstr "Hallo, Welt"

在 CMake 中，此模块提供的 gettext_process_po_files() 命令会自动创建应用程序在运行时加载所需的 .mo 文件，具体取决于系统环境变量（如 LANG）。在下面的示例中，还使用了 GNUInstallDirs 模块来提供 CMAKE_INSTALL_LOCALEDIR 变量

CMakeLists.txt¶
cmake_minimum_required(VERSION 3.24)
project(GettextExample)
include(GNUInstallDirs)

find_package(Gettext)

if(Gettext_FOUND)
  foreach(language IN ITEMS de_DE)
    gettext_process_po_files(
      ${language}
      ALL
      PO_FILES locale/${language}/MyApp.po
      INSTALL_DESTINATION ${CMAKE_INSTALL_LOCALEDIR}
    )
  endforeach()
endif()

add_executable(example main.cxx)

# Find and link Intl library to use gettext() from libintl.h
find_package(Intl)
target_link_libraries(example PRIVATE Intl::Intl)

install(TARGETS example)

代码说明

  • cmake_minimum_required(VERSION 3.24) project(GettextExample) include(GNUInstallDirs) find_package(Gettext) if(Gettext_FOUND) foreach(language IN ITEMS de_DE) gettext_process_po_files( ${language} ALL PO_FILES locale/${language}/MyApp.po INSTALL_DESTINATION ${CMAKE_INSTALL_LOCALEDIR} ) endforeach() endif() add_executable(example main.cxx) # Find and link Intl library to use gettext() from libintl.h find_package(Intl) target_link_libraries(example PRIVATE Intl::Intl) install(TARGETS example)：调用 `cmake_minimum_required()` 命令，声明工程所需的最低 CMake 版本。

$ cmake -B build
$ cmake --build build
$ DESTDIR=$(pwd)/stage cmake --install build

代码说明

  • $ cmake -B build $ cmake --build build $ DESTDIR=$(pwd)/stage cmake --install build：这是一条命令行输入，调用 CMake 命令。

要利用翻译，系统上需要启用 de_DE 区域设置（请参阅 locale -a）。然后可以运行本地化的输出

$ LANG=de_DE.UTF-8 TEXTDOMAINDIR=./stage/usr/local/share/locale \
  ./stage/usr/local/bin/example

代码说明

  • $ LANG=de_DE.UTF-8 TEXTDOMAINDIR=./stage/usr/local/share/locale \ ./stage/usr/local/bin/example：这是一条命令行输入，用来演示终端中实际执行的命令。

示例：处理 POT 文件¶

gettext_process_pot_file() 命令将位于当前二进制目录中的 .po 翻译文件处理为 .mo 文件

CMakeLists.txt¶
find_package(Gettext)

if(Gettext_FOUND)
  set(languages de_DE fr zh_CN)

  foreach(language IN LISTS languages)
    configure_file(locale/${language}.po ${language}.po COPYONLY)
  endforeach()

  gettext_process_pot_file(
    MyApp.pot
    ALL
    INSTALL_DESTINATION ${CMAKE_INSTALL_LOCALEDIR}
    LANGUAGES ${languages}
  )
endif()

代码说明

  • find_package(Gettext) if(Gettext_FOUND) set(languages de_DE fr zh_CN) foreach(language IN LISTS languages) configure_file(locale/${language}.po ${language}.po COPYONLY) endforeach() gettext_process_pot_file( MyApp.pot ALL INSTALL_DESTINATION ${CMAKE_INSTALL_LOCALEDIR} LANGUAGES ${languages} ) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。

示例：创建翻译¶

使用简化的 gettext_create_translations() 命令来创建 .mo 文件

CMakeLists.txt¶
find_package(Gettext)

if(Gettext_FOUND)
  gettext_create_translations(
    MyApp.pot
    ALL
    locale/de_DE.po
    locale/fr.po
    locale/zh_CN.po
  )
endif()

代码说明

  • find_package(Gettext) if(Gettext_FOUND) gettext_create_translations( MyApp.pot ALL locale/de_DE.po locale/fr.po locale/zh_CN.po ) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。
