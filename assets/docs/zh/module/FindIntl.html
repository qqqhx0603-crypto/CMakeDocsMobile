FindIntl¶

版本 3.2 中新增。

查找包含消息翻译函数（例如 gettext()）的国际化支持

find_package(Intl [<version>] [...])

这些函数源自 GNU libintl 库，它是 GNU gettext 工具的一部分，但也可能由标准 C 库提供。

导入的目标¶

此模块提供以下 导入目标

Intl::Intl

在 3.20 版本中添加。

封装 Intl 使用需求的 Target，如果找到 Intl 则可用。

结果变量¶

此模块定义了以下变量

Intl_FOUND

布尔值，指示是否找到（请求版本的）Intl。

Intl_VERSION

3.21 版本新增。

找到的 Intl 实现或库的版本，格式为 x.y.z。

注意

某些 Intl 实现不会将版本嵌入到其头文件中。在这种情况下，变量 Intl_VERSION* 将为空。

Intl_VERSION_MAJOR

3.21 版本新增。

找到的 Intl 的主版本号。

Intl_VERSION_MINOR

3.21 版本新增。

找到的 Intl 的次版本号。

Intl_VERSION_PATCH

3.21 版本新增。

找到的 Intl 的补丁版本号。

Intl_INCLUDE_DIRS

包含使用 Intl 所需头文件的包含目录。

Intl_LIBRARIES

使用 Intl 需要链接的库。

缓存变量¶

以下缓存变量也可以设置

Intl_IS_BUILT_IN

在 3.20 版本中添加。

布尔值，指示找到的 Intl 功能是否由标准 C 库提供，而不是单独的 libintl 库。

Intl_INCLUDE_DIR

包含 libintl.h 头文件的目录。

Intl_LIBRARY

Intl 库的路径（如果有）。

注意

在某些平台上，例如使用 GNU libc 的 Linux，gettext 函数存在于 C 标准库中，不需要 libintl。在这种情况下，Intl_LIBRARY 和 Intl_INCLUDE_DIR 将为空。

示例¶

查找 Intl 支持并在项目中将导入的目标用于使用

find_package(Intl)
target_link_libraries(app PRIVATE Intl::Intl)

另请参阅¶

  • FindGettext 模块用于查找和使用 GNU gettext 工具（msgmerge、msgfmt 等）。
