TestForSTDNamespace¶

此模块检查 CXX 编译器是否支持 C++ 标准库的 std 命名空间。

在 CMake 项目中加载此模块，使用

include(TestForSTDNamespace)

代码说明

  • include(TestForSTDNamespace)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

早期版本的 C++（C++98 之前）没有要求 C++ 标准模板库 (STL) 组件（例如 list 等）和其他 C++ 标准库部分（例如 I/O 流 cout、endl 等）具有专用的命名空间，因此它们在全局范围内可用。

此模块定义了以下缓存变量

CMAKE_NO_STD_NAMESPACE

一个缓存变量，包含检查结果。如果支持 std 命名空间（C++ 98 及更高版本），则其值为 0；如果不支持（ANSI C++），则其值为 1。

注意

在 C++ 98 标准中正式引入了 std 命名空间，从而使这个问题变得过时。

示例¶

包含此模块将检查 std 命名空间的支持，并定义 CMAKE_NO_STD_NAMESPACE 缓存变量

CMakeLists.txt¶
include(TestForSTDNamespace)
file(
  CONFIGURE
  OUTPUT config.h
  CONTENT "#cmakedefine CMAKE_NO_STD_NAMESPACE"
)

代码说明

  • include(TestForSTDNamespace) file( CONFIGURE OUTPUT config.h CONTENT "#cmakedefine CMAKE_NO_STD_NAMESPACE" )：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

然后可以在 C++ 程序中使用该变量来定义缺少的命名空间

example.cxx¶
#include "config.h"

#ifdef CMAKE_NO_STD_NAMESPACE
#  define std
#endif

代码说明

  • #include "config.h" #ifdef CMAKE_NO_STD_NAMESPACE # define std #endif：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。
