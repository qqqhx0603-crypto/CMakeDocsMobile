FindOpenSP¶

在 3.25 版本中新增。

查找 OpenSP 库

find_package(OpenSP [<version>] [...])

代码说明

  • find_package(OpenSP [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

OpenSP 是 SGML（标准通用标记语言）解析器的开源实现。

导入的目标¶

此模块提供以下 导入目标

OpenSP::OpenSP

封装 OpenSP 库使用需求的 target，仅在找到 OpenSP 时可用。

结果变量¶

此模块定义了以下变量

OpenSP_FOUND

布尔值，指示是否可用（请求版本的）OpenSP。

OpenSP_VERSION

找到的 OpenSP 的版本。

OpenSP_VERSION_MAJOR

OpenSP 的主版本号。

OpenSP_VERSION_MINOR

OpenSP 的次版本号。

OpenSP_VERSION_PATCH

OpenSP 的补丁版本号。

OpenSP_INCLUDE_DIRS

包含使用 OpenSP 库所需的头文件的包含目录。

OpenSP_LIBRARIES

需要链接的库，以便使用 OpenSP。这些可以传递给 target_link_libraries() 命令，当不使用 OpenSP::OpenSP 导入 target 时。

缓存变量¶

以下缓存变量也可以设置

OpenSP_INCLUDE_DIR

OpenSP 包含目录。

OpenSP_LIBRARY

osp 库的绝对路径。

OpenSP_MULTI_BYTE

如果 OpenSP 的 config.h 头文件中定义了 SP_MULTI_BYTE，则为 True，这表示 OpenSP 库是使用对多字节字符的支持编译的。 消耗 target 需要定义 SP_MULTI_BYTE 预处理器宏，以匹配此值，以避免字符解码问题。

示例¶

查找 OpenSP 库并将其链接到项目 target

find_package(OpenSP)
target_link_libraries(project_target PRIVATE OpenSP::OpenSP)

代码说明

  • find_package(OpenSP) target_link_libraries(project_target PRIVATE OpenSP::OpenSP)：调用 `find_package()` 命令，查找并加载外部依赖包。
