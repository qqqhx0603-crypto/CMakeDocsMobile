CheckCSourceRuns¶

该模块提供了一个命令，用于检查 C 源代码是否能够编译并运行。

在 CMake 项目中加载此模块，使用

include(CheckCSourceRuns)

代码说明

  • include(CheckCSourceRuns)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_c_source_runs¶

检查给定的 C 源代码是否能编译、链接成可执行文件并随后成功运行。

check_c_source_runs(<code> <variable>)

代码说明

  • check_c_source_runs(<code> <variable>)：调用 `check_c_source_runs()` 命令，具体行为由这一行传入的参数决定。

<code> 中提供的 C 源代码必须至少包含一个 main() 函数。检查结果将存储在由 <variable> 指定的内部缓存变量中。如果代码构建成功且运行时的退出码为 0，则返回布尔值 true 表示成功。如果构建或运行失败，则返回布尔值 false（例如空字符串或错误消息）。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

这是一个空格分隔的字符串，包含传递给编译器的额外标志。分号分隔的列表在此处无效。CMAKE_<LANG>_FLAGS 及其关联的特定于配置的 CMAKE_<LANG>_FLAGS_<CONFIG> 变量的内容会自动添加到编译器命令中，并置于该变量内容之前。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

以下示例演示了如何使用此模块来检查 C 源代码是否受支持且在运行时正常工作。检查结果存储在内部缓存变量 HAVE_NORETURN 中。

include(CheckCSourceRuns)

check_c_source_runs([[
  #include <stdlib.h>
  #include <stdnoreturn.h>
  noreturn void f(){ exit(0); }
  int main(void) { f(); return 1; }
]] HAVE_NORETURN)

代码说明

  • include(CheckCSourceRuns) check_c_source_runs([[ #include <stdlib.h> #include <stdnoreturn.h> noreturn void f(){ exit(0); } int main(void) { f(); return 1; } ]] HAVE_NORETURN)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。
