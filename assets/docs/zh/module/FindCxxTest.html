FindCxxTest¶

查找 CxxTest（一个 C++ 单元测试框架套件），并提供一个辅助命令来创建测试运行器并将它们与 CTest 集成。

find_package(CxxTest [...])

代码说明

  • find_package(CxxTest [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

结果变量¶

此模块定义了以下变量

CxxTest_FOUND

版本 4.2 中添加。

布尔值，指示是否找到了 CxxTest 框架。

CXXTEST_INCLUDE_DIRS

包含使用 CxxTest 所需头文件的目录。

CXXTEST_TESTGEN_EXECUTABLE

找到的 CxxTest 测试生成器脚本（基于 Perl 或 Python）的路径，根据找到的解释器或用户指定的偏好进行选择。

CXXTEST_TESTGEN_INTERPRETER

如果需要（例如，在不支持脚本 shebang 行的平台上），则为运行测试生成器脚本所找到的 Perl 或 Python 解释器的路径。

缓存变量¶

以下缓存变量也可以设置

CXXTEST_PYTHON_TESTGEN_EXECUTABLE

基于 Python 的 CxxTest 测试生成器脚本的路径。

CXXTEST_PERL_TESTGEN_EXECUTABLE

基于 Perl 的 CxxTest 测试生成器脚本的路径。

注意

基于 Perl 的测试生成器脚本已在 CxxTest 4.0 版本中移除，转而使用基于 Python 的脚本。

提示¶

在调用 find_package(CxxTest) 之前，该模块接受以下变量

CXXTEST_TESTGEN_ARGS

可以设置此变量以指定传递给 CxxTest 代码生成器的命令行选项列表（以分号分隔）。如果未设置，默认值为 --error-printer。

命令¶

如果找到 CxxTest，该模块提供以下命令

cxxtest_add_test¶

创建一个 CxxTest 运行器并将其添加到 CTest 测试套件中

cxxtest_add_test(<test-name> <gen-source-file> <input-files-to-testgen>...)

代码说明

  • cxxtest_add_test(<test-name> <gen-source-file> <input-files-to-testgen>...)：调用 `cxxtest_add_test()` 命令，具体行为由这一行传入的参数决定。

参数

<test-name>

要创建并在 CTest 套件中注册为测试的测试可执行目标名称。

<gen-source-file>

要由 CxxTest 代码生成器生成的源文件的名称。这必须是相对路径。它是相对于当前二进制目录（CMAKE_CURRENT_BINARY_DIR）进行解释的。

<input-files-to-testgen>...

包含派生自 C++ 类 CxxTest::TestSuite 的测试套件类的头文件列表，这些文件将被包含在测试运行器中。这些必须以绝对路径形式给出。

已弃用变量¶

以下变量提供用于向后兼容性

CXXTEST_FOUND

4.2 版本弃用：请使用 CxxTest_FOUND，其值相同。

布尔值，指示是否找到了 CxxTest 框架。

CXXTEST_USE_PYTHON

2.8.3 版本弃用：在较早版本的 CMake 中，此提示变量用于强制使用基于 Python 的测试生成器而不是 Perl 生成器，而不管安装了哪种脚本语言。现在仅在同时找到 Perl 和 Python 解释器时才会考虑它。

一个布尔提示变量，当设置为 true 时，如果同时找到 Perl 和 Python 解释器，则优先选择 Python 代码生成器。此变量仅在使用 CxxTest 3 版本时相关。

示例¶

以下示例演示了如何在此模块中将 CxxTest 与 CMake 一起使用。如果找到 CxxTest

  • 在项目中手动创建额外的接口 导入目标，以封装 CxxTest 的使用需求并将其链接到指定的测试。此类目标在处理多个测试时非常有用。

  • 调用测试生成器，从位于当前源目录的输入头文件 foo_test.h 在当前二进制目录中创建 foo_test.cc。

  • 构建一个名为 unit_test_foo 的可执行文件，并将其在 CTest 中注册为一项测试。

CMakeLists.txt¶
find_package(CxxTest)

# Create interface imported target:
if(CxxTest_FOUND AND NOT TARGET CxxTest::CxxTest)
  add_library(CxxTest::CxxTest INTERFACE IMPORTED)
  set_target_properties(
    CxxTest::CxxTest
    PROPERTIES INTERFACE_INCLUDE_DIRECTORIES "${CXXTEST_INCLUDE_DIRS}"
  )
endif()

# Add test:
if(CxxTest_FOUND)
  enable_testing()

  cxxtest_add_test(
    unit_test_foo
    foo_test.cc
    ${CMAKE_CURRENT_SOURCE_DIR}/foo_test.h
  )

  target_link_libraries(
    unit_test_foo
    PRIVATE
      CxxTest::CxxTest
      # Link any project targets as needed, if test depends on them:
      foo
  )
endif()

代码说明

  • find_package(CxxTest) # Create interface imported target: if(CxxTest_FOUND AND NOT TARGET CxxTest::CxxTest) add_library(CxxTest::CxxTest INTERFACE IMPORTED) set_target_properties( CxxTest::CxxTest PROPERTIES INTERFACE_INCLUDE_DIRECTORIES "${CXXTEST_INCLUDE_DIRS}" ) endif() # Add test: if(CxxTest_FOUND) enable_testing() cxxtest_add_test( unit_test_foo foo_test.cc ${CMAKE_CURRENT_SOURCE_DIR}/foo_test.h ) target_link_libraries( unit_test_foo PRIVATE CxxTest::CxxTest # Link any project targets as needed, if test depends on them: foo ) endif()：调用 `find_package()` 命令，查找并加载外部依赖包。

foo_test.h¶
#include <cxxtest/TestSuite.h>

class MyTestSuite : public CxxTest::TestSuite
{
public:
  void testAddition(void)
  {
    TS_ASSERT(1 + 1 > 1);
    TS_ASSERT_EQUALS(1 + 1, 2);
  }
};

代码说明

  • #include <cxxtest/TestSuite.h> class MyTestSuite : public CxxTest::TestSuite { public: void testAddition(void) { TS_ASSERT(1 + 1 > 1); TS_ASSERT_EQUALS(1 + 1, 2); } };：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。
