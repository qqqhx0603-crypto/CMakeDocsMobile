FindODBC¶

3.12 版本新增。

查找 Open Database Connectivity (ODBC) 库，该库实现了用于访问数据库系统的标准 API。

find_package(ODBC [...])

ODBC 使应用程序能够使用一组通用的函数与不同的数据库管理系统 (DBMS) 进行通信。与特定数据库的通信通过 ODBC 驱动程序处理，这些驱动程序由库在运行时加载。

在 Windows 上，使用 Visual Studio 构建时，此模块假定 ODBC 库由可用的 Windows SDK 提供。

在类 Unix 系统上，此模块搜索由 unixODBC 或 iODBC 实现的 ODBC API 提供的 ODBC 库。默认情况下，此模块会查找 ODBC 配置程序以确定 ODBC 库和包含目录，优先顺序为先 unixODBC，后 iODBC。如果未找到配置程序，它将在标准位置搜索 ODBC 头文件和库。

导入的目标¶

此模块提供以下 导入目标

ODBC::ODBC

封装 ODBC 使用要求的目标，在找到 ODBC 时可用。

结果变量¶

此模块定义了以下变量

ODBC_FOUND

指示是否找到 ODBC 的布尔值。

ODBC_INCLUDE_DIRS

包含使用 ODBC 所需头文件的目录。

ODBC_LIBRARIES

使用 ODBC 进行链接时所需的库。

缓存变量¶

以下缓存变量也可以设置

ODBC_INCLUDE_DIR

包含 sql.h 和其他 ODBC 头文件的目录路径。在 Windows 上可能为空，因为对应于预期 Windows SDK 的包含目录已在编译环境中可用。

ODBC_LIBRARY

ODBC 库的路径或库名称。在 Windows 上，这可能仅是一个库名称，因为对应于预期 Windows SDK 的库目录已在编译环境中可用。

ODBC_CONFIG

如果找到或指定了 ODBC 配置程序，则为其路径。例如，unixODBC 使用 odbc_config，或 iODBC 使用 iodbc-config。

限制¶

  • 在 Windows 上，此模块不会搜索 iODBC。

  • 在类 Unix 系统上，没有内置机制来优先选择 unixODBC 或 iODBC。要绕过此限制，请明确将 ODBC_CONFIG 变量设置为所需 ODBC 配置程序的路径。

  • 此模块不支持搜索或选择特定的 ODBC 驱动程序。

示例¶

示例：查找并使用 ODBC¶

查找 ODBC 并将其链接到项目目标

CMakeLists.txt¶
find_package(ODBC)
target_link_libraries(project_target PRIVATE ODBC::ODBC)

示例：查找自定义 ODBC 安装¶

以下示例适用于类 Unix 系统，并演示了如何在 CMake 配置阶段设置提示和缓存变量，以帮助此模块查找自定义的 ODBC 实现（例如默认不支持的实现）。

使用 CMAKE_PREFIX_PATH 指定安装前缀

$ cmake -D CMAKE_PREFIX_PATH=/path/to/odbc-installation -B build

或者使用专门的 ODBC_ROOT 变量

$ cmake -D ODBC_ROOT=/path/to/odbc-installation -B build

手动指定 ODBC 配置程序（如果可用），以便可以根据配置工具自动确定 ODBC 安装

$ cmake -D ODBC_CONFIG=/path/to/odbc/bin/odbc-config -B build

手动指定 ODBC 库和包含目录

$ cmake \
    -D ODBC_LIBRARY=/path/to/odbc/lib/libodbc.so \
    -D ODBC_INCLUDE_DIR=/path/to/odbc/include \
    -B build
