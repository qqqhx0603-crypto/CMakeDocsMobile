CMakePushCheckState¶

此模块提供了一些命令，用于管理那些会影响各种 CMake 检查命令（例如 check_symbol_exists() 等）执行方式的变量状态。

在 CMake 项目中加载此模块，使用

include(CMakePushCheckState)

代码说明

  • include(CMakePushCheckState)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

此模块提供了以下命令，它们对于作用域配置非常有用，例如在 CMake 模块中或在受控环境中执行检查时，确保临时修改仅隔离在检查的作用域内，而不会传播到构建系统的其他部分。

  • cmake_push_check_state()

  • cmake_reset_check_state()

  • cmake_pop_check_state()

受影响的变量¶

此模块的命令会保存、重置和恢复以下 CMake 变量：

CMAKE_REQUIRED_FLAGS

这是一个空格分隔的字符串，包含要传递给编译器的附加标志。分号分隔的列表在此处无效。CMAKE_<LANG>_FLAGS 的内容及其相关的特定配置变量 CMAKE_<LANG>_FLAGS_<CONFIG> 会自动置于此变量内容之前并传递给编译器命令。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

CMAKE_EXTRA_INCLUDE_FILES

3.6 版本新增：此前已由 check_type_size() 命令使用；现在也由此模块支持。

执行检查时要包含的额外头文件的分号分隔列表。

注意

其他 CMake 变量（如 CMAKE_<LANG>_FLAGS）无论使用本模块提供的何种命令，都会传播到所有检查中，因为这些基础变量旨在影响构建系统的全局状态。

命令¶

cmake_push_check_state¶

将上述变量的当前状态压入（保存到）堆栈中。

cmake_push_check_state([RESET])

代码说明

  • cmake_push_check_state([RESET])：调用 `cmake_push_check_state()` 命令，具体行为由这一行传入的参数决定。

在为特定检查进行临时修改之前，请使用此命令保留当前的配置。

RESET

指定此选项时，该命令不仅会保存所列变量的当前状态，还会将它们重置为空，从而允许从干净的状态开始重新配置。

cmake_reset_check_state¶

将上述所列变量的内容重置（清除）为空状态。

cmake_reset_check_state()

代码说明

  • cmake_reset_check_state()：调用 `cmake_reset_check_state()` 命令，具体行为由这一行传入的参数决定。

当执行多个需要全新配置的连续检查时，请使用此命令，以确保没有任何先前的配置被意外地保留下来。

cmake_pop_check_state¶

将上述所列变量的状态恢复为最近一次 cmake_push_check_state() 调用时的值。

cmake_pop_check_state()

代码说明

  • cmake_pop_check_state()：调用 `cmake_pop_check_state()` 命令，具体行为由这一行传入的参数决定。

使用此命令可撤销检查期间所做的临时更改。为防止出现意外行为，请将每个 cmake_push_check_state() 与对应的 cmake_pop_check_state() 配对使用。

示例¶

示例：带有编译定义的独立检查¶

在以下示例中，对 C 符号 memfd_create() 的检查是在附加了 _GNU_SOURCE 编译定义的情况下执行的，且不会影响全局编译标志。RESET 选项用于确保在检查之前明确清除与检查相关的变量的任何先前值。

include(CMakePushCheckState)

# Save and reset the current state
cmake_push_check_state(RESET)

# Perform check with specific compile definitions
set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE)
include(CheckSymbolExists)
check_symbol_exists(memfd_create "sys/mman.h" HAVE_MEMFD_CREATE)

# Restore the original state
cmake_pop_check_state()

代码说明

  • include(CMakePushCheckState) # Save and reset the current state cmake_push_check_state(RESET) # Perform check with specific compile definitions set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE) include(CheckSymbolExists) check_symbol_exists(memfd_create "sys/mman.h" HAVE_MEMFD_CREATE) # Restore the original state cmake_pop_check_state()：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

示例：嵌套配置作用域¶

在以下示例中，变量状态被多次压入堆栈，从而允许进行连续或嵌套的检查。每个 cmake_pop_check_state() 都会恢复最近压入的状态。

include(CMakePushCheckState)

# Save and reset the current state
cmake_push_check_state(RESET)

# Perform the first check with additional libraries
set(CMAKE_REQUIRED_LIBRARIES ${CMAKE_DL_LIBS})
include(CheckSymbolExists)
check_symbol_exists(dlopen "dlfcn.h" HAVE_DLOPEN)

# Save current state
cmake_push_check_state()

# Perform the second check with libraries and additional compile definitions
set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE)
check_symbol_exists(dladdr "dlfcn.h" HAVE_DLADDR)

message(STATUS "${CMAKE_REQUIRED_DEFINITIONS}")
# Output: -D_GNU_SOURCE

# Restore the previous state
cmake_pop_check_state()

message(STATUS "${CMAKE_REQUIRED_DEFINITIONS}")
# Output here is empty

# Reset variables to prepare for the next check
cmake_reset_check_state()

# Perform the next check only with additional compile definitions
set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE)
check_symbol_exists(dl_iterate_phdr "link.h" HAVE_DL_ITERATE_PHDR)

# Restore the original state
cmake_pop_check_state()

代码说明

  • include(CMakePushCheckState) # Save and reset the current state cmake_push_check_state(RESET) # Perform the first check with additional libraries set(CMAKE_REQUIRED_LIBRARIES ${CMAKE_DL_LIBS}) include(CheckSymbolExists) check_symbol_exists(dlopen "dlfcn.h" HAVE_DLOPEN) # Save current state cmake_push_check_state() # Perform the second check with libraries and additional compile definitions set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE) check_symbol_exists(dladdr "dlfcn.h" HAVE_DLADDR) message(STATUS "${CMAKE_REQUIRED_DEFINITIONS}") # Output: -D_GNU_SOURCE # Restore the previous state cmake_pop_check_state() message(STATUS "${CMAKE_REQUIRED_DEFINITIONS}") # Output here is empty # Reset variables to prepare for the next check cmake_reset_check_state() # Perform the next check only with additional compile definitions set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE) check_symbol_exists(dl_iterate_phdr "link.h" HAVE_DL_ITERATE_PHDR) # Restore the original state cmake_pop_check_state()：调用 `include()` 命令，加载并执行另一个 CMake 脚本。
