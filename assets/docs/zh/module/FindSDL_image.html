FindSDL_image¶

查找用于将各种格式的图像加载为 SDL（Simple DirectMedia Layer）表面的 SDL_image 库

find_package(SDL_image [<version>] [...])

注意

本模块专门针对 SDL_image 1 版本。从 2.6 版本开始，SDL_image 在使用 CMake 构建时会提供 CMake 软件包配置文件，应使用 find_package(SDL2_image) 来查找。同样地，SDL_image 3 版本可以使用 find_package(SDL3_image) 来查找。这些较新的版本提供了 导入目标 (Imported Targets)，封装了使用需求。有关更多信息，请参考 SDL 官方文档。

结果变量¶

此模块定义了以下变量

SDL_image_FOUND

3.3 版本中新增。

布尔值，指示是否找到了（所请求版本的）SDL_image 库。

SDL_image_VERSION

版本 4.2 中添加。

包含所找到的 SDL_image 版本的人类可读字符串。

SDL_IMAGE_INCLUDE_DIRS

使用 SDL_image 库所需的头文件所在目录。

SDL_IMAGE_LIBRARIES

使用 SDL_image 库进行链接时所需的库。

提示¶

此模块接受以下变量

SDLDIR

环境变量，可用于帮助定位安装在自定义位置的 SDL 库。它应指向配置、构建和安装 SDL 库时使用的安装目标：./configure --prefix=$SDLDIR。

已弃用变量¶

以下变量提供用于向后兼容性

SDL_IMAGE_VERSION_STRING

自 4.2 版本起弃用：请使用 SDL_image_VERSION，其值相同。

包含所找到的 SDL_image 版本的人类可读字符串。

SDL_IMAGE_FOUND

自 4.2 版本起弃用：请使用 SDL_image_FOUND，其值相同。

SDLIMAGE_FOUND

自 2.8.10 版本起弃用：请使用 SDL_image_FOUND，其值相同。

SDLIMAGE_INCLUDE_DIR

自 2.8.10 版本起弃用：请使用 SDL_IMAGE_INCLUDE_DIRS，其值相同。

SDLIMAGE_LIBRARY

自 2.8.10 版本起弃用：请使用 SDL_IMAGE_LIBRARIES，其值相同。

示例¶

查找 SDL_image 库并创建用于将其链接到项目目标的导入接口目标

find_package(SDL_image)

if(SDL_image_FOUND AND NOT TARGET SDL::SDL_image)
  add_library(SDL::SDL_image INTERFACE IMPORTED)
  set_target_properties(
    SDL::SDL_image
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${SDL_IMAGE_INCLUDE_DIRS}"
      INTERFACE_LINK_LIBRARIES "${SDL_IMAGE_LIBRARIES}"
  )
endif()

target_link_libraries(project_target PRIVATE SDL::SDL_image)

使用 SDL_image 2 版本时，上游软件包直接提供 SDL2_image::SDL2_image 导入目标。它可以在项目中直接使用，而无需调用本模块。

find_package(SDL2_image)
target_link_libraries(project_target PRIVATE SDL2_image::SDL2_image)

同样地，对于 SDL_image 3 版本也是如此。

find_package(SDL3_image)
target_link_libraries(project_target PRIVATE SDL3_image::SDL3_image)
