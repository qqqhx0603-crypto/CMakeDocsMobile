FindQt4¶

查找 Qt4，这是一个用于创建图形用户界面和应用程序的跨平台应用程序开发框架。

find_package(Qt4 [<version>] [...])

代码说明

  • find_package(Qt4 [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

此模块定义了多个导入目标、宏和变量，以便在项目中使用 Qt4。

注意

此模块适用于 Qt 4 版本。从 Qt 5 版本开始，Qt 上游也提供了用于查找 Qt 的导出配置。新代码应遵循 cmake-qt(7)，而不是使用此模块。

若要检测 Qt4 软件包，需要 Qt4 的 qmake 工具，并且该工具必须在系统路径中可用。

注意

当使用 导入目标 (Imported Targets) 时，qtmain.lib 静态库会自动链接到 Windows（微软桌面系统） 上的 WIN32 可执行文件中。要全局禁用此功能，请在查找 Qt4 之前设置 QT4_NO_LINK_QTMAIN 变量。若要为特定可执行文件禁用此功能，请在该可执行文件上将 QT4_NO_LINK_QTMAIN 目标属性设置为 TRUE。

Qt 构建工具¶

Qt 依赖一些捆绑工具进行代码生成，例如用于元对象代码生成的 moc、用于窗口小部件布局和填充的 uic，以及用于虚拟文件系统内容生成的 rcc。如果满足适当的条件，这些工具可以由 cmake(1) 自动调用。更多信息请参阅 cmake-qt(7)。

导入的目标¶

Qt 库可以使用其相应的 导入目标 并通过 target_link_libraries() 命令进行链接。

target_link_libraries(myexe Qt4::QtGui Qt4::QtXml)

代码说明

  • target_link_libraries(myexe Qt4::QtGui Qt4::QtXml)：调用 `target_link_libraries()` 命令，为目标链接依赖库。

链接到导入目标时，在构建 myexe 时会自动应用正确的包含目录和编译定义。

导入目标还会管理它们的依赖关系，因此如果另一个 Qt 库依赖于 Qt4::QtCore，则无需显式列出它。同样，在链接 Qt4::QtDeclarative 时会自动包含 Qt4::QtGui。可以使用 if(TARGET) 命令检查目标是否存在。

如果 Qt 工具包库的调试版和发布版都可用，CMake 会根据 构建配置 选择合适的版本。

结果变量¶

此模块定义了以下变量

Qt4_FOUND

布尔值，指示是否找到了（所请求版本的）Qt4。

QT_FOUND

与 Qt4_FOUND 相同。布尔值，指示是否找到了（所请求版本的）Qt4。该变量可用于与其他 Qt 查找模块兼容。

QT_VERSION_MAJOR

找到的 Qt 主版本号。

QT_VERSION_MINOR

找到的 Qt 次版本号。

QT_VERSION_PATCH

找到的 Qt 修补程序版本号。

提示¶

QT4_NO_LINK_QTMAIN

如果在查找 Qt4 之前设置为布尔值 true，则在 Windows（微软桌面系统） 上全局禁用链接 qtmain.lib 静态库。

宏¶

在某些情况下，以更手动的方式调用 Qt 构建工具可能是必要或有用的。此模块提供了以下宏来为此类用途添加目标：

qt4_wrap_cpp¶

为在给定输入文件列表上运行 moc 创建构建规则。

qt4_wrap_cpp(<variable> <files>... [TARGET <target>] [OPTIONS <options>...])

代码说明

  • qt4_wrap_cpp(<variable> <files>... [TARGET <target>] [OPTIONS <options>...])：调用 `qt4_wrap_cpp()` 命令，具体行为由这一行传入的参数决定。

此宏创建用于处理包含带有 Q_OBJECT 声明的 Qt 类的输入文件列表 <files> 的构建规则。还会添加目录级别的预处理器定义。

<variable>

存储生成的输出文件列表的变量名称。

<files>

一个或多个输入源文件。

TARGET

如果指定了 <target>，则来自该目标的 INTERFACE_INCLUDE_DIRECTORIES 和 INTERFACE_COMPILE_DEFINITIONS 属性将被传递给 moc。

OPTIONS

传递给 moc 的可选选项列表，例如执行 moc -help 时显示的选项。

注意

除了使用 qt4_wrap_cpp() 之外，还可以设置 CMAKE_AUTOMOC 变量来自动使用 moc 处理源文件。

qt4_wrap_ui¶

为在给定的 Qt 设计器 ui 输入文件列表上运行 uic 创建构建规则。

qt4_wrap_ui(<variable> <files>... [OPTIONS <options>...])

代码说明

  • qt4_wrap_ui(<variable> <files>... [OPTIONS <options>...])：调用 `qt4_wrap_ui()` 命令，具体行为由这一行传入的参数决定。

<variable>

存储生成的输出文件名列表的变量名称。

<files>

一个或多个 Qt 设计器 ui 输入源文件。

OPTIONS

传递给 uic 的可选选项列表，例如执行 uic -help 时显示的选项。

注意

除了使用 qt4_wrap_ui() 之外，还可以设置 CMAKE_AUTOUIC 变量来自动使用 uic 处理 ui 文件。

qt4_add_resources¶

为在给定的输入 Qt 资源文件列表上运行 rcc 创建构建规则。

qt4_add_resources(<variable> <files>... [OPTIONS <options>...])

代码说明

  • qt4_add_resources(<variable> <files>... [OPTIONS <options>...])：调用 `qt4_add_resources()` 命令，具体行为由这一行传入的参数决定。

<variable>

存储生成的输出文件名列表的变量名称。

<files>

一个或多个 Qt 资源输入源文件。

OPTIONS

传递给 rcc 的可选选项列表，例如执行 rcc -help 时显示的选项。

注意

除了使用 qt4_add_resources() 之外，还可以设置 CMAKE_AUTORCC 变量来自动使用 rcc 处理资源文件。

qt4_generate_moc¶

创建一个通过在给定输入文件上运行 moc 来生成输出文件的构建规则。

qt4_generate_moc(<input-file> <output-file> [TARGET <target>])

代码说明

  • qt4_generate_moc(<input-file> <output-file> [TARGET <target>])：调用 `qt4_generate_moc()` 命令，具体行为由这一行传入的参数决定。

此宏为 <input-file> 创建一个生成 <output-file> 的构建规则。如果由于某种原因无法使用 qt4_wrap_cpp()（例如需要 moc 文件使用自定义文件名等），请使用此宏。

TARGET

如果指定了 <target>，则来自该目标的 INTERFACE_INCLUDE_DIRECTORIES 和 INTERFACE_COMPILE_DEFINITIONS 属性将被传递给 moc。

qt4_add_dbus_interface¶

从接口 XML 文件创建接口头文件和实现文件。

qt4_add_dbus_interface(<variable> <interface-file> <basename>)

代码说明

  • qt4_add_dbus_interface(<variable> <interface-file> <basename>)：调用 `qt4_add_dbus_interface()` 命令，具体行为由这一行传入的参数决定。

此宏从给定的接口 XML 文件 <interface-file> 创建接口头文件（<basename>.h）和实现文件（<basename>.{cpp,moc}），并将其添加到包含源文件列表的变量（指定为变量名 <variable>）中。

通过在输入文件 <interface-file> 上设置以下源文件属性，可以将其他参数传递给 qdbusxml2cpp 调用：

INCLUDE

指定的文件将被包含在生成的接口头文件中。

CLASSNAME

生成的类的名称。

NO_NAMESPACE

生成的类将不会被包装在命名空间中。

qt4_add_dbus_interfaces¶

从多个接口 XML 文件创建接口头文件和实现文件。

qt4_add_dbus_interfaces(<variable> <interface-files>...)

代码说明

  • qt4_add_dbus_interfaces(<variable> <interface-files>...)：调用 `qt4_add_dbus_interfaces()` 命令，具体行为由这一行传入的参数决定。

此宏为所有列出的接口 XML 文件 <interface-files> 创建接口头文件和实现文件。基本名称将根据 XML 文件的名称自动确定。生成的输出文件列表存储在变量 <variable> 中。

qt4_add_dbus_interface() 中描述的源文件属性也适用于此处。

qt4_add_dbus_adaptor¶

为 D-Bus 接口生成适配器类。

qt4_add_dbus_adaptor(<variable> <xmlfile> <parent-header> <parent-classname>
                     [<basename>] [<classname>])

代码说明

  • qt4_add_dbus_adaptor(<variable> <xmlfile> <parent-header> <parent-classname> [<basename>] [<classname>])：调用 `qt4_add_dbus_adaptor()` 命令，具体行为由这一行传入的参数决定。

从描述接口的 XML 文件创建 D-Bus 适配器（头文件和实现文件），并将其添加到源文件列表中。该适配器将调用转发到在 <parent-header> 中定义并命名为 <parent-classname> 的父类。生成的输出文件名将为 <basename>adaptor.{cpp,h}，如果未给出 <basename>，则默认为 XML 文件的基本名称。如果提供了 <classname>，它将用作适配器本身的类名。生成的输出文件名存储在变量 <variable> 中。

qt4_generate_dbus_interface¶

从给定的头文件生成 D-Bus XML 接口文件。

qt4_generate_dbus_interface(<header> [<interface>] [OPTIONS <options>...])

代码说明

  • qt4_generate_dbus_interface(<header> [<interface>] [OPTIONS <options>...])：调用 `qt4_generate_dbus_interface()` 命令，具体行为由这一行传入的参数决定。

此宏创建一个构建规则，用于从给定的 <header> 文件中提取声明，以生成相应的 XML 接口文件。

<header>

从中生成 XML 接口文件的头文件路径。

<interface>

生成的 XML 接口文件的路径。如果省略此可选参数，接口文件的名称将由头文件的基本名称构造，并附加后缀 .xml。相对路径被解释为相对于 CMAKE_CURRENT_BINARY_DIR。

OPTIONS

可以传递给 qdbuscpp2xml 的选项列表，例如执行 qdbuscpp2xml --help 时显示的选项。

qt4_create_translation¶

创建用于生成 TS 和 QM 文件的构建规则。

qt4_create_translation(<qm-files-var> <directories>... <sources>...
                       <ts-files>... [OPTIONS <options>...])

代码说明

  • qt4_create_translation(<qm-files-var> <directories>... <sources>... <ts-files>... [OPTIONS <options>...])：调用 `qt4_create_translation()` 命令，具体行为由这一行传入的参数决定。

此宏创建构建规则，通过 lupdate 从给定的 <directories> 和/或 <sources> 生成 TS（翻译源文件 .ts）文件，并由 lrelease 生成 QM（Qt 消息文件 .qm）文件。TS 文件在源树中创建和/或更新（除非给出完整路径）。QM 文件在构建树中生成。

<qm-files-var>

生成的 QM 文件列表存储在此变量中。可以通过将 <qm-files-var> 添加到项目库/可执行文件的源列表中来更新翻译，从而使它们始终保持更新，或者通过添加自定义目标来控制它们何时更新/生成。

<directories>

包含源文件的目录列表。

<sources>

源文件列表。

<ts-files>

TS（翻译源）文件列表。

OPTIONS

传递给 lupdate 的可选标志列表，例如 -extensions，用于指定目录扫描的文件扩展名。

qt4_add_translation¶

创建从给定 TS 文件生成 QM 文件的构建规则。

qt4_add_translation(<qm-files-var> <ts-files>...)

代码说明

  • qt4_add_translation(<qm-files-var> <ts-files>...)：调用 `qt4_add_translation()` 命令，具体行为由这一行传入的参数决定。

此宏创建从给定 TS 文件生成 QM 文件的构建规则，并将生成的 QM 文件名列表存储在 <qm-files-var> 变量中。<ts-files> 必须存在，且不会以任何方式更新。

qt4_automoc¶

自 2.8.11 版本起已弃用：请改用 CMAKE_AUTOMOC 变量提供的功能。

在输入文件上运行 moc。

qt4_automoc(<source-files>... [TARGET <target>])

代码说明

  • qt4_automoc(<source-files>... [TARGET <target>])：调用 `qt4_automoc()` 命令，具体行为由这一行传入的参数决定。

此宏可用于自动处理 moc。例如，如果有 foo.h 和 foo.cpp 文件，并且在 foo.h 中某个类使用了 Q_OBJECT 预处理器宏，则必须在其上运行 moc。如果不希望使用 qt4_wrap_cpp()（尽管它很可靠且成熟），可以将 #include "foo.moc" 插入到 foo.cpp 中，然后将 foo.cpp 作为参数提供给 qt4_automoc()。这将在配置阶段扫描所有列出的 <source-files> 以查找此类包含的 moc 文件，如果找到它们，则会生成一个规则，在构建时对附带的头文件 foo.h 运行 moc。如果源文件设置了 SKIP_AUTOMOC 属性，则该文件将被此宏忽略。

TARGET

如果指定了 <target>，则来自该目标的 INTERFACE_INCLUDE_DIRECTORIES 和 INTERFACE_COMPILE_DEFINITIONS 属性将被传递给 moc。

qt4_use_modules¶

自 2.8.11 版本起已弃用：请改用带有 导入目标 的 target_link_libraries()。

为项目提供 Qt 模块以链接到目标。

qt4_use_modules(<target> [<LINK_PUBLIC|LINK_PRIVATE>] <modules>...)

代码说明

  • qt4_use_modules(<target> [<LINK_PUBLIC|LINK_PRIVATE>] <modules>...)：调用 `qt4_use_modules()` 命令，具体行为由这一行传入的参数决定。

此函数使 <target> 使用来自 Qt 的 <modules>。使用 Qt 模块意味着链接到该库，添加相关的模块包含目录，并添加用于使用该模块的相关编译器定义。模块大致等同于 Qt4 组件。

LINK_PUBLIC 或 LINK_PRIVATE

可选的链接模式，用作 target_link_libraries() 调用中的相应参数。

例如，调用 qt4_use_modules(myexe Core Gui Declarative) 将在项目目标 myexe 上使用 QtCore、QtGui 和 QtDeclarative 组件。

已弃用变量¶

以下变量提供用于向后兼容性

QT4_FOUND

自 2.8.11 版本起已弃用：请使用具有相同值的 Qt4_FOUND。

示例¶

查找 Qt4 的典型用法可能如下：

set(CMAKE_AUTOMOC ON)
set(CMAKE_INCLUDE_CURRENT_DIR ON)
find_package(Qt4 4.4.3 REQUIRED QtGui QtXml)
add_executable(myexe main.cpp)
target_link_libraries(myexe PRIVATE Qt4::QtGui Qt4::QtXml)

代码说明

  • set(CMAKE_AUTOMOC ON) set(CMAKE_INCLUDE_CURRENT_DIR ON) find_package(Qt4 4.4.3 REQUIRED QtGui QtXml) add_executable(myexe main.cpp) target_link_libraries(myexe PRIVATE Qt4::QtGui Qt4::QtXml)：调用 `set()` 命令，把变量 `CMAKE_AUTOMOC` 设为给定的值。
