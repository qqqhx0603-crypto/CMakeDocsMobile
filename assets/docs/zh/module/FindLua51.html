FindLua51¶

注意

此模块专门用于 Lua 5.1 版本分支，该分支已过时且不再维护。在新代码中使用最新的受支持 Lua 版本和与版本无关的模块 FindLua。

查找 Lua 库

find_package(Lua51 [<version>] [...])

代码说明

  • find_package(Lua51 [<version>] [...])：调用 `find_package()` 命令，查找并加载外部依赖包。

Lua 是一种可嵌入的脚本语言。

使用 Lua 时，其库头文件应包含在项目源代码中，如下所示：

#include <lua.h>

代码说明

  • #include <lua.h>：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

而不是

#include <lua/lua.h>

代码说明

  • #include <lua/lua.h>：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

这是因为 Lua 头文件的位置可能因平台而异，并且可能存在于 lua/ 以外的位置。

结果变量¶

此模块定义了以下变量

Lua51_FOUND

3.3 版本中新增。

布尔值，指示是否找到（请求版本的）Lua。

Lua51_VERSION

版本 4.2 中添加。

找到的 Lua 5.1 的版本。

缓存变量¶

以下缓存变量也可以设置

LUA_INCLUDE_DIR

包含 Lua 头文件（例如 lua.h、lualib.h 和 lauxlib.h）的目录，这些文件是使用 Lua 所必需的。

LUA_LIBRARIES

使用 Lua 时需要链接的库。

已弃用变量¶

以下变量提供用于向后兼容性

LUA51_FOUND

自版本 4.2 起已弃用：使用 Lua51_FOUND，其值相同。

布尔值，指示是否找到（请求版本的）Lua。

LUA_VERSION_STRING

自版本 4.2 起已弃用：使用 Lua51_VERSION，其值相同。

找到的 Lua 5.1 的版本。

示例¶

查找 Lua 5.1 库并创建一个接口 导入目标，封装其用于链接到项目目标的使用要求

find_package(Lua51)

if(Lua51_FOUND AND NOT TARGET Lua51::Lua51)
  add_library(Lua51::Lua51 INTERFACE IMPORTED)
  set_target_properties(
    Lua51::Lua51
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${LUA_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${LUA_LIBRARIES}"
  )
endif()

target_link_libraries(project_target PRIVATE Lua51::Lua51)

代码说明

  • find_package(Lua51) if(Lua51_FOUND AND NOT TARGET Lua51::Lua51) add_library(Lua51::Lua51 INTERFACE IMPORTED) set_target_properties( Lua51::Lua51 PROPERTIES INTERFACE_INCLUDE_DIRECTORIES "${LUA_INCLUDE_DIR}" INTERFACE_LINK_LIBRARIES "${LUA_LIBRARIES}" ) endif() target_link_libraries(project_target PRIVATE Lua51::Lua51)：调用 `find_package()` 命令，查找并加载外部依赖包。
