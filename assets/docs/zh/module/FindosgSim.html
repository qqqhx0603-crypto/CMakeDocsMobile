FindosgSim¶

查找 OpenSceneGraph 工具包中的 osgSim NodeKit。

注意

在大多数情况下，建议改用 FindOpenSceneGraph 模块并将 osgSim 列为组件。这将自动处理 OpenThreads 和核心 osg 库等依赖项。

find_package(OpenSceneGraph COMPONENTS osgSim)

代码说明

  • find_package(OpenSceneGraph COMPONENTS osgSim)：调用 find_package() 命令，查找并加载外部依赖包。

该模块由 FindOpenSceneGraph 内部使用，用于查找 osgSim NodeKit。在通常使用 find_package() 命令时，不建议直接引用该模块。但是，对于需要更精细检测控制的高级使用场景，它也可作为独立模块使用。例如，显式查找 osgSim 或绕过自动组件检测。

find_package(osgSim)

代码说明

  • find_package(osgSim)：调用 find_package() 命令，查找并加载外部依赖包。

OpenSceneGraph 和 osgSim 头文件在 C++ 项目源代码中应按如下方式引用：

example.cxx¶
#include <osg/PositionAttitudeTransform>
#include <osgSim/ImpostorSprite>
// ...

代码说明

  • #include <osg/PositionAttitudeTransform>：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

  • #include <osgSim/ImpostorSprite>：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

  • // ...：这是代码注释。此处省略了与当前要点无关的代码。

使用 OpenSceneGraph 工具包时，可能还需要其他库，例如 OpenGL（开放图形库）。

结果变量¶

此模块定义了以下变量

osgSim_FOUND

3.3 版本中新增。

布尔值，指示是否找到了 OpenSceneGraph 工具包的 osgSim NodeKit。

OSGSIM_LIBRARIES

使用 osgSim 进行链接所需的库。

OSGSIM_LIBRARY

结果变量，其值与 OSGSIM_LIBRARIES 变量相同。

缓存变量¶

以下缓存变量也可以设置

OSGSIM_INCLUDE_DIR

包含使用 osgSim 所需头文件的目录。

OSGSIM_LIBRARY_DEBUG

osgSim 调试库的路径。

提示¶

此模块接受以下变量

OSGDIR

当 OpenSceneGraph 工具包（包括其 osgSim NodeKit）安装在自定义位置时，可设置此环境变量以帮助定位它。它应指向配置、构建和安装时使用的 OpenSceneGraph 安装前缀：./configure --prefix=$OSGDIR。

已弃用变量¶

以下变量提供用于向后兼容性

OSGSIM_FOUND

4.2 版本弃用：请使用 osgSim_FOUND，其值相同。

布尔值，指示是否找到了 OpenSceneGraph 工具包的 osgSim NodeKit。

示例¶

使用此模块显式查找 osgSim，并创建一个 导入目标 (imported target)，该目标封装了将其链接到项目目标所需的使用要求。

find_package(osgSim)

if(osgSim_FOUND AND NOT TARGET osgSim::osgSim)
  add_library(osgSim::osgSim INTERFACE IMPORTED)
  set_target_properties(
    osgSim::osgSim
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${OSGSIM_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${OSGSIM_LIBRARIES}"
  )
endif()

target_link_libraries(example PRIVATE osgSim::osgSim)

代码说明

  • find_package(osgSim)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(osgSim_FOUND AND NOT TARGET osgSim::osgSim)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；AND 表示逻辑与：两侧条件都为真时结果为真；TARGET 表示判断给定构建目标是否已定义。

  • add_library(osgSim::osgSim INTERFACE IMPORTED)：调用 add_library() 命令，定义一个库目标。

  • set_target_properties(：这一行继续补充当前命令或表达式的内容。

  • osgSim::osgSim：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • INTERFACE_INCLUDE_DIRECTORIES "${OSGSIM_INCLUDE_DIR}"：这一行继续补充当前命令或表达式的内容；变量说明：OSGSIM_INCLUDE_DIR：包含目录、目录相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • INTERFACE_LINK_LIBRARIES "${OSGSIM_LIBRARIES}"：这一行继续补充当前命令或表达式的内容；变量说明：OSGSIM_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

  • target_link_libraries(example PRIVATE osgSim::osgSim)：调用 target_link_libraries() 命令，为目标链接依赖库。
