Findosg¶

查找核心 OpenSceneGraph osg 库（libosg）。

注意

在大多数情况下，建议改用 FindOpenSceneGraph 模块，它会自动查找 osg 核心库及其所需的依赖项（如 OpenThreads）。

find_package(OpenSceneGraph)

该模块被 FindOpenSceneGraph 内部用于查找 osg 库。在常规使用 find_package() 命令时，不建议直接包含此模块。但是，它也可以作为一个独立的模块供高级用户使用，以便在需要对检测过程进行更细粒度控制时（例如显式查找 osg 库或绕过自动组件检测）使用。

find_package(osg)

OpenSceneGraph 核心库头文件在 C++ 项目源码中的包含方式如下：

example.cxx¶
#include <osg/PositionAttitudeTransform>

使用 OpenSceneGraph 工具包时，可能还需要其他库，例如 OpenGL。

结果变量¶

此模块定义了以下变量

osg_FOUND

3.3 版本中新增。

指示是否找到 osg 库的布尔值。

OSG_LIBRARIES

使用 osg 库进行链接时所需的库文件。

OSG_LIBRARY

结果变量，其值与 OSG_LIBRARIES 变量相同。

缓存变量¶

以下缓存变量也可以设置

OSG_INCLUDE_DIR

包含使用 osg 库所需头文件的包含目录。

OSG_LIBRARY_DEBUG

osg 调试库的路径。

提示¶

此模块接受以下变量

OSGDIR

当 OpenSceneGraph 工具包安装在自定义位置时，可设置此环境变量以帮助定位其 osg 库。它应该指向配置、构建和安装时所使用的 OpenSceneGraph 安装前缀：./configure --prefix=$OSGDIR。

已弃用变量¶

以下变量提供用于向后兼容性

OSG_FOUND

4.2 版本弃用：请使用 osg_FOUND，其值相同。

指示是否找到 osg 库的布尔值。

示例¶

使用此模块显式查找 osg 库，并创建一个 导入目标 (imported target) 接口，该接口封装了将其链接到项目目标时所需的各项要求。

find_package(osg)

if(osg_FOUND AND NOT TARGET osg::osg)
  add_library(osg::osg INTERFACE IMPORTED)
  set_target_properties(
    osg::osg
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${OSG_INCLUDE_DIR}"
      INTERFACE_LINK_LIBRARIES "${OSG_LIBRARIES}"
  )
endif()

target_link_libraries(example PRIVATE osg::osg)
