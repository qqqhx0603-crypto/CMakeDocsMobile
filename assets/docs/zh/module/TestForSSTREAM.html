TestForSSTREAM¶

此模块检查 C++ 标准头文件 <sstream> 是否存在且功能正常。

在 CMake 项目中加载此模块，使用

include(TestForSSTREAM)

代码说明

  • include(TestForSSTREAM)：调用 include() 命令，加载并执行另一个 CMake 脚本。

在 C++ 的早期版本（C++98 之前），<sstream> 头文件并未正式标准化，可能不可用。

此模块定义了以下缓存变量

CMAKE_NO_ANSI_STRING_STREAM

一个缓存变量，指示 <sstream> 头文件是否可用。如果 <sstream> 可用（C++ 98 及更高版本），则其值为 0；如果 <sstream> 缺失（ANSI C++），则其值为 1。

CMAKE_HAS_ANSI_STRING_STREAM

一个缓存变量，与 CMAKE_NO_ANSI_STRING_STREAM 相反（如果 <sstream> 可用则为 true，如果 <sstream> 缺失则为 false）。

注意

在 C++ 98 标准中正式引入了 <sstream> 头文件，这使得此检查对于现代编译器来说已经过时。

示例¶

包含此模块将检查 <sstream> 支持并定义 CMAKE_NO_ANSI_STRING_STREAM 缓存变量

CMakeLists.txt¶
include(TestForSSTREAM)
file(
  CONFIGURE
  OUTPUT config.h
  CONTENT "#cmakedefine CMAKE_NO_ANSI_STRING_STREAM"
)

代码说明

  • include(TestForSSTREAM)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • file(：这一行继续补充当前命令或表达式的内容。

  • CONFIGURE：这一行继续补充当前命令或表达式的内容。

  • OUTPUT config.h：这一行继续补充当前命令或表达式的内容。

  • CONTENT "#cmakedefine CMAKE_NO_ANSI_STRING_STREAM"：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

然后它可以在 C++ 程序中使用

example.cxx¶
#include "config.h"

#ifndef CMAKE_NO_ANSI_STRING_STREAM
#  include <sstream>
#endif

int main() { ... }

代码说明

  • #include "config.h"：这一行引入头文件，使当前源文件可以使用其中声明的类型、函数或宏。

  • #ifndef CMAKE_NO_ANSI_STRING_STREAM：这一行开始预处理条件分支，按宏条件决定后续代码是否参与编译。

  • # include <sstream>：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文。

  • #endif：这一行切换或结束预处理条件分支。

  • int main() { ... }：这一行补充当前 C/C++ 代码逻辑，具体作用取决于其上下文；变量说明：main：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。
