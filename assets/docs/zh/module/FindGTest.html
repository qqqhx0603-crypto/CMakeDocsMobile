FindGTest¶

查找 GoogleTest，即 Google 的 C++ 测试和模拟框架

find_package(GTest [...])

GoogleTest 框架还包含 GoogleMock，这是一个用于编写和使用 C++ 模拟（mock）类的库。在某些系统上，GoogleMock 可能会作为单独的软件包分发。

当 GoogleTest 和 GoogleMock 库的调试（debug）和发布（release/optimized）版本均可用时，本模块会根据当前的 构建配置 选择相应的变体。

3.20 版本新增: 如果 GoogleTest 是使用其基于 CMake 的构建系统构建并安装的，它会提供一个 软件包配置文件 (GTestConfig.cmake)，该文件可用于 find_package() 的 配置模式。默认情况下，本模块现在会搜索该配置文件，如果找到，则直接返回结果而无需进一步操作。如果未找到上游配置文件，本模块将回退到 模块模式 并搜索标准位置。

导入的目标¶

此模块提供以下 导入目标

GTest::gtest

在 3.20 版本中添加。

封装 GoogleTest gtest 库使用需求的目标，在找到 GoogleTest 时可用。gtest 库提供了 GoogleTest 测试框架的核心功能。

GTest::gtest_main

在 3.20 版本中添加。

封装 GoogleTest gtest_main 库使用需求的目标，在找到 GoogleTest 时可用。gtest_main 库提供了一个 main() 函数，允许无需手动定义即可运行测试。

仅当 GoogleTest 应为可执行文件提供 main() 函数时，才链接到 GTest::gtest_main。如果项目自行提供了 main() 实现，则仅链接到 GTest::gtest。

GTest::gmock

在版本 3.23 中添加。

封装 GoogleMock gmock 库使用需求的目标，在找到 GoogleTest 及其 Mock 库时可用。gmock 库提供了在 C++ 中编写和使用模拟类的工具。

GTest::gmock_main

在版本 3.23 中添加。

封装 GoogleMock gmock_main 库使用需求的目标，在找到 GoogleTest 和 gmock_main 时可用。gmock_main 库提供了一个 main() 函数，允许无需手动定义即可运行 GoogleMock 测试。

仅当 GoogleTest 应为可执行文件提供 main() 函数时，才链接到 GTest::gmock_main。如果项目自行提供了 main() 实现，则仅链接到 GTest::gmock。

结果变量¶

此模块定义了以下变量

GTest_FOUND

3.3 版本中新增。

布尔值，指示是否找到了 GoogleTest。

提示¶

本模块在调用 find_package(GTest) 之前接受以下变量：

GTEST_ROOT

GoogleTest 安装的根目录（也可以设置为环境变量）。此变量仅在以 模块模式 找到 GoogleTest 时使用。

GTEST_MSVC_SEARCH

使用 MSVC 编译时，此变量控制根据运行时库链接模型搜索哪个 GoogleTest 构建变体。此变量仅在以 模块模式 找到 GoogleTest 时使用，并接受以下值之一：

MD

（默认）搜索构建为链接动态 C 运行时的 GoogleTest 共享库变体。这些库通常使用 MSVC 运行时标志 /MD 或 /MDd（分别用于 Release 或 Debug）进行编译。

MT

搜索构建为链接静态 C 运行时的 GoogleTest 静态库变体。这些库通常使用 MSVC 运行时标志 /MT 或 /MTd 进行编译。

已弃用的项目¶

已弃用变量¶

以下变量提供用于向后兼容性

GTEST_INCLUDE_DIRS

4.1 版本弃用: 请改用 GTest::gtest 导入的目标，该目标通过其 INTERFACE_INCLUDE_DIRECTORIES 目标属性公开所需的包含目录。

提供使用 GoogleTest 所需头文件的包含目录的结果变量。仅在以 模块模式 找到 GoogleTest 时，才能保证此变量可用。

GTEST_LIBRARIES

4.1 版本弃用: 请改用 GTest::gtest 导入的目标。

提供链接 GoogleTest gtest 库所需库的结果变量。请注意，除了此变量指定的库外，项目还负责链接合适的线程库。

GTEST_MAIN_LIBRARIES

4.1 版本弃用: 请改用 GTest::gtest_main 导入的目标。

提供链接 GoogleTest gtest_main 库所需库的结果变量。

GTEST_BOTH_LIBRARIES

4.1 版本弃用: 请改用 GTest::gtest 和 GTest::gtest_main 导入的目标。

提供同时包含 gtest 和 gtest_main 库的结果变量。

GTEST_FOUND

4.2 版本弃用: 请使用 GTest_FOUND，其值相同。

布尔值，指示是否找到了 GoogleTest。

已弃用的导入目标¶

为了向后兼容，本模块还提供以下导入的目标（自 CMake 3.5 起可用）：

GTest::GTest

3.20 版本弃用: 请改用 GTest::gtest 导入的目标。

链接 GTest::gtest 库的导入目标。

GTest::Main

3.20 版本弃用: 请改用 GTest::gtest_main 导入的目标。

链接 GTest::gtest_main 库的导入目标。

示例¶

示例：查找 GoogleTest¶

查找 GoogleTest

find_package(GoogleTest)

或者，查找 GoogleTest 并将其设为必需（如果未找到，处理将停止并报错）

find_package(GoogleTest REQUIRED)

示例：使用导入的目标¶

在下面的示例中，GTest::gtest 导入的目标被链接到项目目标，从而能够使用核心 GoogleTest 测试框架

find_package(GTest REQUIRED)

target_link_libraries(foo PRIVATE GTest::gtest)

在下一个示例中，GTest::gtest_main 导入的目标也链接到可执行文件，并注册了一个测试。GTest::gtest_main 库提供了一个 main() 函数，因此无需手动编写。GTest::gtest 库仍然需要被链接，因为测试代码直接使用了 GTest::gtest 提供的功能，而最佳实践是直接链接所使用的库。

enable_testing()

find_package(GTest REQUIRED)

add_executable(foo foo.cc)
target_link_libraries(foo PRIVATE GTest::gtest GTest::gtest_main)

add_test(NAME AllTestsInFoo COMMAND foo)

与 CTest 的深度集成¶

本模块通常与 GoogleTest 模块一起使用，该模块提供了 gtest_discover_tests() 和 gtest_add_tests() 命令，以帮助将 GoogleTest 基础设施与 CTest 集成。

find_package(GTest)
target_link_libraries(example PRIVATE GTest::gtest GTest::gtest_main)

include(GoogleTest)
gtest_discover_tests(example)

# ...

3.9 版本变更: 之前的 CMake 版本在此模块中定义了 gtest_add_tests() 命令。
