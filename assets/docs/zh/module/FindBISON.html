FindBISON¶

查找 Bison 命令行解析器生成器，并提供一个 CMake 命令，用于生成使用 Bison 的自定义构建规则。

find_package(BISON [<version>] [...])

Bison 是一个解析器生成器，它取代了早期的 Yacc（Yet Another Compiler-Compiler）。在类 Unix 系统上，最常见的实现是 GNU Bison。在 Windows 上，如果已安装，此模块会查找与 Windows 兼容的 Bison。

结果变量¶

此模块定义了以下变量

BISON_FOUND

布尔值，指示是否找到了（所请求版本的）Bison。

BISON_VERSION

找到的 Bison 版本。

缓存变量¶

以下缓存变量也可以设置

BISON_EXECUTABLE

bison 命令行程序的路径。

命令¶

如果找到 bison，此模块提供以下命令：

bison_target¶

创建一个自定义构建规则，使用 Bison 从 Yacc 文件生成解析器文件。

bison_target(
  <name>
  <input-yacc-file>
  <output-parser-file>
  [DEFINES_FILE <header>]
  [VERBOSE [<file>]]       # The [<file>] argument is deprecated
  [REPORT_FILE <file>]
  [OPTIONS <options>...]
  [COMPILE_FLAGS <string>] # Deprecated
)

3.14 版本更动：当策略 CMP0088 设置为 NEW 时，bison 将在 CMAKE_CURRENT_BINARY_DIR 目录中运行。

<name>

用作此命令调用标识符的字符串。

<input-yacc-file>

输入 Yacc 源文件 (.y) 的路径。如果作为相对路径给出，它将被解释为相对于当前源目录 (CMAKE_CURRENT_SOURCE_DIR)。

<output-parser-file>

由 Bison 生成的输出解析器文件的路径。如果作为相对路径给出，它将被解释为相对于当前的 Bison 工作目录。

DEFINES_FILE <header>

3.4 版本新增。

默认情况下，Bison 可以生成一个包含令牌列表的头文件。此选项允许指定一个由 Bison 生成的自定义 <header> 文件。如果作为相对路径给出，它将被解释为相对于当前的 Bison 工作目录。

VERBOSE [<file>]

启用详细语法和解析器报告的生成。默认情况下，报告文件在当前的 Bison 工作目录中创建，并命名为 <output-parser-filename>.output。

<file>

3.7 版本弃用：请使用 VERBOSE REPORT_FILE <file>。

指定应将报告文件复制到的路径。此参数仅保留以实现向后兼容性，且仅在 <output-parser-file> 指定为绝对路径时有效。

REPORT_FILE <file>

3.7 版本中新增。

与 VERBOSE 结合使用，用于为报告输出 <file> 指定自定义路径，从而覆盖默认位置。如果作为相对路径给出，它将被解释为相对于当前的 Bison 工作目录。

OPTIONS <options>...

4.0 版本新增。

添加到 bison 命令行中的额外选项，以 分号分隔的列表 形式提供。

COMPILE_FLAGS <string>

4.0 版本弃用：已被 OPTIONS <options>... 取代。

一个以空格分隔的字符串，包含添加到 bison 命令行中的额外选项。分号分隔的列表 在此无效。

命令变量

此命令还定义了以下变量：

BISON_<name>_DEFINED

布尔值，指示该命令是否成功调用。

BISON_<name>_INPUT

输入源文件，是 <input-yacc-file> 的别名。

BISON_<name>_OUTPUT_SOURCE

由 bison 生成的输出解析器文件。

BISON_<name>_OUTPUT_HEADER

由 bison 生成的头文件（如有）。

BISON_<name>_OUTPUTS

由 bison 生成的文件列表，包括输出解析器文件、头文件和报告文件。

BISON_<name>_OPTIONS

4.0 版本新增。

用于 bison 命令的命令行选项列表。

BISON_<name>_COMPILE_FLAGS

4.0 版本弃用：已被具有相同值的 BISON_<name>_OPTIONS 变量取代。

用于 bison 命令的命令行选项列表。

示例¶

示例：查找 Bison¶

查找 Bison

find_package(BISON)

查找具有最低版本要求的 Bison

find_package(BISON 2.1.3)

查找 Bison 并将其设为必需（如果未找到 Bison，处理将停止并报错）

find_package(BISON 2.1.3 REQUIRED)

示例：生成解析器¶

查找 Bison，并将输入 Yacc 源文件 parser.y 添加到构建阶段，由 Bison 处理为 parser.cpp 源文件，并生成头文件 parser.h。

find_package(BISON)

if(BISON_FOUND)
  bison_target(MyParser parser.y parser.cpp DEFINES_FILE parser.h)
endif()

add_executable(Foo main.cpp ${BISON_MyParser_OUTPUTS})

示例：命令行选项¶

可以将额外的命令行选项列表传递给 bison 可执行文件。例如，添加 -Wall 选项以报告所有警告，添加 --no-lines (-l) 以不生成 #line 指令。

find_package(BISON)

if(BISON_FOUND)
  bison_target(MyParser parser.y parser.cpp OPTIONS -Wall --no-lines)
endif()

生成器表达式 可用于 OPTIONS <options>... 参数。例如，仅对 Debug 构建类型添加 --debug (-t) 选项。

find_package(BISON)

if(BISON_FOUND)
  bison_target(MyParser parser.y parser.cpp OPTIONS $<$<CONFIG:Debug>:-t>)
endif()
