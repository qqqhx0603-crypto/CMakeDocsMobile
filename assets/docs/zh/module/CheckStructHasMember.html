CheckStructHasMember¶

该模块提供了一个用于检查结构体或类是否具有指定成员变量的命令。

在 CMake 项目中加载此模块，使用

include(CheckStructHasMember)

代码说明

  • include(CheckStructHasMember)：调用 include() 命令，加载并执行另一个 CMake 脚本。

命令¶

此模块提供以下命令

check_struct_has_member¶

一次性检查给定的结构体或类是否具有指定的成员变量

check_struct_has_member(
  <struct>
  <member>
  <headers>
  <variable>
  [LANGUAGE <language>]
)

代码说明

  • check_struct_has_member(：这一行继续补充当前命令或表达式的内容。

  • <struct>：这一行继续补充当前命令或表达式的内容。

  • <member>：这一行继续补充当前命令或表达式的内容。

  • <headers>：这一行继续补充当前命令或表达式的内容。

  • <variable>：这一行继续补充当前命令或表达式的内容。

  • [LANGUAGE <language>]：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

该命令用于在包含给定的头文件 <headers>（应在其中声明原型）后，检查结构体或类 <struct> 是否包含指定的 <member>。多个头文件可以在一个参数中以字符串形式指定，并使用 分号分隔列表。检查结果存储在内部缓存变量 <variable> 中。

选项包括

LANGUAGE <language>

使用 <language> 编译器执行检查。可接受的值为 C 和 CXX。如果未指定，则默认为 C。

影响检查的变量

在调用此命令之前，可以设置以下变量来修改检查的运行方式

CMAKE_REQUIRED_FLAGS

一个以空格分隔的字符串，包含要传递给编译器的附加标志。使用 分号分隔列表 是无效的。CMAKE_<LANG>_FLAGS 及其关联的特定配置变量 CMAKE_<LANG>_FLAGS_<CONFIG> 的内容会自动在编译命令中此变量的内容之前被预置。

CMAKE_REQUIRED_DEFINITIONS

分号分隔的编译器定义列表，每个定义的形式为 -DFOO 或 -DFOO=bar。还将自动添加对检查命令的 result variable 参数指定的名称的定义。

CMAKE_REQUIRED_INCLUDES

要传递给编译器的头文件搜索路径的分号分隔列表。这些将是唯一使用的头文件搜索路径；将忽略 INCLUDE_DIRECTORIES 目录属性的内容。

CMAKE_REQUIRED_LINK_OPTIONS

3.14 版新增。

要添加到链接命令的选项的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LIBRARIES

要添加到链接命令的库的分号分隔列表。这些可以是系统库的名称，也可以是 导入的目标（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_LINK_DIRECTORIES

在版本 3.31 中添加。

要传递给链接器的库搜索路径的分号分隔列表（有关更多详细信息，请参阅 try_compile()）。

CMAKE_REQUIRED_QUIET

版本 3.1 中新增。

如果此变量评估为布尔真值，则与检查关联的所有状态消息都将被抑制。

示例¶

示例：检查 C 结构体成员¶

在下面的示例中，该模块在包含 <sys/select.h> 头文件后，检查 C 结构体 timeval 是否具有成员变量 tv_sec。检查结果存储在内部缓存变量 HAVE_TIMEVAL_TV_SEC 中。

include(CheckStructHasMember)

check_struct_has_member(
  "struct timeval"
  tv_sec
  sys/select.h
  HAVE_TIMEVAL_TV_SEC
)

代码说明

  • include(CheckStructHasMember)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • check_struct_has_member(：这一行继续补充当前命令或表达式的内容。

  • "struct timeval"：这一行继续补充当前命令或表达式的内容。

  • tv_sec：这一行继续补充当前命令或表达式的内容。

  • sys/select.h：这一行继续补充当前命令或表达式的内容。

  • HAVE_TIMEVAL_TV_SEC：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

示例：使用编译定义进行隔离检查¶

在下面的示例中，使用 CMakePushCheckState 模块，在临时修改的编译定义下执行检查。

include(CheckStructHasMember)
include(CMakePushCheckState)

cmake_push_check_state(RESET)
  set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE)

  check_struct_has_member(
    "struct utsname"
    domainname
    sys/utsname.h
    HAVE_UTSNAME_DOMAINNAME
  )
cmake_pop_check_state()

代码说明

  • include(CheckStructHasMember)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • include(CMakePushCheckState)：调用 include() 命令，加载并执行另一个 CMake 脚本。

  • cmake_push_check_state(RESET)：调用 cmake_push_check_state() 命令，具体行为由这一行传入的参数决定。

  • set(CMAKE_REQUIRED_DEFINITIONS -D_GNU_SOURCE)：调用 set() 命令，把变量 CMAKE_REQUIRED_DEFINITIONS 设为给定的值；变量说明：CMAKE_REQUIRED_DEFINITIONS：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • check_struct_has_member(：这一行继续补充当前命令或表达式的内容。

  • "struct utsname"：这一行继续补充当前命令或表达式的内容。

  • domainname：这一行继续补充当前命令或表达式的内容。

  • sys/utsname.h：这一行继续补充当前命令或表达式的内容。

  • HAVE_UTSNAME_DOMAINNAME：这一行继续补充当前命令或表达式的内容。

  • )：这一行继续补充当前命令或表达式的内容。

  • cmake_pop_check_state()：调用 cmake_pop_check_state() 命令，具体行为由这一行传入的参数决定。
