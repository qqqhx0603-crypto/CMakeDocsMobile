该模块提供了用于生成启用/禁用功能摘要的命令。

使用 CMake 加载此模块

include(FeatureSummary)

代码说明

  • include(FeatureSummary)：调用 `include()` 命令，加载并执行另一个 CMake 脚本。

此模块提供的命令可用于为构建树生成已启用和已禁用的包和/或功能的摘要，例如

-- The following features have been enabled:

 * Example, usage example

-- The following OPTIONAL packages have been found:

 * LibXml2 (required version >= 2.4), XML library, <http://xmlsoft.org>
   Enables HTML-import in MyWordProcessor
   Enables odt-export in MyWordProcessor
 * PNG, image library, <http://www.libpng.org/pub/png/>
   Enables saving screenshots

-- The following OPTIONAL packages have not been found:

 * Lua, the Lua scripting language, <https://lua.ac.cn>
   Enables macros in MyWordProcessor
 * OpenGL, Open Graphics Library

全局属性¶

此模块使用以下全局属性

FeatureSummary_PKG_TYPES¶

版本 3.8 新增。

此全局属性定义了一个 分号分隔的列表，包含 FeatureSummary 模块使用的包类型。

此列表中的顺序很重要，列表中第一个包类型的重要性最低，最后一个最高。包类型只能更改为重要性更高的类型。

默认包类型为 RUNTIME、OPTIONAL、RECOMMENDED 和 REQUIRED，其重要性排序为 RUNTIME < OPTIONAL < RECOMMENDED < REQUIRED。

FeatureSummary_REQUIRED_PKG_TYPES¶

版本 3.8 新增。

此全局属性定义了一个 分号分隔的列表，包含被视为必需的包类型。

如果这些类别中的一个或多个包未找到，且 feature_summary() 命令以启用的 FATAL_ON_MISSING_REQUIRED_PACKAGES 选项调用，CMake 将中止。

此全局属性的默认值为 REQUIRED。

FeatureSummary_DEFAULT_PKG_TYPE¶

版本 3.8 新增。

此全局属性定义了默认的包类型。

当调用 feature_summary() 命令，且用户未显式设置某个包的类型时，其类型将被设置为该属性的值。

此值必须是 FeatureSummary_PKG_TYPES 全局属性中定义的类型之一。

此全局属性的默认值为 OPTIONAL。

FeatureSummary_<TYPE>_DESCRIPTION¶

版本 3.9 中添加。

可以为每个包 <TYPE> 定义此全局属性，其字符串将用于 feature_summary() 命令的输出标题中。例如

The following <FeatureSummary_<TYPE>_DESCRIPTION> have been found:

代码说明

  • The following <FeatureSummary_<TYPE>_DESCRIPTION> have been found:：这一行给出示例文本或输出，用来说明：The following <FeatureSummary_<TYPE>_DESCRIPTION> have been found:

如果未设置，则使用默认字符串 <TYPE> packages。

打印功能摘要¶

feature_summary¶

打印有关项目已启用或已禁用包和功能的信息

feature_summary(
  WHAT (ALL
    | PACKAGES_FOUND | PACKAGES_NOT_FOUND
    | <TYPE>_PACKAGES_FOUND | <TYPE>_PACKAGES_NOT_FOUND
    | ENABLED_FEATURES | DISABLED_FEATURES)
  [FILENAME <file>]
  [APPEND]
  [VAR <variable-name>]
  [INCLUDE_QUIET_PACKAGES]
  [FATAL_ON_MISSING_REQUIRED_PACKAGES]
  [DESCRIPTION <description> | DEFAULT_DESCRIPTION]
  [QUIET_ON_EMPTY]
)

代码说明

  • feature_summary( WHAT (ALL | PACKAGES_FOUND | PACKAGES_NOT_FOUND | <TYPE>_PACKAGES_FOUND | <TYPE>_PACKAGES_NOT_FOUND | ENABLED_FEATURES | DISABLED_FEATURES) [FILENAME <file>] [APPEND] [VAR <variable-name>] [INCLUDE_QUIET_PACKAGES] [FATAL_ON_MISSING_REQUIRED_PACKAGES] [DESCRIPTION <description> | DEFAULT_DESCRIPTION] [QUIET_ON_EMPTY] )：调用 `feature_summary()` 命令，具体行为由这一行传入的参数决定。

此命令可用于打印有关项目已启用或已禁用包和功能的信息。默认情况下，仅打印功能/包的名称及其指定时的所需版本。使用 set_package_properties() 添加更有用的信息，例如各包的主页 URL 或其在项目中的用途。

参数为

WHAT

这是唯一必需的选项。它指定将打印哪些信息

ALL

打印所有内容。

ENABLED_FEATURES

所有已启用功能的列表。

DISABLED_FEATURES

所有已禁用功能的列表。

PACKAGES_FOUND

所有已找到包的列表。

PACKAGES_NOT_FOUND

所有未找到包的列表。

对于由 FeatureSummary_PKG_TYPES 全局属性定义的每个包类型 <TYPE>，还可以使用以下信息

<TYPE>_PACKAGES_FOUND

仅已找到的类型为 <TYPE> 的包列表。

<TYPE>_PACKAGES_NOT_FOUND

仅未找到的类型为 <TYPE> 的包列表。

3.1 版本变更: WHAT 选项现在是多值关键字，因此可以组合这些值（ALL 值除外）以自定义输出。例如

feature_summary(WHAT ENABLED_FEATURES DISABLED_FEATURES)

代码说明

  • feature_summary(WHAT ENABLED_FEATURES DISABLED_FEATURES)：调用 `feature_summary()` 命令，具体行为由这一行传入的参数决定。

FILENAME <file>

如果提供了此选项，信息将打印到此文件而不是终端。相对 <file> 路径被解释为相对于当前源目录（即 CMAKE_CURRENT_SOURCE_DIR）。

APPEND

如果提供了此选项，输出将追加到 FILENAME 选项提供的 <file> 中，否则如果文件已存在，它将被覆盖。

VAR <variable-name>

如果提供了此选项，信息将存储在指定的变量 <variable-name> 中，而不是终端。

INCLUDE_QUIET_PACKAGES

如果提供了此选项，使用 find_package(... QUIET) 搜索的包也将被列出。默认情况下它们会被跳过。

FATAL_ON_MISSING_REQUIRED_PACKAGES

如果提供了此选项，如果标记为 FeatureSummary_REQUIRED_PKG_TYPES 全局属性中列出的包类型之一的包未找到，CMake 将中止并报错。

DESCRIPTION <description>

将打印在实际内容上方的描述或标题。如果没有此选项，如果仅请求了一种包类型，则不会打印标题，除非使用此选项显式设置自定义字符串，或者使用输出所请求类型默认标题的 DEFAULT_DESCRIPTION 选项。

DEFAULT_DESCRIPTION

版本 3.9 中添加。

打印在内容上方的默认描述或标题，而不是可自定义的 DESCRIPTION <description>。

QUIET_ON_EMPTY

版本 3.8 新增。

如果提供了此选项，当仅请求一种包类型且未找到属于该类别的包时，则不会打印任何输出（包括 DESCRIPTION），也不会将其添加到 FILENAME 或 VAR 变量中。

包属性¶

set_package_properties¶

设置包属性

set_package_properties(
  <PackageName>
  PROPERTIES
    [URL <url>]
    [DESCRIPTION <description>]
    [TYPE (RUNTIME|OPTIONAL|RECOMMENDED|REQUIRED)]
    [PURPOSE <purpose>]
)

代码说明

  • set_package_properties( <PackageName> PROPERTIES [URL <url>] [DESCRIPTION <description>] [TYPE (RUNTIME|OPTIONAL|RECOMMENDED|REQUIRED)] [PURPOSE <purpose>] )：调用 `set_package_properties()` 命令，具体行为由这一行传入的参数决定。

使用此命令配置并提供有关名为 <PackageName> 的包的信息，然后可以使用 feature_summary() 命令显示这些信息。此命令可以直接在相应的 Find 模块 中调用，也可以在调用 find_package() 后在项目中使用该模块时调用。可以设置信息的特性在 find_package() 命令之后自动确定。

参数为

<PackageName>

包的名称。例如，如 find_package(<PackageName>) 参数中指定的。

PROPERTIES

指定要设置的属性

URL <url>

这应该是包的主页或类似内容。理想情况下，这已经在 Find 模块 中直接设置了。

DESCRIPTION <description>

简要描述该包的内容，最多一句话。理想情况下，这已经在 Find 模块 中直接设置了。

TYPE <type>

使用该项目的项目对该包有什么类型的依赖。

默认的 <type> 是 OPTIONAL。在这种情况下，这是一个在构建时可用时可由项目使用的包，但项目在没有它的情况下也能工作。

RECOMMENDED 包类型类似于 OPTIONAL，即如果该包不存在，项目仍然会构建，但生成的二进制文件的功能将受到严重限制。如果 REQUIRED 包在构建时不可用，项目甚至可能无法构建。这可以与 feature_summary(FATAL_ON_MISSING_REQUIRED_PACKAGES) 命令选项结合使用。

最后，RUNTIME 包是在构建过程中完全不使用，但运行生成的二进制文件时必需的包。因此，如果缺少此类包，项目仍然可以构建，但以后可能无法工作。

如果多次为同一个包调用 set_package_properties() 且具有不同的 TYPE，则 TYPE 仅更改为更高的 TYPE（RUNTIME < OPTIONAL < RECOMMENDED < REQUIRED），较低的 TYPE 会被忽略。TYPE 属性是项目特定的，因此不能由 Find 模块 设置，必须在项目中设置。

可以通过设置 FeatureSummary_PKG_TYPES 全局属性来更改接受的类型。

PURPOSE <purpose>

这描述了该包在项目中启用了哪些功能，即它告诉用户在生成的二进制文件中获得了什么功能。如果为包多次调用 set_package_properties()，则所有 PURPOSE 属性都会附加到项目中包的用途列表中。与 TYPE 属性一样，PURPOSE 属性也是项目特定的，因此不能由 Find 模块 设置，必须在项目中设置。

添加功能信息¶

add_feature_info¶

添加功能信息

add_feature_info(<name> <condition> <description>)

代码说明

  • add_feature_info(<name> <condition> <description>)：调用 `add_feature_info()` 命令，具体行为由这一行传入的参数决定。

使用此命令添加有关由给定 <name> 标识的功能的信息。

参数为

<name>

所添加功能的标识名称。

<condition>

指定确定该功能是启用还是禁用的条件。

<condition> 参数可以是

  • 单个条件（例如变量名）。

  • 3.8 版本新增: 多个条件的 分号分隔的列表。

  • 4.0 版本新增: if(<condition>) 子句中使用的完整 条件语法。请参阅策略 CMP0183。这启用了使用完整条件语法（例如用括号对条件分组等）。

<description>

描述该功能的文本。这些信息可以使用 feature_summary() 分别为 ENABLED_FEATURES 和 DISABLED_FEATURES 显示。

示例：将功能摘要追加到文件¶

在以下示例中，功能摘要输出将追加到指定文件，而不是打印

include(FeatureSummary)
feature_summary(WHAT ALL FILENAME ${CMAKE_BINARY_DIR}/all.log APPEND)

示例：将功能摘要存储在变量中¶

在以下示例中，已启用功能的功能摘要存储在指定的变量 enabledFeaturesText 中，包括 QUIET 包

include(FeatureSummary)

  DESCRIPTION "Enabled Features:"
  VAR enabledFeaturesText
)

message(STATUS "${enabledFeaturesText}")

示例：添加自定义包类型¶

在以下示例中，添加了一个自定义包类型，并仅打印非空的类别

set_package_properties(FOO PROPERTIES TYPE BUILD)

  WHAT BUILD_PACKAGES_FOUND
  DESCRIPTION "Build tools found:"
  QUIET_ON_EMPTY
)

feature_summary(
  WHAT BUILD_PACKAGES_NOT_FOUND
  DESCRIPTION "Build tools not found:"
  QUIET_ON_EMPTY
)

示例：打印功能摘要¶

在以下示例中，此模块用于在配置结束时输出功能摘要。如果未找到任何必需的包，处理将在配置阶段结束时停止并显示错误消息。

cmake_minimum_required(VERSION 3.15)
project(Example)

add_library(example example.c)

include(FeatureSummary)

find_package(CURL)
set_package_properties(CURL PROPERTIES TYPE REQUIRED)
target_link_libraries(example PRIVATE CURL::libcurl)

find_package(LibXml2 QUIET)
set_package_properties(LibXml2 PROPERTIES TYPE RECOMMENDED)
if(LibXml2_FOUND)
  target_link_libraries(example PRIVATE LibXml2::LibXml2)
endif()

feature_summary(
  WHAT ALL
  INCLUDE_QUIET_PACKAGES
  DESCRIPTION "Feature summary:"
  FATAL_ON_MISSING_REQUIRED_PACKAGES
)

代码说明

  • cmake_minimum_required(VERSION 3.15) project(Example) add_library(example example.c) include(FeatureSummary) find_package(CURL) set_package_properties(CURL PROPERTIES TYPE REQUIRED) target_link_libraries(example PRIVATE CURL::libcurl) find_package(LibXml2 QUIET) set_package_properties(LibXml2 PROPERTIES TYPE RECOMMENDED) if(LibXml2_FOUND) target_link_libraries(example PRIVATE LibXml2::LibXml2) endif() feature_summary( WHAT ALL INCLUDE_QUIET_PACKAGES DESCRIPTION "Feature summary:" FATAL_ON_MISSING_REQUIRED_PACKAGES )：调用 `cmake_minimum_required()` 命令，声明工程所需的最低 CMake 版本。
