FindRuby¶

查找 Ruby 安装及其实际包含文件和库的位置

find_package(Ruby [<version>] [COMPONENTS <components>...] [...])

代码说明

  • find_package(Ruby [<version>] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

Ruby 是一种通用编程语言。本模块支持 Ruby 2.0 到 4.0 版本。同时也支持诸如 RVM 或 RBENV 之类的虚拟环境。

组件¶

4.3 版本新增。

此模块支持以下组件

Interpreter

Ruby 解释器可执行程序。

Development

构建 Ruby 扩展或嵌入 Ruby 所需的头文件和库。

如果未指定组件，则会搜索 Interpreter 和 Development。

导入的目标¶

4.3 版本新增。

本模块定义了以下 IMPORTED 目标

Ruby::Interpreter

Ruby 解释器。若找到 Interpreter 组件，则定义此目标。

Ruby::Ruby

用于在 C/C++ 应用程序中嵌入 Ruby 的 Ruby 库。若找到 Development 组件，则定义此目标。

Ruby::Module

用于构建 Ruby 扩展模块的 Ruby 库。若找到 Development 组件，则定义此目标。当创建将通过 require 加载到 Ruby 中的原生扩展时，请使用此目标。在大多数平台上，扩展模块不会直接链接到 libruby。包含适当的符号可见性设置。

结果变量¶

此模块定义了以下变量

Ruby_FOUND

3.3 版本中新增。

布尔值，表示是否找到（所请求版本的）ruby。

Ruby_VERSION

找到的 ruby 版本，例如 3.2.6。

Ruby_VERSION_MAJOR

Ruby 主版本号。

Ruby_VERSION_MINOR

Ruby 次版本号。

Ruby_VERSION_PATCH

Ruby 修订版本号。

Ruby_EXECUTABLE

ruby 二进制文件的完整路径。

Ruby_INCLUDE_DIRS

使用 ruby 库时所需的包含目录。

Ruby_LIBRARIES

在 3.18 版本中新增。

从 C 使用 ruby 所需的库。

3.18 版本变更：之前的 CMake 版本对所有变量都使用了 RUBY_ 前缀。

提示¶

此模块接受以下变量

Ruby_FIND_VIRTUALENV

在 3.18 版本中新增。

此变量定义了对虚拟环境的处理方式。它可以留空，也可以设置为以下值之一：

  • FIRST：优先搜索虚拟 Ruby 环境，

    然后再搜索系统 Ruby 安装。这是默认设置。

  • ONLY：仅搜索虚拟环境

  • STANDARD：仅搜索系统 Ruby 安装。

虚拟环境可能由以下方式提供：

rvm

要求已定义 MY_RUBY_HOME 环境变量。

rbenv

要求 rbenv 安装在 ~/.rbenv/bin 中，或已定义 RBENV_ROOT 环境变量。

已弃用变量¶

以下变量提供用于向后兼容性

4.0 版本弃用：以下变量已弃用。参见策略 CMP0185。

RUBY_FOUND

同 Ruby_FOUND。

RUBY_VERSION

同 Ruby_VERSION。

RUBY_EXECUTABLE

同 Ruby_EXECUTABLE。

RUBY_INCLUDE_DIRS

同 Ruby_INCLUDE_DIRS。

RUBY_INCLUDE_PATH

同 Ruby_INCLUDE_DIRS。

RUBY_LIBRARY

同 Ruby_LIBRARY。

示例¶

查找 Ruby 并指定所需的最低版本

find_package(Ruby 3.2.6 EXACT REQUIRED)
# or
find_package(Ruby 3.2)

代码说明

  • find_package(Ruby 3.2.6 EXACT REQUIRED)：调用 find_package() 命令，查找并加载外部依赖包。

  • # or：这是 CMake 注释。下面给出另一种可选写法。

  • find_package(Ruby 3.2)：调用 find_package() 命令，查找并加载外部依赖包。
