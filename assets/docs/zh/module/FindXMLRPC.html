FindXMLRPC¶

查找用于 C 和 C++ 的原生 XML-RPC 库

find_package(XMLRPC [...] [COMPONENTS <components>...] [...])

代码说明

  • find_package(XMLRPC [...] [COMPONENTS <components>...] [...])：调用 find_package() 命令，查找并加载外部依赖包。

XML-RPC 是一种标准的网络协议，它能够在系统之间启用远程过程调用 (RPC)。它以 XML 编码请求和响应，并使用 HTTP 作为传输机制。

组件¶

XML-RPC C/C++ 库包含各种特性（模块），提供特定的功能。这些特性的可用性取决于已安装的 XML-RPC 库版本和系统配置。某些特性也依赖于其他特性。

要列出系统上可用的特性，可以使用命令行实用程序 xmlrpc-c-config。

在 CMake 中，可以使用 find_package() 命令将这些特性指定为组件

find_package(XMLRPC [COMPONENTS <components>...])

代码说明

  • find_package(XMLRPC [COMPONENTS <components>...])：调用 find_package() 命令，查找并加载外部依赖包。

组件可能包括

c++2

C++ 包装器 API，取代了旧版 c++ 特性。

c++

旧版 C++ 包装器 API（已被 c++2 取代）。

client

XML-RPC 客户端函数（也作为旧版基于 libwww 的特性 libwww-client 提供）。

cgi-server

基于 CGI 的服务器函数。

abyss-server

基于 Abyss 的服务器函数。

pstream-server

基于 pstream 的服务器函数。

server-util

基本服务器函数（它们会自动包含在 *-server 特性中）。

abyss

Abyss HTTP 服务器（不需要 abyss-server）。

openssl

OpenSSL 便利函数。

如果未指定任何组件，则此模块将搜索 XML-RPC 库及其包含目录，而无需额外特性。

结果变量¶

此模块定义了以下变量

XMLRPC_FOUND

布尔值，指示是否找到了 XML-RPC 库及其所有请求的组件。

XMLRPC_INCLUDE_DIRS

包含 xmlrpc.h 和使用 XML-RPC 库所需的其他头文件的包含目录。

XMLRPC_LIBRARIES

链接到 XML-RPC 库及其请求的特性所需的库列表。

示例¶

查找 XML-RPC 库及其 client 特性，并有条件地创建一个 导入目标，该目标封装了其用于链接到项目目标的用法要求

find_package(XMLRPC REQUIRED COMPONENTS client)

if(XMLRPC_FOUND AND NOT TARGET XMLRPC::XMLRPC)
  add_library(XMLRPC::XMLRPC INTERFACE IMPORTED)
  set_target_properties(
    XMLRPC::XMLRPC
    PROPERTIES
      INTERFACE_INCLUDE_DIRECTORIES "${XMLRPC_INCLUDE_DIRS}"
      INTERFACE_LINK_LIBRARIES "${XMLRPC_LIBRARIES}"
  )
endif()

target_link_libraries(example PRIVATE XMLRPC::XMLRPC)

代码说明

  • find_package(XMLRPC REQUIRED COMPONENTS client)：调用 find_package() 命令，查找并加载外部依赖包。

  • if(XMLRPC_FOUND AND NOT TARGET XMLRPC::XMLRPC)：开始一个 if() 条件判断，根据条件结果选择是否执行后续语句；条件运算符：NOT 表示逻辑非：反转后续条件的真假值；AND 表示逻辑与：两侧条件都为真时结果为真；TARGET 表示判断给定构建目标是否已定义。

  • add_library(XMLRPC::XMLRPC INTERFACE IMPORTED)：调用 add_library() 命令，定义一个库目标。

  • set_target_properties(：这一行继续补充当前命令或表达式的内容。

  • XMLRPC::XMLRPC：这一行继续补充当前命令或表达式的内容。

  • PROPERTIES：这一行继续补充当前命令或表达式的内容。

  • INTERFACE_INCLUDE_DIRECTORIES "${XMLRPC_INCLUDE_DIRS}"：这一行继续补充当前命令或表达式的内容；变量说明：XMLRPC_INCLUDE_DIRS：包含目录相关变量，其精确取值和作用域以当前页面或脚本上下文为准。

  • INTERFACE_LINK_LIBRARIES "${XMLRPC_LIBRARIES}"：这一行继续补充当前命令或表达式的内容；变量说明：XMLRPC_LIBRARIES：当前示例使用的变量，其值由前面的命令、调用参数或运行环境提供。

  • )：这一行继续补充当前命令或表达式的内容。

  • endif()：这一行结束或切换上一段控制结构的分支。

  • target_link_libraries(example PRIVATE XMLRPC::XMLRPC)：调用 target_link_libraries() 命令，为目标链接依赖库。
