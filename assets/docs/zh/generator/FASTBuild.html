FASTBuild¶

版本 4.2 中添加。

生成一个 fbuild.bff 文件，可用于使用 FASTBuild 构建项目。

用法¶

在调用 cmake(1) 时指定生成器

cmake [<options>] -G FASTBuild -B <path-to-build> [-S <path-to-source>]

这会将一个名为 fbuild.bff 的 FASTBuild 配置文件写入 <path-to-build>。

注意

该生成器还会为 Visual Studio 和 Xcode 生成 IDE 项目文件，这些文件位于

  • <path-to-build>/VisualStudio – Visual Studio 解决方案和项目

  • <path-to-build>/XCode – Xcode 工作区和项目

这些 IDE 文件可以通过构建 xcode 或 solution 目标来生成，并将使用 FASTBuild 作为后端进行构建。

缓存 (Caching)¶

要启用 FASTBuild 缓存，请使用 CMAKE_FASTBUILD_CACHE_PATH 变量或 FASTBUILD_CACHE_PATH 环境变量设置缓存路径。

然后使用 -cache 标志进行构建

cmake --build <path-to-build> -- -cache

编译器行为变量 (Compiler Behavior Variables)¶

以下变量控制在生成的 fbuild.bff 中如何发出编译器节点。这些设置可能会影响构建确定性、调试信息路径、包含处理以及编译器参数格式化

  • CMAKE_FASTBUILD_ALLOW_RESPONSE_FILE

  • CMAKE_FASTBUILD_CLANG_GCC_UPDATE_XLANG_ARG

  • CMAKE_FASTBUILD_CLANG_REWRITE_INCLUDES

  • CMAKE_FASTBUILD_COMPILER_EXTRA_FILES

  • CMAKE_FASTBUILD_FORCE_RESPONSE_FILE

  • CMAKE_FASTBUILD_SOURCE_MAPPING

  • CMAKE_FASTBUILD_USE_DETERMINISTIC_PATHS

  • CMAKE_FASTBUILD_USE_LIGHTCACHE

  • CMAKE_FASTBUILD_USE_RELATIVE_PATHS

配置变量 (Configuration Variables)¶

以下变量可用于配置此生成器

  • CMAKE_FASTBUILD_CACHE_PATH

  • CMAKE_FASTBUILD_CAPTURE_SYSTEM_ENV

  • CMAKE_FASTBUILD_ENV_OVERRIDES

  • CMAKE_FASTBUILD_IDE_ARGS

  • CMAKE_FASTBUILD_TRACK_BYPRODUCTS_AS_OUTPUT

  • CMAKE_FASTBUILD_VERBOSE_GENERATOR

目标属性 (Target Properties)¶

以下目标属性可用于针对每个目标微调行为

  • FASTBUILD_CACHING – 为特定目标禁用缓存。

  • FASTBUILD_DISTRIBUTION – 为特定目标禁用分布式编译。

注意事项 (Notes)¶

  • 此生成器不支持将目录作为自定义命令的输出。如果您确实指定了一个目录作为输出，则应将其标记为 SYMBOLIC 属性，以避免出现不正确的行为。

  • 强烈建议自定义命令仅使用一个输出。如果指定了多个输出，生成器将发出一条附加规则在构建时检查这些输出。这会增加开销，但这是必要的，因为 FASTBuild 本身只支持每个自定义步骤一个输出。

示例¶

cmake [<options>] -G FASTBuild -B <path-to-build> -DCMAKE_BUILD_TYPE=Release
cmake --build <path-to-build> --target my_app

这会在 <path-to-build> 中生成 fbuild.bff，并使用 FASTBuild 构建 my_app 目标。
