Visual Studio（集成开发环境） 15 2017¶

版本 3.7.1 中新增。

生成 Visual Studio（集成开发环境） 15 (VS 2017) 项目文件。

项目类型¶

只能生成 Visual C++ 和 C# 项目（以及与 Intel 编译器集成的 Fortran）。不支持其他类型的项目（JavaScript、Powershell（命令解释器）、Python 等）。

实例选择¶

3.11 版本新增。

VS 2017 支持在同一台机器上进行多次安装。可以使用 CMAKE_GENERATOR_INSTANCE 变量来选择其中一个。

平台选择¶

默认的目标平台名称（架构）为 Win32。

可以设置 CMAKE_GENERATOR_PLATFORM 变量，也许可以通过 cmake -A 选项来指定目标平台名称 (架构)。例如：

  • cmake -G "Visual Studio 15 2017" -A Win32

  • cmake -G "Visual Studio 15 2017" -A x64

  • cmake -G "Visual Studio 15 2017" -A ARM

  • cmake -G "Visual Studio 15 2017" -A ARM64

版本 4.0 变更：此前，为了兼容 3.1 之前的 CMake 版本，可以在生成器名称末尾可选地指定目标平台名称。该方式已被移除。此方式仅支持

Visual Studio 15 2017 Win64

指定目标平台 x64。

Visual Studio 15 2017 ARM

指定目标平台 ARM。

工具链选择¶

默认选择 Visual Studio（集成开发环境） 15 2017 附带的 v141 工具集。可以通过 cmake -T 选项设置 CMAKE_GENERATOR_TOOLSET 选项，以指定其他工具集。

版本 3.8 新增：对于此版本 Visual Studio（集成开发环境） 附带的每个工具集，都有专门为 32 位 (x86) 和 64 位 (x64) 主机编译的变体（与它们的目标架构无关）。默认情况下，即使在 64 位主机上，此生成器也会使用 32 位变体。可以通过在工具集规范中添加 host=x86 或 host=x64 来显式请求使用 32 位或 64 位主机工具。详情请参阅 CMAKE_GENERATOR_TOOLSET 变量。

版本 3.14 新增：增加了对 host=x86 选项的支持。
