CodeLite¶

自 3.27 版本起已弃用： 对 Extra Generators（扩展生成器） 的支持已被弃用，将在未来的 CMake 版本中移除。IDE 可以使用 cmake-file-api(7) 来查看 CMake 生成的项目构建树。

生成 CodeLite 项目文件。

CodeLite 的项目文件将在顶层目录以及每个包含 project() 调用的 CMakeLists.txt 文件的子目录中创建。 默认 all 目标可以通过适当的 make 程序构建项目。 此外还提供了一个 install 目标。

3.7 版本新增：CMAKE_CODELITE_USE_TARGETS 变量可以设置为 ON，以将默认行为从项目更改为目标，作为项目文件的基础。

此“额外”生成器可以指定为

CodeLite - MinGW Makefile（构建脚本）

使用 MinGW Makefile（构建脚本）s 生成。

CodeLite - NMake Makefile（构建脚本）

使用 NMake Makefile（构建脚本）s 生成。

CodeLite - Ninja（构建工具）

使用 Ninja（构建工具） 生成。

CodeLite - Unix Makefile（构建脚本）

使用 Unix Makefile（构建脚本）s 生成。
