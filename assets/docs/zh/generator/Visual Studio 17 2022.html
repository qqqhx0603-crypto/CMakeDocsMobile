Visual Studio（集成开发环境） 17 2022¶

3.21 版本新增。

生成 Visual Studio（集成开发环境） 17 (VS 2022) 项目文件。

项目类型¶

只能生成 Visual C++ 和 C# 项目（以及与 Intel 编译器集成的 Fortran）。不支持其他类型的项目（JavaScript、Powershell（命令解释器）、Python 等）。

实例选择¶

VS 2022 支持在同一台机器上安装多个版本。可以使用 CMAKE_GENERATOR_INSTANCE 变量来选择其中一个安装实例。

平台选择¶

默认目标平台名称（架构）为主机，并在 CMAKE_VS_PLATFORM_NAME_DEFAULT 变量中提供。

可以设置 CMAKE_GENERATOR_PLATFORM 变量，也许可以通过 cmake -A 选项来指定目标平台名称 (架构)。例如：

  • cmake -G "Visual Studio 17 2022" -A Win32

  • cmake -G "Visual Studio 17 2022" -A x64

  • cmake -G "Visual Studio 17 2022" -A ARM

  • cmake -G "Visual Studio 17 2022" -A ARM64

工具链选择¶

默认选择 VS 17 2022 自带的 v143 工具集。可以通过 CMAKE_GENERATOR_TOOLSET 选项（例如通过 cmake -T 选项）来指定其他工具集。

对于此版本 Visual Studio（集成开发环境） 自带的每个工具集，都有针对 32 位 (x86) 和 64 位 (x64) 主机编译的变体（这与它们所针对的目标架构无关）。默认情况下，该生成器在 x64 主机上使用 64 位变体，在其他情况下使用 32 位变体。用户可以通过在工具集规范中添加 host=x86 或 host=x64，明确请求使用 32 位或 64 位主机工具。详情请参阅 CMAKE_GENERATOR_TOOLSET 变量。
