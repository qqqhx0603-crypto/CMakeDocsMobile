Visual Studio（集成开发环境） 14 2015¶

4.2 版本起已弃用：此生成器已被弃用，并将在 CMake 的未来版本中移除。仍然可以使用 Visual Studio 15 2017（或更高版本）生成器并通过将 CMAKE_GENERATOR_TOOLSET 设置为 v140，或者使用 NMake Makefile（构建脚本）s 生成器来使用 VS 14 2015 工具进行构建。

版本 3.1 中新增。

生成 Visual Studio（集成开发环境） 14 (VS 2015) 项目文件。

项目类型¶

只能生成 Visual C++ 和 C# 项目（以及与 Intel 编译器集成的 Fortran）。不支持其他类型的项目（JavaScript、Powershell（命令解释器）、Python 等）。

平台选择¶

默认目标平台名称（架构）为 Win32。

可以设置 CMAKE_GENERATOR_PLATFORM 变量，也许可以通过 cmake -A 选项来指定目标平台名称 (架构)。例如：

  • cmake -G "Visual Studio 14 2015" -A Win32

  • cmake -G "Visual Studio 14 2015" -A x64

  • cmake -G "Visual Studio 14 2015" -A ARM

4.0 版本更改：此前，为了与 3.1 之前的 CMake 版本兼容，可以在生成器名称末尾可选地指定目标平台名称。此功能已被移除。这仅支持以下情况：

Visual Studio 14 2015 Win64

指定目标平台为 x64。

Visual Studio 14 2015 ARM

指定目标平台为 ARM。

工具链选择¶

默认选择 Visual Studio（集成开发环境） 14 2015 自带的 v140 工具集。可以通过 cmake -T 选项设置 CMAKE_GENERATOR_TOOLSET 选项，以指定其他工具集。

3.8 版本新增：对于此版本的 Visual Studio（集成开发环境） 自带的每个工具集，都有专门为 32 位 (x86) 和 64 位 (x64) 主机编译的变体（与它们的目标架构无关）。默认情况下，此生成器即使在 64 位主机上也会使用 32 位变体。用户可以通过在工具集规范中添加 host=x86 或 host=x64 来显式请求使用 32 位或 64 位主机工具。详情请参阅 CMAKE_GENERATOR_TOOLSET 变量。

3.14 版本新增：增加了对 host=x86 选项的支持。

VS 2015 的 Windows（微软桌面系统） 10 SDK 最高版本¶

3.19 版本新增。

微软在“Windows（微软桌面系统） 10 2018 年 10 月更新”博客文章中指出，Windows（微软桌面系统） 10 SDK 版本（15063、16299、17134、17763）不受 VS 2015 支持，仅由 VS 2017 及更高版本支持。因此，默认情况下，CMake 会自动忽略高于 10.0.14393.0 的 Windows（微软桌面系统） 10 SDK。

然而，针对某些驱动程序/Win32 构建还有其他建议，指出情况并非如此。用户可以通过将 CMAKE_VS_WINDOWS_TARGET_PLATFORM_VERSION_MAXIMUM 设置为假值，或者将该变量设置为所需最高版本的字符串值（例如 10.0.15063.0）来覆盖此行为。
